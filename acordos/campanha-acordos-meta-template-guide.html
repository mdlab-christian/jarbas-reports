Possíveis Motivos de Rejeição
Análise dos riscos e como mitigar
  • ⚠️
    RISCO ALTO — Categoria UTILITY vs MARKETING
    A Meta define UTILITY como mensagens que respondem a uma ação do usuário (confirmação de pedido, atualização de envio). Nossa mensagem é um contato outbound B2B (proposta de acordo não solicitada), o que se encaixa melhor em MARKETING. Se rejeitado por categoria errada, basta recriar o template com category: "MARKETING".

    Diferença de preço: UTILITY (~$0,008/msg) é mais barato que MARKETING (~$0,016/msg).
    Plano B: recriar como MARKETING se rejeitado
  • ⚠️
    RISCO MÉDIO — Linguagem de Solicitação/Negociação
    Frases como "verificar a possibilidade de composição amigável" e "permanecemos à disposição para construir uma solução consensual" podem ser interpretadas pela IA da Meta como texto de marketing/prospecção. Caso rejeitado por conteúdo, sugerimos reformular para linguagem mais formal e transacional.
  • ✅
    OK — Comprimento do Corpo
    ~420 caracteres estimados. Limite da Meta é 1.024 caracteres. Dentro do limite com folga.
  • ✅
    OK — Formatação Bold
    Uso de *{{1}}*, *{{2}}*, *{{3}}* (asteriscos para negrito) é suportado no corpo do template.
  • ✅
    OK — Exemplos no Campo example
    Fornecemos exemplos reais em body_text: [["João da Silva", "1234567-89.2023.8.21.0001", "Escritório Souza"]]. Isso é obrigatório para aprovação.
  • ✅
    OK — Não contém links, URLs ou números de telefone no body
    Conteúdo sem elementos que geralmente causam rejeição automática pela política da Meta.
  • ℹ️
    INFO — parameter_format não explicitado
    Não enviamos o campo parameter_format. O default da Meta é POSITIONAL, que é exatamente o que usamos ({{1}}, {{2}}, {{3}}). Não deve causar rejeição, mas é boa prática explicitar em futuras criações.

Como Verificar e Gerenciar na Meta
Passo a passo completo no WhatsApp Business Manager
1

Acessar o WhatsApp Business Manager

Acesse o gerenciador de templates diretamente:

🔗 business.facebook.com/wa/manage/message-templates/

Selecione a conta da empresa → aba Message Templates → filtre por acordo_proposta_v1.

2

Verificar Status via API (Alternativo)

Execute este comando para checar o status atual do template:

source ~/.env curl -X GET \ "https://graph.facebook.com/v19.0/$META_WABA_ID/message_templates?name=acordo_proposta_v1" \ -H "Authorization: Bearer $META_ACCESS_TOKEN" \ | jq '.data[0] | {id, name, status, category, rejected_reason}'

Resposta esperada quando APROVADO:

{ "id": "950518744009681", "name": "acordo_proposta_v1", "status": "APPROVED", "category": "UTILITY", "rejected_reason": null }
3

Se APROVADO ✅

O workflow n8n já está pronto. Siga direto para a Seção 6 — Disparar Campanha. Teste primeiro com 1 número real de WhatsApp antes de enviar em lote.

4

Se REJEITADO ❌

Verifique o rejected_reason na API ou no Business Manager. Ações possíveis:

Causa: categoria errada → Recriar com "category": "MARKETING" (ver Passo 5)
Causa: conteúdo promocional → Reformular o texto para ser mais formal/transacional
Causa: violação de política → Ler as diretrizes oficiais de templates
5

Recriar Template (se necessário) — Versão MARKETING com parameter_format explícito

Delete o template atual e recrie com as correções:

Deletar template atual
source ~/.env # Deletar pelo nome curl -X DELETE \ "https://graph.facebook.com/v19.0/$META_WABA_ID/message_templates?name=acordo_proposta_v1" \ -H "Authorization: Bearer $META_ACCESS_TOKEN"
Recriar como MARKETING (plano B)
curl -X POST \ "https://graph.facebook.com/v19.0/$META_WABA_ID/message_templates" \ -H "Authorization: Bearer $META_ACCESS_TOKEN" \ -H "Content-Type: application/json" \ -d '{ "name": "acordo_proposta_v1", "language": "pt_BR", "category": "MARKETING", "parameter_format": "POSITIONAL", "components": [{ "type": "BODY", "text": "Olá! Representamos *{{1}}* nos autos do processo nº *{{2}}* e identificamos que o escritório *{{3}}* atua pela parte ré.\n\nEntramos em contato para verificar possibilidade de composição amigável.\n\nAtt,\nDr. Christian Dener Paz — OAB/SP 535.101", "example": { "body_text": [["João da Silva", "1234567-89.2025.8.26.0001", "Escritório Souza e Associados"]] } }] }' | jq
Nota: Se o nome acordo_proposta_v1 já foi rejeitado, use um nome diferente como acordo_proposta_v2. Templates rejeitados ficam bloqueados por 30 dias no mesmo nome.
