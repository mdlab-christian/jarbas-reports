🏛️ Contexto — O que é o Advbox

Sistema Legado (Advbox)

Sistema de gestão jurídica que o escritório MP Advocacia (Perinotti Paz Advogados) usa hoje. Concentra tarefas, intimações, prazos e acesso a tribunais (EPROC RS/SC, JusBrasil SP).

  • Lista centralizada de intimações judiciais
  • Criação de tarefas vinculadas a processos
  • Fluxo manual: Advbox → e-proc → Advbox
  • ~116 intimações ativas por dia (SP)
  • Taskscore: métrica de complexidade/pontuação
  • ~8.879 processos cadastrados atualmente

Novo Sistema (MdFlow)

SaaS jurídico em construção pela MdLab que substituirá o Advbox. Já está sendo distribuído em produção — mas os dois sistemas precisam estar 100% sincronizados durante a transição.

  • Stack: Supabase + React + TypeScript + n8n
  • 5.800 processos migrados, meta: 12.000 monitorados
  • Coluna advbox_id na tabela processos
  • Tabelas de staging já criadas: advbox_processos_raw
  • Skill de validação já implementada

⚠️ O Problema Atual

Como o MdFlow ainda não substituiu o Advbox mas já está sendo usado em paralelo, processos distribuídos no MdFlow não estão sendo automaticamente cadastrados no Advbox. Resultado: os dois sistemas estão fora de sincronia. Advogados que trabalham no Advbox não veem os processos novos. O MdFlow já tem os dados de empresa ré, mas eles não estão sendo enviados para o Advbox — o fluxo MdFlow → Advbox está quebrado.

Exemplo de chamada completa

// Padrão de chamada com rate limit
let _last = 0;
async function advboxGet(ep) {
  const elapsed = Date.now() - _last;
  if (elapsed < 2200) await new Promise(r => setTimeout(r, 2200 - elapsed));
  _last = Date.now();
  const r = await fetch('https://app.advbox.com.br/api/v1' + ep, {
    headers: {
      'Authorization': `Bearer ${process.env.ADVBOX_API_TOKEN}`,
      'Accept': 'application/json',
      'X-Requested-With': 'XMLHttpRequest',
      'User-Agent': 'Mozilla/5.0',
    }
  });
  if (r.status === 429) {
    console.warn('Rate limit — aguardando 65s');
    await new Promise(r => setTimeout(r, 65000));
    return advboxGet(ep); // retry
  }
  const t = await r.text();
  try { return { status: r.status, data: JSON.parse(t) }; }
  catch { return { status: r.status, data: t.slice(0, 300) }; }
}

// Buscar processo por CNJ
const resultado = await advboxGet('/lawsuits?process_number=5048484-53.2020.8.21.7000');
const advboxId = resultado.data?.data?.[0]?.id; // salvar no Supabase

Query padrão — buscar processos sem advbox_id (para pareamento)

SELECT id, numero_processo, created_at, organizacao_id
FROM processos
WHERE advbox_id IS NULL
  AND numero_processo IS NOT NULL
  AND created_at >= '2026-03-10'
ORDER BY created_at DESC;
-- Retorna os 583 processos que precisam de pareamento

Query padrão — atualizar advbox_id após pareamento

UPDATE processos
SET advbox_id = {advbox_id},
    advbox_sincronizado_em = NOW()
WHERE id = '{processo_uuid}';

Query padrão — buscar empresa ré faltando (a ser preenchida via Advbox)

SELECT p.id, p.numero_processo, p.advbox_id, p.empresa_re
FROM processos p
WHERE p.advbox_id IS NOT NULL
  AND (p.empresa_re IS NULL OR p.empresa_re = '')
LIMIT 500;
