🔄 Movimentações Processuais

Endpoint: GET /movements/{lawsuit_id}

  • Extrair todas as movimentações TRIBUNAL e MANUAL
  • Salvar em tabela movimentacoes ou no campo detalhes jsonb
  • Verificar se já existem movimentações do EPROC — evitar duplicar
  • Dedup por: data + tipo + texto (hash SHA1)
  • Endpoint /last_movements dá visão geral sem iterar processo a processo
🗞️ Publicações (Diário da Justiça)

Endpoint: GET /publications

  • Intimações capturadas automaticamente pela Advbox
  • Pode complementar o que o EPROC já captura
  • Salvar em tabela publicacoes ou intimacoes
  • Dedup por número do processo + data + texto
  • P2 — fazer após processos e clientes
📋 Histórico de Tarefas por Processo

Endpoint: GET /history/{lawsuit_id}

  • Retorna todas as tarefas (pending + completed)
  • Valor histórico — registros de o que foi feito em cada processo
  • Salvar em detalhes.tarefas_advbox ou tabela separada
  • Atenção: não suporta paginação — retorna tudo de uma vez
  • P2 — baixa prioridade
💰 Transações Financeiras

Endpoint: GET /transactions

  • Honorários recebidos, custas, despesas
  • Pode enriquecer relatórios financeiros históricos
  • Mapear para tabela financeira do MdFlow
  • P2 — fazer por último
  • Pode ser dispensável se financeiro já está no MdFlow
📂 4. Sobre Documentos (Cloud Advbox)
⚠️ Situação atual: não há endpoint documentado para download de arquivos Os 19 endpoints documentados na API pública não incluem nenhum endpoint de arquivo/documento. O campo folder nos processos é um campo de texto livre (ex: "Pasta Maria Silva 2023"), não um path de arquivo.
🔍 O que investigar ao receber o token
  • Testar GET /files, GET /documents, GET /cloud — endpoints não documentados que podem existir
  • Verificar se o campo folder de cada processo tem um ID numérico — pode ser a chave para buscar arquivos
  • Checar se a Advbox tem API de Cloud/Storage separada (pode ser S3 com URL pré-assinada)
  • Contatar suporte Advbox perguntando sobre endpoint de download de documentos
  • Verificar package npm n8n-nodes-advbox — pode ter mais endpoints implementados lá
🟢 Plano B: se não tiver API de documentos
  • Baixar documentos manualmente via interface web antes de cancelar
  • Usar Playwright para automatizar o download da interface web (bulk download)
  • Priorizar documentos dos processos ativos — históricos podem ser dispensáveis
  • Os documentos críticos (petições, contestações) já devem estar no EPROC via download de peças
🔗 5. Estratégia de Deduplicação
👤 Clientes — Chave de matching
Ordem de prioridade:
1º CPF/CNPJ→Match exato — chave confiável
2º Email→Match secundário se CPF vazio
3º Nome + telefone→Fuzzy match — revisar manual
-- Query de matching
SELECT id FROM clientes
WHERE cpf_cnpj = $advbox_cpf
  AND organizacao_id = $org_id
LIMIT 1;
⚖️ Processos — Chave de matching
Ordem de prioridade:
1º Número CNJ→Chave universal — padrão NNNNNNN-DD.AAAA.J.TT.OOOO
2º Protocol number→Se CNJ não bater
-- Query de matching
SELECT id FROM processos
WHERE numero_cnj = $advbox_process_number
   OR numero_processo = $advbox_process_number
  AND organizacao_id = $org_id
LIMIT 1;

FASE 1
Extração Completa de Clientes
⏱️ ~2-4 horas (dependendo do volume) · Paralelo ao EPROC
  • Pull completo de GET /customers com paginação (limit=100, offset incremental)
  • Salvar dump raw em ~/jarbas/advbox-data/customers-raw.json
  • Para cada cliente: match por CPF/CNPJ no Supabase
  • Enriquecer clientes existentes (telefone, cidade, CEP, profissão, nascimento)
  • Logar não-encontrados em ~/jarbas/advbox-data/customers-unmatched.json
  • Relatório final: X enriquecidos / Y não encontrados / Z já completos
Campos prioritários: telefone · whatsapp · data_nascimento · cidade · cep · profissao · email
FASE 2
Extração Completa de Processos
⏱️ ~4-8 horas · ~5.800 processos para processar
  • Pull completo de GET /lawsuits com paginação
  • Para cada processo: match pelo número CNJ no MdFlow
  • Enriquecer processos.detalhes com dados da Advbox (fase, tipo, grupo, responsável)
  • Salvar advbox_id no jsonb para poder buscar movimentações depois
  • Identificar processos na Advbox que NÃO estão no MdFlow (possíveis lacunas)
  • Logar processos com campos financeiros preenchidos (honorários esperados)
⚠️ Rate limit: 30 GET/min = 1.800/hora. Com 5.800 processos em bulk = ~3 minutos só para listar. Movimentações por processo = adicionar 1-2 seg de delay entre chamadas.
FASE 3
Extração de Movimentações por Processo
⏱️ ~6-12 horas · Maior volume — requer delay estratégico
  • Para cada processo que tem advbox_id salvo: buscar GET /movements/{id}
  • Extrair movimentações TRIBUNAL e MANUAL
  • Deduplicar contra movimentações já existentes (do EPROC) por data + texto
  • Salvar movimentações novas na tabela movimentacoes ou detalhes
  • Alternativa mais eficiente: usar GET /last_movements com paginação (sem iterar por processo)
  • Processar em lotes de 100 processos com rate limiter de 2s entre requests
FASE 4
Publicações (Diário da Justiça)
⏱️ ~1-2 horas · Baixa prioridade
  • Pull de GET /publications com filtros de data
  • Comparar com intimações já cadastradas no MdFlow
  • Importar apenas as que não existem ainda
  • Pode identificar intimações perdidas que o EPROC não capturou
FASE 5
Financeiro + Histórico de Tarefas
⏱️ ~2-4 horas · Opcional — avaliar ROI
  • Extrair transações financeiras históricas (GET /transactions)
  • Importar para relatórios financeiros do MdFlow
  • Histórico de tarefas por processo (GET /history/{id}) — valor histórico/auditoria
  • Avaliar se vale o esforço antes de executar

🎛️ Configuração do Rate Limiter
// rate-limiter.mjs
const REQUESTS_PER_MINUTE = 25; // abaixo do limite de 30
const DELAY_MS = Math.ceil(60000 / REQUESTS_PER_MINUTE);

async function advboxGet(endpoint) {
  await sleep(DELAY_MS); // 2.4 segundos
  const res = await fetch(
    `https://app.advbox.com.br/api/v1${endpoint}`,
    { headers: { Authorization: `Bearer ${ADVBOX_TOKEN}` } }
  );
  if (res.status === 429) {
    await sleep(60000); // retry após 1 minuto
    return advboxGet(endpoint);
  }
  return res.json();
}

🏁 9. Checklist para Cancelar o Plano Advbox
✅ Condições para cancelamento seguro
  • Clientes extraídos: todos os clientes com CPF/CNPJ enriquecidos no MdFlow
  • Processos mapeados: todos os processos com advbox_id salvo + dados enriquecidos
  • Movimentações extraídas: histórico completo salvo no MdFlow ou arquivo
  • Publicações exportadas: intimações históricas preservadas
  • Documentos resolvidos: API de docs investigada + download manual feito se necessário
  • Financeiro exportado: transações históricas salvas (se relevante)
  • Backup completo: dump JSON de tudo em ~/jarbas/advbox-data/ + git push
  • Validação: spot check de 50 processos aleatórios — dados baterram entre sistemas
