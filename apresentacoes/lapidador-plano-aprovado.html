00 · Respostas diretas

Você perguntou. Resposta direta.

🤖 JARBAS vai usar Claude Code como?

Livremente, em qualquer fase. Não só na execução. Na investigação (análise de código), na geração do PRD, na limpeza, no redesign. Sempre que precisar editar um arquivo, rodar um comando, testar algo — Claude Code entra. Auto-approve ativo. JARBAS dirige, Claude Code executa.

Claude Web: JARBAS não consegue acessar suas conversas diretamente (é interface de browser). O fluxo é: você passa o contexto/handoff → JARBAS usa. Já planejado assim — você disse que ia passar de qualquer forma. ✅

🎯 Vai dar certo?

Sim. Você disse que o projeto já está muito bem lapidado. Isso muda tudo: não é uma reconstrução, é um polimento. Para cada página, a maior parte do trabalho é testar e corrigir detalhes, não reconstruir do zero.

O que garanto: funcionamento real (cliques, modais, n8n), código limpo, design consistente DS v5, 0 erros TS, tudo testado ponta a ponta.

Honestidade: páginas com n8n complexo (olivia, controller, financeiro) podem surpreender. Margin de segurança no tempo pessimista cobre isso.

03 · Micro-detalhes

O diabo mora nos detalhes.

JARBAS revisa cada interação, não só o funcionamento macro. O exemplo do tooltip escuro em dark mode é exatamente o nível de atenção que vai ser dado.

Interações & Hover
🖱️
Tooltips — fundo deve ser rgba() semi-transparente, texto legível. Nunca preto sólido em dark mode. Font: JetBrains Mono ou Inter sm
📊
Gráficos — hover deve mostrar valores com contraste adequado. Recharts/Tremor: customizar TooltipContent com bg var(--card)
🔘
Botões — hover state visual claro (não só cursor pointer). Disabled state distinto. Loading spinner correto
🏷️
Badges/Tags — cores semânticas consistentes (verde=ativo, vermelho=erro, amarelo=pendente)
📝
Inputs — focus ring visível, placeholder contrastado, error state com border-red + mensagem
Comportamento de Página
🔄
Refresh pós-ação — após criar/editar/deletar, a listagem atualiza automaticamente (invalidateQueries)
🔍
Filtros — aplicar filtra a lista. Limpar reseta para o estado original. URL params persistem no reload?
🔎
Search box — busca em tempo real (debounce 300ms), sem loading excessivo
📋
KPIs/Cards — valores numéricos formatados (R$, número de processos), sem NaN/undefined visível
📑
Abas — estado ativo visível, conteúdo correto por aba, URL sincroniza com aba ativa
📱
Scroll — sem overflow horizontal indesejado, tabelas com scroll interno
💡
Regra do micro-detalhe: se JARBAS encontrar algo que parece "aceitável mas poderia ser melhor" — ele registra no HTML #1 como sugestão de melhoria. Christian decide se entra no PRD ou fica para depois. Nada é ignorado, mas nada é implementado sem aprovação.

06 · JARBAS + Claude Code + Claude Web

Como os três trabalham juntos.

🦞 JARBAS
O arquiteto.
Investiga, planeja, orquestra, valida ponta a ponta, pergunta quando duvida, entrega o HTML. Conversa com você pelo Telegram. Tem memória persistente, contexto do sistema inteiro, acesso direto ao Supabase/n8n/Git. Opera 24/7.
⚡ Claude Code
O executor.
Edita arquivos, refatora código, aplica C-prompts, usa 21.st Magic, roda testes, faz commits. JARBAS dispara quando necessário — em qualquer fase, não só na execução. Auto-approve ativo. JARBAS lê o log e valida o resultado.
🌐 Claude Web
O consultor contextualizado.
JARBAS não acessa diretamente (é interface de browser). O fluxo: você passa o handoff/contexto da sua conversa Claude Web → JARBAS usa como input. Já planejado. Você é a ponte. Nada muda no plano.
🔄 Pair programming na prática
Fluxo real:
1. Você passa o nome/conteúdo da conversa Claude Web para o JARBAS sobre a página
2. JARBAS usa como contexto primário da investigação (junto com o spec e seu áudio)
3. JARBAS dispara Claude Code para análise de código e correções
4. JARBAS valida o resultado do Claude Code (0 TS errors, testa no browser, verifica n8n)
5. Resultado vai para você via Telegram com HTML

Resumo: Claude Web dá contexto de produto. JARBAS dá contexto técnico/operacional. Claude Code executa. Você aprova.
