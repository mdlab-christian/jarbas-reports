Webhook Boa Vista — Gap no Multi-Add Manual

Análise completa do fluxo de criação de pasta Boa Vista no CRM — onde funciona, onde falha e o que corrigir.

MdFlow CRM Gap Identificado sync-storage-drive ModalAdicionarRestricoes 2026-03-27
📋

Sumário Executivo

🎯 O que foi pedido

Verificar se o webhook de criação de pasta Boa Vista (que já roda automaticamente no fluxo de auditorias via bureau) pode ser reaproveitado para o fluxo de cadastro manual de lead — onde o usuário adiciona as restrições na mão, com o órgão restritivo "SCPC/Boa Vista".

Regra de negócio: todas as restrições de um lead que tiverem orgao_nome = 'SCPC/Boa Vista' devem criar uma pasta Boa Vista e somar os valores juntos.

✅
Fluxo auditoria (bureau) já funciona
✅
FormRestricao (1 por vez) já funciona
❌
ModalAdicionarRestricoes (multi-add) NÃO tem webhook
🔍

Contexto da Análise

A análise partiu de dois cenários de criação de restrições num lead:

🤖 Via Auditoria (Bureau)
  • Usuário aciona auditoria no CRM
  • Chama rpc_disparar_auditoria no Supabase
  • n8n recebe, consulta bureau, preenche restrições
  • Pasta Boa Vista criada automaticamente pelo n8n no backend
  • Funciona
✋ Manual (pelo usuário)
  • Usuário abre drawer do lead → aba Restrições
  • Clica "Adicionar" → ModalAdicionarRestricoes
  • Preenche empresa, valor, data, órgão = Boa Vista
  • Insert direto no Supabase sem chamar nenhum webhook
  • Pasta Boa Vista NÃO é criada
📄 Também mapeado: FormRestricao (1 restrição por vez)

O FormRestricao.tsx (modal de edição/adição de UMA restrição) JÁ dispara o webhook quando o órgão é "SCPC/Boa Vista". Mas o ModalAdicionarRestricoes (que permite adicionar N restrições de uma vez) não tem essa lógica.

Webhooks Mapeados

ATIVO POST /webhook/mdflow/sync-storage-drive — Cria/sincroniza pastas no Drive
Usado por FormRestricao, RestricaoPastaCard, DocumentosLeadPaineis, ClienteDrawerTabDocumentosLead. Aceita action: "criar_pasta_restricao" com campos: organizacao_id, lead_id, cliente_id, orgao_nome.
ATIVO rpc_disparar_auditoria (Supabase RPC) — Dispara consulta bureau
Aceita p_bureaus: ['apifull_115'] ou ['serasa', 'boa_vista', ...]. Retorna auditoria_id. O n8n processa no backend e cria a pasta BV automaticamente.
TIPADO / NÃO USADO rpc_cadastrar_lead_boa_vista (Supabase RPC) — Cadastra lead via auditoria BV
Existe nos tipos do Supabase (p_auditoria_id, p_organizacao_id, p_user_id), mas não é chamado em nenhum lugar do frontend. Pode ser rota alternativa no backend para criar lead a partir de auditoria Boa Vista.
⚠️

Gap Identificado

❌ ModalAdicionarRestricoes — sem chamada de webhook

O modal de multi-adição de restrições (ModalAdicionarRestricoes.tsx) faz supabase.from('restricoes').insert(registros) direto no banco e encerra. Não verifica se alguma das restrições tem orgao_nome = 'SCPC/Boa Vista' e não dispara o webhook de criação de pasta.

Isso significa que quando o usuário cadastra um lead manual e adiciona N restrições Boa Vista de uma vez, a pasta no Drive não é criada e o valor consolidado não é somado.

Fluxo Componente Webhook Boa Vista Status
Auditoria via bureau useDispararAuditoria n8n cria no backend OK
Editar/add 1 restrição FormRestricao.tsx sync-storage-drive (fire-and-forget) OK
Multi-add manual ModalAdicionarRestricoes.tsx ❌ Nenhum GAP
Import CSV ClienteDrawerTabRestricoes handleImport ❌ Nenhum Não verificado

RPC Supabase — rpc_cadastrar_lead_boa_vista

📌 O que é e por que existe

A RPC rpc_cadastrar_lead_boa_vista está tipada no supabase.ts mas não é chamada em nenhum componente do frontend. Pela assinatura (p_auditoria_id, p_organizacao_id, p_user_id), parece ser uma rota para criar/vincular um lead a partir de uma auditoria Boa Vista já existente — provavelmente chamada pelo próprio n8n internamente após processar o bureau.

// supabase.ts — tipos existentes
rpc_cadastrar_lead_boa_vista:
| {
Args: {
p_auditoria_id: string
p_organizacao_id: string
p_user_id: string
}
Returns: Json
}
| {
Args: { p_auditoria_id: string; p_user_id?: string }
Returns: Json
}
🔧

A Correção

✅ Solução: adicionar chamada webhook após insert no ModalAdicionarRestricoes

Após o supabase.from('restricoes').insert(registros), verificar se alguma das restrições tem orgao_nome === 'SCPC/Boa Vista' (ou qualquer variação). Se sim, disparar UMA única chamada ao webhook sync-storage-drive com action: 'criar_pasta_restricao'. O n8n já sabe consolidar/somar todos os valores Boa Vista do lead.

⚙️ Detalhes da implementação
  • Onde: ModalAdicionarRestricoes.tsx → função handleSalvar, após o if (error) throw error
  • Trigger: registros.some(r => r.orgao_nome === 'SCPC/Boa Vista')
  • Uma chamada só: não precisa disparar N vezes. O webhook consolida todos os valores Boa Vista do lead de uma vez.
  • Fire-and-forget: mesmo padrão do FormRestricao — .catch(() => {}) silencioso.
  • Também verificar: import CSV em ClienteDrawerTabRestricoes → handleImport tem o mesmo gap.
💻

Código da Fix

Adicionar no ModalAdicionarRestricoes.tsx, após o if (error) throw error no handleSalvar:

// Após: const { error } = await supabase.from('restricoes').insert(registros)
// Após: if (error) throw error

// Disparar webhook Boa Vista se houver ao menos 1 restrição com SCPC/Boa Vista
const temBoaVista = registros.some(r => r.orgao_nome === 'SCPC/Boa Vista')
if (temBoaVista) {
const entityId = leadId ?? clienteId
if (entityId) {
const webhookUrl = `${import.meta.env.VITE_N8N_BASE_URL ?? 'https://primary-production-e209.up.railway.app'}/webhook/mdflow/sync-storage-drive`
fetch(webhookUrl, {
method: 'POST',
headers: { 'Content-Type': 'application/json', mdflow: 'MdL1501@' },
body: JSON.stringify({
organizacao_id: organizacaoId,
lead_id: leadId ?? null,
cliente_id: clienteId ?? null,
orgao_nome: 'SCPC/Boa Vista',
action: 'criar_pasta_restricao',
}),
}).then(() => queryClient.invalidateQueries({ queryKey: ['drive-pastas', entityId] }))
.catch(() => { /* silencioso — pasta criada no próximo sync */ })
}
}

Opcionalmente, usando o wrapper callWebhook que já existe no projeto:

// Adicionar import no topo do arquivo:
import { callWebhook } from '@/lib/webhook'

// No handleSalvar, após o insert:
const temBoaVista = registros.some(r => r.orgao_nome === 'SCPC/Boa Vista')
if (temBoaVista && (leadId ?? clienteId)) {
callWebhook('sync-storage-drive', {
organizacao_id: organizacaoId,
lead_id: leadId ?? null,
cliente_id: clienteId ?? null,
orgao_nome: 'SCPC/Boa Vista',
action: 'criar_pasta_restricao',
}).then(() => queryClient.invalidateQueries({ queryKey: ['drive-pastas', leadId ?? clienteId] }))
.catch(() => {})
}
📌 Também verificar — Import CSV

A função handleImport em ClienteDrawerTabRestricoes.tsx faz insert de linhas do CSV diretamente no Supabase sem verificar Boa Vista. A mesma lógica deve ser aplicada lá, após o insert bem-sucedido, verificando se alguma linha tem orgao_nome contendo "Boa Vista".
