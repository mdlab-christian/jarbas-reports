📌 O que foi feito

O script fix-advbox-processos.mjs leu os processos do Supabase, e para cada processo com advbox_id registrado, acessou a API do AdvBox para:

  • Adicionar a parte autora (cliente do processo)
  • Adicionar a parte contrária (empresa/réu) — ex: KINEI CALÇADOS LTDA
  • Verificar se as partes já existiam antes de tentar inserir

Erros — Rate Limit (27 processos)

📋 Causa única: HTTP 429 — AdvBox rate limit

Todos os 27 erros foram causados por 429 Too Many Requests da API AdvBox após 3 tentativas com retry. O script tinha delay de 1.5s entre requisições, mas em momentos de pico a API throttleou.

// Exemplo de erro: "GET /lawsuits/1798907 → 429: rate limit após 3 tentativas" // IDs afetados (sample): 1798907, 1798542, 1799791, 1798562, 1798850, 1798528, 3243813, 3243821, 3252400, 3290747, 3290748, 3303611, 3303617, 3303693, 3303722, 3303716, 3303831, 3303879... // + 9 outros
Solução recomendada

Rodar novamente o script com --limit=50 e DELAY_MS=3000 (3s) para reprocessar apenas os 27 falhados. Ou aguardar horário de menor carga na API AdvBox.

Processos Sem Empresa (76)

📋 O que significa

76 processos no Supabase não têm empresa vinculada (campo empresa_id ou parte_contraria nulo). Nesses casos, o script adiciona a parte autora normalmente mas não consegue inserir a parte contrária por falta de dados.
