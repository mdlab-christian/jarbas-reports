👥 CRM
Rota: /crm — Arquivo: src/pages/crm/CrmPage.tsx
⚡ CORE ✓ Realtime Multi-view Deep-link URLs
Missão da Página
Porta de entrada dos leads. Gerencia todo o ciclo comercial: do primeiro contato (via MdZap) até a conversão jurídica. Pipeline Kanban ou lista de leads por etapas. Cada lead passa por: Novo → Triagem → Negociação → Documentação → Pronto para Distribuir → Arquivado.

Funções principais: criar leads, triagem, vincular documentos, consultar auditorias de crédito, vincular advogado, distribuir (converte em processo).

⚖ Distribuição
Rota: /distribuicao — Arquivo: src/pages/distribuicao/DistribuicaoPage.tsx
⚡ CORE ✓ Realtime ⚡ Ponto de virada do sistema
Missão da Página
O coração da conversão. É aqui que um lead/restrição deixa de ser comercial e se torna um processo jurídico. Quando o usuário efetua a distribuição:

1. O lead no CRM é arquivado automaticamente
2. O lead é convertido em cliente (se ainda não for)
3. A restrição vira um processo na página Processos
4. O processo some da UI de Distribuição
5. O cliente é atualizado com o novo processo vinculado

Gerencia também petições iniciais via OlivIA e agendamento de distribuições futuras.
Ligações

👥 CRM (lead com restrições prontas)
→ GERA / ENVIA PARA

🔔 Controller
Rota: /controller — Arquivo: src/pages/controller/
⚡ CORE Alta complexidade ✓ Realtime Integração EPROC
Missão da Página
Central de comando jurídico. Monitora todas as intimações e movimentações processuais vindas do EPROC (sistema externo). Para cada intimação recebida, o Controller decide automaticamente o que fazer:

→ Cria uma Atividade para o advogado responsável, OU
→ Cria um Documento Pendente que vai para a fila de docs pendentes

Controla prazos, permite anotações, monitoramento por equipe, arquivamento, exportação e treinamento do modelo de IA para classificação de intimações.

→ GERA EM
✅ Atividades (tarefa para advogado) 📎 Docs Pendentes (documento a criar) 🤖 OlivIA (draft de petição)

Vínculo com Usuários (Crítico)
⚡ Lógica de visibilidade por usuário
Quando o Controller atribui uma Atividade a um usuário → a atividade aparece somente para esse usuário em /atividades.

Quando o Controller cria um Documento Pendente → aparece para todos com acesso à página /documentos-pendentes.

Isso é fundamental para o correto funcionamento dos filtros de equipe.

Padrão de Input de Processo (SearchBoxCNJ)
⚡ Regra crítica — Inputs de Número de Processo
Todo input que solicite número de processo (CNJ) deve usar SearchBoxCNJ — um input com dropdown que:
1. Busca processos conforme o usuário digita
2. Exibe: número CNJ + nome do cliente no dropdown
3. Ao selecionar: preenche automaticamente cliente, empresa, advogado e restrições relacionadas

Exceção: O input de "efetuar distribuição" — que cria o processo em si — usa campo texto simples pois o processo ainda não existe.
Padrão de Realtime (Supabase)
Todas as páginas core usam supabase.channel() com postgres_changes filtrado por organizacao_id.
Ao receber evento → queryClient.invalidateQueries() para atualizar React Query.
Canais são removidos no cleanup (supabase.removeChannel(channel)).
