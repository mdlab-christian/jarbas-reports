Bugs Críticos

4 issues — impacto direto na UX
B1 Drag-and-drop do Kanban NÃO funciona corretamente ✅ Corrigido
📄 CrmKanban.tsx › DraggableLeadCard

dragStartedRef.current é NUNCA setado para true. Após um drag bem-sucedido, o onClick sempre dispara e abre o painel do lead, pois a guarda if (dragStartedRef.current) nunca passa. O resultado visual é que o DnD "parece não funcionar" pois o painel cobre o kanban.

✅ Fix aplicado
// Adicionar useEffect para setar o ref quando isDragging muda useEffect(() => { if (isDragging) { dragStartedRef.current = true; } }, [isDragging]);
Crítico @dnd-kit/core Fase 1
B2 Modal Distribuição exibe dados do lead ERRADO ✅ Corrigido
📄 ModalDistribuicao.tsx › useQuery

staleTime: 60 * 1000 — os dados ficam em cache por 60 segundos. Se o modal é aberto para o Lead A, fechado e aberto para o Lead B, o React Query pode retornar dados em cache antes do refetch. Na primeira renderização, os dados antigos aparecem brevemente.

✅ Fix aplicado
// Antes staleTime: 60 * 1000, // ← BUG: 60 segundos de cache // Depois staleTime: 0, // ← sempre refetch quando modal abre
Crítico React Query Fase 1
B3a Serasa Premium habilitada mas não deveria estar ✅ Corrigido
📄 pages/auditorias/types.ts › BUREAUS[]

disponivel: true para Serasa Premium. O botão de consulta Serasa fica ativo no painel, mas a integração não está implementada. Usuários podem tentar usá-la e se frustrar.

✅ Fix aplicado
{ id: 'serasa', nome: 'Serasa Premium', custo_centavos: 600, disponivel: false }, // Exibe "Em breve" e desabilita o botão
Crítico auditorias Fase 1
B3b CPF Completo (ApiFull) sem resposta ⏳ Pendente
📄 CrmAuditoriaPanel.tsx + supabase/functions/

O RPC rpc_disparar_auditoria cria o registro em auditorias mas não há Edge Function visível que faça a chamada HTTP para a API ApiFull. As funções Supabase existentes são: audio-transcribe, encrypt-credentials, invite-user, olivia-chat, olivia-smart-blocks.

Possíveis causas
  • Token da ApiFull não configurado nas secrets do Supabase
  • Edge Function de processamento ausente ou não deployed
  • Webhook/trigger do banco não está configurado
  • URL da API ApiFull incorreta ou serviço fora do ar
Crítico Edge Function Não corrigido

Problemas Funcionais

4 issues — funcionalidade incorreta
P1 Botão "Triagem" no drawer não move o lead ✅ Corrigido
📄 ClienteDrawerHeader.tsx → CrmPage.tsx → ModalTriagem.tsx

O botão "Triagem" aciona setTriagemLeadId que abre o ModalTriagem. Porém o modal valida que o lead já deve estar na coluna "triagem" — e fecha imediatamente com erro toast. O comportamento correto seria abrir o ModalDistribuicao (checklist de preparação).

✅ Fix aplicado
// CrmPage.tsx — antes case 'triagem': setTriagemLeadId(leadId); break; // Depois case 'triagem': setChecklistLeadId(leadId); break;
Alta Fase 1
P2 Toggle "Aguardando resposta" ativo por padrão ✅ Corrigido
📄 hooks/crm/useLeadFilters.ts

Filtros são persistidos no localStorage. Se o usuário ativou awaiting_human: true em sessão anterior, o spread {'{ ...DEFAULT_FILTERS, ...JSON.parse(raw) }'} mantém o valor true na próxima sessão. Usuário vê o kanban filtrado sem saber o motivo.

✅ Fix aplicado
// Forçar reset do awaiting_human ao carregar do localStorage return { ...DEFAULT_FILTERS, ...JSON.parse(raw), awaiting_human: null };
Alta localStorage Fase 1
P3 WhatsApp salvo sem máscara de formatação ✅ Corrigido
📄 modals/editar-lead/FormContato.tsx

O campo WhatsApp usa applyPhoneMask apenas no onChange, não no valor inicial. Ao abrir o modal de edição, o campo mostra 51999990000 sem máscara.

✅ Fix aplicado
// No reset do form ao carregar dados form.reset({ ...leadData, whatsapp: leadData.whatsapp ? applyPhoneMask(leadData.whatsapp) : '', });
Alta react-hook-form Fase 2
P5 Distribuição — sem botão "Cadastrar novo advogado" ✅ Corrigido
📄 modals/ModalTriagem.tsx

O select de advogados não oferece ação de cadastro. Quando a lista está vazia ou o advogado não existe, o usuário fica preso sem saber como proceder.

✅ Fix aplicado
// Adicionar após o <Select> <Button variant="link" size="sm" className="text-xs h-auto p-0 mt-1" onClick={() => navigate('/configuracoes?tab=advogados')}> + Cadastrar novo advogado </Button>
