🚨 Achados Críticos

❌ F3/F4 — Campo Email Ausente no Form de Edição de Dados
O formulário de edição na aba Dados do drawer não inclui o campo Email. O form edita: CPF, Data de Nascimento, Profissão, Origem do Lead, Endereço completo, CEP. Email só pode ser definido na criação, mas não há campo visível para isso também. Leads ficam sem email editável.
⚠️ D1/D2/D3 — Dropdowns de Filtro com Problemas de Automação (Radix Select)
Os dropdowns de atendente, origem e estado usam Radix UI Select, que não responde a clicks sintéticos via JavaScript. Não foi possível testar a abertura dos dropdowns automaticamente. Funcionam manualmente conforme evidência visual. Recomenda-se verificação manual.
⚠️ G1 — Drag-and-Drop Kanban Não Testado Automaticamente
O DnD do Kanban não foi testado via automação (requer simulação de eventos de ponteiro complexos). A mudança de coluna foi validada via API direta + reflexo no frontend. Testar drag manualmente para garantir funcionamento.
⚠️ D4 — Toggle Aguardando Ativa por Padrão no Kanban
Ao abrir o Kanban, o toggle "Aguardando resposta" estava ativo por padrão, mostrando kanban vazio. Isso pode confundir usuários que esperam ver todos os leads ao entrar no Pipeline.
ℹ️ A4 — 13 Erros de Acessibilidade Radix (DialogContent sem DialogTitle)
Erro recorrente em todos os modais/dialogs da aplicação. Não impede funcionalidade mas prejudica acessibilidade para leitores de tela (WCAG 2.1 - 4.1.2). Ocorre no modal de Novo Lead, Novo Auditoria e outros dialogs.
ℹ️ F3 — Validação de CPF Bloqueia CPF Fictício 000.000.000-00
O sistema corretamente valida CPFs matematicamente. CPF "000.000.000-00" é rejeitado (todos dígitos iguais são inválidos pelo algoritmo da Receita Federal). Comportamento CORRETO — apenas documentado aqui pois o checklist especificava usar esse CPF.
