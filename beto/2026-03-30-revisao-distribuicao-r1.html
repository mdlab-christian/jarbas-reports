Erros Críticos — P0

🔴
P0-1 · ModalEfetuarDistribuicao — RPC recebe UF string mas espera UUID
ModalEfetuarDistribuicao.tsx : linha 295
O campo p_estado_id do RPC deveria receber um UUID da tabela estados, mas o código passa a string UF (ex: "RS"). Distribuição falha silenciosamente ou insere valor inválido no banco.
// BUG: passa UF string ("RS") onde RPC espera UUID
const estadoUf = estados.find(e => e.id === estadoId)?.uf ?? null;
p_estado_id: estadoUf ?? null, // ← "RS" ao invés de UUID
P0modalrpctypescript
🔴
P0-2 · DistribuicaoDrawerPreparacao — podeAvancar não valida empresa_id
drawers/DistribuicaoDrawerPreparacao.tsx : linha 33
A condição para mover processo de Preparação → Aguardando não verifica se empresa_id está preenchido. Processo chega à distribuição sem empresa ré vinculada → falha no backend.
// BUG atual
const podeAvancar = !!advogadoId && docsDisponiveis >= 3;

// FIX sugerido
const podeAvancar = !!advogadoId && docsDisponiveis >= 3 && !!processo?.empresa_id;
P0drawervalidação
🔴
P0-3 · ModalDistribuicaoLote — Sem validação de estado pré-distribuição
modals/ModalDistribuicaoLote.tsx
Não há checkpoint que valide se os processos estão em etapa_distribuicao = 'aguardando'. Pode tentar distribuir processos já em distribuido ou em_distribuicao, causando falhas parciais sem feedback claro por item.

Análise: Efetuar Distribuição

Petição obrigatória? → NÃO ✅

A validação foi removida de TabAguardando.tsx:131-134. allSelectedHavePeticaoOk retorna true sempre que há processo selecionado. Distribuição funciona sem petição gerada — comportamento intencional.

CNJ (validado async) Advogado Estado Justiça
Cidade (opcional) Vara (opcional) Foro Regional (opcional) Juiz (opcional)

Análise: Aguardando Preparação

O que determina que processo fica em "preparacao"?
  • Criado via ModalCriarProcesso → RPC rpc_criar_processo_rascunho seta etapa_distribuicao = 'preparacao'
  • Retornado de "aguardando" via botão voltarPreparacao em TabAguardando
Empresa e Restrições — exibidas?
✅ Empresa exibida no card ✅ Restrição como badge ❌ Empresa NÃO validada ao avançar etapa (P0-2)
⚠️ Investigação: ~2.795 processos pendentes

Processos migrados do sistema legado provavelmente sem todos os campos preenchidos. Para confirmar status de cada processo:

  • Verificar campo distribuicao_manual no banco (boolean)
  • Confirmar se empresa_id está nulo nesses registros
  • Confirmar se restricao_id está nulo nesses registros
  • Verificar flag no sistema legado se já foi distribuído

Campos Transferidos → Processos

Criação (rpc_criar_processo_rascunho)
  • cliente_id
  • empresa_id
  • tipo_id (tipo_peticao)
  • advogado_id
  • restricao_tipo / contrato / data / valor
  • valor_causa
  • prioridade
  • observacoes
  • tipo_processo
Efetuar distribuição (rpc_efetuar_distribuicao_manual)
  • processo_id
  • numero_cnj
  • data_distribuicao
  • p_estado_id ⚠️ (bug P0-1)
  • cidade / vara / foro_regional
  • julgador_nome
  • advogado_id (read-only)
  • usuario_id (ator)
Campos que já estão no processo (não re-passados)

empresa_id, restricao_*, valor_causa já estão vinculados ao processo na criação. A distribuição só atualiza os campos processuais (CNJ, vara, juiz).
