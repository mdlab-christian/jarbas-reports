Achados Detalhados
P2 FRONTEND JSX
SR2-001 — DropdownMenu trigger sem aria-label
Arquivo: components/AcordosHeader.tsx:96
O botao “…” (MoreHorizontal) que abre o menu de acoes (Export, Gerenciar Plataformas) nao tem aria-label nem title. Screen readers leem apenas “button” sem contexto.
Fix: Adicionar aria-label="Menu de acoes" ao Button linha 96.
P2 FRONTEND JSX
SR2-002 — 13x “as any” type casts
Arquivos: ModalEnvioLote, ModalPropostaForm, ModalAcordoForm, useAcordos, useAcordosStats, useEnvioLotePolling
13 instancias de as any espalhadas pela pagina, principalmente em chamadas RPC Supabase e acesso a nested joins. Comprometem type safety e dificultam refactoring. Nao causam bugs em runtime mas impedem o TypeScript de pegar erros de tipo.
Fix: Criar interfaces tipadas para os retornos das RPCs (AcordoListarResult, AcordoStatsResult) e para os nested joins do PostgREST. Substituir as any pelas interfaces. Escopo grande — considerar como tech debt para sprint dedicada.
P2 FRONTEND JSX
SR2-003 — 3x console.error DEV-gated
Arquivo: modals/ModalGerenciarPlataformas.tsx:77,171,190
Tres chamadas console.error protegidas por if (import.meta.env.DEV). Nao chegam a producao mas sao desnecessarias — os erros ja sao tratados com toast.error() nas mesmas linhas. Ruido em dev.
Fix: Remover as 3 linhas de console.error. O toast ja informa o usuario.
P2 BACKEND DADOS
SR2-004 — 3 webhook_requests stuck “processing”
Tabela: webhook_requests | Contexto: acordos.classificar_resposta
3 registros na tabela webhook_requests com status “processing” desde 2026-03-06 (26 dias). Provavelmente orfaos de uma execucao n8n que falhou sem atualizar o status. Nao afetam o funcionamento atual mas poluem o log e podem confundir monitoramento.
Fix: UPDATE webhook_requests SET status = 'timeout' WHERE status = 'processing' AND created_at < NOW() - INTERVAL '1 day' AND contexto ILIKE '%acordo%';
