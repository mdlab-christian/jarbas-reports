Achados Restantes (P2 — Tech Debt)

⚠️ DB2-001 (P2/RLS) — atividades_insert_own_org policy usa org_id() em vez de get_org_id()

DB-009 do Ciclo 1 corrigiu as policies de SELECT/UPDATE/DELETE, mas a policy INSERT ainda usa a função legada org_id(). Funcionalmente não causa bug (ambas as funções retornam a mesma org), mas é inconsistência técnica — pode quebrar se org_id() for removido.

-- Corrigir: DROP POLICY IF EXISTS "atividades_insert_own_org" ON atividades; CREATE POLICY "atividades_insert_own_org" ON atividades FOR INSERT WITH CHECK (organizacao_id = public.get_org_id());

💡 DB2-002 (P2/SCHEMA) — data_vencimento, data_atividade, prazo_fatal são DATE, não TIMESTAMPTZ

3 campos de data usam tipo DATE simples. O padrão MdFlow exige TIMESTAMPTZ. Em produção com dados já existentes, a migração precisa de atenção especial para preservar valores. Impacto: filtros de data podem ter comportamento inesperado em fuso horário. Não bloqueia funcionalidade hoje.

-- Migração requer atenção (tem dados): ALTER TABLE atividades ALTER COLUMN data_vencimento TYPE TIMESTAMPTZ USING data_vencimento::timestamptz; ALTER TABLE atividades ALTER COLUMN data_atividade TYPE TIMESTAMPTZ USING data_atividade::timestamptz; ALTER TABLE atividades ALTER COLUMN prazo_fatal TYPE TIMESTAMPTZ USING prazo_fatal::timestamptz;

💡 DB2-003 (P2/RPC) — Filtros server-side em fn_atividades_lista_com_leitura ainda ausentes

Herdado do Ciclo 1 (DB-006 pulado). A RPC não aceita filtros de status, responsavel_id, tipo ou busca — toda filtragem é feita no cliente. Com muitos dados, isso impacta performance. Requer refatoração maior da RPC + frontend.
