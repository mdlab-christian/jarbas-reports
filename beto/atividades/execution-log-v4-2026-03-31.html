Evidência: Migration 20260401220001 aparecia como aplicada em supabase migration list, mas o banco ainda executava a versão antiga com CASE uuid/text. Fix real: aplicar via psql direto na connection string postgresql://postgres:FVbfpmJnN9hllGMc@db.qdivfairxhdihaqqypgb.supabase.co:5432/postgres.
