P1 JSX FRONT-001 — Histórico: "undefined → undefined" nos status changes

Categoria: CAT-G (Estados Visuais) · Arquivo: DrawerTabEventos.tsx:244
Problema: A aba Histórico mostra "Sistema Revelia — undefined → undefined" nas transições de status. O payload do trigger não contém os campos status_anterior/status_novo — ou os usa com nomes diferentes. String(undefined) = "undefined", e STATUS_CONFIG["undefined"] é undefined, então o fallback também retorna "undefined".
Fix sugerido: Inspecionar o payload real do trigger (via Supabase → tabela eventos_operacionais) e ajustar os nomes dos campos no lookup. Como fallback imediato: payload.status_anterior ?? payload.de ?? payload.from ?? '?'.

❌ Antes
Sistema alterou status undefined → undefined
✅ Depois
Sistema alterou status Pendente → Em andamento

P1 Hook FRONT-002 — Kanban: 2 items Pendente vs 23 na lista

Categoria: CAT-C (Cards/Listas) · Arquivo: hooks/useAtividadesKanban.ts
Problema: A view Kanban exibe apenas 2 cards em Pendente, enquanto a view Lista e os KPIs mostram 23 pendentes. Possíveis causas: (1) fn_get_atividades_kanban usa p_user_id com escopo diferente do fn_atividades_lista_com_leitura; (2) a RPC retorna apenas itens sem deleted_at mas com condição diferente; (3) filtro client-side ativo afeta o kanban mas não a lista.
Fix sugerido: Inspecionar o hook useAtividadesKanban e comparar os parâmetros enviados ao fn_get_atividades_kanban com os do fn_atividades_lista_com_leitura.

P2 CSS only FRONT-003 — Mobile 375px: botão "Nova Atividade" cortado no header

Categoria: CAT-H (Responsividade) · Arquivo: layout/AtividadesPageHeader.tsx
Problema: O PageHeader render 3 botões em linha (OlivIA, Exportar, Nova Atividade). Em 375px o container flex items-center gap-2 não wrapa, cortando o último botão. A área de toque do CTA verde fica inacessível.
Fix: Esconder "OlivIA" e "Exportar" com hidden sm:flex em mobile. "Nova Atividade" fica sempre visível. Área de toque mínima garantida.

P2 CSS only FRONT-005 — KPI NumberTicker usa font-display (não é token DS)

Categoria: CAT-B (Design System) · Arquivo: kpis/AtividadesKpiGrid.tsx:54

P2 CSS only FRONT-006 — Cores Tailwind raw (text-blue-500, text-emerald-500) em vez de tokens DS

Categoria: CAT-B (Design System) · Arquivos: AtividadesKpiGrid.tsx, AtividadesKanbanBoard.tsx
Problema: text-blue-500 aparece em 3 lugares (KPI "Em andamento", Kanban coluna, Drawer). text-emerald-500 em 1. O DS v5 define --info: #60a5fa (→ text-info) e --success: #22c55e (→ text-success).
Fix: text-blue-500 → text-info | text-emerald-500 → text-success | bg-blue-500/15 → bg-info/15.
