Detalhes dos Fixes

POLISH-001 — e.preventDefault() no atalho N

Problema: O handler de teclado abria o modal mas não prevenia o evento padrão. Com autoFocus no campo Título, o browser propagava o keydown para o input recém-focado — resultando em form.titulo = 'n' e isDirty = true imediatamente.

Fix: e.preventDefault() antes de openNovaAtividade(). Analogia: o shortcut / já tinha e.preventDefault() — o N estava faltando.

POLISH-002 — Capitalização do título do calendário

Problema: className="... capitalize ..." aplica CSS text-transform: capitalize que transforma a primeira letra de cada palavra. Resultado: "Março De 2026" (D maiúsculo).

Fix: Removido capitalize do className. Adicionado IIFE para capitalizar só o primeiro caractere da string: s.charAt(0).toUpperCase() + s.slice(1). format(date, "MMMM 'de' yyyy", { locale: ptBR }) já retorna "março de 2026" — só precisava capitalizar o M.

POLISH-004 — Histórico unificado (eventos + comentários)

Problema: DrawerTabEventos lia apenas eventos_operacionais. Comentários ficavam isolados na aba "Comentários" e não apareciam no histórico cronológico.

Fix: Query paralela em atividades_comentarios. Normalização para TimelineItem unificado. Merge + sort por created_at descrescente. Comentários renderizados com bg-primary/5 border-primary/20 para distinção visual. Realtime subscription adicionada para atividades_comentarios também. Filter chip "Comentários" funciona em ambas as fontes.
