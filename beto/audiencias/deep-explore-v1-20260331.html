🔴 P1 — Alta Prioridade

BUG-001 P1 FUNC STATE Botão "Criar Audiência" dispara dialog "Descartar alterações?" ao invés de submeter
Arquivo
ModalAudiencia.tsx linha ~455 (DialogFooter Button onClick)
Descrição
Ao preencher o modal de Nova Audiência e clicar em "Criar Audiência", o sistema exibe o dialog "Descartar alterações? Você tem alterações não salvas." ao invés de executar o handleSaveWithParticipantes. O webhook audiencias/crud é funcional (confirmado via curl direto), mas a submissão via browser está bloqueada pelo dirty state interceptando o onClick.
Causa Raiz Provável
O Dialog onOpenChange={handleClose} está sendo chamado no momento do click no botão do footer. O isDirty=true (form tem dados) faz o handleClose disparar setConfirmClose(true) antes do handleSaveWithParticipantes executar. Possível propagação de evento ou o Radix Dialog tratando o click no botão como fechamento.
Impacto
Usuário não consegue criar audiências via UI. Criação só funciona via webhook direto. Funcionalidade P0 da página comprometida.
Solução Proposta
Adicionar e.stopPropagation() no onClick do botão Criar. Ou verificar se o handleSave não está chamando onOpenChange(false) antes de completar. Verificar também se o useEffect de reset está disparando incorretamente.
BUG-002 P1 RESP Overflow horizontal em mobile 375px — sidebar não colapsa
Arquivo
Layout global (sidebar) / MAIN com lg:ml-[var(--sidebar-collapsed)]
Descrição
Em viewport 375px, o elemento MAIN tem left: 256px causando scrollWidth=583 vs innerWidth=375. O conteúdo da página extravasa 208px para fora da viewport. Os cards KPI são cortados à direita.
Causa Raiz
A classe lg:ml-[var(--sidebar-collapsed)] no MAIN aplica margin-left em breakpoints < lg. No mobile, a sidebar não está colapsada para 0 e a margem empurra o conteúdo para fora.
Impacto
Experiência móvel completamente quebrada. Usuários mobile (provavelmente advogados acessando via telefone) veem conteúdo cortado.
Solução Proposta
Adicionar overflow-x: hidden no html/body OU garantir que ml-0 seja aplicado quando sidebar está recolhida em mobile. Verificar se há bug de detecção de breakpoint.

🟡 P2 — Média Prioridade

BUG-003 P2 UX Campo Processo não preenche texto no modal de edição
Arquivo
ModalAudiencia.tsx ~linha 70 (useEffect selectedProcesso)
Descrição
Ao abrir o modal de edição, o campo "Buscar por CNJ ou cliente..." permanece vazio mesmo após 3+ segundos. O processo_id está correto no banco, o selectedProcesso carrega via useEffect async (query Supabase), mas o ProcessoSearchField não reflete o valor carregado no texto do input.
Causa Raiz
O ProcessoSearchField espera uma prop de valor inicial ou controlled value. O setSelectedProcesso é chamado mas o componente mantém estado interno (uncontrolled) e não atualiza o display após montagem.
Impacto
UX confusa — usuário abre edição e vê campo Processo vazio, pensando que precisa re-selecionar. Pode causar erro se usuário salvar sem selecionar processo (processo_id já está no form.processo_id, mas visualmente confuso).
BUG-004 P2 FUNC Exportar audiências falha — workflow n8n audiencias/exportar INACTIVE
Arquivo
AudienciasTabContent.tsx (função handleExport)
Descrição

🟢 P3 — Baixa Prioridade

BUG-007 P3 UI Hora exibida na lista em UTC (11:00) vs modal em local time (14:00)
Arquivo
AudienciaCard.tsx linha ~73 (formatDateTime) / useAudienciaForm.ts linha ~200 (inserção)
Descrição
O webhook insere a data como 2026-04-15T14:00:00.000Z (UTC), que representa 14:00 UTC = 11:00 BRT (-3). O formatDateTime usa Intl.DateTimeFormat que converte para timezone local do browser. No browser headless (UTC), exibe 14:00→11:00. No modal de edição, o datetime-local input mostra o valor sem conversão (14:00). Inconsistência visual entre lista e modal.
Solução
O webhook deve interpretar a data como horário local BRT (-3) ao inserir. Ou o formatDateTime deve usar timezone explícito: timeZone: 'America/Sao_Paulo'.
BUG-008 P3 UI Botões prev/next do calendário sem aria-label (acessibilidade)

AudienciasCalendarView.tsx (botões de navegação)
Descrição
Os botões < e > do calendário não possuem aria-label. Leitores de tela não conseguem identificar a ação dos botões.
Solução
Adicionar aria-label="Mês anterior" e aria-label="Próximo mês".
