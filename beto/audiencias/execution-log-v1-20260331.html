Detalhes das Correcoes

P0 BUG-001 — staleTime 5min → 30s

5 hooks React Query tinham staleTime: 5 * 60 * 1000 (5 minutos), violando o padrao CLAUDE.md que exige 30 segundos. Dados ficavam desatualizados por muito tempo, especialmente com realtime ativo.

Fix: Substituido por staleTime: 30_000 em todos os 5 hooks.

Arquivos: useAudiencias.ts:70, useAudienciasKPIs.ts:81, useAudienciaCalendar.ts:46, useCorrespondentes.ts:77, useCorrespondenteSelect.ts:31

P1 BUG-002 — RPC reverter_pagamento sem validacao

A RPC reverter_pagamento_audiencia retornava success: true mesmo quando o p_audiencia_id nao existia no banco. A RPC irma registrar_pagamento_audiencia ja validava corretamente.

Fix: CREATE OR REPLACE FUNCTION com SELECT INTO + IF NOT FOUND + validacao de estado pago = true antes de reverter.

Verificacao: Testado via curl com UUID inexistente — agora retorna {"success": false, "action": "toast_error", "message": "Audiencia nao encontrada"}

P1 BUG-003 — Header buttons mobile

No viewport 375px, os 3 botoes do header (OlivIA, Sync Calendar, Nova Audiencia) ficavam truncados/inacessiveis. OlivIA e Sync Calendar sao secundarios.

Fix: hidden sm:inline-flex nos botoes OlivIA e Sync Calendar. Botao "Nova Audiencia" mantido com texto abreviado (<span class="hidden sm:inline">Nova</span> Audiencia).

P2 BUG-004 — Cegueira financeira inconsistente

CorrespondentesTab mostrava "---" quando acesso_financeiro=false, enquanto o padrao MdFlow e R$ •••••.

Fix: Logica ternaria ajustada para exibir R$ ••••• quando sem acesso financeiro.

P2 BUG-005 — Cast as never desnecessario

ModalAudiencia usava .from('audiencias_participantes' as never) em 3 locais, assumindo que a tabela nao estava nos types gerados. Verificacao mostrou que a tabela JA existe em types.ts desde a geracao.

Fix: Removido as never dos 3 locais. Supabase SDK agora valida tipos corretamente.

P2 BUG-006 — useAudienciaIA manual state

Hook usava useState + useEffect manual para fetch, sem cache, retry, dedup ou staleTime. Realtime atualizava via setState direto em vez de invalidar cache.

Fix: Refatorado para useQuery com staleTime: 30_000. Realtime agora faz queryClient.invalidateQueries para manter consistencia com o cache.
