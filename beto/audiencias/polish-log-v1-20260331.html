Detalhes dos Fixes

POLISH-001 — Dirty state nao bloqueia mais o save

Causa raiz: O Radix Dialog propagava o click do botao "Criar Audiencia" como evento de fechamento. Como isDirty=true (form tinha dados), o handleClose abria o dialog "Descartar alteracoes?" antes do save executar.

Fix:

  • Adicionado e.stopPropagation() no onClick do botao de save para evitar propagacao
  • Adicionado guard if (isSaving) return no handleClose para nao interceptar durante save

POLISH-002 — Click popover calendario abre modal

Causa raiz: O Radix Popover consumia o evento do click antes de executar onEditAudiencia. O setOpen(false) fechava o popover sincronamente e o unmount impedia o callback.

Fix: Deferido o onEditAudiencia para o proximo tick com setTimeout(() => onEditAudiencia(id), 0), permitindo que o Popover feche antes da abertura do modal.

POLISH-003 — Campo Processo exibe CNJ ao editar

Causa raiz: O ProcessoSearchField e semi-controlled — o value (processo_id) era sincronizado, mas o inputValue (texto exibido) nao era atualizado quando o valor vinha de carregamento async externo.

Fix:

  • Adicionada prop displayValue ao ProcessoSearchField (componente shared)
  • useEffect que sincroniza inputValue quando displayValue + value estao presentes e inputValue esta vazio
  • ModalAudiencia passa displayValue={selectedProcesso?.numero_cnj_formatado || selectedProcesso?.numero_cnj}

Nota: Componente shared processo_search_field.tsx modificado. A nova prop e opcional — zero breaking change para outras 9 paginas que usam o componente.

POLISH-004 — Aria-labels calendario

Fix: Adicionado aria-label="Mes anterior" e aria-label="Proximo mes" nos botoes de navegacao do calendario.
