Achados Detalhados

P1 N8N LOG
SR2-001 — webhook_requests não registra chamadas de audiencias
Carry-over de SR-009 v1 · Requer edição manual no editor n8n
Nenhum dos 9 workflows de audiencias registra chamadas na tabela webhook_requests. O Manual de Automações (seção 5, item 4) exige registro de todas as chamadas com request_id, contexto e status. Confirmado: SELECT * FROM webhook_requests WHERE contexto LIKE 'audiencia%' retorna 0 registros.
Fix: Adicionar nó "Insert webhook_requests" em cada workflow n8n — requer editor n8n manual, não automatizável via API.
P2 FRONTEND A11Y
SR2-002 — 34 Labels sem htmlFor nos modais e filtros
ModalAudiencia.tsx · ModalCorrespondente.tsx · ModalConfirmacaoPagamento.tsx · ModalReverterPagamento.tsx · AudienciasFilterBar.tsx
Apenas 1 de 35 Labels tem htmlFor (AudienciasFilterBar:59). Os Labels e Inputs são siblings (não parent-child), então htmlFor é necessário para associar label→input via screen reader. Impacto: screen readers não anunciam o label ao focar o input.
Fix: Adicionar htmlFor="campo-id" em cada Label e id="campo-id" no Input correspondente.
P2 FRONTEND PERF
SR2-003 — select('*') em 3 hooks
useAudienciaForm.ts:60 · useCorrespondentes.ts:53 · useCorrespondenteForm.ts:51
Três hooks usam .select('*') em vez de selecionar colunas específicas. Em tabelas com poucas colunas (audiencias: 23, correspondentes: 19) o impacto é baixo, mas viola o padrão CLAUDE.md "Sem select('*') — selecionar apenas colunas necessárias".
Fix: Substituir .select('*') por lista explícita de colunas usadas pelo componente. Ex: .select('id, processo_id, data_audiencia, tipo, ...')
P2 FRONTEND CODE
SR2-004 — UploadAta.tsx insert sem .select('id')
src/pages/audiencias/components/UploadAta.tsx:61
O insert em audiencias_arquivos (linha 61) não tem .select('id'). Padrão MdFlow exige RETURNING id em todo INSERT. Os outros inserts da página já foram corrigidos pelo SR v1. 
// ANTES (atual):
await supabase.from('audiencias_arquivos').insert({...});

// DEPOIS:
await supabase.from('audiencias_arquivos').insert({...}).select('id').single();
Fix: Adicionar .select('id').single() após o insert.
P2 BACKEND SCHEMA
SR2-005 — audiencias_arquivos sem deleted_at + trigger updated_at órfão
Tabela: audiencias_arquivos · Supabase
A tabela audiencias_arquivos não tem coluna deleted_at (viola soft-delete obrigatório) nem coluna updated_at, mas possui trigger set_updated_at que referencia uma coluna inexistente (no-op ou erro silencioso). As demais 4 tabelas da página já têm soft delete correto.
Fix: Migration para adicionar updated_at TIMESTAMPTZ DEFAULT now() NOT NULL e deleted_at TIMESTAMPTZ em audiencias_arquivos.
P2 BACKEND RLS
SR2-006 — audiencias_participantes usa get_org_id() vs org_id()
Tabela: audiencias_participantes · pg_policies
Todas as tabelas de audiencias usam org_id() nas policies, exceto audiencias_participantes que usa get_org_id(). Ambas as funções existem e funcionam (isolamento confirmado com org fictício = 0 rows), mas a inconsistência pode causar confusão em manutenção futura.
Fix: Recriar policy de audiencias_participantes usando org_id() para consistência com as demais tabelas.
P2 FRONTEND CODE
SR2-007 — ModalAudiencia.tsx em 502 linhas
src/pages/audiencias/modals/ModalAudiencia.tsx (502 linhas)
Arquivo excede o limite de 500 linhas do projeto (era 496 antes dos fixes SR v1, agora 502). Candidatos à extração: bloco de Participantes (linhas 414-448), bloco de Correspondente+Valor (linhas 352-386), bloco de opções de pagamento (linhas 387-401).
Fix: Extrair ParticipantesSection e/ou CorrespondenteValorFields como sub-componentes.
