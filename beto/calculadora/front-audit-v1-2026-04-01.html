Two-pane layout NÃO colapsa em mobile/tablet <768px

O layout principal em index.tsx:149 usa flex-[3] / flex-[2] sem nenhum breakpoint responsivo. Em 375px, editor e resultado ficam lado a lado comprimidos, com texto cortado e alertas ilegíveis. O painel de resultado deveria empilhar abaixo do editor em mobile.

Arquivo: src/pages/calculo/index.tsx:149

Correção: flex flex-col lg:flex-row no container + resultado com max-h-[50vh] lg:max-h-none

FRONT-003 CSS Esforço: médio CAT-H Responsividade

Eventos row layout muito comprimido em mobile

CalculoEventos.tsx:72 usa flex items-center gap-2 flex-wrap com widths fixos (w-36). O flex-wrap ajuda, mas em 375px os 4 campos ocupam 2 linhas desalinhadas. Melhor usar grid responsivo.

Correção: grid grid-cols-1 sm:grid-cols-[auto_auto_auto_1fr_auto] gap-2 items-end

FRONT-004 CSS Esforço: baixo CAT-H Responsividade

Encargos: controles inline wrapping irregular em mobile

CalculoAdicionais.tsx:88 — flex items-center gap-2 flex-wrap com inputs de width fixo (w-20, w-44). Em mobile, "Taxa" e "Base" ficam em linhas separadas sem alinhamento. Melhor empilhar verticalmente.

Correção: flex flex-col sm:flex-row + remover widths fixos em mobile

FRONT-005 CSS Esforço: baixo CAT-A Hierarquia Visual

Header "Resultado" sem Manrope — inconsistente com H1

index.tsx:189 — O título "Resultado" do painel direito usa text-sm font-semibold (Inter), enquanto o H1 "Calculadora Judicial" usa font-display (Manrope). Para consistência, "Resultado" deveria usar Manrope.

Correção: h2 className="text-sm font-display font-bold"

FRONT-006 JSX Esforço: baixo CAT-E Botões e Ações

Botão "Exportar PDF" duplicado — header + resultado

O header tem o botão "PDF" (CalculoHeader.tsx:64) e o resultado também tem "Exportar PDF" full-width (CalculoResultadoPane.tsx:189). Redundância confusa — o header é acessível sempre, o do resultado é redundante.

Correção: Remover o botão de export do ResultadoPane ou torná-lo visível apenas em mobile (quando o header fica longe).

FRONT-007 CSS Esforço: baixo CAT-A Hierarquia Visual

Section headers (Valor, Períodos, etc.) — peso visual baixo

Todos os headers de seção usam text-sm font-medium text-muted-foreground — funcionalmente ok, mas poderiam ser text-sm font-semibold text-foreground para dar mais hierarquia entre seções e campos.

Correção: text-sm font-semibold text-foreground nos títulos de seção (Valor a ser atualizado, Períodos, etc.)

FRONT-008 CSS Esforço: baixo CAT-C Cards

Cards de período sem hover state

CalculoPeriodos.tsx:109 — O card do período usa rounded-lg border border-border bg-surface p-3 sem hover. Os cards de encargos têm transition-colors e mudam quando ativos. Período deveria ter ao menos um hover:border-border/80.

Correção: Adicionar hover:border-primary/20 transition-colors
