append_cliente_documento

P0-001: Sem verificacao de organizacao_id P0 CRITICO
A funcao append_cliente_documento faz UPDATE em clientes usando apenas WHERE id = p_cliente_id sem verificar organizacao_id. Sendo SECURITY DEFINER, bypassa RLS. Qualquer usuario autenticado pode adicionar documentos a qualquer cliente de qualquer organizacao.
-- ATUAL (vulneravel):
UPDATE clientes
SET documentos = coalesce(documentos, '[]'::jsonb) || jsonb_build_array(v_doc)
WHERE id = p_cliente_id;

-- CORRETO:
UPDATE clientes
SET documentos = coalesce(documentos, '[]'::jsonb) || jsonb_build_array(v_doc)
WHERE id = p_cliente_id
  AND organizacao_id = get_org_id()
  AND deleted_at IS NULL;

ensure_cliente_stub

P1-001: Aceita p_org_id como parametro (sem validacao JWT) P1
A funcao ensure_cliente_stub recebe p_org_id como parametro externo em vez de derivar de get_org_id(). Embora usada apenas internamente (Jarbas), permite criacao de clientes em qualquer org se chamada via PostgREST. Mitigacao: RLS da tabela bloqueia INSERT cross-org, mas como e SECURITY DEFINER, o RLS e bypassado.

9 Parecer Sincero

Achados por Severidade

P0-001: append_cliente_documento sem isolamento multi-tenant P0
SECURITY DEFINER + UPDATE sem WHERE organizacao_id. Permite cross-tenant data injection via PostgREST. Fix imediato necessario: adicionar AND organizacao_id = get_org_id() AND deleted_at IS NULL no WHERE.
-- FIX:
CREATE OR REPLACE FUNCTION public.append_cliente_documento(p_cliente_id uuid, p_documento jsonb)
RETURNS jsonb LANGUAGE plpgsql SECURITY DEFINER SET search_path TO 'public' AS $$
DECLARE v_doc jsonb; v_uid text;
BEGIN
  -- ADICIONAR: Verificacao de ownership
  IF NOT EXISTS (
    SELECT 1 FROM clientes WHERE id = p_cliente_id AND organizacao_id = get_org_id() AND deleted_at IS NULL
  ) THEN
    RAISE EXCEPTION 'cliente_not_found';
  END IF;

  v_uid := coalesce(p_documento->>'uid', gen_random_uuid()::text);
  v_doc := jsonb_build_object(
    'uid', v_uid, 'tipo', coalesce(p_documento->>'tipo', 'outro'),
    'nome', coalesce(p_documento->>'nome', 'documento'),
    'situacao', coalesce(p_documento->>'situacao', 'pendente'),
    'drive_status', 'pendente', 'storage_path', p_documento->>'storage_path',
    'created_at', now()::text
  );
  UPDATE clientes SET documentos = coalesce(documentos, '[]'::jsonb) || jsonb_build_array(v_doc), updated_at = now()
  WHERE id = p_cliente_id AND organizacao_id = get_org_id();
  RETURN v_doc;
END; $$;
P1-001/002/003: RPCs com p_organizacao_id como parametro externo P1
3 funcoes (ensure_cliente_stub, fn_cliente_resumo, fn_processos_por_cliente) recebem org_id como parametro em vez de derivar do JWT. Como sao SECURITY DEFINER, qualquer caller pode fornecer org_id de outra org. Risco mitigado pelo fato de que essas RPCs sao usadas internamente (Jarbas, n8n), mas expostas via PostgREST. Sugestao: adicionar guard IF p_org_id != get_org_id() AND get_org_id() IS NOT NULL THEN RAISE para bloquear acesso cross-org quando chamada via frontend.
P2-001/002: Webhooks editar e import ausentes no n8n P2
2 webhooks esperados (editar, import) nao existem. Edicao e feita via RPC direto, import via frontend. Sem orquestracao n8n para side-effects.
P2-003: 2 indices redundantes P2
idx_clientes_cpf_org e idx_clientes_org_created sao duplicatas de outros indices. Desperdicam espaco e slows writes marginalmente.
P2-004: Inconsistencia org_id() vs get_org_id() P2
Algumas RPCs usam org_id() (ex: add_cliente_restricao, update_cliente_restricao), outras usam get_org_id() (ex: get_clientes_paginados). Ambas existem e sao equivalentes. Padronizar para uma unica funcao.
P2-005: get_clientes_kpis faz 4 queries sequenciais P2
Pode ser otimizada para 1 query com COUNT(*) FILTER. Impacto real: minimo (9ms total), mas e tech debt.
-- Versao otimizada:
SELECT
  COUNT(*) AS total_clientes,
  COUNT(*) FILTER (WHERE created_at >= date_trunc('month', now())) AS novos_mes_atual,
  COUNT(*) FILTER (WHERE created_at >= date_trunc('month', now()) - INTERVAL '1 month'
                     AND created_at < date_trunc('month', now())) AS novos_mes_anterior,
  CASE WHEN COUNT(*) = 0 THEN 0
    ELSE ROUND(COUNT(*) FILTER (WHERE engajamento_nivel = 'alto') * 100.0 / COUNT(*), 1)
  END AS engajamento_alto_pct
FROM clientes WHERE organizacao_id = v_org_id AND deleted_at IS NULL;
