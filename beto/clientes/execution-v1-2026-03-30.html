EXEC-04 P0 · URGENTE FRONTEND BUG-1: .select() chained ilegal em RPC — advogado_responsavel nunca populado
Contexto

O Supabase JS SDK não suporta .select() com join PostgREST em chamadas RPC. A linha .select('*, advogado_responsavel:advogados!advogado_responsavel_id(id, nome, oab, oab_uf)') é silenciosamente ignorada. O advogado responsável NUNCA aparece no drawer. A solução está no SQL da RPC — o join deve ser feito dentro da query SQL.

Arquivo
~/beta-mdflow/src/hooks/useCliente.ts — linhas 13–16
Dependência
⚡ Independente
Prompt para Claude Code
Projeto: ~/beta-mdflow/
Supabase projeto: qdivfairxhdihaqqypgb

Tarefa: corrigir o chain .select() ilegal em useCliente.ts e garantir que advogado_responsavel
seja retornado pela RPC.

PROBLEMA:
Em ~/beta-mdflow/src/hooks/useCliente.ts, o código atual:
  const { data, error } = await supabase
    .rpc('get_cliente_by_cpf', { p_cpf: cpfSlug! })
    .select('*, advogado_responsavel:advogados!advogado_responsavel_id(id, nome, oab, oab_uf)')
    .maybeSingle();

O .select() após .rpc() é silenciosamente ignorado pelo SDK Supabase. 
O join deve existir dentro do SQL da função get_cliente_by_cpf.

SOLUÇÃO:
1. Verificar a definição atual de get_cliente_by_cpf:
   supabase db dump --schema public --linked | grep -A 80 "get_cliente_by_cpf"

2. Se a função já faz o join com advogados internamente → apenas remover o .select() do hook.
   Se não faz → criar migration que adiciona o join.

3. A migration (se necessária) deve adicionar ao SELECT da função:
   jsonb_build_object(
     'id', adv.id,
     'nome', adv.nome,
     'oab', adv.oab,
     'oab_uf', adv.oab_uf
   ) AS advogado_responsavel
   com LEFT JOIN advogados adv ON adv.id = c.advogado_responsavel_id AND adv.deleted_at IS NULL

4. Atualizar ~/beta-mdflow/src/hooks/useCliente.ts para remover o .select() ilegal:
   const { data, error } = await supabase
     .rpc('get_cliente_by_cpf', { p_cpf: cpfSlug! })
     .maybeSingle();

5. Verificar se o tipo Cliente em ~/beta-mdflow/src/types/cliente.types.ts inclui:
   advogado_responsavel?: { id: string; nome: string; oab?: string; oab_uf?: string } | null
   Se não incluir, adicionar.

6. Aplicar migration se criada: cd ~/beta-mdflow && supabase db push --linked
