Achados

FRONT-001 CSS baixo CAT-H / CAT-E

Botão 'Exportar' cortado a 1280px

A row de ações no header (OlivIA, Importar, Exportar, Novo Cliente) overflow no desktop a 1280px. O texto 'Exportar' fica truncado. As ações secundárias precisam de flex-wrap ou colapsar no dropdown em telas < 1440px.

FRONT-002 CSS baixo CAT-H

KPI cards — 3° card cortado no tablet (768px)

Os 3 KPI cards estão em grid-cols-3 (sm:grid-cols-3). No tablet 768px, o 3° card ('Engajamento Alto') fica cortado. Solução: usar grid-cols-2 no breakpoint sm, com o 3° wrapping para segunda linha ou md:grid-cols-3.

FRONT-003 CSS médio CAT-C

Coluna 'Cadastrado em' mal visível a 1280px

A tabela usa min-w-[1000px] com overflow-x-auto, mas no desktop 1280px (com sidebar de 56px) restam ~1210px — insuficiente para 9 colunas confortáveis. Considerar responsive column visibility: ocultar 'Engajamento' ou 'Cadastrado em' em lg, mostrando apenas em xl.

FRONT-004 JSX baixo CAT-D / CAT-H

Tabs do drawer sem indicador de scroll no mobile

O drawer exibe 6 tabs (Dados, Restrições, Jurídico, Documentos, Pendentes, Histórico). No mobile 375px só 4 são visíveis. Não há gradient/fade indicando que existem mais tabs à direita. Solução: adicionar gradient overlay ou usar ScrollArea no TabsList.

FRONT-005 JSX médio CAT-D

Aba 'Dados' do drawer — layout esparso no desktop

A aba 'Dados' mostra apenas 4 campos (Nome, CPF, Telefone, E-mail) numa área de ~65vw. Muito espaço vazio abaixo. Pode ser parcialmente por dados não preenchidos, mas o layout poderia usar grid de 2 colunas e agrupar mais informações (endereço, origem, advogado, captador).

FRONT-006 CSS baixo CAT-C

Hover state nas rows da tabela muito sutil

As rows têm cursor-pointer mas nenhum background-change visível no hover. O usuário não recebe feedback visual de que a row é clicável. Adicionar hover:bg-muted/30 ou hover:bg-primary/5 no TableRow.

FRONT-007 CSS baixo CAT-A

Título 'Clientes' — verificar uso de Manrope

O H1 'Clientes' deve usar Manrope (font-display). Verificar se o componente PageHeader aplica a classe correta. No modal 'Novo Cliente' o DialogTitle já usa font-display — consistência verificada.

FRONT-008 CSS baixo CAT-H

Botão 'Novo Cliente' poderia ser icon-only no mobile

No mobile 375px o botão 'Novo Cliente' (texto + ícone UserPlus) ocupa espaço significativo ao lado do dropdown menu. Considerar usar size='icon' com tooltip no breakpoint mobile, ou mover para dentro do dropdown.
