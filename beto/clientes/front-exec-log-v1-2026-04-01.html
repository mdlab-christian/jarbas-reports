Detalhes dos Fixes

FRONT-001 (CSS) — Ações secundárias header overflow 1280px
Botões OlivIA, Importar, Exportar transbordavam no header em 1280px. Alterado breakpoint de sm para lg.
Antes: hidden sm:flex / sm:hidden
Depois: hidden lg:flex / lg:hidden
FRONT-002 (CSS) — KPI grid 3 colunas cortando em tablet
Grid de KPIs saltava direto de 1 coluna para 3 no breakpoint sm. Adicionado intermediário sm:grid-cols-2.
Antes: grid-cols-1 sm:grid-cols-3
Depois: grid-cols-1 sm:grid-cols-2 md:grid-cols-3
FRONT-003 (CSS) — Coluna 'Cadastrado em' visível demais
Coluna aparecia em telas 1024px onde já havia scroll horizontal. Movido para xl (1280px+).
Antes: hidden lg:table-cell
Depois: hidden xl:table-cell
FRONT-004 (JSX) — Tabs do drawer cortadas em mobile
TabsList no ClienteDrawer não tinha scroll horizontal; abas ficavam cortadas em 375px. Componente compartilhado — fix conservador.
Antes: <TabsList className="mx-4 mt-3 self-start bg-muted/50">
Depois: <TabsList className="mx-4 mt-3 self-start bg-muted/50 max-w-full overflow-x-auto scrollbar-none">
FRONT-006 (CSS) — Rows da tabela sem hover state
Linhas da tabela eram clicáveis (cursor-pointer) mas sem feedback visual no hover.
Antes: className="cursor-pointer"
Depois: className="cursor-pointer hover:bg-muted/30 transition-colors"
FRONT-008 (CSS) — Botão Novo Cliente grande demais em mobile
Botão com texto ocupava espaço excessivo em 375px. Convertido para icon-only em mobile, texto visível a partir de sm.
Antes: <Button><UserPlus /> Novo Cliente</Button>
Depois: <Button size="icon" className="sm:w-auto sm:px-4 sm:py-2"><UserPlus className="sm:mr-2" /><span className="hidden sm:inline">Novo Cliente</span></Button>

Pendências

⚠️ FRONT-005 (JSX, médio) — Aba Dados layout esparso no desktop

Motivo do skip: A aba "Dados" do ClienteDrawer renderiza até 15+ campos condicionalmente. Com dados completos (nome, CPF, endereço, bancário, etc.), o layout preenche bem. O aspecto "esparso" ocorre quando o cliente tem poucos dados preenchidos — é behavior by design, não bug de layout.

Risco adicional: Componente compartilhado entre /clientes, /crm e /distribuicao. Alterar layout (e.g., grid 2 colunas) pode quebrar contextos com drawer mais estreito.

Recomendação: Não alterar. Se necessário no futuro, considerar grid responsivo grid-cols-1 lg:grid-cols-2 apenas para a seção "Dados Pessoais", com teste nos 3 contextos.
