Detalhamento de Falhas

P0 FT-B07: Popover fecha ao clicar checkbox de engajamento
O Popover de filtros avancados (Radix <Popover>) fecha automaticamente quando o usuario interage com um <Checkbox> (engajamento Alto/Medio/Baixo) ou seu <Label>. O click propaga e o Radix interpreta como interacao "outside" do popover.

Impacto: Impossivel filtrar por engajamento sem workaround.
Fix sugerido: Adicionar onPointerDownOutside={(e) => e.preventDefault()} no <PopoverContent>, ou usar modal={true} no <Popover>. Alternativa: usar onOpenAutoFocus + e.stopPropagation() nos checkboxes.
P0 FT-F02: Webhook clientes/excluir retorna erro NOT NULL
Ao confirmar exclusao, o webhook clientes/excluir retorna:
null value in column "nome" of relation "clientes" violates not-null constraint

O webhook aparenta fazer um UPDATE que seta campos obrigatorios (nome) para NULL em vez de simplesmente setar deleted_at = now().

Impacto: Nenhum cliente pode ser excluido via UI.
Fix sugerido: Revisar o workflow n8n clientes/excluir. O UPDATE deve ser apenas: UPDATE clientes SET deleted_at = now() WHERE id = $1.
P0 FT-J01/J05/J06: Overflow horizontal em TODAS as viewports
O body tem scrollWidth maior que o viewport em todas as resolucoes testadas:
- 375px: body 1317px (3.5x overflow)
- 768px: body 1317px (1.7x overflow)
- 1280px: body 1501px (1.2x overflow)

O container principal .flex.flex-col.gap-6.p-6 nao respeita overflow: hidden ou max-width: 100%. O KPI grid + filtros + tabela combinados estouram o layout.

Impacto: Scroll horizontal indesejado em todas as resolucoes. Mobile praticamente inutilizavel.
Fix sugerido: Adicionar overflow-x: hidden ou max-w-full overflow-hidden no container .flex-1.min-h-0 do layout. Verificar que o wrapper principal da pagina tem w-full overflow-x-hidden.
P1 FT-B04: Limpar busca nao atualiza URL
Quando o usuario limpa o campo de busca (backspace ou clear), o localSearch fica vazio mas o debounce nao propaga a mudanca para o URL. Suspita: a comparacao localSearch !== filters.search no useEffect pode falhar quando ambos sao string vazia por timing.

Impacto: URL mantem ?q=termo_antigo apos limpar busca. Compartilhar URL leva a filtro errado.
Fix sugerido: Verificar o fluxo de debounce no ClienteFilters. Possivelmente o fill do agent-browser nao dispara onChange corretamente (pode ser falso-positivo do teste).
P1 FT-C04: Botao "Proxima" nao responde ao click direto
O botao de paginacao "Proxima" nao responde ao agent-browser click porque esta posicionado abaixo da viewport visivel (abaixo da tabela de 50 rows). O agent-browser nao faz auto-scroll para o elemento. Funciona via eval JS .click().

Impacto: Baixo. O botao funciona normalmente para usuarios humanos que scrollam. E uma limitacao do teste automatizado.
Nota: O botao em si funciona corretamente.
