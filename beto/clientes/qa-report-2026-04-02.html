Bugs Encontrados

P1 CL-BUG-R5-01 — Webhook endpoint diverge do spec

Arquivo: src/pages/clientes/index.tsx:123 | Impacto: Documentacao desatualizada, pode causar confusao futura

O spec documenta os webhooks como:

WH-CLIENTES-003: mdflow/clientes/deletar WH-CLIENTES-004: mdflow/clientes/import

Mas o codigo usa:

callWebhook('clientes/excluir') // nao 'clientes/deletar' callWebhook('clientes/importar') // nao 'clientes/import'

Causa: Evolucao do codigo sem atualizar a documentacao do spec.

Impacto: Se alguem implementar novo cenario n8n baseado no spec, criara endpoints errados.

Solucao: Atualizar spec _resumido___clientes.txt com os nomes reais dos endpoints.

P2 CL-BUG-R5-02 — KPI total (4.350) diverge do total da tabela (4.007)

Arquivo: src/hooks/useClienteKPIs.ts vs src/hooks/useClientes.ts | Impacto: Confusao visual para o usuario

O KPI "Total de Clientes" exibe 4.350, mas a tabela mostra "Mostrando 1-50 de 4.007 clientes".

// useClientes passa p_excluir_cnpj: true → filtra empresas (CNPJ) // useClienteKPIs nao filtra CNPJ → conta TODOS os registros

Causa: get_clientes_kpis conta todos os clientes da org (incluindo CNPJs/empresas), enquanto a listagem filtra empresas com p_excluir_cnpj=true.

Impacto: Discrepancia visual confusa: "4.350 clientes" no KPI mas "4.007 clientes" na paginacao.

Solucao: Alterar RPC get_clientes_kpis para excluir CNPJs (LENGTH(cpf_cnpj) > 11 OR cpf_cnpj LIKE '%/%'), ou passar parametro p_excluir_cnpj.

P2 CL-BUG-R5-03 — Maioria dos clientes sem engajamento_nivel / advogado / whatsapp

Impacto: Dados operacionais | Tabela exibe muitos "--" e "Sem responsavel"

Na amostra de dados reais visualizada no browser, praticamente todos os 50 clientes da pagina 1 exibem:

WhatsApp: — Cidade/UF: — Advogado: Sem responsavel Ultimo contato: — Engajamento: Baixo Processos: —

Causa: Dados importados em massa (possivelmente via Advbox) sem enriquecimento completo.

Impacto: Visual poluido com muitos placeholders. Nao e bug de codigo, e data quality.

Solucao: Executar enrichment batch para preencher whatsapp, advogado_responsavel_id, engajamento_nivel. Considerar esconder colunas totalmente vazias ou mostrar indicador de "dados incompletos".
