Notas Técnicas Relevantes

POLISH-001 — isArchived contradição

O n8n retornou active=True isArchived=True simultaneamente. A flag isArchived no n8n v1 API pode ser artefato de versão. O webhook de produção responde com HTTP 500 (falha de execução por payload inválido), não com "workflow inactive" — o que confirma que o workflow está ativo e processando requests. Recomenda-se desarchive manual no painel n8n para limpar o estado.

POLISH-002 — Estratégia de largura

A classe w-full sm:w-[72vw] está no SheetContent. Em Tailwind, sm: começa em 640px — portanto em 375px apenas w-full está ativo. O drawer ocupa 100% da largura. Em 640px+ assume 72vw com mínimo de 600px. Fix correto e idiomático.

CL-NEW-02 — CPF algoritmo

Implementação correta com dois passes: (1) rejeição de sequências homogêneas via regex /^(\d)\1{10}$/, (2) cálculo módulo 11 para primeiro e segundo dígito verificador. Cobre todos os CPFs inválidos canônicos (111.111.111-11, 000.000.000-00, etc.).

CL-NEW-04 — ORIGEM_LABEL duplo mapeamento

O mapa popula tanto ORIGEM_LABEL[o.id] quanto ORIGEM_LABEL[o.nome.toLowerCase()] — cobrindo clientes com campo origem gravado como UUID e os com nome legado. A colunas CSV usa ?? (c.origem ?? '') como fallback seguro.
