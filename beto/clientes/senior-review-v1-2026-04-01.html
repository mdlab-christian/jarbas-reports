Achados Detalhados

SR-001 P0 BACKEND EXECUÇÃO OBRIGATÓRIA
Busca por nome retorna TODOS os clientes — search completamente quebrada
Função: get_clientes_paginados | Complexidade: RPC (migration SQL)

Descrição: Quando o usuário busca por texto sem dígitos (ex: "MICHELE"), o branch de CPF na RPC get_clientes_paginados faz regexp_replace('MICHELE', '\D', '', 'g') que retorna string vazia. O padrão ILIKE vira '%%' que matcha todos os registros.

Evidência:

-- Busca direta (funciona): 15 resultados
SELECT COUNT(*) FROM clientes
WHERE organizacao_id = get_org_id() AND deleted_at IS NULL
  AND unaccent(lower(nome)) ILIKE '%michele%';
→ 15

-- Via RPC (quebrado): 4007 resultados
SELECT r->>'total' FROM get_clientes_paginados(p_search := 'MICHELE', ...) r;
→ 4007

-- Causa raiz:
regexp_replace('MICHELE', '\D', '', 'g') → '' (string vazia)
'%' || '' || '%' → '%%'
'qualquer_cpf' ILIKE '%%' → TRUE  (matcha tudo!)

Impacto: A busca por nome é a feature mais usada da página /clientes. Com 4.350 clientes, buscar qualquer nome retorna a lista inteira sem filtrar. Funcionalidade core completamente inoperante.

Fix: Adicionar guard no branch de CPF — só ativar quando o search contém pelo menos 1 dígito:

-- ANTES (bugado):
OR regexp_replace(COALESCE(c.cpf_cnpj, ''), '\D', '', 'g')
   ILIKE '%' || regexp_replace(p_search, '\D', '', 'g') || '%'

-- DEPOIS (correto):
OR (
  p_search ~ '\d'   -- só ativa quando search tem dígitos
  AND regexp_replace(COALESCE(c.cpf_cnpj, ''), '\D', '', 'g')
      ILIKE '%' || regexp_replace(p_search, '\D', '', 'g') || '%'
)

Nota: o fix deve ser aplicado nas DUAS ocorrências do WHERE na RPC (COUNT e data query).
