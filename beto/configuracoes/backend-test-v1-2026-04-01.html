P0 — Criticos (bloqueiam em producao)

BK-001 crm_estagios_funil — trigger set_updated_at CRASHA em todo UPDATE

Trigger set_updated_at_crm_estagios_funil chama NEW.updated_at = NOW() mas a tabela nao tem coluna updated_at. Todo UPDATE falha com: record "new" has no field "updated_at".

Impacto: Usuarios nao conseguem editar estagios do funil CRM, reordenar, nem desativar. CrmEstagiosFunil.tsx linhas 72, 97, 115, 117 — todos falham.

Testado: UPDATE crm_estagios_funil SET cor = cor WHERE id = '...' → ERROR 42703

-- FIX (escolher A ou B):
-- A) Adicionar coluna (preferido — alinha com padrao MdFlow):
ALTER TABLE crm_estagios_funil ADD COLUMN updated_at TIMESTAMPTZ DEFAULT now();

-- B) Remover trigger (menos ideal):
DROP TRIGGER set_updated_at_crm_estagios_funil ON crm_estagios_funil;

BK-002 crm_followup_config — mesmo bug: trigger sem coluna updated_at

Trigger set_updated_at_crm_followup_config falha identicamente. CrmFollowupConfig.tsx linha 104 — editar followup config falha.

Testado: UPDATE crm_followup_config SET ativo = ativo WHERE id = '...' → ERROR 42703

ALTER TABLE crm_followup_config ADD COLUMN updated_at TIMESTAMPTZ DEFAULT now();

BK-003 crm_origens — mesmo bug: trigger sem coluna updated_at

Trigger set_updated_at_crm_origens falha identicamente. CrmOrigens.tsx linhas 65, 84 — editar/toggle origem falha.

Testado: UPDATE crm_origens SET ativo = ativo WHERE id = '...' → ERROR 42703

ALTER TABLE crm_origens ADD COLUMN updated_at TIMESTAMPTZ DEFAULT now();

P1 — Importantes

BK-004 estados_distribuicao — RLS public_read permite leitura cross-tenant

Policy estados_distribuicao_public_read com auth.role() = 'authenticated' permite que qualquer usuario autenticado leia estados de TODAS as organizacoes. Risco real baixo (dados sao UF abbreviations), mas viola isolamento multi-tenant. 

-- FIX: Remover policy redundante (ja tem select_own_org):
DROP POLICY estados_distribuicao_public_read ON estados_distribuicao;
-- Tambem remover service_write (ja coberto pelas policies normais + service_role bypassa RLS):
DROP POLICY estados_distribuicao_service_write ON estados_distribuicao;

BK-005 estados_distribuicao — falta coluna created_at

Unica tabela de config sem created_at. Padrao MdFlow exige created_at TIMESTAMPTZ NOT NULL DEFAULT now() em toda tabela. 

ALTER TABLE estados_distribuicao ADD COLUMN created_at TIMESTAMPTZ DEFAULT now() NOT NULL;

BK-006 Valores monetarios usando NUMERIC em vez de BIGINT centavos

3 campos usam NUMERIC para valores monetarios. Padrao MdFlow: BIGINT armazenando centavos.

BK-007 crm_estagios_funil e crm_origens — DELETE nao exige admin

DELETE policies usam organizacao_id IN (SELECT...) sem check de perfil = 'admin'. Qualquer membro da org pode deletar estagios do funil ou origens. Demais tabelas exigem admin para DELETE. 

-- FIX:
DROP POLICY "Org members can delete crm_estagios_funil" ON crm_estagios_funil;
CREATE POLICY "crm_estagios_funil_delete_admin" ON crm_estagios_funil
  FOR DELETE USING (organizacao_id = org_id() AND (auth.jwt() ->> 'perfil') = 'admin');

DROP POLICY "Org members can delete crm_origens" ON crm_origens;
CREATE POLICY "crm_origens_delete_admin" ON crm_origens
  FOR DELETE USING (organizacao_id = org_id() AND (auth.jwt() ->> 'perfil') = 'admin');

BK-008 organizacao_setores — falta trigger updated_at

Tabela nao tem trigger BEFORE UPDATE para updated_at. Alem disso, nao tem coluna updated_at — so tem deleted_at. 

ALTER TABLE organizacao_setores ADD COLUMN updated_at TIMESTAMPTZ DEFAULT now();
CREATE TRIGGER set_updated_at_organizacao_setores BEFORE UPDATE ON organizacao_setores
  FOR EACH ROW EXECUTE FUNCTION set_updated_at();

P2 — Tech Debt

BK-009 RLS pattern inconsistente: 4 tabelas usam IN (SELECT) em vez de org_id()

crm_estagios_funil, crm_origens, docs_categorias, docs_tipos usam o pattern antigo organizacao_id IN (SELECT users.organizacao_id FROM users WHERE users.id = auth.uid()). Funciona mas e menos eficiente e inconsistente com as outras 18 tabelas que usam org_id().

BK-010 mdzap_ai_modules — 3 policies RLS redundantes

Tem org_isolation (ALL), org_select_ai_modules (SELECT) e org_write_ai_modules (ALL) — todas com get_org_id(). As policies ALL ja cobrem SELECT. Consolidar em 4 policies padrao (SELECT, INSERT, UPDATE, DELETE).

BK-011 11 funcoes duplicadas de updated_at

Banco tem 11 funcoes fazendo exatamente o mesmo: set_updated_at, update_updated_at, update_updated_at_column, fn_auto_updated_at, fn_set_updated_at, fn_update_updated_at, trigger_set_updated_at, trg_set_updated_at, trg_honorarios_updated_at, fn_mdchat_members_updated_at, fn_mdchat_channels_updated_at. Todas fazem NEW.updated_at = NOW(); RETURN NEW;. Consolidar em uma unica funcao.

BK-012 controller_dominios — campos tipo e acao sem CHECK constraint

Campos tipo e acao sao TEXT sem restricao de valores. Demais tabelas de config tem CHECKs bem definidos.

BK-013 Faltam indices parciais WHERE deleted_at IS NULL

Tabelas com deleted_at (docs_categorias, docs_tipos, organizacao_setores) nao tem indice parcial CREATE INDEX ON tabela(organizacao_id) WHERE deleted_at IS NULL. Com pouco volume atual nao impacta, mas e padrao recomendado.

O que esta bem

  • ✓ RLS habilitado em 22/22 tabelas — zero tabela aberta
  • ✓ organizacao_id NOT NULL em todas — multi-tenant correto
  • ✓ UUID gen_random_uuid() como PK em todas
  • ✓ INSERT WITH CHECK em todas — nenhum INSERT permite cross-tenant
  • ✓ CHECK constraints bem definidos — atividades_tipos (tipo, complexidade), cartoes (bandeira), config_templates (tipo), contas (tipo), mdzap_ai_modules (slug), processos_situacoes (categoria), whatsapp (status, proposito), advogados (oab format), correspondentes (pix_tipo)

  • ✓ Indice em organizacao_id em todas as tabelas
  • ✓ org_id() e SECURITY DEFINER com search_path
  • ✓ DELETE exige admin em 16/18 tabelas (exceto crm_estagios e crm_origens)

Parecer tecnico

Diagnostico geral

O backend de /configuracoes e solido na fundacao: RLS em todas tabelas, zero ENUM nativo, CHECK constraints bem aplicados, indices corretos, org_id() SECURITY DEFINER. O problema concentrado e especificamente nos 3 tabelas CRM (crm_estagios_funil, crm_followup_config, crm_origens) que tiveram triggers criados sem a coluna correspondente. Isso e um P0 que impede qualquer edicao nesses catalagos em producao.

O fix e trivial: 3 ALTER TABLE ADD COLUMN. Nao existem RPCs nesta pagina (uso correto de CRUD direto via PostgREST para catalogo admin). Nao existem webhooks especificos de configuracoes no n8n (controller e whatsapp tem workflows, mas sao de runtime, nao de config CRUD).

Fora dos P0, as inconsistencias sao todas de padronizacao: 4 tabelas com RLS pattern antigo, valores monetarios em NUMERIC, 11 funcoes duplicadas de updated_at. Nenhuma bloqueia funcionalidade. Nota: 72% (7.2/10) por causa dos P0; sem eles seria ~88%.
