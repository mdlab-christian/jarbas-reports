9. Parecer do Auditor

Score: 95% -- Backend saudavel, sem P0.


O backend da pagina /configuracoes esta em bom estado apos as 11 migrations do ciclo v1. Os ganhos foram reais: todas as 21 tabelas tem RLS habilitado com tenant isolation confirmado, as 4 RPCs seguem o padrao correto (SECURITY DEFINER, search_path, auth check), e a performance e excelente (sub-3ms em todas as queries).


O que melhorou desde v1 (72% -> 95%):

  • CHECK constraints adicionados em todas as tabelas relevantes
  • RLS policies completas (CRUD) em todas as 21 tabelas
  • Indexes de organizacao_id em todas as tabelas
  • Triggers de updated_at em todas as tabelas (exceto config_paginas_sistema)
  • Audit trail via log_config_change em 5 tabelas criticas

O que falta para 100%:

  • config_paginas_sistema precisa de created_at/updated_at/deleted_at (P1)
  • 3 UPDATE policies precisam de WITH CHECK (P1 -- seguranca)
  • Campos monetarios NUMERIC -> BIGINT (P1 -- divida tecnica, requer coordenacao front)
  • Consolidar funcoes updated_at duplicadas (P2 -- limpeza)

Sobre soft delete (deleted_at): 15 tabelas nao tem deleted_at. Para tabelas de catalogo, o padrao ativo=false funciona como alternativa aceitavel. O CLAUDE.md diz "soft delete obrigatorio", mas na pratica ninguem deleta etiquetas ou situacoes -- apenas desativa. A unica que realmente precisa e config_paginas_sistema (que tambem falta timestamps).


Recomendacao: Fixar BK-V2-02 (config_paginas_sistema timestamps) e BK-V2-04 (WITH CHECK) na proxima sprint. O resto e divida tecnica gerenciavel.

BK-V2-02 -- config_paginas_sistema timestamps

-- Adicionar timestamps e trigger
ALTER TABLE config_paginas_sistema
  ADD COLUMN IF NOT EXISTS created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  ADD COLUMN IF NOT EXISTS updated_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  ADD COLUMN IF NOT EXISTS deleted_at TIMESTAMPTZ;

CREATE TRIGGER set_updated_at_config_paginas_sistema
  BEFORE UPDATE ON config_paginas_sistema
  FOR EACH ROW EXECUTE FUNCTION set_updated_at();

CREATE INDEX IF NOT EXISTS idx_config_paginas_org_active
  ON config_paginas_sistema(organizacao_id) WHERE deleted_at IS NULL;

BK-V2-04 -- WITH CHECK nas 3 UPDATE policies

-- docs_categorias
DROP POLICY IF EXISTS docs_categorias_update_own_org ON docs_categorias;
CREATE POLICY docs_categorias_update_own_org ON docs_categorias
  FOR UPDATE USING (organizacao_id = org_id())
  WITH CHECK (organizacao_id = org_id());

-- docs_tipos
DROP POLICY IF EXISTS docs_tipos_update_own_org ON docs_tipos;
CREATE POLICY docs_tipos_update_own_org ON docs_tipos
  FOR UPDATE USING (organizacao_id = org_id())
  WITH CHECK (organizacao_id = org_id());

-- mdzap_ai_modules

CREATE POLICY mdzap_ai_modules_update_own_org ON mdzap_ai_modules

  WITH CHECK (organizacao_id = org_id());
