Achados

P1 — Importante (2)

P1 SRv3-001 PermissionGrid tem refetchInterval: 30_000 residual
Arquivo: components/PermissionGrid.tsx:75
O refetchInterval foi removido das tabs (SR-011), mas permaneceu no PermissionGrid. Causa polling desnecessario a cada 30s enquanto o modal ModalEditarUsuario esta aberto. Deve ser removido — staleTime ja garante cache adequado.
P1 SRv3-002 4 casts 'as never' em queries Supabase (type-safety)
Arquivos: CrmFollowupConfig.tsx:53, DistribuicaoEditor.tsx:122,193,215
.from('config_templates' as never) e .from('advogados_distribuicao_itens' as never) etc. Indica que as tabelas nao estao no type generation do Supabase. Deve-se rodar supabase gen types para adicionar estas tabelas ao schema TS, eliminando os casts inseguros.

P2 — Melhoria (5)

P2 SRv3-003 DialogDescription ausente em 14 modais
Nenhum modal inclui <DialogDescription> (Radix exige para leitores de tela). Pode usar <DialogDescription className="sr-only"> se nao houver texto visivel. Impacto: warning no console + acessibilidade prejudicada.
P2 SRv3-004 ModalEditarUsuario sem dirty state (3 sub-tabs)
Arquivo: modals/ModalEditarUsuario.tsx (425 linhas, 3 sub-tabs: Perfil, Permissoes, Funcionario).
E o unico modal de edicao sem dirty state. Complexidade: alta (3 sub-tabs com estados independentes). Aceito como known issue — implementacao requer refator significativo.
P2 SRv3-005 Partial indexes ausentes em 5 tabelas
Tabelas sem index WHERE deleted_at IS NULL: atividades_tipos, processos_situacoes, config_etiquetas_catalogo, controller_dominios, config_templates, config_paginas_sistema.
Impacto baixo (tabelas pequenas de catalogo, <100 rows), mas nao segue o padrao aplicado em docs_categorias/docs_tipos/organizacao_setores.
P2 SRv3-006 ModalConvidarUsuario e ModalGerenciarSetores sem dirty state
ModalConvidarUsuario: create-only (sem edicao), impacto minimo.
ModalGerenciarSetores: gerencia setores inline, sem formulario de edicao complexo.
Aceito como baixa prioridade.
P2 SRv3-007 Responsive layout limitado (5 breakpoint classes, 0 touch targets)
Apenas 5 classes sm:/md:/lg:grid-cols ou flex. Zero touch targets 44x44px explicitamente definidos. Botoes shadcn (size="sm") tem height ~32px, abaixo do target de 44px recomendado para mobile. O dropdown mobile para tabs mitiga o problema principal.
