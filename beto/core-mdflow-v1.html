CRM

Central de gestão de leads e pipeline comercial
🔴 Core Crítico ✅ Estável /crm 📄 CrmPage.tsx

📋 Descrição Core

Pipeline Kanban de leads organizados por etapas do funil (coluna). Suporta visualização Kanban e Lista. Permite triagem, auditorias, distribuição de restrição, gerenciamento de documentos e integração com MDZap para histórico de contato.

📥 Fluxo de Entrada

  • Novos leads via ModalNovoLead (formulário manual)
  • Leads via indicações (página Indicações)
  • Integração n8n / webhooks externos
  • Auditorias disparadas pela ApiFull API
  • Histórico MDZap (whatsapp_conversas)

📤 Fluxo de Saída

  • Lead aprovado → Distribuição (ModalDistribuirLead)
  • Dados do lead → ClienteDrawer (shared)
  • Lead arquivado → tabela leads com status arquivado
  • Vinculação advogado → tabela advogados
  • Auditoria → tabela auditorias
  • MDZap → navega para /mdzap?lead=ID

🔗 Ligações

→ Distribuição → Auditorias → MDZap → Clientes (drawer)

🌳 Componentes Principais

CrmPage.tsx
├── CrmKanban.tsx — Board kanban de leads por etapa
├── CrmListView.tsx — Visualização em tabela/lista
├── CrmDashboard.tsx — KPIs e gráficos do funil
├── CrmRadar.tsx — Radar de leads (visão estratégica)
├── CrmArquivados.tsx — Leads arquivados
├── CrmFilterBar.tsx — Filtros globais
├── MdzapConnectionIndicator — Status conexão MDZap
├── modals/ModalNovoLead.tsx — Cadastro de lead
├── modals/ModalTriagem.tsx — Checklist de triagem (restrições)
├── modals/ModalDistribuirLead.tsx — Efetua distribuição
├── modals/ModalDistribuicao.tsx — Configuração distribuição
├── modals/ModalNovaAuditoria.tsx — Dispara auditoria CPF
├── modals/ModalArquivar.tsx — Arquiva lead com motivo
├── modals/ModalAIReport.tsx — Relatório IA do lead
└── ClienteDrawer (shared) — Painel lateral do lead/cliente

📝 Inputs Principais

  • Nome, CPF, telefone, email do lead
  • Etapa do funil (crm_estagios_funil)
  • Atendente responsável
  • OAB (advogados_oabs) 🔄 Reutilizável

Distribuição

Central de distribuição de processos para advogados
🔴 Core Crítico ✅ Estável /distribuicao 📄 DistribuicaoPage.tsx

📋 Descrição Core

Central que gerencia o pipeline de distribuição de processos para advogados. Recebe leads aprovados do CRM, passa por preparação (documentos + petição inicial), agendamento e efetua a distribuição no sistema externo (E-PROC). Possui painel JARBAS para distribuição automática.

📥 Fluxo de Entrada

  • Lead aprovado no CRM → cria processo em processos
  • Processo manual via ModalCriarProcesso
  • Agendamentos via WizardAgendarDistribuicao
  • JARBAS auto-distribui via rpc_efetuar_distribuicao

📤 Fluxo de Saída

  • Distribuição efetuada → Processo (status: distribuído)
  • Lead arquivado no CRM
  • Cliente criado em clientes (ensure_cliente_stub)
  • Petição gerada via OlivIA
  • Devolução possível → CRM (rpc_devolver_crm)

🔗 Ligações

← CRM → Processos → Clientes → OlivIA (petição)

Controller

Central de intimações, atividades e monitoramento processual
🔴 Core Crítico ✅ Estável /controller 📄 controller/index.tsx

📋 Descrição Core

Hub central de operações jurídicas. Recebe intimações/despachos do E-PROC via JARBAS, exibe em painel de análise com IA. Permite criar Atividades (para usuário específico) ou Documentos Pendentes (para todos). Inclui monitoramento, equipe, dashboard KPIs, e modo treinamento (Playbooks).

  • Análise IA (ControllerResumoAnaliseIA)

📤 Fluxo de Saída

  • Cria Atividade → página Atividades
  • Cria Documento Pendente → Doc. Pendentes
  • Classifica processo (status)
  • Transfere atividades em lote
  • Exporta dados

← E-PROC/Processos → Atividades → Doc. Pendentes → Decisões

🗺️ Mapa de Fluxo

Fluxo completo Lead → Cliente → Processo → Resolução
📱 WhatsApp / Formulário / Indicação→entrada de lead
↓
🎯 CRM→triagem, restrições, documentos, auditoria CPF, MDZap
↓ aprovado
📦 Distribuição→prepara documentos, gera petição (OlivIA), agenda ou distribui
↓ distribuído
🏛️ E-PROC (externo)↔protocolo, despachos, intimações via JARBAS
↓ intimações chegam
🎛️ Controller→análise IA, classifica intimação, cria atividade ou doc pendente
→✅ Atividadestarefa para advogado específico
→📄 Doc. Pendentespara todos com acesso
↓ sentença
⚖️ Decisões→favorável → Execução | desfavorável → recurso
↓ acordo ou execução
🤝 Acordos / Execuções→💰 Financeiro

🔄 Fluxo paralelo: Entidades criadas na distribuição

📦 Distribuiçãocria👤 Cliente(ensure_cliente_stub)
📦 Distribuiçãocria📁 Processo(fn_criar_processo_distribuicao)
🎯 CRM Leadarquivado→movido para leads arquivados

🤖 JARBAS — Agente Automação

  • Monitora E-PROC via jusbrasil_monitoramentos / processos_monitoramento
  • Importa movimentações → intimacoes
  • Distribui automaticamente via fn_jarbas_iniciar_distribuicao
  • Analisa intimações com IA → fn_jarbas_salvar_analise_ia
  • Enriquece processos → fn_jarbas_enriquecer_processo
  • Painel no Controller (ControllerMonitoramento) e Distribuição (TabJarbas)
  • Estado em: jarbas_instances, jarbas_execucoes, jarbas_tarefas
