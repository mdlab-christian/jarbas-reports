Detalhes dos Bugs

P0
BUG-000
google_drive_id ausente no SELECT dos hooks de docs de lead
src/components/shared/cliente_drawer/tabs/documentos_lead_hooks.ts
Causa Raiz

Todos os 4 SELECTs em documentos_lead_hooks.ts omitiam a coluna google_drive_id. O componente DocumentoItem chama resolveUrl() que verifica doc.google_drive_id primeiro — sem esse campo, docs com Google Drive ID nunca abriam.

Impacto

Todos os docs com google_drive_id ficavam sem URL funcional (abrir/baixar quebrado).

Fix Aplicado
- .select('id, nome, arquivo_url, status, created_at') // 4 ocorrências + .select('id, nome, arquivo_url, google_drive_id, status, created_at') // Interface LeadDoc atualizada: + google_drive_id: string | null
P1
BUG-001 + BUG-002
Multi-select de docs + botão excluir restrição na pasta
src/components/shared/cliente_drawer/tabs/RestricaoPastaCard.tsx
BUG-001 — Multi-select

Pastas de restrição tinham apenas ações individuais por documento. Adicionado:

  • Botão "Selecionar múltiplos" (aparece quando há >1 doc)
  • Toolbar de seleção: checkbox select-all, contador, Download (N), Delete (N), X para sair
  • Checkbox por documento com Square / CheckSquare2
  • Bulk download: abre URLs em novas abas com 150ms de delay
  • Bulk delete: soft-delete em batch via update({ deleted_at })
BUG-002 — Botão Excluir Restrição

Header da pasta agora exibe ícone Trash2 (stopPropagation para não fechar o accordion). Prop onDeleteRestricao?: () => void adicionada. Parent (RestricaoPastaWrapper) usa useExcluirRestricao para implementar o handler.

Arquivos Alterados
RestricaoPastaCard.tsx — multi-select + botão delete header ClienteDrawerTabDocumentosLead.tsx — import useExcluirRestricao + handler no wrapper
P1
BUG-003
Auditoria sem nome do lead pré-preenchido
src/pages/crm/CrmPage.tsx + ModalNovaAuditoria.tsx + auditoria_form_card.tsx
Causa Raiz

handleAuditoriaLead em CrmPage selecionava apenas cpf do lead — o nome nunca era buscado nem passado ao modal. O AuditoriaFormCard não tinha prop initialNome.

Fix Aplicado
// CrmPage.tsx - .select('cpf') + .select('cpf, nome') + setAuditoriaNome(data?.nome ?? null) // ModalNovaAuditoria — novo prop: + nome?: string | null + initialNome={nome ?? undefined} // passa para AuditoriaFormCard // AuditoriaFormCard — novo prop: + initialNome?: string + useState(() => initialNome ?? '') // pré-preenche clienteSearch
Resultado

Campo "Cliente" no modal de auditoria agora vem pré-preenchido com o nome do lead. Se existir cliente com esse nome no banco, o dropdown autocompleta e vincula automaticamente.

P2
BUG-004
DistribuicaoDrawerRestricao sem seção de documentos
src/pages/distribuicao/drawers/DistribuicaoDrawerRestricao.tsx
Causa Raiz

O drawer da distribuição (para restrições na lista) mostrava dados do cliente, da restrição e outros processos — mas nunca os documentos vinculados à restrição (Tier 2 + Tier 1 do lead).

Fix Aplicado
+ import { useRestricaoDocs } from '@/.../documentos_lead_hooks' + import { DocumentoItem } from '@/.../DocumentoItem' // Hook adicionado: + const { data: documentos = [] } = useRestricaoDocs( + restricaoId ?? '', lead?.id ?? '', organizacaoId + ) // Seção "Documentos" adicionada (aparece apenas quando documentos.length > 0)
Comportamento

A seção de Documentos aparece condicionalmente (apenas se há docs). Mostra docs Tier 2 (desta restrição) + Tier 1 (docs centrais do lead, que aparecem em todas as pastas). Usa DocumentoItem com ações individuais (abrir, baixar, renomear, excluir).
