N shortcut não guardava contra modal/sheet aberto e contentEditable
O handler do atalho N → Novo Lead em CrmPage.tsx verificava apenas INPUT/TEXTAREA/SELECT, mas não protegia contra: (1) elementos contentEditable, (2) dialogs/sheets abertos (Sheet do drawer, Dialog de modais). Antes
if (e.target instanceof HTMLInputElement || e.target instanceof HTMLTextAreaElement
  || e.target instanceof HTMLSelectElement) return;
Depois
if (e.target instanceof HTMLInputElement || ...) return;
if ((e.target as HTMLElement)?.isContentEditable) return;
if (document.querySelector('[data-state="open"][role="dialog"], ...')) return;

✅ RESP POLISH-002
Kanban DnD sem TouchSensor — scroll horizontal bloqueado em mobile
O CrmKanban usava apenas PointerSensor e KeyboardSensor. Em dispositivos touch, o PointerSensor intercepta o primeiro toque antes que o browser possa iniciar o scroll nativo horizontal. Resultado: em mobile (375px), arrastar o dedo para rolar entre colunas iniciava drag em vez de scroll. Fix: adicionado TouchSensor com delay de 250ms — o browser tem 250ms para iniciar o scroll antes do drag ativar.
Antes
useSensor(PointerSensor, { activationConstraint: { distance: 8 } }),
useSensor(KeyboardSensor)
Depois
useSensor(TouchSensor, { activationConstraint: { delay: 250, tolerance: 8 } }),
useSensor(PointerSensor, { activationConstraint: { distance: 8 } }),
useSensor(KeyboardSensor)
