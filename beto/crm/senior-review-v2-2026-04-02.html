04 Achados P1 — Importantes
ID Tipo Achado Fix
SR2-P1-001 Backend
CrmOrigensChart inclui leads arquivados no cálculo de origens
src/components/crm/charts/CrmOrigensChart.tsx : linha 30–31
A query .is('deleted_at', null) filtra deletados mas NÃO filtra arquivado = false. O gráfico de origens no Dashboard conta leads arquivados, distorcendo a distribuição de onde vêm leads ativos. Com escritórios que arquivam muitos leads, o gráfico pode mostrar origens com proporção inversa à realidade ativa.

Fix: adicionar .eq('arquivado', false) na query.
1 linha
SR2-P1-002 Frontend
Leads com coluna_crm='documentos_pendentes' são invisíveis no kanban E na list view
src/components/crm/CrmKanban.tsx : linha 149 + CrmListView.tsx : linha 52
O CHECK constraint agora permite documentos_pendentes como valor de coluna. Porém o kanban filtra explicitamente por .in('coluna_crm', ['novos', 'em_atendimento', 'aguardando_documentos', 'aguardando_assinatura', 'triagem']) e o list view usa o mesmo array de 5 colunas. Qualquer lead que caia em documentos_pendentes via n8n ou automação DESAPARECE silenciosamente do pipeline. Verificado: nenhum lead em produção tem essa coluna agora (as 5 colunas ativas são as esperadas), mas o risco existe quando automações usarem essa coluna.

Fix: ou adicionar 'documentos_pendentes' nos arrays de coluna dos componentes, ou documentar explicitamente que essa coluna é só do /documentos_pendentes e nunca do CRM pipeline.
Decisão CTO

SR2-P2-002 Frontend
ModalArquivar sem mensagem de orientação quando motivos não estão cadastrados
src/components/crm/modals/ModalArquivar.tsx : linhas 148–158
Quando motivos?.length === 0 (ou undefined ainda carregando), o Select fica vazio e o botão "Arquivar" desabilitado. O usuário não recebe mensagem informando "Configure motivos em Configurações → CRM para arquivar leads". Resultado: o modal parece bugado. Fix: adicionar Alert condicional quando !isLoading && motivos?.length === 0.
UX
SR2-P2-003 Backend
ETAPA_COLORS não tem mapeamento para 'documentos_pendentes'
src/types/lead.types.ts : linha 47–53
ETAPA_COLORS mapeia 5 colunas. CrmListView usa ETAPA_COLORS[lead.coluna_crm] para estilizar o badge de etapa. Leads com coluna_crm='documentos_pendentes' renderizariam badge sem cor (fallback para string vazia). Fix: adicionar entrada documentos_pendentes: 'bg-destructive/10 text-destructive border-destructive/20'.
Visual
SR2-P2-004 Frontend
CrmListView: "carregar mais" é client-side mas query faz fetch de todos os leads sem limit
src/components/crm/CrmListView.tsx : linhas 46–80
A list view faz um único fetch de todos os leads não-arquivados sem paginação server-side ou limit. Com 1000+ leads, isso pode gerar payload grande. O kanban tem .limit(1000), mas a list view não tem. O "carregar mais" é puramente client-side (slice de array). Fix P2: adicionar .limit(500) como proteção, ou adicionar nota de backlog para paginação server-side futura.

Auditorias demoram. Polling a cada 10s pode sobrecarregar se muitos drawers abertos. Considerar aumentar para 30s com realtime fallback.
09 Veredicto Final
Nota 8.2 — Produção condicionada a 2 fixes P1
O /crm evoluiu de 7.6 → 8.2 neste ciclo. Os 3 blockers da v1 foram resolvidos: view arquivado corrigida, console.log removido, QA report documentado.

A arquitetura geral é sólida: drag-and-drop com undo e rollback, dedup antes de inserção, realtime com canais segregados, bulk actions via RPC, FilterBar responsiva, deep links completos, TypeScript limpo, zero hardcode de cores. O padrão de código está consistente e maduro.

2 fixes obrigatórios antes de produção:
1. SR2-P1-001 — CrmOrigensChart.tsx linha 30: adicionar .eq('arquivado', false). Fix 1 linha. Sem isso o gráfico de origens no Dashboard mostra números incorretos.
2. SR2-P1-002 — Decisão CTO: documentar ou incluir 'documentos_pendentes' nos arrays de coluna do kanban/listview. Sem isso, leads que caírem nessa coluna via automação desaparecem silenciosamente.

Os 4 P2 (3 aria-labels, ModalArquivar empty motivos, ETAPA_COLORS, list view sem limit) são polish recomendados mas não bloqueiam produção.
