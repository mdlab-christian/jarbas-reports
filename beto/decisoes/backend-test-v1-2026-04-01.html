Destaques por RPC

rpc_decisoes_aprovar — Excelente

  • Valida org do usuario via users.organizacao_id
  • Verifica status_analise = 'pendente' antes de aprovar
  • Aceita p_dados JSONB para atualizar resultado, tipo, valores
  • Retorna toast_refresh com decisao_id e status

rpc_decisoes_criar_execucao — Excelente

  • Role check: user_role(p_user_id) IN ('admin', 'juridico')
  • Valida resultado IN ('favoravel', 'parcial')
  • Verifica status_analise = 'aprovado' e execucao_id IS NULL
  • INSERT em execucoes com dados completos do processo
  • Arquiva a decisao automaticamente apos criar execucao
  • Retorna redirect com URL /execucoes/{id}

rpc_calcular_valor_atualizado — Excelente

  • Calcula valor corrigido por IPCA/INPC/SELIC/CDI/Manual
  • 196ms tempo de resposta
  • Retorna: valor_base, valor_corrigido, valor_juros, valor_total, fator_correcao, meses_decorridos

rpc_decisoes_inserir_via_pipeline — P2: FK error leak

  • Valida resultado via RAISE EXCEPTION (nao retorna JSONB action em erro de FK)
  • Se p_organizacao_id invalido: retorna raw PostgreSQL FK error 23503
  • Funcional para uso via service_role (Controller Pipeline), mas nao segue padrao JSONB das demais
