3. Achados

P1 — Importantes (2)

P1 N8N SR-001 — Webhook decisoes/insights sem workflow n8n
Arquivo: src/pages/decisoes/components/DecisoesDashboard.tsx:147
Descricao: O botao "Gerar Agora" no dashboard chama callWebhook('decisoes/insights', ...) mas nenhum dos 4 workflows n8n ativos (NOTIFICAR, EXPORT, WH-DEC-02, ANALISAR-v2) corresponde a este endpoint. O webhook falha silenciosamente — o toast mostra erro generico mas a funcionalidade de insights IA nao funciona.
Evidencia: jq '[.data[] | select(.name | test("insight"; "i"))]' retorna [] — zero workflows.
Fix: Criar workflow N8N-DECISOES-INSIGHTS no n8n com path mdflow/decisoes/insights. Pipeline: validar → buscar decisoes aprovadas do mes → gerar insights via Claude → INSERT em decisoes_insights → responder envelope canonico.
Complexidade: N8N-WORKFLOW (medio)
P1 FRONTEND SR-002 — ModalConfirmarRejeicao usa UPDATE direto em vez de RPC existente
Arquivo: src/pages/decisoes/modals/ModalConfirmarRejeicao.tsx:40-50
Descricao: O modal faz supabase.from('decisoes').update({status_analise: 'rejeitado', ...}) diretamente. O comentario L40 diz "Nao existe rpc_decisoes_rejeitar" — mas a RPC existe no banco (verificado via pg_proc). Usar UPDATE direto:
(1) Bypassa qualquer logica server-side da RPC (validacoes, side effects);
(2) Quebra padrao Action-Driven onde RPCs retornam envelope JSONB;
(3) Inconsistente com ModalAprovarDecisao que usa RPC corretamente.
Evidencia: SELECT proname FROM pg_proc WHERE proname = 'rpc_decisoes_rejeitar' retorna 1 row.
Fix: Substituir o UPDATE direto por chamada RPC: supabase.rpc('rpc_decisoes_rejeitar', { p_decisao_id: decisao.id, p_organizacao_id, p_motivo: motivo.trim() }). Remover comentario falso.
Complexidade: JSX (baixo)

P2 — Tech Debt (6)

P2 FRONTEND SR-003 — DecisaoModalShell.tsx excede 500 linhas (514L)
Arquivo: src/pages/decisoes/modals/DecisaoModalShell.tsx (514 linhas)
Descricao: Contem 5 tabs inline (TabDados, TabProcesso, TabDocumentos, TabHistorico, TabAnaliseIA). Regra do projeto: max 500 linhas por arquivo. Extrair cada tab como sub-componente em modals/components/.
Fix: Criar TabDados.tsx, TabProcesso.tsx, TabDocumentos.tsx, TabHistorico.tsx, TabAnaliseIA.tsx. Shell fica ~100 linhas.
Complexidade: JSX (medio — refatoracao sem mudanca funcional)
P2 FRONTEND SR-004 — RecursosVinculados.tsx query sem organizacao_id
Arquivo: src/pages/decisoes/modals/components/RecursosVinculados.tsx:40-46
Descricao: Query em decisoes_recursos filtra apenas por decisao_id, sem .eq('organizacao_id', orgId). RLS protege no banco (policy usa org_id()), mas viola padrao defense-in-depth do projeto. Todas as outras queries do /decisoes incluem organizacao_id.
Fix: Adicionar .eq('organizacao_id', organizacao_id) na query. Requer passar organizacao_id como prop ou via useAuth().
Complexidade: JSX (baixo)
P2 FRONTEND SR-005 — err?.message TS-unsafe em 3 catch blocks
Arquivos: ModalArquivar.tsx:57, ModalNotificarCliente.tsx:96, DecisoesDashboard.tsx:172
Descricao: Em catch blocks TypeScript, err e do tipo unknown. Acessar err?.message sem type guard e um anti-pattern TS. Outros modais da mesma pagina (ModalAprovarDecisao, ModalConfirmarRejeicao) ja usam o padrao correto: err instanceof Error ? err.message : 'erro desconhecido'.
Fix: Substituir err?.message ?? 'erro desconhecido' por err instanceof Error ? err.message : 'erro desconhecido' nos 3 arquivos.
Complexidade: JSX (baixo)
P2 FRONTEND SR-006 — as Record<string, any> em useDecisoesDashboard
Arquivo: src/pages/decisoes/hooks/useDecisoesDashboard.ts:46
Descricao: O resultado da view vw_decisoes_dashboard_stats e tipado como Record<string, any>. O any propaga para o componente DecisoesDashboard. Criar interface DashboardStats com campos tipados.
Fix: Criar type DashboardStats em decisoes.types.ts e usar no hook.
Complexidade: JSX (baixo)
P2 A11Y SR-007 — Acessibilidade minima: zero aria-labels em botoes icone
Arquivos: Toda a pagina /decisoes (24 arquivos)
Descricao: Nenhum aria-label encontrado nos arquivos da pagina. Botoes com apenas icone (editar, arquivar, notificar, aprovar, rejeitar) nao tem texto acessivel. Modais usam DialogTitle do Radix (que adiciona aria-labelledby automaticamente — OK), mas botoes de acao dentro dos cards e da toolbar nao tem aria-label. Focus rings dependem do shadcn/ui default (OK para componentes base).
Fix: Adicionar aria-label em todos os Button size="icon" e no DropdownMenuTrigger do menu mobile. Baixa prioridade — nao bloqueia funcionalidade.
Complexidade: JSX (baixo, repetitivo)
P2 SCHEMA SR-008 — FK bidirecional intimacao_id (cross-page, deferred)
Tabelas: decisoes.intimacao_id + intimacoes.decisao_id
Descricao: Redundancia bidirecional. Ambos campos populados pelo pipeline. Ja identificado no Backend Exec v1 como BK-002 deferred. Nao pode ser resolvido sem recriar v_controller_intimacoes_completa (impacta /controller). Registrado novamente para rastreabilidade.
Fix: Recriar view usando JOIN via decisoes.intimacao_id (inverso), depois dropar intimacoes.decisao_id. Executar junto com revisao do /controller.
Complexidade: MIGRATION + CROSS-PAGE (alto)
