🔴 Bugs Críticos — P0
BUG-001 · CRÍTICO · K2
Import duplicado de differenceInCalendarDays e parseISO em NovoDocumentoModal
📁 src/pages/documentos-pendentes/components/modals/NovoDocumentoModal.tsx — Linhas 7 e 34
P0
O arquivo importa as funções differenceInCalendarDays e parseISO do pacote date-fns em dois lugares distintos. Na linha 7, junto a outros utilitários no import principal. Na linha 34, como import separado repetindo exatamente as mesmas funções. Em builds de produção com bundlers otimizados (Vite/Rollup), imports duplicados de named exports podem causar erros de TreeShaking inconsistente, bundle inflation ou — dependendo do transpiler — referências não-únicas que geram comportamentos imprevisíveis em runtime.
// linha 7 — import principal correto import { format, addDays, differenceInCalendarDays, parseISO } from 'date-fns'; import { ptBR } from 'date-fns/locale'; ... (mais 20 imports) ... // linha 34 — IMPORT DUPLICADO (deve ser removido) import { differenceInCalendarDays, parseISO } from 'date-fns'; // FIX: remover linha 34 completamente // ✅ differenceInCalendarDays e parseISO já importados na linha 7
Causa raiz técnica: Provavelmente resultado de refactor incremental — o import original estava na linha 34 e ao reorganizar imports na linha 7, o desenvolvedor não removeu o import antigo. TypeScript tolera imports duplicados sem erro, mascarando o problema.
Fix: Remover linha 34 integralmente. Zero mudança funcional esperada.
🟡 Bugs Médios — P1
BUG-002 · MÉDIO · K2
openNovoModal acessa groups sem checar isLoading — pode retornar null
📁 src/pages/documentos-pendentes/components/DocumentosPendentesPage.tsx — Linha 173
P1
O callback openNovoModal recebe um contexto opcional com clienteId e faz um groups?.find() para resolver o cliente_nome. Se o modal for aberto enquanto isLoading = true (grupos ainda carregando), groups é undefined e a busca retorna null. O modal abre sem nome do cliente, exibindo "null" ou undefined em campos de texto que dependem de cliente_nome.
// DocumentosPendentesPage.tsx:170-175 const openNovoModal = useCallback((ctx?: { pendenciaId?: string; clienteId?: string; ... }) => { ? groups?.find((g) => g.cliente_id === ctx.clienteId)?.cliente_nome ?? null // Bug: groups pode ser undefined quando isLoading=true // Fix: adicionar guard: if (isLoading) return; ou usar ?? '' em vez de ?? null
Impacto: Race condition ao abrir modal via PendenciaItem durante primeira carga. Fix sugerido: Guard no início do callback: if (!groups || isLoading) return;
BUG-003 · MÉDIO · G4
Exportar não foi testado — webhook destino desconhecido
📁 src/pages/documentos-pendentes/components/DocumentosPendentesPage.tsx — Exportar button
P1
O botão "Exportar" foi identificado visualmente no PageHeader mas não foi possível testar o webhook de forma segura sem risco de side effects. O código usa callWebhook() que dispara para uma URL de integração. Sem mock, o teste poderia trigger exportação real de dados sensíveis. Recomenda-se teste isolado em staging.
Impacto: Funcionalidade não validada em produção. Pode não funcionar ou disparar para webhook incorreto após deploy. Ação: Testar em staging com webhook de test.

BUG-004 · BAIXO · UI

BUG-005 · BAIXO · ENV
Browser instabilidade: targetId não respeitado em algumas ações evaluate
Ambiente de teste — OpenClaw Browser Tool
P2
