BUG-010 🔵 P3 useQuery com meta.onSuccess deprecated — anti-pattern TanStack Query v5 hooks/use_empresa_modal.ts ⑧ Independente — rodar por último
Contexto
  • Causa raiz: Dupla chamada useQuery com mesmo key — uma com meta: { onSuccess: undefined }, outra com enabled: false
  • Arquivo: ~/beta-mdflow/src/hooks/use_empresa_modal.ts
  • Anti-pattern: TanStack Query v5 removeu onSuccess de meta. Race condition: se cache vazia, form fica vazio
  • Fix: Usar apenas um useQuery e reagir ao data via useEffect com dependência correta
