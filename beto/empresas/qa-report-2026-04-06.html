Bugs por Prioridade

P0 — Criticos (bloqueante)

P0 Global: Auth redirect desvia usuario de qualquer pagina
Apos 2-5 segundos em qualquer pagina, o usuario e redirecionado automaticamente para /processos, /clientes ou /homepage. O onAuthStateChange no auth_context.tsx dispara hydrateUser em eventos TOKEN_REFRESHED, que pode resetar o estado e causar o redirect via getRedirectPath(). Impacto: Impossivel navegar, usar modais, paginar ou filtrar de forma confiavel. Afeta TODA a aplicacao, nao apenas /empresas.
Arquivo: src/contexts/auth_context.tsx:68-94
Evidencia: Testado 8+ vezes no browser. URL muda sozinha apos 2-5s consistentemente.

P1 — Importantes

P1 DS-001: Cores hardcoded Tailwind em 4 arquivos
Uso de green-500, red-500, blue-500, yellow-500 em vez de CSS variables (hsl(var(--success)), hsl(var(--destructive)), etc). Viola principio P-08 do Design System.
Arquivos afetados: TabOrgaosContent.tsx:79, TabCategoriasContent.tsx:12-14, ModalCategoria.tsx:136,142,160,221, ModalOrgao.tsx:136
Total: 9 ocorrencias.
P1 BK-001: orgaos_restritivos sem coluna deleted_at
A tabela orgaos_restritivos nao possui coluna deleted_at (soft delete). O ModalOrgao.tsx:115 usa .delete() (hard delete) como unica opcao. O useOrgaos.ts nao filtra por deleted_at IS NULL. Viola regra de soft delete obrigatorio do CLAUDE.md.
Tabela: orgaos_restritivos
Fix: Adicionar coluna deleted_at TIMESTAMPTZ + alterar delete para update + filtrar no hook.
P1 UI-001: Delete confirmation usa nome da empresa (nao "EXCLUIR")
O ModalConfirmacaoDelete pede para digitar o nome completo da empresa (ex: "9.0 SOLUCOES POR OTELIO DREBES") para confirmar exclusao. Nomes longos dificultam a confirmacao — seria mais pratico usar a palavra fixa "EXCLUIR".
Arquivo: modals/ModalConfirmacaoDelete.tsx:117-119
Nota: Funcional, mas UX pode ser melhorada. Classificado como P1 por fricao do usuario.
