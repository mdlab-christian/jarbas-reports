Detalhes dos fixes aplicados

P0 BK-001 — rpc_decisoes_criar_execucao etapa 'fila' → 'cadastrada'

Problema: RPC inseria etapa='fila' que nao existe no CHECK constraint (execucoes_etapa_check). Fluxo Decisao → Execucao estava 100% quebrado.

Fix: Rewrite completo da RPC com etapa='cadastrada' (valor valido no CHECK).

INSERT INTO execucoes (..., etapa, ...) VALUES (..., 'cadastrada', ...)
P0 BK-002 — execucoes_eventos tipo CHECK alinhado com frontend

Problema: CHECK constraint tinha tipos que nao existiam no frontend (deposito_efetuado, penhora_realizada, etc). Zero intersecao com EventoExecucao type.

Fix: DROP + re-CREATE CHECK com tipos reais: CRIACAO, DEPOSITO, ALVARA_PEDIDO, ALVARA_EXPEDIDO, LEVANTAMENTO, PAGAMENTO_FINAL, ARQUIVO, ETAPA_ALTERADA, SITUACAO_ALTERADA, AVANCO_ETAPA, OBSERVACAO.

ALTER TABLE execucoes_eventos ADD CONSTRAINT execucoes_eventos_tipo_check
  CHECK (tipo IN ('CRIACAO', 'DEPOSITO', 'ALVARA_PEDIDO', 'ALVARA_EXPEDIDO',
    'LEVANTAMENTO', 'PAGAMENTO_FINAL', 'ARQUIVO', 'ETAPA_ALTERADA',
    'SITUACAO_ALTERADA', 'AVANCO_ETAPA', 'OBSERVACAO'));
P1 BK-006 — Triggers sync campos denormalizados

Problema: Campos cliente_nome, empresa_nome, advogado_nome_oab em execucoes ficavam stale quando fonte mudava.

Fix: 3 triggers AFTER UPDATE OF nome nas tabelas clientes, empresas e users. SECURITY DEFINER + search_path.

trg_sync_execucoes_cliente_nome   ON clientes  (AFTER UPDATE OF nome)
trg_sync_execucoes_empresa_nome   ON empresas  (AFTER UPDATE OF nome)
trg_sync_execucoes_advogado_nome  ON users     (AFTER UPDATE OF nome)
P2 BK-008 — Indices unaccent para busca textual

Problema: Busca textual sem indice, full scan em cliente_nome e empresa_nome. Wrapper IMMUTABLE necessario para indice funcional.

Fix: Funcao immutable_unaccent(text) + 3 indices: cliente_nome_search, empresa_nome_search, processo_numero.

CREATE FUNCTION public.immutable_unaccent(text) RETURNS text
  LANGUAGE sql IMMUTABLE PARALLEL SAFE STRICT
  AS $$ SELECT public.unaccent('public.unaccent', $1); $$;

CREATE INDEX idx_execucoes_cliente_nome_search ON execucoes (immutable_unaccent(lower(cliente_nome)));
CREATE INDEX idx_execucoes_empresa_nome_search ON execucoes (immutable_unaccent(lower(empresa_nome)));
CREATE INDEX idx_execucoes_processo_numero ON execucoes (processo_numero);
