Achados Restantes

P1 BK-012 Webhook execucoes/evento — n8n chama RPC com params errados

O workflow n8n que atende execucoes/evento (nao visivel via API — possivelmente workflow com webhook path dinamico ou catch-all) esta chamando fn_append_evento_execucao com 5 parametros (incluindo p_organizacao_id), mas o overload de 5 params foi dropado no BK-010. A funcao agora aceita apenas 4 params: (p_execucao_id, p_tipo, p_descricao, p_meta).

Impacto: Todos os eventos de execucao via webhook (depositos, levantamentos, alvaras, pagamento final, observacoes) falham com PGRST202.

Fix: Atualizar o workflow n8n para remover o parametro p_organizacao_id da chamada RPC. A funcao agora usa public.get_org_id() internamente. Nota: Como o workflow nao aparece via API, pode precisar de acesso direto ao dashboard n8n.

Alternativa (banco): Recriar o overload com 5 params que ignora o p_organizacao_id e delega para a versao de 4 params.

P2 BK-013 Webhooks nao logam em webhook_requests

Nenhum dos webhooks de execucoes registra a requisicao na tabela webhook_requests. Rastreabilidade zero para debugging.

Impacto: Baixo — nao bloqueia funcionalidade, mas dificulta debug.

Fix: Adicionar no INSERT de webhook_requests em cada workflow n8n.

P2 BK-014 FK processo_origem_id sem ON DELETE

A FK execucoes_processo_origem_id_fkey nao tem ON DELETE definido (usa RESTRICT default). Se um processo for deletado, a execucao fica orfã com erro de FK.

Fix: ALTER TABLE execucoes DROP CONSTRAINT execucoes_processo_origem_id_fkey, ADD CONSTRAINT execucoes_processo_origem_id_fkey FOREIGN KEY (processo_origem_id) REFERENCES processos(id) ON DELETE SET NULL;

-- Recriar overload que aceita p_organizacao_id mas ignora (delega para versao de 4 params)
CREATE OR REPLACE FUNCTION public.fn_append_evento_execucao(
  p_execucao_id uuid, p_tipo text, p_descricao text DEFAULT NULL,
  p_meta jsonb DEFAULT '{}'::jsonb, p_organizacao_id uuid DEFAULT NULL
)
RETURNS jsonb LANGUAGE plpgsql SECURITY DEFINER SET search_path TO 'public', 'pg_temp'
AS $$
BEGIN
  -- p_organizacao_id ignorado — funcao usa get_org_id() internamente
  RETURN public.fn_append_evento_execucao(p_execucao_id, p_tipo, p_descricao, p_meta);
END;
$$;
