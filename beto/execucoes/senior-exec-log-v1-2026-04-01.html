3. Detalhes por Fix

SR-001 (P0) — Deep link valor reais → centavos

Problema: /decisoes envia valor=15117.06 (NUMERIC reais). ExecucoesPage parseava com parseFloat() sem converter para centavos. CurrencyInput esperava BIGINT centavos → R$ 15.117,06 exibido como R$ 151,17.

Arquivo: src/pages/execucoes/ExecucoesPage.tsx:47

- valor_condenacao: searchParams.get('valor') ? parseFloat(searchParams.get('valor')!) : undefined,
+ valor_condenacao: searchParams.get('valor') ? Math.round(parseFloat(searchParams.get('valor')!) * 100) : undefined,

Validação: ✅ Retest confirmou Math.round(parseFloat(val) * 100) na linha 47. Conversão reais→centavos correta.

SR-002 (P1) — DecisaoCard rota inexistente

Problema: navigate(`/execucoes/${decisao.execucao_id}`) apontava para rota que não existe no router. App.tsx só registra /execucoes sem :id → 404.

Arquivo: src/pages/decisoes/components/DecisaoCard.tsx:236

- onClick={() => navigate(`/execucoes/${decisao.execucao_id}`)}
+ onClick={() => navigate(`/execucoes?id=${decisao.execucao_id}`)}

Validação: ✅ ExecucoesPage já trata ?id= nas linhas 30-36, abrindo o modal de detalhe automaticamente. Navegação funcional.

SR-003 (P1) — Acessibilidade: aria-labels

Problema: Apenas 3 aria-labels em 28 arquivos. Botões icon-only, sort headers e overflow menu sem atributo de acessibilidade.

Arquivos: ExecucoesTable.tsx, ExecucoesHeader.tsx

Adicionados (4 novos):

  • ExecucoesTable.tsx:164 — aria-label={`Ordenar por ${label}`} nos sort headers (dinâmico)
  • ExecucoesTable.tsx:312 — aria-label="Ações da execução" no MoreVertical
  • ExecucoesHeader.tsx:56 — aria-label="Mais ações" no overflow menu mobile
  • SR-006 integrado: DateFilter buttons com aria-label dinâmico

Total: 3 → 7 aria-labels (+133%)

Validação: ✅ grep -rn 'aria-label' src/pages/execucoes/ retorna 7 ocorrências. Todos os botões icon-only cobertos.

SR-004 (P2) — Export modal no dashboard

Problema: Na tab Dashboard, opção "Aba atual com filtros" ficava selecionável mas mostrava "— execuções" (sem count). Dashboard não tem listagem filtrada.

Arquivo: src/pages/execucoes/modais/ModalExportarExecucoes.tsx

+ const isDashboard = activeTab === 'dashboard';
+ const defaultEscopo = hasBulk ? 'selecionados' : isDashboard ? 'todas' : 'tab_atual';
+ <RadioGroupItem value="tab_atual" id="esc-tab" disabled={isDashboard} />
+ <Label ... className={cn('...', isDashboard && 'text-muted-foreground/50')}>

Validação: ✅ Quando activeTab === 'dashboard': radio desabilitado + texto esmaecido + default é "todas".

SR-006 (P2) — DateFilter aria-labels

Problema: Botões trigger do date picker sem aria-label — screen readers só viam o ícone do calendário.

Arquivo: src/pages/execucoes/components/ExecucoesFilterBar.tsx:281

- <Button variant="outline" className={cn(...)}>
+ <Button variant="outline" aria-label={`Filtrar por ${placeholder.toLowerCase()}`} className={cn(...)}>

Validação: ✅ Placeholder é "Data inicial" e "Data final", gerando aria-label="Filtrar por data inicial" e "Filtrar por data final".

SR-005 (P2) — 5 count queries → RPC única ⏭️ Skipped

Problema: Hook useExecucoesTabCounts dispara 5 queries paralelas para contar registros por tab.

Motivo do skip: Requer criação de nova RPC (fn_get_execucoes_tab_counts) + migration + refactor do hook. Impacto em performance é baixo (5 queries HEAD paralelas são rápidas). Classificado como otimização futura — não bloqueia produção.

Recomendação: Implementar quando houver latência perceptível ou como parte de um ciclo de otimização de performance.
