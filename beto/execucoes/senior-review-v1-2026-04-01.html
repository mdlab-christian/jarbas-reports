3. Achados

ID Tipo P Título Arquivo Complexidade
SR-001 FRONTEND P0 Deep link valor unit mismatch — reais recebidos como centavos ExecucoesPage.tsx:47 JSX (1 linha)
Descrição: Quando o usuário clica "Nova Execução" em /decisoes, o deep link passa valor=15117.06 (NUMERIC reais da tabela decisoes). Em ExecucoesPage.tsx:47, parseFloat("15117.06") retorna 15117.06, que é passado diretamente ao CurrencyInput. Porém, CurrencyInput trata o valor como centavos (padrão BIGINT centavos do MdFlow). Resultado: R$ 15.117,06 é exibido como R$ 151,17 no modal de nova execução.
Evidência: decisoes/index.tsx:90 envia valor: String(d.valor_condenacao) onde d.valor_condenacao é NUMERIC reais. ExecucoesPage.tsx:47 faz parseFloat(searchParams.get('valor')!) sem multiplicar por 100. execucao.shared.tsx confirma: valor_condenacao: number | null; // centavos
Fix: parseFloat(searchParams.get('valor')!) * 100 em ExecucoesPage.tsx:47. Alternativa: converter no lado de /decisoes antes de enviar.
SR-002 FRONTEND P1 DecisaoCard navega para rota inexistente /execucoes/:id decisoes/DecisaoCard.tsx:236 JSX (1 linha)
Descrição: navigate(`/execucoes/${decisao.execucao_id}`) aponta para uma rota que não existe no router (App.tsx:187 só tem /execucoes sem :id). Resultado: clicar em "Ver Execução" no card de decisão leva a uma página 404 (NotFound).
Evidência: App.tsx:187-188 confirma apenas <Route path="/execucoes" element={<ExecucoesPage />} />. Nenhuma rota com parâmetro :id.
Fix: Trocar para navigate(`/execucoes?id=${decisao.execucao_id}`) e implementar auto-open do modal quando ?id= está presente.
SR-003 A11Y P1 Acessibilidade mínima — apenas 3 aria-labels em 28 arquivos ExecucoesTable.tsx + ExecucoesFilterBar.tsx JSX (múltiplos)
Descrição: Somente 3 aria-labels encontrados: "Selecionar todos", "Selecionar {nome}" e "Remover filtro {label}". Faltam aria-labels em: botão MoreVertical (ações por linha), botões de sort nas colunas, botões de ícone no header, DateFilter trigger, botões de ação nos drawers (ícone-only em mobile), e botões de paginação.
Evidência: grep -rn 'aria-label' src/pages/execucoes/ retornou apenas 3 resultados. Labels de formulário (<Label>) estão presentes nos drawers/modais (29 ocorrências), mas ausentes em ExecucoesTable, ExecucoesTabs, ExecucoesHeader.
Fix: Adicionar aria-labels em todos os botões icon-only, sort headers, e controles interativos sem texto visível.
SR-004 FRONTEND P2 Export modal mostra "—" no filteredCount da tab Dashboard ModalExportarExecucoes.tsx:175 JSX
Descrição: Quando na tab Dashboard, filteredCount não é passado (ou é undefined), e o info text mostra "— execuções com filtros aplicados". A tab Dashboard não tem listagem paginada, então o count não é calculado.
Fix: Desabilitar opção "Aba atual" quando activeTab === 'dashboard', ou calcular o count total.
SR-005 PERF P2 5 queries paralelas para contagem de tabs — poderia ser 1 RPC hooks/useExecucoesTabCounts.ts RPC
Descrição: Hook dispara 5 queries separadas ao Supabase para contar registros de cada tab (favoráveis, desfavoráveis, parciais, ag. alvará, pagos). Cada query faz .select('id', { count: 'exact', head: true }). Funciona, mas são 5 round-trips desnecessários.
Fix: Criar RPC fn_get_execucoes_tab_counts(p_organizacao_id) que retorna todas as contagens em uma query.
SR-006 A11Y P2 DateFilter trigger buttons sem aria-label ExecucoesFilterBar.tsx JSX
Descrição: Os botões de filtro de data (data início, data fim) não possuem aria-label descritivo. Screen readers anunciariam apenas o conteúdo visual (ícone do calendário).
Fix: Adicionar aria-label="Filtrar por data inicial" e aria-label="Filtrar por data final".
