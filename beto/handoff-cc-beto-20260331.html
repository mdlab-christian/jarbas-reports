🔄 4. Pipeline Completo de Trabalho

1. core-page
→
2. page-qa-tester
→
3. page-executor
→
4. page-finalize

Etapa 1 — core-page (absorver contexto)

  • Gustavo manda áudio descrevendo a página, bugs, comportamento esperado
  • Beto transcreve (Whisper) e salva em ~/.openclaw/workspace-beto/mdflow-pages/{pagina}.md
  • Inclui: o que é a página, fluxo, componentes, bugs conhecidos, pendências por prioridade

Etapa 2 — page-qa-tester (auditoria)

  • Lança subagente com browser — acessa https://mdflow.app
  • Testa TODOS os componentes da página: abas, drawers, modais, filtros, formulários
  • Valida no Supabase (curl) que as ações refletem no banco
  • Gera HTML de QA report em ~/jarbas-reports/beto/{pagina}/qa-report-{DATA}.html
  • Notifica Gustavo via Telegram com resumo + link

Etapa 3 — page-executor (gerar missão CC em loop)

  • Lê o QA report e extrai bugs por prioridade (P0 → P1 → P2)
  • Gera HTML de execução com 1 prompt por bug (nunca agrupar)
  • Cada prompt tem: caminho completo dos arquivos, contexto, solução proposta
  • Inclui mega prompt final: CC gera HTML de resultado + git commit obrigatório
  • LOOP: Gustavo manda HTML de resultado → Beto analisa → gera v{N+1} com pendentes
  • Encerra quando todos os bugs são ✅ e commit confirmado

Etapa 4 — page-finalize (atualizar KB)

  • Após todos os bugs resolvidos: atualizar o core da página com o status final
  • Atualizar mdflow-core-v4-2026-03-30.html com o novo estado da página
  • Git push do jarbas-reports

Como o Claude Code recebe as missões

cd ~/beta-mdflow && claude --permission-mode bypassPermissions --print \
"[prompt gerado pelo page-executor]"

Para tarefas longas (background):

cd ~/beta-mdflow && claude --permission-mode bypassPermissions --print \
"[prompt]

When completely finished, run:
openclaw system event --text 'Done: {pagina} execution-v{N} — {resumo}' --mode now" &
