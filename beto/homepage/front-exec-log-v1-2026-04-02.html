FRONT-004 — Análise detalhada

Diagnóstico: dados insuficientes — não é bug de código

A RPC get_homepage_ranking(p_user_id, p_org_id) está logicamente correta. O problema é que processos.advogado_id contém UUIDs sintéticos de seed (a1000001-0000-4000-8000-000000000001, etc.) que não correspondem a nenhum users.id real. 7.487 de 7.489 processos ativos têm advogado_id preenchido, mas todos apontam para usuários inexistentes na tabela users.

Ação recomendada: migração de dados para mapear advogado_id sintéticos → users.id reais. Decisão do CTO.
