Achados Relevantes

FT-C05 — Charts não presentes em ViewAdmin (esperado, mas documentar)

A ViewAdmin não renderiza HomepageChartJuridico nem HomepageChartComercial. Isso é correto por design — ViewAdmin foca em dados consolidados via HomepageGestorExecutiveBlock. No entanto, o spec menciona "gráficos Recharts" como componente relevante, o que pode causar confusão. Recomendação: confirmar com CTO se ViewAdmin deve ter algum chart de tendência.

FT-C06 — KpiCard hover: presente apenas em ViewJuridico/Comercial, não em ViewAdmin

Os HomepageKpiCard com hover state só aparecem em ViewJuridico e ViewComercial. Em ViewAdmin, os blocos executivos (HomepageGestorExecutiveBlock) não têm hover/navegação. Isso é comportamento correto, mas o teste de hover de KpiCard é N/A para ViewAdmin. Tested via code review — implementação correta.

FT-A03 — Sidebar active state confirmado via inline style

O botão Homepage (índice 0 do nav) recebe background: hsl(var(--primary) / 0.1) via inline style quando a rota é /homepage. Funciona corretamente. Não usa classes CSS para estado ativo — usa inline style programático (padrão da implementação em app_sidebar.tsx).

Observação: agent-browser set viewport triggerando navegação

Durante os testes, o comando agent-browser set viewport frequentemente acionava cliques involuntários nos botões do sidebar durante o resize, causando navegação para /jurimetria, /usuario etc. Isso é uma limitação do ambiente de teste, não um bug da homepage. Os testes de responsividade foram validados via screenshots capturadas antes do resize.
