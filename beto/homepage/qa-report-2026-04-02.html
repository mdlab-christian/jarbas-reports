Confirmação browser Console warning: "[homepage] ranking error: column p.responsavel_id does not exist" — aparece 4x (uma por request, incluindo retries).
Solução — já pronta Aplicar a migration pendente. A função corrigida usa processos p ON p.advogado_id = u.id (campo correto confirmado no schema: advogado_id UUID).

BUG-HP-R02 get_homepage_gestor_comercial — taxa_conversao retorna 200%–241.6% P0 — Regressão
Arquivo da migration corretiva supabase/migrations/20260331100002_fix_homepage_taxa_conversao.sql
Valores observados por período 
hoje:  null (0 clientes no período → divisão por zero → NULLIF correto)
7d:    200.0  (8 clientes criados, 16 processos → 16/8 × 100 = 200%)
30d:  241.6  (43 leads, 11902 conversoes)
Causa raiz A fórmula atual usa COUNT(DISTINCT p.id) / COUNT(DISTINCT c.id) × 100 — contagem de processos dividida por clientes. Como um cliente pode ter múltiplos processos, o resultado ultrapassa 100%. O valor exibido no browser foi confirmado: Taxa Conversao: 200.
Impacto GestorExecutiveBlock "Comercial" exibe valor de taxa de conversão absurdo (200%+) para todo usuário com acesso_gestor. Dado enganoso para gestão executiva.
Solução — já pronta A migration corretiva calcula: clientes_que_têm_processo / total_clientes × 100 usando COUNT FILTER (WHERE EXISTS(...)). Aplicar migration pendente. 
-- Arquivo: supabase/migrations/20260331100002_fix_homepage_taxa_conversao.sql

Bugs Novos — Ciclo v2

BUG-HP-N01 Overflow horizontal em 375px — sidebar header causa scroll lateral P1 — UI/Responsividade
Arquivo Layout principal da aplicação (header fixed top-0) — não é específico da homepage, mas manifesta em 375px.
Detalhe técnico 
Elementos com overflow:
HEADER fixed: scrollWidth 392 > viewport 375
MAIN  .flex-1: scrollWidth 401 > viewport 375
Causa provável: header fixed com min-width ou padding não responsivo
Impacto Em telas 375px (iPhone SE, iPhone 12 mini) a página tem scroll horizontal indesejado. Itens podem ficar parcialmente cortados. Ocorre em todas as páginas que usam o mesmo layout base.
Sugestão de fix Verificar o componente de header (AppHeader/Topbar) — adicionar max-w-full overflow-hidden ou garantir que elementos internos não ultrapassem 100vw. Investigar se há elemento com width fixo em pixels que quebra em mobile.
BUG-HP-N02 HomepageGestorExecutiveBlock exibe label "Taxa Conversao" sem formatação (camelCase/underscore) P2 — UX
Arquivo src/components/homepage/homepage_gestor_executive_block.tsx — linha 29
Código atual 
key.replace(/_org$/, '').replace(/_/g, ' ')
Resultado observado no browser Labels aparecem em minúsculas corridas: "taxa conversao", "decisoes periodo", "peticoes geradas", etc. Não há capitalização de palavras — apenas substituição de underscore por espaço.
Impacto Labels do bloco executivo parecem inacabadas. Afeta a percepção profissional da ferramenta para gestores/admins que veem esses KPIs.
Sugestão de fix — 1 linha 
// Adicionar capitalize após replace:
key.replace(/_org$/, '').replace(/_/g, ' ')
  .replace(/\b\w/g, c => c.toUpperCase())
Ou criar um mapa de labels amigáveis por chave (mais robusto).
