Análise dos False Positives

🔵 P0-03r — Provisioning (False Positive)

O reteste reportou que o provisioning "falha silenciamente". Verificação via curl com anon key: 

curl -X POST
  "https://qdivfairxhdihaqqypgb.supabase.co/rest/v1/rpc/mdchat_provision_default_channels"
  -H "apikey: [ANON_KEY]" -H "Authorization: Bearer [ANON_KEY]"
  -d '{"p_org_id":"55a0c7ba-...", "p_user_id":"bc557356-..."}'

# Resultado: 5 canais retornados — RPC SECURITY DEFINER funciona corretamente

O fix da v1 (verificar mdchat_channel_members em vez de mdchat_channels) foi a correção certa — garante que o provisioning é verificado por membership do usuário, não por existência de canais da org. O reteste pode ter rodado antes do fix ser publicado, ou estava num estado transiente.

🔵 SCHEMA-01 — created_at / member_count ausentes no RPC (Mitigado na v1)

O RPC mdchat_get_channels_with_info não retorna os campos created_at e member_count. A v1 já aplicou fallbacks defensivos em mdChatService.ts: 

created_at: (row.created_at as string) || new Date().toISOString(),
member_count: Number(row.member_count) || 0,

Esses fallbacks garantem que o frontend nunca recebe undefined nesses campos. Os guards de isNaN() adicionados no NEW-01 complementam a proteção. Para corrigir na raiz: atualizar o RPC no banco para retornar esses campos (fora do escopo desta rodada).

Correções Aplicadas (v2)

NEW-01 — Guards isNaN() em datas

Causa raiz: new Date(dateStr) retorna Invalid Date quando dateStr é null, undefined, "" ou string malformada. date-fns format() lança RangeError: Invalid time value ao receber esse objeto.

Solução: Guards if (isNaN(d.getTime())) return '' (ou return null) antes de qualquer chamada a format().

ArquivoLocalFix
MdChatMessageBubble.tsx formatTimestamp() Guard if (isNaN(d.getTime())) return '' após new Date(dateStr)
MdChatDateSeparator.tsx render Guard if (isNaN(d.getTime())) return null após new Date(date)
MdChatChannelHeader.tsx archived_at render Condição ampliada: channel.archived_at && !isNaN(new Date(channel.archived_at).getTime())

NEW-01e — MdChatMembersTab: useAuth destructure

Causa raiz: useAuth() retorna { user, ... } onde user é o objeto com os campos do usuário. O componente estava tentando fazer const { organizacao_id } = useAuth() — mas organizacao_id não existe no nível do hook, só em user.organizacao_id. Isso resultava em orgId = undefined, query desabilitada, e tab de membros sempre vazia.

Solução: const { user } = useAuth(); const orgId = user?.organizacao_id; 

// Antes (incorreto):
// const { organizacao_id } = useAuth(); 
// const orgId = organizacao_id;

// Depois (correto):
const { user } = useAuth();
const orgId = user?.organizacao_id;
