Detalhes dos Fixes

POLISH-001 — member_count via batch query
Antes
RPC mdchat_get_channels_with_info não retorna member_count. Fallback || 0 → header sempre exibia "0 membros".
Depois
Após o RPC, uma batch query em mdchat_channel_members conta membros por canal. Resultado mergeado nos channels antes de retornar.
POLISH-002 — stopPropagation no search input
Antes
Input de busca propagava keydown para handlers globais. Letras podiam acionar atalhos de navegação.
Depois
onKeyDown={'{'}e => e.stopPropagation(){'}'} no Input. Teclado não vaza para fora do campo.
POLISH-003 — EmojiPicker com busca
Antes
24 emojis hardcoded em grid sem filtro. Sem tooltip, sem como encontrar emoji específico.
Depois
48 emojis com dicionário de nomes em pt-BR. Input de busca (auto-focus) com stopPropagation. Tooltip no hover. Empty state "Nenhum emoji encontrado".
POLISH-004 — @mention nomes compostos
Antes
@João Silva → regex /@([A-Za-z...]+)/g só matchava @João. "Silva" ficava sem highlight.
Depois
Inserção usa non-breaking space: @João\u00A0Silva . Regex expandido para capturar \u00A0 no nome. Display: espaço normal. Detecção: determinística.
POLISH-005 — Responsividade mobile
Antes
3 colunas fixas (280px + flex + 360px) sem breakpoints. Em mobile 375px: sidebar overflow, layout quebrado.
Depois
Sidebar: hidden md:flex — oculta em mobile, drawer com overlay ao abrir. Botão PanelLeft no header (visível apenas em md:hidden). Reader Panel: absolute inset-0 em mobile (fullscreen overlay), side panel em desktop. Fechar sidebar ao selecionar canal em mobile.
POLISH-006 — noTruncate no ThreadView
Antes
Mensagem pai na Thread sempre truncada em 500 chars. Contexto da conversa cortado sem motivo.
Depois
noTruncate?: boolean prop em MdChatMarkdownBody e MdChatMessageBubble. ThreadView passa noTruncate apenas para a mensagem pai. Mensagens da lista principal mantêm truncamento em 500 chars.
