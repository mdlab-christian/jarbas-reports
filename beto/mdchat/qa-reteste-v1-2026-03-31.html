P0

REGRESSAO NOVA: Reader Panel crash com RangeError: Invalid time value

Ao clicar no ícone de membros (Users icon) no header do canal — que abre o MdChatReaderPanel com tab 'members' — a página inteira quebra com "Algo deu errado" e error boundary captura um RangeError: Invalid time value.

RangeError: Invalid time value at lt (format-DDn3-cw9.js:1:12201) at Wa (index-BBI7PbAq.js:4:10038)

Causa raiz provável: O canal "adas" criado pelo provisioning manual via service_role tem setor_id inválido. Outro suspeito: MdChatChannelHeader.tsx:50 tem format(new Date(channel.archived_at), ...) com guard {channel.archived_at && ...} — mas se archived_at chega como string vazia "" em vez de null, o guard passa e new Date("") = Invalid Date.
Ação necessária: Corrigir antes de próximo release — bloqueia acesso ao painel de membros/sobre/arquivos/thread.

P0-03

BUG LEGADO CONFIRMADO: Provisioning falha silenciosamente com anon key

Na primeira carga, o usuário Gustavo (bc557356) teve 0 memberships e o hook useMdChatProvisioning chamou o RPC mdchat_provision_default_channels via anon key do frontend. O RPC executou mas não adicionou memberships — o catch {} swallou qualquer erro silenciosamente. A tela exibiu "Nenhum canal disponível".

A chamada manual via service_role corrigiu o estado. Na segunda carga, os canais apareceram. O provisioning RPC precisa ser SECURITY DEFINER com bypass de RLS — ou o frontend precisa mostrar erro em vez de falhar silenciosamente.

Achado Adicional: RPC mdchat_get_channels_with_info

O RPC não retorna os campos created_at e member_count, mas o serviço mdChatService.ts tenta lê-los:

// mdChatService.ts:26 — created_at não vem do RPC created_at: (row.created_at as string) || new Date().toISOString(), // → sempre fallback // mdChatService.ts:33 — member_count não vem do RPC member_count: Number(row.member_count) || 0, // → sempre 0

Impacto: Header do canal sempre mostra "0 membros". Tab About sempre mostra data atual de hoje como "Criado em". Os fixes P0-02 e P1-01 protegem contra crash, mas os dados exibidos são incorretos.
Recomendação: Adicionar created_at, member_count ao retorno do RPC.
