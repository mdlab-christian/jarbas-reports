❌
B1 — Drag-and-Drop do Kanban NÃO funciona
Em CrmKanban.tsx > DraggableLeadCard: dragStartedRef.current nunca é setado para true. Após drag visual, onClick dispara e abre o drawer indevidamente. DnD funciona visualmente mas não persiste a mudança de coluna.
Fix: useEffect(() => { if (isDragging) dragStartedRef.current = true; }, [isDragging]);
❌
B2 — Modal Distribuição mostra dados do lead ERRADO
Em ModalDistribuicao.tsx: staleTime: 60 * 1000 faz React Query retornar cache desatualizado quando CPFs duplicados existem no banco. Modal mostra dados do LEAD FIX3 em vez do lead atual.
Fix: mudar staleTime: 60 * 1000 → staleTime: 0
❌
B3 — Auditoria completamente não funcional
Serasa Premium: marcado como disponivel: true mas retorna erro "bureau não disponível nesta conta ApiFull". CPF Completo (ApiFull): retorna "requisicao invalida: Não houve resposta da API!". Edge function de processamento ausente ou token não configurado.
Fix B3a: BUREAUS[serasa].disponivel = false. Fix B3b: Verificar token ApiFull nas secrets Supabase + deploy da edge function.
🟠
P1 — Botão "Triagem" no drawer não move o lead
Em CrmPage.tsx: case 'triagem' abre ModalTriagem que valida coluna_crm === 'triagem' e fecha com erro pois lead está em outra coluna. Botão nunca move o lead.
Fix: case 'triagem': setChecklistLeadId(leadId) → abre ModalDistribuicao primeiro
🟠
P2 — Toggle "Aguardando resposta" ativo por padrão
Filtros são persistidos no localStorage. Se usuário ativou awaiting_human: true em sessão anterior, fica ativo na próxima sessão.
Fix: return { ...DEFAULT_FILTERS, ...JSON.parse(raw), awaiting_human: null };
🟠
P3 — WhatsApp salvo sem máscara
ModalNovoLead salva normalizeDigits(data.whatsapp). Ao editar, campo mostra "51999887766" sem máscara de formatação.
Fix: form.reset({ ...leadData, whatsapp: applyPhoneMask(leadData.whatsapp) })
🟠
P5 — Sem botão "Cadastrar novo advogado" no ModalTriagem
Quando lista de advogados está vazia, usuário fica preso sem poder cadastrar um novo direto pelo modal.
Fix: Adicionar Button variant=link → navigate('/configuracoes?tab=advogados')
✅
Funcionando: Criação de leads, Upload docs, Gestão de restrições, ClienteDrawer completo, Modo Lista
Testado em 2026-03-30. Login, cadastro de lead, upload de 5 PDFs com categorias, adicionar restrições, modo kanban visual, modo lista.
