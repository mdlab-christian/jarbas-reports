📡 Padrão de Comunicação com Backend

React Query (principal)

Todas as queries usam useQuery / useMutation com chaves específicas. staleTime: 5min, gcTime: 30min global. Invalidate por queryKey após mutations.

Supabase Realtime

Subscriptions em páginas críticas: Acordos, Relatórios, MdChat, CRM (quando ativo). Usa postgres_changes para auto-invalidate queries.

n8n Webhooks

callWebhook(rota, payload) + handleWebhookAction. Usado em: Marketing (insights), MdZap, Parceiros, OlivIA, Distribuição. Timeout configurável (60s default).

Fluxo de autenticação

  1. Usuário preenche email + senha
  2. LoginForm chama supabase.auth.signInWithPassword
  3. Sucesso → AuthContext atualiza → redirect para / (dashboard padrão do perfil)
  4. Erro → toast com mensagem específica (credenciais inválidas, email não confirmado, etc.)
  5. Reset de senha → abre fluxo de /reset-password com magic link

Guards de rota

  • PageGuard — verifica user.perfil e bloqueia rotas não autorizadas
  • Usuário já logado em /login → redirect automático para home do perfil
  • Perfis: admin, juridico, atendente, financeiro, comercial, lite, parceiro

  • Toggle show/hide senha funciona
  • ShaderBackground não trava o browser (performance)
  • Typewriter roda as frases sem congelar
  • Rota /admin mostra AdminLabel (badge vermelho)
  • Usuário já autenticado acessando /login → redireciona (não mostra tela de login novamente)
