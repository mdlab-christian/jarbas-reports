Overview & Stack

MdFlow — SaaS Jurídico Multi-tenant | Supabase: qdivfairxhdihaqqypgb

Frontend

  • React 18 + TypeScript
  • Vite (build)
  • Tailwind CSS + shadcn/ui
  • React Router v6
  • TanStack Query (server state)
  • Zustand 5 (global state)
  • Recharts (gráficos)
  • React Hook Form + Zod
  • Lucide React (icons)
  • Sonner (toasts)

Backend

  • Supabase (auth + db + storage + realtime)
  • PostgreSQL + pgvector
  • n8n Railway (automações)
  • Edge Functions Supabase
  • RPCs via supabase.rpc()
  • Realtime via postgres_changes
  • Storage para exports e documentos

Design System v5

  • Background: #12100e (Warm Charcoal)
  • Primary: #20c997 (Teal)
  • Radius: rounded-xl cards, rounded-lg modais
  • Font display: Inter / system
  • Sombra: shadow-sm border border-border
  • Multi-tenant: organizacao_id em tudo
  • Soft delete: deleted_at IS NULL

Padrões Globais

Webhooks (callWebhook) 
// lib/webhook.ts
callWebhook(endpoint, payload)
// Payload base: { organizacao_id, user_id, request_id, timestamp, data }
// Resposta: { success, message, error_code?, action?, data? }
// Pós-webhook: handleWebhookAction(action, data, { queryClient, navigate, toast })

Exemplos: processos/export, processos/arquivar, processos/arquivar_lote, clientes/excluir

Modais (padrão shadcn Dialog)
  • Sempre Dialog do shadcn/ui
  • Props padrão: open: boolean, onClose/onOpenChange, onSuccess
  • Loading state interno com isSubmitting + disabled no botão submit
Drawers (Sheet lateral)
  • Sheet shadcn/ui, side="right", hideCloseButton
  • ClienteDrawer: compartilhado entre CRM, Clientes e Distribuição via prop context
  • context=crm: w-[72vw] min-w-[720px] max-w-[1100px]
  • context=clientes: w-[65vw] min-w-[600px] max-w-[900px]
Error Boundaries
  • SectionErrorBoundary ou TabErrorBoundary envolvendo seções críticas
  • Fallback: card com mensagem de erro em text-muted-foreground
  • CRM e Processos usam SectionErrorBoundary inline (classe no próprio arquivo)

Login

Route: /login | Entry: src/pages/login/index.tsx
🐛 Bug ativo: Após digitar credenciais e clicar "Entrar", o login não navega. O usuário sempre precisa dar Ctrl+R para entrar. Pendência baixa, mas resolve algo importante.

Fluxo de Login

login_form.tsx → use_login_flow.ts → auth_context.tsx → auth_service.ts
  → supabase.auth.signInWithPassword(email, password)
  → supabase.rpc('fn_login_context')  [timeout: 12s]
  → flushSync → setUser(profile) + navigate(redirectPath)  ← FALHA AQUI

Hipóteses da Causa

🔴
1. Race condition: manualLoginRef vs onAuthStateChange
manualLoginRef suprime eventos do onAuthStateChange. Se flushSync + navigate falhar silenciosamente, o navigate nunca acontece. Reload aciona INITIAL_SESSION → funciona.
🟡
2. flushSync em React 18 Concurrent Mode
flushSync dentro de async pode não funcionar como esperado — ProtectedRoute vê user=null e redireciona de volta ao /login.
🟡
3. Dois clientes Supabase distintos
google_oauth_button.tsx usa @/lib/supabase; demais usam @/integrations/supabase/client. Se forem instâncias diferentes, onAuthStateChange pode não capturar eventos.
🟡
4. processingRef travado
Se processingRef.current ficar true de tentativa anterior, hydrateUser retorna sem setar user. Reload limpa o estado.
Caminho de Investigação
  • Verificar se @/lib/supabase e @/integrations/supabase/client exportam o mesmo singleton
  • Adicionar logs pós-flushSync e pós-navigate no handleSignIn
  • Verificar timing do processingRef — logar quando entra/sai
  • Testar sem flushSync (usar setState + useEffect para navigate)
  • Monitorar console do browser durante login com falha
