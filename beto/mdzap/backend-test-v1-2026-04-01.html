Auditoria de RPCs

P1 fn_mdzap_conversas — lead_awaiting_human hardcoded false
A coluna leads.aguardando_humano não existe no banco. A RPC retorna false fixo com comentário "leads.aguardando_humano não existe". A aba "Aguardando humano" do MdZap nunca mostrará leads reais. 
-- Fix: criar campo na tabela leads e atualizar a RPC
ALTER TABLE leads ADD COLUMN aguardando_humano BOOLEAN DEFAULT false;
-- Então atualizar fn_mdzap_conversas para usar l.aguardando_humano
P1 fn_mdzap_stats — 2 KPIs hardcoded como 0
aguardando_humano e tempo_medio_resposta_min sempre retornam 0. O dashboard de stats exibe métricas falsas para o usuário. 
-- Mesma dependência de leads.aguardando_humano
-- tempo_medio_resposta: requer lógica de análise de timestamps entre mensagens
P1 fn_mdzap_listar_membros — retorna usuários deletados/inativos
Nenhum filtro por deleted_at ou status ativo. A lista de membros para transferência de conversa pode incluir ex-funcionários. 
-- Fix na RPC:
WHERE u.organizacao_id = p_organizacao_id
  AND u.deleted_at IS NULL  -- adicionar
ORDER BY u.nome;
P2 fn_mdzap_conversas — sem paginação
Carrega todas as conversas de uma vez. Com 6 conversas OK; ao crescer para centenas a performance vai degradar. Recomendado adicionar p_limit / p_offset.
P2 fn_mdzap_stats — 494ms (lento)
4 subqueries separadas no mesmo SELECT. Consolidar em um único FILTER reduz de ~500ms para ~100ms. 
-- Fix: usar FILTER em vez de subqueries
SELECT
  COUNT(*) FILTER (WHERE TRUE) as total_conversas,
  COUNT(*) FILTER (WHERE status = 'ativa') as conversas_ativas,
  COUNT(*) FILTER (WHERE TRUE) as numeros_total,
  COUNT(*) FILTER (WHERE status = 'connected') as numeros_conectados
FROM whatsapp_conversas CROSS JOIN ...

💡 Inconsistência: parte do código usa org_id(), parte usa get_org_id()

✅ Constraints TEXT CHECK em todos os campos de status/tipo/direcao (nunca ENUM)

✅ FKs com ON DELETE CASCADE/SET NULL corretos

✅ Índices bem estruturados — organizacao_id indexado em todas as tabelas principais

✅ Triggers set_updated_at nas tabelas que precisam

✅ RLS isolamento testado — org fictícia retorna 0 registros

✅ INSERT com WITH CHECK funcionando corretamente
