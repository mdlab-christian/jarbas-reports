P2 · BAIXO BUG-09 useMdzapSmartReplies — cache via Map fora do React (memory leak potencial) Independente
Causa raiz

O hook mantém const cache = new Map() no escopo do módulo (fora do ciclo React). O cache nunca é limpo entre sessões — em uso intenso com muitas conversas acumula memória indefinidamente.

Arquivo
  • ~/beta-mdflow/src/hooks/mdzap/useMdzapSmartReplies.ts
Prompt para Claude Code
No arquivo ~/beta-mdflow/src/hooks/mdzap/useMdzapSmartReplies.ts, corrija o cache global que causa memory leak.

Situação atual: existe um `const cache = new Map()` declarado fora do hook (escopo de módulo), que nunca é limpo.

Solução preferida — migrar para React Query:
1. Importe useQuery de @tanstack/react-query
2. Remova o Map global
3. Use useQuery com:
   - queryKey: ['mdzap', 'smart-replies', conversaId]
   - queryFn: a mesma função assíncrona que já existe
   - staleTime: 5 * 60_000  (5 minutos — smart replies mudam pouco)
   - enabled: !!conversaId

Se a migração para React Query for muito invasiva (ex: o hook retorna múltiplos valores que outros componentes dependem), use a solução alternativa:
- Mantenha o Map mas limite o tamanho: if (cache.size > 100) cache.clear()
- Adicione limpeza no useEffect de unmount: return () => cache.delete(conversaId)

Mantenha a assinatura do hook compatível com os consumidores atuais (não quebre props/retornos).
Não altere outros hooks ou componentes.
