P1 — Importantes

P1

J6 — Esc não fecha painel lead no contexto MdzapChatPanel

Em /mdzap (AppLayout), pressing Escape não fechou o painel de informações do lead. O hook useMdzapKeyboardShortcuts está wired no MdzapChatPanel, mas leadPanelOpen é state local — possível de o painel estar num Drawer mobile que não responde ao handler. Testar no /mdzap/app: Esc deve fechar painel via setLeadPanelOpen(false) do MdzapAppPage.

P1

I1 — Submit do form de criação de template não dispara via click normal do Playwright

Clicar no botão "Criar template" via agent-browser click ou find não triggava o submit React. Necessário usar eval document.querySelector('button[type="submit"]').click(). Possível issue com event.isTrusted ou preventDefault no form. Não afeta usuários reais mas indica possível problema com synthetic events.

P2

A9 — Scroll da lista de conversas não detectado via JavaScript

document.querySelector('[class*="overflow-y-auto"]').scrollHeight retorna 0 quando a lista está na aba Fila (vazia). Com conversas na aba Todas, a lista preenche mas scrollHeight=clientHeight (não overflow). Com menos de 8 conversas o scroll não é necessário. Não é um bug funcional mas o elemento de scroll pode não estar corretamente dimensionado.

P2

F7 — Tooltip no botão Enviar não verificado (só texto abaixo do input)

"Nenhuma instância WhatsApp conectada" aparece como texto abaixo do input, não como tooltip no botão Send. O código tem title= mas hover de mouse não foi possível testar via agent-browser. Usuário pode não ver tooltip rapidamente.

P2

Pendência conhecida (R4-BUG-07) — Spec diz J/K, código usa ArrowDown/Up como primário

Já registrada na KB: useMdzapKeyboardShortcuts.ts linha 35-36 tem tanto ArrowDown quanto j implementados. Comportamento correto mas spec não clarifica se j/k devem ser atalhos primários. Confirmar com CTO.
