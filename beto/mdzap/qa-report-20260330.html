🔴 Bugs Críticos

BUG-01 CRÍTICO Checklist B5 (tab Gestão) — Tab não acessível via selector MdzapChatPanel.tsx
Descrição

A tab "Gestão" do MDZap não responde a seletores CSS padrão (text=Gestão) no ambiente de teste. Timeout de 8s na tentativa de clique. O componente existe no código (MdzapGestaoPanel) e está registrado no painel de tabs, mas o localizador falha consistentemente.

Causa raiz suspeita

O caractere especial "ã" (U+00E3) pode causar problemas de encoding no seletor Playwright. Alternativa: o componente pode ter lazy-load ou conditional render que impede o selector de funcionar no estado inicial.

Snippet
// MdzapChatPanel.tsx — render condicional das tabs
{activeTab === 'gestao' && <MdzapGestaoPanel organizacaoId={orgId} />}
// Selector falha: text=Gestão — timeout 8000ms
Impacto

⚠️ QA parcial da tab Gestão. Analisado via código-fonte: componente existe e renderiza corretamente quando acessado manualmente. Risco: nenhum bug funcional identificado no código.

BUG-02 CRÍTICO useMdzapMensagens — queryKey sem organizacaoId (vazamento cross-org) src/hooks/mdzap/useMdzapMensagens.ts:14
Descrição

O hook useMdzapMensagens usa apenas conversaId no queryKey, sem incluir organizacaoId. Se dois usuários de orgs diferentes compartilharem o mesmo conversaId (seed/mock), há risco de cache compartilhado entre organizações.

Causa raiz

queryKey definido como ['mdzap', 'mensagens', conversaId] — falta orgId.

Snippet
// useMdzapMensagens.ts linha 14
const queryKey = ['mdzap', 'mensagens', conversaId];
// DEVERIA SER:
const queryKey = ['mdzap', 'mensagens', orgId, conversaId];
Impacto

🔴 Potencial vazamento de dados entre organizações em ambientes multi-tenant. Risco de segurança P0 se IDs de conversa colidirem.

BUG-03 CRÍTICO Radix UI — Warning "Missing Description" em DialogContent (MdzapMsgBubble) src/components/mdzap/MdzapMsgBubble.tsx:68
Descrição

Console exibe warning Radix UI: "Warning: Missing `Description` or `aria-describedby={undefined}` for {DialogContent}.". Apesar de DialogTitle estar presente com className="sr-only", a DialogDescription está ausente.

Causa raiz

O <DialogContent> do preview de imagem não inclui <DialogDescription> nem aria-describedby={undefined}.

Snippet
// MdzapMsgBubble.tsx linha 68–82
<DialogContent className="max-w-3xl p-0 bg-black/90 border-border overflow-hidden">
  <DialogTitle className="sr-only">Preview de imagem</DialogTitle>
  {/* FALTANDO: <DialogDescription className="sr-only">...</DialogDescription> */}
  {/* OU: <DialogContent aria-describedby={undefined}> */}
</DialogContent>
Console evidence
Warning: Missing `Description` or `aria-describedby={undefined}` for {DialogContent}.
  at https://mdflow.app/assets/index-BP4yqHGB.js:555:496
Impacto

🔴 Acessibilidade comprometida. Warning poluindo console em produção. Radix UI pode quebrar comportamento de screen readers.

BUG-04 MÉDIO useMdzapStats e useMdzapStatus — staleTime: 60_000 diverge do padrão 30_000 src/hooks/useMdzapStats.ts:19 | src/hooks/useMdzapStatus.ts:25
Descrição

Os hooks useMdzapStats e useMdzapStatus usam staleTime: 60_000 (60 segundos) enquanto o padrão definido para a página é 30_000 (30 segundos). Isso causa inconsistência nos dados do Dashboard e do ConnectionIndicator.

Snippet
// useMdzapStats.ts linha 19
staleTime: 60_000, // DEVERIA SER 30_000

// useMdzapStatus.ts linha 25
staleTime: 60_000, // DEVERIA SER 30_000
Impacto

🟡 Dados de KPIs e status de conexão podem ficar até 60s desatualizados. Usuário pode ver estado incorreto do número WhatsApp.

BUG-05 MÉDIO useMdzapTemplates — sem React Query (useState local) — sem cache gerenciado src/hooks/mdzap/useMdzapTemplates.ts:8-34
Descrição

O hook useMdzapTemplates usa useState local e useEffect manual em vez de React Query. Sem staleTime, sem queryKey, sem invalidação automática. Cada montagem do componente dispara nova fetch ao Supabase.

Snippet
// useMdzapTemplates.ts — padrão incorreto
const [templates, setTemplates] = useState<WhatsappTemplate[]>([]);
const [isLoading, setIsLoading] = useState(false);
// Sem useQuery, sem queryKey, sem staleTime, sem cache
Impacto

🟡 Performance degradada: re-fetch toda vez que tab Templates é aberta. Sem deduplicação de requests. Inconsistente com padrão dos outros hooks.

BUG-06 MÉDIO useMdzapKeyboardShortcuts — Ctrl+K não implementado (usa Ctrl+/) src/hooks/mdzap/useMdzapKeyboardShortcuts.ts:24
Descrição

O checklist G1 especifica que / ou Ctrl+K deve focar o MdzapSearchBar. Porém o hook implementa apenas Ctrl+/ (ou Cmd+/). Ctrl+K não está mapeado — conflito com o atalho global de busca do MdFlow (⌘K visível no header).

Snippet
// useMdzapKeyboardShortcuts.ts linha 24
if ((e.ctrlKey || e.metaKey) && e.key === '/') {
  // Ctrl+K ausente — deveria ter:
  // if ((e.ctrlKey || e.metaKey) && (e.key === '/' || e.key === 'k')) {
Impacto

🟡 UX inconsistente. Usuário que tenta Ctrl+K no MDZap abre a busca global do MdFlow em vez da busca de conversas.

BUG-07 MÉDIO MdzapConnectionIndicator — sem tratamento de estado de erro explícito src/components/mdzap/MdzapConnectionIndicator.tsx:19-21
Descrição

O MdzapConnectionIndicator retorna null quando !isEnabled && !isLoading, mas não exibe nenhum feedback visual quando há erro de conexão com a API WhatsApp. O hook useMdzapStatus não expõe um campo error explicitamente ao componente.

Snippet
// MdzapConnectionIndicator.tsx linha 19-21
const { isEnabled, isLoading, numeros, numerosConectados, hasConnected } = useMdzapStatus();
if (!isEnabled && !isLoading) return null; 
// Sem: if (error) return <ErrorState />
Impacto

🟡 Se WA API cair, usuário não recebe feedback visual. O componente simplesmente some da tela.

BUG-09 BAIXO useMdzapSmartReplies — cache via Map fora do React (memory leak potencial) src/hooks/mdzap/useMdzapSmartReplies.ts
Descrição

O hook mantém um cache global (const cache = new Map()) fora do ciclo de vida do React. Este cache nunca é limpo entre sessões do usuário na mesma aba. Em uso intenso com muitas conversas, pode acumular memória indefinidamente.

Fix

Migrar para React Query com staleTime: 5 * 60_000 ou limpar o cache no unmount do hook.

BUG-10 BAIXO MdzapChatPanel — 1/2 números conectados exibido no header sem contexto MdzapChatPanel.tsx
Descrição

O header do MDZap exibe "1/2" (números conectados/total) sem tooltip ou contexto visual que explique o que esse número significa para o usuário final.

Fix

Adicionar tooltip: "1 número WhatsApp conectado de 2 configurados".
