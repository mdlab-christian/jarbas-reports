Bugs por Prioridade

P1 — Importantes (3 bugs)

BUG-R4-01 — Webhook mdzap/ai-composer retorna HTTP 500
src/components/mdzap/MdzapAIComposer.tsx:56 | Impacto: AI Composer indisponível para todos os usuários
Causa raiz: Workflow MDZAP-AI-COMPOSER no n8n retorna HTTP 500 "Workflow execution failed" — erro interno no workflow (credenciais IA, bug lógico ou nó com falha)
Evidência: curl -X POST .../webhook/mdzap/ai-composer → HTTP 500 {"message":"Workflow execution failed"}
Impacto usuário: Botão ✨ no MdzapMsgInput falha silenciosamente (toast.error) ao tentar usar AI Composer
Solução proposta: Investigar workflow MDZAP-AI-COMPOSER no n8n — verificar credenciais IA, nó Execute Code, e tratar exceções com envelope canônico
BUG-R4-02 — Webhook mdzap/julia-prompt-save retorna HTTP 500
src/components/mdzap/MdzapAdminIA.tsx:60 | Impacto: Impossível salvar/versionar prompts da Julia
Causa raiz: Workflow MDZAP-JULIA-PROMPT-SAVE no n8n retorna HTTP 500 — execução falha
Evidência: curl -X POST .../webhook/mdzap/julia-prompt-save → HTTP 500 {"message":"Workflow execution failed"}
Impacto usuário: Operadores não conseguem atualizar os prompts da Julia via MdzapAdminIA (tab Gestão). Botão "Salvar Nova Versão" dispara toast.error
Solução proposta: Investigar workflow MDZAP-JULIA-PROMPT-SAVE — verificar query UPDATE em mdzap_julia_prompts, parâmetros data.tipo/data.conteudo
BUG-R4-03 — MdzapJuliaLab usa tabela não tipada (as never) — sem type-safety
src/components/mdzap/MdzapJuliaLab.tsx:71,96,108,156,211,485 | Impacto: Schema drift não detectado em tempo de compilação
Causa raiz: supabase.from('mdzap_julia_lab_sessoes' as never) — tabela existe no banco mas não está nos types gerados do Supabase
Evidência: 6 usos de as never em MdzapJuliaLab.tsx; tabela confirmada existente com 3 rows no banco
Impacto runtime: Não bloqueia — funciona em runtime. Mas qualquer mudança de schema não é detectada pelo TypeScript
Solução proposta: Regenerar types Supabase (supabase gen types typescript) para incluir mdzap_julia_lab_sessoes + mdzap_julia_prompts

P2 — Melhorias (6 bugs)

BUG-R4-04 — MdzapGestaoPanel: React key inválida em mensagensRecentes
src/components/mdzap/MdzapGestaoPanel.tsx:368 | Impacto: Warning React em console, potencial duplicação de keys
Causa raiz: Interface MensagemRecente não tem campo id (não é selecionado na query), mas key={m.id} é usado no render das mensagens recentes → key será undefined
Evidência: Interface linha 35-41 define: contato_nome, contato_numero, direcao, tipo, created_at — sem id. Query seleciona: tipo, direcao, created_at, whatsapp_conversas (JOIN) — sem id
Solução proposta: Adicionar id na interface e no select, ou usar key={`${m.created_at}-${m.contato_numero}`} como alternativa sem alterar query
BUG-R4-05 — Webhook mdzap/transferir-conversa retorna success=false em teste
src/components/mdzap/MdzapTransferModal.tsx:65 | Impacto: Transferência falha com dados de teste (pode ser behavior esperado)
Causa raiz: Workflow MDZAP-TRANSFERIR-CONVERSA (ID: Af4O6x0EDJu8n5Qq) está ATIVO mas retorna success=false para IDs inválidos de teste. Pode ser validação de dados — confirmar com UUID real
Evidência: success=False, action=toast_error para conversa_id="test", usuario_id="test"
Classificação: P2 — comportamento pode ser correto (validação de entrada). Verificar com UUID real de conversa ativa antes de escalar
BUG-R4-06 — MdzapAppPage usa inline styles (fontFamily, height, position)
src/pages/MdzapAppPage.tsx:79,84,90,101,105,112 | Impacto: Viola Design System P-08 (tokens CSS always)
Causa raiz: style={{ fontFamily: 'Manrope, sans-serif' }}, style={{ height: 52 }}, style={{ position: 'fixed', inset: 0 }} — valores hardcoded
Solução proposta: Substituir por className="font-[Manrope]" e h-[52px]; fixed inset-0 para Tailwind equivalents
BUG-R4-07 — realtime UPDATE em whatsapp_conversas não propaga lead_awaiting_human
src/hooks/useMdzapConversas.ts | Impacto: Botão Assumir/Devolver pode não refletir em tempo real na lista
Causa raiz: lead_awaiting_human é computado via JOIN leads em fn_mdzap_conversas. O realtime UPDATE em whatsapp_conversas não inclui esse campo no payload.new (vem de leads). A atualização via updateConversaNoCache só salva {nao_lidas, status, metadata, ultimo_msg_at} — sem lead_awaiting_human
Impacto: Após assumir/devolver, a aba "Fila" pode não atualizar imediatamente o badge Awaiting até próximo refetch (staleTime 30s)
Solução proposta: Após handleAssumir/handleDevolver, além de invalidateQueries, atualizar optimisticamente lead_awaiting_human no cache via updateConversaNoCache
BUG-R4-08 — useMdzapJuliaPrompts usa tabela não tipada (as never)
src/hooks/mdzap/useMdzapJuliaPrompts.ts:20 | Impacto: Schema drift não detectado em compilação
Causa raiz: supabase.from('mdzap_julia_prompts' as never) — tabela existe (confirmada: 3 rows com id, tipo, conteudo, versao) mas não está nos types gerados
Solução proposta: Regenerar types Supabase — mesmo fix do BUG-R4-03
BUG-R4-09 — Atalhos de teclado: spec diz J/K, código usa ArrowDown/ArrowUp
src/hooks/mdzap/useMdzapKeyboardShortcuts.ts:35,36 | Impacto: Divergência entre spec documentada (L3) e implementação
Causa raiz: Spec /mdzap menciona atalhos J/K para navegar conversas. Código implementa ArrowDown/ArrowUp. Busca usa Ctrl+/ ou Ctrl+K (não "/" simples)
Classificação: P2 — atalhos funcionam, apenas diferentes do spec. Verificar com CTO qual é o comportamento desejado
