SR-01 P1 MIGRATION DROP overload fn_mdzap_conversas(uuid, uuid)

Arquivo: supabase/migrations/20260402000001_mdzap_sr01_drop_fn_conversas_overload.sql · Commit: d629069d

SR-010 (ciclo anterior) criou versão 6-params com DEFAULT values mas não removeu a versão 2-params RETURNS TABLE. PostgREST retornava PGRST203 "Could not choose the best candidate function" em chamadas com 2 argumentos, quebrando MdzapConversationList e gráficos do MdzapDashboard.

Antes — 2 overloads coexistindo
-- Overload legado (removido):
fn_mdzap_conversas(p_org_id uuid, p_numero_id uuid)
  RETURNS TABLE(...)

-- Overload novo (SR-010):
fn_mdzap_conversas(p_org_id uuid, p_numero_id uuid,
  p_status text DEFAULT NULL,
  p_search text DEFAULT NULL,
  p_page int  DEFAULT 1,
  p_per_page int DEFAULT 20)
  RETURNS JSONB
-- → PGRST203 em chamadas 2-args
Depois — 1 overload apenas
-- Migration aplicada:
DROP FUNCTION IF EXISTS
  public.fn_mdzap_conversas(uuid, uuid);

-- Apenas versão 6-params JSONB permanece.
-- Chamadas com 2 args usam DEFAULT values.
-- ✅ PGRST203 eliminado
✅ Verificado via Supabase MCP — pg_proc retorna 1 overload | curl REST retorna {total:6, data_count:6}
