SR-008 P2 FRONTEND — GLOBAL Mobile 375px: sidebar ocupa 265px de 375px disponíveis (FRONT-008 — AppLayout global) AppLayout.tsx JSX — impacto global
Screenshot 375px: conteúdo mdzap disponível em apenas ~110px de largura. Inutilizável em mobile.
Já documentado como FRONT-008. Afeta ALL páginas (AppLayout). Requer sessão dedicada.
Ao menos MdzapBottomNav (componente existe) deveria ser usado em mobile como nav alternativa.
SR-009 P2 BACKEND fn_mdzap_stats: v_aguardando_humano atribuído 2x — primeiro scan desperdiçado Supabase RPC fn_mdzap_stats RPC
Primeiro SELECT faz FILTER para whatsapp_conversas.status = 'humano' → salva em v_aguardando_humano.
Linha seguinte: SELECT COUNT(*) FROM leads WHERE aguardando_humano = true → SOBRESCREVE v_aguardando_humano.
O valor do primeiro SELECT é completamente descartado. Fix: remover a atribuição duplicada.
SR-010 P2 BACKEND fn_mdzap_conversas sem paginação server-side — carrega TODAS as conversas Supabase RPC fn_mdzap_conversas RPC
Signature: fn_mdzap_conversas(p_org_id uuid, p_numero_id uuid) — sem p_page, p_per_page, p_search, p_status.
Retorna todas as conversas da org em uma única query. Scalabilidade: O(n) com n = total de conversas.
Para org com 500+ conversas: query lenta + payload grande para o frontend.
Fix: adicionar params p_page, p_per_page, p_search (client-side filtering está correto mas não escala).
