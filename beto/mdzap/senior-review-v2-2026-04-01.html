SR-01 · P1 · BACKEND · OVERLOAD
fn_mdzap_conversas: overload duplo → PGRST203 — lista de conversas e dashboard charts vazios
Arquivos: src/hooks/useMdzapConversas.ts · src/components/mdzap/MdzapDashboard.tsx:58 · supabase/migrations/20260401182559_mdzap_sr010_fn_conversas_paginacao.sql
EXECUÇÃO OBRIGATÓRIA
Descrição: A migration SR-010 criou fn_mdzap_conversas com 6 parâmetros (p_org_id, p_numero_id, p_status, p_search, p_page, p_per_page) retornando JSONB. Porém a função antiga com 2 params (p_org_id uuid, p_numero_id uuid) retornando RETURNS TABLE (record) nunca foi removida. Ambas as versões têm DEFAULT nos params extras, tornando chamadas com 2 args ambíguas. PostgREST retorna PGRST203 "Could not choose the best candidate function". Resultado: lista de conversas mostra vazia + gráficos Dashboard vazios, apesar de 6 conversas existirem no banco.
Evidência: 
-- Confirmado via Management API:
SELECT proname, pg_get_function_identity_arguments(oid) as args, pg_get_function_result(oid) as returns
FROM pg_proc WHERE proname = 'fn_mdzap_conversas';
-- Resultado: 2 funções com assinatura compatível
-- Chamada direta 6-params: retorna {total:6, data_count:6} ✅
-- Chamada 2-params (padrão hook): PGRST203 ❌
Fix: 
DROP FUNCTION IF EXISTS public.fn_mdzap_conversas(uuid, uuid);
-- Remove a versão antiga RETURNS TABLE. A versão 6-params
-- com DEFAULT values passa a ser a única, e chamadas
-- com 2 args usarão os 4 defaults restantes.

Arquivo: src/components/mdzap/MdzapSwipeableCard.tsx:99
MELHORIA
Descrição: A operação de swipe para arquivar/restaurar conversa usa apenas .eq('id', c.id). O RLS protege a leitura, mas defense-in-depth requer filtro explícito de organizacao_id em operações de escrita também, conforme padrão MdFlow.
Evidência: 
// MdzapSwipeableCard.tsx:99
const { error } = await supabase
  .from('whatsapp_conversas')
  .update({ status: novoStatus })
  .eq('id', c.id);  // ← falta .eq('organizacao_id', organizacaoId)
Fix: 
// Adicionar filtro duplo:
.update({ status: novoStatus })
.eq('id', c.id)
.eq('organizacao_id', orgId);  // orgId do contexto useAuth()
