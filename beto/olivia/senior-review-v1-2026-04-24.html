Achados Detalhados

SR-01 CORRIGIDO P0 → FIXED FRONTEND · JSX handleGerar referenciava formato state removido
Step4Download.tsx · linha 83 (antes da correção)
Ao remover a seção duplicada de formato (N05), o state const [formato, setFormato] foi apagado mas a linha const formatoEfetivo = fmt ?? formato permaneceu referenciando a variável inexistente. Em TypeScript strict mode o build falharia silenciosamente no runtime (ReferenceError).
Fix: handleGerar(fmt: FormatoDownload = 'ambos') → const formatoEfetivo = fmt;
SR-02 CORRIGIDO P1 → FIXED SEGURANÇA · MULTI-TENANT allSelectedBlocos query sem organizacao_id e deleted_at
Step2TagsBlocos.tsx · nova useQuery allSelectedBlocos
A nova query que busca todos os blocos selecionados para exibição nos PlaceholderCards (N02) não tinha filtros eq('organizacao_id', orgId) e is('deleted_at', null). A RLS do Supabase protege o acesso entre orgs, mas o filtro explícito de org_id é obrigatório pelo padrão P-04 Multi-tenant Absoluto do projeto. O soft delete era semanticamente incorreto sem o filtro.
Fix: .eq('organizacao_id', orgId).is('deleted_at', null) + enabled: !!orgId
SR-03 CORRIGIDO P1 → FIXED FRONTEND · HTML advogado_enderecamento com \n não renderiza como quebra de linha HTML
assembleHtml.ts · substituição {{campo}}
A RPC retorna advogado_enderecamento com quebras de linha em texto plano (\n). Ao substituir {{advogado_enderecamento}} em templates HTML, o \n vira espaço em branco invisível — o cabeçalho do advogado apareceria numa única linha em vez de 3 linhas separadas.
RPC retorna: "MP Advocacia\nRua dos Andradas...\nChristian Dener Paz, OAB/RS 116571"
Fix: const toHtml = (val) => val.replace(/\n/g, '<br>') — aplicado em todas as substituições {{campo}}
SR-04 P2 FRONTEND · LOG console.error no fallback assembleHtmlLocally
src/pages/olivia/smart-blocks/index.tsx:223
O catch do fallback local tem console.error('[assembleHtmlLocally]', err). Em produção isso polui o console do browser. O erro já é tratado com toast.warning() para o usuário — o console.error é redundante.
Fix: remover console.error ou substituir por silent catch. Baixa prioridade — não afeta funcionalidade.
SR-05 P2 BACKEND · SEMÂNTICA Campo cidade mapeado para comarca, não cidade do escritório
fn_olivia_smart_blocks_preencher_campos — alias cidade
O alias cidade (usado em templates para a linha de data: "Porto Alegre, 24 de abril de 2026") está mapeado para processo_comarca. Em um processo de Barueri/SP, o campo retorna "Barueri", mas a data deveria ser "Porto Alegre" (sede do escritório). O mapeamento correto seria COALESCE(org.cidade_sede, comarca), mas organizacoes.cidade_sede não existe no schema.
Mitigação atual: usuário pode sobrescrever manualmente em "Revisar campos". Fix definitivo: adicionar cidade_sede à tabela organizacoes — fora do escopo desta sessão.
