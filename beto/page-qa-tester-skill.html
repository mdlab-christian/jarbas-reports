🔄 Fluxo de Execução
1

Ler KB ou código-fonte

Carrega o arquivo de conhecimento da página (se existir) e a árvore de componentes do código-fonte.

read mdflow-pages/<pagina>.md ls ~/beta-mdflow/src/pages/<pagina>/ cat ~/beta-mdflow/src/pages/<pagina>/index.tsx
2

Lançar subagente

Usa sessions_spawn com mode: "run" e timeout de 30 minutos. O subagente recebe: credenciais MdFlow, credenciais Supabase, KB da página e árvore de componentes.

3

Aguardar notificação

Subagente executa autonomamente e notifica via Telegram ao terminar. Sem polling. Relatório salvo em 2 caminhos automaticamente.

🛡️ Regras de Ouro

🚨 NÃO corrigir código

Skill é de INVESTIGAÇÃO, não correção. Proibido alterar .tsx, .ts, .sql, .css ou fazer commits. Apenas documente bugs com arquivo + linha.

⚠️ Anti-duplicata obrigatório

ANTES de criar qualquer registro: verificar banco por "[QA]". Usar prefixo "[QA]" em criações de teste. Máximo 1 registro por sessão por página.

✅ Nunca travar

Máximo 2 tentativas no mesmo item. Se travar: anotar PARCIAL e seguir. HTML deve ser gerado mesmo incompleto.

📱 target="host" sempre

Usar target="host" em TODOS os browser calls sem exceção.

🔍 Console após ações

Verificar browser action=console após cada ação importante para capturar erros.

🔢 Metodologia por item

1. AÇÃO → 2. SCREENSHOT → 3. VALIDAR BANCO → 4. RESULTADO (✅/❌/⚠️/➖)

📋 Checklist Dinâmico Universal (11 grupos)

Todos os itens são dinâmicos: se o componente existe na página → testa. Se não existe → pula (marca N/A).

A
Acesso e Carregamento SEMPRE

A1Navegar para rota — sem tela branca ou erro 500
A2PageHeader com título correto (se existir)
A3Console errors ANTES de qualquer ação
A4Guard de perfil (se página tem restrição)
A5Loading/skeleton durante fetch (se existir)
A6Responsivo em 375px

B
KPIs / Dashboard Cards SE EXISTIREM

B1Cards sem "undefined" ou NaN
B2Valores numéricos coerentes
B3Confirmar ≥1 KPI no Supabase
B4Gráficos renderizam sem área vazia
B5Badges de contagem em tabs carregam

C
Listagem Principal SE EXISTIR

C1Tabela/kanban/grid carrega com dados
C2Paginação — navegar para página 2
C3Ordenação por coluna
C4Checkbox de seleção
C5Toggle de modo de visualização
C6Estado vazio — mensagem adequada

D
Filtros e Busca SE EXISTIREM

D1Campo busca filtra por texto parcial
D2Dropdowns de filtro abrem e filtram
D3Tabs de status filtram corretamente
D4Filtro por data
D5Limpar filtros volta ao estado inicial
D6Deep links de filtros (?q=, ?tab=, etc.)
D7URL atualiza ao mudar filtro

E
Criar Registro SE EXISTIR

E0Anti-duplicata: checar banco por [QA] antes
E1Botão "Novo X" abre modal/form
E2Preencher campos com dados "[QA] teste"
E3Validação de campo obrigatório vazio
E4Salvar → toast de sucesso
E5Registro aparece na listagem
E6Confirmar no banco (ORDER BY created_at DESC)

F
Detalhe / Drawer / Modal SE EXISTIR

F1Clicar em item abre detalhe
F2Dados sem "undefined" ou campos vazios
F3Todas as abas do detalhe carregam
F4Editar + salvar → confirmar no banco
F5Fechar volta à lista sem reload

G
Ações Especiais / Fluxos SE EXISTIREM

G1Ações de status (aprovar, arquivar, restaurar)
G2Confirmar mudança de status no banco
G3Ações bulk (selecionar múltiplos)
G4Export CSV/PDF funciona
G5Import — modal abre (não importar de verdade)
G6Webhooks/integrações respondem
G7Keyboard shortcuts funcionam

H
Validação Supabase SEMPRE

H1Após criar → registro existe no banco
H2Após editar → campo atualizado no banco
H3Após mover/status → coluna correta atualizada
H4Dados front BATEM com dados no banco

I
Árvore de Componentes

IxPara cada componente identificado no código: renderiza sem crash? Estado funciona?

J
Modais Secundários SE EXISTIREM

JxPara cada modal: abre? Campos carregam? Submit funciona?

K
Console e Erros

K1Zero erros críticos após fluxo completo
K2Warnings relevantes listados
K3Requests com status 4xx/5xx
K4Erros de tipo TypeScript em runtime
