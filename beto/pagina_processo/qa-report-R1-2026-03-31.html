P0 NOVO BUG-N001 — restricao_tipo é INTEGER no banco, tratado como TEXT no frontend
O campo processos.restricao_tipo armazena um INTEGER (valor 1) mas o frontend espera uma TEXT string como 'comprador', 'devedor', 'fiador' ou 'avalista'. Isso causa exibição incorreta do tipo de restrição — aparece "1" em vez de "Comprador". No modo de edição, o Select não pré-seleciona o valor correto.
Arquivo
src/components/processo/sections/ProcessoRestricaoPrincipal.tsx linha 66 e 106
Também afeta
src/components/processo/tabs/ProcessoTabDados.tsx linha 40 (extractEditableFields)
Causa Raiz
Coluna processos.restricao_tipo é SMALLINT/INTEGER no banco (valor 1 = comprador). A tabela restricoes.tipo usa TEXT ('comprador'). O RPC retorna o valor bruto integer da tabela processos sem converter para o label textual da tabela restricoes.
Impacto
Exibe "1" em vez de "Comprador" em todos os ~2.387 processos com restricao_tipo definido. Edit mode: Select não pré-seleciona valor.
Evidência DB
processos.restricao_tipo = 1 (INTEGER)
restricoes.tipo = "comprador" (TEXT)
2.387 processos afetados
Solução Proposta
Opção A (backend — preferível): Alterar RPC para retornar CASE WHEN restricao_tipo = 1 THEN 'comprador' WHEN ... END
Opção B (frontend): Adicionar mapeamento numérico: {1: 'Comprador', 2: 'Devedor', ...}[tipo] ?? tipo
P1 NOVO BUG-N002 — Aba Documentos: Sub-tab "Inicial" sempre vazia, categorias incompatíveis com DB
A lógica de divisão Inicial/Tramitação em ProcessoTabDocumentos.tsx usa INICIAL_CATS = ['peticao', 'procuracao', 'comprovante']. src/components/processo/tabs/ProcessoTabDocumentos.tsx linhas 89-95 (isTramitacaoDoc)

Mismatch entre constante INICIAL_CATS no frontend e valores reais do campo categoria no banco. Sem docs com tipo='upload' E categoria in ['peticao','procuracao','comprovante'].
Impacto
Sub-tab "Inicial" sempre vazia para todos os processos. Filtros de categoria também afetados: "Petições", "Procurações" e "Comprovantes" retornam 0 resultados mesmo com docs existentes. Filtro "Outros" = 0 (banco usa 'outros' plural).

Atualizar INICIAL_CATS para incluir variações reais:
['peticao', 'peticao_inicial', 'procuracao', 'procuracao_orgao', 'comprovante', 'comprovante_restricao']
