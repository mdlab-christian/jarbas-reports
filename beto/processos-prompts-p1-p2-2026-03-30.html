# P1 — Verificar tipo de armazenamento de valor monetário no Processos

## Contexto
Projeto: MdFlow LegalTech
Stack: React + TypeScript + Supabase + shadcn/ui
Repositório: ~/beta-mdflow/
Arquivos alvo:
- src/components/processo/sections/ProcessoDadosProcesso.tsx — exibe valor_causa e restricao_valor
- src/components/processo/tabs/ProcessoTabRestricoes.tsx — exibe r.valor das restrições
- src/lib/formatters.ts — contém formatBRLReais()
- src/components/processo/drawers/ProcessoPreviewDrawer.tsx — exibe valor_causa no preview
Rota: /processos

## O que fazer

### 1. Diagnóstico (NÃO ALTERAR NADA até concluir)
Verificar como os campos monetários são armazenados no banco:

a) Consultar o schema real no Supabase — verificar tipo dos campos:
   - Tabela `processos`: tipo do campo `valor_causa`
   - Tabela `restricoes` (ou equivalente): tipo do campo `valor` / `restricao_valor`

b) Verificar nas migrations:
   grep -rn "valor_causa\|restricao_valor\|restricoes.*valor" ~/beta-mdflow/supabase/migrations/ \
     | grep -i "int8\|integer\|numeric\|decimal\|float"

c) Consultar dado real do banco (via Supabase client ou RPC):
   SELECT valor_causa FROM processos WHERE valor_causa IS NOT NULL LIMIT 5;
   SELECT valor FROM restricoes WHERE valor IS NOT NULL LIMIT 5;
   (valores plausíveis em reais = 5.000–500.000; em centavos = 500.000–50.000.000)

### 2. Decisão com base no diagnóstico

Se NUMERIC/DECIMAL (reais) → nenhuma correção necessária. Reportar no relatório final.

Se INT8/INTEGER (centavos) → corrigir APENAS a camada de exibição:
  - Criar utilitário formatBRLCents(valor: number) = formatBRLReais(valor / 100)
  - OU substituir chamadas por formatBRLReais(valor / 100) nos arquivos alvo
  - NÃO alterar schema, migrations ou banco de dados

### 3. Aplicar correção (somente se for centavos)
Corrigir exibição em:
- ProcessoDadosProcesso.tsx: formatBRLReais(processo.valor_causa) e formatBRLReais(processo.restricao_valor)
- ProcessoTabRestricoes.tsx: formatBRLReais(r.valor) / formatBRLReais(r.valor_causa)
- ProcessoPreviewDrawer.tsx: formatBRLReais(processo.valor_causa) (~linha 120)

## Referências
- Migration com NUMERIC confirmado: supabase/migrations/20260325000001_fix_distribuir_lead_completo.sql (linha 33: v_valor_causa NUMERIC)
- formatters.ts: ~/beta-mdflow/src/lib/formatters.ts

## Output esperado
- Diagnóstico claro: "campo X é NUMERIC em reais — nenhuma correção" OU "campo X é INT8 em centavos — corrigido"
- Se houve correção: diff dos arquivos alterados

## Critério de aceite
Valor de causa de R$ 50.000,00 exibido como "R$ 50.000,00" (não "R$ 5.000.000,00")
