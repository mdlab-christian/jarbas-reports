# M2 — Aba Documentos: separar em 2 partes (Inicial vs Tramitação)

## Contexto
Projeto: MdFlow LegalTech
Stack: React + TypeScript + Supabase + shadcn/ui + Tailwind CSS
Repositório: ~/beta-mdflow/
Arquivo principal: src/components/processo/tabs/ProcessoTabDocumentos.tsx
Rota: /processos

## Diagnóstico
A tabela `processos_documentos` tem o campo `categoria` (string) e `execucao_id` (FK nullable).
A aba atual mostra TODOS os documentos em lista plana com filtros de categoria.

## Separação desejada
Adicionar 2 sub-tabs DENTRO da aba Documentos (não são abas externas do drawer):

**Sub-tab "Inicial"** — documentos da fase inicial do processo:
- categoria IN ('peticao', 'procuracao', 'comprovante') OU execucao_id IS NULL
- Foco: documentos gerados na distribuição e entrada do processo

**Sub-tab "Tramitação"** — documentos durante o processo:
- execucao_id IS NOT NULL OU categoria NOT IN ('peticao', 'procuracao', 'comprovante')
- tipo IN ('gerado', 'importado', 'eproc', 'tribunal')
- Foco: decisões, intimações, despachos, docs gerados pela IA

## O que implementar

### Lógica de separação (aplicar em memória, não alterar o select do banco)

```tsx
// Dentro de ProcessoTabDocumentos, após buscar docs:
const docsInicial = useMemo(() => 
  (docs ?? []).filter(d => 
    ['peticao', 'procuracao', 'comprovante'].includes(d.categoria ?? '') || 
    (!d.execucao_id && !['gerado','importado','eproc'].includes(d.tipo ?? ''))
  ), [docs]);

const docsTramitacao = useMemo(() => 
  (docs ?? []).filter(d => 
    d.execucao_id != null || 
    ['gerado','importado','eproc'].includes(d.tipo ?? '') ||
    (d.categoria && !['peticao', 'procuracao', 'comprovante'].includes(d.categoria))
  ), [docs]);
```

```tsx
type SubTab = 'inicial' | 'tramitacao';
const [subTab, setSubTab] = useState('inicial');
```
