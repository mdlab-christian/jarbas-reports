Issues Residuais (pós-fixes)
DB-R01
Campos monetários em NUMERIC ao invés de BIGINT centavos
valor_causa, valor_condenacao, restricao_valor, alvara_valor, deposito_judicial_valor usam NUMERIC. Spec define INT8 centavos. Sem impacto em produção imediato — dados estão sendo armazenados como NUMERIC reais. Migração futura necessária para consistência total com o padrão.
P2
DB-R02
processos_restricoes.tipo é INTEGER sem CHECK constraint
Campo tipo em processos_restricoes é INTEGER sem constraint de validação. Conforme padrão MdFlow, deveria ser TEXT CHECK. Valores não documentados na constraint.
P2
DB-R03
estado_id legado (TEXT) coexiste com estado_id_new (UUID)
Duas colunas para o mesmo conceito. Migração de estado_id TEXT → UUID não foi completada. Dado legado presente mas lógica operacional usa estado_id_new.
P2
DB-R04
Search performance acima de 500ms (705ms avg)
GIN indexes existem mas busca full-text via ILIKE + immutable_unaccent está lenta para dataset de 14k registros. Considerar uso de pg_trgm %% operator ou materializar coluna de busca.
P3
DB-R05
olivia_get_analises_processo sem SECURITY DEFINER
Função chamada no contexto OlivIA sem SECURITY DEFINER. Usada apenas via service_role/server-side — risco baixo mas inconsistente com padrão.
