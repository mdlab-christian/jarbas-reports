Achados UI/UX — Melhorias e Issues

P0 — CRÍTICO: Exportação Nunca Conclui (Bug de Sincronização Frontend/Webhook)

O webhook mdflow/processos/export executa de forma SÍNCRONA e retorna action=download_file com URL direta imediatamente. Porém o frontend implementa polling assíncrono via webhook_requests table aguardando status='completed'. O registro criado no banco fica eternamente com status='processing'.

Fluxo atual (quebrado): webhook → retorna URL → frontend ignora URL → inicia polling → polling nunca encontra 'completed' → timeout 5min → "Exportação demorando mais que o esperado..."

Fix necessário (2 opções):

  1. Frontend: após res.success, verificar se res.action === 'download_file' e abrir res.data.url diretamente (sem polling)
  2. Webhook: atualizar webhook_requests status para 'completed' + result_storage_path antes de retornar
P1 — Empty State com Filtros Avançados Ativos Mostra Mensagem Errada
Quando filtros avançados (sem busca textual) retornam 0 resultados, o empty state exibe ícone Scale + "Nenhum processo cadastrado" — mesmo sendo óbvio que existem processos mas os filtros estão restrictivos. O searchQuery é vazio, então o código cai no segundo branch. O ideal seria verificar se há filtros avançados ativos também. 
// Em ProcessosTable.tsx linha 145-155

}
P1 — KPI "Novos no Mês" Sempre Zero
A RPC get_processos_kpis usa data_distribuicao (não created_at) para calcular novos do mês. Como a maioria dos processos tem data_distribuicao=NULL (verificado: 5 primeiros do banco todos nulos), o KPI sempre retorna 0 independente do mês. Deve ser avaliado se o critério correto é created_at ou data_distribuicao.

O cabeçalho "Nº CNJ" usa TableHead estático, não SortableHead. Outros cabeçalhos de dados textuais (Status, Fase, Advogado) também não são sortable. Consistência visual: esses headers não têm ChevronsUpDown icon enquanto Cliente, Empresa, Valor e Distribuído têm.
