5. Detalhes Tecnicos — Codigo Fonte

5.1 Hook useProcessos — Arquitetura de Filtros

O hook principal useProcessos (180 linhas) implementa:

  • Estado de filtros com 14 campos tipados via ProcessosFiltros interface
  • URL sync bidirecional: parseFromURL() le params, syncToURL() escreve params
  • Normalizacao CNJ: strip separadores para busca por digitos (0012345672026 encontra 0012345-67.2026...)
  • Contagem de filtros avancados ativos para badge no botao "Filtros"
  • Reset automatico de pagina ao mudar qualquer filtro (exceto page)

5.5 Export Flow — Async Polling

O fluxo de exportacao implementa polling assincronico:

  1. Frontend envia webhook processos/export com filtros, formato, colunas
  2. Se resposta inclui action: download_file com URL, download direto (sincrono)
  3. Caso contrario, inicia polling a cada 2s na tabela webhook_requests
  4. Quando status = completed, gera signed URL do Supabase Storage bucket 'exports'
  5. Quando status = failed, mostra toast error
  6. Timeout: 150 tentativas = 5 minutos
  7. Cleanup: clearInterval no unmount via useEffect + useRef
// Rate limit handling if (res.error_code === 'RATE_LIMIT') { toast.warning('Aguarde 60 segundos entre exportacoes.'); }

5.6 Import Flow — Upload + Polling

  1. Upload arquivo para Supabase Storage bucket 'importacoes' em path: {org_id}/imports/processos/{request_id}.{ext}
  2. Webhook processos/import com storage_path, formato, modo_upsert, advogado_id
  3. Polling a cada 2s na tabela webhook_requests
  4. Resposta detalhada: inseridos, atualizados, erros
  5. Modal nao fecha durante processamento (Escape prevented)
