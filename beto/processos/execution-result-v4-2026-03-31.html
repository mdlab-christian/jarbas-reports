🔧 BUG-N2 — Insights retornando registros de erro
BUG-N2 P1 Hook useProcessoInsights retorna ia_status='erro' como dado válido ✅ Resolvido
src/hooks/useProcessoInsights.ts
De 50 registros na tabela processos_insights para a org Midas, 49 têm ia_status='erro' (falha na geração por IA). O hook usava .maybeSingle() sem filtrar erros, então retornava um registro de falha como insight válido — fazendo a aba Insights exibir campos null em vez de dados reais.
// ANTES: await supabase .from('processos_insights') .select('*') .eq('processo_id', processoId!) .order('updated_at', { ascending: false }) .maybeSingle(); // DEPOIS — filtra registros com erro antes de pegar o mais recente: await supabase .from('processos_insights') .select('*') .eq('processo_id', processoId!) .not('ia_status', 'eq', 'erro') // ✅ exclui falhas de IA .order('updated_at', { ascending: false }) .maybeSingle();
ia_status: string | null; // ex: 'sucesso' | 'erro' | 'pendente' ia_error: string | null; // mensagem de erro da IA modelo_ia: string | null; // ex: 'claude-3-5-haiku' tokens_utilizados: number | null; // total de tokens parte_representada: string | null;// ex: 'autor' | 'reu'
Query com .not('ia_status', 'eq', 'erro') confirma que, quando não há registro válido, retorna null (correto). A aba Insights exibirá EmptyState em vez de dados corrompidos.
