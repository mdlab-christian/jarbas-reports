FRONT-001 A11Y JSX P1
ProcessoRowActions: icon button sem aria-label
O botao de acoes (MoreVertical) na linha da tabela nao tem aria-label. Screen readers nao conseguem identificar a funcao do botao. O ModalImportarProcessos (linha 326) implementa corretamente com aria-label="Remover arquivo selecionado", mas o RowActions nao segue o mesmo padrao.

FRONT-002 UX CSS P2
KPI card "Total (Filtros)" cortado em 1024px
No breakpoint 1024px, o 4o KPI card ("Total (Filtros)") tem o label truncado mostrando "Total (Filtro..." e o valor "13.75..." parcialmente cortado. O grid de 4 colunas fica apertado com a sidebar de 48px ocupando espaco. Considerar usar text-ellipsis com tooltip, ou reduzir o label para "Total" neste breakpoint.
src/pages/processos/components/ProcessosKPIs.tsx
FRONT-003 UX CSS P2
Botao "Exportar" cortado em 1024px
No breakpoint 1024px, o botao "Exportar" no header aparece como "Expo..." truncado. Os 3 botoes (OlivIA + Importar + Exportar) competem por espaco horizontal. Solucao: em breakpoints menores que ~1100px, mostrar apenas o icone (sem texto) para Importar e Exportar, mantendo o Tooltip para identificacao.
src/pages/processos/index.tsx (header buttons)
FRONT-004 UX CSS P2
375px: KPI cards 2a coluna parcialmente oculta
No viewport 375px, os KPI cards na coluna da direita (Baixados, Total Filtros) mostram apenas o icone, sem espaço para label e valor. A grid grid-cols-2 divide igualmente o espaco mas o padding/gap consome area util. Solucao: em <640px, considerar grid-cols-1 para KPIs ou reduzir padding para p-3.
src/pages/processos/components/ProcessosKPIs.tsx:20
FRONT-005 UX CSS P2
375px: tabs "Baixados" cortada
No viewport 375px, a tab "Baixados" do filtro fica parcialmente cortada. O tablist "Todos | Ativos | Baixados" nao tem scroll horizontal. O botao "Filtros" tambem compete pelo espaco na mesma linha. Solucao: adicionar overflow-x-auto no container das tabs + botao, ou empilhar verticalmente em mobile.
src/pages/processos/index.tsx (filter bar area)
FRONT-006 A11Y JSX P1
Drawer icon buttons: cobertura aria-label incompleta
O drawer tem 6 icon buttons mas a verificacao grep nao encontrou aria-label em todos eles. Os Tooltips cobrem a descoberta visual, mas screen readers precisam de aria-label direto no <Button>. Verificar: botoes de delete em TramitacaoTab, botoes de acao em DecisaoDetalheDrawer.
src/components/processo/tabs/*.tsx, src/components/processo/drawers/*.tsx
FRONT-007 UX UX P3
Colunas Status e Fase com "---" para maioria dos processos
Na tabela principal, as colunas Status e Fase mostram "---" (dash) para a maioria dos registros visiveis. Isso indica que os dados nao estao preenchidos no backend. Nao e um bug de frontend -- os dados simplesmente estao nulos. Mas a UX poderia ser melhorada com um indicador mais descritivo (ex: badge "Pendente" em vez de dash solitario).
src/pages/processos/components/ProcessosTable.tsx
