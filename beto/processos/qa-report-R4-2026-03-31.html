Resumo Executivo

A página /processos está funcionalmente sólida. As RPCs principais (get_processos_paginados e get_processos_kpis) funcionam corretamente. Os webhooks de arquivamento/restauração e o drawer de detalhes estão operacionais. Foram encontrados 2 bugs P1 (crash para perfil operacional + empresa lookup truncado) e 5 bugs P2 (formatação, memory leak, staleTime fora do spec, bucket errado, UX menor). Adicionalmente, há 2 problemas de qualidade de dados que afetam a experiência visual mas não a funcionalidade.

ATENÇÃO: 9.067 de 13.720 processos (66%) têm status_id = NULL. Esses processos exibem "—" na coluna Status. É um problema de dados, não de código — mas impacta diretamente a usabilidade da página.

Bugs Encontrados

P1 BUG-01 — BulkActionBar: onExport obrigatório mas recebe undefined para perfil 'operacional'
Arquivosrc/components/shared/bulk_action_bar.tsx linha 11
Também emsrc/pages/processos/components/ProcessosBulkBar.tsx linha 7
ImpactoRuntime crash (TypeError: onExport is not a function) para usuários com perfil 'operacional' que clicam em "Exportar seleção" na BulkBar

Causa Raiz

BulkActionBar declara onExport: () => void (obrigatório). ProcessosBulkBar declara onExport?: () => void (opcional) e repassa o valor para BulkActionBar. No index.tsx, onExport é definido como canExport ? () => setIsModalExportOpen(true) : undefined. Quando o usuário tem perfil 'operacional' (canEdit=true mas canExport=false), BulkBar é renderizada mas onExport=undefined. O botão "Exportar seleção" sempre renderiza no BulkActionBar sem checar se onExport está definido.

Trecho problemático

// bulk_action_bar.tsx linha 11 — onExport obrigatório
interface Props {
  onExport: () => void;  // ← obrigatório, sem "?"
}

// ProcessosBulkBar.tsx linha 7 — onExport opcional
interface Props {
  onExport?: () => void;  // ← opcional
}

// index.tsx linha 354 — pode ser undefined
onExport={canExport ? () => setIsModalExportOpen(true) : undefined}

Solução Proposta

// Opção A: Tornar onExport opcional no BulkActionBar
interface Props {
  onExport?: () => void;
}
// E proteger o botão:
{onExport && (
  <Button onClick={onExport}>Exportar seleção</Button>
)}

// Opção B: Não renderizar BulkBar para 'operacional' sem canExport
// No index.tsx:
{canEdit && canExport && (
  <ProcessosBulkBar ... onExport={() => setIsModalExportOpen(true)} />
)}
// + versão sem export para operacional sem canExport
P1 BUG-02 — Lookup de Empresas limitado a 100 de 771 ativas
Arquivosrc/pages/processos/hooks/useProcessosLookups.ts linha 35
Impacto671 empresas (87% do total) ficam invisíveis no filtro MultiSelect Empresa. Usuário não consegue filtrar por empresas além das primeiras 100 (ordem alfabética)

// Opção A: Aumentar limit para cobrir todas as empresas

Qualidade de Dados (não são bugs de código)

Recomendação: Verificar se a importação em lote de março/2026 não preenche status_id automaticamente. Considerar migration para atribuir status default ("Cadastro") nos processos sem status.
DATA DATA-02 — 4.137 de 13.720 processos (30%) com cliente_id = NULL

Recomendação: Verificar fluxo de distribuição — processos devem ser vinculados ao cliente automaticamente. Podem ser processos importados que ainda não passaram pelo enriquecimento.

Observações Técnicas Adicionais

Documentação da missão usa p_org_id — na prática é p_organizacao_id

O checklist menciona chamar as RPCs com p_org_id, mas ambas as RPCs usam p_organizacao_id. O código frontend em useProcessos.ts já usa o nome correto. Apenas a documentação da missão está imprecisa.

rpc_processo_get_completo retorna ACESSO_NEGADO com service_role

Este comportamento é correto. A RPC verifica auth.uid() internamente via SECURITY DEFINER. Com service_role o JWT não tem uid. Em produção, o frontend usa anon_key + JWT do usuário logado e a RPC funciona normalmente. Não é um bug.

ProcessosFilterBar — busca 1-3 chars dispara queries sem filtro

Ao digitar 1-3 caracteres, effectiveSearch=null e o RPC é chamado sem filtro de busca (retorna todos). Isso significa queries desnecessárias ao banco a cada keystroke com menos de 4 chars. Seria melhor não disparar a query enquanto o texto tiver menos de 4 chars.

IaStatusBadge: hidden xl:table-cell

A coluna IA (IaStatusBadge) só aparece em telas >1280px. Usuários em 1024-1279px (desktop comum) não veem o status IA. Comportamento intencional por espaço de tela, mas merece atenção.

novos_mes = 4.641 — plausível

Em 2026-03-31, o mês corrente é março/2026. O bulk import de 2026-03-23 (data de criação de milhares de registros) explica o valor alto. Os KPIs estão matematicamente consistentes: 9.218 + 4.502 = 13.720 (total do paginados).
