Bugs Novos Encontrados (P0/P1/P2)

P1 BUG-N1: Schema Drift — campos fantasmas em useProcessoDecisoes
Arquivo
src/hooks/useProcessoDecisoes.ts linhas 30-31
Causa Raiz
O TypeScript interface ProcessoDecisao declara dois campos que não existem no schema do banco: 
prazo_recursal_dias: number | null;  // NÃO EXISTE na tabela decisoes
prazo_tipo: string | null;           // NÃO EXISTE na tabela decisoes
Os campos reais no banco são: prazo_recurso (int, sem dados) e data_prazo_recursal (date, sem dados) e tipo_recurso. A query usa select('*') então não falha — mas o TypeScript interface está errado, causando P-01 violação (UI ≠ Backend).
Impacto
Atualmente sem impacto visual (campos são null, usados como null). Mas viola P-01 e pode causar bugs futuros se alguém usar dec.prazo_recursal_dias esperando o valor do banco.
Fix Sugerido
Remover prazo_recursal_dias e prazo_tipo da interface. Adicionar prazo_recurso: number | null e tipo_recurso: string | null se necessários.
Confirmação Banco
select prazo_recursal_dias FROM decisoes → ERROR: column decisoes.prazo_recursal_dias does not exist
P1 BUG-N2: processos_insights — 49/50 registros com ia_status='erro' sem limpeza
Tabela Afetada
processos_insights — 49 registros ia_status='erro', 1 ia_status='concluido'
Causa Raiz
Falhas de geração de insights IA acumuladas sem limpeza. useProcessoInsights usa maybeSingle() e não filtra por ia_status, então pode retornar um registro de erro para um processo — resultando em dados nulos mesmo com registro existente.
Impacto
Para processos com registro ia_status='erro', a aba Insights mostrará empty state ou KPIs como "—" mesmo tendo uma entrada na tabela. Experiência do usuário degradada.
Fix Sugerido
Opção A (hook): adicionar .eq('ia_status', 'concluido') na query do useProcessoInsights.
Opção B (banco): pg_cron para limpar registros com ia_status='erro' mais antigos de 7 dias.
Opção C (código): mostrar badge "Análise falhou — clique para regenerar" quando ia_status='erro'.
P2 BUG-N3: Audiência de teste com data ano 2222 no banco de produção
Evidência
1 audiência com data_audiencia: 2222-02-23 (processo: 5067690-25.2025.8.21.0001). Status: cancelada. deleted_at: null.
Impacto
Registro de teste não removido em produção. Não impacta funcionalidade (status=cancelada), mas polui o banco e pode afetar relatórios de audiências agendadas.
Fix Sugerido
Soft delete: UPDATE audiencias SET deleted_at = now() WHERE id = '48dfd1ba-e4f4-4328-b579-9302601203f8'
