Detalhes dos Fixes
SR-001 SR-002 SR-006 RPC get_processos_paginados — 3 fixes combinados
A RPC foi recriada via migration para resolver 3 problemas simultâneos sem overhead duplo:
Mudanças
  • SR-001: RETURN inclui total_pages: CEIL(v_total / page_size)::int
  • SR-002: p.polo_representado adicionado ao SELECT interno
  • SR-006: ASC branch usa COALESCE(nome, '~') para forçar NULLs ao final
SR-007 olivia_get_analises_processo → SECURITY DEFINER
Função sem SECURITY DEFINER estava sujeita ao RLS da tabela olivia_analises, o que podia bloquear chamadas do frontend mesmo com JWT válido.
Migration aplicada
  • Adicionado SECURITY DEFINER
  • Adicionado SET search_path TO 'public'
  • Verificação: prosecdef = true confirmado via pg_proc
SR-008 SR-012 ProcessoRestricaoPrincipal — dead code + comentário
Verificado: banco tem 0 registros com restricao_tipo inteiro (distincts: 'devedor', 'comprador'). O guard typeof tipoRaw === 'number' nunca disparava.
Mudanças em ProcessoRestricaoPrincipal.tsx
  • SR-008: Removido RESTRICAO_TIPO_INT_MAP (6 linhas dead code)
  • SR-008: Removido guard typeof tipoRaw === 'number'
  • SR-008: Variável tipoRaw colapsada em tipo direto
  • SR-012: Comentário atualizado com tipos do schema atual
SR-009 useAdvogadoLookup — limit 50 → 200
Com 50 advogados o filtro podia ficar incompleto. Corrigido em 3 locais que tinham o mesmo problema.
Arquivos corrigidos
  • useProcessosLookups.ts — função useAdvogadoLookup(): limit 50→200
  • ModalImportarProcessos.tsx — lookup inline: adicionado .limit(200)
  • ProcessoDadosProcesso.tsx — lookup edit mode: adicionado .limit(200)
SR-010 staleTime 5min → 30s — 6 arquivos
O padrão do projeto é staleTime: 30_000 (30 segundos). Encontrados 8 usos de 5 * 60 * 1000 em 6 arquivos do cluster processo/.
Arquivos normalizados
  • ProcessoRestricaoPrincipal.tsx — 1 ocorrência
  • ProcessoTabRestricoes.tsx — 1 ocorrência
  • ProcessoDadosProcesso.tsx — 3 ocorrências (já era 30_000 em 2 após edit)
  • ProcessoModalShell.tsx — 1 ocorrência
  • DecisaoDetalheDrawer.tsx — 1 ocorrência
  • ProcessoPreviewDrawer.tsx — 1 ocorrência
SR-005 ModalImportarProcessos — bucket documentos → importacoes
O upload de arquivos de importação (.csv/.xlsx) estava sendo enviado para o bucket genérico documentos. O bucket dedicado importacoes existe no projeto.
Mudança
  • supabase.storage.from('documentos') → .from('importacoes')
  • Storage path mantido: {org_id}/imports/processos/{requestId}.{ext}
  • Workflow n8n processos/import permanece pendente (escopo fora desta execução)
