P0 — Critico (Bloqueante)

BUG-USR-001: useBlocker crash — pagina nao renderiza

P0 CORRIGIDO

Problema: useBlocker de react-router-dom requer createBrowserRouter (data router), mas o app usa BrowserRouter. Resultado: crash imediato ao navegar para /usuario.

Erro: useBlocker must be used within a data router

ArquivoLinhaCodigo OriginalFix
src/pages/usuario/index.tsx 2, 47-57 import { useBlocker } from 'react-router-dom';
const blocker = useBlocker(isDirty);		 Removido useBlocker. Protecao SPA mantida via beforeunload. Comentario explicativo adicionado.

FIX APLICADO nesta sessao. Pagina agora renderiza corretamente.

P1 — Importantes

USR-P1-001: Falta bloqueio de navegacao SPA

P1

Com a remocao do useBlocker, a protecao de dirty state na navegacao SPA (clicar sidebar, links) foi perdida. Apenas beforeunload (refresh/fechar aba) protege.

Fix sugerido: Implementar hook customizado useNavigationBlocker usando window.addEventListener('popstate') ou migrar para createBrowserRouter no App.tsx.

ArquivoAcao
src/pages/usuario/index.tsxAdicionar hook de bloqueio SPA customizado

USR-P1-002: Permissao /usuario nao esta na lista do usuario

P1

O PageGuard bloqueia acesso a /usuario para usuarios nao-admin porque a rota nao esta em user.permissoes. Funciona apenas porque o usuario de teste e admin (isAdmin bypassa). Usuarios com perfil juridico/comercial nao conseguem acessar seu proprio perfil via URL direta.

Fix sugerido: Remover o PageGuard de /usuario no App.tsx, pois todo usuario autenticado deve poder acessar seu proprio perfil. Ou adicionar /usuario como permissao default para todos os perfis.

ArquivoLinhaAcao
src/App.tsx258-259Remover <PageGuard rota="/usuario" /> wrapper

USR-P1-003: Falta aria-label em botoes Copy PIX e Editar

P1

O botao de copiar PIX tem title mas falta aria-label. Botoes "Editar" e "Cancelar" nos forms sao texto-somente (OK), mas o botao Descartar no dirty banner usa icone X sem label explicito.

ArquivoLinhaFix
src/pages/usuario/components/UsuarioFinancialForm.tsx245Adicionar aria-label="Copiar chave PIX"
src/pages/usuario/components/UsuarioDirtyBanner.tsx~38Botao Descartar ja tem texto, mas verificar que X icon nao confunde leitores de tela

Scroll suave: Pagina renderiza sem jumps ou flickers em todos breakpoints.

CAT-B — Tipografia 9/10

H1: font-display (Manrope) correto.
Labels: text-sm font-medium text-foreground em todos os forms. Correto.

H3 dos cards: Usam text-lg font-semibold sem font-display. O DS sugere Manrope para titulos. P2-003 registrado.

CAT-C — Cores e Tokens 10/10

Zero cores hardcoded: grep -rn 'bg-\[#\|text-\[#\|border-\[#' retornou vazio. Nenhuma violacao.
Backgrounds: GlassCard usa bg-card/60 backdrop-blur-lg. Correto.
Textos: text-foreground, text-muted-foreground, text-destructive usados corretamente.
Primary: Badge admin usa bg-primary/20 text-primary. Avatar fallback usa bg-primary/20 text-primary. Correto.
Borders: border-border em todos os inputs e cards. Correto.
Strength bar: Usa tokens bg-destructive, bg-warning, bg-success. Correto.

CAT-D — Componentes 9/10

GlassCard: Variante glass (default) com backdrop-filter: blur. Usado em cards (correto). Nao usado em tabelas/inputs/botoes (correto).
Badge perfil: Cores por tipo: admin=primary, juridico=info, comercial=warning, operacional=muted. Tooltips com descricao. Correto.
Inputs readonly vs editable: Readonly usa cursor-default opacity-70. Email usa cursor-not-allowed opacity-60 + icone Lock. Distintos visualmente. Correto.
Hierarquia de botoes: Editar=ghost, Salvar=default(primary), Cancelar=ghost, Sair=outline+destructive. Hierarquia visual clara.
Skeleton loading: Skeletons presentes em Identity, Personal e Financial cards. Dimensoes realistas.

CAT-E — Interacoes 6/10

Avatar hover: Overlay com Camera icon e transicao transition-opacity group-hover:opacity-100. Correto.
Dirty banner: Framer Motion com initial/animate/exit. Respeita prefers-reduced-motion. Excelente.
useBlocker crash: P0 critico que impedia toda interacao. CORRIGIDO nesta sessao.
Navegacao SPA perdida: Apos fix do useBlocker, dirty state nao bloqueia cliques na sidebar. P1-001 registrado.
Focus rings inconsistentes: Avatar tem focus-visible:ring, eye toggles e copy button nao. P2-004 registrado.

CAT-F — Responsividade 8/10

375px (mobile): Identity card muda para coluna (flex-col items-center). Avatar centralizado. Texto centralizado. Sem overflow horizontal. Correto.
768px (tablet): Sidebar oculta (hamburger). Forms ocupam largura total. Sem problemas de layout.
1280px (desktop): Sidebar visivel. Content area centrada com max-w-3xl. Correto.
1440px (wide): max-w respeitado, conteudo nao se estica. Correto.
375px email truncado: Email no identity card trunca corretamente com truncate. Na area de inputs, email longo corta com ellipsis dentro do input. Comportamento aceitavel, nao e violacao.
Dados Pessoais nao tem botao Editar visivel no header em 375px: O botao "Editar" aparece no mesmo row que o titulo. Em telas muito pequenas pode ficar apertado. Verificar se ha wrap.

CAT-G — Modal Avatar 9/10

Dark mode: Modal usa DialogContent do shadcn (herda dark theme). Correto.
Drag & drop area: border-2 border-dashed border-border com hover border-primary/50 hover:bg-muted/30. Visual claro.
Descricao contextual: Muda entre selecting/cropping/uploading. DialogDescription presente (acessibilidade).
Validacao: Tipos aceitos (JPEG/PNG/WebP), limite 2MB, feedback via toast. Correto.
Canvas crop: 512x512 com clip circular. Zoom slider com icons ZoomIn/ZoomOut. Preview 64x64. Completo.
Botoes Voltar/Salvar: flex gap-2 com flex-1 cada. Alinhados. Voltar=outline, Salvar=default. Correto.
Touch support: onTouchStart/Move/End implementados para mobile crop. Bom.

CAT-H — Empty States 9/10

PIX empty state: Icone Wallet com opacity-40, texto text-muted-foreground, botao variant="outline". Centrado com flex-col items-center gap-3 py-10. Correto e bem visual.
Error state: Icone AlertCircle, texto descritivo, botao "Tentar novamente" com RefreshCw. Correto.

CAT-I — Acessibilidade Visual 7/10

Contraste: --foreground (#f2efe8) sobre --card (#1a1714) = ratio ~14:1. Excelente. --muted-foreground (#9e978a) sobre --card = ratio ~5.8:1. Acima de 4.5:1. OK.
Avatar button: aria-label="Alterar avatar". Correto.
Eye toggle: aria-label={visible ? 'Ocultar senha' : 'Mostrar senha'}. Correto.
Disabled states: Botoes disabled usam disabled nativo (opacity reduzida automatica). Correto.
Copy PIX button: Tem title mas falta aria-label. P1-003.
Focus rings: Inconsistentes entre componentes. P2-004.
Breadcrumb button: Nao tem role="navigation" no nav pai... tem, usa <nav> semantico. OK.

  4. Breadcrumb funcional e com semantica HTML correta (<nav>)
  5. Identity card responsivo (coluna em mobile, row em desktop)
  6. Avatar com hover overlay, aria-label e focus ring
  7. Modal avatar completo: drag/drop, crop, zoom, preview, touch support
  8. Badge de perfil com cores por tipo e tooltips descritivos

  10. Password strength bar com tokens (destructive/warning/success)
  11. Dirty banner com Framer Motion e prefers-reduced-motion
  12. beforeunload handler para proteger contra refresh/fechar aba
  13. PIX empty state bem desenhado e centralizado
  14. Error state com icone, texto e acao de retry
