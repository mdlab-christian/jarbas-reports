2. Achados Novos

P1
BUG: extractPathFromUrl nao suporta paths relativos -- storage leak useUserAvatar.ts linhas 7-16: extractPathFromUrl() usa regex que espera URL completa com /storage/v1/object/. Desde a Fase 4, o DB armazena paths relativos (ex: org_id/user_id/avatar_xxx.webp). Quando o usuario faz upload de novo avatar, extractPathFromUrl(currentAvatarUrl) retorna null e o avatar antigo NAO e deletado do storage. Mesmo problema no handleRemove -- ao remover avatar com path relativo no DB, o arquivo persiste no bucket.

src/hooks/usuario/useUserAvatar.ts:7-16, 34-37, 77-79


Fix proposto: 
function extractPathFromUrl(url: string): string | null {
  if (!url) return null;
  // Relative path (new format) -- already is the storage path
  if (!url.startsWith('http')) return url;
  try {
    const match =
      url.match(/\/storage\/v1\/object\/(?:public|sign)\/avatares\/([^?]+)/) ||
      url.match(/\/storage\/v1\/object\/authenticated\/avatares\/(.+)$/);
    return match ? decodeURIComponent(match[1]) : null;
  } catch {
    return null;
  }
}
