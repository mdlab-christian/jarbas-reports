Como funciona o monitoramento
Passo a passo técnico completo — do login até a intimação aparecer no Controller.
1
Login SSO no EPROC
Acesso via Playwright (Chrome headless) com credenciais do Supabase.
Busca credenciais na tabela jarbas_accounts → Navega para a URL do tribunal → O site redireciona para o Keycloak SSO → Preenche usuário (ex: RS116571) e senha → Gera código TOTP de 6 dígitos via segredo Base32 do banco → Insere o código 2FA → Seleciona perfil ADVOGADO → ✅ Dentro do painel EPROC
2
Listagem de todos os processos da OAB
Navega pela lista paginada de processos ordenados por última movimentação.
Acessa o relatório de processos do procurador (ordenado por últimas movimentações) → Extrai todos os CNJs via regex \d{7}-\d{2}\.\d{4}\.\d\.\d{2}\.\d{4} → Navega todas as páginas automaticamente → Resultado: lista completa de CNJs únicos (ex: RS116571 → ~500+ processos)
3
Lookup no banco MdFlow
Carrega todos os 12.898 processos cadastrados e mapeia CNJ → processo_id.
Busca paginada na tabela processos do Supabase (12.898 registros) → Monta mapa: CNJ → UUID do processo → Permite vincular cada intimação ao processo correto sem fazer N queries
4
Acesso processo a processo
Para cada processo (foco nos primeiros 50 por movimentação recente), extrai os dados da tela.
Navega para a URL direta do processo via CNJ → Aguarda carregamento completo → Extrai via page.evaluate():
• Número CNJ completo (ex: 5012667-52.2025.8.21.0015)
• Classe processual (ex: Procedimento Comum, JEC)
• Últimas 5 movimentações com data e texto
• Data da última movimentação
• Indicadores de urgência (bloqueio, hasta pública, audiência)
5
Análise e classificação por IA
Cada conjunto de movimentações é analisado para identificar o tipo de evento e nível de risco.
Verifica palavras-chave nas movimentações:
• Crítico: bloqueio, penhora, hasta pública, sequestro de bens
• Prazo: intimação com prazo, data final contagem
• Decisão: decisão interlocutória, sentença, acórdão
• Citação: citação eletrônica, carta precatória
• Despacho: despacho judicial
• Movimentação geral: andamentos sem urgência

Nível de risco: critico → alto → medio → baixo
6
Inserção no banco de intimações
Cada evento relevante vira uma intimação na tabela do MdFlow, vinculada ao processo.
Insere na tabela intimacoes com:
• cnj — número completo (ex: 5012667-52.2025.8.21.0015)
• processo_id — UUID do processo no MdFlow
• titulo — título legível com CNJ completo (ex: ⏱️ Prazo — 5012667-52.2025.8.21.0015)
• tipo_evento — prazo, decisao, citacao, despacho, movimentacao_geral
• ia_risco_nivel — critico / alto / medio / baixo
• ia_acao_recomendada — ação sugerida preenchida
• payload_raw — texto bruto das movimentações extraídas
• status = nova

Deduplicação via conteudo_hash para evitar duplicatas em rodadas seguintes.
7
Notificação no Telegram
Resumo enviado ao grupo de notificações com os casos críticos e estatísticas.
Ao final da rodada, envia resumo para o grupo -5184596601:
• Quantidade de processos verificados
• Casos críticos identificados (lista com CNJ completo)
• Casos de alto risco
• Total de intimações inseridas

Taxonomia de eventos
Como a CECILIA classifica cada movimentação processual.
A
🚨 Crítico — Ação Imediata
Bloqueio patrimonial, penhora, hasta pública, sequestro de bens. Prazo máximo: hoje.
B
⚠️ Alto Risco
Decisão interlocutória desfavorável, citação em novo processo, prazo curto (<3 dias).
C
⏱️ Prazo Processual
Intimação com prazo determinado. Verificar data final e adotar medida cabível.
D
⚖️ Decisão / Sentença
Decisão judicial, sentença, acórdão. Analisar conteúdo e verificar recurso cabível.
E
📋 Despacho / Citação
Despacho para cumprimento, citação eletrônica. Cumprir no prazo legal.
F
📄 Movimentação Geral
Andamento sem urgência imediata. Acompanhar — CIÊNCIA sem prazo, juntada de doc.

Melhorias identificadas
Itens a implementar para deixar o sistema production-ready.

    Corrigir login TJRS (Keycloak) — o campo senha no Keycloak tem dois elementos: um visível (toggle de exibição) e um hidden (campo real). O script atual às vezes preenche o errado. Fix: usar [name=password] via JavaScript evaluate no campo hidden.
  • P1
    Auto-cadastrar processos novos — 5 processos do EPROC não existiam no MdFlow (distribuídos em 2026). Ao inserir uma intimação sem processo_id, a CECILIA deve criar automaticamente o registro na tabela processos com os dados básicos extraídos.
  • P2
    Cron automático — 7h e 19h — varredura completa das 5 OABs duas vezes por dia. Hoje a CECILIA só roda quando acionada manualmente pelo Christian.
  • P2
    Deduplicação robusta — evitar reinserir a mesma movimentação em rodadas consecutivas. Usar conteudo_hash com hash do texto + CNJ + data.
  • P2
    Expandir para todas as OABs por rodada — hoje o E2E roda em uma OAB por vez. Implementar loop sequencial: RS116571 → RS131163 → SC074025 → SC075466 → SP535101.
  • P3
    Integrar taxonomia A-G do vault Obsidian — o vault ~/mdlab-vault/01_projetos/controller/02-PLAYBOOKS/taxonomia-intimacoes.md tem a taxonomia completa definida pelo João. A CECILIA deve ler e aplicar essas regras na classificação, não só palavras-chave.
  • P3
    Corrigir permissões do vault Obsidian — as pastas 02-PLAYBOOKS/, 01-FUNCIONAMENTO/ etc. estão bloqueadas por permissão de SO. Executar sudo chmod -R +rx ~/mdlab-vault/01_projetos/controller/ para a CECILIA ter acesso completo ao cérebro operacional.
  • P3
    Relatório HTML diário para o João — ao final de cada varredura, gerar e publicar automaticamente um HTML com todos os casos críticos, prazos próximos e estatísticas, com link enviado via Telegram.

Filosofia operacional
Como a CECILIA toma decisões.
⚠️
Regra de ouro do vault: "Nada sem validação do João." A CECILIA sugere, classifica e popula o Controller — mas a decisão final de ação processual é sempre do advogado responsável. O sistema não executa petições, prazos ou protocolos de forma autônoma sem aprovação explícita.

Foco único
A CECILIA existe para uma coisa: garantir que nenhuma intimação relevante passe despercebida pela equipe.

Aprende com o João
