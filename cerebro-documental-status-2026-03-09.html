🔍 Problemas Encontrados & Status

✅ Corrigido
commit 7490176
URL relativa montada incorretamente
O EPROC usa hrefs relativos como controlador.php?acao=acessar_documento.... O código usava window.location.origin (só o domínio) e perdia o /eproc/ do path — resultando em HTTP 404.
Fix aplicado
Trocado para new URL(href, window.location.href) — resolve o path relativo corretamente incluindo /eproc/.
✅ Corrigido
Supabase Storage
Bucket processos-docs não existia
O upload para Supabase Storage falhava com "Bucket not found" porque o bucket nunca foi criado no projeto. O downloader tentava fazer upload sem o destino existir.
Fix aplicado
Bucket processos-docs criado via API REST do Supabase. Limite: 50MB por arquivo. Pronto para upload.
✅ Corrigido
commit 12b3652
Skill cerebro_fase0 ausente do constraint
A coluna tipo da tabela jarbas_tarefas tem um CHECK constraint enumerando os tipos válidos. cerebro_fase0 e cerebro_documental não estavam na lista.
Fix aplicado
Constraint atualizado via Supabase SQL API para incluir os novos tipos.
🔧 Fix hoje
Bloqueador principal
Download captura HTML viewer em vez do PDF
O EPROC serve os documentos via viewer HTML com iframe. O fetch(href) no browser retorna o HTML do viewer, não o arquivo PDF. O pdftotext/pdfplumber então falham com "No /Root object — Is this really a PDF?"
Fix a aplicar
Navegar ao viewer → extrair iframe[src] ou embed[src] → fazer fetch do PDF real. Fallback: page.waitForEvent('download') ao clicar no link. Implementação: hoje.
📋 Arquitetural
Descoberta relevante
Roteamento OAB ↔ Tribunal necessário
Na Fase 0 os processos eram acessados por todas as 5 OABs. Resultado: OABs de SC viram 0 documentos para processos TJRS/TJSP — e vice-versa. Cada OAB só enxerga documentos do seu tribunal.
Solução
Em produção: rotear por tribunal. TJRS → RS116571, RS131163   TJSP → SP535101   TJSC → SC074025, SC075466. Reduz trabalho redundante em 3x.
