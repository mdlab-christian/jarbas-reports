🔴 Pontos de Quebra 6 problemas

🔴
P0 810 intimações "nova" acumuladas — triagem parada
Há 810 intimações com status='nova' e ia_status='concluido'. O pipeline IA já processou tudo, mas a equipe não está conseguindo dar vazão. Sem um workflow de triagem automática (ou priorização mais agressiva), esse número vai crescer.
Fix: Implementar OLIVIA_PRAZO_GUARDIAN para auto-processar intimações com prazo vencido + criar filtro "Não lidas com IA concluída" como view padrão na abertura da página.
🔴
P0 65 erros hoje — causa não identificada
O heartbeat reporta 65 erros _errosHoje em ~20 minutos de uptime (JARBAS reiniciado recentemente). Sem acesso aos logs detalhados agora, a causa pode ser: falhas de login EPROC, timeouts de navegação Playwright, ou erros de query Supabase.
Fix urgente: cat ~/jarbas/logs/*.log | grep ERROR | tail -50 para identificar padrão. Provavelmente são logins EPROC falhando nas OABs. Verificar credenciais.
🔴
P1 Chat OlivIA no Drawer → webhook 404
A tab "Chat OlivIA" no Drawer chama callWebhook('controller/chat'). Testado: n8n retorna 404. Se um advogado tentar usar o chat, vai aparecer erro genérico. O componente tem graceful error handling mas a feature está não-funcional.
Fix: Criar workflow n8n WH-CTRL-CHAT que recebe {"{"}intimacao_id, messages{"}"}, chama Claude via HTTP Request e retorna {"{"}resposta{"}"}. Ou desabilitar a tab temporariamente.
🔴
P1 skill_ciencia_renuncia.mjs é STUB + WH-CTRL-09 inativo
Botão "Ciência" no drawer chama callWebhook('controller/ciencia') → mas o webhook WH-CTRL-09 está inativo. A skill JARBAS correspondente é um stub aguardando vídeo do João. Qualquer tentativa de ciência vai falhar silenciosamente.
Fix: Ou implementar a skill (aguardar vídeo João), ou desabilitar o botão "Ciência" no drawer com tooltip "Em breve" para não criar falsa expectativa.
🔴
P1 skill_verificar_prazos_hoje.mjs não existe
O painel de Monitoramento lista esta skill como "Não criada" (P0). Sem ela, o sistema não tem monitoramento automático de prazos do dia. A única detecção é via KPIs no front (passiva) — não há notificação proativa ao advogado.
Fix: Criar a skill: query intimacoes WHERE data_prazo = today AND status IN ('nova','lida') → notifica Telegram com lista de prazos urgentes. Registrar em start.mjs como cron matinal.
🔴
P2 Tab Monitoramento: dados hardcoded (não refletem estado real)
As seções "Agentes IA", "Skills JARBAS" e "Webhooks n8n" são arrays hardcoded no TSX. Se um webhook for ativado ou uma skill criada, a UI não vai refletir automaticamente. Isso é enganoso e vai ficar desatualizado.
Fix: JARBAS expor endpoint (ou Supabase table) com status real das skills e webhooks. Heartbeat metadata já tem o array skills — usar isso para gerar lista dinâmica.
