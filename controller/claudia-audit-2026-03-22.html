🐛 Bugs Identificados

🔴
P0 lib/supabase.mjs path errado nas skills herdadas
As skills em src/skills/herdadas/ importam ../lib/supabase.mjs que resolve para src/skills/lib/ (não existe). A lib está em src/lib/.
Fix: criar symlink src/skills/lib → ../lib OU adicionar src/skills/lib/ com re-exports
🔴
P0 Pipeline chama classificador/roteador com payload vazio
O pipeline atual chama classificar({payload: {organizacao_id}}) mas o classificador espera {intimacao_id} ou {intimacao}. Precisa buscar intimações pendentes do Supabase antes de classificar.
Fix: pipeline deve buscar intimacoes com ia_status='pendente' e iterar uma a uma
🟡
P1 jarbas_heartbeat não recebe sinal da CLAUDIA
A página Controller lê jarbas_heartbeat para mostrar status do bot. A Claudia não escreve nessa tabela — aparece como offline na UI.
Fix: skill_claudia_heartbeat deve fazer upsert em jarbas_heartbeat com versao='CLAUDIA_v1'
🟡
P1 CLAUDIA não responde mensagens Telegram
O bot @Claudia_mdbot está ativo mas o start.mjs não tem polling de mensagens. Não é possível mandar um comando e receber resposta.
Fix: adicionar telegram polling loop no start.mjs (getUpdates ou webhook)
🟡
P1 Prompt CLAUDIA_CLASSIFICADOR pode não existir no Supabase
O classificador busca ai_prompts WHERE agent_key='CLAUDIA_CLASSIFICADOR'. Se não existir, o pipeline trava.
Fix: verificar/criar prompt no Supabase OU embutir fallback no código
🟢
P2 config/.env não tem OAB_RS/OAB_SC/OAB_SP explícitos
O start.mjs usa fallback hardcoded. As credenciais EPROC (TOTP, senhas) vêm do Supabase via accounts-loader — OK, mas env vars de OAB ausentes.
Fix: adicionar OAB_RS=RS116571, OAB_SC=SC074025, OAB_SP=SP535101 no config/.env
