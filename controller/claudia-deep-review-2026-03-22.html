🐛 Issues Identificados

CRÍTICO
Sem orquestrador — Claudia não consegue processar nada
As 6 skills estão escritas mas não há nada que as invoque. Falta um start.mjs com cron jobs e orquestração: captura → classifica → roteia → alerta. Sem isso, Claudia é só código morto.
→ Criar ~/claudia/start.mjs com cron de captura (a cada 4h), orquestração das skills e heartbeat (15min)
CRÍTICO
VAULT_PATH incorreto no .env
VAULT_PATH=~/obsidian-claudia mas o vault está em ~/claudia/vault-obsidian/. skill_claudia_vault_sync vai tentar escrever no caminho errado e falhar silenciosamente.
→ Corrigir: VAULT_PATH=~/claudia/vault-obsidian (ou mover para ~/obsidian-claudia)
CRÍTICO
skill_claudia_visual_fallback não existe
O classificador chama Vision fallback se confiança < 80%, mas não há referência ao script correto. skill_clayton_visual_fallback.mjs existe nas herdadas mas nunca foi renomeado/adaptado.
→ Copiar e adaptar skill_clayton_visual_fallback → skill_claudia_visual_fallback com cache 7 dias em claudia_visual_memory
IMPORTANTE
Skills herdadas não corrigidas
Os 3 bugs principais do JARBAS (hash fraco, batch de 20, sem tipo_prazo) ainda existem nas skills herdadas. São cópias diretas sem as correções previstas no plano v4.
→ Aplicar correções: SHA256 hash, batch 10, tipo_prazo, queue serializada RS→SC→SP
IMPORTANTE
vault_synced_at em controller_feedback — não verificado
skill_claudia_vault_sync busca registros com .is('vault_synced_at', null) e depois faz update dessa coluna. Se a coluna não existir, vai falhar silenciosamente.
→ Verificar: ALTER TABLE controller_feedback ADD COLUMN IF NOT EXISTS vault_synced_at TIMESTAMPTZ
IMPORTANTE
Index unique em intimacoes(signature_hash) não confirmado
A deduplicação por hash depende desse index. Sem ele, duplicatas podem ser inseridas em situações de race condition.
→ Verificar: CREATE UNIQUE INDEX IF NOT EXISTS idx_intimacoes_hash ON intimacoes(signature_hash) WHERE signature_hash IS NOT NULL
