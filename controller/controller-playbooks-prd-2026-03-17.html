Spec: Interface de Chat de Treinamento
A principal inovação do projeto — como João instrui o JARBAS
🎯 Entrada: Atividade sem playbook
Quando o Controller recebe uma intimação cujo tipo não tem playbook → aparece na tabela Playbooks com badge "⏳ Aguardando instrução". João clica em "Instruir JARBAS".
💬 Chat abre no lado direito
Sheet/drawer lateral com interface de chat. JARBAS abre com: "Vi uma nova atividade: [tipo]. Como devo agir quando isso aparecer? Me explique por texto ou áudio."
🎤 Áudio → Prompt
João grava áudio diretamente no chat. Frontend envia para JARBAS → Whisper transcreve → Claude reformula como instrução de IA clara → exibe para João revisar.
✅ Aprovação e salvamento
João vê o prompt gerado. Pode editar inline, aceitar ou pedir reformulação ("torna mais específico"). Ao confirmar → salva em jarbas_activity_playbooks com status "ativo".
🔄 Uso em runtime
Na próxima captura, quando JARBAS encontrar o mesmo tipo de atividade → lê o playbook da tabela → usa como instrução no prompt de análise → classifica corretamente automaticamente.
📊 Tabela geral de playbooks
Visão de planilha com todas as atividades: tipo, instrução resumida, última atualização, total de usos, % acerto. João pode editar qualquer instrução diretamente na tabela.

Integração Advbox — Como vai funcionar
Garantir que nenhuma intimação cai no vão entre os dois sistemas
1. JARBAS puxa intimações do Advbox via API
A skill_monitorar já tem acesso ao Advbox. Expandir para chamar endpoint de intimações pendentes do Advbox. Parâmetros: últimas 48h, todos os processos cadastrados.
2. Cross-check por signature_hash
Para cada intimação do Advbox: calcular hash (CNJ + data + texto primeiros 100 chars). Se já existe na tabela intimacoes → ignorar. Se nova → inserir com fonte='advbox'.
3. Intimações do Advbox entram no pipeline normal
Após inserção, entram como ia_status='pendente'. O pipeline de análise (skill_analise_intimacoes_bulk) processa igual ao EPROC. João vê tudo unificado no Controller.
4. Dashboard de cobertura na aba Monitoramento
Cards mostrando: total processos no Advbox, total monitorados pelo JARBAS, % cobertura, gap (processos não monitorados). Alerta se cobertura < 90%. Última sync Advbox com timestamp.

🟢 Banco Supabase: JARBAS tem acesso total via Management API. Todas as DDLs serão executadas pelo JARBAS sem necessidade de Gustavo acessar o painel.
