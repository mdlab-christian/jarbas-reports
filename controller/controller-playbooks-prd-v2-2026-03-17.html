O problema que vamos resolver
Por que a situação atual não funciona
🤖❌
JARBAS classifica errado
O agente OLIVIA_INTIMACAO_MODAL usa um prompt genérico. Não tem instrução específica para cada tipo de atividade que o escritório usa. Resultado: classificações erradas que frustraram João nos vídeos anteriores.
🎓❌
Aba Treinamentos não funciona
A aba atual tem UI bonita mas nenhuma das seções conecta com o JARBAS de verdade. Regras cadastradas lá não chegam ao agente de análise. Vídeos entram mas nada é extraído. É teatro.
🎥❌
Vídeos consomem demais
João gravava vídeos longos mostrando o problema. O JARBAS demorava muito pra processar, travava frequentemente, e mesmo quando funcionava a extração era superficial. Custo alto, resultado baixo.
🏛️🔀
Advbox + EPROC sem deduplicação
O escritório usa o Advbox como sistema jurídico antigo. Ele também monitora intimações. Sem integração, intimações podem entrar duplicadas, ou pior: capturadas pelo Advbox mas não pelo JARBAS.
🔄❌
Nenhum loop de aprendizado
Quando João corrige uma classificação errada, esse aprendizado não vai a lugar nenhum. JARBAS comete o mesmo erro infinitamente. Não existe mecanismo de feedback que alimente o agente.

Nova Aba "Playbooks" — O que vai existir
Reconstrução completa do ControllerTreinamentoHub

💡 A grande ideia: Tabela de Tipos → Instrução

Ao invés de tentar ensinar o JARBAS via vídeo longo, a gente faz uma tabela simples: tipo de atividade → como o JARBAS deve agir. João preenche isso em linguagem natural ou via áudio. JARBAS transforma em instrução de IA. Na próxima captura, JARBAS lê essa tabela antes de analisar qualquer intimação. Resultado: classificação precisa, consistente, e que João pode corrigir a qualquer hora.

Sistema de Feedback — Loop de Aprendizado
João avalia · JARBAS aprende · Controller melhora com o tempo

🔁 O loop que faz o JARBAS ficar bom de verdade

Cada vez que João dá um 👍 ou 👎 numa classificação, esse dado vai para o banco. JARBAS usa esses exemplos reais como few-shot learning na próxima análise. Com o tempo, os erros diminuem porque o agente aprende com casos reais do escritório Midas. Não é magia — é memória estruturada alimentando o prompt.

A interface principal de cadastro de playbooks
1

Numa linha da tabela Playbooks que está "⏳ Aguardando", ou ao clicar em "Novo playbook". Pode ser de uma intimação específica que foi mal classificada.
JOÃO
2
Chat abre no lado direito
Drawer lateral aparece com JARBAS enviando mensagem inicial: "Entendi que apareceu '[tipo]'. Me explica como devo agir — pode ser texto ou áudio."
INTERFACE
3
João responde (texto ou áudio)
Texto: digita a instrução livremente.
Áudio 🎤: segura o botão, fala, solta. Frontend envia o áudio para o webhook do JARBAS.
JOÃO
4
JARBAS processa e reformula
Transcreve áudio (Whisper). Reformula a instrução de João em linguagem de prompt técnico para IA. Retorna para o frontend exibir ao João para revisão.
JARBAS
5
João confere o prompt gerado
Vê o prompt técnico que será usado. Pode pedir ajuste ("mais específico sobre prazo"), editar direto, ou confirmar como está.
JOÃO
6
Salvo e ativo imediatamente
Playbook salvo em jarbas_activity_playbooks. Status muda para "🟢 Ativo". Intimações que estavam paradas com esse tipo são reprocessadas na próxima captura.

Integração Advbox — Cobertura Total
Garantir que nenhuma intimação cai no vão entre os dois sistemas
⚠️
Pré-condição: JARBAS precisa verificar se a API do Advbox expõe endpoint de intimações por data antes de comprometer a implementação. Se não tiver API, será scraping autenticado. Christian/Gustavo precisam confirmar o tipo de acesso disponível.
Como vai funcionar
Fluxo de sincronização (roda junto com o monitoramento normal): 1. JARBAS chama API Advbox — últimas 48h de intimações
2. Para cada intimação: gera signature_hash (CNJ + data + hash do texto)
3. Verifica se hash já existe em intimacoes
4. Se nova: insert com fonte='advbox', entra no pipeline normal
5. Grava resultado em jarbas_advbox_sync_log
Dashboard de cobertura (aba Monitoramento)
Cards que aparecerão: 📊 Cobertura: X de Y processos monitorados pelo JARBAS
⚠️ Gap: Z processos no Advbox sem cobertura JARBAS
🔄 Última sync Advbox: [timestamp] · [N novas] · [N duplicatas]
✅ Status: OK / Erro / Sem resposta
