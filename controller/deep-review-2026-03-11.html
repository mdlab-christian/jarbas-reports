V3

🎯 O que é o Controller

É o painel central de triagem de intimações judiciais do escritório Midas. Recebe intimações capturadas automaticamente do EPROC (RS/SC/SP), processa via IA (OlivIA), e apresenta uma fila de trabalho onde os advogados tomam ações — classificar, delegar, concluir, gerar petição.

🗂️ Abas principais da página

  • Intimações — fila principal com filtros, busca, bulk actions
  • Monitoramento — processos sendo monitorados pelo JARBAS no EPROC
  • Documentos — docs pendentes do escritório
  • Equipe — distribuição por advogado
  • Atividades — atividades geradas pelo controller
  • Dashboard — KPIs e gráficos
  • Arquivados — intimações concluídas/ignoradas
  • Playbooks — guias de triagem por tipo de intimação

🔍 Filtros disponíveis (V3)

  • Status — nova, lida, processada, ignorada
  • IA Status — pendente, processando, concluído, erro
  • Urgente / Importante — flags
  • Advogado responsável
  • Prazo — vencendo hoje, vencido
  • Busca livre — título, CNJ, cliente, resumo
  • Ocultar DJEN — filtra publicações DJE (padrão: ativo)
  • Ocultar Pauta Julgamento
  • Mostrar Duplicadas
  • Paginação — 200 / 400 / 800 / 1200 por página

⚡ KPIs do topo (V3)

  • Total de intimações novas
  • Prazos vencendo hoje (banner de alerta vermelho)
  • Prazos críticos (≤5 dias)
  • Aguardando análise IA
  • Intimações urgentes
  • Processadas hoje

🎹 Keyboard Shortcuts

  • ESC — fechar drawer
  • A — criar atividade (drawer aberto)
  • P — processar intimação
  • I — ignorar intimação
  • ⌘K — Command Palette

Agente Controller (Claude)

olivia-controller

🧠 O que é o Agente Controller

É um agente Claude especializado em triagem jurídica que o JARBAS pode invocar via cc: [tarefa] ou skill. Ele conhece todos os playbooks do escritório, as regras do EPROC (RS/SC/SP) e a árvore de decisão de intimações.

✅ Age autonomamente (sem escalar)

  • Ciência com Renúncia ao Prazo (intimações informativas)
  • Registrar audiências novas no sistema
  • Solicitar documentos AJG ao cliente
  • Lançar atividade de Réplica
  • Lançar Contrarrazões quando apelação do réu identificada
  • Marcar intimação como lida/processada
  • Identificar prazo (úteis vs corridos) por tipo

🛑 SEMPRE escala para humano

  • Sentença/acórdão de mérito → Dra. Eloísa
  • Valor de dano moral inconsistente
  • Decisão ambígua sobre custas
  • Sem regra clara no playbook
  • Embargos à execução
  • Impugnação ao cumprimento

Pipeline Completo: EPROC → Ação

🏗️ Fluxo completo de uma intimação

① EPROC RS/SC/SP
→
skill_monitorar detecta evento novo
② skill_captura_intimacoes
→
extrai texto, grava na tabela intimacoes com sha256 dedup, status='nova'
③ skill_analise_bulk
→
Claude Haiku analisa, grava analise_ia (resumo, ação, flags, prazo)
④ /controller (V3)
→
aparece na fila com badge IA ✓, resumo, flags, prazo
⑤ Advogado/JARBAS age
→
cria atividade, delega, processa ou ignora

🔍 O que a IA extrai de cada intimação

  • resumo_controller — frase de 80 chars para o card
  • acao_recomendada — o que fazer
  • proximos_passos — lista priorizada
  • flag_ciencia_renuncia — true/false
  • flag_deposito_judicial — true/false
  • tipo_intimacao — classificação
  • nivel_urgencia — normal/alto/crítico
  • fase_processual — em que fase está
  • responsavel_sugerido_nome — quem deve agir
  • resultado_decisao — favorável/desfavorável/parcial

📊 Dados pré-calculados na view SQL

  • dias_ate_prazo — dias até vencer
  • status_prazo — sem_prazo/hoje/crítico/vencido
  • is_duplicada — intimação duplicada de outra
  • decisao_resultado — join com tabela decisões
  • ciencia_renuncia_status — pendente/executada
  • advogado_oab + advogado_oab_uf
  • empresa_nome — join com empresas
  • auto_cadastro_pendente — cliente não cadastrado

Gaps e Pendências

🔴
Bug #17 — Lentidão na execução de comandos
Interface percebida como lenta em ações (processar, ignorar, bulk). Causa provável: invalidateQueries muito amplo ou latência do webhook. Precisa profiling.
🔴
Ciência c/ Renúncia — STUB (aguardando vídeo João)
A skill ciencia_renuncia está como STUB. Delega para JARBAS via WH-CTRL-09 mas não executa peticionamento real no EPROC. Precisa do vídeo do João com o fluxo completo.
🟡
Duplicatas — sem agrupamento real
Bug #9 foi parcialmente resolvido (ícone de duplicata na row). Mas não há agrupamento visual "N intimações para o mesmo processo" com colapso/expansão. Seria necessário refatorar V3IntimacaoGroup.
🟡
Aba Financeiro — sem tabela dedicada
Feature 2 (Financeiro) salva em metadata.financeiro como JSONB. Funciona bem para MVP mas dificulta queries analíticas. Criar tabela financeiro_intimacoes no futuro.
🟡
Prazo dinâmico — apenas para tipos mapeados
Feature 1 calcula prazo para ~12 tipos de ação. Intimações com tipo não mapeado não mostram prazo dinâmico (depende do prazo calculado pelo DB). Expandir o PRAZO_MAP conforme surgem novos tipos.
🟡
RAG ChromaDB não populado
A base vetorial processos_jarbas / intimacoes_jarbas ainda não foi indexada. O skill rag_mdflow não funciona sem ela. Rodar: python3 ~/jarbas/src/rag/indexar-mdflow.py
🟢
Monitor N8N — funcional
7 workflows críticos sendo monitorados. JARBAS alerta via Telegram se qualquer um cair. Heartbeat verifica a cada hora.
🟢
Realtime Supabase — ativo
useControllerRealtime hook ativo — quando nova intimação chega no Supabase, a fila atualiza automaticamente sem refresh.
