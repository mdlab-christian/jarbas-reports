🗺️ Plano de Melhorias — Priorizado

P1 — Crítico

Sync de campos raiz após análise IA

Os campos resumo_ia, tipo_evento, prazo_dias, data_prazo e urgente não estão sendo preenchidos após o processamento da IA. A análise rica existe no JSON mas a UI do Controller não consegue consumir.

  • No finalizador da skill de análise IA, após salvar analise_ia, executar UPDATE nos campos raiz
  • resumo_ia ← analise_ia._analista_raw.resumo_controller
  • tipo_evento ← analise_ia._analista_raw.atividade_tipo (mapeado para enum da tabela)
  • prazo_dias ← analise_ia._analista_raw.atividade_sugerida.prazo_dias
  • urgente ← prazo_dias <= 5 OR flag_recurso_necessario = true
  • Script de backfill para os 72 registros já processados
P1 — Crítico

Reprocessar 11 registros com modelo antigo

11 intimações usaram a estrutura legada (sem _analista_raw) — sem roteamento, sem fase, sem responsável. Precisam ser reprocessadas pelo novo modelo.

  • Identificar: SELECT * FROM intimacoes WHERE analise_ia->>'_analista_raw' IS NULL AND ia_status = 'concluido'
  • Resetar ia_status = 'pendente' nos 11 registros
  • Deixar o ciclo normal reprocessar nas próximas horas
P2 — Alto

Capturar conteudo_completo antes de analisar (fase recursal)

7 casos com confiança < 75% são todos da fase recursal com conteúdo truncado. O EPROC envia apenas o cabeçalho — a IA não tem base suficiente para classificar com segurança.

  • Para intimações com fase_processual = 'recursal' e conteúdo < 300 chars: enfileirar download do PDF do evento antes de processar IA
  • Preencher conteudo_completo com o texto extraído do PDF
  • Reprocessar IA com o conteúdo completo disponível
  • Resultado esperado: confiança > 80% para esses casos
P2 — Alto

Ativar flag_recurso_necessario no prompt da IA

flag_recurso_necessario está null em 100% dos registros mesmo com 2 decisões desfavoráveis detectadas. O campo existe no schema mas o prompt não instrui a IA a preenchê-lo.

  • Adicionar instrução explícita no system prompt do analista IA
  • Lógica: se flag_decisao_desfavoravel = true E fase_processual IN ('defesa', 'recursal') → avaliar necessidade de recurso
  • Resultado: alertas de recurso automáticos no Controller
P3 — Melhoria

Dashboard de qualidade IA — monitoramento contínuo

Hoje não há visibilidade sobre a qualidade do processamento IA ao longo do tempo. Criar painel simples no Controller com métricas semanais.

  • % de registros com confiança > 80% (baseline atual: 88,5%)
  • Distribuição de tipos de atividade por semana
  • Contagem de decisões favoráveis vs. desfavoráveis por tribunal
  • Alertas automáticos quando confiança média cair abaixo de 80%
P3 — Melhoria

Enriquecer títulos automáticos com valor da causa

Os títulos gerados pela IA já incluem valores (ex: "Trânsito em julgado — R$15.051,39") — ótimo. Mas 61 intimações ainda têm tipo_evento = null, o que prejudica filtros e agrupamentos na UI.

  • Mapear atividade_tipo para tipo_evento enum: ciencia_renuncia → 'ciencia', cumprimento_sentenca → 'execucao', etc.
  • Atualizar via UPDATE após processamento (parte do P1)

📌 Resumo Executivo

O que melhorou com o upgrade
100% das intimações processadas sem erro. Confiança média de 86,9% (muito boa). Roteamento automático por responsável funcionando. Detecção de trânsitos em julgado e valores identificando ~R$81k em execuções a protocolar. Identificação de ciência de renuncia em 48 intimações.
