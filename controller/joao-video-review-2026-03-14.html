🚨 Gap Crítico — Falta uma Engine de Delegação Automática
1
O que João quer: 3 tipos de ação, não 1
O João mostrou que o trabalho do controller tem 3 saídas: A) Ciência com Renúncia (EPROC + fechar), B) Criar Tarefa para alguém, C) Fechar sem ação. A skill atual só faz A e C. O tipo B — delegação — não existe.
"Atualmente no nosso sistema, quando tem decisão improcedente, a gente manda pra Doutora Heloisa como IMPROCEDENTE 1º GRAU. Quando é do Boa Vista, manda pro Leo."
2
Regras de delegação que o João mostrou (vídeos 1303)
Sentença improcedente → Heloisa (IMPROCEDENTE 1º GRAU) | Boa Vista improcedente → Leonardo | Procedente / parcialmente procedente → Cris + Helo | Decisão 2º grau → Leonardo (7-8d) | SISBAJUD → João Victor (urgente) | Contestação → Yuri ou advogado responsável (RÉPLICA 15d) | AJG Indeferido → AGRAVO 15d
3
O que precisa ser criado: skill_regras_delegacao ou integrar à captura
Após a ciência automática decidir "não é ciência", a skill precisa detectar o TIPO da decisão e CRIAR uma atividade no MdFlow para a pessoa certa com prazo correto. Isso é o coração do que João pediu nos vídeos de triagem.
"Eu não tô aqui pra fazer o trabalho do sistema. Eu tô aqui pra revisar o que o sistema decidiu."

79% entregue — Gap Crítico na Delegação Automática

Os bugs de UX e features de frontend estão bem resolvidos. O problema real está no backend: o João mostrou em 10+ vídeos que o coração do controller é delegar para a pessoa certa automaticamente. A skill atual só faz ciência ou ignora — não cria tarefas para Heloisa, Leonardo, João Victor, etc. Esse é o gap que transforma 79% em 95%.
