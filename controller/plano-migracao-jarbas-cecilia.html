2 Bug Crítico — Por que Cecília encontra 0 intimações

🐛 BUG-01: JANELA_DIAS_MAX = 30 no JARBAS (deve ser 3)

O arquivo skill_captura_intimacoes.mjs do JARBAS tem JANELA_DIAS_MAX = 30 hardcoded. João Victor definiu que a janela deve ser 3 dias. A Cecília pode estar herdando esse valor errado se a skill foi copiada sem ajuste.

// ERRADO (skill JARBAS atual):
const JANELA_DIAS_MAX = 30; // ← deve ser 3

// CORRETO para Cecília:
const JANELA_DIAS_MAX = 3;

🐛 BUG-02: extractIntimacoesDaListagem depende de estrutura DOM do EPROC

A função tenta ler células da tabela por índice fixo (cells[7], cells[6]). O EPROC tem layouts diferentes por tribunal (TJRS vs TJSC vs TJSP) e por versão. Se a estrutura mudou ou o painel carregou de forma diferente, retorna 0 resultados.

🐛 BUG-03: Sessão expira antes de processar — isSessionExpired() falha silenciosamente

A skill importa isSessionExpired de skill_monitorar.mjs. Se a Cecília não tem esse arquivo ou a sessão expirou sem tratamento, a captura termina em 0 intimações sem erro visível.

🐛 BUG-04: instance_id incorreto — tarefas não chegam na Cecília

A tabela jarbas_tarefas tem tarefas captura_intimacoes com instance_id = 'mac-mini-m4-w1' (worker do JARBAS). A Cecília usa instance_id = 'cecilia-mac-mini'. As tarefas nunca chegam nela se o consumidor filtra por instance_id.

  • Verificar se skill_monitorar.mjs está copiado para Cecília (dependência de isSessionExpired)
