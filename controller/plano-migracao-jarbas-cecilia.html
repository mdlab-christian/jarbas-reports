🔄 3. Fluxo Correto de Captura (baseado no que funcionava no JARBAS)

Os logs do JARBAS confirmam capturas bem-sucedidas: RS131163 encontrou 12 intimações, SC074025 e SC075466 encontraram 0 (sem intimações novas), SP535101 funcionou depois de resolver o problema de login.
1
Login SSO + TOTP
loginEproc(account, { closeBrowser: false }) — retorna { success, browser, page }. Retry até 3x com backoff exponencial (10s → 20s → 40s). Se falhar 3x → pular OAB, logar erro, continuar para próxima.
⚠️ Problema atual: SP535101 falhava 3x. Verificar se TOTP está sincronizado.
2
Verificar sessão válida
isSessionExpired(page) — verifica se ainda está logado. Se expirou mid-ciclo → chamar reloginFn() que fecha o browser atual e abre nova sessão. Isso era crítico pois ciclos longos (5 OABs) podiam expirar no meio.
3
Navegar para Relatório de Processos
Navegação direta via URL (mais confiável que clicar em menus):
page.goto(base + 'controlador.php?acao=relatorio_processo_procurador_listar&ord_ultimas_movimentacoes=S')

Fallback hierárquico: seletor CSS → menu lateral → Vision Adaptativa (Clayton).
Problema reportado: redireciona para principal quando sessão não está ativa corretamente.
⚠️ JANA disse que corrigiu para navegação direta via URL. Verificar se Cecília implementou isso.
4
Ordenar por data do último evento
Clicar no cabeçalho da coluna "Data/Hora Evento" duas vezes (para ordenar decrescente). Isso garante que os processos com eventos mais recentes aparecem primeiro, otimizando a captura dentro da janela de 3 dias.
5
Vision Screening (Haiku pré-triagem)
Screenshot da listagem → Claude Haiku analisa visualmente quais linhas têm ícone de intimação (vermelho/amarelo). Descarta páginas sem intimações antes de processar DOM — economiza chamadas de IA.

🔴 4. Problema Atual — Por que a captura parou

Sintoma: Últimas 3 intimações no banco são de 23/03/2026, ia_status: pendente, sem campo tribunal preenchido. Zero capturas desde então.

🔍 Hipóteses (do mais ao menos provável)

  1. Redirecionamento para /principal
    O relatorio_processo_procurador_listar redireciona para principal quando a sessão SSO não está corretamente ativa. O JARBAS usava fallback com Clayton (vision) quando isso ocorria. A Cecília talvez não tenha esse fallback ou a navegação direta via URL.
  2. start.mjs da Cecília não disparou o ciclo
    A Cecília foi iniciada pelo LaunchDaemon mas talvez o schedule de execução não esteja configurado (equivalente ao MONITORAR_SCHEDULE=6,13,19 do JARBAS).
  3. Erro silencioso no pipeline v4.0
    Um erro não tratado pode estar fazendo o ciclo falhar sem logar. Necessário sudo tail -f /Users/cecilia/cecilia/logs/cecilia.log.
  4. TOTP dessincronizado
    O SP535101 já estava falhando 3x no JARBAS. Se o TOTP perdeu sincronização nas outras OABs, o ciclo inteiro pode estar abortando.
  5. Sessão expirada mid-ciclo sem re-login
    O JARBAS tinha um reloginFn explícito. Se a Cecília não tem esse callback, ciclos longos falham silenciosamente na 2ª OAB em diante.

🔑 Solução do redirecionamento (baseada no que funcionava no JARBAS)

⚖️ 7. Regras Críticas do João Victor

Estas regras são inegociáveis. A Cecília foi treinada com base nelas. JARBAS as mantém como referência.

REGRA 1
🗓️ Janela Temporal = 3 dias
Intimações com data > 3 dias são descartadas sem processar. JANELA_DIAS_MAX = 3. Não gasta Claude com lixo histórico sem prazo relevante.
REGRA 2
🎯 Gatilho correto
"Expedida/Certificada a Intimação Eletrônica" com prazo vermelho ou amarelo no EPROC. Verde = expirado = ignorar.
REGRA 3
🔇 Eventos ruído = descartar
DJEN, conclusos, confirmação de petição, citações eletrônicas da ré, trânsito em julgado. Descarte antes de chamar Claude.
REGRA 4
💰 Multa = sempre Classe E
Qualquer intimação com multa é Classe E (estratégico) — nunca automatizar, nunca reclassificar para baixo.
REGRA 5
🔬 Perícia nomeada = desistência
Quando perito é nomeado → desistência obrigatória. Classe E urgente. Alertar João imediatamente.
REGRA 6
🏦 SISBAJUD = alerta imediato
Bloqueio SISBAJUD/BACENJUD = alerta para João Victor no mesmo dia. Não pode aguardar ciclo noturno.
REGRA 7
🤖 Ciência automática = 3 condições
Somente Classe B + confiança ≥ 90% + delay 2h + validação do João. As 3 condições são obrigatórias.
REGRA 8
🔒 Protocolo = João valida primeiro
NUNCA protocolar no EPROC sem validação explícita do João Victor. Sem exceções.
