📚 Playbooks Controller — Plano Completo

Brainstorming + Arquitetura + Frontend + Backend · Gerado em 17/03/2026 · Revisão: Christian + João

🔍 Problema Raiz — Por que os Playbooks precisam existir
    🎥
    Conhecimento preso em vídeos e áudios João grava, eu assisto e absorvo — mas isso não escala. A cada sessão nova, o contexto pode degradar. O conhecimento precisa estar num banco de dados estruturado, não em arquivos de mídia.
    🤖
    JARBAS segue prompts estáticos — sem manual editável As regras de triagem estão hardcoded no código. João não consegue ajustar sem chamar o Christian. O objetivo: João edita direto na interface, JARBAS lê e executa.
    📝
    Novos tipos de intimação → conhecimento perdido Quando JARBAS encontra um tipo novo, não tem onde cadastrar a regra. Precisa de uma interface onde JARBAS mesmo propõe o playbook e João confirma/ajusta.
    🗣️
    João pensa em voz, não em texto estruturado A forma natural do João comunicar regras é falando. Pedir que ele escreva prompts formatos vai gerar atrito. Solução: ele fala, IA organiza o prompt.
💡 Brainstorming — Minha Visão do Projeto Ideal
🏗️ Banco de regras vivo, não hardcode

Tabela controller_playbooks no Supabase. Cada linha = uma instrução. JARBAS consulta antes de classificar. João pode adicionar, editar, desativar.

🎤 Input por voz com transcrição nativa

Botão de microfone na interface → Whisper transcreve → texto aparece no campo. João fala natural: "quando chegar réplica do Bradesco, lança pra Stephanie com 15 dias"

✨ Botão "Melhorar Prompt" com IA

João digitou/falou algo humanizado → clica "Melhorar" → Claude/GPT transforma em instrução estruturada: trigger, ação, responsável, prazo, nível de confiança. João revisa e salva.

🤖 JARBAS propõe novos playbooks automaticamente

Quando JARBAS encontra um tipo de intimação sem playbook correspondente → cria rascunho automático → aparece na aba com badge "Sugerido por JARBAS" aguardando aprovação do João.

📊 Métricas de eficácia por playbook

Cada playbook mostra: quantas vezes foi acionado, % de acertos (João não corrigiu), última vez usado. João sabe o que funciona e o que precisa ajuste.

🔗 n8n como engine de IA (opcional)

Alternativa: o agente de melhoria de prompt pode rodar direto no frontend via API Anthropic/OpenAI, sem n8n. Mais simples, menos latência. n8n só se precisar de pipeline complexo.

✅ Por que essa abordagem resolve o problema

  • João edita regras sem precisar chamar Christian
  • JARBAS sempre atualizado sem redeploy
  • Input por voz elimina fricção de escrever prompts
  • Sugestões automáticas do JARBAS = aprendizado contínuo
  • Rastreabilidade: quem criou, quando, quantas vezes usou
  • Desativa regra sem deletar (histórico preservado)

⚠️ Riscos e mitigações

  • Regra mal feita → JARBAS errar → revisar antes de ativar
  • Conflito entre regras → prioridade explícita por campo
  • João criar regra ambígua → IA detecta e avisa ao salvar
  • Custo API melhoria de prompt → estimativa ~R$0,02/uso

🏗️ Arquitetura do Sistema
🎨 Layer 1 — Frontend (React / MdFlow) ~/beta-mdflow
  • Nova aba Playbooks em /pages/controller/ControllerPlaybooks.tsx
  • Lista de playbooks com status (ativo/pausado/sugerido), métricas de uso, botões editar/pausar/aprovar
  • Formulário de criação com botão 🎤 Gravar → Web Speech API / MediaRecorder → transcreve via Whisper edge function
  • Botão ✨ Melhorar com IA → chama edge function improve-playbook-prompt → retorna estrutura normalizada
  • Preview do prompt melhorado antes de salvar (João revisa o que a IA fez)
  • Badge 🤖 Sugerido pelo JARBAS para playbooks pendentes de aprovação
  • Realtime Supabase: novos playbooks sugeridos aparecem sem refresh
🗃️ Layer 2 — Banco de Dados (Supabase) qdivfairxhdihaqqypgb
  • Nova tabela controller_playbooks — ver schema completo abaixo
  • Tabela controller_playbook_execucoes — log de cada vez que um playbook foi aplicado
  • RLS: leitura pública (org), escrita apenas para roles controller/admin
  • Realtime habilitado na tabela playbooks
⚡ Layer 3 — Edge Functions (Supabase) 2 funções
  • transcribe-voice — recebe áudio (blob), chama Whisper, retorna texto
  • improve-playbook-prompt — recebe texto humanizado, chama Claude Haiku, retorna JSON estruturado com trigger/ação/responsável/prazo/confiança
🤖 Layer 4 — JARBAS (skill atualizada) skill_captura_intimacoes.mjs
  • Antes de classificar cada intimação: consulta controller_playbooks ativos
  • Match por keywords do trigger (texto da intimação) → aplica ação correspondente
  • Se não encontrar playbook → cria sugestão automática na tabela (status sugerido)
  • Registra execução na tabela controller_playbook_execucoes para métricas
  • Fallback: se nenhum playbook cobre → lógica hardcode atual (retrocompatível)
