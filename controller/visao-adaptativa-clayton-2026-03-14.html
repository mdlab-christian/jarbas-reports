🧠 Auto-melhoria contínua

À medida que o Clayton executa e a memória acumula acertos, o JARBAS pode gerar automaticamente novos seletores Playwright a partir das decisões Vision de alta confiança. O que hoje precisa de Vision, amanhã já é Playwright puro — sem nenhuma intervenção manual.

🔧 Melhorias da Lapidação

O que precisamos implementar para a Visão Adaptativa ficar 100%

01

Contexto rico — passar URL + título da página + histórico de ações

Hoje o Vision recebe só "contexto: ciencia". Precisa receber: URL atual, título da página, últimas 3 ações executadas, e o que o Clayton estava tentando fazer. Isso reduz erros de interpretação.

const ctx = {
  url: await page.url(),
  titulo: await page.title(),
  historico_acoes: ultimasAcoes.slice(-3),
  objetivo: contexto,
};
02

Validação CSS no DOM antes de clicar

Quando Vision sugere um seletor CSS, confirmar que ele existe no DOM antes de executar. Evita clicar no lugar errado quando o seletor não bate.

const existe = await page.$(decisao.seletor_css);
if (!existe) {
  // fallback para coordenadas ou nova chamada Vision
}
03

Retry inteligente — 2 tentativas com novo screenshot

Se Vision executar e falhar (elemento sumiu, clicou errado), tirar novo screenshot e tentar mais uma vez com contexto adicional: "Tentei clicar em X mas não funcionou. O que está errado?"

04

Modelo adaptativo por complexidade — Sonnet para casos simples

Botão deslocado → Sonnet 4.6 (R$0,02). Fluxo complexo com múltiplos passos → Opus (R$0,50). Economia de ~80% nos custos Vision sem perder precisão.

const modelo = ehCasoSimples(contexto)
  ? 'claude-sonnet-4-6'   // ~R$0,02/screenshot
  : 'claude-opus-4-5';    // ~R$0,50/screenshot
05

Auto-geração de seletor Playwright a partir da memória

Quando uma memória atinge 10+ acertos com 85%+ de confiança, o JARBAS gera automaticamente um novo seletor Playwright e atualiza o skill. Da próxima vez, nem chega na camada Vision.

06

Notificação ao JARBAS quando Vision é ativado

Toda vez que o Vision é ativado (não a memória — o Vision mesmo), enviar uma notificação para o Telegram com: qual tribunal, qual contexto, o que detectou, se resolveu. Permite monitorar saúde dos tribunais em tempo real.

// Telegram: "⚡ Vision ativado no TJRS
// Contexto: peticionamento
// Detectou: novo botão 'Protocolar' (94%)
// Status: ✅ Resolvido — decisão salva"
