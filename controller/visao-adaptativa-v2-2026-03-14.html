🗺️

1. Contexto Rico novo

O Vision agora recebe URL da página, título, últimas 3 ações da sessão e o erro anterior. Isso reduz drasticamente interpretações erradas — o Claude sabe exatamente onde o Clayton está e o que já tentou.

// v1: contexto genérico
analisarComVision('ciencia', tribunal)

// v2: contexto rico
{
  url: 'https://eproc.tjrs...',
  titulo: 'EPROC - Processo 0001234',
  historico_acoes: ['login (✅)', 'busca (✅)', 'peticionamento (❌)'],
  tentativa: 2, erroAnterior: 'seletor não encontrado'
}
✅

2. Validação CSS no DOM novo

Antes de executar qualquer clique (da memória ou do Vision), o Clayton verifica se o seletor realmente existe na página. Se não existe, tenta o seletor alternativo. Se ambos falharem, vai para o próximo retry com novo screenshot.

const existe = await page.$('button[data-action="protocolar"]');
if (!existe) {
  // tenta alternativa ou novo screenshot
}
🔁

3. Retry Inteligente (2x) novo

Se a primeira tentativa Vision falhar (confiança baixa, seletor errado, execução falhou), o Clayton aguarda 1s, tira novo screenshot e tenta novamente — passando o erro anterior para o Vision não repetir o mesmo caminho.

for (let tentativa = 1; tentativa <= MAX_RETRIES_VISION; tentativa++) {
  // novo screenshot + erro anterior como contexto
  const { decisao } = await analisarComVision({ erroAnterior });
  if (sucesso) break;
  await sleep(1000); // aguarda antes do próximo retry
}
💰

4. Modelo Adaptativo novo

Contextos simples (login, confirmação, listagem, busca) usam Sonnet 4.6 (~R$0,02/screenshot). Contextos complexos (peticionamento, ciência, fluxos multi-passo) usam Opus (~R$0,50). Economia estimada de 80% nos custos Vision.

📲

5. Notificação Telegram novo

Toda vez que o Vision é ativado (não a memória — o Vision mesmo), o JARBAS enfileira uma notificação no Telegram informando tribunal, contexto, o que detectou, se resolveu e o custo. Monitoramento em tempo real da saúde dos tribunais.

⚡ Vision ativado — TJRS
Contexto: `peticionamento`
Detectou: botão 'Protocolar' na barra superior
Status: ✅ Resolvido (claude-opus-4-5)
Custo estimado: ~R$0,50
🚀

6. Auto-Promoção para Playwright novo

Quando uma memória atinge 10+ acertos com 90%+ de confiança, ela é marcada com promover_para_playwright: true. O JARBAS pode então ler esses candidatos e gerar automaticamente novos seletores Playwright — tornando aquele fluxo mais rápido e sem custo para sempre.

// Nova função: listar candidatos à promoção
const candidatos = await listarCandidatosPromocao();
// → memórias com 10+ acertos e 90%+ confiança
// → pronto para gerar novo seletor Playwright

🧠 Filosofia da Visão Adaptativa

O Clayton não é mais um robô frágil que trava com qualquer mudança no EPROC. Ele é um agente que vê, interpreta e resolve — como um humano faria olhando para a tela. A diferença é que o Clayton nunca esquece: cada problema resolvido vira conhecimento permanente. Com o tempo, os tribunais param de ser fontes de falhas e passam a ser territórios mapeados, onde o Clayton navega com total segurança.
