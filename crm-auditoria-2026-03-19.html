Webhook de geração de contrato sem URL → trava toda a geração

Causa raiz confirmada no código | ENV missing
ENV Front Bloqueante
▾
O que encontrei:
O hook useGerarContrato.ts lê import.meta.env.VITE_N8N_CONTRATO_WEBHOOK_URL para saber para onde mandar o payload. Se a variável não existe, ele lança um toast de erro imediatamente e retorna false.

Resultado: clicar em qualquer botão "Gerar Contrato" ou "Gerar Procuração" mostra o erro "URL do webhook não configurada" e nada é gerado.

Confirmado: o arquivo .env do projeto tem VITE_N8N_WEBHOOK_URL mas NÃO tem VITE_N8N_CONTRATO_WEBHOOK_URL.

Agravante: o workflow n8n WH-DOC-CONTRATO-HONORARIOS (ID: 5M8iuY8Uq8lSCHKo) está INATIVO — mesmo que a variável existisse, o webhook não responderia.
✅ Fix — 2 passos obrigatórios
  • Passo 1: Criar o workflow n8n completo (ver P1-A abaixo)
  • Passo 2: Adicionar no ~/beta-mdflow/.env e .env.production:
    VITE_N8N_CONTRATO_WEBHOOK_URL="https://primary-production-e209.up.railway.app/webhook/gerar-contrato"
  • Passo 3: git push → Lovable faz rebuild com a variável
⚠️ Atenção

Variáveis VITE_ precisam estar no .env do repositório E configuradas no ambiente de build do Lovable. Só colocar no arquivo local não basta para produção.

Workflow n8n de contratos está inativo e vazio — precisa recriar

ID: 5M8iuY8Uq8lSCHKo | Status: INATIVO | Conteúdo: vazio
n8n Bloqueante para geração
▾
O que encontrei:
O workflow WH-DOC-CONTRATO-HONORARIOS existe no n8n mas está INATIVO e com 0 nodes funcionais (foi criado como placeholder vazio).

Sem esse workflow, o botão "Gerar Contrato" no front nunca vai funcionar, mesmo com a variável ENV corrigida.

O que o workflow precisa fazer:
1. Receber POST /webhook/gerar-contrato com {lead_id, restricao_id, tipo, oab}
2. Buscar dados do lead + restrição no Supabase
3. Buscar template em documento_templates pelo tipo + OAB
4. Usar Google Docs API ou gerar PDF via HTML
5. Salvar no Supabase Storage + processos_documentos
6. Retornar {success: true, url: "URL_DO_PDF"}

Bloqueante crítico: Google OAuth2 precisa estar configurado no n8n para usar Google Docs API. Se não estiver, precisamos de alternativa (HTML→PDF via Puppeteer no n8n ou via API externa).
✅ Plano de implementação
  • Verificar se Google OAuth2 está configurado no n8n (GET /api/v1/credentials)
  • Se sim → recriar workflow usando Google Docs createFromTemplate + export PDF
  • Se não → usar abordagem alternativa: gerar PDF via template HTML + wkhtmltopdf ou html-pdf no n8n
  • Ativar o workflow após criação
  • Testar com payload real do leadfix3
⚠️ Decisão pendente com Christian

Se optar por Google Docs API: precisa do OAuth2 configurado (processo manual no n8n UI). Se optar por HTML puro: mais simples de implementar mas pode perder formatação dos documentos word.

Geração de contratos: Google Docs API vs HTML puro?

✅ Google Docs API (igual ao Make)

  • + Mantém formatação exata dos .docx
  • + Templates já existem no Google Docs
  • + Mesmo fluxo do Make = mais seguro
  • − Requer OAuth2 configurado no n8n
  • − Mais complexo de montar

🔵 HTML puro (mais simples)

  • + Sem OAuth2 necessário
  • + Mais rápido de implementar
  • + Sem dependência do Google
  • − Precisa converter os .docx para HTML
  • − Pode perder alguma formatação
