Achado de Schema — Documentado

⚠️ Campo arquivado é boolean, não status string

O CC identificou durante a execução que o schema real usa arquivado (boolean) e coluna_crm = 'arquivados' (plural, conforme CHECK constraint) — não status = 'arquivado' como estava nos SQLs iniciais. A correção foi aplicada silenciosamente durante a execução e documentada na migration 20260324000003_cpf_duplicatas_resolvidas.sql. Verificar se há outros pontos no frontend/backend que assumem status = 'arquivado' incorretamente — candidato para próxima investigação.
