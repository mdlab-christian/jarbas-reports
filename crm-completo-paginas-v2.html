📌 Regra de Distribuição
  • Cada restrição = 1 processo individual
  • Botão Distribuir aparece só em leads na Triagem
  • Ao distribuir: lead → ARQUIVADOS (motivo: "Cliente convertido")
  • Lead se torna Cliente no módulo Clientes
💰 Regra de Valor
  • CRM → mostra valor da restrição original (ex: R$ 230)
  • Distribuição → ainda mostra valor da restrição
  • Ao distribuir → salva valor_causa = restricao.valor + R$15.000
  • Processo → mostra valor_causa (ex: R$ 15.230)
🔗 Regra de Documentos
  • Tipo 1 (específico/restrição): sobe vinculado à restrição → vai só pro processo dela
  • Tipo 2 (específico/processo): sobe no processo → aparece no processo e na restrição
  • Tipo 3 (geral/lead): sobe sem vínculo → vai pra TODOS os processos do cliente

⚡ Migrations Necessárias (E28)
3 alterações no banco — executadas na E28 antes de qualquer frontend

Migration 1 — FK orgão restritivo em restrições

Hoje o órgão restritivo é só um campo texto. Precisa de FK para poder mostrar dados completos do órgão (CNPJ, processavel) em todos os drawers.

ALTER TABLE restricoes ADD COLUMN orgao_id UUID REFERENCES orgaos_restritivos(id); -- Popular registros existentes por nome: UPDATE restricoes r SET orgao_id = o.id FROM orgaos_restritivos o WHERE r.orgao_nome = o.nome;

Migration 2 — restricao_id em processos_documentos

Para suportar documentos vinculados a restrições específicas (Tipo 1 e 2 da lógica de docs).

ALTER TABLE processos_documentos ADD COLUMN restricao_id UUID REFERENCES restricoes(id);

Migration 3 — tipo_documento (nome exato)

Campo para armazenar o nome exato do documento (ex: "Contrato de Honorários – Pessoa Física"). Lista de nomes a confirmar com Christian via Drive.

ALTER TABLE processos_documentos ADD COLUMN tipo_documento TEXT; -- Exemplos de valores esperados: -- 'Identidade — {nome_lead}' -- 'Comprovante de Residência — {nome_lead}' -- 'Contrato de Honorários' -- 'Procuração — {empresa/orgao}'
📎 Lógica de Documentos — 3 Tipos
Como documentos fluem entre CRM, Distribuição, Processo e Cliente
Tipo Como entra Onde fica salvo Onde aparece
Tipo 1
Específico por restrição		 Upload no CRM vinculado à restrição X processos_documentos WHERE restricao_id = X Só no processo da restrição X (quando distribuído)
Tipo 2
Específico por processo		 Upload direto no processo processos_documentos WHERE processo_id = P No processo P + na restrição vinculada a P
Tipo 3
Geral do lead/cliente		 Upload no lead sem vínculo de restrição processos_documentos WHERE lead_id = L, restricao_id IS NULL Em TODOS os processos e restrições do cliente
⚠️ Documentos Numerados (3-5)
  • Doc 3: id-3-{nome_lead} → Identidade/CNH
  • Doc 4: comprovante-residencia-{nome_lead} → Comprovante de Residência
  • Doc 5: variável — múltiplos possíveis (CTPS, sit. cadastral, renda)
  • Estes são Tipo 3 (gerais) — vão para todos os processos do cliente
  • Nomes exatos a confirmar via Drive (5 pastas Christian + 5 Heloísa)
