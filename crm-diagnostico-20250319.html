2. Diagnóstico técnico — o que o código confirma

P0 BUG CONFIRMADO

Gerar Contrato / Procuração sem URL

O hook useGerarContrato.ts depende de VITE_N8N_CONTRATO_WEBHOOK_URL. Esta variável não existe no .env local nem de produção.

// useGerarContrato.ts linha 31-34 const webhookUrl = import.meta.env.VITE_N8N_CONTRATO_WEBHOOK_URL if (!webhookUrl) { toast.error('URL do webhook não configurada (VITE_N8N_CONTRATO_WEBHOOK_URL)') return false }

Estado atual do .env:

VITE_N8N_WEBHOOK_URL="https://primary-production-e209.up.railway.app/webhook" VITE_N8N_WEBHOOK_SECRET="MdL1501@" # VITE_N8N_CONTRATO_WEBHOOK_URL → ❌ NÃO EXISTE

Impacto: Toda tentativa de "Gerar Contrato" ou "Gerar Procuração" falha imediatamente com toast de erro, sem nem chegar ao n8n.

Caminhos afetados: Tab "Documentos" do drawer de lead CRM → botões Gerar Contrato e Gerar Procuração por restrição + botão "Gerar Tudo".

P1 BUG CONFIRMADO

Botão "Gerar Contrato" — cor errada

O botão principal de contrato usa variant={temContrato ? 'outline' : 'default'}. Quando não há contrato gerado, fica default (cinza/neutro no Design System v5). A especificação pede botão com cor primária (teal #20c997).

// ClienteDrawerTabDocumentosLead.tsx linha ~280 <Button variant={temContrato ? 'outline' : 'default'} // ← deveria ser 'default' com className text-primary ou variant próprio >

O variant="default" no Design System v5 resulta em bg-primary... mas o botão de procuração usa variant="outline" sempre — sem destaque visual.

P1 BUG CONFIRMADO

Documentos Pendentes — formato não conforme

A página /documentos-pendentes usa uma estrutura de accordion agrupado por cliente com processos aninhados. O formato combinado (cards visuais por tipo de documento, com prazo, urgência e ação rápida por linha) não foi implementado.

O que existe hoje:

  • Accordion por cliente → Accordion por processo → lista plana de pendências
  • Cada item mostra: checkbox, nome do documento, prazo, status badge, botões de ação
  • Sem cards visuais, sem agrupamento por tipo, sem destaque de prazo crítico no card principal

O PendenciaItem.tsx existe mas não tem o layout de "card por documento" que foi acordado — é uma linha compacta.
