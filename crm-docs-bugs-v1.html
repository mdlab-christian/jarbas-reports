🔴 Bug #1 — Botão "Baixar" não baixa (abre nova aba igual ao Abrir)
Prioridade: P0 • Descoberto por: Gustavo (teste direto) • Causa: código incorreto
P0

Comportamento atual: Clicar em "Baixar" abre o arquivo em nova aba igual ao botão "Abrir". Nenhum download acontece.

Arquivo afetado:

src/components/shared/cliente_drawer/tabs/DocumentoItem.tsx

Causa raiz (linhas ~55–60): o handleBaixar usa window.open idêntico ao handleAbrir:

❌ Código atual (errado): 
const handleBaixar = async () => {
  const url = await resolveUrl()
  // mesmo código do handleAbrir ↓
  if (url) window.open(url, '_blank', 'noopener,noreferrer')
  else toast.error('Documento sem URL disponível')
}
✅ Código correto (fix): 
const handleBaixar = async () => {
  const url = await resolveUrl()
  if (!url) { toast.error('Documento sem URL disponível'); return }
  try {
    const response = await fetch(url)
    const blob = await response.blob()
    const blobUrl = URL.createObjectURL(blob)
    const a = document.createElement('a')
    a.href = blobUrl
    a.download = doc.nome || 'documento'
    document.body.appendChild(a)
    a.click()
    document.body.removeChild(a)
    URL.revokeObjectURL(blobUrl)
  } catch {
    // fallback CORS: link direto com atributo download
    const a = document.createElement('a')
    a.href = url; a.download = doc.nome || 'documento'
    a.target = '_blank'
    document.body.appendChild(a); a.click(); document.body.removeChild(a)
  }
}

src/components/shared/cliente_drawer/tabs/documentos_lead_hooks.ts

Como deveria funcionar:

  • Upload central → salva com restricao_id = NULL + categoria = 'documento_lead'

Hipótese B: O queryClient.invalidateQueries após upload só invalida a query central, não as pastas individuais de cada restrição:

❌ Provável problema atual:

queryClient.invalidateQueries({ queryKey: ['lead-docs-upload', lead.id] })

restricoes.forEach(r => {
  queryClient.invalidateQueries({
    queryKey: ['restricao-docs-upload', r.id, lead.id, organizacaoId]
  })
})

O hook useRestricaoDocs já busca os Tier 1 corretamente — provavelmente só falta o invalidate trigger.

🟡 Bug #4 — Upload não converte imagem para PDF
Prioridade: P1 • Descoberto por: Gustavo (arquivo JPG enviado) • Causa: falta implementação
P1

Comportamento atual: Ao fazer upload de JPG/PNG, o arquivo é salvo como imagem. Deveria ser convertido para PDF automaticamente.

Arquivo afetado:

src/hooks/useDocumentUpload.ts

Solução: Instalar jsPDF e converter antes do upload para storage:

✅ Implementação (usando jsPDF): 
// npm install jspdf
import jsPDF from 'jspdf'

async function converterParaPdf(file: File): Promise<File> {
  if (!file.type.startsWith('image/')) return file // PDF/DOC → pass-through
  return new Promise((resolve) => {
    const reader = new FileReader()
    reader.onload = (e) => {
      const img = new Image()
      img.onload = () => {
        const pdf = new jsPDF({
          orientation: img.width > img.height ? 'landscape' : 'portrait',
          unit: 'px', format: [img.width, img.height]
        })
        pdf.addImage(e.target?.result as string, 'JPEG', 0, 0, img.width, img.height)
        const blob = pdf.output('blob')
        const nome = file.name.replace(/\.[^.]+$/, '.pdf')
        resolve(new File([blob], nome, { type: 'application/pdf' }))
      }
      img.src = e.target?.result as string
    }
    reader.readAsDataURL(file)
  })
}

Chamar converterParaPdf(file) ANTES do upload para o Storage no useDocumentUpload.ts.
