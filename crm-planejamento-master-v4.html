📄 Lógica Completa de Documentos
Como documentos se propagam entre CRM, restrições e processos
Regra geral: O campo tipo_documento deve existir obrigatoriamente para mapear o nome exato de cada documento. Nomes vêm do padrão das pastas no Drive dos processos "Aguardando Distribuição" (ver 10 pastas — 5 Christian + 5 Heloísa).
Tipo de Upload
Comportamento
Destino
Documento geral do lead
(nível cliente)
Uploaded no lead, sem vínculo com restrição específica. Serve para TODOS os processos daquele cliente.
→ Todas as restrições do cliente
Documento por restrição específica
Vinculado a uma restrição X específica. Ao distribuir, vai junto para o processo daquela restrição. Se múltiplos docs de restrições → vão em conjunto na distribuição.
→ Só aquela restrição/processo
Upload geral (fora de restrição)
(drag and drop avulso)
Documento enviado sem contexto de restrição. Deve ser vinculado a TUDO — todos os processos e todas as restrições do cliente.
→ Todos processos + todas restrições
Ao distribuir processo
Documentos da restrição original são copiados/revinculados para o processo criado. Busca pela restrição original para localizar os documentos.
→ Processo criado na distribuição
