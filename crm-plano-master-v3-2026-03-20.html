📁 Sistema de Documentos — Conceito Central

A lógica que se replica em todos os drawers e páginas

Conceito

A aba de Documentos funciona como um gerenciador de arquivos: uma área de upload central no topo + pastas (uma por restrição processável) abaixo. Documentos no upload central vão para todas as pastas. Documentos arrastados sobre uma pasta vão só para ela. Cada pasta é um accordion que abre para mostrar os arquivos vinculados. Esta lógica é idêntica em CRM, drawer do cliente, drawer do processo, página cliente e página processo — os dados são sincronizados.

🔗 Vinculação de Pastas entre Páginas

Upload ou geração em qualquer lugar = atualiza em todos

🎙️ Regra de Sincronização

As pastas de documentos são vinculadas pelo mesmo lead_id / restricao_id / processo_id. Upload ou geração em qualquer contexto (CRM, Cliente, Processo) atualiza a visão em todos os demais. Não há cópia de dados — o mesmo registro em processos_documentos é exibido em múltiplos contextos.

Diagrama de Vinculação

AçãoOnde salva no bancoAparece em
Upload central no Drawer CRM processos_documentos (restricao_id=null, lead_id=X) Drawer CRM · Drawer Cliente · Pág. Cliente
Upload individual no Drawer CRM (pasta Boa Vista) processos_documentos (restricao_id=Y, lead_id=X) Drawer CRM (pasta Y) · Drawer Cliente (pasta Y) · Pág. Cliente (pasta Y)
Gerar Petição no Drawer Distribuição processos_documentos (processo_id=Z, categoria='peticao_inicial') Drawer Distribuição · Drawer Processo · Pág. Processo · Pág. Cliente
Distribuição (RPC) Docs Tier 1 copiados para cada processo criado Todos os drawers/páginas dos processos criados

fn_documentos_por_contexto — uso por página

ContextoParâmetros passadosRetorna
Drawer CRM (lead) p_lead_id=X, p_cliente_id=null, p_restricao_id=null Docs do lead (Tier 1 + Tier 2 + Tier 3)
Drawer Cliente / Pág. Cliente p_lead_id=null, p_cliente_id=C, p_restricao_id=null Todos os docs do cliente (inclui pós-processo)
Drawer Processo / Pág. Processo p_lead_id=null, p_cliente_id=null, p_restricao_id=R + processo_id=P Docs daquele processo + pasta da restrição específica
