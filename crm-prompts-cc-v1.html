- Imagens: `image/jpeg`, `image/png`, `image/webp`

   a) Bloqueia qualquer arquivo cujo `file.type` começa com `application/vnd.google-apps`
   b) Exibe mensagem clara: "Arquivos do Google Docs/Sheets não são suportados. Por favor, baixe o arquivo como PDF ou .docx antes de enviar."
   c) Impede que o upload prossiga se o arquivo for Google nativo
4. Idealmente, também atualize o atributo `accept` do `` para listar apenas os tipos aceitos
5. Se houver um drag-and-drop, aplique a mesma validação

## Critério de sucesso

# TAREFA: Corrigir lookup de empresas para usar variações de nome no fluxo Nova Auditoria → Criar Lead

## Contexto
No fluxo de Nova Auditoria, quando o sistema processa o resultado da consulta (bureau PFU/Boa Vista/Serasa),
ele recebe o nome da empresa que consta no comprovante de restrição.
Esse nome precisa ser mapeado para o nome oficial da empresa cadastrada no sistema.

**Problema:** A consulta retorna nomes como "OmniBanco" ou "Banco Santander S.A." mas o sistema tem
cadastrado "Banco Bradesco S.A." ou "Santander" (nomes diferentes). O lookup falha porque compara
o nome exato sem usar as variações.

**Histórico:** O agente JARBAS fez uma varredura em todos os processos com pasta vinculada no Google Drive,
comparando nomes dos comprovantes com nomes do sistema, e salvou essas variações na tabela de empresas.

## O que investigar

### 1. Encontrar onde as variações estão salvas
1. Abra a tabela de empresas no banco (provavelmente `empresas`, `credores`, `orgaos` ou similar)
2. Procure campos como: `variacoes`, `nomes_alternativos`, `nome_fantasia`, `aliases`
3. Confirme que existem registros com variações populadas (ex: empresa "Banco Bradesco S.A." tem variação "OmniBanco")

### 2. Encontrar onde o lookup é feito
1. Procure a função que recebe o nome da empresa da consulta e tenta encontrar no banco
2. Pode ser: `findEmpresaByNome`, `matchEmpresa`, uma query Supabase com `.eq('nome', nomeConsulta)` ou RPC

### 3. Corrigir o lookup para usar variações
O lookup atual provavelmente faz:
```sql
SELECT * FROM empresas WHERE nome = 'OmniBanco'
-- Não encontra porque o nome cadastrado é "Banco Bradesco S.A."
```

Deve ser corrigido para:
```sql
-- Opção A: buscar por nome OU em array de variações
SELECT * FROM empresas 
WHERE nome ILIKE '%OmniBanco%' 
   OR nome_fantasia ILIKE '%OmniBanco%'
   OR 'OmniBanco' = ANY(variacoes)

-- Opção B: se variações for JSONB array
SELECT * FROM empresas 
WHERE nome = 'OmniBanco'
   OR variacoes @> '["OmniBanco"]'::jsonb

-- Opção C: criar uma função no banco para fazer o match
```

4. Confirme qual é o campo e formato das variações e aplique o lookup correto

### 4. Fallback quando não encontrar
Se após buscar com variações ainda não encontrar a empresa:
- Mostrar na UI o nome como veio da consulta e permitir que o usuário selecione/corrija manualmente
- NÃO quebrar o fluxo — deixar o usuário prosseguir com o nome da consulta

## Critério de sucesso
- "OmniBanco" é corretamente mapeado para "Banco Bradesco S.A." (ou equivalente)
- O lookup de empresas usa as variações de nome cadastradas
- Quando não encontrar correspondência, permite correção manual sem quebrar
