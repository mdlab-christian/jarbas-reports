CRM-02

Validação de advogado na RPC de distribuição

✅ OK
✅
Investigação confirmou: RPC não validava advogado_id antes da correção A função recebia p_lead_id + p_organizacao_id apenas. Passava v_lead.advogado_id (possivelmente NULL) para fn_criar_processo_distribuicao sem guard.
✅
Validação adicionada no banco (camada correta) Guard inserido na RPC logo após IF NOT FOUND, antes de qualquer operação de escrita. Aplicado via Supabase MCP em qdivfairxhdihaqqypgb.
✅
Mensagem amigável retornada ao usuário RPC retorna {success: false, message: "Lead sem advogado vinculado. Vincule um advogado antes de distribuir."} — capturado pelo frontend em ModalDistribuirLead.tsx:77 e exibido via toast.error.
✅
Comentário inline documenta a decisão -- 2. Validar advogado — obrigatório para criar processos [CRM-02] no arquivo de migration local. Arquivo sincronizado com o banco.
Arquitetura da correção: A validação ficou no banco (não no frontend) porque é a camada canônica para regras de negócio no MdFlow (princípio P-02: lógica nunca no React). O frontend já possui gate soft no ModalDistribuicao (checklist), mas a RPC agora é a última linha de defesa — garante integridade independente de qualquer caminho de UI.
