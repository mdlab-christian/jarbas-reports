🔍 Causas do Gasto Elevado

1. Restarts de sessão = rebuild de cache (causa principal)

O backfill AdvBox causou múltiplos restarts de sessão (o processo morria por timeout). Cada restart cria uma nova sessão do zero — sem cache do Anthropic aproveitável — e o custo de cacheWrite é pago novamente para recarregar todo o system prompt (SOUL.md, MEMORY.md, AGENTS.md, TOOLS.md, etc). Com 100k tokens de contexto, cada nova sessão custa ~$0.10–0.30 só para montar o contexto.

2. Processamento de arquivo ZIP grande (103k tokens/call)

A sessão b212 processou o ZIP das conversas Wati com 103k+ tokens por call — 7 calls = $1.03. O conteúdo do arquivo inflou o contexto permanentemente nessa sessão.

3. Contexto acumulado de 100k tokens

Config atual: contextTokens: 100000. A sessão principal acumulou 2.5M tokens ao longo do dia. O pruning de keepLastAssistants: 3 ajuda, mas o system prompt já pesa ~40k tokens isolado.

4. Reset de sessão: 30 dias

idleMinutes: 43200 (30 dias). Sessões nunca resetam — context acumula indefinidamente dentro de uma sessão longa.

5. Crons ativos (custo controlado)

Os crons (GROUP_CONTEXT_CHECKPOINT a cada 3h + RELATORIO_CUSTO_DIARIO às 23h) são normais e custaram apenas $1.21 na madrugada. Não são o problema.

🛠️ Recomendações de Otimização

  • 🔴
    Reduzir contextTokens de 100k para 60-80k P0
    O system prompt do JARBAS (SOUL.md + MEMORY.md + AGENTS.md + TOOLS.md) pesa ~40-50k tokens. Reduzir para contextTokens: 80000 no config ainda tem margem confortável e reduz o custo de cacheWrite por sessão em ~20%.

    Config: agents.defaults.contextTokens: 80000
  • 🔴
    Workspace context: manter abaixo de 20KB P0
    MEMORY.md menciona a regra: "Workspace context > 20KB = flood". Verificar tamanho atual dos arquivos de workspace. SOUL.md sozinho tem 4.5KB; MEMORY.md tem 5KB; AGENTS.md tem ~6KB — estão no limite. Considerar mover seções menos usadas para arquivos auxiliares não carregados no context window.
  • 🟡
    Evitar processar arquivos grandes inline P1
    O ZIP da Wati com 103k tokens por call custou $1.03 em 7 calls. Para arquivos grandes (>50k tokens), usar sessions_spawn com sessão isolada que descarta o contexto ao terminar — não contamina a sessão principal.
  • 🟡
    Para tarefas longas como backfill: usar script standalone P1
    O backfill AdvBox custou ~$4 pela combinação de múltiplos restarts de sessão. O script já existe e roda standalone (nohup node backfill-advbox.mjs) — não precisa de monitoramento inline no chat a cada 10 minutos.
  • 🟢
    MEMORY.md: compactar seções menos usadas P2
    Seções como "OABs / EPROC" e "Lições Consolidadas" em AGENTS.md são carregadas em toda sessão mas raramente usadas em DMs. Avaliar mover para skill específica ou arquivo carregado on-demand.
  • 🟢
    Heartbeat: manter em 2h (atual é bom) P2
    O heartbeat a cada 2h é adequado. Não reduzir — cada heartbeat consome ~$0.05-0.10 pela sessão acumulada.
