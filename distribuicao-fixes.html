⚠️ Fix-A — Drawer fecha antes da mutation completar

PARCIAL P2-C · TabPreparacao.tsx · UX de erro degradada

Problema

Em TabPreparacao.tsx, o setDrawerOpen(false) ocorre antes da mutation enviarParaDistribuicao.mutate(id). Se a mutation falhar, o drawer já fechou e o toast de erro aparece sem contexto visual.

Código atual (linhas 253–256):
onEnviarParaDistribuicao={(id) => { setDrawerOpen(false); setDrawerProcessoId(null); enviarParaDistribuicao.mutate(id); }}
Comportamento esperado: Drawer fecha após sucesso da mutation. Em caso de erro, o drawer permanece aberto e o toast de erro aparece com contexto.

Prompt CC

Arquivo: src/pages/distribuicao/components/TabPreparacao.tsx PROBLEMA: O drawer de preparação fecha ANTES da mutation completar. Se a mutation falhar, o drawer já fechou e o toast de erro aparece "solto" sem contexto visual. CORREÇÃO NECESSÁRIA: 1. Remover setDrawerOpen(false) e setDrawerProcessoId(null) do callback onEnviarParaDistribuicao 2. Mover ambos para dentro do onSuccess da mutation enviarParaDistribuicao ANTES (padrão atual): ``` const enviarParaDistribuicao = useMutation({ mutationFn: ..., onSuccess: () => { queryClient.invalidateQueries(...); toast.success('Processo enviado para distribuição'); // drawer NÃO fecha aqui }, }); onEnviarParaDistribuicao={(id) => { setDrawerOpen(false); // ← fecha antes de saber se deu certo setDrawerProcessoId(null); enviarParaDistribuicao.mutate(id); }} ``` DEPOIS (correção): ``` const enviarParaDistribuicao = useMutation({ mutationFn: ..., onSuccess: () => { setDrawerOpen(false); // ← fecha SÓ após sucesso setDrawerProcessoId(null); queryClient.invalidateQueries(...); toast.success('Processo enviado para distribuição'); }, onError: (error) => { toast.error(`Erro ao enviar para distribuição: ${error.message}`); // drawer permanece aberto para o usuário tentar novamente }, }); onEnviarParaDistribuicao={(id) => { enviarParaDistribuicao.mutate(id); // apenas dispara }} ``` CRITÉRIOS DE SUCESSO: - [ ] Clicar "Enviar para Distribuição" com N8N/Supabase OK → drawer fecha, toast de sucesso aparece - [ ] Simular erro (desativar o workflow ou forçar erro na mutation) → drawer PERMANECE aberto, toast de erro aparece no contexto do drawer - [ ] Sem regressão no fluxo normal de envio para distribuição
