#001 Distribuição não efetiva nos processos agendados
🔴 Urgente Corrigido
✅ Código OK. O ModalEfetuarDistribuicao busca o processo via view_processos_distribuicao sem filtrar por etapa — portanto aceita agendados. A RPC rpc_efetuar_distribuicao_manual só bloqueia se etapa_distribuicao = 'em_distribuicao' ou 'distribuido'. Processos com etapa 'agendado' passam normalmente.
✅ O TabAgendados tem o menu "Efetuar Distribuição" que abre o modal com setDistribProcessoId(p.id) — fluxo correto.
TabAgendados.tsx ModalEfetuarDistribuicao.tsx rpc_efetuar_distribuicao_manual
#002 Comprovantes de restrição não exibidos
🔴 Urgente Corrigido
✅ Lógica implementada. O ModalPreparoProcesso exibe o item "Comprovante de Restrição" no checklist. O botão "Ver" aparece quando viewUrl = comprovanteState.url ?? processo.comprovante_restricao_url. A view já retorna comprovante_restricao_url e comprovante_restricao_status.
✅ O DistribuicaoProcessoCard também tem lógica para abrir o comprovante quando status = 'sucesso' e URL disponível.
⚠️ Atenção: O item só aparece se processo.restricao_tipo não for null. Se o processo não tem restricao_tipo preenchido, o botão "Gerar Comprovante" fica desabilitado e o status é sempre 'pendente'. Confirmar se todos os processos têm esse campo preenchido.
ModalPreparoProcesso.tsx useGerarComprovante.ts comprovante_restricao_url restricao_tipo (obrigatório)
#003 Tipo de restrição / órgão restritivo não exibido
🟠 Alto Corrigido
✅ Campo presente no código. O DistribuicaoProcessoCard exibe processo.restricao_orgao_nome (linha 132–134) e processo.restricao_tipo (linha 242). A view_processos_distribuicao inclui ambos os campos.
ℹ️ A exibição é condicional: restricao_orgao_nome só aparece se não for null. Se o campo estiver vazio no banco, o órgão não aparece — mas isso seria problema de dado, não de código.
DistribuicaoProcessoCard.tsx restricao_orgao_nome restricao_tipo
#004 Botão Exportar não abre nada
🟠 Alto Corrigido
✅ Exportar implementado. Ambas as abas (TabAguardando e TabAgendados) usam o componente ImportExportBar com onExport que chama exportToCSV via csv_utils. O componente tem handleExport com feedback de toast e download.
ℹ️ O export gera CSV com os dados filtrados da listagem atual. Se a lista estiver vazia, exibe toast "Nenhum processo para exportar".
TabAguardando.tsx TabAgendados.tsx ImportExportBar csv_utils.ts
#005 Modal com dois botões X de fechar
🟡 Médio Pendente — verificar na tela
⚠️ Não consigo confirmar só pelo código. Os modais usam DialogContent do shadcn/ui, que por padrão renderiza um botão X automático. Se algum modal também tiver um botão X manual no JSX, apareceria duplicado.
ℹ️ Nos modais auditados (ModalEfetuarDistribuicao, ModalPreparoProcesso, WizardAgendarDistribuicao) não há X manual — apenas o DialogContent padrão + botão "Cancelar" no footer. O X duplo pode ser específico de um modal não auditado, ou de uma versão do componente Dialog que adiciona X automaticamente mesmo quando há botão de fechar customizado.
❌ Ação necessária: Gustavo precisa identificar em qual modal específico aparece o X duplo para corrigir. Quando identificar, a correção é adicionar hideCloseButton prop ao DialogContent (se suportado) ou garantir que não há dois componentes de fechamento montados.
Dialog.tsx (shadcn) Precisa de teste na tela Modal a identificar
#006 Agendamento para o mesmo dia (hoje)
🔵 Investigar Parcialmente ok
✅ minDate = startOfDay(hoje). O WizardAgendarDistribuicao define minDate = startOfDay(new Date()) e usa disabled={(d) => d < minDate}. Isso significa: hoje está habilitado (não é menor que startOfDay de hoje).
ℹ️ Tecnicamente o código permite selecionar hoje. Se o comportamento ainda parece errado na tela, pode ser um problema de timezone (UTC vs BRT) ou de como o datepicker compara as datas. Requer confirmar no teste.
WizardAgendarDistribuicao.tsx startOfDay(new Date()) Verificar timezone
