#007 Card não sai da lista de Agendados após distribuição 🔴 Urgente ✨ Novo
Confirmado em teste — Bug ativo

Ao efetuar distribuição a partir da aba Agendados, o toast de sucesso aparece corretamente ("Processo distribuído com sucesso"), mas o card do processo permanece visível na lista. A invalidação de query/cache não está funcionando para esta aba.

Já confirmado no banco: ambos os processos de teste tiveram etapa_distribuicao = 'distribuido' e distribuido = true após a ação. O problema é puramente de UI — o React Query não está refazendo o fetch após a mutação.

Resultado do Teste (18/03/2026)

✅ Banco OK — processo distribuído com sucesso
❌ UI — card permanece na lista após distribuição
Confirmado via query direta no Supabase por JARBAS

Correção Necessária

Em TabAgendados.tsx, após mutação bem-sucedida de distribuição, garantir que queryClient.invalidateQueries(['processos_distribuicao', 'agendado']) está sendo chamado. Verificar se o onSuccess do modal de distribuição propaga o invalidate para a aba correta.
