⛔ TJRS usa 0610 (Práticas Abusivas). No TJSP isso está errado — usar 022003.

Estratégia de preenchimento

Clicar no jstree NÃO preenche os hidden fields. A estratégia correta é:

  1. Clicar na ⭐ (filtro de favoritos)
  2. Extrair ID numérico do DOM via exibeOcultaInfos()
  3. Preencher hdnCodAssuntoJudicial + hdnIdAssuntoJudicial
  4. Chamar window.transportar()
  5. Fallback: btnIncluirAssunto
// Estratégia TJSP — transportar() após preencher hiddens
const transportarOk = await page.evaluate(({ cod, id, desc, tipo }) => {
  const set = (elId, val) => { const el = document.getElementById(elId); if (el) el.value = val; return !!el; };
  set('hdnCodAssuntoJudicial', cod);    // '022003'
  set('hdnIdAssuntoJudicial', id);      // '826'
  set('txtDesAssunto', desc);           // 'Indenização por dano moral'
  set('hdnTipoAssunto', tipo);          // 'P'
  if (typeof window.transportar === 'function') { window.transportar(); return 'transportar()'; }
  const btn = document.getElementById('btnIncluirAssunto');
  if (btn) { btn.click(); return 'btnIncluirAssunto'; }
  return 'nenhum';
}, { cod: '022003', id: '826', desc: 'Indenização por dano moral', tipo: 'P' });
❌ Gap: O ID numérico 826 é baseado em inferência/extrapolação de testes do TJRS. Ainda não foi testado ao vivo no TJSP. Se 826 não funcionar, precisamos abrir a árvore e extrair o ID do DOM do SP.

8. Gaps e Incertezas — Honesto

❌ CRÍTICO — ID numérico do assunto 022003: O valor 826 foi inferido da estrutura do TJRS. O TJSP pode ter um ID diferente para o mesmo assunto. Precisamos abrir o EPROC SP ao vivo, ir na árvore de assuntos, inspecionar o onmouseenter do nó 022003 e confirmar o ID real.
❌ CRÍTICO — Seletor de unidade: #selInfraUnidades pode não existir em todas as contas SP, ou o value que começa com SP- pode ter outro prefixo. Precisamos verificar ao vivo.
❌ ALTO — Etapa 5 upload: Fine Uploader no TJSP pode ter seletores de tipificação diferentes dos do TJRS. Os campos #txtTipo_N e #selIdSigilo_N precisam ser validados.
❌ MÉDIO — OAB SP: Não temos a OAB SP configurada. O login requer inscrição na seccional SP. Precisamos da OAB SP para poder logar no TJSP.
⚠️ MÉDIO — Réu entidade: Boa Vista, Serasa etc. são tratados como "entidade" (sem CNPJ no EPROC). No TJSP, esse comportamento precisa ser validado — pode ser diferente.

9. Regras de Operação

Regras críticas (toda distribuição)

  • Pasta ativa: /tmp/distribuicao-ativa/ — todos os docs ficam aqui
  • Fonte de verdade: ler a petição inicial da pasta (nunca usar memória/contexto)
  • Antes de finalizar: enviar screenshot + resumo completo para Gustavo/Christian no Telegram
  • Só apagar pasta após confirmação de sucesso pelo humano
  • Anti-litigância: max 1 distribuição por autor+réu por dia
  • Upload: todos os arquivos de uma vez (setInputFiles), nunca em loop
  • Contexto pesado: notificar antes de 80% do limite — nunca perder contexto no meio da distribuição

Fluxo de execução

  1. Baixar docs do processo para /tmp/distribuicao-ativa/
  2. Ler petição para extrair: nome autor, CPF, réu, valor causa, tribunal
  3. Logar no EPROC SP com a OAB SP correspondente
  4. Selecionar unidade SP-1G (#selInfraUnidades)
  5. Navegar para Petição Inicial
  6. Executar Etapa 1 → 2 → 3 → 4 → 5
  7. PARAR antes de finalizar → enviar resumo para humano
  8. Aguardar confirmação → finalizar
  9. Limpar pasta ativa
  10. Atualizar MdFlow com número do processo gerado
