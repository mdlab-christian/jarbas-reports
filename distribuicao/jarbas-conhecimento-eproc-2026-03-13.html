📋 2. E0 — MdFlow (pré-distribuição)
  • Prioridade 1: aba Agendado → se houver processo agendado, pegar esse primeiro
  • Prioridade 2: aba Aguardando Distribuição → pegar o primeiro da fila
  • Abrir drawer do processo → manter aberto durante toda a distribuição
  • Confirmar qual advogado está cadastrado no processo para logar no EPROC correto
  • Verificar se a OAB para o estado está disponível (RS116571, SP535101, etc.)
  • NUNCA distribuir processo contra CEF (Caixa Econômica Federal)
  • Anti-litigância: máximo 1 distribuição do mesmo cliente contra a mesma empresa por dia

❌ O problema do jstree
jstree no EPROC SP não funciona via Playwright. Todos os métodos abaixo falharam e NÃO devem ser usados:
// ❌ NÃO funciona:
page.click('[id="022003_anchor"]')                          // fecha contexto
jQuery('#divArvore').jstree('select_node', '022003')   // retorna []
carregarMaisInfo('022003')                                   // contexto não persiste
✅ RS — Método correto (jstree funciona)
// RS e SC: incluirAssunto() via JavaScript — único método confiável
await page.evaluate((parentId) => {
  window.jQuery('#divArvore').jstree('open_node', parentId);
}, '0220');
await page.waitForTimeout(2000);  // aguardar filhos carregarem

await page.evaluate((nodeId) => {
  incluirAssunto(nodeId);        // função nativa do EPROC
}, '022003');
await page.waitForTimeout(2000);

// Selecionar competência (campo dinâmico que aparece após incluir assunto)
await page.selectOption('#selNumCodCompetencia', '716');
✅ SP — Bypass via campos hidden
SP usa jstree diferente — a única solução confiável é preencher os campos hidden diretamente e chamar transportar().
// SP: bypass completo do jstree — injetar nos campos hidden
await page.evaluate(() => {
  document.getElementById('hdnCodAssuntoJudicial').value = '022003';
  document.getElementById('hdnIdAssuntoJudicial').value  = '826';
  document.getElementById('txtDesAssunto').value          = 'Indenização por dano moral';
  document.getElementById('hdnTipoAssunto').value         = 'P';
  document.getElementById('txtGlossario').value           = '';
  document.getElementById('hdnInfoCodAssunto').value      = '022003';
  transportar();  // função nativa EPROC SP
});

// Origem do ID 826 — como o EPROC descobre:
// URL AJAX: controlador_ajax.php?acao_ajax=consultar_info_assunto_judicial&idClasse=0000000029&idLocalidade=68
// Payload: codAssunto=022003
// Resposta: { idAssunto:"826", tipoAssunto:"P", podeSerPrincipal:"S", sinLancavel:"N" }
👤 5. Etapa 3 — Parte Autora (3/5)
URL: processo_cadastrar_3_autor · Título: Peticionamento Eletrônico (3 de 5) - Partes (autores)
✅ Cenário A — Cliente JÁ existe no EPROC
  • Adicionar via incluirParte() com CPF limpo
  • EPROC encontra → traz dados na tabela #tblPartes
  • Selecionar Justiça Gratuita: value='6' = Requerida
  • Avançar via document.getElementById('btnProxima')?.click()
  • NÃO preencher endereço novamente — já existe no cadastro
⚠️ Cenário B — Cliente NOVO
  • EPROC abre formulário de cadastro de pessoa
  • Sexo: #selSexo → 'M' ou 'F' (ver foto no doc 3)
  • CEP: #txtCep + Tab → aguardar auto-complete (2.5s)
  • SP: bairro não auto-preenche → buscar em viacep.com.br/ws/{CEP}/json/
  • Cidade: é a cidade do AUTOR, não da comarca
  • Clicar #btnIncEnd → Salvar → reconsutar CPF → Cenário A
💻 Código: incluirParte()
// Método confirmado: v17/v18 — único confiável para autor
const addResult = await page.evaluate(({ tipo, nome, doc }) => {
  try {
    incluirParte(
      tipo,        // 'PF', 'PJ', 'ENT'
      '0',         // ID da pessoa (0 para nova busca)
      '0',         // índice na tabela
      false,       // validar duplicata
      'N',         // entidade assistencial
      false,       // requisita entidade
      nome,        // nome completo
      doc          // CPF limpo (11 dígitos)
    );
    return { ok: true };
  } catch (e) { return { ok: false, error: e.message }; }
}, { tipo: 'PF', nome: autor.nome, doc: cpfLimpo });

// Verificar se formulário de cadastro abriu (cliente novo)
const needsCadastro = await page.evaluate(() => {
  const sexoSelect = document.querySelector('#selSexo, select[name*="sexo"]');
  const cepInput   = document.querySelector('#txtCep, input[name*="cep"]');
  return !!(sexoSelect || cepInput);
});

🏢 6. Etapa 4 — Parte Ré (4/5)
URL: processo_cadastrar_3_reu · Título: Peticionamento Eletrônico (4 de 5) - Partes (Réus)
PJ Ré com CNPJ (padrão)

💻 Código: objTabParte.adicionar() — método confiável para réu
Atenção: incluirParte() NÃO funciona de forma confiável para réu — selQualificacaoReu bloqueia. Usar objTabParte.adicionar() diretamente.
const addResult = await page.evaluate(({ tipo, nome, doc }) => {
  const keys = window.arrAtributosTabelaPartesPeticialInicialKeys;
  // Colunas (ordem): id_pessoa, tipo_pessoa, nome_pessoa, cpf_cnpj, polo_parte,
  // cod_tipo_parte, des_tipo_parte, sin_principal, coluna_representacao,
  // id_representacao, tipo_representacao, des_representacao, inf_extra,
  // sin_gera_combo_tipo_parte, sin_parte_processo_originario,
  // coluna_justica_gratuita, id_justica_gratuita (3=Não Req.), sin_isento_custas, ...

  const attrs = {
    id_pessoa: '0',
    tipo_pessoa: tipo,             // 'PJ' ou 'ENT'
    nome_pessoa: nome,
    cpf_cnpj: doc,                 // CNPJ limpo ou '00000000000' para entidade
    polo_parte: document.getElementById('hdnPoloParte').value, // 'R'
    cod_tipo_parte: '52',          // RÉU
    des_tipo_parte: 'RÉU',
    sin_principal: 'Sim',
    id_justica_gratuita: '3',      // Não Requerida para réu
    sin_isento_custas: 'N',
    // demais campos: 'null'
  };

  const ordered = keys.map(k => attrs[k] || 'null');
  window.objTabParte.adicionar(ordered);
  document.getElementById('hdnPartesFlag').value = 'S';
  return { ok: true, rowCount: window.objTabParte.obterItens()?.length };

🏛️ 9. Modal de Citação SC (exclusivo TJSC)
Exclusivo SC: Após finalizar no EPROC SC, aparece um modal pedindo o endereço para citação da ré. Não é agência bancária — é o endereço da empresa.
  • Aguardar modal aparecer: #modalCitacao, .modal-citacao, [id*="citacao"]
  • Raspar tabela de endereços → buscar match com SC + Florianópolis/Capital
  • Fallback 1: primeiro endereço com SC
  • Fallback 2: qualquer endereço em SC
  • Clicar no radio correspondente → clicar #btnContinuar
  • Se modal não aparecer em 10s → SC pode ter mudado comportamento
  • SC pode vir pré-marcado no checkbox Juízo 100% Digital — verificar antes de marcar

⚖️ 14. Regras de Negócio

Anti-litigância

  • Máximo 1 distribuição do mesmo cliente contra a mesma empresa por dia
  • Ex: cliente com 2x Boa Vista + 3x outros → distribuir 4 (1 Boa Vista + 3 outros)
  • Verificar via jarbas_tarefas com status=concluido do dia

Regras bloqueantes

  • NUNCA distribuir contra CEF (Caixa Econômica Federal)
  • NUNCA trocar de processo no meio de uma distribuição
  • Manter drawer do MdFlow aberto durante toda a operação

Prioridade de fila

  • 1º: processos com status Agendado
  • 2º: processos Aguardando Distribuição
  • JARBAS sabe qual cliente está distribuindo (baixou os docs)
  • Nunca perguntar qual processo é — ler da petição na pasta ativa
