Pendências — O que falta para o sistema estar "pronto"
🔴 P0 — Segurança (DEVE corrigir antes de qualquer uso em produção)
N8N-DIST-DISTRIBUIR e N8N-DIST-PETICAO: webhook sem autenticação (qualquer pessoa pode acionar), SUPABASE_SERVICE_ROLE_KEY e ANTHROPIC_API_KEY hardcoded no código do workflow (se alguém ver o n8n, vê a chave).
🔴 P0 — Error handlers retornam HTTP 500
Quando dá erro nos workflows de distribuição, o n8n retorna HTTP 500. O frontend espera sempre HTTP 200 com { success: false }. Isso pode causar falhas silenciosas na UI.
🟡 P1 — Bug no N8N-DIST-CRON
O workflow que reverte processos presos em "em_distribuicao" referencia a coluna status na tabela processos_agendamentos, mas essa coluna não existe. O campo correto é concluido (boolean). Isso faz o cron falhar silenciosamente.
🟡 P1 — N8N-DIST-PRIORIZAR desativado
O workflow de priorização de lote está inativo. Não é bloqueante para distribuição básica, mas impede priorização manual de lotes.
🟡 P1 — 3 hooks faltantes (spec vs implementado)
useDistribuicaoConfig, useAgendamentos, useChatOlivia não existem como hooks. Lógica está inline nos componentes. Funciona, mas viola a spec e dificulta manutenção.
🟡 P2 — TabAguardando.tsx próximo do limite
479 linhas (limite da spec: 500). Uma refatoração pode ser necessária em breve.
🔵 P2 — Pipeline de comprovante não implementado
A RPC rpc_solicitar_comprovante_restricao marca status='gerando' mas o pipeline que gera o PDF via bureau não existe. Aguardando definição do fluxo.
