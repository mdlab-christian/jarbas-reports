EPROC — Download de Documentos

Gravação: João · 13/03/2026 · 11min · TJRS + TJSC + TJSP
⚡ Diagnóstico — Por Que a Skill Não Estava Baixando
Problema identificado: A skill atual tenta baixar como se todos os documentos fossem arquivos acessíveis por URL direta. Mas o EPROC tem dois fluxos completamente diferentes de download — e o mais comum (DESPADEC1, SENT1, ATORD1) exige simular o fluxo de impressão do navegador para gerar o PDF. A skill não implementa esse segundo fluxo.
Segundo problema: Documentos em lote (petição inicial + anexos) são baixados como .zip, não como PDF direto. A skill provavelmente espera um PDF e ignora o zip.
Solução: Implementar 3 estratégias de download separadas, detectadas pelo tipo de link do documento.

Os 3 Fluxos de Download (Crítico para a Skill)

📦 Tipo 1 — Download em Lote (ZIP) Mais simples

Quando: Eventos com vários anexos (petição inicial, contestação com documentos)

Como: Clica no ícone de download (seta ↓) ao lado do evento → nova aba abre e fecha automaticamente → arquivo .zip baixa direto

Nome do arquivo: 50222480920258210010_Event1.zip → descompacta em PDFs individuais

Lógica de detecção do tipo de documento

function detectarTipoDownload(linkTexto) {
  // Tipo 1: ícone ZIP — múltiplos arquivos
  if (linkTexto.includes('download') || linkTexto === 'zip') return 'LOTE_ZIP';
  
  // Tipo 3: documentos do juízo — precisam de "impressão"
  const tipoJuizo = ['DESPADEC', 'SENT', 'ATORD', 'DESP', 'DEC', 'ACORD'];
  if (tipoJuizo.some(t => linkTexto.toUpperCase().includes(t))) return 'VISOR_IMPRESSAO';
  
  // Tipo 2: documentos das partes — PDF direto no navegador
  return 'PDF_NAVEGADOR';
}
