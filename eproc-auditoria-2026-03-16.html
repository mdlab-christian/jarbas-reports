🏗️ Arquitetura Atual do Sistema

🔄 Fluxo de Login (State Machine v7.1)

  • Preflight HTTP
    isEprocAvailable() — abort imediato se offline
  • Chromium Persistent Profile
    chrome-profiles/[oab]/ com cookies salvos
  • detectState() loop (12 iter.)
    Estados: EPROC_LOGIN → SSO_LOGIN → 2FA → PROFILE → PANEL
  • TOTP automático
    generateTOTP() + retry automático se código expirou
  • Seleção de Perfil (ADVOGADO)
    Compatível TJRS (texto) e TJSC (OAB no texto)
  • Vision Fallback no UNKNOWN
    Estado desconhecido → adaptiveAction() com Claude Vision
  • maxRetries = 2 + 15s wait
    2 tentativas de login por invocação

🛡️ Resiliência Implementada

  • Circuit Breaker (task-consumer)
    3 falhas → pausa 15min + notifica Telegram
  • Reagendamento automático
    Erros transitórios → +2h → +4h → +8h (máx 3x)
  • Orphan recovery
    Tarefas em_andamento > 10-90min → reset automático
  • Circuit breaker de conta
    ≥5 falhas consecutivas → conta desativada + alerta
  • Anti-bot: jitter inter-task
    importar: 5s, captura: 5s, download: 2s entre tarefas
  • reloginFn dentro da skill
    Sessão expirada no meio → relogin sem re-enfileirar
  • Sem pool de sessões
    Cada tarefa abre/fecha browser (oportunidade de melhoria)

🐛 Bug Crítico Encontrado
🔴
[BUG-01] skill_eproc_cnj.mjs — URL do TJSP incorreta (P0)

O arquivo skill_eproc_cnj.mjs tem um mapeamento hardcoded errado para TJSP:
'8.04': { nome: 'TJSP', url: 'https://esaj.tjsp.jus.br/' }
Porém a conta real no Supabase aponta para https://eproc1g.tjsp.jus.br (EPROC, não ESAJ).
O ESAJ é o sistema do TJSP para consulta pública, não o EPROC para advogados. Todo acesso por CNJ ao TJSP vai para o sistema errado.

// ❌ ATUAL (errado) const TRIBUNAL_MAP = { '8.21': { nome: 'TJRS', url: 'https://www.eproc2g.tjrs.jus.br/eproc/' }, '8.12': { nome: 'TJSC', url: 'https://eproc1.tjsc.jus.br/eproc/' }, '8.04': { nome: 'TJSP', url: 'https://esaj.tjsp.jus.br/' }, // ← ERRADO }; // ✅ CORREÇÃO '8.04': { nome: 'TJSP', url: 'https://eproc1g.tjsp.jus.br' }, // ← CORRETO

⚠️ Além disso, as URLs do TJRS no mapeamento também divergem: hardcoded usa eproc2g mas a conta usa eproc1g. A skill eproc_cnj deve usar as URLs do Supabase, não o mapeamento hardcoded.

🎯 Recomendações de Melhoria

P0 — BUG Corrigir TRIBUNAL_MAP em skill_eproc_cnj.mjs

Substituir URLs hardcoded por lookup nas contas do Supabase (já carregadas via loadAccounts). O mapeamento fixo cria dessincronia toda vez que um tribunal atualiza sua URL. Solução: detectar tribunal via CNJ → buscar a conta ativa correspondente → usar account.eproc_url.

P1 Tratar "Execution context destroyed" com guard de estado

Antes de qualquer page.evaluate(), verificar se a página ainda está no estado esperado com isSessionExpired() ou checar URL. O erro "context destroyed" é silencioso e mata a tarefa sem entrar no circuit breaker (tentativas=0). Adicionar try/catch específico com reloginFn trigger.

P1 Reagendamento offline para captura_intimacoes sem CNJ

Quando EPROC está offline, captura_intimacoes falha mas não é reagendada (o reagendamento atual requer numero_cnj que a captura não tem). Resultado: a captura daquele ciclo é simplesmente perdida. Solução: adicionar lógica de reagendamento sem CNJ, usando oab como chave + delay de 2-4h.

P1 Fallback Vision → Playwright com auto-promoção de seletores

O sistema já tem a lógica (threshold: 10 acertos + 90% confiança → promover a seletor Playwright). Porém a tabela clayton_visual_memory não tem a coluna seletor que a skill espera — ela foi renomeada para decisao.seletor_css dentro do JSON. Validar se a promoção está funcionando corretamente.

P2 Implementar pool de sessões (browser reuse)

Hoje cada tarefa abre e fecha o Chromium. Com 5 contas e tarefas frequentes, isso gera latência desnecessária (~3-5s por login quando sessão ainda válida). Um pool simples com 1 browser por OAB, com keepalive e re-login lazy quando expirar, reduziria tempo médio de execução em ~40%.

P2 Expandir memória visual para TJSP

TJSP tem apenas 9 acertos na memória visual (vs RS: 5.702). Isso significa que o TJSP ainda depende mais de Vision pago do que de cache. Adicionar contextos pré-mapeados para TJSP: login_tela, seleção_perfil, busca_processo, relatorio_processos.

P3 Health check proativo a cada hora com alerta de degradação

Hoje o sistema detecta offline no momento da tentativa. Adicionar um cron leve (a cada 1h) que faz preflight em todos os tribunais e notifica Christian se qualquer um ficar offline — antes de a captura agendar e falhar. Já tem o isEprocAvailable(), basta expor no monitor_n8n ou heartbeat.
