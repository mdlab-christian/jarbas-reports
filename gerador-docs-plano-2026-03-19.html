📊 Resumo Executivo

Como funcionava (Make)

🔵 1 webhook recebe: tipo + lead_id + OAB
🔵 Filtra por tipo (contrato_midas / contrato_orgao / contrato_empresa)
🔵 Filtra por OAB do advogado
🔵 Copia Google Doc template → novo arquivo
🔵 Faz replace dos placeholders
🔵 Salva PDF no Drive do cliente
🔵 Renomeia pasta do cliente

O problema (era improviso)

❌ IDs Google Docs hardcoded no Make
❌ 210 módulos repetidos por OAB
❌ Cada advogado = bloco duplicado
❌ Sem banco de templates centralizado
❌ Difícil manutenção
❌ Make tem limite de operações/mês
❌ Sem rastreio de quais docs foram gerados

Como vai ficar (n8n)

✅ 1 workflow limpo no n8n
✅ Supabase como fonte de verdade dos templates
✅ documento_templates com google_doc_id_original
✅ Lógica: busca template → copia doc → substitui
✅ Rastreio em processo_documentos
✅ Sem operações Pay-per-use (n8n self-hosted)
🗺️ Fluxo do Make (lógica recriada)
1
Webhook recebe payload
tipo + dados_origem.id (lead_id) + dados_origem.advogado.oab
↓
2
Busca lead no Supabase
Tabela leads filtrando por id — pega nome, CPF, endereço, advogado
↓
3
Router por tipo de documento
contrato_midas → gera 1.1 + 1.2
contrato_orgao → gera 1.6, 1.7, 1.8, 2.2
contrato_empresa → gera 1.3, 1.4, 1.5, 2.1
↓
4
Router por OAB do advogado
RS/116571 · RS/131163 · SC/74025 · SC/75466 · SP/535101
Cada OAB tem seus Google Docs específicos
↓
5
Copia Google Doc template → nova instância
API Google Docs: createFromTemplate com substituição de variáveis
↓
6
Exporta PDF e salva no Drive do cliente
Pasta do cliente nomeada como {codigo_advogado} - {nome} [{CPF}]
↓
7
Deleta rascunho do Docs (mantém só PDF)
Google Drive: deleteAFile após upload do PDF
Tabelas Supabase usadas no Make:
📦 leads — dados do cliente (nome, CPF, OAB, endereço)
📦 restricoes_orgao — quando tipo = contrato_orgao
📦 outra tabela (empresa_re) — quando tipo = contrato_empresa
Tabelas que o n8n vai usar:
📦 leads ou organizacoes — dados do cliente
📦 documento_templates — Google Doc IDs (a popular)
📦 processo_documentos — salvar links dos docs gerados
📦 restricoes_orgao / contratos_empresa — dados extras
⚠️ Pontos de atenção:
• RS/131163 e SC/75466 — quem são esses advogados? Verificar.
• 1.7, 1.8 e 1.4, 1.5 têm o mesmo Google Doc ID para múltiplas OABs (é intencional — template genérico)
• Drive do cliente: pasta precisa existir previamente ou ser criada
• Make usava conta Google específica — n8n precisa da mesma
