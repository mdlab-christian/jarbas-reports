🎯 Matriz de Riscos

⛔ EPROC bloqueia por bot detection

O TJRS e TJSC já têm histórico de rate limiting. Com 6.751 processos para navegar, risco de bloqueio temporário de IP/conta.

Mitigação: 3–5s delay, pausa de 2min a cada 100, re-login automático, detector de captcha com fallback de 10min.

⛔ n8n Railway cai durante sync

Se o n8n cair durante a operação, os batches enviados para o webhook vão falhar silenciosamente. Com 6.751 processos, alta probabilidade de algum downtime.

Mitigação: Verificar status do webhook antes de cada batch. Usar checkpoint para retomada. Monitorar n8n durante a operação.

⚠️ Credenciais TOTP dessincronizan

TOTP é baseado em tempo. Se o relógio do Mac Mini drift mais de 30s, os códigos ficam inválidos e o login falha.

Mitigação: Verificar sync NTP antes de iniciar. A skill já tenta código ±1 intervalo.

⚠️ Timeout do Playwright em processos grandes

Processos com muitas movimentações (100+) podem timeout na extração. A skill tem TIMEOUT_MS=180s.

Mitigação: Aumentar timeout para 300s para a operação em massa. Implementar paginação de movimentações se necessário.

✅ Duplicação de movimentações

Re-importar processo que já tem movimentações pode criar duplicatas.

Mitigação: A skill usa TURBO_SKIP_KNOWN=true por default. Para a sync de movs, usar ON CONFLICT DO NOTHING no Supabase.

✅ Custo API (IA analysis)

4.698 análises IA podem custar R$200–500 se usar Sonnet. Com Haiku, custo seria R$20–50.

Mitigação: Usar Haiku para batch. Sonnet apenas para processos P0 (depósito, alvará, sentença favorável).
