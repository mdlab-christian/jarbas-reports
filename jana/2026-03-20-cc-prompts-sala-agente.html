OlivIA · Skills Necessárias

7 Skills que a OlivIA Precisa Criar

A OlivIA tem conhecimento mas não tem ferramentas. Abaixo as 7 skills prioritárias para que ela funcione como agente autônomo de petições — em ordem de prioridade.

P0 · Criar primeiro
Skill 1: gerar-peticao

Skill principal. Executa o fluxo completo de geração: identifica réu/tipo/estado, consulta playbook no banco, busca RAG, complementa com tags, monta peça com template HTML do Supabase.

Triggers
"gerar petição", "replica de [réu]", "apelação do processo X"
Tabelas usadas
olivia_playbooks, olivia_modelos_peticao, olivia_tipos_peticao, fn_rag_search
Output
Peça formatada + blocos usados + jurisprudências + flag revisão
Regra crítica
Consultar banco ANTES de gerar. Nunca inventar argumento.
P0 · Criar primeiro
Skill 2: analisar-prova

Recebe descrição ou texto de um documento. Classifica em olivia_tipos_prova (forte/moderado/fraco). Busca bloco de impugnação via RAG. Retorna análise com jurisprudência do estado.

Triggers
"analisa essa prova", "como impugnar [documento]", "contestação de [réu]"
Tabelas usadas
olivia_tipos_prova, fn_rag_search, olivia_jurisprudencias
Output
Tipo/nível da prova + argumento de impugnação + jurisprudência
Filtro obrigatório
Estado (RS/SC/SP) → tribunal correto na jurisprudência
P1 · Criar depois do P0
Skill 3: triagem-processo

Ponto de entrada universal — substitui o Playbook Raiz em draft. Lê o contexto do processo (texto livre, PDF ou estruturado), identifica: réu, tipo de ação, estado, fase processual, prazo. Recomenda a ação correta com playbook correspondente.

Triggers
"analisa esse processo", "o que fazer com isso", [envia PDF/texto]
Tabelas usadas
olivia_playbooks (dinâmico — consultar sempre), fn_rag_search
Output
Diagnóstico + playbook recomendado + próxima ação sugerida
Regra crítica
Ler histórico COMPLETO — não só último evento
P1
Skill 4: consultar-jurisprudencia

Busca semântica nas 1.353 jurisprudências do banco. Filtra por réu, estado (TJRS/TJSC/TJSP) e situação. Retorna ementa + link + tribunal. Awareness especial: Boa Vista tem tema repetitivo STJ; Serasa tem IRDR de dívidas prescritas.

Triggers
"jurisprudência de [réu] em [estado]", "decisões sobre negativação de [banco]"
Funções
fn_rag_search, match_rag_chunks_by_text, olivia_jurisprudencias direto
P1
Skill 5: rag-enriquecimento

Indexa em massa blocos de texto e jurisprudências no RAG. Hoje apenas ~8% está indexado. Essa skill automatiza o processo de chamar fn_rag_ingest_document em batch para todos os registros pendentes.

Triggers
"enriquece o RAG", "indexa jurisprudências", "rag: enriquecer"
Função
fn_rag_ingest_document — batch de olivia_blocos_texto + olivia_jurisprudencias
Impacto
RAG de 8% → 100% — qualidade das buscas melhora drasticamente
Segurança
Processar em lotes de 20 — respeitar rate limit da API OpenAI
P2
Skill 6: memoria-processual

Popula e consulta olivia_agente_contexto — memória por processo. Guarda: última análise, última peça gerada, observações do advogado, histórico de interações. Permite que a OlivIA "lembre" o que foi discutido sobre um processo específico.

Triggers
"lembra o processo X", "contexto do proc. X", "o que foi feito nesse processo"
Tabela
olivia_agente_contexto — atualmente vazia, precisa ser populada
P2
Skill 7: ajg-por-estado

Retorna os documentos necessários para AJG (Gratuidade de Justiça) por estado. RS: IR + contracheques + declaração. SC: extratos + DETRAN + imóveis. SP: Registrato + declarações + extratos. Também orienta sobre peculiaridades de cada tribunal.

Triggers
"AJG RS", "documentos gratuidade SC", "o que pedir para AJG em SP"
Dados
Hardcoded no SKILL.md — dados estáveis por estado
