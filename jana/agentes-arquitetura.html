⚠️ Problema Atual

JANA e JARBAS estão sobrecarregados porque múltiplos usuários e tarefas competem pela mesma instância OpenClaw. O gargalo não é hardware — é isolamento de processo.

🗺️ Plano de Implementação
1

Criar usuários macOS

No MacBook Air: sudo sysadminctl -addUser olivia. No Mac Mini: sudo sysadminctl -addUser clayton e sudo sysadminctl -addUser midas-agent. Cada um com home própria (/Users/olivia, etc.).

2

Criar bots Telegram no BotFather

Um bot por agente: @Olivia_mdbot, @Clayton_mdbot, @MidasAgent_mdbot. Cada um com token próprio.

3

Instalar OpenClaw por usuário

Login como cada usuário macOS → openclaw setup com conta OpenClaw própria + bot token do agente. Workspace isolado em ~/.openclaw/workspace-[nome]/.

4

Criar SOUL.md por agente

Identidade, permissões e escopo de cada agente. Clayton só lê tabelas jurídicas. Olivia só acessa WhatsApp/leads. Credenciais mínimas por função.

5

Configurar gestão JARBAS + JANA

JARBAS gestora dos agentes do Mac Mini via ssh mac-mini -l clayton. JANA gestora dos agentes do MacBook. Handoffs via tabela jarbas_handoffs do Supabase.

6

Prioridade sugerida de implementação

1º Olivia (maior impacto — atendimento em tempo real já congestionado) → 2º Clayton (processos jurídicos 24/7) → 3º Midas-agent (financeiro isolado) → depois avaliar dev-agent.
