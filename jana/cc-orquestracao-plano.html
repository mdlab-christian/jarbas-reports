🔍 Diagnóstico Atual

Hoje tudo passa pela JANA ou pelo JARBAS — inclusive tarefas de codificação longa que poderiam rodar em paralelo no terminal via Claude Code. Isso gera dois problemas:

  • 📌 Contenção de turno: JANA/JARBAS processam 1 mensagem por vez. Lapidação longa = grupo bloqueado.
  • 📌 Contexto inflado: sessões pesadas com subagentes aumentam o uso de tokens e a latência de todas as respostas.
  • 📌 Dependência total: se JANA/JARBAS travam, o desenvolvimento para.
Raiz do problema: Claude Code pode rodar em múltiplos terminais em paralelo, sem consumir contexto da JANA ou do JARBAS. Hoje esse recurso não é usado sistematicamente.

🛠️ Skill Proposta — cc-prompt-gen

O que a skill faz:

1

Detecção automática

Quando a tarefa é longa/complexa, JANA avisa: "Isso é melhor no CC — gero o prompt?" ou gera direto sem perguntar.

2

Geração do prompt estruturado

JANA monta um prompt completo com: contexto do projeto, arquivos relevantes, objetivo claro, critérios de sucesso e restrições.

3

Você roda no terminal

Copia o prompt, roda claude --print --permission-mode bypassPermissions "prompt" no terminal em paralelo com outras conversas.

4

CC responde → você cola o resultado

Manda o output do CC de volta pra JANA para revisão, validação ou próxima etapa.

Trigger sugerido

Você manda qualquer uma dessas variações:

gera prompt cc: lapida /controller/drawer com foco em performance
cc prompt: refatora o hook useProcessos
gera missão: [descrição da tarefa]

JANA retorna um bloco ```bash pronto para executar no terminal.

📊 100% CC vs Híbrido (JANA/JARBAS + CC)

✅ 100% Claude Code Manual

  • Zero carga nos agentes
  • Totalmente paralelo
  • Sem risco de travar grupos
  • Controle total do terminal

❌ Limitações

  • Você monta o contexto manualmente
  • Sem histórico automático de decisões
  • Sem integração Supabase/n8n direto
  • Sem checkpoint de missão

✅ Híbrido (Recomendado)

  • JANA/JARBAS gerenciam contexto
  • CC executa tarefas pesadas
  • Histórico + checkpoint automático
  • Supabase/n8n via JANA

❌ Limitações

  • Requer copiar/colar prompts
  • Um passo extra no fluxo
  • JANA ainda precisa revisar output

🎯 Recomendação

Modelo híbrido com skill de geração de prompt.
JANA/JARBAS ficam como orquestradores e gerentes de contexto. Claude Code fica como executor paralelo de tarefas pesadas. A skill detecta automaticamente quando delegar pro CC e gera o prompt completo — você só roda no terminal. Resultado: mais conversas simultâneas, sem travar, sem perder contexto.
