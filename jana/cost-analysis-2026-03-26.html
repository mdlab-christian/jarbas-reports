🔍 Causas Raiz Identificadas

1

🧠 Contexto Gigante — Cache de 120-174k Tokens por Call

Cada chamada ao Claude carrega em média 120.000 tokens via prompt cache. Isso inclui SOUL.md, MEMORY.md, AGENTS.md, TOOLS.md, HEARTBEAT.md + histórico do grupo + skills injetadas. O system prompt tem ~174k tokens total. A cada rebuild de cache (TTL 30min), o custo é de $0.65 por chamada — só para montar o contexto.

⚡ Impacto: ~60% do custo total | Cache write = $3.75/M tokens (12.5x mais caro que cache read)
174k tokens × $3.75/M = $0.65/write
TTL 30min → 48 rewrites/dia = $31/dia só em writes
2

📱 Muitos Grupos Telegram com requireMention=false

São 5+ grupos ativos + wildcard (*) onde TODA mensagem aciona o Claude. Os grupos Olivia (G1-G6) sozinhos geraram 3.000+ calls em 3 dias. O grupo "S3 Modo Express" custou $271 em 3 dias. O wildcard (*) significa que qualquer grupo Telegram não listado também aciona a JANA.

⚡ Impacto: ~35% do custo | Grupos Olivia + S3/S4 = causa principal dos picos
3

💗 Heartbeat Rodando Constantemente com Contexto Grande

A sessão de heartbeat (27e25c3f) fez 890 calls em 3 dias (~297/dia) e custou $110. O heartbeat inclui verificações de SSH, subagentes, crons — e carrega todo o contexto de 174k tokens. Mesmo quando retorna HEARTBEAT_OK, o contexto é processado. Calls caras: $0.65/vez quando o cache expira.

⚡ Impacto: ~16% do custo | Cadência muito alta para a complexidade do que faz
4

🔄 Cache TTL Curto (30min) → Muitos Rewrites Caros

O config atual tem cache-ttl: 30m. Com contexto de 174k tokens e TTL de 30min, o cache é recriado muitas vezes por dia. Cada rewrite custa $0.65. A sessão S3 teve 294 cache writes = $137 só em writes. Aumentar o TTL ou usar mode: stable reduziria drasticamente esse custo.

⚡ Impacto: multiplica o custo das causas 1 e 2
5

🤖 claude-sonnet-4-6 para TODO tipo de tarefa

100% das chamadas usam claude-sonnet-4-6 — inclusive heartbeats simples (HEARTBEAT_OK), respostas curtas em grupo, verificações de status. Heartbeats que retornam 2 palavras custam a mesma base de contexto que tarefas complexas. Sonnet 4.6 é caro para uso "always-on".

⚡ Impacto: potencial de redução de 40-60% com modelo mais barato para tarefas simples

✅ Recomendações de Otimização

🔇

1. Ativar requireMention=true nos grupos menos críticos

Grupos Olivia (G1-G6) e "Apresentação Viver de IA" — configurar para só acionar quando mencionada. Grupos de dev intenso podem manter mention=false, mas com mais seletividade.

💰 Potencial: reduzir calls em 60-70% nos grupos Olivia → ~$150-200/mês economizados
openclaw config set channels.telegram.groups."-5264633357".requireMention true
📉

2. Reduzir tamanho do contexto injetado — Slim Mode

SOUL.md tem 9kb, AGENTS.md tem 4kb, TOOLS.md tem 2.3kb — mas a maior parte do custo vem de histórico de conversa carregado no cache. Revisar o que é realmente necessário no system prompt. Criar versões "slim" dos arquivos de contexto.

💰 Potencial: se reduzir de 174k para 80k tokens → custo de cache write cai de $0.65 para $0.30
⏰

3. Aumentar o TTL do cache ou usar modo stable

Config atual: cache-ttl: 30m. Aumentar para 2h ou 4h reduziria drasticamente o número de cache writes. Cada write economizado = $0.65 poupado.

💰 Potencial: Se reduzir de 48 para 12 writes/dia → $23/dia de economia
contextPruning.ttl: "2h"
🤏

4. Heartbeat com modelo mais barato (Haiku) ou menos frequente

O heartbeat retorna HEARTBEAT_OK 90% das vezes mas usa claude-sonnet-4-6 com 174k de contexto. Usar claude-haiku para heartbeats simples seria 20x mais barato. Ou reduzir a cadência de heartbeat.

💰 Potencial: heartbeat com Haiku custaria ~$0.003/call vs $0.05-0.65 atual → ~$95/mês economizados
🗂️

5. Revisar grupos ativos — remover grupos inativos ou de teste

O wildcard (*) aciona a JANA em qualquer grupo. Grupos como "Apresentação Viver de IA" ($8 em 3 dias) provavelmente não precisam de resposta automática. Revisar e desativar grupos que não precisam de agente ativo.

💰 Potencial: eliminar 3-4 grupos → $30-50/semana economizados
🧹

6. Sessões de dev intenso (Olivia) → usar compaction + limitar histórico

Sessões longas como S3/S4 acumulam histórico até 215.753 tokens de contexto. Compaction automática mais agressiva manteria o contexto menor. Configurar keepLastAssistants: 3 em vez de 5.

💰 Potencial: reduzir contexto médio de 120k para 60k → cortar custo ao meio nas sessões longas
