E5-GerarPeticaoBase — Gera petição prévia dentro da análise

Após E4-ParseResultado, inserir um novo nó que usa o resultado da análise + o modelo de petição para gerar uma versão base da petição já com os blocos das tags resolvidas substituídos.

Lógica:

  • Busca o conteudo_template do modelo ativo (como o Express faz em B2)
  • Para cada tag com confiança ≥ 0.7: substitui a tag pelo conteúdo do bloco escolhido
  • Tags com confiança < 0.7: mantém como placeholder {{TAG_NOME}} para o usuário preencher
  • Pode criar novos blocos de texto para lacunas não cobertas pelo modelo e posicioná-los
  • Salva em olivia_analises.resultado.peticao_base (campo novo no JSONB)

E2-PrepararInput-Claude — Adicionar instrução de geração de petição base

O system prompt da OlivIA precisa receber instrução adicional para além da análise probatória, também retornar a estrutura de petição no JSON de saída.

Adicionar ao schema de saída:

"peticao_base": {
  "blocos_resolvidos": [
    {
      "tag_id": "uuid",

      "bloco_id": "uuid",      // null se tag nova criada pelo agente
      "titulo_secao": "string",
      "conteudo_html": "HTML do bloco de texto substituído",
      "tag_nova": false,       // true = criado pelo agente (não existe no banco)
      "justificativa": "por que este conteúdo aqui"
    }
  ],
  "tags_novas_criadas": [
    {
      "tag_nome": "nome da tag nova",
      "posicao_entre": ["tag_id_anterior", "tag_id_posterior"],
      "conteudo_html": "conteúdo gerado pelo agente"
    }
  ],
  "instrucao_gas": "instrução para o Google Apps Script de como montar o documento"
}

Regra de jurisprudência: OlivIA só pode referenciar jurisprudências presentes no <jurisprudencia_rag> do prompt — que vem do nosso banco via RAG. Sem invenção.
