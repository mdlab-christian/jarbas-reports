⚠️ Aviso #2

Triagem não é automática — análise IA não pré-preenche o Step 2

O Express não conecta o resultado da ANALISE-V3 ao Step 2 automaticamente. O campo analise_resultado.analise_tags já existe no resultado da análise — cada tag tem bloco_id, confianca e justificativa — mas o StepTriagem não lê isso para pré-preencher as respostas.

O que precisamos fazer:

  • Quando analise_id existe no wizardState, mapear analise_tags → respostas no StepTriagem
  • Tags com confianca ≥ 0.7: pré-preencher automaticamente (bloco selecionado)
  • Tags com confianca < 0.7: mostrar perguntas_triagem para revisão humana
  • Resultado: usuário só responde perguntas onde a IA não teve confiança
⚠️ Aviso #3

Express não usa estrutura Smart Blocks como base

O B3-MontarPrompt usa o conteudo_template do modelo selecionado. Se o modelo não tiver um template no formato Smart Blocks, a petição gerada terá estrutura diferente.

Duas opções em discussão:

Opção A (mais simples): Injetar o template Smart Blocks como <template_base> no B3-MontarPrompt para todos os modos Express — garante estrutura consistente sem mudar o fluxo.

Opção B (mais poderosa): Gerar a petição Smart Blocks dentro da própria ANALISE-V3. Quando avança Step 2 → Step 3, a petição base já está pronta. Usuário só revisa. Economiza tempo mas muda a arquitetura — a análise passaria a fazer análise + geração.

⚠️ Aviso #4

Formatação Google Docs — HTML não confirmado

O B4-ChamarRedator gera a petição em JSON estruturado (seções + conteúdo). O B5 converte para texto Markdown-like. O B9-AppsScript recebe o texto e cria o Google Doc.

Problema: se o Apps Script recebe texto plano ou Markdown, perde negritos, espaçamentos e a formatação jurídica. Precisamos confirmar se o Apps Script converte HTML → Google Docs nativamente, ou se precisamos mudar B4/B5 para gerar HTML diretamente.

Solução recomendada: B4 gera em HTML (tags <h1>, <b>, <p>) → Apps Script usa Body.appendParagraph() com formatação preservada via setText() + setBold(), ou usa a Docs API com insertText + updateTextStyle.
