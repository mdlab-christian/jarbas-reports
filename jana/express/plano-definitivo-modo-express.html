🔴 Bugs Críticos — Bloqueiam o Fluxo Executar PRIMEIRO
  • 🔴
    BUG-01 — Upload dispara análise ANTES do Avançar (auto-trigger com debounce 1s)
    Diagnóstico real: UploadProvasInline.tsx tem um autoTriggerRef com setTimeout 1s que chama handleAnalisar() automaticamente quando arquivos são adicionados. Isso dispara o webhook ANALISE-V4 antes do usuário clicar Avançar.

    Impacto: Análise inicia sem usuário saber → pode usar modelo errado ou processo errado se o usuário ainda está selecionando.

    Fix: Remover o bloco autoTriggerRef de handleFilesChange(). Upload apenas armazena arquivos em estado local. Análise só inicia via botão explícito "Analisar agora" ou ao clicar Avançar (no nextStep() do useWizardTabs que já existe e funciona corretamente).
    UploadProvasInline.tsx → handleFilesChange → remover bloco autoTriggerRef
    CRÍTICO Frontend 30min
  • 🔴
    BUG-02 — Modelo sem versão publicada não bloqueia avanço
    Diagnóstico real: useWizardTabs.ts → canAdvance verifica apenas processo_id, tipo_peticao_id, modelo_grupo_id, advogado_id — não verifica versao_id. Quando versoes.length === 0, o usuário pode avançar sem versão selecionada, e o n8n falha.

    Fix 1 — canAdvance: Adicionar && !!activeState.wizardState.versao_id ao canAdvance do Step 0.

    Fix 2 — UI: Quando versoes.length === 0, exibir alerta vermelho visível + desabilitar botão Avançar no footer.

    Fix 3 — Dropdown de modelos: Não listar modelos sem active_version_id preenchido.
    useWizardTabs.ts → canAdvance + StepIdentificacao.tsx → versoes.length === 0
    CRÍTICO Frontend 45min
  • 🔴
    BUG-03 — OLIVIA-ANALISE-V4 B2 silencia quando tags/blocos zerados → loading infinito
    Diagnóstico real: Quando B2-HTTP-Tags retorna array vazio (categoria do modelo sem tags), o B2-Process produz total_tags: 0, total_blocos: 0 e prossegue silenciosamente. O frontend fica em polling até timeout (120s) e exibe "Análise demorou mais que o esperado — avançando" sem feedback real.

    Fix n8n: Adicionar IF após B2-Process: se total_tags === 0 AND total_blocos === 0 → setar olivia_analises.status = 'ERRO' com mensagem "Modelo sem tags cadastradas" → frontend detecta ERRO e exibe toast explicativo.

    Fix Supabase: Verificar tabela olivia_tags e olivia_blocos_texto para a categoria "Instituições Financeiras" — pode ser problema de dados, não de código.

    Fix Frontend: Timeout de 90s com botão "Tentar sem análise" visível.
    OLIVIA-ANALISE-V4 → B2-Process → IF node + olivia_tags (verificar dados)
    CRÍTICO n8n + Dados 1h

  • 🟠
    RISK-03 — Progress bar de análise frágil (baseada em string matching)

    Fix: Substituir por meta.progresso numérico retornado diretamente do Supabase (já existe em olivia_analises.meta.progresso — o polling já busca isso). Usar pct direto para a barra.

    ALTO Frontend 30min

    Situação: O polling tem timeout de 120s no useWizardTabs.nextStep(), mas o frontend mostra apenas spinner sem opção de cancelar. Usuário não sabe que pode abortar.

    Fix: Exibir botão "Pular análise" no overlay de loading após 15s. O hook já tem abortAnalise() disponível — só precisa expô-lo no UI.
