🎬 1. Diagnóstico do Vídeo de Teste
🔴
BUG-EXPRESS-01 — Upload dispara análise ANTES do clique em "Avançar"
CRÍTICO Frontend

Ao arrastar/soltar um documento na zona de upload do Step 1, o sistema disparou imediatamente a análise IA sem o usuário ter clicado em "Avançar". O usuário não teve chance de classificar o tipo de documento nem selecionar o modelo correto.

Causa provável: UploadProvasInline.tsx tem um onChange ou onDrop que aciona callWebhook diretamente. O webhook de análise só deve ser disparado no handler de nextStep().

Fix: Remover trigger automático no upload. Upload só salva os arquivos em estado local (wizardState.provas_urls); análise só inicia quando usuário clica "Avançar" e o useWizardTabs chama nextStep().
🔴
BUG-EXPRESS-02 — Modelo sem versão ativa não bloqueia avanço
CRÍTICO Frontend + Validação

O campo "Versão" exibiu "Nenhuma versão publicada para este modelo" mas o botão "Avançar" permaneceu ativo. O usuário avançou e o workflow n8n falhou com toast "Modelo sem versão ativa — contate o administrador". A validação deve ocorrer no Step 1, não no n8n.

Fix Step 1: Em StepIdentificacao.tsx, adicionar validação antes de permitir nextStep():
if (!wizardState.versao_id) { toast.error("Selecione um modelo com versão publicada"); return; }

Fix UX: Exibir alerta vermelho visível no campo Versão quando versoes.length === 0. Desabilitar botão "Avançar" nesses casos.

Fix Dropdown: Não listar modelos em olivia_modelos_grupo que não tenham active_version_id preenchido (ou versão com status publicado).
🔴
BUG-EXPRESS-03 — OLIVIA-ANALISE-V4 trava no B2 com tags/blocos vazios
CRÍTICO n8n

O workflow OLIVIA-ANALISE-V4 chegou ao node B2-Process com total_tags: 0 e total_blocos: 0. O B2-HTTP-Tags retornou array vazio porque a categoria do modelo não tem tags cadastradas. O workflow não falhou com erro explícito — simplesmente produziu output vazio, levando ao loading infinito.

Fix B2-Process: Adicionar guard após B2-HTTP-Tags: se total_tags === 0 AND total_blocos === 0, retornar erro explícito com action: 'toast_error' e mensagem "Modelo sem tags cadastradas — configure em /playbooks".

Fix loading: Adicionar timeout de 90s no frontend. Se olivia_analises.status não for concluido após timeout → exibir erro e botão "Tentar novamente".

ALTO n8n + Config

Christian reportou "não quero que use claude-haiku, só claude-sonnet para cima". Auditar todos os fluxos para garantir que claude-haiku-4-5 não seja usado em nenhum agente de produção.
