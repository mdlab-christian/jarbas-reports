➕ Adicionar Etapa
Triggers: "adiciona etapa", "nova etapa: [desc]", "incluir etapa"

JANA coleta título, camada, componente, página, escopo e complexidade. Gera PRD rascunho automaticamente. Se complexidade = complexa, pergunta se quer quebrar em sub-etapas.

Granularidade mínima: 1 aba | 1 componente | 1 modal | 1 migration | 1 workflow. Nunca: página inteira em 1 etapa.
📋 Gerar PRD Geral
Triggers: "gera prd do projeto", "prd da Olivia", "documenta o projeto"
Importante

JANA gera um prompt CC para investigar e documentar o projeto inteiro (frontend + backend + automações). O PRD fica salvo no banco e é injetado em todos os prompts futuros das etapas — evita que CC perca contexto arquitetural.

Por que fazer isso? Sem PRD geral, o CC só vê 1 etapa de cada vez e pode tomar decisões arquiteturais erradas. Com o PRD, ele sabe onde cada coisa se encaixa no projeto inteiro.

💡 Sistema de Ideias — projetos_ideias

Novo em v3: Tabela unificada que substitui jana_ideias e jarbas_ideias. Uma ideia pode vir de qualquer pessoa — Christian, Gustavo, JANA, JARBAS, ou até alguém externo.

Campo especial: ideia_equipe

✅ ideia_equipe = false

Ideia do time de desenvolvimento. Christian, Gustavo, JANA ou JARBAS. Pode virar etapa de projeto diretamente.

Ideia de alguém externo (ex: João — não é dev). Precisa de avaliação antes de virar etapa. Indica quem deu a ideia.

🔄 Como o Loop Funciona

O loop segue a spec até o fim. Cada etapa executada deve atender ao critério de aceite. O projeto só termina quando 100% das etapas estão testado=true.

1

JANA identifica próxima etapa pendente

Verifica status, dependências resolvidas, tentativas < 2

2

JANA gera prompt CC completo

Inclui: resumo PRD geral, PRD da etapa, critério de aceite, arquivos alvo, regras anti-alucinação, instrução de screenshot

3

CC executa no terminal

Implementa código + tira screenshots (Playwright) + gera relatório obrigatório

4

JANA valida o resultado

vision (screenshot) | e2e (banco + frontend) | n8n-review (workflows) | manual

5

JANA atualiza banco + notifica

testado=true + evidencia_url + screenshots registrados + progresso_pct atualizado

Volta ao passo 1

Repete até 100% das etapas concluídas. Ao final: relatório completo para Christian/Gustavo.

🛑 Loop para quando: Tentativas >= 2 na mesma etapa → JANA alerta Christian imediatamente. Nunca continua no escuro.

📏 Regras — O que você precisa saber

❌ O que nunca acontece

  • CC improvisa além do prompt
  • Banco alterado sem instrução explícita
  • Etapa marcada testada sem evidência real
  • Loop continua após 2 falhas na mesma etapa
  • Etapa executada antes de dependência concluída
  • Modo automático sem autorização explícita

✅ Garantias do sistema

  • Projeto não se perde se a sessão fechar
  • Screenshots de cada componente criado
  • Update após cada etapa concluída
  • Alerta imediato se algo quebrar
  • PRD geral previne decisões erradas
  • Spec é imutável — mudança pede confirmação

Limite de segurança — Modo Automático

Modo automático = máximo 3 etapas por rodada sem check-in. JANA manda update após cada etapa. Você pode pausar a qualquer momento com: "para o loop".
