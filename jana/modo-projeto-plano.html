⚡ Skills a Criar

📋 skill-spec — Geração de SPEC

Absorve contexto (áudios, texto, código) e gera SPEC técnica completa com mapa mental + checklist.

  1. Lê todos arquivos relevantes do repo (frontend + backend)
  2. Mapeia workflows n8n envolvidos (por nome/ID)
  3. Gera mapa mental de tudo que foi dito
  4. Gera checklist de absorção (nada pode ficar de fora)
  5. Gera SPEC estruturada (estado atual / arquitetura / gaps / plano)
  6. Publica HTML no GitHub Pages e retorna link
  7. Cria registro em jana_projetos com spec_url

🚀 skill-projeto-executar — Modo Loop

Executa todas as tarefas de um projeto em loop, uma por vez, com validação.

  1. Lê projeto + tarefas do Supabase (status=pendente, ordenado por prioridade)
  2. Para cada tarefa: gera prompt CC ou executa diretamente
  3. Modo Manual: gera prompt → Christian roda no terminal
  4. Modo Automático: usa sessions_spawn com runtime=acp
  5. Após execução: valida (Vision para frontend, lê workflow para n8n)
  6. Se aprovado: marca testado=true e testado_em=now()
  7. Se falhou: cria nova tarefa de fix com contexto do erro
  8. Loop até todas tarefas com testado=true

💡 skill-ideias — Captura de Ideias

Salva ideias avulsas no banco sem virar projeto ainda.

  1. Trigger: Christian diz "salva essa ideia" ou "tem uma ideia"
  2. Extrai: título, categoria, página, autor, prioridade
  3. Salva em jana_ideias
  4. Confirma com resumo: "💡 Ideia salva: [título] · categoria: [X] · prioridade: [Y]"

🔍 skill-validar-tarefa — Validação Pós-Execução

Valida que a tarefa foi realmente executada com sucesso.

  1. Frontend: abre browser, navega para a página, tira screenshot Vision
  2. n8n: acessa API n8n, lê nodes do workflow, verifica se está ativo e correto
  3. Supabase: executa query de verificação
  4. Retorna: aprovado / reprovado + evidência
  5. Atualiza testado + evidencia_url no banco

🔄 Fluxo Completo — Do Áudio à Execução

Fase 1 — Planejamento (modo /plan ativo)

Christian manda áudios/texto → JANA transcreve + absorve → skill-spec executa → HTML gerado (mapa + checklist + spec) → jana_projetos criado (status=rascunho)

Fase 2 — Aprovação do plano

Christian revisa o HTML → "pode executar" / ajustes → JANA cria tarefas no banco → jana_tarefas (status=pendente)

Fase 3 — Execução Manual (CC)

JANA lê próxima tarefa pendente → Gera prompt CC estruturado → Christian roda no terminal → JANA valida (Vision/n8n) → testado=true → próxima tarefa

Fase 4 — Execução Automática (noturna)

JANA lê tarefas pendentes → sessions_spawn (runtime=acp) → Aguarda resultado → Valida + atualiza banco → loop até projeto concluído

⚙️ Modo Manual vs Automático

🖐 Modo Manual (padrão)

  • Christian opera o CC no terminal
  • JANA gera o prompt pronto para copiar
  • Mais controle — vê o que está sendo feito
  • Sem risco de loop infinito ou drift
  • JANA valida automaticamente após cada passo
  • Ideal para tarefas críticas ou novas

🌙 Modo Automático (noturno)

  • JANA spawna CC via sessions_spawn
  • Loop com validação automática
  • Ideal para tarefas repetitivas / fixes simples
  • Requer tarefas bem definidas no banco
  • Notifica Christian ao concluir ou travar
  • Fallback: cria tarefa de revisão manual se falhar 2x

❓ Perguntas para Alinhar Antes de Executar

  • 🤔 Skill vs Agente: Skills são arquivos SKILL.md que a JANA lê e segue. Agentes (subagents) são sessões paralelas spawned. Para esse fluxo, skills são o certo — controladas, sem overhead de sessão nova.
  • 🤔 RLS nas tabelas: As tabelas jana_* / jarbas_* precisam ser acessíveis pelos bots (service_role) e pelo MdFlow (authenticated)? Ou só pelos bots via service_role?
  • 🤔 Projeto piloto: Confirmar que OlivIA Modo Agente é o primeiro projeto a entrar no novo fluxo?
  • 🤔 Modo noturno: Qual horário? Manter o padrão de idle 30 dias (sem reset automático) ou criar um cron explícito para rodar tarefas?
