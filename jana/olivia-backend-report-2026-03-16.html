🔧 Problemas Encontrados e Resolvidos
✅ HTML gigante quebrava o RAG (chunks > 8.192 tokens)
Google Docs exporta HTML com CSS inline imenso. Alguns blocos chegaram a 872.972 chars (~218k tokens), muito acima do limite de embedding da OpenAI (8.192 tokens).
Fix: scripts olivia_13 e olivia_14 — converter HTML → texto limpo antes de indexar. Blocos de ~870k reduziram para ~3k chars.
✅ fn_rag_complete_document "not in schema cache"
n8n não encontrava a função RPC no PostgREST após falha de embedding. Era efeito colateral do erro de tokens — a função existe no banco.
Fix: resolvido automaticamente após corrigir o tamanho dos chunks.
✅ 28 docs travados em status "processando"
n8n processou os docs mas não atualizou o status (falha silenciosa após erro de embedding). Ficaram presos em "processando" indefinidamente.
Fix: olivia_14 — limpar HTML dos travados + resetar para "pendente" e re-indexar.
✅ RAG sync interrompido (token OAuth expirado mid-run)
Token Google OAuth expirou durante extração de blocos (válido por ~43min). Parte dos docs teve erro 401.
Fix: script reenova token automaticamente via refresh_token. Docs com erro foram re-processados na segunda rodada.
⚠️ Pendências Abertas
🔒
244 blocos + 104 modelos sem permissão no Drive
Os Google Docs referenciados existem mas a conta OAuth não tem permissão de leitura. Precisam ser compartilhados com o email da conta de serviço (ou tornar público dentro do Drive).
Plano de correção: compartilhar as pastas do Drive com a conta OAuth → rodar olivia_11_blocos_content.mjs novamente → RAG sync automático via trigger.
