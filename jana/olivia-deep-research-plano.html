O prompt do olivia_analise tem a legislação correta (CDC 43, Súmula 385, 479, 359), está especializado em negativação indevida e os casos de TJSP/TJRS. 🚨 Gaps Identificados — Por Prioridade
Todos os problemas encontrados no Deep Research de hoje
🚨

[P0-A] perguntas_triagem da IA não chegam no Step 2

O StepTriagem sempre chama mdflow/olivia/triagem → fn_olivia_get_triagem_by_grupo que retorna perguntas genéricas do template. As 9 perguntas contextualizadas que o Claude gerou em olivia_analises.resultado.perguntas_triagem ficam no banco e nunca são exibidas ao advogado. Solução: no StepTriagem, verificar se analiseResultado?.resultado?.perguntas_triagem?.length > 0. Se sim → usar essas perguntas. Se não → fallback para fn_olivia_get_triagem_by_grupo.

P0 — Crítico StepTriagem.tsx lines ~180-240 useWizardTabs + analise_resultado ~2h dev
🚨

[P0-B] Score mapeado errado no TriagemAnaliseCard

O workflow retorna chance_sucesso: 1|2|3 (BAIXA/MÉDIA/ALTA) mas o card espera score: 0-100. A barra de progresso fica em 1-3% ao invés de representar a chance real. Solução: mapear na renderização: 1 → 25, 2 → 60, 3 → 90. Mostrar também o label textual "Chances Baixas/Médias/Altas".

P0 — UX quebrado TriagemAnaliseCard.tsx lines ~15-45 ~30min dev
⚠️

[P1-A] tokens_usados e custo_usd sempre 0

O nó Anthropic do n8n entrega usage dentro de content[].usage (não em output.usage). O nó E4-ParseResultado não itera o array content para extrair input_tokens e output_tokens. Solução no n8n (E4): const usage = content.find(c => c.usage)?.usage || {}; const tokens = (usage.input_tokens||0) + (usage.output_tokens||0);

P1 — Custo E4-ParseResultado (n8n OLIVIA-ANALISE-V3) ~20min
⚠️

[P1-B] visao_geral e documentos_faltantes ausentes no card

O campo visao_geral contém o resumo narrativo completo gerado pelo Claude — é o primeiro item mais importante para o advogado entender o caso. Os documentos_faltantes (8 no E2E) indicam o que precisa pedir ao cliente. Ambos estão no banco mas não exibidos no TriagemAnaliseCard. Solução: adicionar seção "Visão Geral" no topo e seção "Documentos Pendentes" no card.

P1 — UX importante TriagemAnaliseCard.tsx ~1h dev
⚠️

[P1-C] Verificar rota olivia/analise-decisao vs mdflow/olivia/analise

O UploadProvasInline.tsx chama olivia/analise-decisao via callWebhook(). O webhook real do OLIVIA-ANALISE-V3 está em mdflow/olivia/analise. Se callWebhook() adiciona prefixo mdflow/ automaticamente → OK. Caso contrário → o upload de provas está chamando um endpoint errado e nunca dispara o V3. Solução: verificar src/lib/webhook.ts e confirmar roteamento.

P1 — Conectividade UploadProvasInline.tsx + src/lib/webhook.ts ~1h investigação + fix
⚠️

[P1-D] Tipo de documento não diferencia análise no prompt

O Step 1 permite classificar cada PDF como: extrato_bancario, contrato, comprovante_pagamento, notificacao, procuracao, documentos_pessoais, laudo_pericial, outros. Essa classificação é enviada ao webhook mas o E2-PrepararInput-Claude não a usa para adaptar a análise. Um "extrato_bancario" e um "contrato" merecem prompts diferentes — o contrato precisa análise de IP/assinatura, o extrato precisa análise de valores/datas. Solução: no E2, adicionar seção no prompt: [DOC 1: contrato — analisar: IP, data, assinatura digital, cédula de crédito]

P1 — Qualidade IA E2-PrepararInput-Claude (n8n) ~2h dev
💡

[P2-A] fase_processual vazia sem PDFs

O workflow retorna fase_processual: "" quando não há documentos. O processo tem campos grau, fase_id, ultima_decisao_tipo que poderiam ser usados para inferir automaticamente. Solução no E2: mapear grau+fase_id para texto (ex: grau=1 → "Primeiro Grau — fase conhecimento").

P2 — Qualidade E2-PrepararInput-Claude ~1h
💡

[P2-B] Progress bar real durante análise (0%→100%)

O workflow já tem nós de UPDATE de progresso (F-ProgressBar intermediário) mas o campo progresso_pct não existe na tabela. Adicionar coluna → o frontend já tem Realtime → exibir barra real ao invés de spinner estático.

P2 — UX Migration + UploadProvasInline + n8n nós F intermediários ~3h dev
✨

[P3-A] Banner "análise sem documentos" no Step 2

Quando análise não teve PDFs → exibir aviso visual contextual para o advogado entender as limitações.

P3TriagemAnaliseCard.tsx~30min
