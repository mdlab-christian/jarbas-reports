Fase 1 — Peticao Completa (P0)

FIX-01: Resolver truncamento de peticao

Problema: Prompt de 63k chars + max_tokens baixo = peticao truncada em 27k chars.

Opcoes (escolher uma):

OpcaoComoProsContras
A) Aumentar max_tokens Setar max_tokens: 32768 ou 65536 no olivia-gera-peticao.mjs Fix mais simples. Sonnet 4.6 suporta ate 64k output. Custo por peticao aumenta ~4x. Tempo de geracao pode dobrar (120-360s).
B) Chunked generation Dividir SmartBlocks em 3-4 chunks (intro, fundamentacao, conclusao). Gerar cada chunk separadamente. Cada chunk tem prompt menor e output menor. Mais controle. Complexidade significativa. Precisa manter coerencia entre chunks. Transicoes podem ser artificiais.
C) Adapt-only mode Em vez de "reescrever template inteiro", instruir LLM a "retornar APENAS trechos modificados com marcadores de posicao". SmartBlocks template e retornado intacto com patches aplicados. Output minimo (~5k chars). Rapido. Barato. Template preservado 100%. Requer parser de patches. LLM pode errar marcadores. Menos flexibilidade criativa.
D) Hibrido A+C (RECOMENDADO) max_tokens=32768 + instrucao "adapte o template, nao reescreva do zero. Mantenha toda a estrutura e paragrafos existentes. Modifique apenas onde necessario." Balanco custo/qualidade. Template preservado. Output completo. Custo ~2x maior que C mas ~2x menor que A puro.

Recomendacao: Opcao D — aumentar max_tokens para 32768 + mudar instrucao para "adaptar" em vez de "reescrever".

Relevancia para formatacao de peticoes

O padrao Blade Humanizer pode ser aplicado na OlivIA para:

  • Labels de status no frontend — olivia_historico.status (PENDENTE → "Pendente", EM_PROGRESSO → "Em Progresso")
  • Labels de qualidade_ia — rapida → "Rapida (Sonnet)", avancada → "Avancada (Opus)"
  • Labels de secoes no outline editor — introducao → "Introducao", fundamentacao → "Fundamentacao Juridica"
  • NAO e relevante para a skill de formatacao de peticoes (S9) — a formatacao ABNT e sobre HTML/CSS/DOCX styling, nao sobre labels de UI.

Conclusao: O padrao Blade Humanizer ja e seguido nas partes certas do frontend. Nao precisa de mudanca na skill olivia-formata-peticao.mjs.
