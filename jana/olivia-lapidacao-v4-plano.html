2. Checklist de Requisitos vs Realidade

2.1 Pipeline de Geracao

  • Fase 1 roda ao avancar do Step 1: analisa-provas → busca-juris → gera-perguntas
  • Status muda corretamente: PROCESSANDO → AGUARDA_RESPOSTAS → PROCESSANDO nao setado → CONCLUIDO
  • Fase 2 roda ao submeter respostas: smartblocks → gera-peticao → formata → revisor → cria-doc
  • BUG ATIVO: Status PROCESSANDO NAO e setado no inicio da Fase 2 (server.mjs)
  • Progresso vai de 0% a 100% sem travar
  • Polling fallback 15s no Step 4 (implementado)

2.2 SmartBlocks como Base

  • SmartBlocks SEMPRE roda antes do gera-peticao
  • Busca tags via active_version_id do modelo_grupo_id
  • Output >= 10.000 chars — sem validacao explicita (funciona na pratica)
  • Claude refina o template SmartBlocks — nunca reescreve do zero
  • Output gera-peticao >= 80% do input SmartBlocks (48.9K chars)

2.3 Analise de Provas

  • Le texto_extraido dos documentos do processo
  • Usa slice de 2000 chars por doc (nao 300)
  • Resultado injetado como === FATOS EXTRAIDOS DOS DOCUMENTOS ===
  • Dados concretos aparecem na peticao (agencia, contrato, valor)
  • Campo tom normalizado (string/objeto)
  • meta.relatorio salvo com >= 3 secoes

2.4 Jurisprudencias

  • Busca via olivia_modelo_juris (2.440 vinculos)
  • Filtra por status='published' (nao 'ativo')
  • DOCX: fonte 10pt + recuo 3cm + italico
  • Marcadores semanticos >>> JURIS ... <<< JURIS
  • Sem markdown literal no texto final

2.5 Perguntas ao Advogado

  • Maximo 5-6 perguntas (slice(0, 5))
  • Baseadas nas tags do modelo (nao genericas)
  • Usa nome do reu e fatos concretos
  • Tags ja respondidas NAO geram pergunta
  • Dados do processo NAO geram pergunta

2.6 Qualidade do DOCX

  • DOCX gerado (25KB+)
  • PDF gerado (269KB)
  • Titulo correto (# REPLICA)
  • Data preenchida automaticamente
  • Nomes internos (CONTR7/8/9) NUNCA aparecem
  • Partes corretas (autor/reu do processo)

2.7 Formatacao ABNT

  • Times New Roman 12pt
  • Margens: 3cm superior/esquerda, 2cm inferior/direita (1134 twips = 2cm)
  • Paragrafos 1.5 espacamento
  • Recuo 1.25cm
  • Juris: 10pt + 3cm + italico
  • Sem markdown literal no DOCX final

2.8 Regras de Desenvolvimento

  • Skills .mjs NUNCA usam fetch() — grep: 0 ocorrencias
  • ZERO hardcode de cor no frontend — grep: 0 hex
  • Max 400-500 linhas por .tsx — OliviaGeradorExpress = 498L
