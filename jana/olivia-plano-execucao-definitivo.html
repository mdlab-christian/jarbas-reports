Regras Anti-Alucinação e Gestão de Contexto

🚫 O que NUNCA fazer

❌ Prompt gigante para CC

Nunca mandar o PRD inteiro para o CC resolver de uma vez. Máximo: 1 componente ou 1 fix por prompt.

❌ Assumir que CC entende contexto n8n

Sempre incluir explicitamente no prompt: "use require('https'), fetch() não existe". CC não sabe as regras específicas do n8n.

❌ Marcar tarefa concluída sem E2E

Código compilando ≠ funcionando. Sempre testar fluxo completo com dados reais antes de considerar concluído.

❌ Esquecer git push

Cada mudança deve ser commitada e pusheada imediatamente. Lovable desincronizado = confusão. Sem exceções.

✅ O que SEMPRE fazer

✅ Ler relatórios anteriores ao entrar

Cada sala lê sua pasta de relatórios antes de qualquer ação. Contexto dos blocos anteriores já está lá.

✅ Salvar relatório após cada CC

Toda resposta do CC que resulte em mudança de código deve ter relatório salvo na pasta da sala.

✅ Deep review do relatório CC

CC pode ter implementado parcialmente. Sala deve comparar o relatório com o master plan e procurar gaps.

✅ Prompts em fila

Gerar todos os prompts de correção de uma vez, em ordem. Christian executa um por um no terminal.

// ✅ ÚNICO padrão válido para HTTP em Code nodes n8n
const https = require('https');
function httpRequest(options) {
  return new Promise((resolve, reject) => {
    const { method, hostname, path, headers, body } = options;
    const req = https.request({ method: method || 'GET', hostname, path, headers, port: 443 }, (res) => {
      let data = '';
      res.on('data', chunk => data += chunk);
      res.on('end', () => {
        try { resolve({ status: res.statusCode, data: JSON.parse(data) }); }
        catch(e) { resolve({ status: res.statusCode, data }); }
      });
    });
    req.on('error', reject);
    req.setTimeout(25000, () => { req.destroy(); reject(new Error('Timeout')); });
    if (body) req.write(typeof body === 'string' ? body : JSON.stringify(body));
    req.end();
  });
}
