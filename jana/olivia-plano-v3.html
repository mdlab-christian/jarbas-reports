🦊 Protocolo por bloco (obrigatório)

  • 1. Ler projetos_status → contexto atual
  • 2. Consultar RAG → documentação do componente
  • 3. Verificar spec do modo → bloco segue spec?
  • 4. ≤ 3 arquivos por bloco (evitar contexto longo)
  • 5. Executar o bloco
  • 6. Gerar prompt de revisão para CC
  • 7. CC revisa + aplica correções + gera relatório
  • 8. JANA valida relatório CC
  • 9. Salvar em projetos_reports
  • 10. Atualizar projetos_status
  • 11. Próximo bloco

🤖 Template prompt de revisão CC

  • Contexto: projeto, sala, bloco executado
  • O que JANA fez: lista de arquivos criados/modificados
  • Spec de referência: path do doc + excerto relevante
  • Workflows canônicos: IDs da tabela (se bloco usa n8n)
  • Regras invioláveis: fetch() proibido, hex hardcoded proibido, etc.
  • O que NÃO pode mudar: lista explícita
  • O que CC deve fazer: revisar, corrigir se necessário, gerar relatório
❌ Regras absolutas (CC e JANA)
  • fetch() em Code nodes n8n → usar require('https') sempre
  • ORG_ID hardcoded → sempre via parâmetro ou variável de ambiente
  • Hex colors hardcoded → sempre via CSS vars do DS v5
  • Workflow n8n por nome → sempre por ID (tabela olivia_workflows_canonicos)
  • Arquivo >500 linhas → dividir em sub-componentes antes de continuar
  • Concluir modo sem E2E passando → proibido, independente do estado do código
