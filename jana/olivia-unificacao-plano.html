📝 Prompt Universal — Mande em Cada Conversa de Grupo

Copie e mande nas 3 conversas de grupo (Smart Blocks, Express, Modo Agente). Cada uma deve gerar uma spec completa para ser colada na conversa central com a JANA.

Preciso que você gere uma SPEC COMPLETA deste modo do gerador de petições OlivIA. Vou centralizar tudo numa única conversa com a JANA para finalizar os 3 modos juntos. A spec deve conter: ## 1. ESTADO ATUAL - % de conclusão estimado (o que funciona, o que não funciona) - Lista de bugs conhecidos (P0/P1/P2/P3) com descrição técnica - O que foi testado e confirmado funcionando - O que nunca foi testado até o fim ## 2. ARQUITETURA FRONTEND - Arquivo orquestrador principal (caminho completo no repo) - Hook principal usado (ex: useOliviaWizard, useSmartBlocks, useOliviaAgente) - Todos os steps do wizard (nome + componente + arquivo) - Estado gerenciado (campos do WizardState ou equivalente) - Componentes críticos e seus caminhos ## 3. BACKEND E INTEGRAÇÕES - Workflow(s) n8n: nome + ID + status (ativo/inativo) + o que faz - Endpoints chamados: URL completa + método + payload esperado - Polling/SSE: como o frontend sabe que terminou (tabela, campo, intervalo) - Google Drive: folder ID usado, como o doc é gerado - Se usa JARBAS: endpoint + skill + comportamento SSE ## 4. BANCO DE DADOS (SUPABASE) - Tabelas usadas: nome + campos relevantes para este modo - Tabela de histórico/status: qual campo indica conclusão, valores possíveis - RLS relevante se houver ## 5. DECISÕES TOMADAS NESTA CONVERSA - Lista de decisões de arquitetura tomadas aqui (com o porquê) - Mudanças de design/UX decididas - O que foi descartado e por quê ## 6. O QUE FALTA PARA 100% - Lista priorizada de tarefas restantes - Dependências entre tarefas - Estimativa de complexidade (horas ou dias) ## 7. ARQUIVOS E RELATÓRIOS - Links de relatórios HTML gerados nesta conversa (GitHub Pages) - Commits relevantes (hash + descrição) - Qualquer arquivo de spec/doc gerado Acesse o repositório GitHub (beta-mdflow) e leia os arquivos relevantes antes de gerar a spec.
