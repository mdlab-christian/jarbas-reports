OlivIA Upload — Plano Definitivo

Bucket dedicado olivia-docs · estrutura por processo · integração Agente · 2026-03-21

1. Por que não funcionou até agora

Bucket documentos — 3 problemas simultâneos

ProblemaDetalheStatus
RLS policy de INSERT exige path começando com organizacao_id Código usava processos/{id}/... → primeiro folder = "processos" → bloqueado Parcialmente corrigido
Tabela processos_documentos — campo entidade_tipo NOT NULL Insert sem esse campo viola constraint Corrigido
Campo tipo com CHECK constraint Código enviava categoria (ex: "contestacao") mas CHECK aceita apenas 'gerado','upload','importado','eproc' Corrigido

Mesmo com os últimos fixes, o bucket documentos tem mime types limitados (só PDF/imagens, não DOCX) e file size limit de 10MB. Além disso, mistura documentos de toda a plataforma — dificulta acesso isolado pelo Agente OlivIA.

2. Decisão — Novo bucket olivia-docs

✅
Viável e recomendado. Bucket dedicado para a OlivIA com estrutura clara, RLS simples e integração direta com o Agente. Sem impacto em outros módulos do MdFlow.
OpçãoPrósContrasDecisão
Continuar com documentos Zero migration RLS complexa, mime types limitados, mistura de dados, difícil acesso isolado pelo Agente Descartado
Usar olivia-provas (já existe) Já existe, público, mime types ok Público = sem RLS de org, nome confuso para contexto de processo Descartado
Novo bucket olivia-docs RLS limpa, estrutura por processo, mime types corretos, acesso isolado pelo Agente, 50MB limit Migration necessária (simples) ✅ Escolhido

3. Estrutura do bucket olivia-docs

olivia-docs/
  {organizacao_id}/ ← folder 1: org (satisfaz RLS)
    processos/
      {processo_id}/ ← folder 3: isolamento por processo
        {uuid}-contestacao.pdf
        {uuid}-comprovante.pdf
        {uuid}-contrato.docx

O Agente acessa: olivia-docs/{org_id}/processos/{processo_id}/* para listar e baixar todos os documentos do processo de forma isolada.

4. Plano de Execução — 4 etapas

1
Criar bucket + RLS via migration Supabase

Migration SQL que cria o bucket e as 4 policies (SELECT/INSERT/UPDATE/DELETE).

-- Criar bucket olivia-docs
INSERT INTO storage.buckets (id, name, public, file_size_limit, allowed_mime_types)
VALUES (
  'olivia-docs',
  'olivia-docs',
  false,
  52428800, -- 50MB
  ARRAY[
    'application/pdf',
    'application/msword',
    'application/vnd.openxmlformats-officedocument.wordprocessingml.document',
    'text/plain',
    'image/jpeg',
    'image/png'
  ]
);

-- Policy SELECT: usuários da mesma org
CREATE POLICY "olivia_docs_select_org"
ON storage.objects FOR SELECT
USING (
  bucket_id = 'olivia-docs'
  AND (storage.foldername(name))[1] IN (
    SELECT organizacao_id::text FROM users WHERE id = auth.uid()
  )
);

-- Policy INSERT: usuários da mesma org
CREATE POLICY "olivia_docs_insert_org"
ON storage.objects FOR INSERT
WITH CHECK (
  bucket_id = 'olivia-docs'
  AND (storage.foldername(name))[1] IN (
    SELECT organizacao_id::text FROM users WHERE id = auth.uid()
  )
);

-- Policy UPDATE: usuários da mesma org
CREATE POLICY "olivia_docs_update_org"
ON storage.objects FOR UPDATE
USING (
  bucket_id = 'olivia-docs'
  AND (storage.foldername(name))[1] IN (
    SELECT organizacao_id::text FROM users WHERE id = auth.uid()
  )
);

-- Policy DELETE: apenas admin da org
CREATE POLICY "olivia_docs_delete_org"
ON storage.objects FOR DELETE
USING (
  bucket_id = 'olivia-docs'
  AND (storage.foldername(name))[1] IN (
    SELECT organizacao_id::text FROM users WHERE id = auth.uid()
  )
  AND (auth.jwt() ->> 'perfil') = 'admin'
);

4
Como o Agente OlivIA acessa os documentos

O Agente (Step 1 ou Step 2) busca documentos do processo via:

// Opção A — via tabela (preferida, já filtra por org via RLS):
const { data: docs } = await supabase
  .from('processos_documentos')
  .select('id, nome, categoria, storage_path, arquivo_url')
  .eq('processo_id', processoId)
  .eq('entidade_tipo', 'processo')
  .eq('tipo', 'upload')
  .eq('status', 'disponivel')
  .is('deleted_at', null);

// Para cada doc, gerar URL assinada (bucket privado):
const { data: signed } = await supabase.storage
  .from('olivia-docs')
  .createSignedUrl(doc.storage_path, 3600); // 1h de validade

// Enviar signed.signedUrl para o webhook do Agente junto com o contexto

O Agente recebe um array de URLs assinadas + metadados (categoria, nome) e pode fazer download/análise de cada documento dentro do timeout da sessão.
