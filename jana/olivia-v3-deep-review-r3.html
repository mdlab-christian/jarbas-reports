2. Bugs Found (12)

IDSeverityFile : LineDescription
B-01 P1 shared/GeradorStep1.tsx 548 lines violates 500-line limit (D-FE-08). Dual v2/v3 interface with discriminated union adds complexity. Contains JARBAS health check, placeholder validation, upload drawer, cadastro rapido, search, processo enrichment -- all in one file.
B-02 P0 hooks/useExpressGeneration.ts : L167 startPolling not in dispararGeracao dependency array. The function startPolling (defined at L203) is called inside dispararGeracao (L157) but is NOT listed in its useCallback deps at L167. This means if startPolling's identity changes (e.g. updateState changes), dispararGeracao will still use the stale closure. Since startPolling itself is a useCallback with [updateState] as dependency, and updateState is stable (useCallback([], [])), this is a latent bug -- currently safe but fragile.
B-03 P2 express/ExpressStep2Triagem.tsx : L59-61 DEV mode silently falls back to mock data on ANY error. If the webhook fails in development, mock triagem questions are shown without any warning to the developer. Should at least log a console.warn or show a toast.
B-04 P0 agente/AgenteStep4Geracao.tsx : L73-75 qualidadeIa maps to invalid DB CHECK values.
const qualidadeApi = session.qualidadeIa === 'avancado' ? 'pensamento_estendido' : session.qualidadeIa === 'premium' ? 'opus' : session.qualidadeIa;
The values 'pensamento_estendido' and 'opus' are NOT valid olivia_historico.qualidade_ia CHECK constraint values. Spec defines only 'padrao' and 'avancado'. The INSERT at L80-91 will fail with a CHECK constraint violation when qualidade is 'avancado' or 'premium'.
B-05 P0 agente/AgenteStep6Conclusao.tsx : L32 Writes to non-existent columns.
.update({ feedback_nota: nota, feedback_comentario: comentario || null })
Schema has feedback_advogado (TEXT), NOT feedback_nota/feedback_comentario. This UPDATE silently fails -- Supabase ignores unknown columns with anon key.
B-06 P1 smart-blocks/SmartBlocksStep3Geracao.tsx : L189 feedback_advogado written as JSONB object.
feedback_advogado: { nota, comentario, enviado_em }
If the column is TEXT (as Memory states), this will write [object Object]. If JSONB, this is correct. Inconsistent with Express (writes string number) and Agente (writes to wrong columns).
B-07 P2 express/ExpressStep3Outline.tsx : L48-50 useEffect fires onChange on every sections change, potentially causing re-render loops.
useEffect(() => { onChange(sections); }, [sections, onChange]);
If parent's onChange causes a re-render that passes new props, this creates a loop. Currently mitigated because handleOutlineChange in parent is useCallback-stable, but fragile pattern.
B-08 P2 hooks/useSmartBlocks.ts : L170 console.debug left in production code.
console.debug('[useSmartBlocks] modeloPeticaoId:', modeloPeticaoId, ...)
Should be removed or gated behind DEV flag.
B-09 P2 types.ts : L70 OliviaHistoricoTipo includes 'MANUAL' which is not in DB CHECK. CHECK is: EXPRESS|SMART_BLOCKS|JARBAS|VIBELAW|BULK. 'MANUAL' is legacy v2 value. The type allows 'IA_ASSISTIDA' which also doesn't exist.
B-10 P1 types.ts : L183 AgenteSession.qualidadeIa includes 'premium' which is not in spec or in the QUALIDADE_IA_OPTIONS constant (constants.ts L23-26 defines only padrao/avancado). The type and the AgenteStep3Editor UI both offer 3 options. The 'premium' tier maps to 'opus' in AgenteStep4 which would fail the DB CHECK.
B-11 P2 gerador/OliviaGeradorExpress.tsx : L138 Back button uses <Link to="/olivia"> instead of parent onBack callback. This navigates to /olivia (full page), bypassing the mode selection state managed in index.tsx. Should call handleBackToLanding like Agente mode does, but Express doesn't receive this callback. Functional but breaks the SPA pattern.
B-12 P2 agente/AgenteStep3Editor.tsx : L16, L115-144 3 quality options in UI but spec says 2. UI renders padrao/avancado/premium RadioGroup items. The 'premium' option (Opus 4.5) is not in QUALIDADE_IA_OPTIONS or the Master Plan. Creates inconsistency between Express (2 options) and Agente (3 options).
