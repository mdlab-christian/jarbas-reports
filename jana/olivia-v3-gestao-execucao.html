Comunicar status ao Christian a cada wave
Query Supabase para verificações de estado
Monitorar terminais CC ativos (watchdog)
Merge de resultados de terminais paralelos
Editar/lapidar código diretamente (bug fixes óbvios)
💻 Claude Code — Executor
Responsabilidades
Implementar componentes React conforme spec
Aplicar fixes nos workflows n8n (JSON export)
Executar migrations SQL no Supabase
Criar/modificar hooks TypeScript
Implementar skills JARBAS (Mac Mini via SSH)
Testes unitários + E2E Playwright
Refatoração de código (max 500 linhas/arquivo)
Integração com callWebhook() centralizado
Configurar TipTap v2 + dnd-kit
Implementar SSE streaming (VibeLaw futuro)
Git commits semânticos após cada etapa

🔀 Fluxo por etapa

JANA → gera cc_prompt enriquecido (com contexto, gaps, spec) ↓ JANA → confirma pré-requisitos (banco, arquivos, dependências) ↓ CC → executa etapa (implementação, fix, migration) ↓ CC → faz commit + cria evidência (screenshot ou output) ↓ JANA → revisa: compara spec vs implementado ↓ JANA → se nota ≥ 8/10 → aprova + próxima etapa JANA → se nota < 8/10 → identifica gaps → manda CC corrigir ↓ Ao final de cada wave → JANA envia status ao Christian

n8n Workflows
