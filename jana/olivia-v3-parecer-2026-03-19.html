Análise: olivia_flow vs Flow de Modelos

O que é olivia_flow (no banco):

Tabela que armazena o "flow" de tags por modelo_grupo — basicamente é a lista ordenada de perguntas (tags) que compõem o wizard Smart Blocks para cada modelo. É necessária — o Smart Blocks Wizard usa para carregar as tags na ordem correta.

olivia_flow: 0 rows (vazia)
Precisa ser populada via E11 seed SQL
Campos: flow_ativo, validation_status, flow (JSONB)

Funcionalidade "Flow" na página /modelos:

Interface UI para gerenciar os flows (editar ordem de tags por modelo). Você disse que nunca usou — o sistema foi para o mapa conceitual mas nunca virou feature real. A tabela existe e precisa de dados, mas a UI de gerenciamento de flows na página Modelos pode ser simplificada.

Minha recomendação:

Manter a tabela olivia_flow (é necessária para Smart Blocks), mas remover a UI de gerenciamento de flows da página Modelos. O seed automático via SQL (E11) elimina a necessidade da interface manual. Se algum dia precisar editar flows, pode ser via Supabase diretamente ou uma UI futura.
