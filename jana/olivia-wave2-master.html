🐛 Bugs P0 Encontrados e Corrigidos
⚠️
Detectados no teste real no Lovable — corrigidos antes da Wave 2

Você reportou: modelos não aparecem no Step 1, erro de RLS no upload. Diagnóstico feito via consulta ao banco real. 2 bugs raiz encontrados e corrigidos.

BUG 1 · P0 ModeloPeticaoSelector — modelos nunca aparecem
Causa raiz 1: Query usava .is('deleted_at', null) mas a coluna deleted_at não existe em olivia_modelos_peticao. Isso fazia a query falhar silenciosamente → lista vazia.

Causa raiz 2: enabled exigia processoId preenchido para ativar (props v2). No modo Express, o processo só é selecionado via SearchBox — antes de selecionar, modelos nunca carregavam.

Fix: Removido .is('deleted_at', null). enabled simplificado para !!effectiveOrgId — modelos carregam assim que a página abre, prontos para quando processo for selecionado.

Arquivo: src/pages/olivia/components/shared/ModeloPeticaoSelector.tsx
BUG 2 · P0 UploadDrawer — erro RLS ao listar e enviar documentos
Causa raiz: O componente acessava processo_documentos (sem 's') mas a tabela real no banco é processos_documentos (com 's'). Além disso, usava o campo storage_url que não existe — o campo real é arquivo_url.

Isso causava erro de RLS porque a tabela errada nem existe, gerando "permission denied" ou "relation does not exist".

Fix: Corrigido nome da tabela em todas as chamadas (query + insert). Corrigido campo storage_url → arquivo_url.

Arquivo: src/pages/olivia/components/shared/UploadDrawer.tsx
