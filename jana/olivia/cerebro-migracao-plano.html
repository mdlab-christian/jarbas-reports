RAG — Limpar e Reconstruir com Hierarquia

Embeddings com metadados completos · pgvector · text-embedding-3-small
⚠️
Problema crítico a resolver:

As jurisprudências têm tag_ids = NULL. Isso quebra a busca semântica por tag. Durante a migração, JANA vai inferir tags das jurisprudências baseado no conteúdo (texto_integral) × lista de tags existentes, e popular tag_ids no banco.

O que muda no RAG novo

AspectoRAG atualRAG novo
MetadadosIncompletostag_id + codigo + categoria + secao + tribunal
Texto embedadoconteudo brutoMarkdown limpo (html2md) — mais semântico
Juris → tagsNULLInferido por Claude + populado no banco
BuscaSó semânticaFiltros estruturais + re-rank semântico

Metadados no embedding (crítico)

// O texto que vai ser embedado (blocos) // Inclui metadados no início para melhor semântica texto_para_embedding = ` TAG: BV - SMS | SEÇÃO: fundamentacao CATEGORIA: Órgão Restritivo TÍTULO: Da Ausência de Comunicação Formal [conteúdo markdown do bloco] ` // Para jurisprudências: texto_para_embedding = ` TRIBUNAL: TJRS | CATEGORIA: Instituições Financeiras TÍTULO: Restrição indevida sem contrato EMENTA: [campo ementa] TEXTO: [primeiros 500 chars do texto_integral] `

Estratégia de busca na skill OlivIA

// Função de seleção de blocos para uma tag async function selecionarBlocoParaTag(tagId, categoriaEmpresaId, uf, contextoProcesso) { // Passo 1: filtro estrutural (rápido) const candidatos = await supabase .from('olivia_blocos_texto') .select('id, titulo, conteudo, embedding') .contains('tag_ids', [tagId]) // tag correta .eq('categoria_empresa_id', categoriaEmpresaId) // categoria correta .eq('status', 'published') .is('deleted_at', null); // Passo 2: re-rank por similaridade semântica const embCtx = await gerarEmbedding(contextoProcesso); const ranked = candidatos.data .map(b => ({ ...b, score: cosineSim(b.embedding, embCtx) })) .sort((a, b) => b.score - a.score); return ranked[0]; // melhor bloco para o contexto }
