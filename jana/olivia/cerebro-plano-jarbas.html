O que vai acontecer — Explicado de forma simples

Visão geral humanizada de cada etapa

O JARBAS vai executar um pipeline de migração em loop, processando etapa por etapa e te avisando a cada ~2 minutos. Enquanto ele roda em background, você pode trabalhar no frontend normalmente.

A
Preparação do ambiente ~10min
JARBAS cria o bucket privado no Supabase, testa a conexão com o Google Drive (usando as credenciais já no .env), e cria a pasta compartilhada /Users/Shared/olivia-vault/ que tanto o JARBAS quanto a OlivIA conseguem acessar.
B
Exportar os 177 modelos de petição do Google Drive ~3h background
Para cada modelo de petição (template jurídico), o JARBAS baixa o Google Doc como HTML, converte também para Markdown, cria um arquivo de metadados (meta.json) com todas as vinculações (tipo de petição, categoria de empresa, tags), e salva no bucket. Se algum arquivo estiver corrompido no Drive, pula e registra no log. Avisa a cada 2min: "✅ 45/177 modelos OK".
C
Exportar os 1.244 blocos de texto do Google Drive ~2h background
Os blocos são o conteúdo jurídico real das petições — argumentos, impugnações, fundamentações. JARBAS exporta cada um do Google Drive como HTML e Markdown, organizados por tag (pasta). Se o Drive falhar, usa o conteúdo HTML que já temos no banco como backup. Nunca perde um bloco.
D
Exportar as 999 jurisprudências + inferir tags ~2h background
JARBAS exporta cada jurisprudência do Drive (HTML + Markdown), separadas por tribunal (TJRS/TJSC/TJSP). Depois faz algo inteligente: usa Claude em batches de 20 para inferir quais tags jurídicas cada jurisprudência representa — e salva isso no banco. Isso resolve um problema existente (tag_ids estava NULL em quase todas).
E
Criar as 183 tags como Markdown ~5min
Tags são metadados puros (não têm Google Doc). JARBAS gera um arquivo .md para cada tag com seus metadados (seção, obrigatória, código) e lista de blocos vinculados. Isso cria o mapa de navegação do Obsidian.
F
Gerar os 10 playbooks jurídicos com IA ~30min
Após ter todos os dados, JARBAS pede para o Claude criar um playbook estratégico para cada categoria de empresa (Energia Elétrica, Instituições Financeiras, etc.), baseado nos títulos dos blocos e jurisprudências já importados. Resultado: 10 documentos Markdown + HTML com a estratégia jurídica de cada categoria.
G
Enriquecimento do banco novo ~30min
JARBAS aproveita o fluxo para enriquecer o backend: preenche campos que estavam vazios (tipo_peticao_id nas versões, tag_ids nas jurisprudências), cria os índices de categorias, e atualiza os playbooks na tabela olivia_playbooks do banco.
H
Reconstruir o RAG com hierarquia correta ~45min
Por último, JARBAS limpa os embeddings antigos e recria todos com metadados enriquecidos no texto (TAG: X | CATEGORIA: Y | SEÇÃO: Z). Usa text-embedding-3-small em batches de 20. Faz uma busca de validação ao final.
⏱
Tempo total estimado: ~8h rodando em background

B, C e D podem rodar em paralelo (3 terminais). Aviso a cada 2min. Você pode fechar o chat — ele continua rodando no Mac Mini e te notifica via Telegram ao concluir cada etapa.

Problemas Identificados e Soluções

Issues encontrados na revisão — todos resolvidos
Problema 1 — Permissão

JARBAS não tem sudo para escrever em /Users/olivia/

JARBAS é usuário admin mas não tem senha para sudo -u olivia. Não consegue escrever diretamente no vault.

✅ Solução

Usar /Users/Shared/olivia-vault/ — pasta do macOS acessível por todos os usuários sem sudo. JARBAS escreve lá. OlivIA lê de lá (a skill vai usar este caminho).

# Criar pasta compartilhada (JARBAS consegue) mkdir -p /Users/Shared/olivia-vault/{playbooks,blocos,jurisprudencias/TJRS,jurisprudencias/TJSC,jurisprudencias/TJSP,tags,modelos,categorias} chmod -R 777 /Users/Shared/olivia-vault
Problema 2 — Node/fetch

node não estava no PATH do SSH

Ao fazer SSH no Mac Mini, o PATH não carrega o nvm automaticamente. Scripts precisam apontar para o node correto.

✅ Solução
# Usar caminho absoluto ou source nvm primeiro source ~/.nvm/nvm.sh 2>/dev/null NODE=$(which node || echo /usr/local/bin/node) # fetch é nativo no Node 21+ — não precisa node-fetch
Problema 3 — Rate limit Drive

Drive API tem limite de ~10 req/s

Exportar 1.244 blocos + 999 juris + 329 modelos = ~2.572 requests. Sem controle, vai dar 429.

✅ Solução

500ms entre requests + retry automático com 10s de espera no 429. Blocos têm fallback no campo conteudo do banco. Juris têm fallback em texto_integral. Só modelos não têm fallback — esses são pulados e logados.

Problema 4 — Loop sem feedback

Script longo sem retorno = parece travado

Rodar 8h sem sinal de vida é arriscado — não dá pra saber se travou.

✅ Solução — Heartbeat

Script envia mensagem via Telegram Bot a cada 2min com progresso atual. Se o script morrer, o heartbeat para — e o silêncio é o alerta.

// A cada 2min: POST para Telegram "⏳ Migração em andamento: B=45/177 C=120/1244 D=0/999"

// ─── Heartbeat ───────────────────────────────────────────────────────────────
async function telegram(msg) {
  try {

function iniciarHeartbeat(getStatus) {
  return setInterval(async () => {
    await telegram(`⏳ Migração OlivIA\n${getStatus()}`);
  }, 2 * 60 * 1000); // 2 minutos
}

// ─── Utilitários ─────────────────────────────────────────────────────────────
async function sleep(ms) { return new Promise(r => setTimeout(r, ms)); }

function log(msg) { console.log(`[${new Date().toISOString().slice(11,19)}] ${msg}`); }

function logerro(msg) { appendFileSync(ERROS, msg + '\n'); log('ERRO: ' + msg); }

function htmlToMd(html) {
  return html
    .replace(//gi, '\n')
    .replace(/<\/p>/gi, '\n\n').replace(/]*>/gi, '')
    .replace(/(.*?)<\/strong>/gi, '**$1**')
    .replace(/(.*?)<\/b>/gi, '**$1**')
    .replace(/(.*?)<\/em>/gi, '*$1*')
    .replace(/]*>(.*?)<\/h[1-6]>/gi, (m,t) => '## ' + t + '\n')
    .replace(/]*>(.*?)<\/li>/gi, '- $1\n')
    .replace(/<[^>]+>/g, '')
    .replace(/&/g,'&').replace(/</g,'<').replace(/>/g,'>')
    .replace(/ /g,' ').replace(/é/g,'é').replace(/ç/g,'ç')
    .replace(/ã/g,'ã').replace(/õ/g,'õ').replace(/í/g,'í')
    .replace(/ó/g,'ó').replace(/ú/g,'ú').replace(/á/g,'á')
    .replace(/ê/g,'ê').replace(/ô/g,'ô').replace(/â/g,'â')
    .replace(/Ã/g,'Ã').replace(/É/g,'É').replace(/Í/g,'Í')
    .replace(/\n{3,}/g,'\n\n').trim();
}

async function getToken() {
  const r = await fetch('https://oauth2.googleapis.com/token', {
    method:'POST', headers:{'Content-Type':'application/x-www-form-urlencoded'},
    body: new URLSearchParams({
      client_id: process.env.GOOGLE_CLIENT_ID,
      client_secret: process.env.GOOGLE_CLIENT_SECRET,
      refresh_token: process.env.GOOGLE_REFRESH_TOKEN,
      grant_type: 'refresh_token'
    })
  });
  const d = await r.json();
  if (!d.access_token) throw new Error('Token Drive inválido: ' + JSON.stringify(d));
  return d.access_token;
}

async function exportDrive(docId, retries = 1) {
  try {
    const tkn = await getToken();
    const r = await fetch(
      `https://www.googleapis.com/drive/v3/files/${docId}/export?mimeType=text/html`,
      { headers: { Authorization: `Bearer ${tkn}` } }
    );
    if (r.status === 429) {
      if (retries > 0) { await sleep(10000); return exportDrive(docId, retries - 1); }
      logerro(`429_LIMIT ${docId}`); return null;
    }
    if (!r.ok) { logerro(`HTTP${r.status} ${docId}`); return null; }
    const html = await r.text();
    // Extrair body
    const body = html.match(/]*>([\s\S]*)<\/body>/i)?.[1] || html;
    return `${body}`;
  } catch(e) {
    logerro(`EXCEPTION ${docId}: ${e.message}`); return null;
  }
}

async function uploadBucket(path, content, type = 'text/html') {
  const { error } = await supabase.storage.from('olivia-modelos')
    .upload(path, Buffer.from(content, 'utf8'), { contentType: type, upsert: true });
  if (error) logerro(`UPLOAD_ERR ${path}: ${error.message}`);
}

function vaultWrite(relPath, content) {
  const full = `${VAULT}/${relPath}`;
  mkdirSync(full.substring(0, full.lastIndexOf('/')), { recursive: true });
  writeFileSync(full, content, 'utf8');
}

// ─── ETAPA B: Modelos ─────────────────────────────────────────────────────────
async function migrarModelos() {
  log('=== ETAPA B: Modelos ===');
  const { data: grupos } = await supabase.from('olivia_modelos_grupo')
    .select('id,nome,tipo_peticao_id,categoria_empresa_id,active_version_id,tags_ids')
    .eq('organizacao_id', ORG).eq('status','published').is('deleted_at',null);
  const { data: versoes } = await supabase.from('olivia_modelos_peticao')
    .select('id,nome,google_doc_id').eq('organizacao_id', ORG).is('deleted_at',null);
  const { data: modeloTags } = await supabase.from('olivia_modelo_tags')
    .select('modelo_id,tag_id,obrigatoria,ordem').is('deleted_at',null);
  const { data: tags } = await supabase.from('olivia_tags').select('id,codigo,nome').eq('organizacao_id',ORG);
  const { data: tipos } = await supabase.from('olivia_tipos_peticao').select('id,nome');
  const { data: cats } = await supabase.from('empresas_categorias').select('id,nome');

  const tagMap = Object.fromEntries((tags||[]).map(t=>[t.id,t]));
  const tipoMap = Object.fromEntries((tipos||[]).map(t=>[t.id,t]));
  const catMap = Object.fromEntries((cats||[]).map(c=>[c.id,c]));
  const mtMap = {};
  for (const mt of (modeloTags||[])) {
    if (!mtMap[mt.modelo_id]) mtMap[mt.modelo_id] = [];
    mtMap[mt.modelo_id].push(mt);
  }

  let ok = 0, skip = 0;
  const total = grupos?.length || 0;

  const hb = iniciarHeartbeat(() => `B: ${ok}/${total} modelos | ${skip} pulados`);

  for (const g of (grupos||[])) {
    const versao = versoes?.find(v => v.id === g.active_version_id);
    if (!versao?.google_doc_id) { skip++; continue; }

    const html = await exportDrive(versao.google_doc_id);
    if (!html) { skip++; await sleep(500); continue; }

    const md = htmlToMd(html);
    const tagsModelo = (mtMap[versao.id]||[])
      .sort((a,b) => a.ordem - b.ordem)
      .map(mt => ({ ...mt, tag: tagMap[mt.tag_id] }))
      .filter(mt => mt.tag);

    const meta = {
      grupo_id: g.id, nome: g.nome,
      tipo_peticao_id: g.tipo_peticao_id,
      tipo_peticao_nome: tipoMap[g.tipo_peticao_id]?.nome || '',
      categoria_empresa_id: g.categoria_empresa_id,
      categoria_empresa_nome: catMap[g.categoria_empresa_id]?.nome || '',
      active_version_id: versao.id,
      versoes: [{ id: versao.id, arquivo: 'v1.html', google_doc_id: versao.google_doc_id }],
      tags: tagsModelo.map(mt => ({
        tag_id: mt.tag_id, codigo: mt.tag.codigo, nome: mt.tag.nome,
        obrigatoria: mt.obrigatoria, ordem: mt.ordem
      })),
      migrado_em: new Date().toISOString()
    };

    await uploadBucket(`modelos/${g.id}/v1.html`, html);
    await uploadBucket(`modelos/${g.id}/v1.md`, md, 'text/markdown');
    await uploadBucket(`modelos/${g.id}/meta.json`, JSON.stringify(meta,null,2), 'application/json');

    const vaultMd = `---\ngrupo_id: ${g.id}\ntipo: ${meta.tipo_peticao_nome}\ncategoria: ${meta.categoria_empresa_nome}\ntags_count: ${tagsModelo.length}\n---\n\n# ${g.nome}\n\n**Tipo:** ${meta.tipo_peticao_nome}  \n**Categoria:** ${meta.categoria_empresa_nome}\n\n## Tags\n${tagsModelo.map(mt=>`- [${mt.tag.codigo}] ${mt.tag.nome}${mt.obrigatoria?' *(obrigatória)*':''}`).join('\n')}\n`;
    vaultWrite(`modelos/${g.id}.md`, vaultMd);

    ok++;
    if (ok % 10 === 0) log(`B: ${ok}/${total}`);
    await sleep(500);
  }

  clearInterval(hb);
  const msg = `✅ ETAPA B concluída!\nModelos: ${ok} OK | ${skip} pulados\nVer /tmp/erros-migracao.log`;
  await telegram(msg);
  log(msg);

// ─── ETAPA C: Blocos ──────────────────────────────────────────────────────────
async function migrarBlocos() {
  log('=== ETAPA C: Blocos ===');
  const { data: blocos } = await supabase.from('olivia_blocos_texto')
    .select('id,titulo,conteudo,tag_ids,categoria_empresa_id,google_doc_id,version')
    .eq('organizacao_id', ORG).eq('status','published').is('deleted_at',null)
    .range(0, 1500);
  const { data: tags } = await supabase.from('olivia_tags').select('id,codigo,nome').eq('organizacao_id',ORG);
  const { data: cats } = await supabase.from('empresas_categorias').select('id,nome');
  const tagMap = Object.fromEntries((tags||[]).map(t=>[t.id,t]));
  const catMap = Object.fromEntries((cats||[]).map(c=>[c.id,c]));

  let ok = 0, fallback = 0, skip = 0;
  const total = blocos?.length || 0;
  const hb = iniciarHeartbeat(() => `C: ${ok}/${total} blocos | ${fallback} fallback | ${skip} erros`);

  for (const b of (blocos||[])) {
    let html = await exportDrive(b.google_doc_id);
    if (!html) {
      // Fallback: usar campo conteudo do banco
      if (b.conteudo) {
        html = `${b.conteudo}`;
        fallback++;
      } else { skip++; await sleep(500); continue; }
    }
    const md = htmlToMd(html);
    const tagCodigo = tagMap[b.tag_ids?.[0]]?.codigo || 'sem_tag';
    const catNome = catMap[b.categoria_empresa_id]?.nome || '';
    const tagNome = tagMap[b.tag_ids?.[0]]?.nome || '';

    const frontmatter = `---\nid: ${b.id}\ntitulo: "${(b.titulo||'').replace(/"/g,"'")}"\ntag_ids: [${(b.tag_ids||[]).join(', ')}]\ntag_codigo: ${tagCodigo}\ntag_nome: "${tagNome}"\ncategoria_empresa_id: ${b.categoria_empresa_id||''}\ncategoria_nome: "${catNome}"\nversao: ${b.version||1}\nsupabase_id: ${b.id}\n---\n\n## ${b.titulo||'Bloco'}\n\n${md}`;

    await uploadBucket(`blocos/${tagCodigo}/${b.id}.html`, html);
    await uploadBucket(`blocos/${tagCodigo}/${b.id}.md`, frontmatter, 'text/markdown');
    vaultWrite(`blocos/${tagCodigo}/${b.id}.md`, frontmatter);

    ok++;
    if (ok % 50 === 0) log(`C: ${ok}/${total}`);
    await sleep(500);
  }

  clearInterval(hb);
  const msg = `✅ ETAPA C concluída!\nBlocos: ${ok} OK | ${fallback} via fallback | ${skip} erros`;
  await telegram(msg);
  log(msg);
}

async function inferirTagsJuris(juris) {
  log('=== Inferindo tags das jurisprudências ===');
  const { data: tags } = await supabase.from('olivia_tags').select('id,codigo,nome').eq('organizacao_id',ORG);
  const tagLista = (tags||[]).map(t=>`${t.codigo}: ${t.nome}`).join('\n');
  const BATCH = 20;
  let atualizadas = 0;

  for (let i = 0; i < juris.length; i += BATCH) {
    const lote = juris.slice(i, i + BATCH);
    const prompt = `Você é advogado especialista em direito do consumidor.
Lista de tags disponíveis (codigo: nome):
${tagLista}

Para cada jurisprudência abaixo, selecione 0 a 4 tags mais relevantes.
Retorne APENAS JSON válido: [{"id":"uuid","tag_ids":["uuid1"]}]

Jurisprudências:
${lote.map((j,idx) => `${idx+1}. id:${j.id} | titulo:${j.titulo?.slice(0,80)} | ementa:${j.ementa?.slice(0,150)}`).join('\n')}`;

    try {
      const resp = await fetch('https://api.anthropic.com/v1/messages', {
        method:'POST',
        headers:{'Content-Type':'application/json','x-api-key':process.env.ANTHROPIC_API_KEY,'anthropic-version':'2023-06-01'},
        body: JSON.stringify({ model:'claude-haiku-4-5', max_tokens:2000, messages:[{role:'user',content:prompt}] })
      });
      const d = await resp.json();
      const text = d.content?.[0]?.text || '[]';
      const jsonMatch = text.match(/\[[\s\S]*\]/);
      if (!jsonMatch) continue;
      const resultados = JSON.parse(jsonMatch[0]);
      for (const r of resultados) {
        if (!r.id || !r.tag_ids?.length) continue;
        await supabase.from('olivia_jurisprudencias')
          .update({ tag_ids: r.tag_ids })
          .eq('id', r.id);
        atualizadas++;
      }
    } catch(e) { log('Erro inferência tags lote ' + i + ': ' + e.message); }
    await sleep(1000);
  }
  log(`Tags inferidas e atualizadas no banco: ${atualizadas}`);
}

async function migrarTags() {
  log('=== ETAPA E: Tags ===');

}
