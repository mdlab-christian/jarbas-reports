🐛 Bug P0-1 — SmartBlocks NÃO chega ao gera-peticao

Causa Raiz

O server.mjs linha 321 passa o objeto inteiro do SmartBlocks com o nome errado:

// ❌ ERRADO — passa o objeto como campo "smart_blocks_base"
const peticaoResult = await geraPeticao({
  ...
  smart_blocks_base: smartBlocksBase,  // ← campo desconhecido pela skill
  ...
});

A skill gera-peticao.mjs desestrutura do input diretamente:

// gera-peticao.mjs — desestrutura no topo
const {
  peticao_base = "",
  peticao_base_texto = "",   // ← espera isso
  peticao_html_base = "",    // ← e isso
  ...
} = input;

Como smart_blocks_base não bate com nenhum campo esperado, o template fica vazio → o LLM gera do zero.

Fix

// ✅ CORRETO — expandir os campos do smartBlocksBase no nível raiz
const peticaoResult = await geraPeticao({
  historico_id, organizacao_id, processo_id,
  tipo_peticao_id, modelo_id, versao_id,
  user_id, advogado_id, qualidade_ia,
  perguntas_respondidas: perguntas,
  analise_provas: analiseProvas,
  jurisprudencias,
  // ✅ Expandir SmartBlocks no nível raiz
  peticao_base_texto: smartBlocksBase?.peticao_base_texto || '',
  peticao_html_base: smartBlocksBase?.peticao_html_base || '',
  instrucoes_ia: smartBlocksBase?.instrucoes_ia || '',
  blocos_usados: smartBlocksBase?.blocos_usados || [],
  observacoes,
  dados_processo,
});

🐛 Bug P0-2 — Step 2 pula perguntas (race condition)

Causa Raiz

Race condition entre polling inicial e Realtime:

  1. Frontend monta Step 2 → carregarPerguntas() roda imediatamente
  2. Se status ainda é PROCESSANDO → isLoading continua true ✅
  3. Mas se status AINDA é PENDENTE ou AGUARDA_RESPOSTAS já chegou antes do Realtime se estabelecer → isLoading=false, perguntas=[]
  4. Auto-advance useEffect: isLoading=false && perguntas.length===0 → avança após 1.5s
  5. Realtime chega 200ms depois com as 13 perguntas... mas já avançou

Adicionalmente: quando se volta para o Step 2, carregarPerguntas() lê diretamente do banco e encontra as perguntas → aparecem.

Fix

// ExpressStep2Triagem.tsx — modificar auto-advance
// ❌ ATUAL: avança sempre que isLoading=false e sem perguntas
useEffect(() => {
  if (!isLoading && perguntas.length === 0 && !fetchError && historicoId) {
    const timer = setTimeout(() => onSubmit(...), 1500);
    return () => clearTimeout(timer);
  }
}, [isLoading, perguntas.length, ...]);

// ✅ FIX: só avançar se status for CONCLUIDO/SUCESSO (não AGUARDA_RESPOSTAS)
// + aumentar delay para 3s para dar tempo ao Realtime
const [statusFinal, setStatusFinal] = useState<string | null>(null);

// No Realtime handler e polling: salvar o status em state
// setStatusFinal(row.status)

useEffect(() => {
  const podeAvançar = 
    !isLoading && 
    perguntas.length === 0 && 
    !fetchError && 
    historicoId &&
    (statusFinal === 'CONCLUIDO' || statusFinal === 'SUCESSO'); // ← só sem AGUARDA_RESPOSTAS
  if (podeAvançar) {
    const timer = setTimeout(() => onSubmit(...), 3000); // ← 3s para Realtime chegar
    return () => clearTimeout(timer);
  }
}, [isLoading, perguntas.length, statusFinal, ...]);

🐛 Bug P0-3 — Petição genérica sem fatos concretos

Problema

A petição gerada para Thaina × BB não usou os dados concretos dos documentos:

  • ❌ Cita "Documentos CONTR7, CONTR8, CONTR9" (nomes de arquivo internos)
  • ❌ Não menciona datas reais: 26/03/2019 e 01/02/2022
  • ❌ Não menciona autenticações eletrônicas: 4FA8F2A269D4A8AD, 1FA75DEA3AB6553F
  • ❌ Não cita valores específicos das 8 negativações: R$5.433,44 + R$634,57 + R$2.900,41...
  • ❌ Argumento errado: diz que contratos "não provam anuência" mas Thaina assinou eletronicamente via senha
  • ❌ Data em branco no rodapé: "Porto Alegre, _____ de _____"

Causa

Dupla causa: SmartBlocks não chega (P0-1) + analise_por_doc não injeta fatos específicos no prompt.

Fix no gera-peticao — enriquecer análise com dados concretos

// Em montarPromptAdaptacao() — enriquecer analise_por_doc
if (analise_provas?.analise_por_doc?.length) {
  partes.push("Documentos analisados pelo banco (use descrições jurídicas, NÃO nomes de arquivo):");
  analise_provas.analise_por_doc.forEach(doc => {
    // Sanitizar nome: remover UUIDs e números de processo
    const nomeJuridico = sanitizarNomeDoc(doc.nome);
    partes.push(`  - ${nomeJuridico}: ${doc.sugestao_impugnacao || ''}`);
  });
}

// Função sanitizarNomeDoc
function sanitizarNomeDoc(nome) {
  return nome
    .replace(/^52396427\d+_\d+_/, '') // remove número do processo
    .replace(/---[a-f0-9-]{36}$/, '')  // remove UUID
    .replace(/\.pdf$/i, '')
    .replace(/CONTR(\d+)/, 'Contrato de Adesão')
    .replace(/CONT(\d+)/, 'Contrato')
    .replace(/PROC(\d+)/, 'Petição Inicial')
    .replace(/ANEXO(\d+)/, 'Anexo $1')
    .replace(/RG(\d+)/, 'Documento de Identidade')
    .trim() || nome;
}

Fix — data automática

// No prompt de dados_processo
const hoje = new Date().toLocaleDateString('pt-BR', {day:'numeric', month:'long', year:'numeric'});
partes.push(`Data da petição: ${hoje}`);
