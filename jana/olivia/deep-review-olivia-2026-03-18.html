Pontos fortes do design

  • Hierarquia tags/blocos bem documentada e aplicada nos 3 modos de forma consistente
  • Perguntas dinâmicas JSON no Express — decisão excelente: IA gera perguntas baseadas no caso real, não genéricas
  • Drawer lateral upload (>50% tela) em vez de modal — UX muito melhor para selecionar documentos do EPROC + upload manual
  • Botão "Atualizar JARBAS" no drawer — integração rica: JARBAS baixa docs do processo automaticamente
  • Validação ao avançar com modal de preenchimento de campos faltantes — evita o problema clássico de processos antigos sem dados
  • Anti-fingerprint já implementado no skill_olivia_agente_v2.mjs com seed baseado em hash CNJ — 4 camadas de variação de estilo jurídico

Pontos que merecem atenção

⚠️ Sistema de abas multi-petição — estado não especificado

D-FE-07 define o conceito mas não define onde persiste o estado. Sugiro zustand slice com array de abas, cada aba com: {id, modo, step, processoId, modeloId, versaoId, stepPayloads[], status}. Dirty-state guard ao fechar.

⚠️ Chat OlivIA — "consulta mas não altera" precisa de spec de UX

A decisão de deixar o chat só para consulta é correta (evita UX confusa). Mas precisa especificar: como o usuário solicita alterações na petição? Sugestão: campo input dedicado abaixo do editor com botão "Aplicar sugestão" + diff visual aceitar/rejeitar.

⚠️ Modo Agente Step 3 — editor de rich text não especificado

O plano diz "Rich text editor" mas não define qual lib: TipTap, Quill, Plate, ou mesmo um textarea com markdown. Recomendo TipTap com extensões mínimas (bold, italic, heading) — leve, extensível, React-friendly.
