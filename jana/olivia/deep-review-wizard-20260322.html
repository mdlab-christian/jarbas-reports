Step 5 — Conclusão
❌ Gaps críticos Download, Google Doc, Conclusão
❓ O download .docx/.pdf está funcionando?
❌ NÃO funciona. Bucket olivia-peticoes é privado — a URL pública retorna 404. O usuário clica em Download e nada acontece.

Soluções possíveis:
  1. Tornar o bucket público (risco de segurança — qualquer um com a URL baixa)
  2. Gerar signed URL via Edge Function com expiração de 15min (recomendado)
  3. Passar o download pelo servidor OlivIA (proxy)

A skill olivia-cria-doc.mjs tem função signUrl() já implementada — usar essa abordagem.
❓ Onde está o Google Doc? PRD especifica isso como CTA principal.
❌ Google Doc não é gerado. O pipeline atual (OliviaMDBot) não cria Google Doc — decisão arquitetural da sessão anterior substituiu N8n (que criava GDocs) pelo OliviaMDBot (que só gera DOCX+PDF). A rota /olivia/criar-doc do servidor OlivIA deveria ser chamada pelo JARBAS para criar o GDoc, mas não está integrada ao pipeline.
❓ O score "0.0" no Step 5 está correto?
❌ Bug de campo. O revisor salva score_revisor no banco como 0 porque o servidor passa o campo errado. A skill olivia-revisor.mjs espera peticao_express mas o servidor passa peticao_html. Com campo errado, o Claude recebe contexto vazio → score 0.
❓ Tem como melhorar UI/UX?
  • 🎯 Animação de celebração ao chegar na conclusão (confetti ou shimmer no card)
  • 🎯 Mostrar resumo: tipo petição, processo, tempo gerado, tokens
  • 🎯 Avaliação 1-5 estrelas clicável com campo comentário
  • 🎯 Botão "Ver no histórico" — navega para /olivia com aba fila mostrando esse registro
  • 🎯 Compartilhar link (signed URL com 24h) para outro advogado revisar
