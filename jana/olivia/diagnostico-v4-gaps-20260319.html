🔍 Causa Raiz — Por Que o CC Não Cumpriu Tudo

1

categoriaEmpresaId nunca é propagada — o maior gap funcional

O ProcessoSelector retorna o processo mas não busca categoria_empresa_id. O usePetitionModels recebe categoriaEmpresaId como param mas todos os callers passam null. Resultado: todos os 177 modelos aparecem sem filtro de empresa — a lista é inútil na prática.

→ Fix: ProcessoSelector deve fazer SELECT processos JOIN empresas e retornar categoria_empresa_id. Passar para usePetitionModels nos Steps 1 de todos os modos.

2

Prompts do CC não incluíam decisões de spec D-FE-xx

O prompt dos terminais focou em "criar estrutura e componentes" mas não listou as decisões de frontend do PLANO-MESTRE como restrições. Ex: D-FE-12 (sem seletor advogado), D-FE-13 (sem seletor qualidade no Express), D-FE-09 (sem playbook). CC implementou o que faz "sentido óbvio" em vez do que foi decidido.

→ Fix no próximo projeto: seção DECISÕES FIRMES no topo de cada prompt com lista numerada de o que NÃO fazer.

3

Landing incompleta — KPIs e histórico ausentes

O PRD seção 4.2 é explícito: Landing tem KPI row (fn_olivia_kpis), PeticaoTabBar (rascunhos em andamento), histórico últimas 5. O CC criou só os 3 cards de modo — parte mais simples. Partes que requerem queries ao banco foram ignoradas.

→ Fix: Implementar fn_olivia_kpis + histórico + rascunhos na OliviaGeradorLanding.

4

hook useSmartBlocksGeneration não foi criado

O componente OliviaGeradorSmartBlocks.tsx importava o hook mas ele não existia. CC provavelmente assumiu que o arquivo já estava lá (de sessão anterior) ou esqueceu de criar. Derrubou o build inteiro.

→ Corrigido por JANA. Já commitado.

5

Chat e Fila são placeholders

OliviaChatPage e OliviaFilaTab existem como componentes mínimos. O Chat é funcional na v3 (OliviaChatPage com threads), mas na v4 não foi reconstruído com o novo design. A Fila existe como aba separada quando deveria ser integrada ao Gerador.

→ Fix: Reconstruir Chat com modal início + timeline + composer. Mover Fila para dentro da aba Gerador.

### CORREÇÃO 1 — P0: categoriaEmpresaId no ProcessoSelector (GAP CRÍTICO)
**Problema:** O seletor de modelo lista TODOS os 177 modelos sem filtrar por empresa do processo.
**Causa:** ProcessoSelector não retorna categoria_empresa_id. usePetitionModels recebe null.

**Fix no ProcessoSelector (src/pages/olivia/components/shared/ProcessoSelector.tsx):**
Alterar a interface ProcessoSelecionado para incluir:
  categoria_empresa_id?: string | null;

Alterar SearchBoxCNJ onChange para fazer lookup:
Quando processo é selecionado, fazer query:
  SELECT processos.id, processos.numero_cnj, clientes.nome as cliente_nome,
         empresas.nome as empresa_nome, empresas.categoria_id as categoria_empresa_id,
         processos.advogado_id
  FROM processos
  LEFT JOIN clientes ON processos.cliente_id = clientes.id
  LEFT JOIN empresas ON processos.empresa_id = empresas.id
  WHERE processos.id = [processoId]

Retornar categoria_empresa_id no objeto ProcessoSelecionado.

**Fix nos Steps 1 de cada modo:**
