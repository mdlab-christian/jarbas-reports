🔍 Diagnóstico: Por que o Frontend Atual Não Funciona?

O frontend atual é um remendar sobre remendar — foi construído para N8n, migrado para OlivIA, corrigido em cima, e ficou com lógica partida. O problema não é o backend: o servidor OlivIA v2.0 está rodando, as 9 skills existem, o banco está certo. O problema é o frontend.

4. Fluxo V2 — Correto e Simplificado

1 Step 1 — Identificação

Seleciona processo + modelo. Clica "Continuar" →
  ① INSERT olivia_historico (status=PENDENTE)
  ② POST /olivia/gerar com fase=1 (dispara análise em background)
  ③ Navega imediatamente para Step 2

2 Step 2 — Análise & Perguntas

Exibe OliviaProgressCard com substeps animados
Supabase Realtime escuta olivia_historico.id=historicoId:
  → quando perguntas_geradas chega → exibe perguntas com animação
  → quando status=AGUARDA_RESPOSTAS → habilita botão "Avançar"
Advogado responde. Clica "Avançar" → vai para Step 3.

3 Step 3 — Estrutura

Outline DnD com instruções por tópico. Clica "Gerar Petição" →
  POST /olivia/gerar com fase=2 + respostas + outline
  Servidor processa: smartblocks → gera-peticao → revisor → cria-doc
  Navega para Step 4

4 Step 4 — Geração (Loading → Rascunho)

Loading animado com progresso real (Realtime). Quando CONCLUIDO:
  → Exibe rascunho editável (Textarea)
  → Score do revisor
  → Botão "Aprovar" → vai para Step 5

5 Step 5 — Conclusão

Animação de sucesso (Framer Motion). Botões: Download DOCX, Download PDF.
Score de qualidade + tempo de geração. Input feedback 1-10.
"Gerar Nova Petição" → reset para Step 1.

7. Hooks V2 — Design Correto

useOliviaWizard.ts — Hook Principal (Substituir useExpressGeneration)

// Problema do hook atual: estado atualizado após re-render, timing issues
// Solução: usar ref para o historicoId + callback pattern

export function useOliviaWizard() {
  const historicoIdRef = useRef<string | null>(null)  // ← ref, não state!
  const [state, setState] = useState<OliviaWizardState>(INITIAL)

  // Step 1 → 2: cria historico + dispara Fase 1
  const iniciarAnalise = useCallback(async () => {
    // 1. INSERT historico → pega ID imediatamente
    const { data } = await supabase.from('olivia_historico').insert({...}).select('id').single()
    const hId = data.id
    historicoIdRef.current = hId                    // ref atualiza IMEDIATAMENTE
    setState(s => ({ ...s, historicoId: hId, step: 2 }))  // state é async

    // 2. POST /olivia/gerar fase=1 (non-blocking)
    callWebhook(OLIVIA_WEBHOOKS.GERAR, { historico_id: hId, fase: 1, ... })
      .catch(err => setState(s => ({ ...s, status: 'ERRO', errorMessage: err.message })))

    // 3. Iniciar Realtime com o ID da ref (disponível imediatamente)
    startRealtime(hId)
  }, [...deps])

  // Realtime usa a ref para evitar stale closure
  const startRealtime = useCallback((hId: string) => {
    const channel = supabase
      .channel(`olivia-${hId}`)
      .on('postgres_changes', { filter: `id=eq.${hId}` }, (payload) => {
        const row = payload.new
        // Atualiza estado baseado no que chegou
        if (row.perguntas_geradas?.length) {
          setState(s => ({ ...s, perguntas: row.perguntas_geradas, status: 'AGUARDA_RESPOSTAS' }))
        }
        if (row.status === 'CONCLUIDO') {
          setState(s => ({ ...s, status: 'CONCLUIDO', textoGerado: row.texto_gerado, step: 5 }))
        }
        if (row.status === 'ERRO') {
          setState(s => ({ ...s, status: 'ERRO', errorMessage: row.error_message }))
        }
        // Progresso
        if (row.progresso_pct) setState(s => ({ ...s, progresso: row.progresso_pct }))
        if (row.progresso_msg) setState(s => ({ ...s, progressoMsg: row.progresso_msg }))
      })
      .subscribe()
    realtimeRef.current = channel
  }, [])

8. Fix do Problema de Timing (Causa Raiz Principal)

❌ Problema Atual

// OliviaGeradorExpress.tsx
const handleStep1NextV3 = useCallback(async () => {
  updateState({ processoId, modeloId, ... })  // 1. Atualiza state (async!)
  setStepIndex(1)                              // 2. Navega

// useExpressGeneration.ts — dispararAnalise lê state:
const dispararAnalise = useCallback(async () => {
  // state.processoId PODE ainda ser null aqui se o closure está stale!
  if (!state.processoId || !state.modeloId) {
    toast.error('Dados incompletos')  // ← ESSE é o erro silencioso
    return null
  }
}, [state.processoId, state.modeloId, ...])

✅ Solução V2

// Passar dados DIRETAMENTE, não ler do state
const handleStep1 = useCallback(async (dados: Step1Data) => {
  // dados vem do componente, não do state — sem timing issue
  await iniciarAnalise(dados)  // hook recebe dados explicitamente
}, [iniciarAnalise])

// Hook recebe parâmetros, não lê state
const iniciarAnalise = useCallback(async (dados: {
  processoId: string, modeloId: string, versaoId: string, ...
}) => {
  const { data } = await supabase.from('olivia_historico').insert({
    processo_id: dados.processoId,  // ← direto do parâmetro
    modelo_id: dados.modeloId,
  }).select('id').single()
  
  const hId = data.id
  historicoIdRef.current = hId    // ref: disponível imediatamente
  startRealtime(hId)              // inicia escuta
  
  callWebhook(OLIVIA_WEBHOOKS.GERAR, {
    historico_id: hId,
    processo_id: dados.processoId,  // ← direto
    modelo_id: dados.modeloId,      // ← direto
    fase: 1,
  })
}, [])
