🔴 Bugs Críticos — Plano de Correção

BUG #1 — vite build falha: ExpressStep5Conclusao.tsx JSX malformado

O bloco {viewMode === 'conclusao' && ( que abre na linha 200 fecha com )} dentro do return mas os elementos filhos (Google Doc, downloads, metrics, feedback, etc.) estão FORA da condição — como irmãos no JSX pai. O esbuild rejeita a sintaxe.

Fix: Envolver todo o conteúdo do bloco conclusão (linhas 200–337) em um único fragmento <>...</> dentro do {viewMode === 'conclusao' && ( ... )}, garantindo que todos os filhos estejam dentro da condição.

BUG #2 — callWebhook() continua usando VITE_N8N_WEBHOOK_URL (não OlivIA)

O olivia-url.ts foi criado corretamente mas o callWebhook() em src/lib/webhook.ts ainda aponta para import.meta.env.VITE_N8N_WEBHOOK_URL. As chamadas EXPRESS_GERAR e EXPRESS_TRIAGEM chegam no Railway n8n, não no OliviaMDBot.

Fix: Adicionar VITE_OLIVIA_WEBHOOK_URL=https://olivia.jarbas-mdlab.com ao .env.local e atualizar o callWebhook() para detectar paths olivia/* e usar a URL OlivIA (ou criar callOliviaWebhook() separado).

BUG #3 — useExportarDocumento usa jarbas.jarbas-mdlab.com hardcoded

O hook de exportação DOCX/PDF usa const JARBAS_URL = 'https://jarbas.jarbas-mdlab.com' hardcoded e chama /olivia/cria-doc-local. Este endpoint é do JARBAS (Mac Mini), não do OliviaMDBot — mas este é um caso legítimo porque o JARBAS gera os arquivos. O problema é a ausência do header de auth correto e URL hardcoded sem fallback env.

Fix: Substituir JARBAS_URL por JARBAS_BASE_URL importado de @/lib/jarbas-url (já existe). Manter endpoint /olivia/cria-doc-local no JARBAS (correto por decisão arquitetural).

BUG #4 — Chat drawer no Step 4 é visual-only (botão Send sem handler)

O drawer abre/fecha mas o Textarea não tem estado controlado e o botão Send não tem onClick. O chat não envia nada.

Fix: Adicionar const [chatInput, setChatInput] = useState("") + handler handleSendChat que chama callWebhook("olivia/chat", ...) com o historicoId e o texto. Resposta adicionada a uma lista local de mensagens.
