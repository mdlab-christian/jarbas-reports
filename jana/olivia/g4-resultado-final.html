4. Proximos passos

P0 — CRITICO: Gateway nao tem API routing

O OpenClaw gateway v2026.3.13 so serve /health + SPA static. Todos os POST para /olivia/* retornam 404.

Opcoes:

  • A) Implementar webhook middleware no OpenClaw gateway (requer feature request ao OpenClaw)
  • B) Criar um servidor Express/Fastify custom na porta 18794 do user olivia, com proxy no Cloudflare tunnel
  • C) Rotear via n8n (PRD diz "OlivIA nao usa N8n" — conflito arquitetural)
  • D) Usar o Telegram bot da OlivIA como canal de comunicacao (via API Telegram)

Recomendacao: Opcao B — servidor Express leve que recebe POST, valida token, e enfileira tarefas para o OpenClaw agent via filesystem/supabase.

P1 — Frontend precisa migrar auth

Hoje: header mdflow: MdL1501@ (via webhook.ts)

PRD exige: Authorization: Bearer olivia-gateway-token-2026

Arquivos a alterar:

  • src/lib/olivia-url.ts — trocar OLIVIA_AUTH por OLIVIA_TOKEN
  • src/lib/webhook.ts:33 — usar header Authorization ao inves de mdflow

So faz sentido migrar DEPOIS que o gateway aceitar POST (P0 resolvido)
