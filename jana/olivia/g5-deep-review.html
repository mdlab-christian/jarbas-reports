BUG-G5-03 · P0 · TEMPLATE MISMATCH
Placeholders do template-base.docx não batem com os dados enviados pela skill
🔴 P0
template-base.docx espera: {cabecalho} {titulo} {corpo} {conclusao}
olivia-cria-doc.mjs envia: {tribunal} {numero_processo} {autor} {reu} {secoes}
Resultado: docxtemplater gera DOCX com placeholders literais sem substituição.

✅ Correção (2 opções)

Opção A (recomendada): Atualizar gerarDocx() em olivia-cria-doc.mjs para mapear dados → {cabecalho}/{titulo}/{corpo}/{conclusao}:
cabecalho = escritório | título = tipo_ação | corpo = secoes concatenadas | conclusao = cidade + data + advogado

Opção B: Recriar template-base.docx com {tribunal}, {numero_processo}, {secoes} etc.

BUG-G5-04 · P1 · URL PÚBLICA EM BUCKET PRIVADO
Frontend recebe URL pública mas bucket olivia-peticoes é privado → HTTP 400
🟡 P1
Bucket criado com public: false. Skill gera docx_url como URL pública /storage/v1/object/public/.
Bridge repassa apenas docx_url (público) para o frontend. Resultado: HTTP 400 ao tentar baixar.
A skill gera corretamente docx_signed_url mas bridge ignora esse campo.

✅ Correção

Atualizar bridge para repassar docx_signed_url e pdf_signed_url em vez de docx_url/pdf_url.

BUG-G5-05 · P1 · PATH DE STORAGE DIVERGENTE
Upload usa orgId/baseName — spec pede historico_id/peticao.docx
🟡 P1
PRD especifica: olivia-peticoes/[historico_id]/peticao.docx
Implementação atual: olivia-peticoes/{orgId}/peticao-{numero}-{ts}.docx
Impacto: UPDATE de historico_id salva URL errada e impossível localizar arquivo por historico_id.

✅ Correção

Atualizar docxStoragePath em olivia-cria-doc.mjs para {historico_id}/peticao.docx quando historico_id estiver disponível.
