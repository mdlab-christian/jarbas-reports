🔍 G6 OlivIA — Diagnóstico: Petição não usa modelo

2026-03-22 · Investigação: por que a petição gerada ignora o conteudo_template do banco?

🚨 Root Cause: S5 e S6 nunca buscam o conteudo_template

O template real da petição (olivia_modelos_peticao.conteudo_template) tem 19.743 caracteres com toda a estrutura jurídica, placeholders {{contrato_cartao_credito}}, {{provas_unilaterais}}, jurisprudências fixas, etc. Nenhuma das skills S5 ou S6 consulta essa tabela. O Claude recebe apenas os blocos de SmartBlocks concatenados como texto livre e gera a petição do zero — sem estrutura, sem jurisprudências reais do template, sem formatação oficial.

Fluxo do pipeline — o que cada skill faz com o modelo

S4 olivia-gera-perguntas ✅ OK
Busca as tags do modelo via olivia_modelos_grupo.active_version_id → olivia_modelo_tags → olivia_tags e retorna a lista de perguntas para o advogado. Correto — 14 tags para o modelo Contrarrazões IF CC.
Faz: GET olivia_modelo_tags WHERE modelo_id = active_version_id
Não faz: não precisa do template aqui — ok
S5 olivia-smartblocks ❌ NÃO USA TEMPLATE
Busca blocos de texto por tag, monta INTRODUÇÃO + FUNDAMENTAÇÃO + CONCLUSÃO como texto livre. Nunca consulta olivia_modelos_peticao nem conteudo_template. O resultado é um texto concatenado de blocos, sem a estrutura oficial da petição.
DEVERIA FAZER:
1. GET olivia_modelos_peticao WHERE id = versao_id
2. Pegar conteudo_template (19.743 chars com {{placeholders}})
3. Para cada placeholder {{codigo_tag}}: buscar bloco correspondente
4. Substituir {{codigo_tag}} → conteúdo do bloco
5. Retornar template preenchido como peticao_base_texto

FAZ: concatena blocos como texto livre → Claude não tem estrutura base
S6 olivia-gera-peticao ❌ GERA DO ZERO
Recebe o texto livre do S5 como "âncora" e pede ao Claude para "gerar a petição adaptada ao caso". Sem estrutura base, o Claude cria a petição do zero com sua própria formatação — resultado diverge completamente do template oficial.
DEVERIA FAZER:
1. Receber o template JÁ PREENCHIDO do S5
2. Pedir ao Claude apenas para: adaptar ao caso concreto (número, datas, valores), preencher variáveis dinâmicas, personalizar com análise das provas

FAZ: Claude gera petição nova do zero → estrutura, formatação e conteúdo totalmente diferentes do modelo oficial
S7 olivia-revisor ✅ Revisor correto
Revisa a petição gerada pelo S6. Dá score alto porque o texto jurídico é bom — mas não é o texto do modelo oficial. O revisor não valida se a estrutura corresponde ao template.
