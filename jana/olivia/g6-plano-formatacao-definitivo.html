2. Erros encontrados — detalhamento técnico

Erro 1 — Instruções internas dos blocos aparecem na petição

Os blocos de texto têm instruções para o advogado embutidas no meio do conteúdo como <span>TEXTO EM CAPS</span>. A função de sanitização atual só detecta padrões como "SE FOR", "INSERIR" etc., mas não cobre todos:

Padrões de instrução encontrados nos blocos (> 150 chars): 5x | JUNTAR O PRINT DA FATURA 2x | COLOCAR O PRINT DA DATA DO CADASTRO... 2x | JUNTAR O PRINT DA DATA COM A FATURA COM A DATA DA RESTRICAO 2x | SE FOR O CASO DE JUNTAREM SO DOCUMENTO OU SO SELFIE, ALTERAR... 2x | [DATA DA RESTRIÇÃO] ← variáveis com colchetes 2x | [VALOR DA RESTRIÇÃO] 2x | [DATA DA FATURA] 2x | [VALOR DA FATURA] ...

Fix: ampliar regex de sanitização para cobrir padrões JUNTAR/COLOCAR/ALTERAR/[TEXTO] e qualquer span cujo texto seja >80% maiúsculas E tenha mais de 10 chars e seja texto plano (sem tags).

Erro 2 — Subtítulo do bloco + subtítulo do Claude = duplicado

Os blocos GDocs têm o subtítulo como primeiro parágrafo (ex: "DA AUSÊNCIA DE VALOR JURÍDICO..."). O S6 pede ao Claude para adaptar e o Claude também gera o subtítulo da seção. Resultado: aparece duas vezes.

Fix: o S6 deve informar ao Claude que os subtítulos já estão nos blocos — o Claude deve apenas adaptar o corpo, não adicionar novos títulos de seção.

Erro 3 — Blocos sem mammoth (HTML entities, sem negrito)

O importer olivia-gdoc-importer.mjs ainda não rodou para os 1.244 blocos. Resultado: blocos usam conteudo (HTML entities &ccedil;) em vez de conteudo_html_rico com <strong>/<em>.

Fix: rodar o importer. Estimativa: ~45 minutos para 1.244 blocos com rate limit 120ms/req.

Erro 4 — Recuo e espaçamento diferentes do GDocs original

O PDF gerado pelo puppeteer não reproduz o firstLineIndent e spaceBefore do GDocs. O mammoth converte para HTML mas não transfere estilos CSS de parágrafo.

Fix via GAS (ver abaixo): exportar o DOCX real do GDocs e usar como template base — toda a formatação fica preservada no formato Word nativo.

3. Plano definitivo — Abordagem GAS (Google Apps Script)

A abordagem mais fiel ao GDocs original usa o GAS para copiar a formatação Word diretamente, substituindo os placeholders no próprio DOCX — sem converter para HTML. Isso preserva 100% da formatação: recuos, espaçamentos, estilos de parágrafo, negrito, itálico, notas de rodapé etc.

Por que GAS é melhor que só mammoth?

❌ Abordagem atual (HTML + puppeteer)

• mammoth converte DOCX → HTML (perde recuos, spaceBefore)

• puppeteer converte HTML → PDF (fonte OK, sem estilos Word)

• Blocos misturados com texto do Claude (estrutura inconsistente)

• Instruções internas precisam de sanitização manual

✅ Abordagem GAS + docxtemplater

• DOCX original → mantém 100% da formatação Word

• Placeholders substituídos no próprio XML Word

• Blocos inseridos como formatted content (runs Word)

• PDF gerado a partir do DOCX (via Google Drive export ou puppeteer com CSS fiel)

5. Script GAS — Como implementar

O GAS que precisamos fazer (função que já temos OAuth funcionando com drive.readonly):

// Google Apps Script — olivia-docx-exporter.gs // Roda no GAS da conta perinottipazadvogados.com function exportarTodosParaStorage() { // 1. Buscar todos os google_doc_id do banco via supabase REST const modelos = buscarDoSupabase('/olivia_modelos_peticao?select=id,google_doc_id&google_doc_id=not.is.null'); const blocos = buscarDoSupabase('/olivia_blocos_texto?select=id,google_doc_id&google_doc_id=not.is.null&status=eq.published'); // 2. Para cada um: exportar como DOCX e salvar no bucket for (const m of modelos) { const docxBlob = DriveApp.getFileById(m.google_doc_id).getBlob(); // Upload para olivia-modelos/{id}.docx via Supabase Storage API uploadParaStorage('olivia-modelos', m.id + '.docx', docxBlob); } // 3. Mesma coisa para blocos → olivia-blocos/{id}.docx } // Vantagem: DriveApp.getFileById() no GAS acessa qualquer arquivo do Drive // sem precisar de OAuth externo — tem acesso nativo via conta do usuário

Credenciais: OAuth atual (GOOGLE_CLIENT_ID_MDFLOW + GOOGLE_CLIENT_SECRET_MDFLOW + GOOGLE_REFRESH_TOKEN) funciona com escopo drive — já testado e exportando o template principal (325KB) com sucesso.

Blocos .txt vs .docs: Os blocos estão como arquivos Google Docs (importados de .txt) — o export como DOCX via Drive API funciona normalmente para ambos os tipos.
