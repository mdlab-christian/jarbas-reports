🤔 Opção A vs Opção B

Opção A — Formatação dentro do S6 (Claude)

Pedir ao Claude para formatar junto com a geração: marcadores JURIS, títulos, recuos, etc.

Prós
  • Pipeline mais simples (sem etapa extra)
  • Claude vê o contexto inteiro ao formatar
Contras
  • Prompt já tem 46KB — mais instruções = qualidade jurídica pior
  • Claude às vezes "improvisa" a formatação e sai do padrão
  • Não dá pra aplicar retroativamente em petições já geradas
  • Difícil testar formatação isolada do conteúdo

⭐ Opção B — Skill S9 Formatadora Separada RECOMENDADA

Skill Node.js determinística que transforma o HTML gerado pelo Claude aplicando todas as regras de formatação ABNT/escritório.

Prós
  • Claude foca 100% em qualidade jurídica
  • Formatação determinística — sempre igual, testável
  • Sem LLM = sem custo adicional, latência <1s
  • Revisor (S7) vê o HTML já formatado → score mais justo
  • Aplica em petições antigas sem regregar
  • Fácil de ajustar sem mexer no prompt do Claude
Contras
  • Mais uma etapa no pipeline
  • Precisa de regras bem definidas pra cada caso

⚙️ O que a S9 Formatadora faz

📜 Jurisprudências

Detecta <blockquote> → aplica 10pt, recuo 4cm esq + 2cm dir, itálico, metadados alinhados à direita

🔤 Títulos de seção

Parágrafos com >65% CAPS que iniciam com "DA ", "DO ", "DAS " → <strong> + espaçamento ABNT

¶ Recuo de parágrafo

Primeiro parágrafo após título sem recuo, demais com 1.25cm text-indent via CSS

🗑️ Instruções internas

Remove qualquer parágrafo residual com JUNTAR/COLOCAR/INSERIR/LEMBRAR em CAPS

📐 Espaçamento

Normaliza margin-top/bottom entre seções: 12pt antes de título, 6pt entre parágrafos

🔗 Entidades HTML

Decodifica &ccedil;, &atilde;, etc. — blocos sem mammoth têm entities cruas

✂️ Subtítulos duplicados

Se bloco começa com mesmo texto do título da seção → remove o duplicado do bloco

📏 Fonte consistente

Garante Times New Roman 12pt em todo o corpo; jurisprudência 10pt; metadados 9pt
