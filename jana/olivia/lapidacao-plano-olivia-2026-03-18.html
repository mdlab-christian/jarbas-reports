⚙️ N8N — Análise de Viabilidade do Fix

Viabilidade do fix $helpers.httpRequest()

✅ Fix é viável — método confirmado

$helpers.httpRequest() é o método oficial do n8n para chamadas HTTP dentro de Code nodes no Task Runner (v1.21+). Documentação confirma que substitui fetch() 1:1 para chamadas simples.

Padrão de substituição:


// ANTES (quebrado)
const resp = await fetch(url, { method: 'POST', headers: {...}, body: JSON.stringify(data) });
const result = await resp.json();

// DEPOIS (fix)
const result = await $helpers.httpRequest({
  method: 'POST',
  url: url,
  headers: {...},
  body: data,  // já objeto, não string
  json: true   // auto-parse JSON
});

OLIVIA-ANALISE-V40 nodesZeroLimpo. Não tocar.

💡 Consideração para o CC (E05)

Para os nodes que chamam a Anthropic API (B4-ChamarRedator, A3b-GerarIA): em vez de substituir por $helpers.httpRequest(), considerar criar um HTTP Request node nativo do n8n — mais fácil de debugar, mais performático, sem código JavaScript. O Code node pode apenas preparar o payload e passar para o HTTP Request node via $json.

Documentar isso no PRD do E05 para que o CC saiba que tem essa opção.
