🎯 Visão Geral — O Problema que Estamos Resolvendo
🚨
O problema central: Projetos complexos como a Olivia têm horas de planejamento, dezenas de decisões, centenas de campos. Quando se vai executar, o Claude Code começa com contexto parcial → alucina → inventa estrutura → gera retrabalho. Cada sessão perdida = horas de replanejamento.
❌ Fluxo Anterior (frágil)
  • 📱 Horas de áudios com Christian
  • 🧠 JANA processa e executa na mesma sessão
  • 🔄 Quebra de contexto → replanejamento
  • 🤖 CC começa tarefa sem contexto completo
  • 💥 Alucinação → retrabalho → mais horas perdidas
  • 📉 Conhecimento acumulado se perde ao encerrar sessão
✅ Fluxo Novo (robusto)
  • 📱 Horas de áudios com Christian
  • 📄 JANA consolida tudo em HTML estruturado (Plano Mestre)
  • 🤖 CC analisa HTML + áudios → sugere complementos
  • ✅ Christian valida → HTML vira "Super Plano Mestre"
  • 📋 JANA gera tarefas com PRD completo cada uma
  • 🔁 Execução em loop → cada CC sabe exatamente o que fazer
  • 🗄️ Tudo persistido no banco → imune a quebra de contexto
💡 A Insight Central

O HTML não é documentação pela documentação. É o único artefato de contexto que sobrevive à quebra de sessão, pode ser enviado integralmente ao CC, e serve como "memória externalizada" do projeto. Quanto mais rico o HTML, mais o CC acerta de primeira — menos iterações, menos custo, menos retrabalho. A tabela de tarefas com PRD por tarefa é a segunda camada de proteção: o CC executa UMA tarefa por vez, com contexto COMPLETO dessa tarefa, sem precisar carregar o projeto inteiro.

FUTURO

Loop até todas as tarefas testado=true → Notifica Christian
100% das tarefas concluídas e testadas = projeto encerrado. Relatório final publicado no GitHub.
🛡️ Anti-Alucinação Sistêmica — Como o Sistema Garante Qualidade
🧱 Camadas de Proteção
Camada 1 HTML Plano Mestre — todo o conhecimento estruturado. CC nunca parte do zero.
Camada 2 Revisão CC + Christian — sugestões do CC são validadas por quem conhece o projeto.
Camada 3 PRD por tarefa — CC só precisa do contexto da tarefa atual. Contexto cirúrgico.
Camada 4 Banco de tarefas — imune a quebra de sessão. Estado persiste entre sessões JANA.
Camada 5 Validação com evidência — testado=true só com screenshot/e2e real.
Camada 6 2 falhas = PARA — nunca continua em loop de erro silencioso.
💡 Por que PRD por Tarefa Agrega Muito
✅ Sim, vale muito. O PRD em markdown por tarefa resolve o principal problema de projetos longos.
  • 🎯 CC recebe contexto completo de 1 tarefa — sem ruído das outras 80
  • 📋 PRD define: o quê, por quê, como, arquivos alvo, regras, o que NÃO fazer
  • 🔒 Critério de aceite no PRD = CC sabe quando parar
  • 🗄️ PRD no banco = nunca se perde, pode ser regenerado
  • 🔄 JANA pode regenerar PRD se o plano mestre for atualizado
  • 📊 Auditable — Christian pode ler qualquer PRD e ver se está correto

- Usar ProcessoWithRelations tipado — zero "as any"

- [ ] Campos corretos para Empresa vs Órgão
- [ ] Zero TypeScript errors

- Não criar nova lógica de fetch — usar hooks existentes
- Não alterar olivia_historico — escopo desta tarefa é apenas frontend

💬 Resposta JANA — O que Acho dessa Estratégia
🦊 Sim, funciona. E é exatamente a arquitetura certa para esse tipo de projeto.

O que você está propondo é, essencialmente, criar um sistema de memória externalizada e versionada para projetos de longa duração. Não é algo que fui arquitetada para fazer nativamente, mas é a solução mais sólida que existe dado o constraints do ambiente (quebra de contexto, sessões independentes, CC sem memória de longo prazo).

O fluxo que você descreveu resolve os três principais vetores de falha: (1) perda de contexto → resolvido pelo HTML e banco; (2) alucinação do CC → resolvido pelo PRD por tarefa e regras anti-alucinação; (3) execução antes de plano sólido → resolvido pela sequência confirmada (tarefas só depois do Super Plano Mestre validado).

O único ponto que eu acrescentaria: quando as tarefas forem geradas no passo F, cada PRD deve referenciar explicitamente a seção do HTML que o fundamenta. Tipo: "ver plano-mestre seção #wireframes-express step 3". Isso garante rastreabilidade total — se alguém questionar uma tarefa, tem de onde veio.
