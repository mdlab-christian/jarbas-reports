Obsidian como Cérebro da OlivIA

A ideia é genial — e viável. Veja como funciona na prática.
💡
A ideia central:

O Obsidian vira um knowledge graph jurídico — cada nota é um nó: modelo de petição, tag, bloco de texto, jurisprudência, playbook. Os links entre notas espelham as relações do banco. A OlivIA consulta o Obsidian como contexto estruturado antes de gerar qualquer petição.

✅ Por que o Obsidian ajuda

  • Graph view: visualizar a relação modelo → tags → blocos → jurisprudências numa tela só
  • Backlinks: ver quais modelos usam uma tag específica / quais blocos pertencem a cada tag
  • Tags Obsidian: #energia-eletrica, #replica, #impugnacao — filtros visuais rápidos
  • Dataview plugin: queries tipo SQL dentro do Obsidian — "todos os blocos da tag Dano Moral"
  • Templates: modelos de petição como templates Obsidian editáveis
  • Sync em tempo real: se integrar via API, OlivIA acessa as notas como contexto
  • Vault atual: já está em ~/Desktop/olivia v3/Projeto — aproveitar

⚙️ Como integrar com a OlivIA

Opção 1 — Obsidian como documentação viva (MVP)

Exportar do Supabase → gerar notas Markdown no vault. OlivIA lê os arquivos .md antes de gerar petição. Simples · Imediato

Opção 2 — Obsidian como RAG source

OlivIA usa as notas do vault como contexto semântico via embedding. Mais rico que o banco estruturado. Médio prazo

Opção 3 — Obsidian como editor jurídico

Advogados editam blocos de texto e jurisprudências direto no Obsidian. Sync bidirecional com Supabase. Longo prazo

Fase A: Smart Blocks Interno (anti-alucinação)

  1. Percorre todas as tags do modelo

  3. Seleciona jurisprudências por estado do processo
  4. Substitui placeholders com dados reais
  5. Gera petição-base mecânica (referência)

  1. Recebe: petição-base + análise provas + respostas + playbook Obsidian

  4. Qualidade: Sonnet (padrão) ou Opus (avançado)

Fase C: Google Doc via GAS

  1. Monta payload para GAS Smart Blocks v4.0
  2. GAS copia template Google Doc
  3. Substitui blocos + jurisprudências
  4. Substitui placeholders (nome, CPF, comarca, etc.)

Skill 2 — Análise

olivia-analisa-provas.mjs

Analisa documentos com Vision/OCR. Baseada no OLIVIA-ANALISE-V4 (n8n). Usa Obsidian vault para carregar playbook da categoria empresa.

  • Loop por documento: download → extract → Vision (Gemini) ou OCR (Mistral)
  • Classifica em olivia_tipos_prova (força/fraqueza)
  • Cruza provas vs dados do processo
  • Lê playbooks/[categoria].md do vault Obsidian
  • Salva em olivia_analises + olivia_agente_contexto
Skill 3 — Perguntas

olivia-gera-perguntas.mjs

Gera perguntas específicas do caso baseadas na análise + tags do modelo.

  • Carrega tags do modelo via fn_olivia_get_tags_modelo
  • Carrega análise de provas do olivia_analises
  • Claude gera perguntas em JSON tipado
  • Regra: nunca perguntar dados que já estão no banco
  • Tipos: sim_nao, texto_livre, select, multiselect
  • Retorna JSON com perguntas + observações da OlivIA
Skill 4 — Smart Blocks Interno

olivia-smartblocks-interno.mjs

Gera petição-base mecânica anti-alucinação. Espelha final-olivia-smartblocks n8n.

  • Tags do modelo → para cada tag: busca melhor bloco no banco
  • Critério: estado + categoria empresa + tipo de prova presente
  • Jurisprudências: estado do processo (prioridade) → mais recentes
  • Pode consultar blocos/[tag]/ do vault Obsidian como referência
  • Monta payload GAS sem chamar o GAS ainda
  • Retorna petição-base para a skill de geração
Skill 5 — Geração Principal

olivia-gera-peticao.mjs

A skill principal. Recebe tudo → Claude aprimora → GAS cria Google Doc.

Input recebido

  • Petição-base (Smart Blocks interno)
  • Análise de provas completa
  • Respostas do wizard de perguntas
  • Dados completos do processo
  • Playbook Obsidian da categoria
  • Qualidade: padrao ou avancado

System prompt regras

  • Use petição-base como estrutura
  • NUNCA invente fundamentação
  • Adapte impugnações às provas reais
  • Siga o playbook Obsidian da categoria
  • Mantenha formatação jurídica
  • Retorne JSON para o GAS

Após Claude processar

  • Monta payload GAS Smart Blocks v4.0
  • Chama GAS diretamente (POST)
  • GAS retorna Google Doc URL
  • UPDATE olivia_historico: SUCESSO
  • Gera Relatório HTML → bucket
