Arquitetura — Modo Agente 2.0

Fluxo completo de ponta a ponta, sem n8n, 100% OlivIA MD Bot
💡
Decisão de arquitetura (nova):

O Modo Agente 2.0 é 100% OlivIA MD Bot via skills. Nenhuma dependência de n8n. O frontend chama o gateway da OlivIA (olivia.jarbas-mdlab.com) que executa skills em sequência. O n8n fica como backup para Modo Express apenas.

Antes de qualquer petição final, OlivIA SEMPRE executa Smart Blocks internamente para gerar petição-base como referência. A IA aprimora — não inventa do zero.

Geração da Petição CRIAR DO ZERO
OlivIA executa Smart Blocks interno + aprimora com IA + cria Google Doc

Fase A: Smart Blocks Interno (anti-alucinação)

  1. Carrega todas as tags do modelo selecionado
  2. Para cada tag (pergunta): busca o melhor bloco (resposta) do banco
  3. Critério de seleção: estado do processo + categoria empresa + respostas do usuário
  4. Seleciona jurisprudências por estado (prioridade) + assunto
  5. Substitui placeholders com dados reais do processo
  6. Gera petição-base mecânica (esqueleto completo)

Fase B: Aprimoramento IA

  1. Claude recebe: petição-base + análise provas + respostas wizard + contexto processo
  2. Regra: NUNCA inventa — segue estrutura da petição-base
  3. Pode reposicionar tags, refinar texto, adicionar argumentos específicos
  4. Adapta impugnações para as provas analisadas
  5. Mantém formatação jurídica do modelo

Fase C: Criação do Google Doc

  1. OlivIA chama GAS Smart Blocks v4.0 com payload completo
  2. GAS copia template Google Doc
  3. Substitui placeholders (blocos + jurisprudências)
  4. Substitui tags de dados (nome, CPF, comarca, etc.)
  5. Retorna URL do Google Doc criado
  6. UPDATE olivia_historico (google_doc_url, status: SUCESSO)

Arquitetura das skills:

Cada skill é um arquivo .mjs no workspace da OlivIA. A skill orquestradora coordena as demais em sequência. O frontend chama apenas a skill orquestradora via gateway olivia.jarbas-mdlab.com. As skills conversam entre si via parâmetros e banco Supabase.

⚡ CRIAR — Skill 1

olivia-orquestra

Skill orquestradora principal. Recebe payload do frontend e executa as skills em sequência.

Responsabilidades

  • Receber: historico_id, processo_id, modelo_id, docs_urls, respostas_wizard
  • Executar em sequência: analisa-provas → gera-perguntas → smartblocks-interno → gera-peticao
  • Propagar contexto entre skills via olivia_agente_contexto
  • Atualizar progresso no olivia_historico em cada etapa
  • Tratar erros: retry automático 2x por etapa
// Endpoint exposto: POST olivia.jarbas-mdlab.com/skills/olivia-orquestra Payload: { historico_id, processo_id, versao_id, docs_urls[], qualidade } Retorna: { status: 'em_andamento', poll_key: historico_id } // Frontend faz polling em olivia_historico.progresso
⚡ CRIAR — Skill 2

olivia-analisa-provas

Análise de documentos/provas com Vision + OCR. Baseada no workflow OLIVIA-ANALISE-V4 do n8n.

Lógica (inspirada no OLIVIA-ANALISE-V4)

  • Loop por documento: download → extract PDF/imagem → análise
  • PDFs de texto: pdftotext simples
  • PDFs de imagem/prints: Gemini Vision ou Mistral OCR
  • Classifica cada doc em olivia_tipos_prova (força/fraqueza)
  • Cruza com dados do processo: valor_restricao, contrato, orgao_restritivo
  • Identifica: contestação, SMS, extrato, contratos, etc.
  • Resultado: JSON com análise por documento + pontos de impugnação
  • Salva em olivia_analises + olivia_agente_contexto
⚡ CRIAR — Skill 3

olivia-gera-perguntas

Com base na análise de provas + tags do modelo, gera perguntas específicas para o caso.

Lógica

  • Carrega tags do modelo via fn_olivia_get_tags_modelo
  • Carrega análise de provas do olivia_analises
  • Carrega dados do processo (restrições, empresa, comarca)
  • Carrega playbook da categoria da empresa
  • Claude gera perguntas específicas em formato JSON tipado
  • Tipos: sim_nao, texto_livre, select, multiselect, date
  • Regra: nunca perguntar o que já está no banco (nome, CPF, valor, etc.)
  • Foco: validar pontos específicos das provas + intenção processual
⚡ CRIAR — Skill 4

olivia-smartblocks-interno

Executa o fluxo Smart Blocks mecanicamente para gerar petição-base. Anti-alucinação.

Lógica (espelha final-olivia-smartblocks n8n)

  • Carrega tags do modelo via fn_olivia_get_tags_modelo
  • Para cada tag: busca o melhor bloco de texto no banco
  • Critério: match com estado + categoria empresa + tipo de prova presente
  • Se múltiplos blocos: seleciona por similaridade com análise de provas
  • Seleciona jurisprudências: estado do processo (prioridade) → fallback mais recente
  • Monta payload para GAS Smart Blocks v4.0
  • Retorna petição-base como referência (não entrega ao usuário diretamente)

Gaps, Riscos e Considerações

O que pode dar errado e como mitigar

⚠️ Gap Crítico: olivia_modelo_juris vazia

0 vínculos entre modelos e jurisprudências. A skill olivia-smartblocks-interno não consegue inserir jurisprudências automaticamente.

Mitigação para o MVP

A skill olivia-smartblocks-interno busca jurisprudências por:
1. Estado do processo (filtro por tribunal_uf)
2. Assunto vinculado à tag
3. Mais recentes com texto_integral
Sem precisar da tabela olivia_modelo_juris por agora.

⚠️ Gap: Jurisprudências sem texto_integral

1.353 jurisprudências, mas 99% sem texto_integral — RAG não funciona para elas.

Mitigação para o MVP

Usar apenas ementa + numero_acórdão como contexto. O texto_integral pode ser preenchido progressivamente (E27 do plano original). Não bloqueia o MVP.

⚠️ Risco: Contexto longo (100+ páginas)

Processos grandes com muitos documentos podem exceder contexto do Claude.

Mitigação

  • Resumir cada documento antes de agregar
  • Usar olivia_agente_contexto para persistir entre calls
  • Análise em batch: 1 documento por vez, contexto acumulado
  • Modo avançado (Opus): contexto maior

⚠️ Risco: Skills expostas no gateway OlivIA

O gateway da OlivIA está público via Cloudflare. Skills precisam de autenticação.

Mitigação

O token olivia-gateway-token-2026 já está configurado no openclaw.json da OlivIA. O frontend passa este token no header Authorization. Apenas o MdFlow conhece o token.

✅ Ponto positivo: GAS já funciona

O GAS Smart Blocks v4.0 e GAS Express v2.0 estão funcionando. O problema era só no n8n.

A skill olivia-gera-peticao pode chamar o GAS diretamente da mesma forma que o n8n faria — sem precisar de nenhum fix no GAS.

✅ Ponto positivo: OLIVIA-ANALISE-V4 é a base perfeita

O workflow de análise no n8n (50 nodes, sem fetch() quebrado) é exatamente o que a skill olivia-analisa-provas precisa replicar.

Nós temos o mapa completo de nodes: Vision, OCR, RAG, Claude. A skill será uma versão .mjs fiel ao workflow.
