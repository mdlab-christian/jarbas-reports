A Questão da Formatação — Google Doc obrigatório

Sim, o Google Doc ID é necessário. Veja o porquê e como funciona.
⚠️
A pergunta do Christian:

"Será que a gente precisa de Google Doc ID nesse modo agente?" — Sim, precisa. A formatação jurídica (recuos, negrito, fonte, espaçamento) só é preservada 100% via Google Docs. HTML ou texto puro perdem a formatação. Mas o fluxo já está implementado na skill_olivia_agente_v2!

Por que Google Doc é obrigatório

  • Formatação jurídica: recuo de 1,5cm, espaçamento 1,5, fonte Arial/Times 12pt
  • Negrito nos títulos de seção (DOS FATOS, DO DIREITO, etc.)
  • Imagens em blocos de texto (reportagens como prova) — só funciona via Doc
  • Cópia de template: o GAS copia o modelo Google Doc e substitui os blocos — toda formatação do modelo é preservada automaticamente
  • Download PDF direto do Drive — sem conversão manual
Fluxo atual: Claude gera texto → GAS usa texto para preencher Google Doc template → preserva formatação 100%

Como a formatação funciona na prática

// skill_olivia_agente_v2.mjs — fluxo interno 1. Claude gera petição como JSON estruturado: { secoes: [ { tipo: "DOS_FATOS", conteudo: "texto..." }, { tipo: "DO_DIREITO", conteudo: "texto..." } ], jurisprudencias: ["..."] } 2. GAS Smart Blocks v4.0 recebe payload: - Copia Google Doc template (modelo_peticao.google_doc_id) - Para cada bloco: substitui {{placeholder}} pelo conteúdo - Preserva: fonte, tamanho, recuo, negrito do template - Insere jurisprudências nos placeholders {{juris_N}} 3. Resultado: Google Doc com formatação perfeita → URL salva em olivia_historico.google_doc_url

E se o modelo não tem google_doc_id?

A skill detecta e usa o Express GAS v2.0 como fallback — cria doc vazio e insere HTML formatado. Funciona mas perde algumas formatações avançadas.
