📄 6. Google Apps Script (GAS) — Estratégia de Adaptação
⚙️ GAS v3.2 Atual — Como Funciona

O GAS foi feito para o Express/Smart Blocks (petições por blocos separados).

ItemExpress/Smart BlocksModo Agente
Formato de entrada secoes[] (cada bloco = Google Doc) HTML contínuo gerado pela IA
Placeholders {{enderecamento}}, {{qualificacao}}… Sem placeholders — texto pronto
Estrutura Blocos separados → montagem Texto único → inserir direto
Estratégia recomendada: htmlParaSecoes(html) na skill
// Na skill_olivia_agente_v2.mjs
function htmlParaSecoes(peticaoHtml) {
  // Converte HTML da petição para o formato de seções do GAS
  // Mantém compatibilidade sem modificar o GAS v3.2
  return [
    {
      placeholder: "{{peticao_completa}}",
      conteudo_texto: stripHtml(peticaoHtml),
      conteudo_html: peticaoHtml
    }
  ];
}

// Enviar para GAS no mesmo formato
const payload = {
  secoes: htmlParaSecoes(peticao_html),
  tags_substituicao: { ...metadados },
  titulo_documento: `Petição - ${processo.numero_cnj}`,
  google_folder_id: config.folder_id
};

✅ Reutiliza GAS existente sem modificação. GAS v3.2 recebe as seções e cria o Google Doc normalmente.
