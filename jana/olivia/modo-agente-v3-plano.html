Os 4 Steps — Adaptados

Mesma estrutura do Express · Motor e lógica trocados
1
Step 1 — Identificação e Upload

Processo + Tipo + Modelo + Documentos

O que o Express já tem:

  • Seleção de processo (ProcessoSelector)
  • Tipo de petição
  • Modelo de petição (ModeloPeticaoSelector)
  • Upload de documentos com tipo por arquivo

O que muda:

  • Ao clicar "Avançar": enviar documentos para OlivIA analisar
  • OlivIA faz OCR/leitura dos PDFs e extrai fatos do processo
  • Resultado da análise vai para o Step 2
  • FK fix: garantir que tipo_documento está correto por arquivo

Endpoint OlivIA: POST /olivia/analisar-documentos → retorna resumo dos fatos + perguntas sugeridas

2
Step 2 — Triagem + Chat ao Vivo

Perguntas baseadas nas provas + AgenteChat aberto

O que o Express já tem:

  • Formulário de perguntas
  • Estrutura de wizard
  • Componente AgenteChat existente

O que muda:

  • Perguntas geradas pela IA (não template fixo) — baseadas na análise das provas do Step 1
  • AgenteChat aberto desde o início do Step 2 — usuário pode tirar dúvidas enquanto preenche
  • Chat conectado ao OpenClaw OlivIA (SSE ou polling)
  • Respostas do formulário alimentam o contexto da geração

Endpoint OlivIA: POST /olivia/gerar-triagem → retorna perguntas JSON. Chat: SSE via /olivia/chat-stream

3
Step 3 — Geração com RAG

OlivIA gera a petição com blocos + modelos + juris

O que o Express já tem:

  • Barra de progresso de geração
  • Polling de status
  • GeracaoProgress component

O que muda:

  • POST para /olivia/gerar-modo-agente no OpenClaw (não n8n)
  • OlivIA usa RAG (blocos + juris + playbooks do cérebro)
  • Score de qualidade retornado pelo agente
  • Revisor automático interno (score < 9.0 → revisão, max 2x)
  • Chat continua aberto durante geração

Endpoint OlivIA: POST /olivia/gerar-modo-agente → retorna tarefa_id. Status: GET /olivia/tarefa-status/:id

4
Step 4 — Resultado

Documento gerado + Download + Feedback

O que o Express já tem:

  • Exibição do documento gerado
  • Botão de download

O que muda:

  • PDF gerado via Puppeteer (HTML→PDF no Mac Mini) — não GAS
  • Score de qualidade exibido
  • Feedback de estrelas para treino futuro
  • Botão "Ajustar com chat" → reabre AgenteChat para refinamento
⚙️

Motor OpenClaw OlivIA

Como o frontend chama a OlivIA
🔌
A OlivIA ainda não tem skills de geração — elas serão criadas após o Cérebro estar migrado. Mas os endpoints já devem ser definidos agora para o frontend se conectar. O frontend bate no endpoint → OlivIA processa com a skill → retorna resultado.
Endpoint OlivIAMétodoO que fazRetorno
/olivia/analisar-documentos POST Recebe PDFs do processo → OCR + extrai fatos → sugere perguntas JSON: {fatos, perguntas[]}
/olivia/gerar-triagem POST Com fatos + tipo + modelo → gera perguntas específicas JSON: {perguntas[]}
/olivia/gerar-modo-agente POST Enfileira geração da petição (assíncrono) JSON: {tarefa_id}
/olivia/tarefa-status/:id GET Polling do status da geração JSON: {status, progresso, resultado?}
/olivia/chat-stream POST (SSE) Chat ao vivo com a OlivIA durante qualquer step SSE stream de tokens
