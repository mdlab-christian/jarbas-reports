8 — Checklist Anti-Alucinação por Prompt CC

Cada prompt para CC deve ter:

  • [CONTEXTO] — Arquivos envolvidos (caminho completo + conteúdo atual)
  • [OBJETIVO] — O que deve ser feito (1 coisa só)
  • [REGRAS] — Restrições específicas (ex: "modelo_id = versao.modelo_grupo_id, não versao_id")
  • [CRITÉRIO DE SUCESSO] — Como verificar que funcionou (ex: "consulta SQL retorna historico com modo=EXPRESS")
  • [NÃO FAZER] — O que explicitamente não alterar (ex: "não alterar OliviaGeradorSmartBlocks.tsx")
  • [VERIFICAR] — Checklist para CC rodar após implementar
