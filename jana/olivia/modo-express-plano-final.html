Princípios Fundamentais (Imutáveis)

🏗️ Smart Blocks é a base de TUDO

Antes de qualquer geração Express, a Olivia SEMPRE gera internamente uma petição pelo Smart Blocks. Essa petição-base serve como: (1) estrutura da petição, (2) style guide do advogado, (3) âncora anti-alucinação. A Olivia aprimora a base — nunca inventa do zero.

🏷️ Tags = Perguntas que a IA responde

As tags do modelo são as mesmas "perguntas" que o usuário respondia no Smart Blocks manual, só que agora a Olivia responde automaticamente usando análise de provas + dados do processo + playbooks. O filtro categoria_empresa + versao reduz centenas de tags para ~15.

👤 Humano no loop — mas rápido

Olivia pode gerar sem ninguém na frente do PC, mas não vai. O humano valida: (1) respostas das perguntas no Step 2, (2) estrutura no Step 3, (3) aprovação do revisor pós-geração. Cada validação é rápida — não substitui o trabalho, acelera.

📊 Relatório = Memória do Processo

Relatório gerado no Step 2 é armazenado vinculado ao processo. Step 5 o atualiza com dados da petição gerada (economiza tokens). Futuramente: nova geração desse processo → Olivia puxa o relatório anterior e dispensa re-upload de documentos.

🎨 Versão = Estilo de Escrita

A petição Smart Blocks gerada internamente usa a versão selecionada pelo usuário. Essa versão tem o estilo de escrita do advogado. A Olivia herda esse estilo ao aprimorar — garantindo que não pareça petição gerada por IA.

📎 Sem documentos = Modo Gracioso

Se o usuário não enviar documentos (petições simples: emendas, manifestações), a Olivia pergunta o que precisa ao usuário no Step 2 — atua como Smart Blocks turbinado. Nenhum upload obrigatório para modelos sem tags de prova.

Fluxo Completo de Uma Geração Express
UI
Step 1 — Usuário preenche e avança

Seleciona processo → frontend puxa dados (restricoes, empresa, comarca, UF, advogado). Seleciona tipo petição → modelo → versão (cascata). Upload de documentos com tipo_prova. Ao clicar "Avançar":

SK
olivia-orquestra dispara (assíncrono)

Frontend faz POST → recebe 202 imediato. Orquestradora começa: UPDATE historico status=EM_PROGRESSO. Frontend navega para Step 2 mostrando loading "Analisando caso..."

AI
olivia-analisa-provas (paralela ao Step 2 loading)

Lê docs do bucket + dados do processo + playbook da categoria empresa. Cruza: contestação justifica a restrição? Provas são válidas? Gera análise estruturada por documento. Salva em olivia_analises. Gera relatório HTML v1 (provisório).

AI
olivia-gera-perguntas (depende da análise)

Recebe análise + tags do modelo filtradas por categoria_empresa (de ~200 tags → ~15). Para cada tag: já tem a resposta? (RAG, banco, análise) → sim: responde automaticamente. Não tem certeza → inclui como pergunta ao usuário. Gera máximo 5-8 perguntas reais ao usuário.

DB
UPDATE olivia_historico — perguntas prontas

SET meta.perguntas = [...], meta.respostas_auto = {...}, progresso = 30. Supabase Realtime notifica o frontend → perguntas aparecem no Step 2 (fim do loading).

UI
Step 2 — Usuário responde perguntas

Perguntas específicas do caso aparecem. Botão "Ver Relatório" disponível (relatório v1). Usuário responde + pode adicionar observações por voz (Whisper). Ao submeter: frontend faz PATCH historico.meta.triagem_respostas.

UI
Step 3 — Usuário valida estrutura

Outline vem das tags do modelo (já filtrado). DnD para reordenar. Campo de instrução por tag + mic. Botão "Ver Relatório" persistente. Ao avançar: olivia-orquestra continua.

SK
olivia-smartblocks-interno (sem interação do usuário)

Para cada tag: seleciona melhor bloco de texto (olivia_blocos_texto filtrado por tag + UF + categoria_empresa). Seleciona jurisprudência correta (mesmo estado do processo, mesma situação). Monta petição-base substituindo placeholders. Essa petição-base é o "modelo de partida".

AI
olivia-gera-peticao (Claude melhora a base)

Input: petição-base Smart Blocks + análise de provas + respostas do usuário + instruções por tag + qualidade_ia. Claude instrução: "Esta é a petição-base no estilo do advogado. Melhore-a com base no contexto do caso. Não invente fatos. Preserve o estilo." Gera petição aprimorada.

AI
olivia-revisor (agente revisor pós-geração)

Compara petição Express com petição Smart Blocks base. Score de qualidade (0-10). Se score ≥ 7 → aprovado automaticamente. Se score < 7 → devolve para usuário com as considerações. Usuário pode: aprovar mesmo assim, pedir ajuste específico, ou regenerar.

UI
Step 4 — Geração em progresso (Realtime)

Barra de progresso real (via Supabase Realtime em olivia_historico.progresso). Mensagens contextuais por etapa. Se revisor pede validação → mostra para o usuário neste step. Botão "Ver Relatório" disponível durante espera.

📄
olivia-cria-doc + atualiza relatório

POST JARBAS /olivia/criar-doc (versão do advogado como template base). Retorna google_doc_url. Atualiza relatório HTML v1 → v2 final (adiciona: petição gerada, blocos usados, score revisor, métricas). Salva relatório no Supabase Storage vinculado ao processo_id.

UI
Step 5 — Conclusão

Google Doc + Relatório HTML v2 completo + métricas + feedback. Relatório salvo e acessível futuramente pelo processo.
