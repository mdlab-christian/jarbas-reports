3 — Arquitetura de Skills (Workflow Completo, não 2 skills)
💡
Decisão de Arquitetura Não 2 skills monolíticas. Sim: 1 skill orquestradora que chama sub-skills em sequência — igual a um workflow N8n, mas como código Node.js. Cada sub-skill tem responsabilidade única, é testável isoladamente, e pode ser evoluída sem afetar as outras.

🎯 olivia-orquestra.mjs — A Maestrina

Endpoint exposto: POST /api/v1/skills/olivia-orquestra · Responde 202 imediatamente · Processa em background

Sequência: analisa-provas → gera-triagem → aguarda respostas do usuário → smartblocks-interno → gera-peticao → cria-doc

⬇

📊 olivia-analisa-provas.mjs

Baseada no workflow de análise (ID: 8ntAvTGMhCfrlfRt)

  • Download e OCR de cada documento enviado
  • Classificação por tipo (olivia_tipos_prova)
  • Cruzamento com dados do processo (restrição, valor, contrato)
  • Lê playbook da categoria empresa (via bucket ou Obsidian)
  • Output: análise estruturada por documento + pontos fortes/fracos
  • Salva em olivia_analises + gera relatório HTML parcial

❓ olivia-gera-triagem.mjs

Substitui Fluxo A do N8n Express

  • Recebe análise de provas + dados do processo + tags do modelo
  • Claude gera perguntas específicas do CASO (não genéricas)
  • Regra: nunca perguntar nome, CPF, valor, comarca (já está no banco)
  • Tipos: sim/não, texto livre, select, multiselect
  • Obs da OlivIA: "Contestação fraca — Súmula 385 STJ aplica"

🧱 olivia-smartblocks.mjs

Baseada no workflow SmartBlocks (ID: hyGajjic2kZBVGvW)

  • Percorre tags do modelo (olivia_modelo_tags)
  • Para cada tag: busca melhor bloco (estado + categoria empresa)
  • Seleciona jurisprudências por UF do processo
  • Gera petição-base mecânica (anti-alucinação)
  • Não chama GAS ainda — só monta o payload

✍️ olivia-gera-peticao.mjs

Baseada no Fluxo B do N8n Express (B3→B7)

  • Recebe: petição-base + análise + respostas triagem + instrucoes
  • Claude Sonnet (padrão) ou Opus (avançado)
  • Aprimora sem inventar — segue petição-base como estrutura
  • Sistema de revisão com score (B7-Revisor)
  • Re-gera se score baixo (até 2x)
⬇

📄 olivia-cria-doc.mjs

Chama JARBAS POST /olivia/criar-doc → retorna Google Doc URL. UPDATE olivia_historico: SUCESSO + google_doc_url + score_revisor + tokens_usados + tempo_geracao_ms + modelo_usado. Gera URL pública do relatório HTML → url_publica.

// olivia-orquestra.mjs — sequência completa async function orquestrar(payload) { const { historico_id, processo_id, versao_id, docs, outline, triagem_respostas, instrucoes, qualidade_ia } = payload; // 1. Análise de provas (paralela ao carregamento do step 2) const analise = await analisa_provas({ processo_id, docs }); await updateHistorico(historico_id, { progresso: 20 }); // 2. Gera triagem → retorna para o frontend via olivia_historico const triagem = await gera_triagem({ processo_id, versao_id, analise }); await updateHistorico(historico_id, { progresso: 35, triagem_data: triagem }); // PAUSE: frontend detecta triagem_data via Realtime → exibe perguntas // Aguarda UPDATE com triagem_respostas (frontend faz PATCH) await aguardaRespostas(historico_id); // polling em jarbas_tarefas // 3. SmartBlocks interno (petição-base) const base = await smartblocks({ processo_id, versao_id, analise }); await updateHistorico(historico_id, { progresso: 60 }); // 4. Gera petição aprimorada const peticao = await gera_peticao({ base, analise, triagem_respostas, instrucoes, qualidade_ia }); await updateHistorico(historico_id, { progresso: 85 }); // 5. Cria Google Doc via JARBAS const doc = await cria_doc({ peticao, processo_id, versao_id }); await updateHistorico(historico_id, { status: 'SUCESSO', progresso: 100, google_doc_url: doc.url, score_revisor: peticao.score, relatorio_html_url: geraRelatorioURL(analise, peticao) }); }
