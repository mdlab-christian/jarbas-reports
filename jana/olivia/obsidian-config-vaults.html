💡 O que os 3 vídeos ensinaram (aplicado à nossa realidade)
Dave Swift — "A memória não funciona sem vault"

Configurar o vault no config do agente. Sem espaços no nome. Agente indexa ao reiniciar. Vault = memória real dos agentes.

Alex McFarland — "user.md é o arquivo mais importante"

Um CLAUDE.md/SOUL.md por vault com filosofia do agente. Sync entre máquinas via Obsidian Sync. Plugin HTML Reader para visualizar dashboards.

Rafael Quintanilha — "Headless Sync = agente sem interface"

obsidian-headless CLI para Mac Mini rodar sync sem abrir o app. obsync antes de ler. Vault contínuo sincronizado enquanto os agentes operam.

🎯
Consenso dos 3: vault separado por propósito, nunca misturar.

Cada agente tem seu vault com filosofia própria. CLAUDE.md/SOUL.md ensina o agente como navegar. INDEX.md como ponto de entrada. Sem espaços nos nomes de pasta.

📱 Fase 1 — MacBook Air (posso executar agora)
1
Renomear claudia-vault → cecilia-vault e consolidar

O ~/claudia-vault/ tem 2 subpastas com conteúdo idêntico: vault-obsidian/ e cecilia-vault/. Renomear e usar cecilia-vault/ como base.

mv ~/claudia-vault ~/cecilia-vault # Atualizar obsidian.json para refletir novo path
2
Instalar obsidian-headless no MacBook Air

CLI oficial da Obsidian para sync sem interface. Requer Node.js 22+.

npm install -g obsidian-headless ob login # autenticar com conta Obsidian do Christian (interativo)
3
Criar os 3 remote vaults na conta Obsidian Sync

Criar uma vez na nuvem. Todos os devices conectam ao mesmo remote.

ob sync-create-remote --name "mdlab-vault" --encryption e2ee ob sync-create-remote --name "olivia-vault" --encryption e2ee ob sync-create-remote --name "cecilia-vault" --encryption e2ee ob sync-list-remote # confirmar que os 3 aparecem
4
Conectar vaults locais aos remotos e fazer upload inicial
cd ~/mdlab-vault && ob sync-setup --vault "mdlab-vault" --device-name "MacBook-Air-Jana" cd ~/olivia-vault && ob sync-setup --vault "olivia-vault" --device-name "MacBook-Air-Jana" cd ~/cecilia-vault && ob sync-setup --vault "cecilia-vault" --device-name "MacBook-Air-Jana" # Fazer upload de cada vault: cd ~/mdlab-vault && ob sync cd ~/olivia-vault && ob sync cd ~/cecilia-vault && ob sync
5
Plugins Obsidian (você faz no app)

Community Plugins → instalar:

  • Data Files Editor — visualizar/editar JSON, TXT no Obsidian
  • HTML Reader — renderizar HTMLs de dashboards direto no app

Settings → Sync → "Sync all other types" ON em cada vault (sincroniza HTML, JSON, TXT)

🖥️ Fase 2 — Mac Mini (quando SSH voltar)
⚠️ Mac Mini está offline agora (SSH timeout). Esta fase executa quando voltar.
6
Verificar usuários e criar cecilia se necessário
ssh mac-mini "sudo dscl . -list /Users | grep -v '^_'" # Deve mostrar: jarbas, olivia, cecilia # Se cecilia não existir: ssh mac-mini "echo '010597' | sudo -S dscl . -create /Users/cecilia"
7
Mover olivia-vault de /Users/Shared/ para /Users/olivia/

Rafael enfatizou: vault por usuário, não compartilhado. OlivIA deve ter seu próprio espaço isolado.

ssh mac-mini "echo '010597' | sudo -S mv /Users/Shared/olivia-vault /Users/olivia/olivia-vault" ssh mac-mini "echo '010597' | sudo -S chown -R olivia:staff /Users/olivia/olivia-vault"

Atualizar /Users/olivia/.openclaw/openclaw.json memorySearch.extraPaths para novo path.

8
Criar cecilia-vault para o usuário cecilia
ssh mac-mini "echo '010597' | sudo -S mkdir -p /Users/cecilia/cecilia-vault" ssh mac-mini "echo '010597' | sudo -S chown -R cecilia:staff /Users/cecilia/cecilia-vault" # Conteúdo vai chegar via Obsidian Sync ao conectar
9
Instalar obsidian-headless em cada usuário do Mac Mini
# Como jarbas (já tem npm): ssh mac-mini "/opt/homebrew/bin/npm install -g obsidian-headless" # Como olivia: ssh mac-mini "echo '010597' | sudo -S su - olivia -c '/opt/homebrew/bin/npm install -g obsidian-headless'" # Como cecilia: ssh mac-mini "echo '010597' | sudo -S su - cecilia -c '/opt/homebrew/bin/npm install -g obsidian-headless'"
10
Conectar cada vault ao Obsidian Sync remoto

⚠️ ob login é interativo — precisa da senha da conta Obsidian do Christian. Fazer via SSH com PTY.

# jarbas — mdlab-vault: ssh -t mac-mini "cd /Users/jarbas/mdlab-vault && ob login && ob sync-setup --vault 'mdlab-vault' --device-name 'Mac-Mini-Jarbas'" # olivia — olivia-vault (pull-only): ssh -t mac-mini "echo '010597' | sudo -S su - olivia -c 'cd ~/olivia-vault && ob login && ob sync-setup --vault olivia-vault --device-name Mac-Mini-Olivia'" ssh mac-mini "echo '010597' | sudo -S su - olivia -c 'cd ~/olivia-vault && ob sync-config --mode pull-only'" # cecilia — cecilia-vault: ssh -t mac-mini "echo '010597' | sudo -S su - cecilia -c 'cd ~/cecilia-vault && ob login && ob sync-setup --vault cecilia-vault --device-name Mac-Mini-Cecilia'"
11
Configurar sync contínuo como serviço (launchd)

Rafael: o agente precisa que o vault esteja sempre atualizado antes de operar. launchd garante isso.

# Criar 3 services — exemplo mdlab-vault (jarbas): # /Library/LaunchDaemons/com.jarbas.obsidian-mdlab.plist cat << 'EOF' | ssh mac-mini "echo '010597' | sudo -S tee /Library/LaunchDaemons/com.jarbas.obsidian-mdlab.plist" <?xml version="1.0" encoding="UTF-8"?> <plist version="1.0"><dict> <key>Label</key><string>com.jarbas.obsidian-mdlab</string> <key>ProgramArguments</key><array> <string>/opt/homebrew/bin/ob</string> <string>sync</string> <string>--path</string><string>/Users/jarbas/mdlab-vault</string> <string>--continuous</string> </array> <key>UserName</key><string>jarbas</string> <key>RunAtLoad</key><true/> <key>KeepAlive</key><true/> <key>StandardOutPath</key><string>/tmp/obsidian-mdlab.log</string> <key>StandardErrorPath</key><string>/tmp/obsidian-mdlab.err</string> </dict></plist> EOF # Repetir para olivia-vault e cecilia-vault # Carregar todos: ssh mac-mini "echo '010597' | sudo -S launchctl load /Library/LaunchDaemons/com.jarbas.obsidian-mdlab.plist"
