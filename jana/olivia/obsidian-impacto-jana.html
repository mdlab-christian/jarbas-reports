🔴 O problema atual — o que realmente acontece
💀
A JANA "morre" e "renasce" em cada compactação.

Quando o contexto enche (~100k tokens), a conversa é compactada. Tudo que não foi salvo em arquivo — decisões verbais, contexto de projeto, preferências ditas de passagem — some. A próxima sessão começa quase do zero.

🔵 Isolamento de contexto — o que o Obsidian faz
🤔
O Obsidian por si só não isola contexto entre conversas.

Quem isola é o OpenClaw com o arquivo memory/group-[chat_id].md por grupo. O vault é a camada de conhecimento — não a camada de isolamento de sessão.

❌ O vault NÃO faz

  • Não isola contexto entre conversas automaticamente
  • Não impede compactação de acontecer
  • Não carrega tudo na memória o tempo inteiro
  • Não lembra de coisas ditas verbalmente que não foram salvas
  • Não substitui o contexto da conversa ativa

✅ O vault FAZ

  • Persiste decisões mesmo após compactação
  • Persiste lições aprendidas entre sessões
  • Estado dos projetos sempre disponível por busca
  • Reduz o que precisa ser recontextualizado
  • Compartilhado entre JANA, JARBAS e Claude Code

🗂️ Isolamento real de contexto — como funciona hoje
✅
O isolamento entre grupos já existe — via memory/group-[chat_id].md

Cada grupo tem seu próprio arquivo de contexto. Este grupo (-5272336714) nunca contamina o JN Network (-5146490256). O vault complementa isso com conhecimento estruturado.

📁 Camadas de memória (após vault)

  • memory/YYYY-MM-DD.md — log diário (raw)
  • MEMORY.md — memória curada longo prazo
