🚀 O que falta para chegar a 100%
1
Instalar 3 plugins no Obsidian app (você faz — 10 min)

Settings → Community Plugins → Browse → instalar: Dataview (queries no vault), HTML Reader (ver dashboards do GitHub Pages dentro do Obsidian), Templater (templates de notas). Impacto imediato na usabilidade editorial.

2
Adicionar frontmatter + tags nos playbooks do cecilia-vault

Cada arquivo de playbook com tags: [critico, roteamento, sentenca] e nivel: critico. Desbloqueia filtros no app e Dataview queries. Posso fazer automaticamente.

3
Conectar o Obsidian app nos 3 vaults remotos (você faz — 5 min)

Settings → Obsidian Sync → Login → Connect Vault em cada um dos 3. Sem isso o app mostra o conteúdo local mas não sincroniza via UI. O sync CLI já funciona, mas o app não sabe que está conectado.

4
Log de aprendizado automático da Cecília

Quando João corrigir uma sugestão → bot escreve automaticamente no 06-LOG-APRENDIZADO/YYYY-MM.md. Obsidian vira diário de evolução dela. Posso implementar na skill do bot.

5
Adicionar wikilinks entre notas do olivia-vault

Bloco de "banco sem contrato" linkando para "tese assinatura eletrônica inválida". Playbook IF linkando para modelos de réplica. Graph View ficaria uma rede real de conhecimento jurídico.

6
Canvas do fluxo de intimações no cecilia-vault

Mapa visual: Captura EPROC → Classificação → Triagem João → Atividade criada → Protocolo. Documentação viva. Útil para onboarding de novos membros da equipe no escritório.

7
Reindexação automática do ChromaDB quando vault mudar

Hoje o índice RAG da OlivIA é gerado manualmente. Com um watcher de arquivos + reindexação automática, qualquer bloco novo adicionado ao olivia-vault fica disponível para RAG em minutos.

Principal gap: o Obsidian não sabe que está conectado ao Sync

O sync CLI funciona independente. Mas para você ver o indicador de sync no app, receber notificações e usar a UI de sync, precisa conectar os 3 vaults na tela de Settings → Obsidian Sync dentro do app.
