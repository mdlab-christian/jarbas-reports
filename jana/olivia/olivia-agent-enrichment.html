🚀 O Que Vai Ser Enriquecido (Priorizado)
P0 Crítico
Incluir bloco nos source_types do RAG
O agente busca RAG com p_source_types: ['jurisprudencia', 'peticao'] mas ignora os 1.244 blocos de texto que são a base da petição gerada. Sem isso, o agente escolhe blocos às cegas — sem busca semântica.
Arquivo: OLIVIA-ANALISE-V4 → Nó D4-BuscarRAG ANTES: p_source_types: ['jurisprudencia', 'peticao'] DEPOIS: p_source_types: ['jurisprudencia', 'peticao', 'bloco'] Impacto: Claude passa a escolher blocos com contexto semântico real
Impacto: Alto Esforço: Baixo — 1 linha de código
P0 Crítico
Mapear agent_key nos olivia_playbooks
Os 8 playbooks estão publicados mas com agent_key = null. O nó B3-HTTP-Playbook busca por categoria/tipo — mas se a lógica de seleção usa agent_key, vai retornar vazio.
UPDATE olivia_playbooks SET agent_key = 'olivia-analise-provas' WHERE scope = 'analise'; UPDATE olivia_playbooks SET agent_key = 'olivia-gerar-peticao' WHERE scope = 'peticao'; Playbooks de análise (6): Inst. Financeira, Mercado Livre, Energia, Lojas, Telefonia, Análise de Sentença Playbooks de petição (2): Negativação Prescrita, Restrição Geral
Impacto: Alto Esforço: Baixo — 2 UPDATEs
P1 Alta
Injetar instrucoes_ia dos modelos no prompt do Claude
303 modelos de petição têm instrucoes_ia preenchidas (tom, foco, peculiaridades do tipo de petição). Essas instruções não chegam ao Claude atualmente — ele não sabe as regras específicas do modelo que vai montar.
No nó E2-PrepararInput-Claude, adicionar ao USER_MESSAGE: <instrucoes_modelo_peticao nome="${p.modelo_template.nome}"> ${p.modelo_template.instrucoes_ia} </instrucoes_modelo_peticao> Pré-requisito: nó B3 já busca o template — só falta passar instrucoes_ia junto
Impacto: Médio-Alto Esforço: Baixo — editar E2
P1 Alta
Reanimar WH-JURI-02 (Olivia Chat Jurimetria)
O workflow de chat de jurimetria é um stub — recebe a mensagem, valida org_id e retorna um toast genérico. O agente de jurimetria não faz nenhuma análise real. Com 480 assuntos e 1.353 jurisprudências no banco, isso é desperdício puro.
Estrutura proposta para WH-JURI-02: 1. Receber query de jurimetria (texto livre) 2. Gerar embedding da query 3. Buscar fn_rag_search com source_type='jurisprudencia' 4. Montar contexto com ementa + categorias + paths 5. Chamar Claude com contexto das juris encontradas 6. Retornar análise jurisprudencial estruturada
Impacto: Alto Esforço: Médio — ~1h de dev
P1 Alta
Atualizar OpenAI Assistant com instrucoes completas
O Assistant asst_2bAsETPqoTXBcXt6H7LosHKC (usado no workflow legado OlivIA) tem apenas 1 arquivo de 12KB no vector store. Com o novo backend, podemos subir os playbooks, instruções de todos os tipos e os datasets como base de conhecimento.
Atualizar vector store vs_67fb4810a0208191b8280fc2013ab978: - Subir olivia_playbooks.instrucoes (8 playbooks) → ~50KB - Subir olivia_modelos_peticao.instrucoes_ia (303 modelos) → ~300KB - Subir dataset blocos como exemplos (top 100 pares tag→bloco) → ~50KB - Subir jurisprudências_assunto descricoes → ~40KB Total estimado: ~440KB de contexto rico Modelo: migrar de gpt-4-1106-preview para gpt-4o
Impacto: Alto Esforço: Médio — precisa de exports
P2 Normal
Few-shot examples do dataset no system prompt do Claude
Os datasets olivia-dataset-blocos-v1.jsonl (1.244 exemplos tag→bloco) e olivia-dataset-modelos-v1.jsonl (327 exemplos tipo→modelo) foram gerados mas não são usados. Injetar 5-10 exemplos relevantes no system prompt melhora a precisão da escolha de blocos.
Estratégia: no nó E2, antes do USER_MESSAGE, adicionar: <exemplos_few_shot> [filtrar 5 exemplos do dataset que matcham a categoria atual] exemplo 1: tag "contrato não assinado" → bloco "Ausência de Prova de Contratação" exemplo 2: tag "notificação inválida" → bloco "Irregularidade na Notificação Boa Vista" ... </exemplos_few_shot>
Impacto: Médio Esforço: Médio
P2 Normal
Adicionar ementa completa das jurisprudências no RAG
As 1.353 jurisprudências têm ementa preenchida mas o campo texto_integral está vazio. O RAG usa o chunk que foi indexado (ementa), mas com texto completo a busca semântica e o contexto enviado ao Claude seriam muito mais ricos.
Débito conhecido do backend. Caminho: scraping/import do texto completo das jurisprudências. Por ora: garantir que os chunks das ementas são completos (não truncados) e re-indexar se necessário. Verificar: SELECT avg(length(conteudo)) FROM rag_chunks WHERE document_id IN (SELECT id FROM rag_documents WHERE source_table = 'olivia_jurisprudencias')
Impacto: Médio Esforço: Alto — requer fonte externa
P3 Melhoria
agent_key dinâmico por playbook no RAG
O agent_key no D4 é fixo ('olivia-analise-provas') independente do fluxo. Com os playbooks mapeados, o agent_key pode ser lido do playbook carregado no B3, tornando a busca RAG mais precisa por tipo de análise.
No nó D4-BuscarRAG: ANTES: p_agent_key: 'olivia-analise-provas' DEPOIS: p_agent_key: $('B3-Process').item.json.agent_key || 'olivia-analise-provas' Pré-requisito: P0 — mapear agent_key nos playbooks primeiro
Impacto: Baixo-Médio Esforço: Mínimo
