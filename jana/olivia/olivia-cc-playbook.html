O que faz:

  3. Roda playbook de 22 perguntas para avaliar qualidade probatória
  4. Define tom: incisivo (>70% docs fracos) | neutro | defensivo
  5. Salva em olivia_historico.analise_provas
⚠️ Docs sem texto_extraido são ignorados. Sempre extrair texto após upload.

📄 S8 — olivia-cria-doc.mjs

Gera DOCX + PDF + Google Doc

Processo de geração:

  1. Gera DOCX via docx-generator.mjs (lib docx npm):
    • Times New Roman 12pt
    • Jurisprudências: 10pt, recuo 30mm (3cm), itálico — detecta <blockquote>
    • Subtítulos em negrito
  2. Gera PDF via Puppeteer (headless Chrome)
  3. Upload DOCX + PDF para bucket olivia-peticoes no Supabase
  4. Chama JARBAS (/olivia/criar-doc) para Google Doc (opcional)
  5. Salva URLs em olivia_historico
  6. Seta status = 'CONCLUIDO'
💡 JARBAS cria o Google Doc — OlivIA só gera DOCX/PDF. Sem menção ao "JARBAS" na UI.

🖥️ Frontend — 5 Steps do Wizard

Step 1 — Identificação

src/pages/olivia/components/gerador/OliviaGeradorExpress.tsx

Componentes:

  • SearchBoxProcesso — busca por CNJ ou nome do cliente
  • SeletorModelo — query: olivia_modelos_grupo WHERE status='published' (coluna status, não ativo)
  • SeletorVersao — usa active_version_id do grupo
  • SeletorTipoPeticao — 10 tipos disponíveis
  • UploadDrawer — upload docs do réu

UploadDrawer (detalhe crítico):

  • Bucket: olivia-docs
  • Path: {'{'}org_id{'}'}/processos/{'{'}processo_id{'}'}/{'{'}timestamp{'}'}_{'{'}nome{'}'}.pdf
  • Após upload → chama /olivia/extrair-texto em background (fire-and-forget)
  • Toggle "Analisar documentos com IA" — default ON

Ao clicar "Gerar":

  1. Insere olivia_historico (status=PENDENTE)
  2. Chama callWebhook(OLIVIA_WEBHOOKS.EXPRESS_GERAR, {"{"} fase: 1, modelo_grupo_id: dados.modeloId, ... {"}"})
  3. Transição para Step 2

Step 2 — Triagem / Perguntas

src/pages/olivia/components/express/ExpressStep2Triagem.tsx

Escuta via Supabase Realtime em olivia_historico:

  • AGUARDA_RESPOSTAS → exibe perguntas de perguntas_geradas.perguntas_usuario
  • CONCLUIDO/SUCESSO (sem perguntas) → auto-advance para Step 3 (delay 3s)
  • NUNCA avançar automaticamente com status AGUARDA_RESPOSTAS
⚠️ Race condition corrigida em commit 4beaabb7. Estado triageStatus controla o fluxo.
