🔴 Problema Atual: Upload Manual é Frágil

Por que análise de provas falha hoje

O UploadDrawer depende de ação humana — o usuário precisa:

  1. Entrar no EPROC manualmente
  2. Identificar quais documentos são relevantes para o tipo de petição
  3. Baixar cada documento
  4. Fazer upload no drawer do MdFlow
  5. Aguardar extração de texto via pdftotext

Na prática isso nunca acontece — os documentos ficam vazios e a petição é gerada sem análise real de provas. O ag. 4359 nunca aparecia por isso.

❌ Hoje

  • Upload manual = raramente feito
  • Análise de provas opera no vazio
  • Petição genérica sem fatos concretos
  • Texto truncado em 300 chars por doc (fix aplicado hoje: 2.000)
  • Depende da memória do advogado sobre o processo

✅ Com Auto-Análise EPROC

  • OlivIA busca os docs automaticamente
  • Seleciona documentos relevantes pelo tipo de petição
  • Análise completa com dados reais do processo
  • Ag. 4359, datas, valores — tudo na petição
  • Zero trabalho do advogado

💡 Solução Proposta: Botão "Analisar processo" + Skill S0.5

Conceito: substituição do UploadDrawer por um toggle

Em vez do drawer de upload manual, o Step 1 ganha um toggle boolean "Analisar processo automaticamente" (default: ON).

Quando ativo, ao avançar para Step 2, a OlivIA:

  1. Acessa o EPROC com as credenciais do advogado (já armazenadas no banco)
  2. Navega ao processo pelo número CNJ
  3. Identifica quais documentos baixar com base no tipo de petição selecionado (contexto jurídico)
  4. Baixa os PDFs dos eventos relevantes (contestação, sentença, contrato, etc.)
  5. Faz upload no bucket olivia-docs e extrai texto automaticamente
  6. Continua o pipeline com análise de provas real

Onde roda o Playwright

O Playwright roda exclusivamente no Mac Mini (usuário jarbas) — mesma infraestrutura do monitoramento EPROC do JARBAS. Não precisa criar usuário separado para a OlivIA.

A skill olivia-eproc-baixar.mjs já existe e já delega para as skills do JARBAS via import direto. O eproc-delegator.mjs já implementa:

  • loginEproc() — state machine com Keycloak/TOTP/2FA
  • navegarProcesso() — extrai movimentações e links de documentos
  • downloadDocumentosEproc() — 3 fluxos: ZIP, PDF direto, visor impressão

⚡ Concorrência: Múltiplas Petições Simultâneas

⚠️ Playwright é single-browser por padrão — mas tem solução

Cada instância Playwright abre um browser. Se 3 advogados gerarem petições ao mesmo tempo, temos 3 browsers simultâneos no Mac Mini.

Capacidade estimada do Mac Mini M4

  • RAM disponível: ~20GB (24GB - SO - JARBAS - Vite)
  • Por browser Chromium: ~300–600MB
  • Máximo seguro: 3–4 gerações simultâneas
  • Tempo por processo: ~45–90s (login + navegar + download 5–10 docs)

✅ Estratégia: Fila com concorrência limitada

// olivia-eproc-auto.mjs — controle de concorrência
const MAX_CONCURRENT = 3;
let activeBrowsers = 0;

export async function executar(input) {
  // Aguardar slot disponível
  while (activeBrowsers >= MAX_CONCURRENT) {
    await sleep(5000);
    // Atualizar progresso: "Aguardando slot EPROC (fila: X)..."
  }
  activeBrowsers++;
  try {
    return await _executarEproc(input);
  } finally {
    activeBrowsers--;
  }
}

Na prática, para o escritório Midas, dificilmente haverá mais de 2 gerações simultâneas. A fila é segurança extra.

✅ Session reutilização: uma sessão por OAB

Em vez de logar a cada geração, manter sessão Playwright persistente por OAB (mesmo padrão do monitoramento do JARBAS). Login custa ~15s — desnecessário se a sessão estiver ativa.

// Cache de sessões por OAB
const sessions = new Map(); // oab → { page, browser, lastUsed }

async function getOrCreateSession(oab, credenciais) {
  const cached = sessions.get(oab);
  if (cached && !isSessionExpired(cached.page)) {
    return cached; // reutiliza sessão existente
  }
  const session = await loginEproc(credenciais);
  sessions.set(oab, { ...session, lastUsed: Date.now() });
  return session;
}

Com isso, a segunda geração com o mesmo OAB é instantânea (sem login). Tempo cai de ~90s para ~30–45s.

⚠️ Riscos e Mitigações

MÉDIO
EPROC bloqueia acesso automatizado
Alguns TJs detectam Playwright por User-Agent ou comportamento.
Mitigação: skill_login_eproc.mjs já usa User-Agent real de Chrome e delays humanizados. O JARBAS faz isso há meses sem bloqueio no TJRS. TJSC/TJSP — testar antes.
MÉDIO
EPROC offline / VPN necessária
Se o EPROC estiver fora do ar ou exigir VPN, a geração trava.
Mitigação: timeout de 30s + fallback gracioso: "Não foi possível acessar o EPROC. Você pode fazer upload manual ou continuar sem documentos."
BAIXO
Processo não encontrado no EPROC
Processo pode ter número CNJ incorreto ou não estar na OAB do advogado.
Mitigação: verificar via numero_cnj do banco. Se não achar, continuar sem docs (não bloqueia a geração).
BAIXO
Senha do EPROC desatualizada
Advogado mudou a senha sem atualizar no sistema.
Mitigação: detectar erro de login e notificar: "Credenciais EPROC desatualizadas. Atualize em Configurações → Perfil." Pipeline continua sem docs.
BAIXO
Tempo adicional de geração (+45–90s)
O acesso ao EPROC adiciona tempo antes do pipeline iniciar.
Mitigação: barra de progresso com etapa "Acessando EPROC..." visível. Session cache reduz para 30s na segunda geração.
BAIXO
Docs baixados não relevantes para o tipo de petição
EPROC tem muitos eventos — baixar tudo é lento e ineficiente.
Mitigação: classificador por tipo_peticao_id define prefixos prioritários. MAX_DOCS=8 por default. Baixa P1 primeiro (sentença, contestação).

Estrutura da nova skill olivia-eproc-auto.mjs

export async function executar(input) {
  const {
    historico_id, organizacao_id, processo_id,
    numero_cnj, tipo_peticao_id, advogado_oab,
  } = input;

  // 1. Buscar credenciais do advogado no banco
  const cred = await buscarCredenciaisAdvogado(env, advogado_oab);
  if (!cred) return { sucesso: false, motivo: 'sem_credenciais', documentos_baixados: 0 };

  // 2. Login com session cache
  const session = await getOrCreateSession(advogado_oab, cred);
  if (!session?.page) return { sucesso: false, motivo: 'login_falhou', documentos_baixados: 0 };

  // 3. Navegar ao processo
  const { docs_para_baixar, header } = await navegarProcesso(session.page, { cnj: numero_cnj });

  // 4. Classificar quais docs baixar pelo tipo de petição
  const docsAlvo = classificarDocsPorTipoPeticao(docs_para_baixar, tipo_peticao_id);

  // 5. Baixar PDFs (máx 8 docs)
  const baixados = await downloadDocumentos(session.page, docsAlvo.slice(0, 8));

  // 6. Para cada PDF: upload olivia-docs + insert processos_documentos + extrair texto
  for (const doc of baixados) {
    await uploadEInserirDoc(env, doc, processo_id, organizacao_id);
    // extrair-texto roda em background (fire and forget)
  }

  return { sucesso: true, documentos_baixados: baixados.length, header };
}
