⚠️ O que falta configurar/criar

1. Adicionar IDs da equipe no groupAllowFrom

A OlivIA só responde a Christian e Gustavo em grupos. Precisa adicionar os IDs dos advogados/estagiários ao groupAllowFrom no openclaw.json.

Ação: Christian manda os IDs → JANA/JARBAS editam o openclaw.json e reiniciam o gateway.

2. Skill de enriquecimento do vault

Quando a equipe diz "essa tese está errada, use isso", a OlivIA precisa:

  • Entender que é um feedback jurídico (não só conversa)
  • Classificar: nova tese / correção de argumento / nova jurisprudência / erro de formato
  • Salvar no vault no local correto (teses/, playbooks/, juris-novas/)
  • Confirmar para o usuário o que foi salvo

Trigger: palavras-chave como "salva isso", "adiciona ao vault", "nova tese:", "corrija:", "atualiza playbook"

3. Sincronização vault → banco Supabase

O vault é a memória da OlivIA, mas as gerações usam o banco Supabase (olivia_blocos_texto, olivia_jurisprudencias). O feedback salvo no vault precisa eventualmente ser promovido ao banco para impactar as petições geradas.

Fluxo: Vault (rascunho/revisão humana) → aprovação do Christian → INSERT no banco → próxima geração já usa

4. Geração de relatório HTML no Telegram

A equipe pede uma análise e a OlivIA precisa gerar um HTML detalhado (tipo os relatórios que a JANA gera) e mandar o link no chat.

Solução: OlivIA usa o servidor da JANA em ~/jarbas-reports/jana/olivia/ via SSH para publicar, ou usa uma pasta pública dedicada no GitHub Pages.

5. Verificar/habilitar recebimento de mídia

OpenClaw já passa mídia (áudio, imagem) para os agentes em alguns canais. Precisamos confirmar se a OlivIA:

  • Recebe áudio → transcreve via Whisper (Edge Function audio-transcribe)
  • Recebe imagens → analisa via Claude Vision
  • Recebe PDFs → extrai texto e analisa

Provável resposta: sim, tudo funciona via OpenClaw nativamente — a mesma infra que a JANA usa no Telegram.
