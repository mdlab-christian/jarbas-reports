Findings Detalhados — O Que Está Errado na Prática

🔴 pesquisa-web: Reclame Aqui sempre 403 — skill nunca usa dados reais
Impacto: toda análise de reputação do réu é mock. A skill deveria ser uma das mais valiosas (identificar empresa com histórico de fraude reforça tese). Sem dados reais, o SmartBlock de contexto do réu fica vazio.
Causa: GET sem User-Agent, sem cookies, sem Referer. Reclame Aqui usa Cloudflare Bot Protection.
🟡 analisa-provas: campo "pontos_fortes" lista pontos FRACOS do réu
O array pontos_fortes na resposta contém as inconsistências detectadas (pontos fracos do réu / pontos de ataque do autor). Nome enganoso — o frontend pode interpretar errado e exibir ao usuário como forças do réu.
Renomear: pontos_fortes → pontos_ataque_adversario | Adicionar campo separado pontos_alerta (riscos para o autor)
🟡 analisa-provas: chance_ganho fixo em 81% — não reflete os dados
No teste, os 3 pilares estavam com false (nenhum comprovado) mas chance_ganho retornou 81%. O cálculo deveria ser dinâmico: pilares comprovados + peso das provas + nível de impugnação.
pilares_comprovados: 0/3 → esperado chance_ganho ~30–50%, não 81%
🟡 gera-perguntas: perguntas são nomes técnicos das tags, não perguntas para o cliente
"(Empréstimo) Estrutura Probatoria Inicial" não é uma pergunta que o advogado faz ao cliente. A tag é um rótulo interno. Falta uma camada de humanização — simples chamada Claude com lista de nomes de tags, pedindo transformação em perguntas simples em pt-BR.
Input: ["(Empréstimo) Relação Juridica Previa Entre as Partes"] Output esperado: "O banco tem contrato anterior com você (conta corrente, outro produto)?"
🟡 gera-peticao: 22.843 tokens por chamada — custo elevado
O SmartBlock de 47.781 chars de HTML vai direto no prompt. HTML tem muito ruído (tags, atributos, espaços) que não agrega valor semântico para o Claude. Extrair apenas o texto limpo antes de montar o prompt reduziria 60–70% dos tokens.
47.781 chars HTML → ~8.000 chars texto puro → economia de ~14.000 tokens (~R$0,40 por petição)
🔵 cria-doc: warning puppeteer --localstorage-file
Não impede o funcionamento mas indica que o puppeteer está sendo lançado com um argumento inválido/desnecessário. Em versões futuras pode virar erro. Remover o argumento do launch.
Warning: `--localstorage-file` was provided without a valid path
🔵 gera-peticao: petição de reforma usa contexto de âncora de primeira instância
O modo "reforma" (apelação/agravo) no teste usou SmartBlocks de primeira instância. Petições de reforma precisam de contexto diferente: decisão recorrida, fundamentos do recurso, razões de reforma. O fluxo de SmartBlocks deveria filtrar por tipo_peticao.
