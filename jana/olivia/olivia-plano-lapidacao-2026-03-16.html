Antes de mais nada — o diagnóstico honesto

A OlivIA tem estrutura sólida. O banco está populado (1.244 blocos, 329 modelos), os workflows N8N existem, o frontend tem os 5 modos. O problema não é falta de código — é desconexão entre as camadas.

O frontend manda um processo para o N8N, mas os dados chegam incompletos. O N8N tenta gerar a petição mas a IA não sabe quem é o cliente, qual a empresa ré, qual o valor da causa. Resultado: petição genérica. É esse ciclo quebrado que precisamos consertar — e a solução é mais cirúrgica do que parece.

🗺️ O plano em 3 conversas paralelas
1
🔧

Conversa 1 — Backend JARBAS

Abre primeiro · A base de tudo

Aqui é onde o problema raiz mora. O JARBAS é o servidor que busca os dados do processo, monta o contexto e passa pra IA. Se ele mandar os dados errados (ou incompletos), o N8N e o frontend não têm como compensar.

O principal entregável desta conversa é o processo_context completo — um JSON rico com nome do cliente, CPF, endereço, empresa ré, CNPJ, advogado, OAB, valor da causa, tipo de decisão... tudo que a IA precisa pra escrever uma petição de verdade.

P0Criar rota /olivia/processo-context (monta o JSON completo com todos os joins)
P0Corrigir triagem: passar contexto do processo + provas para a IA gerar perguntas relevantes
P0Criar rota /olivia/relatorio (relatório HTML com análise das provas — Modo Agente)
P1Migration RLS: permitir que Modo Agente insira em jarbas_tarefas
P1Validar /olivia/criar-doc e DOCX pipeline (pandoc + template ABNT)
↓

Conversa 2 — N8N + Modo Agente

Abre depois do backend · Automação inteligente

Com o backend entregando o contexto correto, esta conversa garante que os workflows N8N usam esse contexto nos prompts da IA. Atualmente os workflows existem mas não estão recebendo (ou não estão usando) os dados do processo — daí a petição sai genérica.

O Modo Agente entra aqui também porque roda direto no JARBAS (não usa N8N) — são os mecanismos de briefing → estrutura → geração que precisam ser auditados e completados.

P0Auditar os 6 workflows ativos: verificar se processo_context chega e entra no prompt da IA
P0Limpar workflows legados (OlivIA antigo, DIST-GERAR-PETICAO, DIST-PETICAO, RECURSO separado)
P1Modo Agente: auditar skills JARBAS, corrigir steps incompletos e adicionar step de Revisão
P1Decidir: Modo Agente precisa de workflow N8N dedicado ou fica 100% no JARBAS?

Conversa 3 — Frontend

Abre por último · Conecta tudo

Com backend e N8N corretos, o frontend vira trabalho de conexão e UX. A maioria dos bugs B1-B10 some naturalmente quando as camadas de baixo estão certas — o que sobra é garantir que o React está chamando as rotas certas e exibindo os estados corretamente.

P0B1 — useGerarPeticao chama /olivia/processo-context antes de gerar (não manda dados vazios pro N8N)
P1B4 — Loading em tempo real via Supabase Realtime (não polling estático)
P1B5 — Fila: clique no card abre modal de detalhe
P1B6 — VibeLaw: parsear response da edge function (não mostrar JSON bruto)
P1B7, B8, B9, B10 — restante dos bugs de UX/feedback
