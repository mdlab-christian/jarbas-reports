olivia-eproc-baixar.mjs — Acesso EPROC direto
Nova · Substitui agente controller · OlivIA decide quais docs baixar
▼

Input

{ processo_numero, versao_id, historico_id, modelo_tags[], uf, tribunal }

Output

{ documentos_baixados: [{nome, tipo_prova, signed_url, relevancia}], decisoes_relevantes[], status_processo }

Lógica

  1. Login EPROC com credenciais do escritório (da tabela advogados_oabs)
  2. Buscar processo pelo número → listar todos os eventos
  3. Baseado nas tags do modelo selecionado, identificar QUAIS docs são relevantes (ex: se modelo é "réplica banco" → baixar contestação + provas do réu)
  4. Baixar PDFs relevantes → upload para bucket olivia-docs → retornar signed URLs
  5. UPDATE olivia_historico.analise_provas com lista de documentos encontrados
⚠️
Credenciais por escritório Buscar credenciais EPROC em advogados_oabs.meta.eproc_login + eproc_senha por UF. Multi-escritório: cada org tem suas próprias credenciais.

olivia-analisa-provas.mjs — Análise de documentos + chance de ganho
Baseada no Playbook Jurídico · Classificação forte/moderado/fraco · Score 0-100%
▼

Input

{ documentos: [{signed_url, tipo_prova_id, nome}], processo_dados, obs_iniciais, playbook_ids[] }

Output

{ analise_por_doc[], pontos_fortes[], alertas[], chance_ganho: 0-100, relatorio_v1_html, perguntas_preliminares[] }

Lógica de Análise (baseada no Playbook Jurídico)

  • Identifica tipo de ré (banco/fintech/loja/energia/telefonia/cessão/marketplace)
  • Para cada doc: classifica conforme tripé (relação jurídica / contratação específica / origem do débito)
  • Verifica incongruências: valor apontamento ≠ saldo, data divergente, número contrato diferente, IP sem correspondência de endereço
  • Detecta provas fracas: tela sistêmica (unilateral), selfie sem trilha, assinatura eletrônica sem certificação
  • Detecta provas fortes: contrato com assinatura + certificação, certidão cartório, nota fiscal + comprovante entrega
  • Calcula chance_ganho: peso das provas do réu vs teses do escritório disponíveis
  • Identifica preliminares ocultas (litigância má-fé, impugnação gratuidade)
  • Gera lista de incongruências para incluir na petição

Modo Avançado (Opus) · Lista selecionável no Step 3

Output

{ reclame_aqui: [{titulo, nota, url, data, resumo}], noticias: [{titulo, fonte, url, resumo}], score_reputacao: 0-10 }

Lógica

  3. Retornar lista formatada — usuário seleciona no Step 3 quais incluir e em qual tópico
  4. UPDATE olivia_historico.meta.pesquisa_web com resultado completo

olivia-smartblocks.mjs — Base anti-alucinação
Sempre primeiro · Petição-base via RAG + blocos do banco
▼

Input

{ versao_id, modelo_grupo_id, respostas_automaticas{}, triagem_respostas{}, processo_dados }

Output

{ peticao_base_texto, blocos_usados[], juris_usadas[], score_cobertura: 0-100 }

Lógica

  1. Buscar todas as tags do modelo via versao_id → olivia_modelos_peticao.tag_ids
  2. Para cada tag: fn_rag_search(tag_nome, estado_processo, tipo_reu) → top blocos relevantes
  3. Fallback de juris: busca por estado_processo.uf se olivia_modelo_juris VAZIA (está vazia)
  4. Montar petição-base substituindo {{tag_nome}} pelo bloco selecionado
  5. Esta petição-base é a âncora anti-alucinação para olivia-gera-peticao
