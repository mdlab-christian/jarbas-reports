❌ Fix 3 — Step 4 trava em 65% (nunca gera a petição)

Diagnóstico root cause: status AGUARDA_RESPOSTAS quando deveria estar gerando

Evidência do banco (coletada pela JANA):
AGUARDA_RESPOSTAS | pct=65 | msg='Gerando petição com IA...' | e89b8e47

Causa root confirmada: O server.mjs tem duas fases:
- Fase 1: analisa provas → gera perguntas → seta status AGUARDA_RESPOSTAS
- Fase 2: recebe respostas → SmartBlocks → gera petição

O bug é que o status nunca muda de AGUARDA_RESPOSTAS durante a Fase 2. O server.mjs mantém status=AGUARDA_RESPOSTAS enquanto roda SmartBlocks (55%) e Geração (65%). O frontend vê progresso_pct=65 mas status=AGUARDA_RESPOSTAS — e o canal Realtime do wizard só avança quando vê CONCLUIDO/SUCESSO.

Combinado com o polling de 8s adicionado hoje (boa mitigação), mas o status errado faz o wizard não saber que está na fase de geração — pode re-enviar as respostas por engano.
Fix: O server.mjs deve atualizar o status para EM_PROGRESSO (ou similar) quando começa a Fase 2 (pós-respostas). O frontend precisa saber que as respostas foram aceitas e a geração está rodando.

⚠️ Segundo fator: a Fase 2 pode não estar sendo disparada

Verificar se o endpoint que recebe as respostas (/olivia/gerar Fase 2) está sendo chamado corretamente pelo useOliviaWizard.ts. Se o wizard avança para Step 3 (outline) mas depois para Step 4, pode ser que a Fase 2 seja disparada via dispararGeracao() no hook — verificar se essa função envia perguntas_respondidas corretamente.
