🔀 Como o vault é usado no pipeline

S1

EPROC → baixar documentos

Delega ao JARBAS via eproc-delegator.mjs (Playwright)

S2

analisa-provas → Vault: lerPlaybook(categoria)

Playbook do segmento carregado. Juris locais buscadas por assuntos detectados.

S3

busca-juris → Vault: fallback + salvarJurisLocal()

Se banco retorna <4 resultados, usa vault local. Novas juris do banco são salvas no vault automaticamente.

S4

gera-perguntas → humanizarPerguntas() via Claude Haiku

13 tags → 13 perguntas em português natural.

S5

smartblocks → Vault: lerBlocoLocal() + buscarJurisPorTagLocal() + lerPlaybook()

9 blocos do vault + 22 juris por tag + playbook nas instruções do Claude.

S6

gera-peticao → Vault: lerPlaybook() + buscarTesesLocal() + salvarSessao()

Playbook + teses no system prompt. Sessão salva no vault após gerar.
