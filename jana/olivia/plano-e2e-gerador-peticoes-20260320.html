🔴 [P0] Fix arquivo compartilhado Smart Blocks — risco de regressão no Express 
Contexto: OlivIA v3 — beta-mdflow/src/pages/olivia/components/gerador/

Problema (P0): Existe um arquivo compartilhado entre OliviaGeradorSmartBlocks.tsx e 
OliviaGeradorExpress.tsx que está com um bug. Qualquer mudança no Smart Blocks pode 
quebrar o Express (e vice-versa). Isso é um risco de regressão grave.

Tarefa:
1. Identificar o arquivo compartilhado problemático
2. Se for um componente shared que tem lógica específica de modo: extrair a lógica 
   específica para cada modo e manter apenas o que é realmente compartilhado
3. Se for um tipo/interface: garantir que está corretamente tipado para ambos os modos
4. Adicionar um comentário explicando a fronteira de responsabilidade

Critério de aceite:
- Mudar Smart Blocks não quebra Express
- Mudar Express não quebra Smart Blocks
- Código compila sem erro TypeScript
- Componentes compartilhados em /shared/ são realmente genéricos
🟡 [P1] Fix polling multi-tenant — filtro organizacao_id ausente 
Contexto: OlivIA v3 — beta-mdflow/src/pages/olivia/hooks/

Problema (P1): O polling em olivia_historico (useExpressGeneration ou hook similar) 
não filtra por organizacao_id. Isso pode causar cross-tenant data leak — uma organização 
ver dados de outra.

Tarefa:
1. Encontrar todos os pontos de polling em olivia_historico
2. Verificar se há .eq('organizacao_id', organizacao_id) em cada query
3. Garantir que organizacao_id vem do useAuth() (não hardcoded)
4. Verificar também queries de leitura no Step 5 (conclusão)
5. Adicionar .is('deleted_at', null) em todas as queries que não têm

Stack: @tanstack/react-query v5 + Supabase JS SDK. staleTime: 30_000 obrigatório.

Critério de aceite:
- Toda query em olivia_historico filtra por organizacao_id
- Toda query filtra por deleted_at IS NULL
- Sem hardcode de organizacao_id
🟡 [P1] Fix outline stale — não reinicia ao trocar modelo no Step 1 
Contexto: OlivIA v3 — beta-mdflow/src/pages/olivia/components/gerador/express/

Problema (P1): Quando o usuário seleciona um modelo no Step 1, avança para o Step 3 
(Outline), e então VOLTA para o Step 1 e troca o modelo — o outline do Step 3 não 
reinicia. Mostra outline do modelo anterior.

Tarefa:
1. Identificar onde o outline é carregado (useModeloOutline ou similar)
2. Garantir que ao mudar modeloId/versaoId, o outline faz invalidate/reset
3. Se usar React Query: invalidateQueries(['olivia_modelo_tags', modeloId]) ao trocar
4. Se usar state local: resetar array de seções ao trocar modelo
5. Também resetar completedSteps para 'estrutura' quando modelo muda

Critério de aceite:
- Trocar modelo no Step 1 limpa o outline do Step 3
- Outline novo carrega com tags do modelo novo
- Sem resíduo do modelo anterior
🟡 [P1] Implementar relatório IA completo no Step 2 — spec não atendida 
Contexto: OlivIA v3 — beta-mdflow/src/pages/olivia/components/express/ExpressStep2Triagem.tsx

🟡 [P1] Fix label seletor de versão — "V1 · Nome · 20/03" conforme spec
