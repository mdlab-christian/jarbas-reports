🛡️ Princípios que entram na Skill (baseado no guia Anthropic + aprendizados)

Anti-alucinação

  • ✅ CC recebe PRD de 1 etapa — nunca o projeto inteiro
  • ✅ PRD inclui "O que NÃO fazer" explicitamente
  • ✅ CC não inventa — "se ambíguo, para e explica"
  • ✅ CC não altera banco sem instrução explícita no PRD
  • ✅ 2 tentativas falhas → PARA + alerta

Divisão de blocos (nova)

  • ✅ Complexidade: simples (1 fix) / média (1 componente) / complexa (1 aba)
  • ✅ Complexa = JANA quebra em sub-etapas automaticamente
  • ✅ Granularidade: 1 tab, 1 componente, 1 modal, 1 migration
  • ✅ Nunca: página inteira em 1 etapa

Pontos de falha (novo)

  • ✅ Antes de gerar prompt CC → JANA analisa pontos_falha
  • ✅ Se há pontos de falha → inclui no prompt como "investigue"
  • ✅ CC documenta o que encontrou em cada ponto

Persistência de estado (novo)

  • ✅ Toda sessão começa lendo o banco — não a memória
  • ✅ HTML gerado do banco, não da conversa
  • ✅ Projeto ativo sobrevive a múltiplas sessões JANA
  • ✅ Plano é vivo — etapas podem ser alteradas, versao++ ao alterar
