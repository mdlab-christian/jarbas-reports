Hierarquia do Sistema de Petições

Regra fundamental
💡 Esta hierarquia é o núcleo do sistema. IA nunca inventa — sempre segue esta estrutura. Inspirada no sistema antigo validado (milhares de petições geradas).
📂Tipo de PetiçãoEx: Revisional, Ação de Obrigação de Fazer, Ação Anulatóriaolivia_tipos_peticao
📋Modelo GrupoAgrupador de versões. Vinculado ao tipo + categoria_empresaolivia_modelos_grupo
🗂️Versão (Modelo de Petição)Versão publicada do modelo. Esqueleto da petição com {{placeholders}}olivia_modelos_peticao
🏷️TagsSeções/perguntas do modelo. Ex: "Qualificação das partes", "Mérito", "Pedidos". Ordem define estrutura.olivia_tags / olivia_modelo_tags
📝Bloco de TextoConteúdo formatado que vai na tag. Vinculado à categoria da empresa. 1 bloco por versão/tag/categoria.olivia_blocos_texto
⚖️JurisprudênciasPrecedentes vinculados ao bloco. Prioridade: estado do processo. Fallback: mais recente do assunto.olivia_jurisprudencias / olivia_modelo_juris
Como funciona no GAS (lógica validada):
1. Pega versão do modelo (Google Doc template com {{TAG_NOME}})
2. Para cada tag na ordem: busca bloco do banco → substitui {{TAG_NOME}} pelo conteúdo do bloco (preservando formatação do Google Doc do bloco)
3. Substitui jurisprudências: {{JURIS_TAG_1}}, etc.
4. Substitui {{dados}} do processo: {{cliente_nome}}, {{reu_nome}}, etc.
5. Limpa tags residuais não substituídas (marca em vermelho para revisão)

🔧 Fix P0 — B-ProgressStart (N8N)

Node Code JavaScript usando fetch() que não existe no ambiente de task runner do N8N. Substituir por HTTP Request node:

// ANTES (quebrado):
const response = await fetch('https://...', { method: 'POST', ... })

// DEPOIS (opção 1 — HTTP Request node):
// Substituir o Code node por um HTTP Request node configurado

// DEPOIS (opção 2 — dentro de Code node):
const response = await this.$helpers.httpRequest({
  method: 'POST', url: 'https://...', body: { ... }
})
