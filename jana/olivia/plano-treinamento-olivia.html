💡 Insight Chave — Arquitetura do Conhecimento

Tags = Perguntas situacionais / Blocos = Respostas jurídicas

183 tags organizadas em 3 seções: introdução (18), fundamentação (147), conclusão (18). Cada bloco de texto tem 1+ tags que descrevem a situação processual (ex: "Juntou contrato físico sem assinatura"). Esse é um dataset Q&A jurídico de altíssima qualidade para treinar a OlivIA.

Estrutura de Conhecimento Extraída

• 329 modelos = templates completos de petição (saber fazer)
• 1.244 blocos = argumentos jurídicos por situação (raciocínio)
• 1.353 jurisprudências = embasamento legal por assunto/tribunal (autoridade)
• 183 tags = situações processuais mapeadas (diagnóstico)
Juntos formam um corpus de ~4.000 documentos de conhecimento jurídico especializado em direito do consumidor.

Potencial de Treinamento

Com tags → blocos como pares Q&A, podemos: (1) fine-tuning de modelo base; (2) few-shot examples para cada tipo de petição; (3) retrieval augmentado por situação (não só por semântica genérica); (4) playbooks detalhados por categoria de empresa + tipo de situação.

A query anterior ("blocos com tag_ids órfãos: 1244") deu um falso positivo — os tag_ids são para olivia_tags (183 registros), não para assuntos. Os vínculos estão corretos. Mas o script de enriquecimento de playbooks pode ter tentado cruzar com a tabela errada.

❌ 1.353 jurisprudências com texto_integral vazio

A OlivIA usa a ementa para RAG, mas o texto_integral enriqueceria muito a busca. Antes de executar a Fase 3, verificar se o banco antigo tem esse campo preenchido — se não tiver, a expansão via Claude será necessária (custo de tokens).

⚠️ rclone scope readonly — upload Drive vai falhar

O remote gdrive foi configurado com scope drive.readonly. Para upload (Fase 2), é necessário reconfigurar com scope drive ou usar a Google Drive API diretamente via service account. Verificar antes de iniciar a Fase 2.

⚠️ 21 modelos sem RAG — verificar se têm conteudo_template

Antes de forçar o RAG, confirmar que esses 21 modelos têm conteudo_template preenchido. Se estiverem vazios, o RAG não vai indexar nada útil.

⚠️ Dataset de treinamento requer validação jurídica

Antes de usar o dataset para fine-tuning, o Christian precisa validar uma amostra — os blocos foram escritos pelo escritório e podem ter referências a casos específicos que não devem vazar para um modelo geral.
