🚨 Bugs P0 — Bloqueadores Críticos
P0
Select "Tipo de Petição" não abre — Enter navega para landing
Ao pressionar Enter com o Radix Select do "Tipo de Petição" em foco, a aplicação navega de volta para a landing /olivia em vez de abrir o dropdown. O componente Radix Select usa Portal — o click direto no SelectTrigger não dispara a abertura. O evento Enter propaga para cima e aciona algum listener de navegação não esperado.
⚡ Impacto: Impossível avançar para Step 2. Bloqueia 100% do fluxo Express.
Causa provável: Falta de type="button" no SelectTrigger ou ausência de e.stopPropagation() / e.preventDefault() no handler do wizard. O Enter está sendo capturado por um listener externo que interpreta como "voltar".

Fix: Verificar se há um onKeyDown global no wizard container (OliviaGeradorExpress) que captura Enter. Adicionar e.stopPropagation() no SelectTrigger wrapper, ou adicionar onKeyDown={(e) => e.stopPropagation()} no div container do Select.
P0
Radix Select não responde a click programático
O SelectTrigger do "Tipo de Petição" não abre ao receber .click() programático (testado via evaluate e via snapshot ref). Possivelmente o componente exige pointer event real ou há algum overlay bloqueando. Em teste manual (usuário real clicando) isso pode funcionar, mas indica fragilidade do componente.
⚡ Impacto: E2E automatizado quebra completamente neste ponto. Testes Playwright também falharão.
Fix: Garantir que o SelectTrigger tenha data-testid="tipo-peticao-trigger" para testes. Validar se o componente funciona com click real no browser (usuário comum). Se funcionar: problema é só no teste automatizado — adicionar data-testid e usar page.locator('[data-testid="tipo-peticao-trigger"]').click() no E2E.
