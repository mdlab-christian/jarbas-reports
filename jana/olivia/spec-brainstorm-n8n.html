💡 Insights & Decisões de Arquitetura
PADRÃO — Response

Respond 202 sempre que a operação for async

Análise e Geração já usam esse padrão. Manter obrigatoriamente. Frontend faz polling em olivia_analises / olivia_historico. Nunca bloquear o webhook esperando IA responder.

QUALIDADE — Extração

Triple fallback na extração de documentos

PDF text extract → Gemini Vision → Mistral OCR. Threshold 30 chars para acionar Mistral. Esse pipeline está maduro e deve ser preservado integralmente.

IA — Prompt

System prompt por tipo de análise (PROVAS vs PROCESSO)

Análise V2 já tem 2 system prompts distintos no Preparar Input Claude. Preservar e documentar no SPEC N8n. O prompt PROVAS inclui legislação consolidada (CDC art. 43, Súmula 385/479/359 STJ).

FRONTEND — Integração

Contrato de dados obrigatório para conectar com frontend

Cada webhook deve ter: request_id, organizacao_id, actor_user_id. Output salvo no Supabase deve ter schema definido e estável. Mudar output quebra o frontend — documentar no SPEC.

MANUTENÇÃO

Sticky Notes obrigatórios na reconstrução

Cada seção do workflow deve ter Sticky Note explicando: o que faz, qual tabela usa, qual IA chama, o que retorna. Facilita revisão por Christian e manutenção futura.

SEGURANÇA

Usar variáveis de ambiente n8n, não hardcode

Na reconstrução: SUPA_URL, SUPA_KEY, ANTHROPIC_KEY, GEMINI_KEY → variáveis de ambiente n8n (Settings > Variables). Código mais limpo e sem chaves expostas no JSON do workflow.
