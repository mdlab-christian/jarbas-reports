O que é "Pixel Agents"?

🎮 Pixel Agents — Origem do Conceito

O Pixel Agents é originalmente uma extensão para VS Code que cria um escritório virtual em pixel art onde seus agentes do Claude Code ganham vida como personagens. Cada agente vira um bonequinho que anda pela tela, senta na mesa quando está trabalhando, vai para o break room quando termina.

O conceito se tornou viral recentemente e gerou dezenas de repositórios fork/inspirados no GitHub — muitos deles adaptados para Mac, standalone (Electron), ou browser-based.

⚠️ Original: VS Code / Windows-friendly ✅ Forks: funciona em Mac 🔵 Conceito: Claude Code hooks + WebSocket + Canvas
💡 Como funciona tecnicamente: usa os hooks do Claude Code (PreToolUse, PostToolUse, Stop) para capturar eventos em tempo real. Os eventos são enviados via WebSocket para um servidor local, que renderiza os agentes na tela. Funciona em qualquer OS — o pixel art roda no browser.

Seus Agentes Claude Code

📊 JANA + JARBAS: 69 agentes cada

Encontrei 69 agentes em ~/.claude/agents/ tanto no MacBook Air (JANA) quanto no Mac Mini (JARBAS). São idênticos — sincronizados.

❓ "Por que não escolho qual agente usar?"

Esses arquivos em ~/.claude/agents/ não são agentes que você controla manualmente — são sub-agentes especializados que o Claude Code pode invocar autonomamente. Quando você manda uma tarefa, Claude decide internamente se delega para um sub-agente especializado (ex: mdflow-frontend-specialist, supabase-ops) ou faz ele mesmo. Você não escolhe — o modelo decide.

O Pixel Agents / The Forge visualiza exatamente isso: quando Claude Code spawna sub-agentes, você vê os personagens aparecendo na tela!

Dá pra fazer no Lovable?

🔵 Lovable — análise

Lovable gera web apps React. Dá sim, mas com limitações importantes:

✅ O que vai bem Interface visual bonita, dashboard com cards de status, gráficos, kanban. Tudo que é "frontend de visualização".
⚠️ Limitação crítica Lovable não conecta ao backend local (Mac). Você precisaria de uma API/WebSocket server rodando no seu Mac para o Lovable consumir. Não é self-hosted simples.
💡 Melhor alternativa Usar Claude Code para gerar o frontend React + Node.js backend local. Controle total, sem depender do Lovable. Mais rápido e integrado com seu stack atual.
⚠️ Veredicto: Lovable é ótimo para protótipos visuais, mas para uma ferramenta que precisa se comunicar com processos locais (Claude Code, Supabase, agentes), melhor fazer com Claude Code direto. Mais controle, mesma velocidade.

💡 Resposta sobre VS Code / Windows

O Pixel Agents original SIM funciona em Mac se você usar VS Code no Mac — mas os forks (The Forge, Backstage) são melhores e não precisam de VS Code. Windows não é obrigatório para nenhuma das opções acima.

Resumo rápido:

  • Pixel Agents original → VS Code (qualquer OS)
