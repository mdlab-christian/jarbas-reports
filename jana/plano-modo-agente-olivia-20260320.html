02 — Bugs Encontrados no Teste Manual

🔴
BUG-001 — Tipo de Petição: dropdown abre sem opções [P0]
Causa raiz: sessão Supabase expirada (Invalid Refresh Token) bloqueando query de tipos. Solução: relogar + corrigir refresh token handling. Secundário: verificar RLS da tabela olivia_tipos_peticao.
🔴
BUG-002 — olivia_historico queries falhando 400 [P0]
Filtro deleted_at=is.null → 400. Causa: coluna deleted_at pode não existir em olivia_historico e processos_restricoes. Verificar migration + aplicar se ausente.
🟡
BUG-003 — advogados query retorna 406 [P1]
Query com user_id=eq.X retorna 406 (Not Acceptable). Provavelmente .single() em query que retorna múltiplos. Verificar RLS + corrigir para .maybeSingle() ou limit=1.
🔴
BUG-004 — Upload de documentos: RLS violation [P0]
Drawer de documentos: new row violates row-level security policy. Causa: INSERT sem user_id no payload, ou policy de Storage bucket bloqueando. Verificar: (1) payload do upload inclui user_id e org_id; (2) policy INSERT na tabela de docs; (3) Storage bucket olivia-provas com policy correta.
🔴
BUG-005 — Step 2: OlivIA/JARBAS indisponível [P0]
Toast "JARBAS indisponível — use modo Express" imediato. Webhook /olivia/gerar-modo-agente responde 405 no GET (endpoint existe) mas não processa POST corretamente, ou skill_olivia_agente_v3.mjs não está registrado no servidor de webhook do JARBAS. Com a migração para OlivIA, esse bug se resolve pela nova rota.
🔴
BUG-006 — Step 4 trava em "Redigindo petição final..." [P0]
Nenhuma execução de longa duração no n8n. O workflow não dispara ou falha silenciosamente antes de iniciar. Frontend fica em polling eterno. Causa: provavelmente o payload está incorreto ou o servidor de webhook não registrou a skill v3. Resolvido pela migração para OlivIA Skills.

03 — Nova Arquitetura: Modo Agente com OlivIA

💡
Princípio mantido: Antes de gerar qualquer petição, OlivIA SEMPRE executa internamente o fluxo Smart Blocks (tags → blocos → jurisprudências → placeholders) como base anti-alucinação. A petição final aprimora essa base com análise das provas e contexto do caso.

Fluxo Completo do Modo Agente

FRONTEND (React) — /olivia/gerador/modo-agente
        │
        ▼ Step 1 (GeradorStep1 compartilhado)
  Seleciona: processo + tipo petição + modelo + versão + qualidade IA
  Faz upload de documentos (drawer)
  Toggle: Gerar relatório HTML = SIM (padrão no Modo Agente)
        │
        ▼ Step 2 — Análise Profunda
  POST https://olivia.jarbas-mdlab.com/analise-profunda    ← NOVO endpoint OlivIA
  OlivIA recebe e executa via skill-analise:
    1. Carrega contexto completo do processo (Supabase)
    2. Analisa documentos enviados (provas)
    3. Busca jurisprudências via RAG (fn_rag_search)
    4. Gera Smart Blocks internamente (anti-alucinação)
    5. Retorna: { relatorio, perguntas_triagem, smart_blocks_base }
  Frontend: polling olivia_historico.progresso (2s) + progress bar contextual
        │
        ▼ Step 3 — Editor + Chat OlivIA
  Editor 65% | Chat OlivIA 35%
  Chat conectado à instância @olivia_mdbot via OpenClaw
  Thread ID = processo_id → continuidade de contexto
  Usuário responde perguntas + conversa com OlivIA
  Tudo que é dito no chat → contexto para geração
        │
        ▼ Step 4 — Geração
  POST https://olivia.jarbas-mdlab.com/gerar-peticao       ← NOVO endpoint OlivIA
  OlivIA executa via skill-gerar-peticao:
    1. Usa smart_blocks_base do Step 2 como referência
    2. Incorpora contexto do chat do Step 3
    3. Gera petição final com Claude Sonnet/Opus
    4. Score revisor automático (skill-revisor)
    5. Salva em Google Doc + Supabase
  Frontend: polling 3s + substeps animados
        │
        ▼ Step 5 — Revisão Final
  Preview da petição + Score IA (badge colorido)
  Botão Google Doc + PDF
  Botão "Editar" → volta Step 3
        │
        ▼ Step 6 — Conclusão
  Links (Doc, PDF, Relatório HTML)
  Métricas (tokens, custo, tempo, modelo)
  Feedback advogado (1-10 estrelas + comentário)
  Botão "Gerar outra" → nova aba
