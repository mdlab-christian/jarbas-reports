01 — Agente dentro da OlivIA ou só Skills?

🤔
Sua dúvida: "é um workflow que utiliza várias skills em sequência — tem que criar algum agente dentro da OlivIA pra isso?"

✅ Resposta: Skills + Orquestração nativa

A OlivIA já é o agente. O OpenClaw permite que uma skill chame outra skill (sub-skill), ou que a OlivIA decida internamente qual skill usar em cada etapa.

O workflow inteiro fica dentro da skill principal (skill-gerar-peticao) que orquestra as demais em sequência:

skill-gerar-peticao
  → chama: skill-analise-processo
  → chama: skill-smart-blocks (base anti-alucinação)
  → chama: skill-analise-provas (se tiver docs)
  → chama: skill-especialista-[categoria] (playbook)
  → gera petição final com Claude
  → chama: skill-revisor
  → chama: skill-criar-doc (Google Drive)

Não precisa de "agente dentro de agente" — a OlivIA executa skills em sequência como um pipeline.

02 — Workflow Completo da OlivIA (substitui n8n + GAS integralmente)

💡
Google Apps Script eliminado: O GAS era usado para criar o Google Doc com formatação correta. A OlivIA vai criar o Doc diretamente via Google Docs API com OAuth service account. Mesma pasta de destino (google_drive_folder_id do processo), mesmo formato. Zero dependência de GAS.

Pipeline completo — inspirado no Express + SmartBlocks + Agente v3

skill-analise-provas (opcional, só se provas_urls não vazio) — classifica documentos do réu por tipo → busca blocos de impugnação → UPDATE progresso 50%

06 — Cloudflare Tunnel: Guia Passo a Passo

🔧
O JARBAS já tem o tunnel jarbas.jarbas-mdlab.com → localhost:3012. A OlivIA roda na porta 18793 no Mac Mini. Precisa de um tunnel separado.

Pré-requisito: cloudflared instalado no Mac Mini

ssh mac-mini "which cloudflared"  # verificar se existe
ssh mac-mini "cloudflared --version"

Se não existir: ssh mac-mini "brew install cloudflared"

Passo 1 — Autenticar no Cloudflare (se ainda não autenticado)

ssh mac-mini "cloudflared tunnel login"
# Abrirá URL no terminal — cole no browser e autorize
# Credencial salva em ~/.cloudflared/cert.pem

Passo 2 — Criar o tunnel

ssh mac-mini "cloudflared tunnel create olivia-tunnel"
# Saída: Tunnel credentials written to ~/.cloudflared/[UUID].json
# Anote o UUID do tunnel — vai precisar

Passo 3 — Criar arquivo de configuração

ssh mac-mini "cat > ~/.cloudflared/olivia-config.yml << 'EOF'
tunnel: [UUID-DO-TUNNEL]
credentials-file: /Users/olivia/.cloudflared/[UUID-DO-TUNNEL].json

ingress:
  - hostname: olivia.jarbas-mdlab.com
    service: http://localhost:18793
  - service: http_status:404
EOF"

⚠️ Substituir [UUID-DO-TUNNEL] pelo UUID gerado no Passo 2. O arquivo de credentials fica em /Users/jarbas/.cloudflared/ por padrão — copiar para /Users/olivia/.cloudflared/.

Passo 4 — Criar registro DNS na Cloudflare

ssh mac-mini "cloudflared tunnel route dns olivia-tunnel olivia.jarbas-mdlab.com"
# Cria automaticamente o CNAME no DNS da zona jarbas-mdlab.com

Verificar no dashboard Cloudflare → DNS → deve aparecer: olivia.jarbas-mdlab.com CNAME [UUID].cfargotunnel.com

Passo 5 — Instalar como serviço (rodar automaticamente)

ssh mac-mini "sudo cloudflared service install --config /Users/olivia/.cloudflared/olivia-config.yml"
ssh mac-mini "sudo launchctl load /Library/LaunchDaemons/com.cloudflare.cloudflared.plist"

Ou rodar em foreground para testar antes:

ssh mac-mini "cloudflared tunnel --config ~/.cloudflared/olivia-config.yml run"

Passo 6 — Verificar que funciona

curl -s https://olivia.jarbas-mdlab.com/health
# Deve retornar 200 (o OpenClaw responde /health por padrão)
✅
Se retornar 200, o tunnel está funcionando. Atualizar OLIVIA_BASE_URL no frontend para https://olivia.jarbas-mdlab.com.

07 — Google Drive Direto (sem GAS)

Como a skill-criar-doc vai funcionar

O GAS (Google Apps Script) era o intermediário para criar docs no Drive. A OlivIA vai fazer isso diretamente via API. Duas opções:

OpçãoComoVantagemComplexidade
Service Account (recomendado)Service account com acesso à pasta do Google Drive do escritório. Credenciais JSON no workspace OlivIA.Sem OAuth flow, funciona 24/7, sem expiraçãoBaixa — uma vez configurado, funciona sempre
OAuth2 com refresh tokenAutorizar uma vez, salvar refresh token, renovar automaticamenteAcessa a conta Google diretamenteMédia — precisa renovar refresh token periodicamente
⚠️
Ação necessária: Criar uma Service Account no Google Cloud Console e dar permissão de Editor na pasta do Drive onde ficam as petições. Salvar o JSON da service account no workspace OlivIA.

Fluxo da skill-criar-doc:

1. Autenticar via service account (GOOGLE_SERVICE_ACCOUNT_JSON)
2. Criar Google Doc vazio na pasta google_drive_folder_id do processo
3. Inserir conteúdo via batchUpdate (seções formatadas)
4. Substituir placeholders {{cliente_nome}}, {{reu_nome}}, etc.
5. Aplicar formatação: título H1, seções H2, jurisprudências em recuo
6. Retornar: { google_doc_url, google_doc_id }
7. Opcional: compartilhar o doc com o email do advogado
