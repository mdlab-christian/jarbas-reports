05 — Substituição GAS → Docs API com Formatação Preservada

⚠️
Ponto crítico: O GAS copiava a formatação exata do bloco Google Doc (negrito, itálico, recuo, estilos de parágrafo, tabelas, imagens). A substituição deve ter paridade total — senão as petições perdem qualidade.

Problema: 3 tipos de conteúdo nos blocos

📝
Tipo 1: Texto puro (campo conteudo)
Blocos sem google_doc_id. Fácil — usar o texto direto. Sem formatação especial.
📄
Tipo 2: Google Doc com texto formatado
Bloco tem google_doc_id. Precisa preservar: negrito, itálico, sublinhado, recuos, tamanho de fonte, parágrafos estilizados.
🖼️
Tipo 3: Google Doc com imagens/prints
Blocos de jurisprudência ou extratos bancários com imagens incorporadas. Precisa copiar as imagens também.

Solução: Docs API estrutura + Drive Copy

1️⃣
Para blocos simples (sem imagem)
drive.files.export({ mimeType: 'application/vnd.openxmlformats-officedocument.wordprocessingml.document' }) → importar como DOCX no Doc final. Preserva toda formatação Rich Text via Docs API batchUpdate.
2️⃣
Para blocos com imagens (melhor opção)
Usar Drive API files.copy para copiar o bloco como doc separado, depois inserir como conteúdo inline via docs.documents.batchUpdate com InsertInlineImageRequest para cada imagem. Alternativa mais simples: exportar como HTML (inclui imagens como base64) e inserir no Doc final.
3️⃣
Abordagem recomendada (mais simples e confiável)
Criar o Doc final via upload DOCX: montar o documento inteiro como DOCX em memória (usando biblioteca docx no Node.js), incorporando os blocos com formatação preservada, depois fazer drive.files.create convertendo para Google Doc (mimeType: 'application/vnd.google-apps.document').

Fluxo técnico da skill-criar-doc (v3)

async function criarDocModoAgente({ titulo, template_html, blocos, campos, google_folder_id, auth }) {
  // 1. Para cada bloco COM google_doc_id → buscar estrutura completa via Docs API
  const blocosResolvidos = await Promise.all(blocos.map(async (bloco) => {
    if (bloco.google_doc_id) {
      // Exportar como HTML (preserva texto formatado + imagens como base64)
      const html = await drive.files.export({
        fileId: bloco.google_doc_id,
        mimeType: 'text/html',
        auth
      });
      return { ...bloco, html_resolvido: html.data };
    }
    return { ...bloco, html_resolvido: `<p>${bloco.conteudo}</p>` };
  }));

  // 2. Substituir placeholders no template HTML
  let htmlFinal = template_html;
  for (const bloco of blocosResolvidos.sort((a,b) => a.ordem - b.ordem)) {
    const placeholder = bloco.placeholder || `{{BLOCO_${bloco.ordem}}}`;
    htmlFinal = htmlFinal.replace(placeholder, bloco.html_resolvido);
  }

  // 3. Substituir campos dinâmicos
  for (const [key, value] of Object.entries(campos)) {
    htmlFinal = htmlFinal.replaceAll(`{{${key}}}`, String(value ?? ''));
  }

  // 4. Criar Google Doc via Drive API (conversão HTML → Google Doc)
  const { data } = await drive.files.create({
    requestBody: {
      name: titulo,
      mimeType: 'application/vnd.google-apps.document',
      parents: [google_folder_id]
    },
    media: {
      mimeType: 'text/html',
      body: htmlFinal
    },
    fields: 'id, webViewLink',
    auth
  });

  return { google_doc_id: data.id, google_doc_url: data.webViewLink };
}
⚠️
Atenção imagens: A exportação HTML do Google Doc inclui imagens como URLs temporárias (https://lh3.googleusercontent.com/...) com autenticação. Para preservar no Doc final, usar a API Docs batchUpdate com InsertInlineImageRequest referenciando essas URLs — elas são acessíveis com o mesmo token OAuth.
