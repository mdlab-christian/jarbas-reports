P3 — Criar skill olivia-chat.mjs (Chat OlivIA)

Backend (Mac Mini) ~30 min ▶ 
Crie o arquivo `/Users/Shared/olivia-skills/olivia-chat.mjs` — skill de chat conversacional da OlivIA.

## Contexto
- NUNCA usar fetch() — usar apenas require("https") ou o helper httpsPost já padrão nas outras skills
- Ver padrão em olivia-analisa-provas.mjs para referência de estrutura
- Env vars: SUPABASE_URL, SUPABASE_KEY, ANTHROPIC_API_KEY (lidas de /Users/olivia/.env ou ~/jarbas/.env)

## Contrato de entrada (payload do server.mjs)
```javascript
{
  threadId: string,       // ID da thread no Supabase (olivia_threads)
  message: string,        // Mensagem do usuário
  organizacaoId: string,  // organizacao_id
  userId: string,         // actor_user_id
}
```

## O que a skill deve fazer

### 1. Buscar histórico da thread
```
GET olivia_messages?thread_id=eq.{threadId}&order=created_at.asc&limit=20
```
Filtrar apenas role = 'user' | 'assistant'.

### 2. Buscar dados do processo (se thread tiver processo_id)
```
GET olivia_threads?id=eq.{threadId}&select=processo_id,titulo
GET processos?id=eq.{processo_id}&select=numero_cnj,cliente_nome,restricao_valor,restricao_orgao_nome
```

### 3. Montar system prompt
```
Você é OlivIA, assistente jurídica da MdLab especialista em direito do consumidor brasileiro.
{se processo: "Contexto do processo: {numero_cnj} — {cliente_nome} vs {reu} — Restrição: R$ {valor} em {orgao}"}
Responda de forma técnica, clara e objetiva em português jurídico.
Seja concisa mas completa. Máximo 400 palavras por resposta.
```

### 4. Chamar Claude Sonnet
- model: claude-sonnet-4-6
- max_tokens: 32000
- messages: histórico convertido + nova mensagem do usuário

### 5. Salvar resposta em olivia_messages
```
POST olivia_messages
{
  organizacao_id: organizacaoId,
  thread_id: threadId,
  role: 'assistant',
  content: respostaClaude,
  metadata: { model: 'claude-sonnet-4-6', skill: 'olivia-chat' }
}
```

### 6. Atualizar updated_at da thread
```
PATCH olivia_threads?id=eq.{threadId}
{ updated_at: new Date().toISOString() }
```

### 7. Retornar
```javascript
{ success: true, messageId: 'uuid-da-mensagem-salva', content: respostaClaude }
```

## Tratamento de erro
Se qualquer etapa falhar: logar erro, retornar { success: false, error: mensagem }.

## Modo CLI para teste
```javascript

  executar({ threadId: process.argv[2], message: process.argv[3] || 'Olá', organizacaoId: '...', userId: '...' })
```

P4 — Atualizar rota /olivia/chat no server.mjs

Backend (Mac Mini) ~10 min ▶ 
No arquivo `/Users/Shared/olivia-server/server.mjs`, a rota POST /olivia/chat atualmente espera `historico_id + mensagem` mas o frontend envia `data.thread_id + data.message + data.organizacao_id + data.actor_user_id`.

## Mudança necessária
Substituir completamente o bloco da rota /olivia/chat (linhas ~810-882) por:

```javascript
if (req.method === 'POST' && path === '/olivia/chat') {
  const data = body.data || body;
  const threadId = data.thread_id;
  const message = data.message;
  const organizacaoId = data.organizacao_id || body.organizacao_id;
  const userId = data.actor_user_id || body.actor_user_id;

  if (!threadId || !message) {
    return json(res, 400, { ok: false, error: 'thread_id e message obrigatorios' });
  }

  try {
    const { executar } = await import(SKILLS_DIR + '/olivia-chat.mjs');
    const result = await executar({ threadId, message, organizacaoId, userId });
    if (result.success) {
      return json(res, 200, { ok: true, success: true, message: 'Mensagem processada' });
    } else {
      return json(res, 500, { ok: false, error: result.error });
    }
  } catch (e) {
    console.error('[olivia-chat] Erro:', e.message);
    return json(res, 500, { ok: false, error: e.message });
  }
}
```
