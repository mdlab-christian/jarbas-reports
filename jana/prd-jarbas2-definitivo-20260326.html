12. As 10 Regras Invioláveis (do Curso OpenClaw)

1
Crons: sempre isolated + agentTurn + announce
systemEvent + main dispara mas não executa (durationMs ~0ms). Esse bug travou crons por 3 dias numa produção real.
2
NUNCA hardcodar credenciais — tudo no .env
Um vazamento = todas as contas comprometidas. .env com chmod 600 é a última linha de defesa. Nunca em SOUL.md ou arquivos de identidade.
3
dmPolicy: allowlist desde o Dia 1
Com dmPolicy "open", qualquer pessoa que encontrar o bot pode comandar o agente — ler arquivos, acessar APIs, enviar mensagens.
4
Extrair lições ANTES de cada compactação
Lessons → decisions → pending → NUNCA pular. Compactação descarta 80% do contexto. Sem extração prévia, informações críticas se perdem para sempre.
5
Todo agente novo começa L1 (Observer)
Nenhum agente ganha autonomia sem histórico. Promoção via performance review semanal: L1→L2→L3→L4.
6
Split de modelos — Sonnet/Haiku/Opus por contexto
Haiku para heartbeats (~$0.005). Sonnet para crons/execução (~$0.02). Opus só para interação estratégica. Economia de 75-80%.
7
Backup antes de mudanças estruturais
Antes de criar agentes, modificar gateway config, reorganizar workspace: salvar config + ROLLBACK.md. config.patch reinicia gateway e mata crons.
8
Sub-agent travou → retry 2x → avisar humano
NUNCA fire and forget / limbo silencioso. Sucesso = resumo. Falha = retry 2x. Falhou 2x = alerta imediato.
9
SOUL.md genérico = agente genérico
Anti-patterns com exemplos ❌/✅. "Never dos" explícitos. Contexto profundo no USER.md. A diferença entre chatbot e COO é o SOUL.
10
Creators são skills, não agentes separados
1 agente com N skills > N agentes especializados. Menos custo, menos coordenação, menos cold starts, contexto compartilhado.
