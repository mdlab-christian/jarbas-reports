list Lista todos os projetos indexados com contagem de docs e chunks. Tabela: projeto / docs / chunks / última atualização

🔗 Como conectar na revisão em loop (para quando for usar)

Você perguntou onde conectar o RAG. A resposta: no prompt inicial do CC, não numa skill separada. O CC passa a ter uma "instrução base" de sempre consultar o RAG antes de agir.

❌ Sem RAG (hoje)

# Prompt do CC Lê os arquivos: - ~/Desktop/olivia v3/OLIVIA-V3-MASTER-PLAN.md - ~/Desktop/olivia v3/T2-prd-frontend.md - ~/Desktop/olivia v3/T2-prd-frontend-ux-addendum.md - ~/Desktop/olivia v3/T3-prd-backend.md [... mais 4 arquivos ...] Agora revise o componente...

~150k tokens consumidos só lendo spec. Contexto quase cheio antes de começar.

✅ Com RAG

# Prompt do CC Antes de revisar qualquer componente, consulte: node ~/.openclaw/workspace-jana/skills/rag-projeto/scripts/rag_projeto.mjs \ search "[componente que vai revisar]" \ --projeto olivia-v3 Use APENAS os trechos retornados como spec. Não leia arquivos de spec diretamente.

~2-4k tokens por consulta. Contexto livre para código + screenshots + fixes.

Onde fica o RAG na skill de revisão

Quando você pedir a revisão em loop, JANA vai gerar um prompt de CC que inclui esta instrução automática. A skill de revisão vai chamar o RAG internamente, como parte do fluxo. Você não precisa fazer nada manual.

## Fluxo da revisão em loop (futuro, quando você pedir) Para cada componente da pasta src/pages/olivia/: 1. CC consulta RAG → "spec do [componente]" 2. CC lê o arquivo .tsx atual 3. CC compara: código real vs spec RAG 4. Se divergência → corrige + registra em projeto_etapas 5. Se conforme → registra "ok" + passa pro próximo 6. Screenshot + vision nos componentes visuais 7. Repete até todos os componentes revisados
