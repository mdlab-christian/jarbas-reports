# Fix #6 + #9 — Imports não utilizados e useEffect deps incompletas

## Contexto
Projeto: MdFlow — ~/beta-mdflow/
Arquivos alvo (todos os 4 wizards):
- src/pages/playbooks/gerador/wizards/WizardNovaVersao.tsx
- src/pages/playbooks/gerador/wizards/WizardCriarBloco.tsx
- src/pages/playbooks/gerador/wizards/WizardCriarModelo.tsx
- src/pages/playbooks/gerador/wizards/WizardTermosJuris.tsx

## Tarefa 1 — Remover useCallback dos imports
Em todos os 4 arquivos, remover `useCallback` do import do React
(não é utilizado em nenhum deles):
```ts
// antes:
import { useState, useEffect, useCallback } from 'react';
// depois:
import { useState, useEffect } from 'react';
```

## Tarefa 2 — Corrigir deps dos useEffects de webhook
Os useEffects que disparam execWebhook têm arrays de deps incompletos.

ATENÇÃO: A intenção do design é que o webhook só dispare UMA VEZ ao entrar no step
de geração (step 2 ou 3, dependendo do wizard). O padrão de controle usado é:
`!generated && !generating && !webhookFailed` — isso previne re-disparos.

Portanto a correção correta é usar `useRef` para estabilizar as variáveis
que mudam entre renders (execWebhook via useCallback, ou capturar os valores
no momento do disparo com refs).

Opção recomendada: envolver o useEffect num `useRef` de "já executou" ou
adicionar `// eslint-disable-next-line react-hooks/exhaustive-deps` com comentário
explicativo se o comportamento intencional é executar apenas uma vez.

Avaliar cada wizard e aplicar a solução mais conservadora que não altere
o comportamento atual (disparo único por step).

## Restrições
- NÃO alterar o fluxo de geração (deve continuar disparando uma única vez)
- NÃO modificar o payload enviado ao webhook
- NÃO alterar WizardShell.tsx ou outros arquivos
- Se a correção de deps causar re-disparos inesperados → usar eslint-disable com comentário
