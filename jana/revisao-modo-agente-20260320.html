02 — Bugs Ativos (3 encontrados)

🔴
BUG-001 — Tipo de Petição: dropdown abre mas sem opções [P0]
Ao clicar no combobox "Selecione o tipo" após selecionar processo, o dropdown abre mas não exibe nenhuma opção. Suspeita: sessão Supabase expirada (console mostra Invalid Refresh Token: Refresh Token Not Found) bloqueando a query de tipos. A query advogados também retorna 406.
P0 Sessão expirada → relogar
🔴
BUG-002 — olivia_historico queries falhando 400 [P0]
Console tem ~15+ erros 400 nas queries olivia_historico. Causas prováveis: (1) sessão expirada — token inválido; (2) coluna deleted_at não existe na tabela. As stats da landing mostram 0 por isso. Verificar se a migration do deleted_at foi aplicada corretamente na tabela olivia_historico.
P0 Verificar migration + relogar
🟡
BUG-003 — advogados query retorna 406 Not Acceptable [P1]
Query advogados?select=id,nome,oab,oab_uf&...&user_id=eq.X retorna 406. Isso pode causar que o advogadoId não seja propagado corretamente para o wizard, potencialmente bloqueando a geração. Checar RLS da tabela advogados para o user_id atual.
P1 RLS policy issue

03 — Pontos de Atenção (UX/Design)

💡
P1 — "Modo Agente" no header: subtítulo diz "Geração profunda via JARBAS" (ok mas incompleto)
Antes de entrar, o card da landing diz "Geração profunda com JARBAS + Claude Opus". Depois de entrar no modo, o subtítulo simplifica para apenas "Geração profunda via JARBAS". Seria mais coerente manter "JARBAS + Claude Opus" para deixar claro o que está rodando por baixo.
💡
P2 — "Gerar relatório HTML" no Step 1: padrão deveria ser "Sim (JARBAS)"
Atualmente o padrão selecionado visualmente é "Não" (verde). Para o Modo Agente, onde o JARBAS sempre gera um relatório detalhado, faz mais sentido deixar "Sim (JARBAS)" como padrão ativo. O usuário que não quer relatório vai desativar.
💡
P2 — Processo card: "Evento não gerou documento" em destaque desnecessário
No card do processo selecionado, o campo "Evento não gerou documento" aparece com ícone de localização (⊙) mas sem destaque visual negativo. Para o advogado, essa info pode ser importante (significa que não há doc judicial recente). Considerar badge amarelo ou ícone de alerta.
💡
P3 — Step 1 do Agente não tem label "Modo Agente" diferenciado do Express
O Step 1 do Modo Agente e do Express usam exatamente o mesmo componente GeradorStep1. Visualmente, depois que o usuário entra no modo, a única diferença é o header "Modo Agente". Seria útil ter um pequeno indicador contextual dentro do step (ex: chip "Modo Agente ativa" ou bordas coloridas diferentes por modo).
