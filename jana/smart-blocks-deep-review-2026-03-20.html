P0 BUG-01 versaoId ≠ olivia_modelos_peticao.id — FK não verificado em produção

Arquivo: OliviaGeradorSmartBlocks.tsx → useSmartBlocks.ts

Problema: O fix da Sala 8 passou state.versaoId como modeloPeticaoId assumindo que olivia_versoes.id == olivia_modelos_peticao.id. Mas a FK confirmada é:

olivia_modelo_tags.modelo_id → olivia_modelos_peticao.id  (tabela de peticoes)
olivia_versoes → sem FK explícita para olivia_modelos_peticao  (verificado via information_schema)

Se os IDs não forem os mesmos, a query de tags retornará 0 resultados em produção — o wizard fica em branco sem erro visível.

Causa raiz: As tabelas olivia_modelos_peticao (329 rows, 23 cols) e olivia_versoes (329 rows, 10 cols) são 1:1, mas a relação entre olivia_versoes.id e olivia_modelos_peticao.id não foi confirmada. Precisam ter o mesmo UUID para o fix funcionar.

Risco: Se forem IDs diferentes (ou seja, olivia_versoes tem seu próprio UUID e olivia_modelos_peticao tem outro), o bug original P0 ainda existe.

P1 BUG-03 olivia_modelo_tags.obrigatoria ignorada — usa somente tag-level default

Arquivo: useSmartBlocks.ts linhas 159–178 (processamento de tags)

Problema: O schema de olivia_modelo_tags tem coluna obrigatoria (boolean) que representa a obrigatoriedade específica do modelo. O hook atual usa somente t.selecao_obrigatoria da olivia_tags, que é o default global da tag.

// ATUAL — usa apenas o valor global da tag
selecao_obrigatoria: (t.selecao_obrigatoria as boolean) ?? false,

// CORRETO — usar override do modelo quando disponível
selecao_obrigatoria: (row.obrigatoria !== undefined && row.obrigatoria !== null)
  ? (row.obrigatoria as boolean)
  : ((t.selecao_obrigatoria as boolean) ?? false),

Impacto: tags podem ser opcionais globalmente mas obrigatórias num modelo específico (ou vice-versa), e isso é ignorado.

🔧 PROMPT CC — BUG-03 FIX
Arquivo: src/pages/olivia/hooks/useSmartBlocks.ts

TAREFA: Usar olivia_modelo_tags.obrigatoria como override de selecao_obrigatoria da tag.

Na função de processamento de tags (linha ~159), alterar o mapeamento:

// ANTES:

// DEPOIS — obrigatoria do modelo tem prioridade sobre o default da tag:
