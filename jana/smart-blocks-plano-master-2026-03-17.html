🟠 P1 — Alto (bugs funcionais)

P1 Frontend Step 2: blocos não carregam — race condition na cadeia de queries
SmartBlocksStep2Blocos.tsx / query blocosDb

A query de blocos tem enabled: !!organizacao_id && !!state.modelo_id && modeloTagsIds !== undefined. O problema: modeloTagsIds passa por cadeia de 3 queries: modeloGrupo → junctionTagIds → modeloTagsIds.

Se junctionTagIds ainda está em loading, modeloTagsIds = undefined → enabled=false → query de blocos não dispara. Em ambiente lento (Supabase cold start ou conexão instável), isso trava e o Step 2 fica vazio.

E2E confirmou: 0 checkboxes encontrados após selecionar modelo.

✅ Fix — SmartBlocksStep2Blocos.tsx

Atual: enabled: !!organizacao_id && !!state.modelo_id && modeloTagsIds !== undefined

Corrigir para: adicionar skeleton/loading state enquanto loadingJunction === true. Se junctionTagIds.isSuccess && data.length === 0, fazer fallback para modeloGrupo.tags_ids ou carregar todos os blocos.

Também: adicionar isLoading state visível no Step 2 para o usuário não achar que a tela travou.
