🔴 Crítico BUG-02 — Blocos duplicados no Step 2 Frontend + DB
Todos os blocos aparecem em pares (ou mais) no wizard. A query busca por tag_ids mas não filtra pela versão ativa do bloco (bloco_grupo_id → active_version_id). No banco: um bloco_grupo_id com 1244+ registros todos com version=1 — import histórico criou duplicatas.
Root cause (SmartBlocksStep2Blocos.tsx)
// Query atual — sem deduplicação por bloco_grupo_id:
.from('olivia_blocos_texto')
.select('id, titulo, conteudo, tag_ids')
.eq('organizacao_id', organizacao_id)
.is('deleted_at', null)
.eq('status', 'published')
// ← Falta: .eq('version', 1) OU deduplicar por bloco_grupo_id
Fix
// Opção A (Frontend): deduplicar após busca por bloco_grupo_id:
const uniqueMap = new Map();
allBlocos.forEach(b => {
const key = b.bloco_grupo_id || b.id;
if (!uniqueMap.has(key)) uniqueMap.set(key, b);
});
return Array.from(uniqueMap.values());

// Opção B (DB): query com DISTINCT ON bloco_grupo_id via RPC
🟡 Moderado BUG-03 — Preview de blocos renderiza HTML bruto Frontend
O preview do bloco exibe tags HTML como texto (<p><span>...</span></p>) porque usa renderização de texto puro. Alguns blocos têm conteúdo em HTML (importados de Google Docs). A UX fica ilegível.
Root cause (SmartBlocksStep2Blocos.tsx, linha 481)
// PROBLEMA: renderiza como texto puro
{previewBloco?.conteudo}

// FIX: detectar e renderizar HTML quando necessário
{isHtml(previewBloco?.conteudo)
? <div dangerouslySetInnerHTML={{ __html: sanitize(previewBloco.conteudo) }} />
: <p className="whitespace-pre-wrap">{previewBloco.conteudo}</p>
}
🟡 Moderado BUG-04 — Versões indistinguíveis no Select (Step 1) Frontend
O dropdown de versão exibe todas as versões com o mesmo texto ("Instituição Financeira - Cartão de Crédito"). Confirmado no banco: 3 registros com nome idêntico, versões 1, 2 e 3. O código usa v.nome ?? `v${v.versao}` — como nome sempre está preenchido, nunca cai no fallback.
Root cause (SmartBlocksStep1Identificacao.tsx, linha 74)
// PROBLEMA: nome idêntico para todas as versões
{v.nome ?? (v.versao != null ? `v${v.versao}` : `Versão ${v.id.slice(0,8)}`)}

// FIX: sempre mostrar número de versão
{v.versao != null ? `${v.nome} (v${v.versao})` : v.nome}

1. Modelo de Dados olivia_modelos_grupo → olivia_modelos_peticao (versões) → olivia_modelo_tags (junction) → olivia_tags → olivia_blocos_texto (por bloco_grupo_id + version)
2. Fluxo do Wizard (3 steps) Step 1: buscar processo CNJ → selecionar tipo petição → selecionar modelo grupo → selecionar versão (com label v1/v2/v3) → advogado
Step 2: carregar tags do modelo (junction table) → buscar blocos por tag_ids → deduplicar por bloco_grupo_id → agrupados por seção → seleção radio por seção
Step 3: resumo → chamar webhook → polling olivia_historico → sucesso = link Google Doc
3. Workflow n8n (olivia/gerar-manual) Webhook → Validar → Buscar blocos full-text → Montar prompt com contexto do processo → Chamar Claude → Salvar texto → Chamar GAS (sw-create-google-doc) → Atualizar historico com google_doc_url
4. Google Apps Script (GAS v4.0) Copia modelo Google Doc → substitui {{BLOCO_N}} com conteúdo dos blocos selecionados (preservando formatação) → substitui {{tags}} com dados do processo → move para pasta → retorna URL
