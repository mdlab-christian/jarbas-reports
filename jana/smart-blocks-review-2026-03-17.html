⚠️ Pontos de atenção (não bloqueiam release)

⚠️ SUCESSO com google_doc_url nulo MÉDIO

O registro f154fb76 tem status=SUCESSO mas google_doc_url=null. Isso significa que o n8n marcou SUCESSO sem salvar a URL do Google Doc. O frontend mostra a tela de sucesso sem link — confuso para o usuário.

// Sugestão: n8n deve verificar se google_doc_url foi preenchido antes de setar SUCESSO
// Ou: frontend trata status=SUCESSO + url=null como "sem documento gerado" com aviso

⚠️ Registros PENDENTE travados (meta: {}) MÉDIO

4 registros recentes ficaram em PENDENTE indefinidamente com meta:{} — o frontend inseriu o registro mas o n8n não foi acionado ou ignorou. Sem fallback de timeout no banco para resetar PENDENTEs antigos.

-- Sugestão: cron job ou trigger no n8n para detectar PENDENTEs > 5min
-- UPDATE olivia_historico SET status='ERRO', error_message='Timeout sem processamento'
-- WHERE status='PENDENTE' AND created_at < NOW() - INTERVAL '5 minutes'

🔵 eslint-disable em useEffect de populate BAIXO

Em SmartBlocksStep2Blocos.tsx, o useEffect de população tem // eslint-disable-next-line react-hooks/exhaustive-deps para omitir tagsMap e setState das dependências. Funciona, mas pode causar populate desatualizado se tagsMap mudar depois que os blocos já foram carregados.

// Solução: useCallback para o populate, ou usar useMemo para derivar os blocos
// Baixo risco no uso atual porque tagsMap é carregado uma vez por sessão

🔵 Arquivo .bak no repositório BAIXO

OliviaGeradorSmartBlocks.tsx.bak está no diretório do projeto. Não é commitado (provavelmente no .gitignore), mas é ruído no filesystem.
