Sistema está rodando — JARBAS Node (PID 2516) e OpenClaw Gateway (PID 2688) estão ativos. O problema não é que o sistema parou — é que os arquivos de configuração ficaram desalinhados após o crash de contexto.

🎯 Causa Raiz do Crash

O JARBAS sofreu context overflow — muitas tasks simultâneas sobrecarregaram o contexto da sessão OpenClaw. O processo Node.js sobreviveu, mas o alinhamento entre os arquivos de identidade (SOUL.md, MEMORY.md) e o comportamento real do agente se perdeu.


Adicionalmente, existe uma confusão de pastas: ~/jarbas/ (projeto Node.js) e ~/.openclaw/workspace/ (workspace OpenClaw) são entidades distintas mas deveriam ser mais integradas.


Solução: Reconstruir os arquivos de persona pelo Claude Code com acesso direto ao filesystem, garantindo coerência entre os dois ambientes.

Problemas Identificados para Corrigir

P0 — Resposta incoerente no Telegram

JARBAS responde mas não executa comandos no Mac Mini. Sintoma: respostas genéricas sem usar tools de exec/shell.

✅ Fix: Reconstruir SOUL.md enfatizando que JARBAS TEM e DEVE usar exec tool liberalmente.

P0 — Contexto compartilhado entre conversas causando interferência

Quando múltiplas conversas (grupos + DM) estão ativas, tasks de uma interferem na fila da outra.

✅ Fix: Isolar filas por chat_id no task-consumer. Cada conversa tem sua própria fila de tasks independente.

P1 — Notificações de RAM com valores errados

Alerta de RAM disparando com valores incorretos ou threshold mal calibrado.

✅ Fix: skill_status.mjs → corrigir leitura de memória via `vm_stat` + calibrar threshold para 85% de uso real.

P1 — Custo tracker notificando a cada centavo

Notificações de custo chegando com muita frequência e valores mínimos.

✅ Fix: Threshold único de R$200 por ciclo. Relatório de custo consolidado só no resumo diário das 23h.

P2 — Grupos Telegram sem isolamento de contexto

JARBAS não está bem configurado para múltiplos grupos simultâneos com contextos independentes.

✅ Fix: Implementar context router — cada grupo/DM tem namespace de contexto isolado. Grupos autorizados têm acesso total como a DM.

P2 — Pastas ~/jarbas/ e ~/.openclaw/workspace/ desalinhadas

Dois ambientes com arquivos duplicados ou conflitantes causando confusão sobre qual é a "fonte da verdade".

✅ Fix: Claude Code mapeia tudo, elimina duplicações, cria symlinks onde necessário, define ~/jarbas/ como fonte de verdade jurídica/operacional e ~/.openclaw/workspace/ como workspace de agência.

Como funciona a orquestração

O Supabase age como message bus central. Cada Mac Mini tem uma coluna instance_id nas tasks. JARBAS Central distribui tasks para o node certo baseado em tipo (jurídico → #2, dev → #3, urgente → #1). Cada Mac Mini reporta heartbeat a cada 5min. Se um cai, as tasks são redirecionadas.
