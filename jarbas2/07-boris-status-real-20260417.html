O que é o Boris? — 3 módulos Sonnet 4.6

🧠 Boris Planner

Pré-geração · 349 linhas
  • Recebe tópicos + documentos + playbooks + tipo de petição
  • Retorna plano de seções + instruções cruzadas
  • Resolve perda de contexto em Promise.all paralelo
  • Trigger: topicos.length > 1 e modo geração
  • Modelo: Claude Sonnet 4.6
Status: código local · NÃO rodou em prod

🔍 Juris Validator

Pós-geração · 15 linhas (delegator)
  • Recebe peticao_html + citações
  • Valida semanticamente: confere se o acórdão suporta o argumento
  • Detecta alucinações de conteúdo (não só IDs)
  • Trigger: toda geração com jurisprudência
  • Modelo: Claude Sonnet 4.6
Status: código local · substituído por R-02

✂️ Boris Revisor

Adversarial · 333 linhas
  • Revisão condicional pós-geração
  • Dispara se score < 8 ou alucinação detectada
  • Foco dinâmico: 'citacoes' ou 'qualidade'
  • Retorna HTML revisado ou null
  • Modelo: Claude Sonnet 4.6
Status: código local · NÃO rodou em prod

Princípios arquiteturais

1 · Filho direto da Olivia — Boris roda no mesmo processo Node.js, user olivia UID 503. Zero overhead de rede ou processo separado.
2 · Silent degradation — qualquer erro/timeout do Boris retorna null. Pipeline continua sem alterações. Boris nunca bloqueia petição.
3 · Olivia 100% standalone (D-EST-01 Opção B) — Boris é opcional. Sistema funciona sem ele. Ativação via feature flag ENABLE_BORIS.

function validarJurisUsados(topicosGerados, jurisPayload) { const idsValidos = new Set((jurisPayload || []).map(j => j.id)); const todasCitadas = []; for (const topico of (topicosGerados || [])) { for (const id of (topico.juris_usados || [])) { todasCitadas.push(id); } } const naoReconhecidas = [...new Set(todasCitadas.filter(id => !idsValidos.has(id)))]; return { total_citadas: todasCitadas.length, total_validas: todasCitadas.length - naoReconhecidas.length, nao_reconhecidas: naoReconhecidas, score: todasCitadas.length > 0 ? +(totalValidas / totalCitadas).toFixed(2) : 1, }; } // grava em olivia_historico.meta.validacao_juris

06 ⚠️

Lacuna semântica — o que o R-02 não detecta

Casos que R-02 passa ✅ mas deveriam falhar ❌

Alucinações de conteúdo que R-02 não pega porque só valida IDs:

  • IA cita REsp 1.234.567/STJ (ID real no payload) mas descreve a ementa errada (trocou dano moral por dano material)
  • IA usa acórdão do TJRS sobre telefonia para embasar caso de energia elétrica (contexto errado)
  • IA cita súmula correta mas aplica-a no sentido contrário ao que determina
  • IA interpreta decisão de primeira instância como se fosse entendimento consolidado do STJ

Quando Boris v2 faria diferença crítica

🎯 Petições para juízes exigentes

Juízes combativos (categoria "complicador") detectam distorção de citação e podem penalizar. Boris v2 é escudo pré-protocolo.

💰 Casos de alto valor (> R$ 50k)

Risco de honorários sucumbenciais aumenta com o valor. Qualidade semântica da jurisprudência é alavanca direta.

🆕 Teses novas / jurisprudência em formação

Acórdãos recentes ainda sem ementa consolidada. IA tende a super-generalizar — Boris v2 valida se aplica ao caso.

🏦 Petições contra bancos / grandes réus

Departamentos jurídicos robustos (BB, Bradesco, Itaú) atacam citações distorcidas. Boris v2 previne contra-ataques.
