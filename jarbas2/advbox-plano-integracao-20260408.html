O que a IA faz automaticamente

🏷️
Classificação de Urgência
A IA lê o conteúdo de cada intimação e classifica em 4 níveis:
  • Imediata prazo < 3 dias, citação, audiência
  • Alta prazo 3-5 dias, despacho importante
  • Normal prazo > 5 dias, rotina
  • Informativa sem prazo, mero expediente
🔗
Agrupamento por Processo
Intimações do mesmo CNJ são agrupadas automaticamente:
  • • Múltiplas intimações do mesmo CNJ = 1 grupo
  • • Evita retrabalho em duplicatas da ADVBOX
  • • Mostra contagem de intimações por processo
  • • Ordena pelo prazo mais urgente do grupo
📋
Ação Recomendada
A IA sugere o que fazer com base no conteúdo:
  • • "Elaborar réplica" (contestação recebida)
  • • "Agendar audiência" (pauta designada)
  • • "Cumprir despacho" (determinação do juiz)
  • • "Apenas ciência" (mero expediente)
  • João confirma ou ajusta a sugestão
⏰
Controle de Prazo
Prazos calculados e destacados visualmente:
  • ● Vencido — prazo já passou (vermelho)
  • ● Hoje — vence hoje (laranja)
  • ● Crítico — 1-3 dias (amarelo)
  • ● No prazo — mais de 3 dias (verde)
  • Alertas automáticos para prazos críticos

Fluxo Técnico Detalhado

1
Coleta (automática ou sob comando)
A skill /advbox-sync busca movimentações recentes via GET /last_movements.
  • • Filtra apenas movimentações que contêm "INTIMACAO" no título
  • • Busca tarefas "ANALISAR INTIMACAO" via GET /posts
  • • Rate limit respeitado: 30 requisições/minuto (suficiente para ~500 intimações/dia)
2
Deduplicação
Hash do conteúdo (título + CNJ + data) para evitar registros duplicados.
  • • Se já existe no MdFlow → ignora (não importa de novo)
  • • Se é duplicata de outro CNJ → marca como duplicada_de_id
  • • Garante que o João nunca vê a mesma intimação duas vezes
3
Enriquecimento
Busca dados completos do processo e da publicação na ADVBOX.
  • • Busca dados do processo via GET /lawsuits/{id}
  • • Busca publicação completa via GET /publications/{id}
  • • Vincula ao processo existente no MdFlow (match por CNJ)
4
Classificação IA
Modelo o4-mini analisa título + conteúdo de cada intimação.
  • • Define: urgência, tipo_acao, prazo, risco
  • • Custo: ~R$0,01 por intimação (praticamente zero)
  • • Tempo: ~2 segundos por intimação
5
Inserção no MdFlow
Cria registro em intimacoes com provider='advbox'.
  • • Aparece automaticamente no Controller do João
  • • Realtime: atualiza ao vivo na tela (sem precisar dar refresh)
  • • Separado dos dados do EPROC — sem conflito
⚠️

Limitações — Transparência Total

Não dá para CONCLUIR tarefas na ADVBOX via API
A API da ADVBOX não permite marcar tarefas como concluídas (retorna erro 405 — Method Not Allowed). Isso significa que, mesmo após analisar no MdFlow, a tarefa continua "pendente" na ADVBOX.
  • (A) João conclui manualmente na ADVBOX após trabalhar no MdFlow
  • (B) Ignorar a ADVBOX completamente após importar — trabalhar só no MdFlow
  • (C) Futuramente automatizar via browser (Playwright) — mais complexo
Recomendação: opção B — João trabalha SÓ no MdFlow e ignora a ADVBOX
Rate limit de 30 requisições/minuto
Suficiente para importação diária (~200-500 intimações novas). Insuficiente para sync histórico completo (155 mil movimentações levaria ~86 horas).
Solução: importar apenas dos últimos 30 dias + novas a partir de hoje
Intimações não têm campo "tipo" na ADVBOX
A API não diferencia intimação, citação, despacho — tudo vem como "movimentação". Precisa parsear o texto do título para identificar o tipo.
Solução: a IA resolve isso com alta precisão analisando o texto (~99%)
Dados da ADVBOX são read-only no MdFlow
O que for importado fica registrado com provider='advbox'. Não afeta dados existentes do EPROC. Pode rodar em paralelo sem conflito nenhum.
Risco zero de interferir com o que já existe no MdFlow
