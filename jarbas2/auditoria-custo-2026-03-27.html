⚠️ Problemas Identificados

🔴 [CRÍTICO] JARBAS legado com heartbeat a cada 2h rodando Sonnet com vault de 75MB
O JARBAS legado está configurado com heartbeat: { every: "2h" }. Isso significa que a cada 2 horas ele carrega o contexto completo incluindo o vault mdlab-vault (75MB de notas) + memorySearch + HEARTBEAT.md para execução. Com Sonnet 4.6, cada heartbeat pode custar $5-15 USD dependendo do contexto carregado. São 12 heartbeats/dia = $60-180/dia só nesse agente.

Agravante: O heartbeat do JARBAS legado faz queries no Supabase, verifica grupos, verifica JANA — cada verificação é uma chamada de modelo.
🔴 [CRÍTICO] contextTokens de 200.000 em Jarbas 2.0 e OlivIA — carregando contexto máximo desnecessariamente
Jarbas 2.0 e OlivIA estão configurados com contextTokens: 200000 — o máximo possível para Sonnet 4.6. Isso significa que cada requisição potencialmente carrega 200k tokens de contexto, multiplicando o custo de cache miss. O correto é limitar a 80-100k para uso normal e deixar 200k só para tarefas que explicitamente precisam.
🔴 [CRÍTICO] Cron de relatório da OlivIA com timeout de 300s dando erro 3x consecutivos
O cron de relatório diário da OlivIA (18h) está marcado com consecutiveErrors: 3 e lastErrorReason: "timeout". Isso significa que ele está rodando por 5 minutos completos, falhando, e sendo retentado. Cada tentativa com 300s de Sonnet gera custo enorme. O prompt do cron está tentando ler 3 sessions_history completas + gerar HTML — muito pesado para uma sessão isolada.
🟠 [ALTO] Vault da Cecília e JARBAS legado sem limpeza — 69MB e 75MB sendo indexados
O memorySearch carrega todos os arquivos dos vaults para busca semântica antes de cada resposta. Vaults de 69-75MB com milhares de arquivos geram chamadas de embedding pesadas. O vault da Cecília herdou todo o histórico da Claudia (antiga). Precisa de limpeza cirúrgica.
🟠 [ALTO] Crons de monitoramento Cecília (3x/dia) rodando via JARBAS legado com Sonnet
Os 3 crons de monitoramento EPROC (manhã 7h, tarde 12h, noite 18h) + consolidação noturna (0h) estão todos configurados no JARBAS legado e usando Sonnet. São tarefas de execução de script — não precisam de modelo avançado. Poderiam usar Haiku ou ser migrados para scripts diretos.
🟡 [MÉDIO] Jarbas 2.0 sem contextPruning agressivo — sessão atual acumula tokens
Nossa sessão atual está longa (vários comandos SSH, investigações, configs). O keepLastAssistants: 3 está ativo mas o contexto desta sessão já deve estar em 50k+ tokens. Cada mensagem nova carrega o histórico completo até o TTL de 1h.
🟡 [MÉDIO] HEARTBEAT.md do Jarbas 2.0 com 7 verificações complexas
O HEARTBEAT.md do Jarbas 2.0 define 7 verificações: vault, projetos, handoffs Supabase, contexto, Mac Mini SSH, subagentes, watchdog de crons. Se o heartbeat for ativado no futuro com frequência alta, isso vira custo fixo pesado. Por enquanto sem heartbeat configurado — mas risco latente.

✅ Plano de Ação — 7 Otimizações

1
JARBAS legado — desabilitar ou aumentar heartbeat para 6h
Alterar heartbeat.every: "2h" para "6h" ou desabilitar completamente já que Jarbas 2.0 faz a supervisão. Economia estimada: 60-70% do custo do JARBAS legado.
2
Reduzir contextTokens de Jarbas 2.0 e OlivIA de 200k para 80k
200k tokens é desnecessário para uso cotidiano. 80k cobre 95% dos casos. Para tarefas longas o modelo usa compaction automático. Economia: ~40% por requisição em cache miss.
3
Cron OlivIA — reduzir timeout de 300s para 120s + simplificar prompt
O cron está falhando com timeout. Quebrar em 2 etapas: (1) coletar dados 120s, (2) gerar HTML 120s. Ou usar modelo menor. Economia: eliminar 3 runs desperdiçadas/dia = R$300+.
4
Crons de monitoramento Cecília — migrar de Sonnet para Haiku
Os 4 crons (manhã/tarde/noite/noturno) executam scripts bash + reportam resultado. Não precisam de Sonnet. Haiku é 20x mais barato. Economia: ~95% do custo desses 4 crons.
5
Limpar vault Cecília (69MB → <5MB)
O vault herdou histórico de logs, relatórios e dados da Claudia antiga. Manter apenas: SOUL.md, decisões, lições, projetos ativos. Eliminar logs e relatórios HTML. Economia: memorySearch mais rápido e barato.
6
JARBAS legado mdlab-vault (75MB) — externalizar grandes pastas do memorySearch
Excluir do memorySearch as pastas treinamentos/, processos/, dados/. Manter só 02_decisoes/, 03_licoes/, 04_stack/. Reduz vault efetivo de 75MB para <5MB.
7
Jarbas 2.0 — resetar sessão atual e começar sessão limpa
Esta sessão já acumulou ~2h de histórico de comandos e investigações. Após aplicar as otimizações, iniciar nova sessão para aproveitar o contexto limpo e cache-ttl: 1h.
