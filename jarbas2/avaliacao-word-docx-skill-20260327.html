📦 O que é o ivangdavila/word-docx

ℹ️
Essa skill do ClawHub NÃO é uma biblioteca de código. É um arquivo SKILL.md com ~60 regras comportamentais que ensinam o Claude a tratar DOCX como OOXML (ZIP de XML), não como texto. Inclui regras sobre estilos, numeração, track changes, compatibilidade LibreOffice/Google Docs, fields, etc.

💡 O que o word-docx ensina que é genuinamente útil para a OlivIA

⚠️
A OlivIA gera petições do zero — não edita DOCX existentes. Então ~60% das regras do word-docx (track changes, revision workflows, copy-paste de estilos) são irrelevantes para o fluxo atual. Mas algumas regras são aplicáveis.

Regras relevantes para absorver

  • A4 explícito: sempre definir page size como A4 (210x297mm) nos DOCX gerados — o word-docx alerta que US Letter é o default e muda paginação
  • Named styles: usar named styles do Word (Normal, Heading1, ListBullet) em vez de formatação direta — documentos ficam editáveis
  • Numeração via abstractNum: listas numeradas devem usar o sistema correto de numeração do Word, não Unicode manual
  • Table widths explícitas: definir larguras fixas nas tabelas para evitar drift no LibreOffice/Google Docs
  • Round-trip check: verificar que o DOCX gerado abre corretamente no Word antes de subir para o bucket — ainda não fazemos isso
  • Track changes: não relevante — OlivIA não edita docs existentes com revisões
  • Copy-paste entre documentos: não relevante — geração é do zero
  • Fields (TOC, page numbers): raramente usados nas petições geradas

✅ Recomendação Final

O que fazer (em vez de clawhub install word-docx)

  • Absorver as 4 regras relevantes diretamente no skill_olivia_cria_doc_local.mjs como comentários/config — A4, named styles, numeração, table widths
  • Adicionar verificação de round-trip: após gerar o DOCX, fazer um open simples com mammoth (já instalado) para confirmar que não está corrompido
  • NÃO instalar via clawhub install — a OlivIA não usa o Claude Code como runtime, é um agente Node.js puro. O SKILL.md seria para o Claude Code, não para o agente
  • NÃO adicionar novas dependências — todas as libs necessárias já estão instaladas
  • Se no futuro a OlivIA precisar editar DOCX existentes (ex: petições do cliente com track changes), aí sim o word-docx vira referência obrigatória
