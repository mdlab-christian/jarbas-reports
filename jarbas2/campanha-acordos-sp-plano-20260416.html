Decisão Recomendada: Abordagem Híbrida Sem Cecília (inicialmente)

Os processos SP são majoritariamente TJSP (e-SAJ), não EPROC. Cecília foi construída para EPROC (TJRS/TJSC). A abordagem recomendada usa Supabase + Datajud API + Script OAB CNA + n8n + WhatsApp — sem depender de browser automation na etapa inicial. OpenClaw entra apenas na Fase 3 para casos sem dado.

⚠️
Observação Técnica Crítica: TJSP ≠ EPROC

Os processos com código 8.26.xxxx tramitam no TJSP via sistema e-SAJ (portal esaj.tjsp.jus.br). Cecília usa seletores EPROC (#tblPartesERepresentantes, Keycloak SSO) que não funcionam no e-SAJ. Usar Cecília aqui exigiria reescrever os skills de navegação — investimento maior que o valor do piloto. OpenClaw (Computer Use) consegue navegar o e-SAJ por ser genérico.

Plano de Execução — 4 Fases

Plano detalhado para executar a campanha de acordos com o mínimo de esforço e máxima confiabilidade.

01
Curadoria da Lista
Exportar do Supabase os processos filtrados por empresa prioritária, fase correta e OAB SP535101. Gerar planilha inicial com dados já disponíveis.
  • 1
    Executar query Supabase com filtros: nosso_oab = 'SP535101' + empresas da lista João + fases: citacao, defesa, inicial, instrucao, replica
  • 2
    Exportar CSV com: numero_cnj, reu_nome, cliente_nome, fase_processual, valor_causa, vara_distribuicao, advogado_reu_nome, advogado_reu_oab, data_contestacao
  • 3
    Separar em 2 abas: Com dado advogado (~233) e Sem dado advogado (~165)
  • 4
    Criar tabela Supabase campanhas_acordo para tracking de status de cada processo na campanha
  • 5
    Revisão manual por Christian/João para excluir processos inadequados
⏱ 2-4h
🛠 MdOs skill / Script Node.js
💰 ~$0
02
Enriquecimento — Datajud API
Para os ~165 processos sem advogado_reu, buscar na API do CNJ (Datajud) os representantes do polo passivo.
  • 1
    Script Node.js: para cada CNJ sem dado → GET https://api-publica.datajud.cnj.jus.br/api_publica_tjsp/_search com o número do processo
  • 2
    Extrair de dados.representantes onde tipo = 'Advogado' e polo = réu → capturar nome + OAB
  • 3
    Rate: API pública (~3 req/s). 165 processos ≈ ~1 minuto de execução
  • 4
    Atualizar Supabase com advogado_reu_nome + advogado_reu_oab via UPSERT
  • 5
    Para os que Datajud não retornar → marcar como pendente_openclaw para Fase 3
⏱ 4-8h dev + 1min exec
🛠 Script Node.js + n8n
💰 ~$0 (API gratuita)
03
Busca de Contato (CNA-OAB + Google)
Com a OAB do advogado réu em mãos, buscar e-mail e WhatsApp via CNA (Cadastro Nacional de Advogados) da OAB e pesquisa do escritório.
  • 1
    CNA-OAB: Script POST para https://cna.oab.org.br/ com número OAB → scraping do e-mail cadastrado
  • 2
    Consolidar escritórios únicos: agrupar processos pelo mesmo advogado_reu_oab → um contato serve múltiplos processos
  • 3
    OpenClaw (Computer Use): para escritórios sem e-mail no CNA → pesquisa Google nome+escritório+WhatsApp+SP
  • 4
    Busca específica: Egger e Mesquita Advogados → site, e-mail, WhatsApp
  • 5
    Planilha final com: advogado, escritório, WhatsApp, e-mail, processos vinculados, empresa-réu
⏱ 1-2 dias
🛠 Script + OpenClaw (spots)
💰 Baixo (~$5-20 OpenClaw)
04
Disparo — WhatsApp + Tracking
Enviar propostas personalizadas via WhatsApp API oficial (número Christian Paz Advocacia) e rastrear respostas no Supabase.
  • 1
    Configurar API oficial WhatsApp (Meta Cloud API) com o número Christian Paz Advocacia já cadastrado
  • 2
    Criar template de mensagem no WhatsApp Business Manager para aprovação Meta (tempo: ~24h)
  • 3
    n8n workflow: ler planilha/Supabase → personalizar mensagem (nome adv, empresa, CNJ) → enviar via API
  • 4
    Rate: máximo 100 msgs/dia para números frios (limite Meta). Campanha de ~233 contatos = ~3 dias de disparo
  • 5
    Webhook de resposta → n8n → atualizar campanhas_acordo com status: enviado / lido / respondeu / negociando / fechado
  • 6
    Alerta Telegram para Christian/Heloísa quando receber resposta de interesse
⏱ 1-2 dias setup + 3 dias disparo
🛠 n8n + Meta WhatsApp API
💰 ~R$50-100 (msgs API)

🗄️ Tabela Supabase — campanhas_acordo
CREATE TABLE campanhas_acordo (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  organizacao_id UUID NOT NULL,
  processo_id UUID REFERENCES processos(id),
  campanha TEXT NOT NULL,     -- 'sp_sem_condenacao_2026'
  onda SMALLINT DEFAULT 1,    -- 1 ou 2
  status TEXT DEFAULT 'pendente',
    -- pendente | enriquecido | sem_contato
    -- enviado | lido | respondeu
    -- negociando | fechado | recusado
  advogado_reu_nome TEXT,
  advogado_reu_oab TEXT,
  escritorio_nome TEXT,
  contato_whatsapp TEXT,
  contato_email TEXT,
  contato_fonte TEXT,         -- 'banco' | 'datajud' | 'cna' | 'google'
  enviado_em TIMESTAMPTZ,
  respondeu_em TIMESTAMPTZ,
  resposta_texto TEXT,
  notas TEXT,
  created_at TIMESTAMPTZ DEFAULT now(),
  updated_at TIMESTAMPTZ DEFAULT now(),
  deleted_at TIMESTAMPTZ
);
