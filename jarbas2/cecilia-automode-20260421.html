Bug #1 — Budget $2 matando 54% dos ciclos

Bug #2 — RAG com 0 embeddings funcionais
.env tinha GOOGLE_API_KEY, código esperava GOOGLE_AI_API_KEY. 125 registros em rag_controller com embeddings inválidos (geração sempre falhava silenciosamente).
Fix: renomear env var + reprocessar 125 registros + migrar ivfflat → HNSW
Bug #3 — Gênero errado na taxonomia → 0% playbooks disparando
intimacoes.tipo_acao: 'juridica_estrategica' (feminino ✓)
controller_playbooks.trigger_tipo_acao: 'juridico_estrategico' (masculino ✗)
JOIN sempre falhava. Nenhum playbook disparou em toda a existência do sistema.
Fix: UPDATE 6 linhas em controller_playbooks corrigindo masculino → feminino
Bug #4 — SP535101 parado (watchdog 2h < ciclo real 1h37min)
Apenas 23 minutos de margem. Qualquer lentidão na rede ou no EPROC matava o processo antes de concluir.
Fix: watchdog 3h para todas as OABs (margem de 1h23min)
Bug #5 — 55 processos Chromium órfãos (alertas frequentes)
browser.close() não estava em bloco try/finally. Qualquer exceção deixava processos Chromium vivos indefinidamente.
Fix: try/finally em S1, S4, S5 + pkill chromium no startup do daemon
Bug #6 — 3 pipelines paralelos gerando dados incoerentes
F1: classe_cecilia A-G + analise_ia={} vazio

Resultado: 0 análises completas reais em toda a tabela intimacoes. 176/4.704 = 3.7% com dados aproveitáveis.

Plano de Execução Automode

Como funciona o automode do /projeto
  • Cada task segue 3 fases: PLAN → EXECUTE → REVIEW
  • Tasks dentro da mesma wave rodam em terminais paralelos
  • Cada worker acessa Mac Mini via SSH (ssh mac-mini-tail "...")
  • Resultados salvos em .projeto/cecilia/v1/status/
  • Após cada wave: validação antes de avançar
Protocolo de cada task (3 fases)
PLAN → EXECUTE → REVIEW
PLAN (5-15min): - Ler spec da task - Verificar estado Mac Mini (SSH) - Verificar schema Supabase se DDL - Identificar riscos e deps - Confirmar pré-requisitos OK EXECUTE (variável): - Implementar a mudança - Testar no ambiente - Verificar critérios de aceite REVIEW (5-10min): - Rodar checklist de aceite - Registrar em .projeto/.../status/ - Commit se código - Marcar ✅ DONE ou ❌ FAILED
