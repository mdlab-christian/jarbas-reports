Lições Aprendidas

1Validar API keys e modelos no init
gemini-embedding-2 não existia e falhou silenciosamente por semanas. Adicionar health check de embedding na startup do controller-agent — verificar model availability e API key válida antes do primeiro ciclo.
2Testar constraints do banco antes de code review
A constraint intimacoes_modo_operacional_check era conhecida mas o fallback 'auto' nunca foi testado contra ela. Assertions de schema devem fazer parte do pipeline de verificação do controller.
3SKIP vs ABORT na classificação de erros
Erros de navegação do Playwright são inerentemente transitórios. Todo erro de contexto Playwright deve ser SKIP por default, não ABORT. Uma única classificação errada bloqueava uma OAB inteira por múltiplos ciclos.
4Circuit breaker precisa de auto-reset por causa
O auto-reset de 24h é necessário mas insuficiente quando a causa raiz é um bug de código. Bugs de código devem resetar o circuit breaker automaticamente após deploy para evitar intervenção manual recorrente.
5Duplo path de vault — verificar processo em execução
cecilia-vault vs Documents/cecilia-vault são o mesmo filesystem (mesmo inode via symlink), mas é essencial sempre verificar o processo em execução (ps aux | grep cecilia) para confirmar o working directory correto e evitar editar o arquivo errado durante debug em produção.
