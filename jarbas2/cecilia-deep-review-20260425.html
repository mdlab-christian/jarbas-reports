Secao 3 — 6 Bugs Corrigidos (Deep Review)

BUG-A P0 CRITICO S8 sobrescrevia ia_status de records sem classe
S8 sobrescrevia ia_status='erro_s5_vazio' para 'concluido' em records sem classe. 13 records ficavam stuck para sempre — nunca seriam reprocessados.
Fix: guard continue antes do switch no s8-act.mjs. Records sem classe sao ignorados pelo S8.
BUG-B P0 advogado_id nunca persistido no S7
O campo advogado_id nao era incluido no payload do UPDATE/guard do S7. Todos os 50 records ficavam com NULL, quebrando filtros e relatorios por advogado.
Fix: controller passa advogadoId ao s7Persist; s7-persist.mjs inclui no payload do UPDATE.
BUG-C P1 titulo permanecia placeholder apos S5 ler documento real
O titulo continuava como "Movimentacao detectada — aguardando leitura S4/S5" mesmo apos S5 ter lido e processado o documento com conteudo real.
Fix: extrairTituloS7() extrai primeira linha de conteudo_completo. Titulo agora reflete o conteudo real do documento.
BUG-D SQL 13 records stuck nunca seriam reprocessados
13 records com NULL classe + ia_status='concluido' ficavam fora do radar do S3, que filtra por ia_status='pendente'. Seria um loop infinito de exclusao.
Fix: SQL reset desses records para pendente. Agora serao reprocessados no proximo ciclo.
BUG-E SQL Todos 50 records sem advogado_id no banco
O backfill nunca havia sido executado. Todos os 50 records de RS116571 estavam com advogado_id = NULL, quebrando filtros de interface e relatorios de advogado.
Fix: SQL backfill RS116571 → Christian. 100% dos records agora tem advogado_id correto.
BUG-F P1 Claude retornava classes descritivas em vez de A-G
O modelo retornava strings como "ruido", "ciencia_simples", "decisao_favoravel" em vez das classes canonicas A, B, C, D, E, F, G. Nenhum record era classificado corretamente.
Fix: CLASSE_ALIASES em ia-output-parser.mjs mapeia todas as variantes descritivas para as classes A-G. ~0% WARN de validacao agora.
