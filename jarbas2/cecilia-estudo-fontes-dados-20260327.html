Como os Sistemas do Mercado Funcionam

📰 AdvBox / DV Box e similares — Diário Eletrônico
Monitoramento

Esses sistemas monitoram as publicações no DJE (Diário de Justiça Eletrônico) para capturar intimações e despachos publicados oficialmente. O fluxo é: baixam o PDF do diário → fazem OCR/parsing → extraem número do processo + texto da publicação. Mas isso só captura o que foi publicado no diário, não os documentos internos do processo (petições, laudos, etc.). É eficiente para gestão de prazos via intimações, mas limitado para análise documental completa.

⚡ JusBrasil API de Intimações
API paga

A API de intimações do JusBrasil foi projetada para automatizar a coleta de intimações eletrônicas nos tribunais (EPROC, PJe, ESAJ, Projudi, DJE). Ela acessa os sistemas como se fosse o advogado, capturando intimações com seus anexos/documentos das intimações. Mas o escopo é: intimações do portal do advogado, não todos os documentos do processo.

🏛️ DataJud CNJ — API Pública
Gratuita

API pública e gratuita do CNJ com dados de todos os tribunais brasileiros. Fornece: capa processual + movimentações. Não fornece documentos. É excelente para monitorar andamentos, mas para análise documental não serve. Limite de uso por terms of use.

🔍 Codilo / JUDIT e similares
API paga

APIs comerciais que fazem scraping dos tribunais para consulta processual (capa, movimentações, partes). Cobram por consulta/monitoramento. Não disponibilizam os documentos completos do processo — só metadados e movimentações.

Análise para o Caso Específico da Cecília

🎯 O que a Cecília precisa fazer?

  • Monitorar movimentações de ~5.800 processos ativos → várias fontes resolvem
  • Detectar intimações com prazos → DJE ou JusBrasil API resolvem
  • Baixar e analisar os documentos do processo (petições, sentenças, laudos) → SOMENTE via EPROC direto

⚠️ Ilusão do Diário Eletrônico

O DJE publica resumos das intimações e despachos — geralmente o texto "Intimem-se as partes" ou "Cite-se o réu", com o número do processo. Ele não contém os documentos completos (a petição inicial, a contestação, o laudo pericial, a sentença completa). Para análise jurídica de conteúdo, o DJE é insuficiente.

Sistemas como AdvBox usam o DJE apenas para gestão de prazos (saber que houve uma publicação), não para análise do conteúdo dos documentos.

✅ Arquitetura ideal — 2 camadas complementares

  • Camada 1 — Monitoramento em massa (rápido e barato): DataJud CNJ ou DJE → detectar novos andamentos em todos os 5.800 processos sem precisar abrir o EPROC
  • Camada 2 — Acesso documental (focado): EPROC direto via Playwright → entrar apenas nos processos que tiveram movimentação relevante para baixar e analisar os documentos
  • Resultado: Cecília não abre o EPROC processo a processo cegamente — ela vai direto nos processos que precisam de atenção

🏆 Veredicto Final

🔵 Diário Eletrônico serve?

⚠️ Parcialmente. Ótimo para monitorar que houve publicação. Inútil para baixar documentos completos do processo. AdvBox usa DJE para gestão de prazos, não para análise documental.

🔵 APIs externas (JusBrasil, Codilo, JUDIT) resolvem?

⚠️ Movimentações sim, documentos não. Nenhuma API externa disponibiliza os documentos completos do processo. Só o EPROC com credenciais.

🟢 Para análise de documentos: qual o caminho?

✅ EPROC direto é obrigatório. Não existe alternativa via API. O EPROC com login de advogado é a única fonte dos documentos completos.
