⚡ Veredicto: Sistema Híbrido Playwright + Claude API (sem Computer Use)

Computer Use (CU) para 800 processos/dia é financeiramente inviável — custo estimado de R$ 800–2.400/dia só com screenshots. A abordagem correta é: Playwright faz toda a navegação/extração determinística (login, captura de lista, download de documentos) e Claude API analisa o conteúdo textual extraído — sem screenshot, sem Computer Use. Esse modelo já está implementado na Cecília mas quebrado em dois pontos específicos (ativo=false + conteúdo null). Computer Use é reservado para fallback pontual quando o Playwright falha num selector específico (estimativa: 5–10% dos casos).

🆕 Novidades Anthropic Relevantes para a Cecília (2025–2026)
🖥️ Computer Use v2 (Nov 2025)
Beta header: computer-use-2025-11-24. Funciona com Sonnet 4.6 e Opus 4.6. Permite screenshot + mouse + teclado. Estado da arte no WebArena. Novo: tool text_editor_20250728 para edição de texto e bash_20250124 para comandos — mais eficiente que só screenshot.
🧠 Claude Sonnet 4.6 — Mesmo preço, muito mais capaz
$3/MTok input · $15/MTok output. Com cache: $0.30/MTok (90% off). Performance quase de Opus em tarefas jurídicas. Já é o modelo da Cecília (analise_intimacoes_bulk migrado Haiku→Sonnet 4.6, precisão 92%). Ideal para análise de documentos jurídicos longos.
📦 Batch API — 50% mais barato
Para análise de intimações (não precisa de resposta em tempo real): usar Batch API reduz custo pela metade. Sonnet 4.6 Batch: $1.50/$7.50 por MTok. Ideal para o pipeline noturno: captura no dia, análise em lote à noite.
🔍 Extended Thinking (Claude 3.7+)
Para casos ambíguos difíceis (sentença procedente complexa, cálculo de honorários múltiplos), ativar thinking budget de 1000–2000 tokens. Custo extra mas muito superior para análise jurídica profunda. Usar seletivamente — não para todas as 800 intimações.
🛡️ Prompt Caching 1h
Cache de 1 hora para prompts frequentemente reutilizados. O system prompt da Cecília (playbooks + regras + roteamento = ~33k chars) pode ser cacheado por 1h — economizando 90% no custo de input das 800 análises/dia.
🖥️ Computer Use na prática (Sonnet 4.6)
Disponível via API hoje. Screenshot 1024×768 ≈ 1.092 tokens de imagem por captura. Uma tarefa típica de navegação usa 5–15 screenshots + raciocínio = $0.05–$0.30 por processo com Sonnet. Para 800 processos: $40–$240/dia só em CU.

🤔 Quando Faz Sentido Usar Computer Use (Análise Honesta)
✅
Faz sentido usar CU
• Fallback quando Playwright quebra — EPROC atualiza a interface sem avisar
• PDFs embeddados complexos — quando pdftotext falha e precisa de Vision para ler
• Formulários complexos — ciência com renúncia em EPROC SP (interface diferente)
• Detecção de mudanças de interface — usar Vision para verificar se a tela está como esperado
• Debug de novos selectors — quando precisa entender a interface para escrever Playwright novo
❌
NÃO faz sentido usar CU
• Navegação principal — Playwright é 50–100× mais rápido e barato
• Extração de listas — Playwright extrai HTML estruturado em milissegundos
• Análise de texto — Claude API de texto é 50× mais barato que via screenshot
• 800 processos/dia em loop — inviável financeiramente ($37+/dia)
• Login e autenticação — Playwright gerencia cookies/sessão perfeitamente
• Download de PDFs — fetch/axios direto é mais eficiente

💬 Resposta Direta às Suas Perguntas
🤔
Rodar Computer Use para operar o EPROC processo a processo?
Tecnicamente: sim, funciona. Praticamente: inviável para 800/dia. Sonnet 4.6 com CU custa ~$0.047/processo = $37/dia = R$ 200/dia. E demora 6–13 horas. A abordagem certa é Playwright determinístico + Claude analisa texto.
💡
Usar Computer Use para configurar/corrigir o Playwright?
Excelente ideia! Usar CU pontualmente para identificar selectors corretos quando a interface EPROC muda. Claude navega uma vez visualmente, extrai os selectors corretos, atualiza o código Playwright. Custo: $0.05 por "inspeção manual" — totalmente viável.
🔄
Cecília faz login Playwright + depois CU opera?
Tecnicamente viável mas desnecessário para o fluxo normal. Usar apenas quando Playwright falha num passo específico. O login via Playwright é muito mais rápido e estável. CU não precisa refazer o que Playwright já faz bem.
🆕
Novidades Anthropic mais relevantes para Cecília?
1) CU 2025-11-24 funciona com Sonnet 4.6 (viável economicamente como fallback). 2) Batch API — 50% off para análises não urgentes. 3) Cache 1h — 90% off no system prompt de 33k tokens. 4) Extended Thinking — para sentenças complexas.
