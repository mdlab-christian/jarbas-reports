🧪 Análise de Capacidade — GPT-4.1 para Texto Jurídico Brasileiro
✅ Pontos Fortes do GPT-4.1 para esta tarefa
🇧🇷
Português jurídico brasileiro GPT-4.1 tem excelente desempenho em PT-BR, incluindo termos processuais (SISBAJUD, AJG, EPROC, SCR/RMC/RCC). Comparável ao Sonnet para este domínio.
📋
Saída JSON estruturada Com response_format: { type: "json_object" }, o GPT-4.1 tem uma das melhores taxas de conformidade de JSON estruturado do mercado.
📏
Context window de 1M tokens 8× maior que o Sonnet 4.6 (200K). Com o crescimento da base de playbooks/regras, isso é uma vantagem significativa para não truncar o system prompt.
🎯
Instruction following (IFEval) GPT-4.1 é state-of-the-art em seguir instruções precisas — crítico para a regra M6 (ações específicas) e para não inventar responsáveis fora da lista.
⚡
Caching automático Não precisa alterar o prompt com cache_control. O OpenAI detecta automaticamente prefixos repetidos — menos código, menor chance de bug.
⚠️ Riscos e Limitações
🔬
Sem benchmark jurídico BR específico Médio Nenhum benchmark público compara GPT-4.1 vs Claude Sonnet em classificação de intimações EPROC. A qualidade real só será conhecida com testes A/B.
👁️
skill_ler_documento.mjs — Vision Alto O fallback L4 (screenshot → IA Vision) usa a API Anthropic com formato específico. Precisaria migrar para o formato OpenAI Vision separadamente.
💰
Cache menos agressivo que Sonnet Médio Anthropic dá 90% de desconto em tokens cacheados. OpenAI dá 75% (GPT-4.1). Para system prompts muito grandes, Sonnet ainda pode ser mais eficiente com cache ativo.
🔄
Dois providers simultâneos Baixo Se migrar a análise para OpenAI mas manter Vision no Anthropic, o sistema depende de dois provedores — mais pontos de falha, mais chaves para gerenciar.
🧾
Mudança na nomenclatura de tokens Baixo input_tokens → prompt_tokens. Quebra o tracking de custo da cecilia_custo_ia se não atualizar o código.
⚠️ GPT-4.1-mini — análise específica (88% mais barato, mas...)
✅ Bom para
  • Classes A, B, G (simples, alta volumetria)
  • Pre-triagem antes do modelo principal
  • Casos com texto curto e evento claro
❌ Risco alto
  • Classe E (estratégica) — sentença improcedente
  • Classe F (alvará/SISBAJUD) — valor financeiro
  • Nuances de litigância de má-fé
  • Prazos recursais específicos
💡 Uso ideal
  • Reintroduzir como Phase 1 (triagem)
  • GPT-4.1-mini para A/B/C/G
  • GPT-4.1 (ou Sonnet) para D/E/F
  • Economia: ~75% vs Sonnet puro

⚖️ Parecer Final — Jarbas
✅ VIÁVEL — GPT-4.1 como modelo principal de análise

O GPT-4.1 é tecnicamente capaz de realizar a análise de intimações EPROC na taxonomia A-G com qualidade comparável ao Claude Sonnet 4.6. A economia de ~$313/ano (43% de redução) é real e justifica a migração — desde que validada com testes A/B supervisionados antes do go-live.

Fatores positivos: 1M de contexto, instruction following excelente, JSON nativo, caching automático, chave OpenAI já disponível, código de migração simples.
Ponto de atenção: a qualidade em decisões E/F (estratégicas + financeiras) é o critério mais crítico — validar com João Victor antes de produção total.

⚠️ ATENÇÃO — GPT-4.1-mini só como camada de triagem

Usar GPT-4.1-mini como substituto único é arriscado para o domínio jurídico — principalmente em E/F. Porém, reintroduzir um split de modelo (mini para A/B/C/G + 4.1 para D/E/F) seria a opção de menor custo com menor risco, economizando ~75% vs Sonnet puro.

❌ NÃO RECOMENDADO — Trocar sem testes A/B

Em hipótese alguma trocar em produção sem ao menos um ciclo de validação manual das classificações. Uma falha na classe E (sentença improcedente não detectada) pode custar muito mais do que $313 em prêmios advocatícios perdidos. O protocolo de migração com toggle ENV garante rollback instantâneo.
