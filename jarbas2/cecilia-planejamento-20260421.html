Bugs P0 — Corrigir Primeiro (Bloqueia Tudo)

3 bugs — subprojeto-01 prioritario
BUG-P0-01
S6→S7 "analise_ia ausente ou sem classe" — 419 registros corrompidos
Frequencia: Maioria dos ciclos. RS116571 chegou a 110 erros em 1 ciclo.

Sintoma A: S5 retorna conteudo vazio (0 chars) → S6 emite ia_status='erro_s5_vazio' → schema-validator rejeita → S7 nao persiste.

Sintoma B: S6 retorna output do Claude mas sem campo classe no JSON → validator rejeita → S7 nao grava classe_cecilia.

Impacto: 419 intimacoes com classe_cecilia=null no banco. Ciclos marcados como falhou mesmo tendo processado parcialmente. RS116571 opera a 12% de aproveitamento real.

Hipotese principal: Parse do JSON do Claude falha quando resposta contem texto antes/depois do bloco JSON. Campo classe pode ter nome alternativo em algumas respostas (classificacao vs classe). Fix: regex mais robusta para extrair JSON + fallback de alias de campo.
S6 → S7 schema-validator classe_cecilia 419 registros afetados [schema-validator] s6->s7 INVALIDO [S8-ACT] Classe desconhecida
BUG-P0-02
SP535101 / TJSP Completamente Parado — e-SAJ vs EPROC
Sintoma: Todos os ciclos da OAB SP535101 terminam com "Watchdog: ciclo sem conclusao apos 2h". Zero intimacoes processadas historicamente.

Causa provavel: TJSP usa sistema e-SAJ (nao EPROC). Login diferente (nao e Keycloak), XLS diferente ou inexistente, selectors de DOM totalmente diferentes.

Impacto: 1.250+ processos SP sem nenhum monitoramento. Campanha SP535101 Onda-1 (398 prioritarios) completamente sem cobertura.

Acao necessaria: Investigar fluxo de login e estrutura de dados do TJSP e-SAJ antes de codar. Pode exigir modulo S1/S2 dedicado para e-SAJ. Verificar com Joao Victor se ha expectativa de prazo.
SP535101 e-SAJ (nao EPROC) Watchdog 2h 1.250+ processos afetados modulo dedicado necessario
BUG-P0-03
Config Fetch Erro — "fetch failed" centenas de vezes/hora
Sintoma: start-cron.mjs imprime "Config fetch erro: fetch failed — usando defaults" centenas de vezes por hora.

Impacto: Sem controle dinamico de budget ou OABs. Sistema usa defaults hardcoded ($2 budget por OAB) em vez de valores configurados no Supabase. Budget insuficiente para RS116571 e RS131163.

Causa provavel: Endpoint de configuracao dinamica (tabela cecilia_capture_config?) inacessivel no contexto do daemon, URL errada no .env, ou RLS bloqueando acesso.

Fix: Verificar URL da config no .env, checar RLS da tabela, adicionar retry com backoff exponencial e silenciar logs apos primeira falha.
start-cron.mjs cecilia_capture_config budget hardcoded $2 spam de logs RLS suspeito
