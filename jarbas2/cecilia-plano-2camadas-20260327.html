📡 Camada 1 — DataJud CNJ (sensor de movimentações)
API pública gratuita · Elasticsearch · Cobertura nacional · Não precisa logar no EPROC

✅ O que é e como funciona

  • API pública do CNJ com dados de todos os tribunais brasileiros
  • Backend Elasticsearch — suporta queries complexas e batch
  • Fornece: capa processual, movimentações com data e tipo, partes
  • Atualização: próxima ao tempo real (dados enviados pelos próprios tribunais)
  • TJRS → api_publica_tjrs | TJSC → api_publica_tjsc | TJSP → api_publica_tjsp

⚠️ Precisa fazer cadastro?

Sim, mas é gratuito. Acesse datajud.cnj.jus.br, crie conta institucional e solicite chave de API. É um processo simples. A API key é enviada por email. Sem custo, sem aprovação demorada — é um serviço público.

Rate limit: existe, mas é generoso para uso institucional. Para 5.800 processos em batch diário, não há problema.

🔌 Exemplo de query — buscar movimentações dos últimos 2 dias por número CNJ

# Endpoint: POST https://api-publica.datajud.cnj.jus.br/api_publica_tjrs/_search # Header: Authorization: ApiKey <sua_chave> { "query": { "bool": { "must": [ { "term": { "numeroProcesso": "5001234-12.2023.8.21.0001" } } ], "filter": [ { "range": { "movimentos.dataHora": { "gte": "now-2d/d" } } } ] } }, "_source": ["numeroProcesso", "movimentos", "dataAjuizamento"] } # Para busca em LOTE (múltiplos CNJs de uma vez — ideal para 5.800 processos): # POST /_msearch ← multi-search, uma request por tribunal com lista de CNJs

✅ O que o DataJud retorna (dados reais)

  • numeroProcesso: número CNJ completo
  • movimentos[]: lista com dataHora, codigoNacional, nome do movimento (ex: "Juntada de Petição", "Despacho", "Sentença")
  • classeProcessual, assuntos, orgaoJulgador, partes
  • Não retorna o conteúdo dos documentos — apenas os metadados da movimentação

⚠️ Limitações importantes

  • Dados chegam com delay de horas a 1 dia dependendo do tribunal (não é tempo real instantâneo)
  • Processos em segredo de justiça: não aparecem
  • Alguns tribunais têm cobertura parcial — TJRS/TJSC/TJSP são bem cobertos
  • Não entrega documentos, só metadados de movimentação
  • Requer atualização da base de CNJs no Supabase para correlacionar com processos do Midas

🤖 Camada 2 — EPROC Playwright (motor documental)
Acessa SOMENTE processos com movimentação real · Baixa documentos · Análise IA

✅ Por que o EPROC direto ainda é necessário (e sempre será)

  • Nenhuma API externa entrega os documentos completos do processo — petições, contestações, laudos, sentenças completas, recursos
  • Para análise jurídica real (o que a Cecília precisa fazer), o documento é insubstituível
  • Com o filtro do DataJud, ela vai em ~5-10% dos processos por dia, não em todos

🎯 Fluxo Cecília na Camada 2

  • Recebe lista de CNJs com movimentação (do DataJud)
  • Login EPROC com credenciais do escritório
  • Para cada processo relevante: navega, clica nos documentos novos
  • Baixa PDF/texto do documento
  • Envia para LLM: classificação + extração de prazo + resumo
  • Cria tarefa no MdFlow com prazo calculado

🐛 Bug P0 atual: documentos não abrem

  • Cecília já acessa processos e vê movimentações ✅
  • Falha ao clicar nos links de documento ❌
  • Causa provável: documento abre em nova aba / iframe separado / redirect que o Playwright não segue automaticamente
  • Fix: interceptar request de download ou usar page.waitForEvent('popup') para nova aba
  • Isso deve ser corrigido antes de qualquer outra evolução

🔧 Lógica de priorização para a Camada 2

# Classificação de movimentações por prioridade (sugerida) URGENTE → entra HOJE: - "Sentença" | "Acórdão" | "Intimação" | "Julgamento" | "Prazo" - "Despacho" com texto contendo prazo RELEVANTE → entra em até 24h: - "Juntada de Petição da Parte Contrária" - "Despacho" sem menção a prazo - "Certidão" IGNORAR (não entra no EPROC): - "Redistribuição" | "Conclusão ao Juiz" | "Remessa" - Movimentações administrativas sem ação necessária

JusBrasil API — Papel no Sistema

📬 O que a JusBrasil API de Intimações realmente entrega
Pago — avaliar custo

A API JusBrasil (via Digesto — op.digesto.com.br) acessa o portal do advogado nos tribunais com as credenciais fornecidas e coleta as intimações eletrônicas. Ela retorna:

  • Intimações com texto completo (conteudo_textual)
  • Anexos — documentos associados à intimação (PDF quando disponível)
  • Prazo calculado (campo "prazo")
  • Audiências agendadas com data/hora/local
  • Suporte a MFA (necessário para EPROC RS com Portaria CNJ 140/2024)
  • Alternativa ao preposto JusBrasil: você pode fornecer as credenciais + OTP seed do MFA

✅ Onde a JusBrasil API ajuda

  • Captura intimações com documento da intimação direto — sem abrir o EPROC
  • Ideal para despachos e sentenças curtas publicados como intimação
  • Suporta MFA — resolve o problema de autenticação do EPROC RS
  • Pode funcionar em paralelo como validação cruzada com o DataJud

⚠️ Limitações e riscos

  • É pago — preço não está público, precisa negociar
  • Só captura o que chega como intimação formal — petições da parte contrária não chegam assim
  • Dependência de terceiro: se cair, você não tem acesso
  • Pode ter delay maior que acesso direto ao EPROC

Plano de Implementação — Ordem de Execução

1

🐛 [AGORA] Corrigir bug documentos EPROC no Playwright

Antes de tudo. A Cecília precisa conseguir baixar documentos. Provável fix: page.waitForEvent('popup') para nova aba ou interceptação de download com page.waitForEvent('download'). Investigar no EPROC como o link de documento se comporta (nova aba? iframe? redirect para PDF?).

2

🏛️ [PRÓXIMO] Cadastro + integração DataJud CNJ

Criar conta em datajud.cnj.jus.br → solicitar API key institucional (gratuito). Depois: criar job n8n ou cron que consulta os 5.800 CNJs do Supabase por lote (msearch Elasticsearch) uma vez por dia. Resultado: tabela processos_com_movimentacao atualizada diariamente.

3

🔍 Filtro de prioridade sobre resultado do DataJud

Classificar cada movimentação detectada em: URGENTE / RELEVANTE / IGNORAR (tabela de regras no Supabase, configurável). A fila da Cecília no dia é a lista URGENTE + RELEVANTE ordenada por urgência.

4

🤖 Integrar fila do DataJud na Cecília

A Cecília passa a consumir a fila do Supabase em vez de tentar varrer tudo. Para cada processo na fila: abre EPROC → navega até movimentação nova → baixa documento → analisa com LLM → cria tarefa MdFlow com prazo.

5

⚡ [AVALIAR] Contratação JusBrasil API para intimações

Contatar JusBrasil Soluções (suportesolucoes@jusbrasil.com.br) para cotação. Se o custo for viável para ~5.800 processos, integrar como shortcut para intimações: quando DataJud detecta "Intimação" → JusBrasil API já entrega o documento sem abrir EPROC → EPROC só para o que JusBrasil não cobrir.

6

📊 Métricas e fallback

Monitorar: % processos cobertos pelo DataJud, % falsos negativos, tempo de processamento. Fallback: se DataJud estiver fora ou com delay > 24h → Cecília volta ao modo varredura parcial (top processos por data de andamento no Supabase).
