OpenClaw vs LaunchDaemon — Comparação

Pergunta: usar OpenClaw para comandar os crons e heartbeats da Cecilia em vez do LaunchDaemon atual?

🤖 OpenClaw para crons/heartbeats

PRÓS
  • Interface unificada Telegram
  • Feedback em tempo real ao comandar
  • Consistente com outros agentes
  • Sem SSH para ajustar horário
CONTRAS
  • Single point of failure (bot cai → sem controle)
  • OpenClaw não é garantia de nível OS
  • Latência de polling vs execução direta
  • Supabase config table já resolve mesma coisa
  • Adiciona camada de complexidade desnecessária
  • Se rede Tailscale cai, bot perde acesso

🖥️ LaunchDaemon + Supabase config (atual)

PRÓS
  • Nível OS — inicia antes do login
  • Sobrevive a crash do OpenClaw/Telegram
  • Supabase config = fonte da verdade
  • Restart automático pelo launchd
  • Testado e funcionando
CONTRAS
  • Alterar horário requer SSH ou MdOs
  • Menos visível no dia-a-dia
→
Recomendação: Arquitetura em 3 camadas (NÃO substituir LaunchDaemon)

Camada 1 — Execução: LaunchDaemon (OS-level, insubstituível para crons)
Camada 2 — Config: Supabase cecilia_config.schedules_json (fonte da verdade)
Camada 3 — Interface: OpenClaw/Telegram (UI para ler status e alterar config via /monitoramento)

OpenClaw nunca é o motor de execução. É apenas o painel de controle. O LaunchDaemon mantém Cecilia rodando mesmo que o bot Telegram esteja offline.
