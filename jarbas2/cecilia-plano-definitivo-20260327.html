Investigação 1 — DataJud CNJ (teste real com 6 CNJs)

✅ O que funciona no DataJud

  • 5 de 6 processos encontrados (83% de cobertura)
  • Historial completo de movimentos (até 56 por processo)
  • Tipos de movimento: Petição, Conclusão, Publicação, Sentença, Liminar, Tutela
  • API Key simples — header Authorization: APIKey ...
  • CNJ sem pontos/hífens como chave de busca
  • Multi-search (batch) suportado — eficiente para 5.800 CNJs
  • Gratuito, sem limite declarado para uso institucional

❌ Por que o DataJud NÃO serve como Camada 1

  • Delay médio: 19 dias entre movimentação real e indexação
  • Delay máximo: 28 dias
  • Processo de 2026 (distribuído há dias): 0 resultados — ainda não indexado
  • Para prazos processuais, 19 dias é catastrófico — prazo já venceu
  • Não entrega documentos, apenas metadados

⚠️ Uso recomendado para DataJud (real)

  • Job semanal de enriquecimento: preencher campos faltantes no Midas (vara, classe processual, data de ajuizamento)
  • Auditoria de base: identificar processos com meses sem movimentação (possíveis casos encerrados)
  • Validação de CNJs: confirmar se números no sistema estão corretos
  • NÃO usar: monitoramento diário de prazos, priorização de trabalho, detecção de intimações

Investigação 2 — Diário de Justiça Eletrônico (DJE)

❌ DJE não serve como fonte de monitoramento para o Midas

  • Delay: publicações do DJE aparecem 1-2 dias após o despacho/sentença. Além disso, o prazo só conta a partir do dia seguinte à publicação (Lei 11.419/06, art. 4º §3º)
  • Conteúdo: o DJE publica apenas o extrato da comunicação ("Intimem-se as partes", "Cite-se o réu"), não o documento completo
  • Correlação por processo: o DJE TJRS não tem API pública acessível — apenas portal web com busca manual por número/parte/OAB
  • Escala: para 5.800 processos, fazer scraping do DJE é inviável — o portal bloqueia automação e a estrutura do PDF é caótica
  • Redundância: tudo que aparece no DJE já aparece no painel de intimações do EPROC (e antes)

ℹ️ O DJE relevante para vocês é o DJERN (Nacional)

O CNJ criou o Domicílio Judicial Eletrônico (DJe-N) e o Diário de Justiça Eletrônico Nacional (DJEN) — a partir de 16/05 todos os prazos serão contados pelas publicações desses canais. Mas o EPROC já integra isso — a tela de intimações do painel do advogado consolida tudo de forma unificada. Não precisa monitorar DJE separadamente.

⚠️ Novo aviso crítico encontrado na tela de login

O TJRS tem aviso explícito anti-robô na tela de login: "O TJRS informa que estão sendo adotadas medidas de restrição de acesso robotizado [...]. O acesso simultâneo do mesmo usuário em várias máquinas é reconhecido pelo sistema como similar ao uso de robôs, o que pode levar ao bloqueio do usuário."

Isso explica o circuit breaker das OABs. A Cecília estava abrindo múltiplas sessões simultâneas por OAB, causando o bloqueio automático.

🚨

Problema Crítico — Circuit Breaker em Todas as OABs

🔴 Todas as 5 OABs desativadas por circuit breaker (6 falhas consecutivas)

  • RS116571 — TJRS — último sucesso: 2026-03-27 07:48 — circuit breaker ativo
  • RS131163 — TJRS — último sucesso: 2026-03-27 07:49 — circuit breaker ativo
  • SC074025 — TJSC — último sucesso: 2026-03-27 07:49 — circuit breaker ativo
  • SC075466 — TJSC — último sucesso: 2026-03-27 07:50 — circuit breaker ativo
  • SP535101 — TJSP — último sucesso: 2026-03-27 07:50 — circuit breaker ativo
  • TJSC também retornando ERRO 429 (rate limit HTTP)

Causa provável: múltiplas sessões simultâneas ou polling muito agressivo. O TJRS detecta como robô e bloqueia. O circuit breaker da Cecília entrou em ação corretamente, mas nunca resetou.

✅ Como resolver o circuit breaker

  • Resetar manualmente: UPDATE jarbas_accounts SET ativo = true, login_falhas_consecutivas = 0, ultimo_erro = null
  • Aguardar 30-60 min após o reset antes de tentar login (deixar o IP "esfriar")
  • Reduzir agressividade: uma sessão por OAB, no máximo
  • Implementar random delay entre requests (2-5s mínimo)
  • Nunca abrir múltiplas sessões simultâneas com a mesma OAB

Investigação 4 — 2FA TOTP (foi resolvido!)

✅ TOTP gerado programaticamente com sucesso

Os totp_secret estão no Supabase (jarbas_accounts.totp_secret). A geração de código TOTP funciona perfeitamente com Python padrão (sem libs externas). O login de hoje funcionou em ~5 segundos.

# Geração de TOTP para EPROC (puro Python, sem dependências) import hmac, hashlib, time, struct, base64 def get_totp(secret_b32): key = base64.b32decode(secret_b32.upper(), casefold=True) counter = int(time.time()) // 30 msg = struct.pack('>Q', counter) h = hmac.new(key, msg, hashlib.sha1).digest() offset = h[-1] & 0x0f code = struct.unpack('>I', h[offset:offset+4])[0] & 0x7fffffff return str(code % 1000000).zfill(6) # Usar a cada login, sempre com o secret do banco: # SELECT totp_secret FROM jarbas_accounts WHERE oab = 'RS116571'

⚠️ Cuidado: código TOTP válido por 30 segundos

O login precisa ser executado em menos de 30 segundos após gerar o código. No Playwright, isso é simples — gera o TOTP imediatamente antes de submeter o formulário. Nunca gerar o código com antecedência.

Resultado: tarefa criada no MdFlow com descrição do ato + prazo calculado + link do processo.
