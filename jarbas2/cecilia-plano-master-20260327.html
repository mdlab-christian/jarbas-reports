❌ Gaps Críticos — O que Está Quebrado

G1 · FONTE DE CAPTURA ERRADA [CRÍTICO]

A skill_captura_intimacoes.mjs navega para relatorio_processo_procurador_listar (Relatório de Processos com "Últimas Movimentações") e filtra por keywords de intimação na tabela de texto. Isso é frágil e ruidoso.

O correto (validado ao vivo): capturar diretamente da URL citacao_intimacao_prazo_aberto_listar — painel dedicado que já vem filtrado, com CNJ, prazos (início, final), juízo, classe, assunto. Sem filtro de keyword, sem falso positivo, sem movimentação irrelevante.

G2 · FALTA CAPTURA "PENDENTES DE CITAÇÃO" [CRÍTICO]

Christian pediu explicitamente: capturar também "processos pendentes de citação/intimação" — segundo tipo de fila do painel EPROC, além do "prazo em aberto".

URL esperada: citacao_intimacao_pendente_listar (a ser validada no login). Esta skill não existe na Cecília. Precisa ser implementada ou a skill_captura_intimacoes precisa acessar ambas as URLs em sequência.

G3 · LEITURA DE IFRAME DA ÁRVORE DE DOCUMENTOS NÃO IMPLEMENTADA [CRÍTICO]

Nossa grande descoberta ao vivo: os documentos judiciais no EPROC são renderizados como HTML puro em um iframe na Árvore de Documentos. Basta iframe.inner_text() — sem OCR, sem PDF parsing, sem download.

O código atual na skill_captura_intimacoes.mjs usa extractTextoDocumento() que tenta ler #tblEventos e cai no fallback Vision (Claude Screenshot, caro e lento). A função correta seria: navegar para arvore_documento_listar → clicar no último evento relevante → page.frameLocator('iframe').last().locator('body').innerText().

G4 · CHROMIUM NÃO INSTALADO NO USUÁRIO CECILIA [BLOQUEANTE]

Os logs mostram: browserType.launchPersistentContext: Executable doesn't exist at /Users/christian/Library/Caches/ms-playwright/chromium — caminho do usuário christian, não do usuário cecilia.

Fix: ssh mac-mini-tail "sudo -u cecilia npx playwright install chromium"

G5 · CIRCUIT BREAKER ATIVO (5 OABS) [BLOQUEANTE]

Causa raiz: sessões simultâneas com o mesmo OAB → TJRS detectou como robô e bloqueou. Portaria CNJ 140/2024 é severa com isso.

Fix: UPDATE jarbas_accounts SET ativo = true, login_falhas_consecutivas = 0 — depois aguardar 30-60min antes do primeiro novo acesso. Implementar delay mínimo 5min entre OABs + max 2x/dia.

G6 · HASH DO PAINEL NÃO CAPTURADA DINAMICAMENTE [DRIFT]

As URLs do EPROC como citacao_intimacao_prazo_aberto_listar&hash=XXXX são assinadas por sessão e mudam a cada login. O código atual chega no painel e depois navega para "Últimas Movimentações" — não captura o href do link "Prazo em Aberto" do painel.

Fix: após login, no painel, capturar o href exato dos links "Prazo em Aberto" e "Pendentes de Citação" via page.$eval('a[href*="citacao_intimacao_prazo_aberto_listar"]', el => el.href).

G7 · ANÁLISE IA SEM LEITURA AO VIVO DO DOCUMENTO [QUALIDADE]

A skill_analise_juridica_processo.mjs busca movimentações no banco (processos_movimentacoes) e tenta baixar PDFs da URL. O problema é que o banco pode não ter o texto completo do documento, e baixar PDFs é lento e falha frequentemente.

A abordagem correta: ao capturar a intimação, já ler o texto do documento via iframe e salvar em intimacoes.conteudo. A análise IA então usa esse texto rico direto do banco, sem precisar voltar ao EPROC.

LEITURA DE DOCUMENTO VIA IFRAME (nova skill)

// DESCOBERTA ao vivo: docs são HTML puro no iframe da Árvore
// Não há PDF, não há OCR, não há fallback Vision necessário

async function lerDocumentoIframe(page, cnj, baseUrl) {
  // 1. Navegar para Árvore de Documentos
  const arvoreUrl = `${baseUrl}/controlador.php?acao=arvore_documento_listar`
    + `&txtNumProcesso=${cnj}`;
  await page.goto(arvoreUrl, { waitUntil: 'domcontentloaded' });
  
  // 2. Clicar no evento mais recente relevante
  const ultimoEvento = page.locator(
    'tree treeitem, .infraArvore li'
  ).last();
  await ultimoEvento.click();
  await page.waitForTimeout(1000);
  
  // 3. Ler texto do iframe — HTML PURO!
  const texto = await page
    .frameLocator('iframe').last()
    .locator('body')
    .innerText();
    
  return texto; // texto judicial completo, sem OCR
}

Anti-Duplicação — Como Funciona

A Cecília usa signature_hash = SHA256(cnj + data_evento + texto[0:200]) para deduplicar. Se o hash já existe em intimacoes, retorna status: 'duplicada' e não insere nada.

A Unique Constraint no banco (code 23505) garante que mesmo se dois workers rodarem, só um consegue inserir. ✅ Sem duplicatas.

💡 Novas Ideias — Brainstorming

💡 Sessão Maximizada por Login

Login SSO+TOTP tem custo (tempo + risco de detecção). A cada sessão, maximizar o que capturamos: 1 login → captura "Prazo Aberto" + "Pendentes Citação" + snapshot dos contadores do painel em sequência, sem re-autenticar. Economiza tempo e reduz risco de bloqueio.

💡 Cache de Hash por Sessão

Após login, armazenar os hrefs assinados em memória com TTL de 3 horas. Se a Cecília precisar navegar para a mesma URL dentro da mesma janela, reutiliza a sessão. Evita re-login desnecessário.

💡 Análise Seletiva por Urgência

Não analisar todos os documentos via Claude — só os que realmente precisam. Prioridade: (1) prazo_fatal ≤ 5 dias, (2) tipo=sentenca/acordao, (3) SISBAJUD. O resto fica em fila ia_status='pendente' para análise em lote noturno. Custo 60-70% menor.

💡 Detector de Prazos no Texto do Iframe

Ao ler o texto via iframe, regex extrair prazos diretamente do despacho: "no prazo de 15 (quinze) dias", "até 07/04/2026". Isso evita pedir para o Claude calcular prazo — já vem no texto. Confiança alta quando o juiz menciona data explícita.

💡 Snapshot do Painel (Contadores)

Após login, antes de navegar para as tabelas, fazer screenshot dos contadores do painel do advogado (N processos com prazo aberto, N pendentes de citação, etc.) e salvar em jarbas_snapshots para dashboard histórico. Custo zero — só ler o DOM.

💡 Modo de Alcance Lento (off-peak)

Para os 5.800 processos sem movimentação recente no banco, rodar varredura lenta 1x/semana (sábados, 01:00) — lista completa, página a página, comparar com banco. Detecta processos que saíram do radar (redistribuídos, extintos) sem sobrecarregar EPROC nos horários de pico.

💡 Alerta Proativo de Prazo Fatal Próximo

Cron diário às 07:00 — query em intimacoes com prazo_fatal <= HOJE + 2d e status ≠ arquivada. Se houver → Telegram para João com lista de processos urgentes. Prevenção de perda de prazo — mais importante que a captura em si.

💡 Reconciliação Semanal EPROC ↔ Banco

Domingo à noite: skill_listar_processos_eproc.mjs (já existe!) para cada OAB, comparar CNJs do EPROC com o banco. CNJs no EPROC mas não no banco → enfileirar importação. CNJs no banco mas não no EPROC → marcar como arquivado/redistribuído.

💡 Fallback Gracioso por Tribunal

Se TJSC retorna 429 ou timeout, não travar toda a fila. Marcar a OAB como retry_after: +2h e continuar com as outras. Circuit breaker por tribunal, não por OAB global. O TJSC tem sido o mais instável historicamente.

💡 Teste Sintético Diário

1 processo específico de teste (conhecido, com histórico estável) que a Cecília verifica todo dia às 08:25 (antes da captura real). Se conseguir navegar e ler → sinal verde para produção. Se falhar → abortar e notificar Christian. Canário de saúde.
