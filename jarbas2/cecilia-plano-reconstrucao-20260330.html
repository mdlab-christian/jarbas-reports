👁️ O Que o Christian Mostrou no Vídeo
Insight crítico:

A fonte correta de movimentações é "Últimas Movimentações" — não "Prazo em Aberto" nem "Relatório de Processos". Isso muda tudo no fluxo atual da Cecília.

1

Painel do Advogado OK

Após login, está no painel. Contadores visíveis: 169 processos com prazo em aberto, 124 pendentes de citação. Próxima audiência 31/03.

➜ Clicar em "Últimas Movimentações" (botão no topo direito do painel)

2

Lista de Últimas Movimentações RECONSTRUIR

Tela com todos os processos que tiveram movimentação. Colunas incluem: Número do Processo, Classe, Autor, Réu, Localidade, Assunto, Último Evento, Data/Hora, Data Autuação, Valor Causa.

  • Filtrar por data/hora: processos com movimentação desde o último ciclo (ex: últimas 6h)
  • Navegar pelas páginas — processos são muitos, não cabe em 1 página
  • REGRA: clicar no número do processo, NÃO na checkbox
  • Coluna chave: "Data, Hora, Último Evento"
3

Processo Aberto — Verificar se é para Nós RECONSTRUIR

Ao abrir o processo, verificar se a movimentação é para a parte autora (que representamos). Muitos processos na lista são movimentações de outras partes — devem ser filtrados.

  • Identificar a parte que representa: parte-autora
  • Verificar qual evento/movimentação foi para nós
  • Se não for para nós → pular, voltar para lista
4

Acessar Documento do Evento QUEBRADO HOJE

Dentro do processo, identificar o evento relevante (ex: "Evento 5") e clicar no link do documento (despacho, intimação, etc) para abrir e ler o conteúdo.

  • Clica no link do documento — abre em nova aba ou iframe
  • Extrai o texto completo do documento
  • Recurso alternativo: Árvore do Processo — ver todas as movimentações históricas. Usar quando necessário

⚠️ Este é o passo quebrado hoje: conteudo=null em 100% dos casos. A URL da árvore não tem hash → sessão expira.

5

Análise IA + Registro DEPENDENTE

Com o texto do documento em mãos, Claude analisa o conteúdo aplicando os playbooks do João. Resultado salvo no Supabase.

🎯 Decisão Chave: Qual Fonte de Movimentações?

❌ Fonte Atual: Prazo em Aberto

  • Fila de intimações com prazo vencendo
  • Lista pré-filtrada pelo EPROC
  • Já expirado = urgente demais
  • Não cobre todos os processos ativos
  • Cecília está usando isso hoje — errado

Fonte Correta: Últimas Movimentações

  • Todos os processos com movimentação recente
  • Filtrar por janela de tempo (ex: últimas 6h)
  • Precisa filtrar quais são para a parte autora
  • Cobre 100% das movimentações
  • É o que o João mostrou no vídeo
Impacto:

Mudar de "Prazo em Aberto" para "Últimas Movimentações" é a mudança mais importante. Significa reconstruir a skill_captura_intimacoes.mjs inteira com nova lógica de fonte + filtro de parte autora.

🤖 Decisão: Playwright vs Vision — Quando Usar Cada Um

Playwright (95% dos casos)

  • Login + 2FA → sempre Playwright
  • Navegar lista de movimentações → Playwright
  • Clicar no número do processo → Playwright
  • Extrair tabela de eventos → Playwright
  • Clicar no link do documento → Playwright
  • Ler texto do documento → Playwright (iframe/frames)
  • Paginação da lista → Playwright

Vision / Computer Use (5% — fallback)

  • Interface mudou e selector quebrou
  • CAPTCHA ou popup inesperado
  • Documento em formato visual (imagem, não texto)
  • Inspecionar selectors com CU pontual (~$0.05)
  • PDF sem texto extraível

⚠️ Regra do Christian: NUNCA navegar por URL direta. Sempre via clique no elemento da página. Se precisar buscar processo, usar o campo "Nº de processo" no header do EPROC.

📊 Decisão: Estratégia de Volume

Opção A: Processo por Processo

  • Carrega cada processo completo
  • Mais confiável, menos risco de perder contexto
  • Mais lento para alto volume
  • Sessão pode expirar no meio
  • Simples de implementar

Opção B: Extrair Lista → Processar em Lote

  • 1ª passagem: extrai todos os CNJs da lista com movimentação
  • Filtra por janela de tempo (últimas Nh)
  • 2ª passagem: para cada CNJ, abre processo e lê documento
  • Retoma sessão se expirar (re-login automático)
  • Escala bem para 800+ processos/dia
