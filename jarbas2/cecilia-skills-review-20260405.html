🔄 Pipeline — 6 Etapas do Ciclo Diário

O orquestrador executa as 6 etapas sequencialmente para cada OAB. A cadeia completa roda diariamente às 00:00 BRT via cron interno (UTC 03:00).

// Fluxo completo por OAB — executarCiclo(account, '00h') loginEproc(account) → { browser, page, sessionHash } → skillGerarPlanilha(page) → { filePath: XLS } → skillProcessarPlanilha(filePath) → { processos[], xlsDataMap } → [para cada processo] → skillAcessarProcesso(page, cnj) → { eventos[], metadata } → [para cada evento com doc] → skillLerDocumento(page, href) → { texto } → skillAnalisarIA(movimentacoes) → { classe A-G, acao, risco } → persistirIntimacao() → DB (dedup signature_hash) → enrichProcess() → DB (NULL-only fields) → [se classe E/F] notificarImediato() → Telegram
Paralelismo

OABs processadas sequencialmente, com pausa de 5 min entre cada uma. Batch lookups paralelos (Promise.all) para as 3 queries de metadados.

Retry / Resiliência

MAX_RETRY=2 por OAB com backoff de 30s. Re-login automático se sessão expira durante a etapa 4. Circuit breaker por conta com backoff exponencial.

Deduplicação

signature_hash = SHA256(cnj|evento|dataEvento). Check antes de persistir. Também usa evento_eproc como external_id para dedup duplo.
