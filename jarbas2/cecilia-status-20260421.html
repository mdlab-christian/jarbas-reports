Secao 2 — Bugs P0 — Analise Tecnica

BUG-P0-01 S6 to S7 — classe=null em 419 intimacoes

Duas causas raiz distintas. Corrigir em ordem: Causa A primeiro (mais simples, maior impacto imediato).

Causa A — S5 retorna conteudo vazio

Quando o documento EPROC nao carrega, conteudo_completo chega em S6 com 0 chars. S6 detecta e faz SKIP — mas S7 persiste com analise_ia=null e classe=null.

Fix

Quando conteudo vazio, usar titulo do evento para analise minima — classificar como A/B/C sem ler o documento.

Causa B — S6 gera JSON malformado

Claude as vezes retorna analise_ia com JSON invalido ou fora do schema esperado. ia-output-parser.mjs falha silenciosamente — classe nao salva.

Fix

Usar Anthropic SDK tool_use em vez de JSON livre — output estruturado garantido pelo SDK.

BUG-P0-02 SP535101/TJSP — 1.250+ processos sem monitoramento

Root cause: TJSP usa e-SAJ (sistema proprio), nao EPROC. O login atual assume EPROC e falha silenciosamente.

Solucao Imediata

datajud-client.mjs ja existe no codigo — consulta CNJs via API Datajud sem login. Ativar como fallback imediato enquanto e-SAJ e desenvolvido.

Solucao Definitiva

Implementar login e-SAJ — sistema diferente do Keycloak/EPROC usado para RS e SC. Requer desenvolvimento especifico.

Atencao — Confirmacao Obrigatoria com Joao
Confirmar qual OAB e correta para TJSP (SP535101) antes de executar qualquer fix nesta OAB.
