Bug Crítico #1 — Playwright Browser Deletado (4/5 OABs INATIVAS)

P0 browserType.launch: Executable doesn't exist at /Users/jarbas/Library/Caches/ms-playwright
💀 Impacto: RS131163, SC074025, SC075466, SP535101 = 4 de 5 OABs = 80% da cobertura processual = zero análise SC e SP desde 26/04. Apenas RS116571 sobreviveu.
Causa Raiz
A variável PLAYWRIGHT_BROWSERS_PATH no .env aponta para /Users/jarbas/Library/Caches/ms-playwright. O cache do user jarbas foi deletado (limpeza automática macOS ou atualização do Playwright). O user cecilia perdeu acesso ao executável do browser Chromium.
Fix P0 — 5 minutos
// Reinstalar Playwright para o user cecilia no Mac Mini
ssh mac-mini-tail "sudo launchctl asuser 506 sudo -u cecilia bash -c ' cd /Users/cecilia/Documents/cecilia-vault/controller-agent PLAYWRIGHT_BROWSERS_PATH=/Users/cecilia/Library/ms-playwright \ /opt/homebrew/bin/npx playwright install chromium '"
// Atualizar .env após instalação bem-sucedida
PLAYWRIGHT_BROWSERS_PATH=/Users/cecilia/Library/ms-playwright
Resultado esperado: todos os ciclos (6h, 13h, 19h) voltam a funcionar com as 5 OABs. SC e SP retomam análise dos processos parados desde 26/04.

Bug Crítico #2 — S5 L5 Vision OCR captura screenshot em branco (101 falsos positivos)

P0 page.screenshot() sem aguardar carregamento do iframe PDF — gera imagem branca e falso Classe A
🚨 Impacto: 101 processos aparecem na interface /controller como "Urgentes A" quando são erros de leitura. 70% das Classe A no banco são falsos positivos. MIA dispara alertas incorretos para o escritório.
Cadeia de falha (passo a passo)
1.S5 tenta ler documento EPROC via Level 1 → 2 → 3 → 4 — todos falham
2.Fallback para L5 Vision OCR: page.screenshot() executado sem aguardar o iframe PDF terminar de renderizar
3.iframe ainda renderizando → screenshot captura pixels brancos (zero conteúdo visível)
4.Claude recebe imagem branca → responde: "A imagem aparece completamente em branco — não há texto visível..."
5.S6 usa esse texto como conteudo_completo → Claude sem contexto → classifica conservador como Classe A confiança 0.30
6.S7 persiste como urgente → S8 dispara alerta → MIA notifica o escritório incorretamente
Fix Imediato — S6: detectar imagem branca antes de analisar
// skill_analise_ia.mjs — inserir antes de chamar Claude no S6
// Detectar resposta de imagem branca (L5 OCR falhou) const padraoImagemBranca = [ 'imagem aparece completamente em branco', 'nao ha texto visivel', 'image appears completely white', 'no text visible' ]; const isImagemBranca = padraoImagemBranca.some(p => conteudo_completo.toLowerCase().includes(p) ); if (isImagemBranca) { await supabase.from('intimacoes_capturadas').update({ ia_status: 'erro_leitura', ia_erro: 'L5_IMAGEM_BRANCA: documento nao renderizou antes do screenshot', ia_classe: null }).eq('id', intimacao.id); continue; // nao analisar — aguardar proximo ciclo com fix L5 }
Fix Definitivo — S5 L5: aguardar iframe antes do screenshot
// skill_captura_documento.mjs — Level 5 Vision OCR
// Apos navegar para a pagina do documento: await page.waitForTimeout(2500); // aguardar PDF renderer inicializar // Aguardar pelo menos uma imagem com conteudo real await page.waitForFunction(() => { const imgs = document.querySelectorAll('img'); return Array.from(imgs).some(i => i.naturalWidth > 0); }, { timeout: 5000 }).catch(() => {}); // Aguardar iframe PDF visivel await page.waitForSelector('iframe, embed[type="application/pdf"]', { state: 'visible', timeout: 4000 }).catch(() => {}); // So ENTAO capturar screenshot const screenshot = await page.screenshot({ fullPage: false });

Bug P1 — Config Fetch Timeout (ruído nos logs)

P1 Config fetch timeout (10s) — 30+ repetições nos logs, mascara erros reais
O start-cron.mjs faz polling da tabela cecilia_config via REST API Supabase a cada 60s com timeout de 10s. Em conexões lentas (Mac Mini inicializando, WiFi instável), o timeout dispara repetidamente.
Impacto: Não é bloqueante — o sistema usa defaults corretos (6h/13h/19h, 5 OABs). Porém polui os logs impedindo identificar erros reais como o bug do Playwright.
Fix — aumentar timeout de 10s para 30s
// start-cron.mjs — linha ~45
// Antes: const controller = new AbortController(); const timeout = setTimeout(() => controller.abort(), 10_000); // Depois: const controller = new AbortController(); const timeout = setTimeout(() => controller.abort(), 30_000);
