Independencia do Sistema

O sistema roda 100% independente da Cecilia OpenClaw. O LaunchAgent inicia automaticamente no boot do Mac Mini, mantem-se vivo via KeepAlive, e consome tarefas de jarbas_tarefas continuamente.

Qualquer sistema que insira uma linha em jarbas_tarefas com executor='cecilia' comanda o sistema. Nao precisa de OpenClaw, SSH, ou qualquer dependencia externa.

Etapa 1: Login EPROC — skill_login_eproc.mjs

O que faz: Autentica no EPROC via Keycloak SSO com state machine deterministica.

State Machine (12 iteracoes max):

  • EPROC_LOGIN — usuario/senha direto
  • SSO_LOGIN — Keycloak federated
  • SSO_WAIT — redirect em progresso
  • 2FA — TOTP code (condicional)
  • PROFILE — selecao ADVOGADO-TITULAR
  • PANEL — sucesso (extrai hash)
  • ERROR — falha (incrementa counter)

Detalhes Tecnicos:

  • Native Setter bypass para Keycloak com @
  • Session cache 14 min em Supabase
  • Playwright headless, 1280x900, Chrome 122
  • Re-login automatico se sessao expira
  • Circuit breaker: 6 falhas → desativa OAB

E2E: OK sessionHash capturado

Etapa 2: Gerar Planilha — skill_gerar_planilha.mjs

O que faz: Navega ao relatorio de processos e baixa XLS via gerarExcel().

Navegacao: Usa DOM click (link a[href*="relatorio"]) em vez de page.goto() — page.goto quebra sessao EPROC (trigger "Selecao de perfil").

Download: Intercepta evento de download do Playwright, salva em /tmp/cecilia-downloads/movs_{oab}_{ts}.xls

Validacao: Verifica tamanho > 100 bytes. Detecta "sessao expirada" no DOM.

E2E: OK XLS baixado com sucesso

Etapa 3: Processar Planilha — skill_processar_planilha.mjs

O que faz: Parseia XLS e aplica 3 camadas de filtro.

FILTRO 1: Data

Eventos antes de desdeTimestamp descartados. Suporta Excel serial dates, DD/MM/YYYY, ISO.

FILTRO 2: Ruido

Regex patterns: DJE, DJM, Diario, publicacao, conclusao, certidao, remessa → auto Classe A.

FILTRO 3: Polo

OAB so no polo passivo (reu) → descartado. Foco em processos onde somos autores.

Fix aplicado: CNJ com codigo de vara (5047539-51.2024.8.21.0008 (CAN1CIV1J)) limpo via regex.

E2E: OK 3 processos com movimentacao filtrados

Etapa 4: Acessar Processo — skill_acessar_processo.mjs

O que faz: Busca processo por CNJ no EPROC e extrai eventos do DOM.

Busca: Seletor #txtNumProcessoPesquisaRapida, #inpPesquisarProcesso, #txtNumProcesso (3 fallbacks).

Fallback: Se busca falha, usa window.location.href (nao page.goto!) para navegar direto.

Extracao: Tabela #tblEventos / .infraTable. Por linha: data, texto do evento, links de documentos.

E2E: OK 3 processos navegados com sucesso

Etapa 5: Ler Documento — skill_ler_documento.mjs

O que faz: Extrai texto de documentos judiciais com estrategia de 5 niveis de fallback.

NivelMetodoTaxa SucessoCusto
L1DOM innerText (clone body, remove scripts/styles)~60%$0
L2Iframe content (busca iframes, extrai innerText)+20%$0
L3PDF download + pdf-parse+10%$0
L4Claude Vision (screenshot → OCR via Haiku)+8%$0.002

Limite: 50KB por documento. Delay entre documentos: 800-2500ms (anti-bot).

E2E: OK 138 movimentacoes lidas

Etapa 6: Analise IA — skill_analisar_ia.mjs

O que faz: Classifica movimentacoes usando Claude Sonnet 4.6 + playbooks do cecilia-vault.

Filtro Zero-Custo (antes da IA):

  • Eventos ruido → auto Classe A ($0)
  • Polo passivo (reu) → auto Classe B ($0)
  • Economiza ~60% dos tokens

Para o restante:

  • Modelo: claude-sonnet-4-6
  • Batches de 10 movimentacoes
  • System prompt com cache (1h)
  • Custo: ~$0.005/item

Fontes do System Prompt:

  • Vault: 7 arquivos do cecilia-vault/02-PLAYBOOKS/
  • DB: controller_playbooks (max 60 registros)
  • DB: controller_regras_operacionais (max 70)
  • DB: controller_settings (termos gatilho)

Output por item:

classe (A-G), acao, responsavel, confianca, justificativa, prazo_dias, urgente, importante, ciencia_renuncia

E2E: OK 101 classificacoes — A=81, B=8, C=1, D=9, E=2

CRITICO: Schema Mismatch analise_ia

P0-1 — O Controller NAO Consegue Ler os Dados da Cecilia

A funcao normalizeAnaliseIA() em src/utils/controller_analise_ia_utils.ts detecta 3 schemas conhecidos. Cecilia v2 nao corresponde a nenhum.

O QUE CECILIA ESCREVE:

{ "classe": "E", "nome_classe": "Providencia Juridica Estrategica", "acao": "Escalar para Dr. Christian...", "responsavel": "Dr. Christian", "confianca": 0.98, "justificativa": "CS polo passivo...", "ciencia_renuncia": false, "prazo_dias": 5, "tipo_prazo": "urgente", "via_ia": true, "modelo": "claude-sonnet-4-6", "fonte": "cecilia_v2_obsidian" }

O QUE O CONTROLLER ESPERA (Schema NOVO):

{ "_analista_raw": { "flag_ciencia_renuncia": true, "fase_processual": "inicial", "atividade_tipo": "ciencia_renuncia", "responsavel_sugerido_nome": "Leonardo", "resumo_controller": "Texto resumo...", "confianca": 90, "flag_deposito_judicial": false }, "atividade_sugerida": { "titulo": "...", "descricao": "..." }, "risco": "medio", "resumo": "..." }

Deteccao de schema:

// normalizeAnaliseIA() checa: const isNew = '_analista_raw' in raw; // Cecilia: FALSE (nao tem _analista_raw) const isSimples = 'flag_ciencia_renuncia' in raw // Cecilia: FALSE (tem ciencia_renuncia, sem flag_) || 'responsavel_sugerido_nome' in raw; // Cecilia: FALSE (tem responsavel) // Resultado: cai em "OUTROS" → retorna null/false para TUDO

Impacto: No /controller, as 101 intimacoes da Cecilia aparecem com:

  • Flags ciencia_renuncia, deposito_judicial = sempre false
  • Responsavel = null
  • Resumo IA = null
  • Atividade sugerida = null
  • Confianca = null

Fix recomendado (Opcao A — adaptar Cecilia):

Alterar skill_analisar_ia.mjs e test_ciclo_completo.mjs para escrever no formato Schema NOVO com _analista_raw wrapper. Mais robusto porque nao exige deploy do MdFlow.

Fix alternativo (Opcao B): Adicionar deteccao de "Schema CECILIA" ('classe' in raw && 'fonte' in raw) no normalizeAnaliseIA(). Exige deploy do MdFlow.

Como Comandar o Sistema

Principio: Task-Based Architecture

O sistema consome tarefas da tabela jarbas_tarefas. Qualquer sistema que insira uma tarefa comanda a Cecilia. Nao precisa de SSH, OpenClaw, ou acesso direto ao Mac Mini.

Via SQL (Supabase)
INSERT INTO jarbas_tarefas ( organizacao_id, tipo, status, executor, payload, preferred_host ) VALUES ( '55a0c7ba-...', 'orquestrador', 'pendente', 'cecilia', '{"oab":"RS116571","horario":"manual"}', 'mac-mini' );
Via Claude Code (MCP)
// No Claude Code, usar Supabase MCP: execute_sql({ project_id: 'qdivfairxhdihaqqypgb', query: 'INSERT INTO jarbas_tarefas ...' })

Para a Cecilia OpenClaw comandar, basta criar uma skill OpenClaw que faca um INSERT na tabela. Exemplo: skill "comandar_eproc" que recebe parametros e insere tarefa.

Os crons estao configurados corretamente no start.mjs.
