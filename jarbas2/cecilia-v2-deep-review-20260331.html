🔴 Problema central da triagem

O sistema tem 3 filtros de descarte que funcionam (arquivado, data-timestamp, planilha-ruído) e 3 que não funcionam corretamente (preClassificados A, isRuido em analisar_ia, IA retorna A).

O resultado é que todo processo que a Cecília classifica como A vai para o banco, independente de como foi classificado. A única coisa que não vai para o banco é o que foi filtrado no nível do XLS ou por data. Qualquer coisa que chegou no pipeline de IA vai para o banco.

Fix único e completo: Uma única regra em skill_orquestrador.mjs antes de persistirIntimacao() resolve todos os 3 caminhos:

orquestrador.mjs — o único lugar certo para filtrar
for (const r of analise.resultados) {
  // Triagem: A e B não aparecem no controller, não persistem
  // A = irrelevante | B = ciência com renúncia (não precisa de ação)
  if (r.classe === 'A' || r.classe === 'B') {
    // Garantir que timestamp foi atualizado (já está em cnjsProcessados)
    continue;
  }
  const result = await persistirIntimacao(r, cicloId, oab, tribunal, processoIdMap);
  ...
}

⚠️ Decidir sobre B: hoje B (ciência com renúncia) vai pro banco e para no controller. João decide: mostrar B ou não? Se não mostrar, adicionar || r.classe === 'B' no filtro.
