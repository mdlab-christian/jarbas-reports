Como Comandar o Sistema

A Cecilia v2.0 consome tarefas da tabela jarbas_tarefas. Qualquer sistema que insira uma tarefa nessa tabela comanda a Cecilia.

1

Via Supabase (SQL direto)

Insira uma tarefa na tabela. A Cecilia consome automaticamente.

INSERT INTO jarbas_tarefas ( organizacao_id, tipo, status, executor, payload ) VALUES ( '55a0c7ba-1a23-4ae1-b69b-a13811324735', 'ping', -- tipo da skill 'pendente', -- status inicial 'cecilia', -- quem executa '{"test": true}' -- payload JSON );
2

Via n8n (webhook)

Trigger via workflow n8n que insere na tabela.

curl -X POST "$N8N_WEBHOOK_INGEST" \ -H "Content-Type: application/json" \ -H "$N8N_WEBHOOK_AUTH_KEY: $N8N_WEBHOOK_AUTH_VALUE" \ -d '{"agent":"cecilia","skill":"orquestrador","payload":{"oab":"RS116571"}}'
3

Via Claude Code / Jarbas

Insira tarefa via MCP Supabase ou SSH direto.

# Via MCP (Claude Code): execute_sql("INSERT INTO jarbas_tarefas ...") # Via SSH (rodar script ad-hoc): ssh mac-mini-tail 'echo "010597" | sudo -S launchctl asuser 506 \ /bin/bash -c "node /Users/cecilia/cecilia-skills/src/skills/script.mjs"'
4

Via Telegram (futuro)

Bot @jarbas_masterbot pode intermediar comandos. Basta rotear a mensagem para inserir na tabela.

A Cecilia v2 aceita comandos de qualquer fonte via tabela jarbas_tarefas:

  • Jarbas (OpenClaw) — pode inserir tarefas via SQL
