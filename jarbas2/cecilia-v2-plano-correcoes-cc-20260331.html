⚙ Root Cause — Codigo

Arquivo: skill_processar_planilha.mjs linhas 7-63

// COLUMN_ALIASES detecta headers dinamicamente... const COLUMN_ALIASES = { cnj: ['processo', 'cnj', ...], assunto:['assunto', 'assunto principal'], evento: ['ultimo evento', 'evento', 'ultima movimentacao', ...], ... }; // Fallback quando deteccao de header FALHA: const DEFAULT_INDICES = { cnj: 0, classe: 1, autores: 2, reu: 3, localidade: 4, assunto: 5, evento: 6, // <-- FIXO para TJRS. TJSC pode ter ordem diferente! dataEvento: 7, dataAutuacao: 8, valorCausa: 9, };

Mecanismo de falha:

  1. detectColumnIndices(rows) tenta matchear headers da planilha com COLUMN_ALIASES
  2. Se a planilha TJSC usa nomes de coluna ligeiramente diferentes (ex: "Movimentacao" em vez de "Ultimo evento"), a deteccao retorna null
  3. Fallback DEFAULT_INDICES assume layout TJRS: coluna 5 = Assunto, coluna 6 = Evento
  4. Se no TJSC coluna 5 = Evento e coluna 6 = Assunto (ou outra permutacao), grava Assunto como tipo_evento
FIX Solucao Proposta
  1. Adicionar validacao eventoParece() — rejeita tipo_evento que contem virgulas excessivas (padrao de "Assunto"), palavras-chave juridicas como "DIREITO CIVIL", "Responsabilidade", ou length > 200
  2. Expandir COLUMN_ALIASES.evento com mais variantes: ['ultimo evento', 'evento', 'movimentacao', 'ultima movimentacao', 'ult. evento', 'mov.']
  3. Fallback inteligente: se header detection falha, tentar detectar qual coluna "parece" evento (texto curto, sem virgulas, verbos juridicos) vs assunto (categorias separadas por virgula)
  4. Log warning quando fallback e acionado: console.warn('FALLBACK_INDICES usado - layout de colunas nao detectado')

Arquivo: skill_processar_planilha.mjs linhas 7-65

Impacto estimado: Elimina ~72 dos 282 fallback D (-25.5%)

BUG-03 P0 — parseJsonArray() Contamina Batch Inteiro

🔍 O bug mais destrutivo do pipeline
💥
Quando Haiku retorna JSON levemente malformado para UM item do batch, parseJsonArray() retorna [] e TODOS os items do batch caem em fallback D. Um unico evento corrompido (ex: JavaScript como tipo_evento) pode arrastar 9 eventos legitimos para fallback.
// skill_analisar_ia.mjs:385-390 function parseJsonArray(responseText) { try { const jsonMatch = responseText.match(/\[[\s\S]*\]/); return jsonMatch ? JSON.parse(jsonMatch[0]) : JSON.parse(responseText); } catch { return []; // <-- ARRAY VAZIO = todos no batch caem em fallback } } // skill_analisar_ia.mjs:489-493 const classificacoes = parseJsonArray(response.content[0]?.text || ''); for (let j = 0; j < batch.length; j++) { const mov = batch[j]; const cls = classificacoes[j] || { classe: 'D', acao: 'Classificacao Haiku falhou', confianca: 0.3 }; // ^ se classificacoes = [], TODOS os j caem no fallback }

Mecanismo de amplificacao:

  1. Batch de 10 eventos: 1 tem JavaScript como tipo_evento (BUG-06)
  2. Haiku tenta classificar, JSON sai malformado por causa do input garbage
  3. parseJsonArray() falha no parse, retorna []
  4. TODOS 10 eventos recebem classe D, confianca 0.3
  5. Os 9 eventos legitimos que Haiku teria classificado corretamente agora sao fallback
FIX Solucao: parseJsonArrayRobusto()
function parseJsonArrayRobusto(responseText, batchSize) { // Tentar parse normal primeiro try { const jsonMatch = responseText.match(/\[[\s\S]*\]/); const arr = jsonMatch ? JSON.parse(jsonMatch[0]) : JSON.parse(responseText); if (Array.isArray(arr) && arr.length > 0) return arr; } catch {} // Parse individual por objeto — salvar o que for possivel const results = []; const objectPattern = /\{[^{}]*\}/g; let match; while ((match = objectPattern.exec(responseText)) !== null) { try { const obj = JSON.parse(match[0]); if (obj.classe) results.push(obj); } catch {} } return results; // Salva o maximo possivel em vez de descartar tudo }

Impacto estimado: Reduz fallback de batch para apenas items individualmente invalidos, salvando ~155 intimacoes que hoje caem junto.

BUG-05 P0 — TJSC Falha na Geracao de XLS

🔍 Problema

O botao "Gerar Excel" existe no TJSC no mesmo lugar que no TJRS. O problema e o CSS selector usado para clicar nele. O TJSC pode usar um ID ou classe diferente no DOM, fazendo page.click(selector) falhar.

Evidencia: OAB SC074025 processou 392 eventos em 69.2 min mas gastou $0.87 antes de falhar na geracao do XLS em algum ponto do ciclo.

FIX Solucao
  1. Criar mapa TRIBUNAL_SELECTORS com seletores por tribunal
  2. Fallback por texto do botao: page.click('button:has-text("Excel")')
  3. Screenshot de debug quando selector falha: page.screenshot({ path: '/tmp/debug-xls-fail.png' })

Arquivo: skill_gerar_planilha.mjs
