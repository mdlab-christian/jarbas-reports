🐛 Bugs Criticos Corrigidos
P1 — Skills apontando para arquivos inexistentes

21 de 26 skills registradas em start.mjs importavam de ./skills/ mas os arquivos foram movidos para ./skills-archive/. Monitoramento EPROC completamente quebrado.

CORRIGIDO Removidas todas as 21 imports quebradas. Registradas apenas 4 skills legadas + 7 novas v2.0.

P2 — Dois LaunchAgents competindo

ai.cecilia.skills e ai.cecilia.controller ambos apontavam para o mesmo start.mjs, causando risco de processos duplos.

CORRIGIDO ai.cecilia.controller.plist renomeado para .disabled.

P3 — step-executor.mjs importando modulo inexistente

Import de ai-rescue.mjs que nao existe mais. Qualquer skill usando step executor crashava.

CORRIGIDO Reescrito como executor 2 camadas (Script + Retry + Telegram).

P4 — task-consumer.mjs static import quebrado

Import estatico de skill_turbo_modo.mjs (arquivado) no topo do task-consumer crashava o Node.js inteiro na inicializacao.

CORRIGIDO Substituido por stub function inline.

P5 — OpenClaw LaunchDaemon com KeepAlive

/Library/LaunchDaemons/ai.openclaw.cecilia.plist com KeepAlive=true mantinha o OpenClaw gateway respawnando mesmo apos kill.

CORRIGIDO launchctl unload -w + renomeado para .disabled. Processos mortos.

Ciclo EPROC — Fluxo por horario

Cada ciclo processa todas as 5 OABs sequencialmente (TJRS primeiro, depois TJSC, depois TJSP). Entre cada OAB, pausa de 4.5-5.5 minutos (anti-deteccao). Cada OAB passa pelas 6 etapas do pipeline. Retry 2x com backoff de 30s em caso de falha. Tempo estimado por ciclo: ~30-45min para 5 OABs.
