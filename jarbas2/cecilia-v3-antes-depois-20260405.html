O Que Mudou

1 — Campo responsavel
⛔ Antes

Campo responsavel retornado pela IA com nome de pessoa hardcoded. Quando alguem saia da equipe, o sistema continuava atribuindo para eles.

{ C: 'Joao Victor', D: 'Joao Victor', E: 'Dra. Eloisa', F: 'Dr. Christian' }
✅ Depois

responsavel: null sempre. IA nao atribui pessoa. A equipe decide QUEM faz — a IA decide O QUE fazer.

{ tipo_providencia: 'juridica_simples', nivel_automacao: 2, responsavel: null }
2 — Novos campos no output
⛔ Antes

Output nao tinha campos de tipo/nivel. So tinha classe: 'D' e responsavel. Frontend nao sabia o nivel de automacao.

{ "classe": "D", "responsavel": "Joao Victor", "acao": "Elaborar replica" }
✅ Depois

Output agora tem tipo_providencia e nivel_automacao. Tabela intimacoes ganhou 2 novas colunas indexadas. Frontend pode filtrar por tipo.

{ "tipo_providencia": "juridica_simples", "nivel_automacao": 2, "responsavel": null }
3 — Prompts dos agentes
⛔ Antes

Prompts tinham secao "RESPONSAVEIS VALIDOS" listando todos da equipe: Stephanie, Leonardo, Marcelo, Yuri, Heloisa, Dr. Christian. Prompt com 8.000+ tokens carregando lista de pessoas.

## RESPONSAVEIS VALIDOS - Stephanie (comercial) - Leonardo (juridico) - Marcelo (juridico) - Yuri (juridico) - Heloisa (juridico) - Dr. Christian (estrategico)
✅ Depois

Secao removida completamente. Prompt mais enxuto. autoRouteResponsavel so usa cecilia_equipe para copiar prazo_dias (nao nome). Schema JSON sem campo responsavel.

// Schema JSON limpo — sem responsavel { "classe": "D", "tipo_providencia": "juridica_simples", "nivel_automacao": 2, "acao": "..." }
4 — Reprocessamento via n8n
⛔ Antes

Frontend chamava callWebhook('controller/reprocessar_ia', ...) para n8n Railway, workflow N8N-CONTROLLER-REPROCESSAR-v1. 3 nos JavaScript com prompts defasados.

frontend → callWebhook(n8n Railway) → N8N-CONTROLLER-REPROCESSAR-v1 → marca pendente (sem IA)
✅ Depois

Frontend insere diretamente em jarbas_tarefas (Supabase). Cecilia task consumer pega e executa skill_reprocessar_ia.mjs. Sem n8n. Pipeline 100% Node.js.

frontend → INSERT jarbas_tarefas (Supabase) → Cecilia consumer (poll 3min) → skill_reprocessar_ia.mjs
5 — Pipeline de reprocessamento
⛔ Antes

skill_reprocessar_ia nao existia. Havia skill_analise_intimacoes_bulk usando agente OLIVIA_INTIMACAO_MODAL (prompt desatualizado, sem vault). Nenhum polling automatico.

skill_analise_intimacoes_bulk.mjs → OLIVIA_INTIMACAO_MODAL (prompt velho) → sem vault, sem playbooks → trigger: manual apenas
✅ Depois

skill_reprocessar_ia.mjs criada com 3 modos. Usa skillAnalisarIA com vault + 52 playbooks + 67 regras. Poll automatico a cada 3 min.

skill_reprocessar_ia.mjs modos: individual · bulk · enqueue → skillAnalisarIA (vault + 52 playbooks) → poll automatico: 3min (start.mjs)
6 — Dados gravados no banco
⛔ Antes

INSERT em intimacoes gravava responsavel: r.responsavel (nome de pessoa). Campos tipo_providencia e nivel_automacao nao existiam no schema.

INSERT INTO intimacoes SET responsavel = 'Joao Victor', -- sem tipo_providencia -- sem nivel_automacao analise_ia = { responsavel: 'Joao Victor' }
✅ Depois

INSERT grava tipo_providencia e nivel_automacao. responsavel: null. JSONB analise_ia atualizado. 2 novas colunas indexadas no Supabase.

INSERT INTO intimacoes SET responsavel = null, tipo_providencia = 'juridica_simples', nivel_automacao = 2, analise_ia = { tipo_providencia: '...' }

Por Que Isso Importa

🛡️
Resistente a mudancas de equipe
Sistema nao quebra quando advogado sai ou entra. Antes: atribuia para ex-funcionarios automaticamente. Agora: null — a equipe decide na hora da execucao, nao o prompt da IA.
⚡
Pipeline 100% na Cecilia
Zero dependencia do n8n para o controller de prazos. Menos pontos de falha, menor latencia, logs centralizados no Mac Mini. Webhook do n8n pode ser desativado sem impacto operacional.
🔄
Reprocessamento automatico
Poll a cada 3 min garante que intimacoes com status pendente nunca fiquem paradas. Antes dependia de trigger manual via n8n Railway — sujeito a falhas e esquecimento.
🗄️
Dados tipados no banco
tipo_providencia e nivel_automacao permitem filtros, KPIs e automacao futura sem parsing de JSONB. Dados de primeira classe na tabela relacional.
