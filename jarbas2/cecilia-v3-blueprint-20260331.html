🔄 Logica de Roteamento 2-Tier
// Edge Function: analisar-intimacao
const tier1 = await classifyWithHaiku(intimacao, processo);

// Haiku resolve: A (descarte), B (ciencia), G (encerramento simples)
if (['A','B','G'].includes(tier1.classe) && tier1.confianca >= 0.85) {
  await saveSimpleClassification(intimacao_id, tier1);
  return; // ~80% das intimacoes resolve aqui
}

// Sonnet analisa: C, D, E, F + qualquer baixa confianca + decisoes
const tier2 = await analyzeWithSonnet(intimacao, processo, {
  includeDecisionAnalysis: tier1.decisao_detectada,
  includeDocumentAnalysis: tier1.classe === 'C',
  deepArguments: tier1.classe === 'E', // analisa argumentos + provas
});

// Persiste em 8+ tabelas via funcao goldmine
if (tier2.decisao_detectada) {
  await supabase.rpc('fn_jarbas_analise_v2_upsert', {
    p_organizacao_id, p_processo_id, p_analise: tier2
  }); // alimenta decisoes, jurimetria, julgadores, etc.
}
await updateIntimacao(intimacao_id, tier2);
Ver 13 Regras Operacionais do Joao (cecilia-vault)
R01Pauta de Julgamento = SEMPRE irrelevante (A). Nunca criar tarefa.
R02Dedup: SHA256(texto+data+cnj). Janela 2 dias. Ocultar duplicatas.
R03Tarefa aberta recente (5 dias uteis) = nova intimacao e ruido.
R04Historico MdFlow prevalece. Cruzar status atual do processo.
R05LER DISPOSITIVO REAL. Nunca confiar so no titulo do evento.
R06MULTA = SEMPRE Classe E. AJG NAO cobre multas.
R07AJG: 4 cenarios (concedida, pendente, provar, suspensao).
R08Cancelamento distribuicao: track proprio, nao e extincao generica.
R09Cumprimento sentenca: diferenciar credor vs devedor.
R10Alvara: SEMPRE checar 4 pontos (valor, pago, baixa, tipo).
R11Pericia = DESISTENCIA com prioridade. Nunca tratar como prova.
R12SP: SEMPRE checar processo relacionado 2o grau.
R13Reconvencao: verificar polo efetivo.

Vision Fallback (Fase 1)

Problema: Playwright Engessado

Scripts baseados em seletores CSS quebram quando o EPROC muda layout. Telas inesperadas (CAPTCHA, popups, manutencao) travam o pipeline.

Login failurePipeline inteiro para
Layout changeSeletores nao encontram
Popup inesperadoTimeout
CAPTCHAImpossivel automatizar
Solucao: Claude Vision como Fallback

Quando Playwright falha, captura screenshot e envia para Claude Vision para "ler" a tela.

Seletor nao encontradoScreenshot → Vision identifica elemento
Tela desconhecidaVision classifica estado da pagina
Popup inesperadoVision identifica botao de fechar
Layout mudouVision extrai dados mesmo com layout novo
Custo extra: ~$0.01/screenshot (Sonnet Vision). Usado apenas como fallback. Estimativa: <5% das navegacoes. ~$5-10/mes adicional.
