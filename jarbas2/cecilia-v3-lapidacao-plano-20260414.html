Secao 4: Arquitetura Proposta v3.0

Principio Fundamental

1 agente, 1 user, 1 vault, multiplas interfaces

Inspirado na OlivIA: vault-driven architecture onde o vault e a SOUL do agente, playbooks sao hot-reloaded, memoria e cumulativa e auto-corrigida, e multiplas interfaces (MdFlow, MdOs, Telegram) consomem o mesmo core.

Pipeline S0-S9 Corrigido

S0 PLAN

Carrega vault (agente/ + playbooks/ + memoria/). Valida existencia de SOUL.md e MEMORY.md. Define OABs e modo.

S1 LOGIN

Keycloak SSO + TOTP via otplib. Cache sessao 14min. Retry 2x antes de circuit breaker.

S2 SCRAPE

Download XLS EPROC. Parse: A=CNJ, C=autor, D=reu, H=data, I=dist, J=valor. Filtro por data.

S3 FILTER

SHA256 dedup contra existentes. Pre-classify via regex (70% sem IA = sem custo).

S4 NAVIGATE

Navega ao processo no EPROC. Abre pagina de detalhes. Preparacao para S5.

S5 READ_DOCS FIX

5-level fallback: iframe → download → OCR → raw text → screenshot. PERSISTIR conteudo_completo.

S6 ANALYZE FIX

Injetar SOUL + MEMORY + playbooks + conteudo. Sonnet 4.6. Self-review (nota 8.5). Validar tipo_providencia.

S7 PERSIST FIX

UPSERT TODOS campos: ia_classificacao, resumo_ia, conteudo_completo, data_prazo, modelo_ia, custo_ia, tokens_ia.

S8 ACT

Auto-finaliza cat A. Upsert decisao cat E. Alerta urgente cat F (SISBAJUD, multa). Tudo automatico.

S9 REPORT

Metricas do ciclo. Telegram resumo. Append MEMORY.md com licoes aprendidas.

4 Modos Operacionais

A. Monitoring Cron 3x/dia

Ciclo padrao. 300-600 eventos. Scrape + filter + analyze + persist. Pipeline S0-S9 completo.

B. Deep Analysis On-demand

Analise profunda 1-10 processos. TODOS documentos. Relatorio completo. Via MdOs/MdFlow.

C. Batch Harvest MdOs

Lote 50-500 processos. Campanhas, backfill, re-analise massiva. Custo ~$0.62/processo.

D. Training Feedback

Correcoes equipe (Telegram/MdOs), valida, atualiza playbooks e MEMORY.md. Auto-learning.

Secao 7: Brainstorming — Capacidades Futuras

7.1 Navegacao Ilimitada EPROC

Alem de XLS, navegar diretamente nas paginas do processo EPROC para extrair dados que o XLS nao contem: movimentacoes completas, documentos anexos, audiencias, valores atualizados. Potencial: +40% dados por processo.

7.2 Campanhas em Lote (Modo C)

Processar 50-500 processos de uma vez para campanhas especificas: re-analise de processos antigos, backfill de campos vazios, auditoria de classificacoes. Custo estimado: ~$0.62/processo.

7.3 Integracao Cecilia-OlivIA

Quando Cecilia detecta intimacao cat D (prazo), aciona OlivIA para gerar peticao automaticamente. Pipeline end-to-end: deteccao → leitura → classificacao → peticao → revisao. ~$0.62/peticao completa.

7.4 Auto-Enrichment de Processos

Usar dados capturados para enriquecer tabela processos: juiz atualizado, vara, status, valor causa, advogado reu. Backfill automatico dos 136 campos. 5.800 processos enriquecidos.

7.5 Guardian de Prazos

Alertas escalonados: D-5 (aviso), D-3 (urgente), D-1 (critico), D-0 (alerta vermelho). Via Telegram, email, MdFlow toast. Prioridade maxima para cat F.

7.6 Anthropic Managed Agents

Quando estabilizar, migrar para multi-agente: agent monitor, agent reader, agent classifier, agent actor. Decisao D18: aguardar maturidade.
