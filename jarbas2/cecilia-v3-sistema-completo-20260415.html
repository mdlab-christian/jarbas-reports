O Que Precisa de Correcao

🔴 Issues P1 — Serios (7 issues) Corrigir antes de confiar 100% ▼
P1-01
S3 nao e scoped por OAB
CRITICO
S3 query busca TODAS as intimacoes pendentes (ate 200), nao apenas da OAB atual. Em multi-OAB run, OAB 1 pode processar docs da OAB 2 com sessao Playwright errada (tribunal diferente). Documentos lidos com sessao incorreta.
Fix: Adicionar filtro oab ou tribunal na query S3
P1-02
Reprocessar individual usa n8n (deprecated)
INCONSISTENCIA
Botao "Reprocessar IA" no drawer chama callWebhook n8n. Bulk reprocessar usa corretamente jarbas_tarefas. Dois caminhos diferentes para a mesma acao — comportamento imprevisivel.
Fix: Substituir callWebhook por UPDATE direto + INSERT jarbas_tarefas
P1-03
ciencia_renuncia_status mismatch
BUG SILENCIOSO
Realtime escuta 'concluida', drawer verifica 'executada'. Valores diferentes — toast nunca dispara corretamente. Advogado nao recebe confirmacao visual da ciencia.
Fix: Alinhar para um unico valor ('concluida' ou 'executada') em todo o sistema
P1-04
Ignorar/Arquivar ambos setam 'finalizada'
SEMANTICA PERDIDA
Ambas as acoes bulk resultam em status='finalizada'. Impossivel distinguir no banco e nos relatorios o que foi resolvido vs. ignorado.
Fix: Implementar status distintos ('ignorada', 'arquivada') ou usar campo separado
P1-05
SC075466 circuit breaker permanente
OAB OFFLINE
OAB TJSC Heloisa com 3+ falhas consecutivas. Skipada em todo ciclo. Sem mecanismo de auto-reset. Intimacoes do TJSC Heloisa NAO estao sendo monitoradas ha multiplos ciclos.
Fix: Reset manual no Supabase (tabela circuit_breakers) OU implementar auto-reset apos 24h sem falha
P1-06
L4 PDF nao implementado (TODO-W3)
FUNCIONALIDADE AUSENTE
Documentos em PDF nunca sao lidos via extracao real. Cai diretamente para L5 (Vision OCR via screenshot). Docs extensos podem nao ser capturados — especialmente sentencas longas.
Fix: Implementar download PDF via Playwright + pdf-parse (npm) no estagio L4
P1-07
Budget cap apenas avisa, nao para
RISCO FINANCEIRO
Se custo ultrapassa $2.00/ciclo, o sistema envia Telegram mas continua processando. Em caso de bug que dispare muitas chamadas Claude, custo pode escalar sem limite automatico.
Fix: Adicionar flag para abortar pipeline apos budget excedido (com override manual via env var)
🟡 Issues P2 — Melhorias (9 issues) Nao bloqueantes ▼
P2-01
v_monitoramento_oab_status com schema drift
View falta colunas intimacoes_ultimo_ciclo e custo_ultimo_ciclo. Frontend tenta exibir, recebe undefined.
P2-02
Filtro com_deposito no caminho errado do JSONB
Busca em analise_ia.flag_deposito_judicial (root), mas dado esta em analise_ia._analista_raw.flag_deposito_judicial. Filtro nunca retorna resultados.
P2-03
KPI "Acuracia IA" hardcoded 'N/A'
Nunca foi implementado. Calcular a partir de cecilia_feedback (positivo/negativo) via query simples.
P2-04
Colunas duplicadas no schema
duplicada + is_duplicada fazem a mesma coisa. urgente (bool) + urgencia (text) sao redundantes. Confunde queries.
P2-05
merito_julgamentos FK sem ON DELETE
Se intimacao for deletada (hard delete acidental), registro em merito_julgamentos vira orfao sem referencia.
P2-06
39 indices em intimacoes — 4 pares redundantes
Indices duplicados aumentam overhead de INSERT/UPDATE. Dropar os 4 pares redundantes. Adicionar WHERE deleted_at IS NULL em 6 indices parciais.
P2-07
S5 extracao L1/L2 falham consistentemente
L1 (.infraAreaTexto) e L2 (6 seletores EPROC) falham. L3 retorna apenas ~138 chars. Documentos reais raramente extraidos via seletor DOM — layout do EPROC provavelmente mudou.
P2-08
skipDedup=true — codigo de dedup no S3 e dead code
Dedup feito via unique constraint no banco (correto). Codigo de dedup no S3 nunca executa. Pode ser removido para clareza.
P2-09
ia_risco_nivel nao preenchido no path Guard
Quando Guard finaliza classe A (FIX-13), campo ia_risco_nivel fica null. Inconsistencia nos relatorios de risco.
