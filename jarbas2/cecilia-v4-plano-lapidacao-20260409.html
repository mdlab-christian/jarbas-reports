9. Workflow por Etapas (Fluxo Ideal)

Etapa 1: SCRAPE
Login EPROC (state machine v7)
Listar processos com movimentacao desde ultimo ciclo da OAB
Salvar lista em memoria
Etapa 2: TRIAGE
Para cada processo: analisar titulo do evento (regex)
Pre-classificar: A/B → descartar | C-F → marcar para analise
Registrar: evento_id, cnj, read_doc flag
Etapa 3: DOCUMENTS
Acessar processos com read_doc=true
Navegar via ARVORE de documentos do EPROC
Baixar/extrair texto relevante → salvar em ~/cecilia/docs/{cnj}/
Etapa 4: ANALYSIS
Sonnet 4.6 + vault ON-DEMAND
Classificacao A-G com justificativa | Extracao de dispositivo
Sugestao de atividade | [Chamar Olivia se contestacao]
Etapa 5: PERSIST
INSERT/UPDATE intimacoes com campos promovidos
Upsert decisoes | Alertas Telegram (E/F/SISBAJUD)
Atualizar fase processo
Etapa 6: REPORT
Atualizar cecilia_ciclos com metadata JSONB
Telegram resumo ciclo | Log em cecilia_pipeline_log

10. Sistema de Fila/Log (Backend)

Proposta: tabela cecilia_pipeline_log
CREATE TABLE cecilia_pipeline_log ( id UUID DEFAULT gen_random_uuid() PRIMARY KEY, organizacao_id UUID NOT NULL REFERENCES organizacoes(id), ciclo_id UUID REFERENCES cecilia_ciclos(id), oab TEXT NOT NULL, cnj TEXT, etapa TEXT CHECK (etapa IN ('scrape','triage','documents','analysis','persist','report')), status TEXT CHECK (status IN ('pendente','executando','concluido','erro','skip')), input JSONB, output JSONB, erro TEXT, duracao_ms INT, created_at TIMESTAMPTZ DEFAULT now(), updated_at TIMESTAMPTZ DEFAULT now() ); CREATE INDEX ON cecilia_pipeline_log(organizacao_id) WHERE status = 'erro'; CREATE INDEX ON cecilia_pipeline_log(ciclo_id); ALTER TABLE cecilia_pipeline_log ENABLE ROW LEVEL SECURITY; CREATE POLICY "tenant_isolation" ON cecilia_pipeline_log USING (organizacao_id = public.get_org_id());
Beneficios
✓ Rastrear exatamente onde cada processo parou
✓ Identificar processos com erro em etapa especifica
✓ Reprocessar somente processos que falharam
✓ Dashboard de saude do pipeline
✓ Auditoria completa (LGPD compliance)
Queries Uteis
-- Processos com erro na analise SELECT cnj, erro, created_at FROM cecilia_pipeline_log WHERE etapa = 'analysis' AND status = 'erro' ORDER BY created_at DESC LIMIT 20;
-- Duracao media por etapa SELECT etapa, AVG(duracao_ms) as avg_ms, COUNT(*) FILTER (WHERE status='erro') as erros FROM cecilia_pipeline_log GROUP BY etapa;
