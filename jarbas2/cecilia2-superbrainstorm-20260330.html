🔄 3. Workflow Definitivo — Cecília 2.0
3 ciclos/dia: 00:00 · 12:00 · 18:00 — janela de 6h entre cada ciclo. ~200-300 movimentações por ciclo por OAB.
1

Login + Sessão MANTER — skill_login_eproc.mjs

SSO Keycloak + 2FA TOTP + seleção de perfil ADVOGADO. Pool de sessões por OAB. Re-login automático em caso de expiração. Funciona 100% — não tocar.

2

Captura Lista "Últimas Movimentações" NOVA SKILL

Clica em "Últimas Movimentações" via link do painel (com hash). Extrai CNJs com movimentação nas últimas 6h. Pagina automaticamente até atingir o limite de tempo. Retorna array de {cnj, data, evento, href_processo}.

  • Volume: ~200-300 CNJs por ciclo
  • Armazena href do processo para clique posterior (não usa URL direta)
  • Filtro de deduplicação por hash antes de processar
  • Filtro de ruído zero-custo (eventos DJEN, conclusos, pautas)
3

Processar Processo — Clique + Extração NOVA SKILL

Para cada CNJ: clica no número do processo via href capturado. Verifica se movimentação é para a parte autora. Identifica evento relevante e clica no documento. Extrai texto.

  • Filtro parte autora: descartar movimentações da parte contrária/réu
  • Acesso ao documento: via clique no link (nunca URL direta)
  • Fallback: Árvore do processo quando documento não está acessível diretamente
  • Volume: processa ~50-80 por ciclo após filtros (estimativa)
  • Re-login automático se sessão expirar no meio do lote
4

Filtro Zero-Custo MANTER — lógica existente

Aplicar lista de eventos-ruído ANTES de chamar Claude. Se evento está na lista (DJEN, concluso, pauta, citação do réu, etc.) → Classe A automático, zero tokens gastos.

  • Economiza ~40-60% dos calls de IA
  • João Victor: "não preciso saber disso — jogar fora sem analisar"
5

Análise IA com Playbooks MANTER — refinar injeção

Claude Sonnet 4.6 analisa o texto real do documento com RAG dos playbooks do João. Classifica A-G, define responsável, prazo e ação sugerida. Cache 1h no system prompt (90% off).

  • Injetar: taxonomia A-G + regras operacionais + matriz roteamento + eventos ruído
  • Cruzar com histórico MdFlow: fase, responsável, tarefas abertas
  • Saída estruturada: classificação + responsável + prazo + justificativa
6

Salvar + Notificar João MANTER

Gravar no Supabase (conteudo, ia_analise, ia_status, responsavel, prazo_sugerido). Notificar João com resumo do ciclo: N intimações analisadas, X urgentes, Y pendências.

7

Ciência Automática (B ≥90%) FASE 2

Para Classe B com confiança ≥90%: executar ciência com renúncia automaticamente. Notificar João após execução. Sempre com log auditável.

8

Fallback + Alertas P0 PERMANENTE

Se falhou em qualquer etapa → registrar como status='falha_manual' + notificar João imediatamente. Nunca perder prazo silenciosamente.

📐 Regras de Navegação (invioláveis — Christian definiu)
  • ❌ NUNCA navegar por URL direta construída manualmente
  • ❌ NUNCA usar page.goto(baseUrl + '/controlador.php?...') sem hash
  • ❌ NUNCA clicar na checkbox do processo — clicar no número
  • ❌ NUNCA fazer page.evaluate() logo após page.goto() sem esperar
  • ✅ SEMPRE extrair href do link da página e usar para navegar
  • ✅ SEMPRE aguardar waitForLoadState('domcontentloaded') após goto
  • ✅ SEMPRE usar page.frames() em vez de page.evaluate() para iframes
  • ✅ Para buscar processo: usar campo "Nº de processo" no header do EPROC
🛡️ Arquitetura de Fallback (à prova de erro)
// Camada 1 — Playwright normal await page.click(href_extraído_do_dom) // sempre via link capturado await page.waitForLoadState('domcontentloaded') // Camada 2 — Playwright retry (3x) for (let i = 0; i < 3; i++) { try { return await lerDocumento(page, cnj) } catch { await page.waitForTimeout(1500 * (i+1)) } } // Camada 3 — Re-login se sessão expirou if (await isSessionExpired(page)) { await loginEproc(oab) // re-login automático return await lerDocumento(page, cnj) } // Camada 4 — Vision fallback (5% dos casos) const screenshot = await page.screenshot() const selector = await claudeVisionInspect(screenshot, 'link do documento') await page.click(selector) // Camada 5 — Falha definitiva (nunca silencioso) await supabase.update({ status: 'falha_manual', erro: msg }) await notificarJoao('❌ Processo ${cnj} precisa de revisão manual')
