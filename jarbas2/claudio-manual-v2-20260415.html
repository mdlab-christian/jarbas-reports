Runtime que conecta Telegram ao Claude. Mantem o bot 24/7 via LaunchAgent macOS. Cada mensagem vira um prompt para o LLM, que pode executar bash diretamente no Mac.

📄 SKILL.md

O cerebro de cada comando. Sao arquivos de instrucao LLM, nao scripts executaveis. Claudio le o skill e decide como agir com base em linguagem natural mais bash inline.

🔍 cc-scanner.sh

Script que detecta todos os terminais com processos claude ativos via tmux ls mais AppleScript mais correlacao PID e TTY. Base do /status e /read.

🔄 monitor.sh

Daemon rodando a cada 3s que auto-aprova prompts CC detectados via padrao de texto. Cobre tmux e Terminal.app. Ativado por /autoapprove on.

🔔 monitor-mode.sh

Daemon separado para notificacoes ricas de mudanca de estado. Envia mensagem no Telegram quando qualquer terminal muda de active para waiting ou error.

👁️ fswatch

Monitora ~/beta-mdflow/reports/*.html e publica automaticamente no GitHub Pages ao detectar novos arquivos. LaunchAgent: ai.claudio.fswatch.

Identificadores de Terminais

§ 3

Claudio reconhece tres tipos de identificadores. Cada um tem caracteristicas diferentes de precisao e persistencia.

term:N
term:3
Referencia a janela N do Terminal.app pelo indice. O numero e atribuido pelo macOS na ordem de abertura das janelas.
Aviso: N varia se janelas forem fechadas
tmux:nome
tmux:mdflow
Sessao tmux identificada pelo nome. Persistente mesmo fechando todas as janelas do Terminal.app. Recomendado para projetos longos.
Persistente — prefira tmux sempre que possivel
cc:PID
cc:87335
Referencia diretamente pelo PID do processo claude. Uso interno pelo cc-scanner. Util para operacoes avancadas quando o indice da janela e desconhecido.
Uso avancado / interno
Identificador Persistencia Precisao keystroke Scrollback Recomendacao
term:N Nao — muda se janelas fecharem Media — vai para pane ativo Nao — so tela visivel Use para leitura rapida e screenshot
tmux:nome Sim — sobrevive ao logout Alta — send-keys preciso Sim — historico completo Use para projetos e auto-aprovacao
cc:PID Nao — PID muda a cada execucao Indireta — correlacionado ao TTY Depende do tipo Uso interno pelo cc-scanner

📊 4.1 Status e Visibilidade
/status
Estado geral de todos os terminais — Terminal.app e tmux
detalhes
O que faz

Usa cc-scanner internamente para enumerar todos os terminais com processos claude. Agrupa por tipo, classifica o estado de cada um e exibe emoji de status com contagem total e alertas.

Emojis de estado
● Ativo — rodando normalmente ● Aguardando — prompt pendente ● Erro — erro detectado no output ● Idle — processo inativo ● Desconhecido — estado indeterminado
Exemplo de output
🐙 Status — 14:35 ──────────────────────────── 📟 Terminal.app (22 janelas) 🟢 term:1 implement-load-memoria — ativo 🟢 term:2 beta-mdflow — ativo 🟡 term:6 Claude Code — aguardando (!) 🔴 term:3 Project controller — erro 🔵 term:9 ~/Documents — idle ... + 17 terminais 📦 tmux (2 sessoes) 🟢 tmux:mdflow beta-mdflow — ativo 🟢 tmux:test ~/beta-mdflow/tests — ativo ──────────────────────────── Total: 24 | 1 aguardando | 1 erro
/sessions
Lista detalhada com TTY, PID e diretorio de trabalho
detalhes
O que faz

Versao expandida do /status. Para cada terminal exibe o TTY associado (/dev/ttysNNN), o PID do processo claude, o diretorio de trabalho atual (via lsof) e o estado classificado.

Exemplo
🐙 Sessions — 14:36 ────────────────────────────────────── term:6 TTY: /dev/ttys006 PID: 87335 CWD: /Users/christian/beta-mdflow Status: aguardando
/read term:N | /read tmux:nome
Conteudo visivel do terminal especificado
detalhes
Diferenca entre term: e tmux:

tmux:nome — scrollback completo

  • Acessa historico via tmux capture-pane
  • Buffer de ate 5000 linhas (padrao tmux)
  • Contexto rico do que esta rodando

term:N — so tela visivel

  • AVISO: AppleScript so le o que esta na tela
  • AVISO: Sem acesso ao historico de scroll
  • DICA: Use /screen para captura visual
Uso
/read term:6 /read tmux:mdflow
/tail term:N [linhas]
Ultimas N linhas do terminal. Default 100, max 500.
detalhes
Exemplos
/tail term:3 # ultimas 100 linhas (padrao) /tail term:3 50 # ultimas 50 linhas /tail tmux:mdflow 200 # ultimas 200 linhas da sessao tmux
✅ 4.2 Aprovacao
/approve
Varre todos os terminais e aprova prompts pendentes
detalhes
Como funciona

Varre todos os 24 terminais (Terminal.app e tmux), le o conteudo de cada um, identifica padroes de prompt e decide se aprova (envia "y" + Enter) ou bloqueia (notifica sem agir).

Padroes APROVADOS automaticamente

  • Do you want to proceed
  • (y/n)
  • Allow this action
  • [1] Yes
  • Continue?
  • Proceed?

Padroes BLOQUEADOS — notifica sem aprovar

  • rm -rf
  • force push
  • DROP TABLE
  • DROP DATABASE
  • irreversible
  • permanently delete
Exemplo de output
Aprovacao manual — 14:36 Varrendo 24 terminais... • term:6 → "Do you want to proceed? (y/n)" → y • term:12 → nenhum prompt detectado • term:3 → "rm -rf dist" — BLOQUEADO 1 aprovado | 1 bloqueado | 22 sem prompts
/autoapprove on | off
Liga/desliga daemon de auto-aprovacao continua (monitor.sh)
detalhes
Funcionamento

Quando ativo, o monitor.sh roda em loop a cada 3 segundos, varrendo todos os terminais e aprovando prompts seguros automaticamente. Notifica no Telegram apenas quando ha aprovacoes ou bloqueios.

⚠ Use com cuidado em sessoes onde operacoes destrutivas podem surgir. O auto-approve nunca aprova padroes da lista de bloqueio, mas pode aprovar prompts novos nao mapeados se o texto for ambiguo.
Uso
/autoapprove on # liga daemon monitor.sh — loop de 3s /autoapprove off # desliga daemon
🖱️ 4.3 Controle de Terminais
/run term:N <cmd> | /run tmux:nome <cmd>
Envia comando para o terminal especificado
detalhes

/run tmux:nome — send-keys (preciso)

  • OK: tmux send-keys — texto exato
  • OK: Nao depende do app em foco
  • OK: Funciona mesmo com vim aberto

/run term:N — keystroke (best-effort)

  • AVISO: AppleScript System Events
  • AVISO: Vai para o pane ativo
  • AVISO: Pode falhar com vim/nano aberto
Exemplos
/run term:6 y # envia "y" + Enter via AppleScript /run tmux:mdflow git status # executa no tmux (preciso) /run tmux:test npm run build # build no terminal tmux:test
/new <titulo> <cmd>
Abre nova janela Terminal.app ou cria sessao tmux
detalhes
Exemplos
/new "olivia-w10" "cd ~/beta-mdflow && claude" /new tmux:auditor "cd ~/beta-mdflow && claude"
/kill term:N | /kill tmux:nome
Encerra terminal — SEMPRE pede confirmacao antes
detalhes
⚠ O /kill sempre exibe uma confirmacao antes de executar. Responda "sim" para confirmar ou "nao" para cancelar. Nunca age imediatamente.
Exemplos
/kill term:3 # fecha janela 3 do Terminal.app /kill tmux:old # mata sessao tmux "old"
/broadcast <msg>
Envia texto para TODOS os terminais ativos
detalhes
ℹ O /broadcast exibe uma previa com todos os terminais que receberao o texto antes de enviar. Util para interromper uma operacao em todos os terminais simultaneamente.
Exemplo
/broadcast Ctrl+C # envia Ctrl+C para todos os terminais

Mecanismos Tecnicos em Detalhe

§ 5

Como o cc-scanner detecta terminais — 7 passos

Listar sessoes tmux — nomes, janelas e panes ativos
tmux ls
Contar e enumerar janelas do Terminal.app via AppleScript
osascript -e 'tell application "Terminal" to count windows'
Para cada janela: obter o TTY do tab selecionado
osascript -e 'tell app "Terminal" to tty of selected tab of window N'
Listar todos os processos claude em execucao com seus TTYs
ps aux | grep claude
Correlacionar TTY com PID: /dev/ttys006 → campo tty "s006" no ps
TTY /dev/ttys006 → ps tty "s006" → PID 87335
Obter diretorio de trabalho atual do processo
lsof -p 87335 | grep cwd
Ler conteudo visivel e classificar estado do terminal
analise de padroes: "Do you want", "Error:", "Done" → ativo / aguardando / erro / idle

Como o AppleScript keystroke funciona

# Enviar "y" + Enter para janela N do Terminal.app tell application "System Events" tell process "Terminal" tell window N keystroke "y" key code 36 # key code 36 = Enter/Return end tell end tell end tell
⚠ Limitacao critica: o keystroke vai para o pane ativo na janela. Se vim, nano ou qualquer outro editor estiver aberto, o caractere "y" sera inserido no editor, nao no prompt CC. Use tmux send-keys para precisao garantida.

Como o screencapture de janela especifica funciona

# 1. Obter CGWindowID — ID interno da janela no macOS WID=$(osascript -e 'tell application "Terminal" to get id of window N') # 2. Capturar APENAS aquela janela (nao toda a tela) screencapture -l $WID /tmp/claudio-screen-$(date +%Y%m%d-%H%M%S).png # 3. Publicar no GitHub Pages cp /tmp/claudio-screen-*.png ~/jarbas-reports/jarbas2/ cd ~/jarbas-reports git add jarbas2/claudio-screen-*.png git commit -m "screen: term:N $(date +%Y%m%d-%H%M%S)" git push origin main # 4. Retornar link publico no Telegram echo "https://mdlab-christian.github.io/jarbas-reports/jarbas2/claudio-screen-TIMESTAMP.png"

Limitacoes Conhecidas e Workarounds

§ 7
Limitacao Causa Tecnica Workaround
/read term:N sem scrollback AppleScript so acessa o conteudo visivel na tela no momento da leitura Use /screen term:N para captura visual ou tmux para scrollback completo
/run term:N pode errar Keystroke via System Events vai para o pane ativo — vim/nano intercepta o caractere Prefira sempre /run tmux:nome com send-keys
Numeracao term:N muda macOS renumera janelas quando janelas anteriores sao fechadas Use /sessions para encontrar o indice atual antes de interagir
/screen requer GUI ativa screencapture -l depende do WindowServer do macOS Sempre disponivel quando o Mac esta no desktop. Nao funciona via SSH puro
Auto-approve pode ser cego ao pane monitor.sh verifica conteudo mas nao sabe qual app tem foco no pane exato O sistema sempre verifica o texto antes de enviar "y". Usar tmux quando possivel
Latencia de screenshot Processo inclui captura + git add + commit + push Leva 5-10s dependendo da conexao. Use /read para leitura rapida de texto
/broadcast sem confirmacao de recebimento Entrega de keystroke via AppleScript e fire-and-forget Apos broadcast, use /status para confirmar o estado dos terminais
