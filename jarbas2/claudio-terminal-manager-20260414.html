02 — Arquitetura

A Solução: 4 Camadas de Controle

De baixo para cima — fundação que já funciona, mais camadas progressivas de visibilidade.

1
CC Settings — Auto-Aprovacao Nativa
Ja Implementado

dangerouslySkipPermissions: true no settings.json. Resolve ~99% dos prompts nativamente. Funciona para TODOS os terminais sem nenhum daemon.

↕
2
AppleScript Monitor — Terminal.app Scan
Novo — Core

Daemon escaneia as 21 janelas Terminal.app a cada 3s. Detecta prompts, auto-aprova via keystroke direcionado. Correlacao precisa via TTY.

# Correlacao PID ↔ Janela ps -p 87335 -o tty= # → "s001" tty of selected tab of window N # → "/dev/ttys001" # Match! Janela term:2 encontrada.
↕
3
cc-wrapper — Log Persistence
Opt-in

Shell function que envolve claude com script -q. Captura todo I/O em /tmp/cc-logs/. Logs persistem apos fechar janela. Permite /read de sessoes antigas.

function claude() { SESSION_ID="cc-$(date +%s)" LOG="/tmp/cc-logs/$SESSION_ID.log" script -q "$LOG" command claude "$@" }
↕
4
tmux — Controle Total
Recomendado

Para novos projetos: alias cc abre sessao tmux automaticamente. Scrollback infinito, input preciso via tmux send-keys. Zero ambiguidade.

alias cc='_cc_new' _cc_new() { local nome="${1:-cc-$(date +%H%M)}" tmux new-session -d -s "$nome" "claude" \ && tmux attach -t "$nome" }

06 — Ideia I2

Watchdog — Sessoes Travadas

Se um processo CC ficar 10+ minutos sem output, Claudio alerta com contexto completo.

⚠️ term:7 parou de responder
Processo: claude (PID 75815)
Working dir: ~/beta-mdflow
Ultimo output: ha 18min
📸 Screenshot 🔪 Kill ⏩ Ignore
Quando dispara

Se um processo CC ficar 10+ minutos sem output, Claudio alerta. Com screenshot opcional para ver o estado visual antes de decidir.

O ⏩ Ignore silencia o alerta por 30min — util quando o processo esta fazendo algo lento propositalmente (build, download grande).

09 — Honestidade

Limitacoes — Ser Honesto e Importante

⚠️
Terminal.app: sem scrollback
AppleScript so le o que esta VISIVEL na tela. Para historico completo: use /screen para screenshot ou prefira tmux que tem scrollback infinito.
⚠️
Auto-approve e best-effort
O keystroke vai para a janela. Se vim/nano estiver aberto, pode ir pro lugar errado. O monitor verifica o conteudo visivel antes de agir para minimizar falsos positivos.
⚠️
Numeracao instavel
term:3 hoje pode ser term:2 amanha se voce fechar uma janela. Usamos titulo da janela como referencia secundaria para maior resiliencia.
💡
tmux e o modo ideal
Para novos projetos, use cc nome. Scrollback infinito, input preciso, zero ambiguidade. Terminal.app e para o que ja esta aberto agora.
