Grupo B — Médio Porte

~30–60 min cada
06
Filtro de advogados: listar todos (remover dedup por user_id)
Médio
Arquivo src/pages/controller/ControllerV4FilterBar.tsx
Bug atual A query em linha 176 busca todos os advogados sem filtro ativo, porém o useMemo em linha 191 remove duplicatas por user_id — Christian Paz tem OAB/RS e OAB/SC, então aparece como 1 entrada só. Isso é intencional pela lógica atual mas o usuário quer ver todos os advogados individualmente.
Solução Remover o useMemo de dedup e exibir cada advogado individualmente no dropdown, com nome + OAB/UF como label (ex: "Christian Paz — OAB/RS 116571"). Ajustar a lógica de seleção para não usar advogadoIdsByUser e sim IDs diretos.
⚠️ Impacto no filtro
Hoje selecionar "Christian Paz" automaticamente filtra pelos IDs de ambas as OABs dele. Com dedup removido, cada OAB vai ser filtrada separadamente. Confirmar se é comportamento desejado.
07
Bug: "Ver processo" e "Outros processos" → URL errada
Médio
Arquivo src/pages/controller/ControllerV3DrawerTabProcesso.tsx
Bug Linha 208: navigate(`/processos/${processoId}`) — usa UUID
Linha 405: navigate(`/processos/${p.id}`) — usa UUID
A rota /processos/:cnjSlug espera o CNJ convertido para slug (pontos → hifens).
Solução Linha 208: mudar para navigate(`/processos/${cnjToSlug(processo?.numero_cnj ?? '')}`)}
Linha 405: mudar para navigate(`/processos/${cnjToSlug(p.numero_cnj!)}`)}
O objeto processo já inclui numero_cnj (linha 54 da query). O objeto p dos outros processos também inclui numero_cnj (linha 98 da query). Importar cnjToSlug de @/utils/cnjUtils.
08
Documentos anexados: adicionar botão de excluir
Médio
Arquivo src/pages/controller/ControllerV3DrawerTabDocumentos.tsx
Situação atual Docs do bucket controller-documentos só têm botão de visualização (Eye). Upload funciona via supabase.storage.from('controller-documentos').upload(path, file) com path {orgId}/{intimacao.id}/{timestamp}_{filename}.
Solução Adicionar botão Trash2 (lucide) ao lado do Eye para docs do bucket controller-documentos. Implementar handleDeleteDoc(doc) que chama supabase.storage.from('controller-documentos').remove([`${orgId}/${intimacao.id}/${doc.nome}`]) e invalida a query após delete. Botão delete só aparece para bucket controller-documentos — docs inline e do bucket processos ficam sem delete (pois são do processo, não da intimação).
Adicionar confirmação (window.confirm ou AlertDialog) antes do delete.

12
Bug: docs pendentes do processo não aparecem no drawer
Investigação
Arquivo src/pages/controller/ControllerV3DrawerTabDocumentos.tsx
Query atual Linha 39–47: busca documentos_pendentes WHERE processo_id = intimacao.processo_id AND organizacao_id = orgId AND deleted_at IS NULL. Query parece correta.
Possíveis causas 1. RLS da tabela documentos_pendentes — pode estar restringindo acesso pelo perfil do usuário logado.
2. processo_id mismatch — o intimacao.processo_id pode não corresponder ao campo processo_id em documentos_pendentes.
3. Query key stale — staleTime: 30_000 pode estar servindo cache vazio.
Diagnóstico Executar no Supabase MCP: SELECT * FROM documentos_pendentes WHERE processo_id = 'uuid-do-processo' AND deleted_at IS NULL LIMIT 10;
Verificar RLS com SELECT * FROM pg_policies WHERE tablename = 'documentos_pendentes';
