1. Executive Summary

Diagnostico Geral

O sistema Controller + Cecilia esta essencialmente parado ha 7 dias. A taxa de sucesso e de 31% (40 de 128 ciclos completados). As 15 intimacoes ativas sao todas de hoje, capturadas no TJRS em ciclo manual.

O problema nao e a IA (vault training funciona, 2-tier classification OK) nem as skills individuais (todas funcionam em serie). O root cause e concorrencia: o orquestrador executa Promise.all() em todos os tribunais simultaneamente, causando conflitos de sessao, race conditions e bloqueios mutuos no Playwright.

4 Root Causes Identificados:
  • RC-1: Promise.all() executa 5 OABs em paralelo — skills compartilham browser context
  • RC-2: Session cache grava expires_at='1970-01-01' (epoch) — toda sessao e tratada como expirada
  • RC-3: SP535101/TJSP do2FA timing — campo OTP nao renderizado quando fill() executa
  • RC-4: Dual pipeline (Cecilia + n8n ingest) com schemas divergentes

10. Brainstorming — Ideias para Evolucao

🛡️ Idea 1: Self-Healing Skill
Quando login falha 2x consecutivas:
  • Screenshot do estado atual do browser
  • Envia para Claude Vision (Haiku) → identifica: "campo senha vazio", "CAPTCHA apareceu", "pagina timeout"
  • Aplica fix automatico: retry com insertText, esperar 30s, trocar user-agent
  • Se falhar 3x → alerta Telegram + desativa OAB temporariamente
Custo estimado: ~$0.02/investigacao
RECOMENDADO — Sprint 3
🤖 Idea 2: Super-Agent Unificado (pattern OlivIA)
Consolidar 11 skills em 1 agente com 6 modos:
  • CAPTURE (login + XLS)
  • PROCESS (parse + metadata)
  • ANALYZE (IA 2-tier)
  • MONITOR (health + alerts)
  • HEAL (self-correction)
  • REPORT (HTML + vault)
Pros: contexto unificado, menos handoff. Contras: 1 bug afeta tudo, mais dificil de debugar.
POSTERGAR — Estabilizar primeiro
⚡ Idea 3: Edge Function Trigger (pattern OlivIA)
Em vez de cron push, usar DB trigger event-driven:
  • INSERT em cecilia_capture_queue → Edge Function dispara
  • EF chama Mac Mini via webhook → skill executa
  • Resultado volta via Supabase insert
Pros: event-driven, escalavel. Contras: complexidade adicional, latencia webhook.
AVALIAR — Apos Sprint 2
🎯 Idea 4: Confidence Cascading
Se analise_ia.confianca < 0.7:
  • Re-analisar com Opus 4.6 (modelo superior)
  • Se ainda < 0.7 → flag para revisao humana
  • Decisao humana alimenta feedback loop
Custo: ~$0.15/re-analise (vs $0.03 Sonnet)
RECOMENDADO — Sprint 3
📊 Idea 5: Dashboard Unificado
Conectar monitoring tab diretamente ao agente:
  • Botao "Executar ciclo agora" por OAB
  • Timeline visual dos 6 stages com status real-time
  • Graficos: success rate 7d/30d, custo por ciclo, intimacoes por dia
  • Alertas inline quando OAB cai
