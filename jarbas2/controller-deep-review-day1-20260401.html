Roadmap de Melhorias

15 ideias priorizadas
QUICK WIN

1. Circuit Breaker Inteligente

Implementar backoff progressivo em vez de desativacao permanente. Apos 5 falhas, aguardar 30min e tentar novamente. Nunca desativar todas OABs simultaneamente — manter pelo menos 1 ativa.

QUICK WIN

2. Auto-Recovery de OABs

Cron a cada 1h que verifica contas desativadas e tenta reativar com um login de teste. Se sucesso, reativa automaticamente. Elimina intervencao manual.

QUICK WIN

3. Fix Telegram Notifications

Configurar chat_id correto na Cecilia. Adicionar fallback: se Telegram falhar, enviar via Supabase push + email. Alertas urgentes (E/F) nao podem ser silenciosos.

QUICK WIN

4. Auto-Reporting por Ciclo

Gerar .md em ~/Desktop/relatorios controller/ apos cada ciclo. Incluir: OABs, processos, classificacoes, custos, erros. Consolidado diario automatico.

MEDIO ESFORCO

5. Otimizacao de Custo IA

Aumentar cobertura do regex pre-filtro para classe A (60-70% das movimentacoes). Elevar threshold Haiku de 85% para 90% antes de escalar para Sonnet. Meta: $0.34/ciclo.

MEDIO ESFORCO

6. Persistir Classe A no Banco

Salvar registros classe A com flag classe_cecilia='A' para metricas de volume. View ja filtra A, entao UI nao e afetada. Permite calcular taxa real de triagem.

MEDIO ESFORCO

7. Dashboard Operacional

Pagina /admin/cecilia com: status em tempo real de cada OAB, ultimo ciclo, erros, custo acumulado, grafico de volume por dia. Usar cecilia_ciclos + cecilia_custo_ia.

MEDIO ESFORCO

8. Alinhar Schema analise_ia

Padronizar output JSON da IA para incluir: classificacao, responsavel_sugerido, prazo.data, prazo.tipo. Atualizar prompt da Cecilia + validar campos no orquestrador.

MEDIO ESFORCO

9. Health Check Endpoint

Endpoint HTTP na porta 19200 que retorna: status do servico, ultima execucao, OABs ativas, erros recentes, memoria/CPU. Util para monitoramento externo (Uptime Robot).

MEDIO ESFORCO

10. Retry Inteligente por OAB

Se uma OAB falha no ciclo, agendar retry isolado em 15min em vez de esperar proximo ciclo (6h). Maximiza cobertura sem sobrecarregar EPROC.

GRANDE ESFORCO

11. Mode C — Harvest Batch

Popular historico dos 14.455 processos. 100-200/dia, ~90 dias. Preencher analise_ia para processos sem classificacao. Pre-requisito: Modes A+B estaveis por 2 semanas.

GRANDE ESFORCO

12. Comparacao Automatica Advbox

Importar planilha Advbox periodicamente e gerar diff: "Advbox capturou X que Cecilia nao pegou". Util durante transicao. Pode rodar como cron semanal.

GRANDE ESFORCO

13. Prompt Evolution Loop

Feedback loop: controller marca intimacoes como "classificacao errada" -> acumula -> ajusta prompt da IA mensalmente. Meta: reduzir classe D (fallback) para menos de 15%.

MEDIO ESFORCO

14. Alertas Escalonados

Classe E/F -> Telegram imediato. Classe C urgente -> digest 1x/hora. Classe B -> digest diario. Evita fadiga de notificacao. Configuravel por usuario.

QUICK WIN

15. Log Rotation + Error Tracking

Garantir que erros vao para cecilia_erros (nao so jarbas_logs). Adicionar campo error_count no cecilia_ciclos. Facilita diagnostico pos-mortem.
