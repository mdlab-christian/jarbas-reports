I-01
Taxa de Login 32% -- Sistema Inoperante 68% do Tempo
CRITICAL 68% downtime
39 concluidos / 123 total em 7 dias = 31.7%. O sistema falha no Stage 1 (login) na maioria dos ciclos. Provavel rate limiting ou SSL issues no Keycloak EPROC. SP535101 e a pior OAB (18.4%).
Correcao: Investigar SSO_TIMEOUT + exponential backoff + KC-restore agressivo | Esforco: 4-8h
I-02
Campo ultimo_login_sucesso NAO Atualizado
CRITICAL Metricas falsas
Todas as OABs mostram 2026-03-27 como ultimo login, mas ciclos concluidos hoje existem. O UPDATE nao dispara apos login bem-sucedido.
Fix: skill_orquestrador.mjs -> UPDATE jarbas_accounts SET ultimo_login_sucesso = NOW() | Esforco: 30min
I-03
Session Cache expires_at = epoch (1970-01-01)
CRITICAL KC-restore quebrado
Todas as 5 OABs com expires_at = '1970-01-01 00:00:00'. O TTL nao esta sendo gravado no INSERT/UPDATE. KC-restore nunca funciona porque session sempre parece "expirada".
Fix: skill_login_eproc.mjs -> SET expires_at = NOW() + INTERVAL '14 minutes' | Esforco: 30min
