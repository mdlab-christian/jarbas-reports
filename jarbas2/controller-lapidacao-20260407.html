Problemas Críticos

P0 — Bloqueante
P0 Bloqueante
524 Chromiums com leak
Cleanup não funciona, sistema vai crashar. Precisa forçar kill de processos órfãos + limitar pool de browsers. Cada ciclo adiciona instâncias sem remover as anteriores.
P0 Bloqueante
State machine login UNKNOWN — TJRS/TJSC/TJSP
Após SSO Keycloak, sistema detecta estado UNKNOWN em vez de PANEL. Pipeline para na etapa 1 para todas as OABs afetadas. Raiz: fluxo de redirecionamento SSO não mapeado na state machine.
P1 — Alto Impacto
P1 Alto Impacto
15.267 itens pendentes na fila enrichment
73% dos 20.883 itens em cecilia_enrichment_queue estão pendentes. Velocidade de processamento não acompanha entrada. Possíveis itens stale com status 'processando' há mais de 1h.
P1 Alto Impacto
Sem LaunchAgent instalado
Cecilia v2.3 rodando manualmente. Reboot do Mac Mini = sistema para completamente. Nenhuma auto-recuperação em caso de crash.
P1 Alto Impacto
Crons só às 00:00 — sem ciclos diurnos
Intimações geradas pelo EPROC às 14h só são processadas à meia-noite. Urgências da tarde ficam sem análise por até 10 horas.
P1 Alto Impacto — Frontend
ImportModal OAB — dropdown incompleto
Dropdown mostra apenas OABs de cecilia_capture_config, não todos os advogados da tabela. Impede importar lote por advogados não configurados no ciclo automático.
P1 Alto Impacto — Frontend
Botão "Analisar" sem estado de carregamento
Quando análise já está pendente na fila, botão não exibe loading state. Usuário clica múltiplas vezes pensando que não funcionou, gerando análises duplicadas.
P1 Alto Impacto — Frontend
Gerar Petição abre na mesma aba
Modal abre /olivia na mesma aba, perdendo o contexto do /controller. Deveria usar window.open('_blank').
P1 Alto Impacto — Frontend
Rodapé do drawer com ação incorreta
"Analisar Processo" no rodapé deveria ser substituído por "Gerar Petição" como CTA primário. Hierarquia de ações invertida.
P2 — Melhoria
P2 Melhoria
Taxa de sucesso 60.7% (34/56 ciclos)
Meta é chegar a 90%+. Principal causa raiz: state machine UNKNOWN + leaks de browser que acumulam ao longo dos ciclos.
P2 Melhoria
5 queries paralelas no useMonitoramentoCecilia
Hook faz 5 queries separadas para fila de monitoramento. Deveria ser 1 COUNT GROUP BY status via RPC.
P2 Melhoria
Sem indicador visual de Cecilia offline
Se Cecilia ficar offline por mais de 3 minutos sem heartbeat, frontend não exibe alerta visualmente distinto. Usuário não sabe se sistema está parado.
