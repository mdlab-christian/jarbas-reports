6. Issues Criticos — Acao Necessaria

P0 Realtime query key mismatch no frontend
useControllerRealtime.ts invalida controller-intimacoes e controller-kpis, mas os componentes V4 usam controller-v4-intimacoes e controller-v4-kpis. Resultado: quando Cecilia insere novas intimacoes, a pagina nao atualiza em tempo real.
Fix: Atualizar query keys em useControllerRealtime.ts para usar prefixo 'controller-v4-'
P1 RPC rpc_controller_finalizar nao existe
PRD S11.2 define a RPC para finalizar intimacoes individuais (botao Finalizar no Drawer). A RPC nunca foi criada. O frontend usa batch_update_status como workaround, mas sem atomicidade de concluida_em/concluida_por_id.
Fix: Criar RPC conforme spec S11.2 — UPDATE status=finalizada + concluida_em + concluida_por_id
P1 6 campos IA metrics com 0% populacao
resumo_ia, data_prazo, modelo_ia, custo_ia, tokens_ia, analise_versao — todos 0%. As colunas existem mas o pipeline E2E nao as popula. O script e2e_v3_ultima_mov.mjs nao seta esses campos. A Edge Function deveria popular, mas o pipeline atual bypassa a Edge Function (classifica no Node.js via skillAnalisarIA).
Fix: Quando ativar crons Mode A, garantir que Edge Function popula modelo_ia, custo_ia, tokens_ia, analise_versao. Para resumo_ia e data_prazo, adicionar ao prompt de classificacao.
P1 Inflacao de urgentes/importantes na classificacao
45.8% urgentes e 70.1% importantes vs expectativa PRD de ~10-15% e ~20-30%. Classe E em 30.6% vs meta de 3-5%. Se tudo e urgente, nada e urgente. O prompt de classificacao provavelmente esta com threshold muito baixo para essas flags.
Fix: Revisar prompt da Edge Function. Calibrar com Joao apos 1-2 semanas. Considerar: urgente=true APENAS para E/F, importante=true APENAS para C/D/E/F.
P1 KPI cards "Urgentes" e "Vencem hoje" nao filtram
Em ControllerV4KPIs.tsx, os cards "Urgentes" e "Vencem hoje" tem filterPatch: { status: [] }. Clicar neles nao filtra a lista — diferente dos cards "Nao lidas" e "Estrategicas E/F" que funcionam.
Fix: Implementar filterPatch com filtro urgente=true e prazo=hoje respectivamente
P1 advogado_reu a 0.3% — routing matrix quebrada
Apenas 40 de 14.455 processos tem advogado_reu_nome preenchido. O playbook de Joao define routing por empresa (Stephanie=financeiras, Leonardo=Boa Vista, Marcelo=varejo). Sem advogado_reu, o routing automatico nao funciona.
Fix: Priorizar Phase 6 (Mode C harvest) para popular esse campo. Investigar seletor DOM no EPROC.
P2 5 registros com analise_ia sem _analista_raw wrapper
96.5% dos registros tem o wrapper correto. 5 registros legados (provavelmente cecilia_v2 antigos) usam formato flat. O normalizer no frontend (controller_analise_ia_utils.ts) lida com ambos formatos, entao nao ha impacto funcional imediato.
Fix: Migrar os 5 registros via SQL UPDATE para formato _analista_raw (baixa prioridade)
P2 Crons Cecilia PAUSED — zero monitoramento automatico
Os crons 3x/dia (06h, 13h, 19h) estao desativados. Todo dado atual veio de execucoes manuais E2E. Sem crons, o mecanismo nao alimenta automaticamente.
Fix: Verificar deploys no Mac Mini via SSH, reativar LaunchAgent apos confirmacao
