Executive Summary

O Super Agente Controller e um pipeline S0-S9 que automatiza a triagem de intimacoes judiciais do EPROC para o escritorio MP Advocacia (Midas). Apos investigacao completa com 8 terminais paralelos, os resultados sao:

DEAD ON ARRIVAL
Zero intimacoes processadas apesar de 61 ciclos registrados como "concluidos". Pipeline inteiro travado por cascata de falhas.
RECUPERAVEL
14 fixes atomicos desbloqueiam o pipeline. Frontend 75% funcional. Vault 7.9/10 de qualidade. Estimativa: 4-5 sessoes CC.
Causa raiz: cascata de falhas iniciando em ghost columns (SELECT referencia colunas inexistentes), propagando por status invalidos (CHECK constraints ignorados) e culminando em return values descartados. O pipeline executa sem erros visiveis, mas processa zero dados.

Sidebar Bug — Root Cause + Fix

Problema:

Sidebar EPROC fica colapsada em TJSC/TJSP. page.$(selector) encontra elementos no DOM oculto, click() dispara mas nao navega. Falha silenciosa — zero erros no console.

Consequencia:

CNJ retorna null (nao causa — consequencia). A skill tenta navegar via sidebar, falha, e toda captura da OAB retorna vazia.

FIX A (RECOMENDADO): Mover URL navigation para Tier 1

const baseUrl = page.url().match( /^https?:\/\/[^\/]+/ )?.[0]; await page.evaluate((url) => { window.location.href = url; }, `${baseUrl}/eproc/controlador.php` + `?acao=relatorio_processo_` + `procurador_listar` + `&ord_ultimas_movimentacoes=S`);

Esta unica mudanca resolve SIDEBAR BUG + CNJ=null simultaneamente.

URLs EPROC sao assinadas (hash CSRF)
Nunca construir URLs EPROC manualmente. Sempre extrair do DOM ou usar URL fixa do painel (controlador.php). Links dentro do sistema contem hash de sessao que expira.

CronJob + LaunchAgent

start-cron.mjs
Scheduler atual com PID lock, schedule 6h/13h/19h BRT, suporte daemon + --once
node start-cron.mjs --once # executa 1x node start-cron.mjs # daemon node start-cron.mjs --dry-run # nao persiste
Problema atual
Se Mac reinicia ou terminal fecha, cron morre silenciosamente. Nenhum mecanismo de restart automatico.
LaunchAgent Proposto
<plist version="1.0"> <dict> <key>Label</key> <string>com.mdflow.controller-agent.dev</string> <key>KeepAlive</key> <true/> <key>RunAtLoad</key> <true/> <key>StandardOutPath</key> <string>~/Library/Logs/mdflow/controller.log</string> <key>StandardErrorPath</key> <string>~/Library/Logs/mdflow/controller-err.log</string> </dict> </plist>
Instalar em ~/Library/LaunchAgents/. KeepAlive garante restart automatico apos crash ou reboot.
