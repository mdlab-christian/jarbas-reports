Advogado — lógica de fonte

Christian perguntou: puxar de advogados ou de intimações + processos?

Resposta: Puxar de advogados da org com JOIN em intimações — só mostrar advogados que TÊM intimações pendentes no período selecionado. Isso garante que a lista não fica gigante com advogados inativos.

query sugerida
SELECT DISTINCT a.id, a.nome, a.oab
FROM advogados a
JOIN intimacoes i ON i.advogado_id = a.id
WHERE a.organizacao_id = $org
  AND i.status != 'arquivada'
  AND i.classe_cecilia != 'A'
ORDER BY a.nome

🎓 Aba Playbooks — Especificação

Conceito — Christian explicou direitinho

A ideia é ter uma interface onde o João configura o comportamento da IA sem precisar editar scripts diretamente. A IA que analisa intimações (Haiku + Sonnet via Cecília) lê os playbooks do cecilia-vault/playbooks/ como contexto. O frontend precisa ser a interface de edição desses playbooks.

Funcionalidades

  • Lista de playbooks ativos: taxonomia-intimacoes, regras-joao, ruido-patterns, matriz-roteamento
  • Editor inline de cada playbook (Markdown)
  • Novos padrões detectados: lista de eventos que a IA encontrou com frequência mas não tem regra — João classifica aqui
  • Histórico de decisões humanas: o João tomou X → IA sugeria Y — aprendizado
  • Toggle de regras: ativar/desativar padrões de ruído individualmente

Fluxo de novo padrão

  • Cecília detecta evento novo com frequência ≥5x no ciclo
  • Registra em tabela cecilia_padroes_novos
  • Badge na aba "Playbooks" com count de novos padrões
  • João abre → vê o evento + exemplos → classifica (ruído / classe D / classe E / etc.)
  • Sistema atualiza RUIDO_PATTERNS ou playbook no cecilia-vault via webhook n8n

Backend necessário

  • cecilia_padroes_novos — eventos novos detectados (criar tabela)
  • cecilia_playbooks — conteúdo dos playbooks sincronizado do Obsidian (criar tabela)
  • Webhook n8n: /controller/playbook/update — quando João salva alteração → atualiza arquivo .md no cecilia-vault

Christian disse: "fazer um sistema inteligente onde o frontend alimenta o backend e a IA puxa do backend ou do Obsidian do Cecília Vault".

Proposta simplificada para v1: Frontend edita cecilia_playbooks no Supabase → cron diário sincroniza para os arquivos .md no cecilia-vault via ssh. Simples, confiável, sem polling de Obsidian.

Chat de treinamento (v2)

Christian mencionou chat via Telegram polling. Para a v1, incluir uma caixa de input: "Ensinar novo padrão" → envia para webhook → n8n processa e atualiza playbook. É mais simples que um chat completo e entrega o mesmo resultado.
