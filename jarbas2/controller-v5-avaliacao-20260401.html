💡 Observações Técnicas do Jarbas

1. Edge Function — risco de deploy novo

A migração da análise IA para Supabase Edge Function (Deno runtime) é a mudança arquitetural mais ousada. O Deno tem quirks (sem node_modules, imports por URL, sandbox). O mecanismo documenta a estrutura de arquivos mas não tem nenhum snippet de código da Edge Function — só a estrutura de diretórios e o contrato.

Isso é o único ponto onde o PRD mecanismo fica vago. O CC vai precisar implementar do zero com contexto limitado sobre como o Deno do Supabase funciona com a Anthropic API. Sugestão: adicionar um snippet de exemplo da Edge Function antes do CC executar.

2. Training Loop (Layer 4) — sub-especificado

O PRD menciona "CC Agent executa 1x/dia, lê 4 vaults, atualiza prompts em ai_agents_config". Mas não especifica:

  • Qual é o trigger do CC Agent? Cron do OpenClaw?
  • Como o feedback do João (correções de classificação) chega ao vault?
  • Qual a estrutura do ai_agents_config? Tabela Supabase ou arquivo?

Para v1, eu simplificaria: João edita playbooks no frontend → salva em controller_playbooks (Supabase) → Edge Function lê direto do Supabase. Elimina a dependência de vault sync.

3. Premissa 6 vs KPI "Estratégicas" — correto

Premissa 6 diz classes ocultas. Mas o KPI card "Estratégicas" usa internamente classe_cecilia IN ('E','F'). Isso não é uma violação — o frontend mostra o label "Estratégicas" sem mencionar a classe. O filtro opera no backend. Comportamento correto.

4. Divisão de responsabilidades — impecável

O PRD do mecanismo produz exatamente o que o PRD frontend consome. View schema alinhada. RPC signatures idênticas. Realtime channels definidos. Webhooks catalogados. É o primeiro PRD do projeto que tem handshake bilateral explícito.

Historicamente, Jarbas (mecanismo) e JANA/Beto (frontend) implementavam em silos e descobriam incompatibilidades no E2E. Isso resolve esse problema estruturalmente.

5. Ordem de execução importa

O CC não pode começar pelo frontend. A ordem obrigatória é:

  1. Executar migrations SQL (status enum + analise_ia + novas colunas)
  2. Resetar OABs circuit-broken
  3. Descomentar crons no start.mjs + reiniciar Cecília
  4. Validar 1 ciclo completo com dados novos
  5. Então: CC implementa frontend v5

Se o CC do frontend começar antes das migrations, vai quebrar tudo.
