Secao 2 — Problema Critico: Acesso a Documentos (S5 sempre null)

P0 CRITICO
⚠️
Sintoma: conteudo_completo permanece null em 100% dos processos. O S5 e executado mas nao consegue extrair o texto dos documentos em nenhuma das 5 camadas de extracao.
Diagnostico das 5 camadas de extracao (L1-L5)
L1
.infraAreaTexto — Seletor so existe na pagina do documento, nao na pagina do processo. navigateToDocumento() usa a[href*="documento"] — seletor errado para EPROC. Formato real: controlador.php?acao=processo_evento_visualizar&...
Resultado: navegacao nao ocorre, extracao tenta na pagina errada → null
FALHA
L2
Multi-selector fallback — #txtSentenca, #txtDespacho, #divConteudoDocumento todos dependem de estar na pagina do documento. Iframes EPROC: iframe#ifrVisualizacao com src cross-origin → SecurityError silencioso.
Resultado: timeout ou "frame not found" → null
FALHA
L3
Event row text — Extrai o TEXTO DA LINHA DA TABELA, nao o documento completo. Seletores #tblEventos tr funcionam na pagina do processo. Retorna titulo/descricao curta do evento, nao o conteudo do documento.
Resultado: parcial — metadados, nao conteudo real
PARCIAL
L4
PDF download — EPROC nao serve PDFs via URL direta. Usa POST + redirect autenticado com token de sessao. a[href*=".pdf"] raramente existe; links PDF usam onclick com funcao JavaScript.
Resultado: 403 ou redirect para login → null
FALHA
L5
Vision OCR — Ultimo recurso, nunca ativado. Timeout de 30s por documento antes de tentar L5. L1-L4 falham silenciosamente retornando null (sem acionar L5).
Resultado: nunca executado em nenhum processo
INATIVO
Root cause resumido — fluxo de falhas encadeadas
navigateToDocumento() nao navega
→
Playwright na pagina do processo
→
L1/L2: sem .infraAreaTexto → null
→
L3: texto curto da linha
→
L4: URL invalida → null
→
L5: nunca ativado

Secao 4 — Solucoes Detalhadas

P0 Fix 1 — Filtro de data null no insert (URGENTE — RISCO ZERO) skill_captura_intimacoes.mjs L86 5 min ▼
🔴
Problema: intimacaoEhMuitoAntiga(null) retorna false → flood historico. Itens sem data passam o filtro e sao inseridos mesmo sendo historicos.
// ANTES -- BUG: function intimacaoEhMuitoAntiga(dataEvento) { if (!dataEvento) return false; // BUG: sem data = passa filtro const d = new Date(dataEvento); const cutoff = new Date(); cutoff.setDate(cutoff.getDate() - 7); return d < cutoff; } // DEPOIS -- CORRIGIDO: function intimacaoEhMuitoAntiga(dataEvento) { if (!dataEvento) return true; // sem data = descartar (historico nao confirmado) const d = new Date(dataEvento); if (isNaN(d.getTime())) return true; // data invalida = descartar const cutoff = new Date(); cutoff.setDate(cutoff.getDate() - 7); return d < cutoff; }
✅
Impacto: Imediato. Previne o flood. Zero risco de regressao — itens sem data sao quase sempre historicos. Aplicar agora.
P0 Fix 2 — Corrigir navegacao ao documento no EPROC s5-read-docs.mjs ~L270 2h ▼

Sem este fix o pipeline nao tem valor real: S6 nao tem contexto para classificar com qualidade, analises ficam rasas e o vault do Joao e subutilizado.
