Análise técnica de como integrar o Domicílio Judicial Eletrônico Nacional no sistema atual — o que resolve, o que não resolve, e como você mesmo implementa.

⚡ Veredicto direto

Sim, e mais do que você imagina. A Cecília provavelmente tem um bug hoje causado pela mudança do DJEN em maio/2025: o filtro S3 descarta eventos rotulados "Diário de Justiça Eletrônico Nacional" — mas esses são exatamente as intimações formais com prazo que precisam ser capturadas. Além disso, adicionar o DJEN ao S2 resolve dois problemas de uma vez: cobertura de 100% das intimações formais e cálculo correto do prazo (D+3 úteis). O melhor de tudo: é HTTP puro, sem Playwright, sem browser. Você implementa isso sozinho em um novo skill_scrape_djen.mjs.

1

O Problema Que Provavelmente Existe Hoje

O que mudou com o DJEN e como isso afeta a Cecília

🔴
Bug provável no S3: O filtro DJEN_FILTER_KEYWORDS da Cecília descarta eventos do EPROC que contêm "Diário de Justiça Eletrônico Nacional". Antes de maio/2025 fazia sentido — era só publicação. Hoje, a intimação formal com prazo vai exclusivamente pelo DJEN. Quando o EPROC mostra esse evento na lista, é o aviso de que existe uma intimação com prazo no DJEN. A Cecília está descartando esse aviso.

📅 Como era (antes de maio/2025)

Intimação aparecia diretamente no painel do EPROC com conteúdo e prazo visíveis. Filtrar eventos de "publicação em diário" era correto — era ruído real.

Filtro S3 correto ✅

📅 Como é agora (após 16/mai/2025)

Intimação formal vai exclusivamente pelo DJEN. No EPROC aparece apenas um aviso: "Publicado no DJEN em dd/mm/yyyy". O conteúdo e o prazo estão no DJEN.

Filtro S3 descarta intimação real ❌

🗓️ Novo ciclo de prazos com o DJEN

O prazo não começa na data que o EPROC mostra o evento. Começa D+3 úteis após o envio ao DJEN. Se a Cecília está calculando pela data do EPROC, os prazos estão errados.

D0
Tribunal envia ao DJEN
→
D1
DJEN disponibiliza
→
D2
DJEN publica
→
D3
Prazo começa aqui

Se a Cecília captura em D0 (data que o EPROC mostra o aviso) e calcula o prazo a partir daí, está entregando ao advogado uma data de vencimento 3 dias úteis mais cedo do que o real.

O Que a API do DJEN Oferece

É REST puro — zero Playwright. Você implementa isso com fetch()

✅
Boa notícia: A API do DJEN é oficial do CNJ, estável, documentada e usa OAuth2 padrão via Keycloak — o mesmo tipo de auth que o EPROC usa. Não precisa de browser, não tem CSRF, não tem React inputs. É só HTTP.

URLs de Produção

  • API Base:
    domicilio-eletronico.pdpj.jus.br
  • SSO Token:
    sso.cloud.pje.jus.br/auth/realms/
    pje/protocol/openid-connect/token
  • Swagger (HML):
    gateway.stg.cloud.pje.jus.br/
    domicilio-eletronico-hml/swagger-ui

Credenciais necessárias

Por OAB/CPF monitorado, você precisa de:

  • client_id + client_secret — gerados em domicilio-eletronico.pdpj.jus.br → "Gerenciar credenciais API" (login gov.br do advogado)
  • CPF do advogado — usado no header On-behalf-Of
  • Guardar junto com login/senha/totp_secret na tabela jarbas_accounts

Autenticação — token OAuth2

// skill_auth_djen.mjs — obter token (sem Playwright)
async function getDJENToken(cpf, clientId, clientSecret) {
  const res = await fetch(
    'https://sso.cloud.pje.jus.br/auth/realms/pje/protocol/openid-connect/token',
    {
      method: 'POST',
      headers: { 'Content-Type': 'application/x-www-form-urlencoded' },
      body: new URLSearchParams({
        grant_type:    'client_credentials',
        client_id:     clientId,
        client_secret: clientSecret,
      })
    }
  );
  const { access_token, expires_in } = await res.json();
  // Cachear token por expires_in segundos (tipicamente 300s)
  return access_token;
}

// GET /comunicações — retorna intimações pendentes de ciência
async function buscarIntimacoesDJEN(token, cpf, dataInicio, dataFim) {
  const params = new URLSearchParams({
    statusCiente: 'em curso',    // só as não lidas
    dataInicio,                      // 'YYYY-MM-DD'
    dataFim,                         // máx 7 dias de intervalo
    size: '100',
    sort: 'dataEnvio,desc'
  });

  const res = await fetch(
    `https://domicilio-eletronico.pdpj.jus.br/comunicações?${params}`,
    {
      headers: {
        'Authorization': `Bearer ${token}`,
        'On-behalf-Of': cpf,          // CPF sem pontuação, 11 dígitos
        'Content-Type':  'application/json'
      }
    }
  );
  return res.json();

      method: 'PUT',
      headers: {
        'Authorization': `Bearer ${token}`,
        'On-behalf-Of': cpf
      }
    }
  );
  // Após esse PUT: status muda para 'ciente' no DJEN
  // IMPORTANTE: isso oficializa a abertura. O prazo começa a contar.
  // Fazer APENAS quando quiser registrar ciência formal.
}
⚠️
Atenção ao registrar ciência: O PUT que "abre" a comunicação registra formalmente a ciência no sistema do CNJ. O prazo D+3 começa da dataPublicacao, independente de quando você faz o PUT. Use o PUT apenas na ação deliberada do advogado, não automaticamente no S2.

A nova skill: skill_scrape_djen.mjs

// skill_scrape_djen.mjs — integra ao pipeline como S2B
export async function scrapesDJEN({ cpf, clientId, clientSecret, diasRetroativos = 2 }) {
  // 1. Autenticar
  const token = await getDJENToken(clientId, clientSecret);

  // 2. Definir janela de 7 dias (limite da API)
  const hoje = new Date();
  const inicio = new Date(hoje - diasRetroativos * 86400000);

  // 3. Buscar intimações não lidas
  const resultado = await buscarIntimacoesDJEN(
    token, cpf,
    inicio.toISOString().split('T')[0],
    hoje.toISOString().split('T')[0]
  );

  // 4. Normalizar para o mesmo formato do S2A (EPROC)
  return (resultado.content || []).map(com => ({
    cnj:               com.numeroProcesso,
    tipo_evento:       com.tipoComunicacao,
    titulo:            com.assunto,
    data_evento:       com.dataEnvio,
    data_publicacao:   com.dataPublicacao,    // ← chave para calcular D+3
    conteudo_raw:      com.conteudo || '',
    fonte:             'djen',
    djen_id:           com.numeroComunicacao,
    status_djen:       com.status,
  }));
}

Cálculo de prazo correto

// S7 persist — calcular prazo baseado na fonte
function calcularDataPrazo(evento, diasPrazo) {
  if (evento.fonte === 'djen' && evento.data_publicacao) {
    // Prazo conta a partir de D+3 úteis da publicação no DJEN
    const inicio = addBusinessDays(new Date(evento.data_publicacao), 3);
    return addBusinessDays(inicio, diasPrazo);
  }
  // EPROC: manter cálculo atual (a partir da data do evento)
  return addBusinessDays(new Date(evento.data_evento), diasPrazo);
}

O Que o DJEN Não Resolve

Honestidade sobre as limitações — para você não se surpreender

❌ Não elimina o Playwright

Movimentações gerais (despachos, decisões que não são intimações formais), enriquecimento de dados do processo e download de documentos completos continuam exigindo Playwright no EPROC. O DJEN cobre apenas as intimações eletrônicas formais.

❌ Não cobre todos os eventos

Audiências marcadas, despachos de andamento, decisões interlocutórias, movimentações de cartório — essas coisas continuam visíveis apenas pelo "Últimas Movimentações" do EPROC. O DJEN é específico para intimações com prazo.

❌ Documentos incompletos

O DJEN pode ter o texto da intimação, mas o processo completo com todos os documentos (petições das partes, laudos, etc.) continua no EPROC. Para alimentar a OlivIA com contexto completo, o Playwright ainda é necessário.

⚠️ Credenciais por CPF, não por OAB

As credenciais do DJEN são vinculadas ao CPF do advogado (via gov.br), não ao número OAB. Para escritórios com advogados colaboradores, cada CPF precisa do seu próprio par de credenciais. Adicione djen_cpf à tabela jarbas_accounts.

💡
A combinação certa: DJEN (HTTP REST) para intimações formais com prazo + EPROC via Playwright para movimentações gerais e documentos. As duas fontes se complementam e juntas cobrem 100% do que você precisa monitorar.
