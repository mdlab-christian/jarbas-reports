O problema que isso resolve

O Jarbas (OpenClaw) é ótimo pra conversas, orquestração e tarefas rápidas. Mas às vezes uma tarefa é grande demais — muitos arquivos, muito código, muitas iterações — e o Claude Code (CC) é muito mais eficiente pra isso.

O problema: o CC não sabe nada sobre o contexto do Jarbas. Não sabe das credenciais, não conhece os vaults do Obsidian, não sabe quem é o Christian, não conhece a rede de agentes, não tem as skills.

A solução: o comando /migrar-cc gera um arquivo .md com tudo que o CC precisa saber, já formatado. É basicamente um "briefing completo" — cola e o CC assume o trabalho como se fosse o Jarbas.

💡

A analogia

É como um CTO passando o trabalho pro dev sênior. Você não explica tudo do zero — você entrega um documento com: contexto, ferramentas, credenciais, onde estão as coisas e o que precisa ser feito. O dev assume e entrega.

O que o arquivo gerado contém

🐙 Identidade do agente

Nome, hardware, usuário macOS, papel na rede, quem supervisiona e quem responde a quem.

🔧 Ferramentas e acesso

SSH pro Mac Mini, como rodar scripts como cecilia via launchctl, Supabase curl, n8n, GitHub Pages.

🔑 Credenciais (paths)

Onde ficam os .env, quais variáveis existem. Sem valores — o CC já tem acesso pelo ambiente.

📂 Skills disponíveis

Lista completa das 44 skills OpenClaw com descrição — o CC sabe o que pode usar e onde fica cada uma.

🧠 Vault Obsidian do projeto

Busca automática nos vaults por arquivos relacionados ao projeto. CONTEXTO.md, PROJETOS-ATIVOS.md e contexto do Christian sempre incluídos.

📊 Últimos HTMLs gerados

Links dos relatórios e planos relacionados ao projeto — o CC entende o estado atual sem precisar perguntar.

📋 Projetos em andamento

Estado atual de todos os projetos do PROJETOS-ATIVOS.md — o CC sabe o que está bloqueado, em andamento e concluído.

⚙️ Comandos prontos

Copy-paste de comandos SSH, Supabase, git, reiniciar agentes — sem precisar lembrar a sintaxe.

Como usar — passo a passo

1

Falar pro Jarbas gerar o contexto

No Telegram, mandar pro Jarbas: "migra pro CC" ou "gera contexto CC" ou "passa pro claude code". Se tiver um projeto específico, mencionar: "migra pro CC o projeto Cecília".

2

Jarbas gera e manda o link do .md

O Jarbas vai rodar o script, montar o arquivo com todo o contexto e mandar o link direto do GitHub raw. Exemplo: https://raw.githubusercontent.com/mdlab-christian/jarbas-reports/main/jarbas2/cc-contexto-20260331.md

3

Abrir o Claude Code no terminal

Com o contexto baixado ou copiado, rodar no terminal do Mac:

# Opção 1: usar o arquivo diretamente
claude --print --permission-mode bypassPermissions "$(cat ~/jarbas-reports/jarbas2/cc-contexto-20260331.md)"

# Opção 2: baixar do GitHub e usar
curl -s https://raw.githubusercontent.com/mdlab-christian/jarbas-reports/main/jarbas2/cc-contexto-20260331.md > /tmp/ctx.md
claude --print --permission-mode bypassPermissions "$(cat /tmp/ctx.md)"

# Opção 3: abrir o CC normalmente e colar o conteúdo do .md no início da conversa
claude
# → colar o conteúdo do arquivo .md como primeira mensagem

Para o Beto (Mac Mini)

O Beto tem o mesmo comando instalado. Rodar diretamente no Mac Mini: node ~/beto/src/skills/skill_migrar_cc.mjs --projeto "nome"

Gerar direto do terminal (sem passar pelo Jarbas)

O que mudou na nossa operação

🐙 O Jarbas 2.0 virou o ponto central de tudo

Antes: tinha JANA no MacBook + JARBAS no Mac Mini — dois bots separados, contexto duplicado, confuso de gerenciar.

Agora: Jarbas 2.0 substituiu os dois. É o único agente principal do Christian, roda no MacBook Air M4, porta 18790. Tem acesso total à rede toda via SSH e supervisiona OlivIA, Cecília, Beto e Jarbinhas.

🧠 Obsidian virou o cérebro

Toda memória de longo prazo do Jarbas vive em dois vaults:

  • jarbas-vault — privado, estado atual, projetos, ideias
  • mdlab-vault — compartilhado com toda a rede, conhecimento técnico

O /migrar-cc busca nesses vaults automaticamente — o CC recebe o contexto real, não uma versão resumida.

💻 Claude Code agora tem identidade Jarbas

O ~/.claude/CLAUDE.md foi reescrito completamente. Agora quando você abre o CC, ele já sabe:

  • Quem é o Christian e o contexto das empresas
  • Toda a rede de agentes e como acessar cada um
  • Como rodar scripts no Mac Mini como cecilia/olivia/beto
  • O padrão de HTML do Jarbas (dark mode, self-contained, publicar no GitHub Pages)
  • As regras P0 e o que confirmar antes de executar
  • Todas as credenciais via process.env (já injetadas no settings.json)
🔌 MCPs do Obsidian configurados no CC

O Claude Code agora tem 3 MCPs ativos que dão acesso direto aos vaults:

  • obsidian-jarbas-vault — leitura/escrita no vault privado do Jarbas
  • obsidian-mdlab-vault — knowledge base técnico da rede
  • filesystem-vaults — acesso direto a todos os vaults + workspace + reports

Regras importantes ao usar o CC como Jarbas

🔴

P0 — Sempre confirmar antes de executar

DELETE em produção, git push --force, migrations irreversíveis, qualquer coisa que afete os ~5.800 processos do Midas. Para tudo e confirma com o Christian.

🚫 O que nunca fazer
  • Hardcodar credenciais — usar process.env (já estão injetadas)
  • Deixar TODO ou placeholder no código — implementar completo
  • Duas sessões Playwright para a mesma OAB ao mesmo tempo
  • Fire and forget em processos críticos — sempre aguardar o resultado
  • Usar keyboard.type() para preencher senha no Keycloak com @ — usar native setter
📋 Relatório ao finalizar (obrigatório)

Ao concluir qualquer tarefa não-trivial, o CC deve entregar:

  • ✅ O que foi feito (arquivo por arquivo)
  • 📁 Paths completos de tudo criado/modificado
  • 🧪 Como testar (passos exatos)
  • ⚠️ O que precisa de revisão manual
  • ❌ O que não foi feito e por quê
  • ⏭️ Próximo passo sugerido
