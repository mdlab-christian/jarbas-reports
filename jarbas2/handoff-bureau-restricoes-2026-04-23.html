Estados do CrmAuditoriaPanel

Componente Compartilhado
CrmAuditoriaPanel e usado tanto no CRM (/crm, /clientes) quanto no MdZap (LeadRestricoesMdzapTab). Tem 5 estados distintos controlados pelos dados da tabela auditorias.
01
Nunca Auditado
Lead/cliente sem registro em auditorias. Exibe botao "Consultar CPF Completo" com icone Shield. Clique chama dispararAuditoria().
02
Processando
Status 'processando'. Progress bar animada — incrementa 1.42%/segundo ate 85%. Nao avanca alem de 85% — aguarda Realtime. Exibe spinner + texto "Consultando bureau..."
03
Concluido — Sem Restricoes
Status 'concluida' + possui_restricoes = false. Exibe CheckCircle2 verde + "CPF sem restricoes ativas". Opcao discreta de baixar PDF.
04
Concluido — COM Restricoes
Status 'concluida' + possui_restricoes = true. Exibe: total de restricoes, valor total (mascarado se !acesso_financeiro), contagem processaveis. Botao "Cadastrar Restricoes" chama onCadastrarClick(auditoriaId).
05
Falha
Status 'falha'. Exibe AlertTriangle + mensagem de erro_mensagem. Botao "Tentar novamente" recria a auditoria.

Matching por Empresa — Algoritmo findBestMatch

Onde Vive
Funcao findBestMatch(credor: string, empresas: Empresa[]) em src/pages/auditorias/hooks/use_cadastrar_restricoes.ts. Executada client-side ao montar o CadastrarRestricoesDrawer.
+---------------------------------------------------------------------------------+ | ALGORITMO DE MATCHING - 3 ESTRATEGIAS | +---------------------------------------------------------------------------------+ INPUT: credor = "BANCO BRADESCO S.A." (nome vindo do bureau, normalizado) +--- ESTRATEGIA 1: Match Exato -------------------------------------------------+ | normalizar(credor) IN empresas[].nome_variacoes[] | | Se encontrar --> score = 1.0 --> confidence = 'alto' VERDE | +-------------------------------------------------------------------------------+ | nao encontrou v +--- ESTRATEGIA 2: Fuzzy Token (Jaccard Similarity) ----------------------------+ | tokens_credor = tokenizar("banco bradesco s a") = {'banco','bradesco'} | | Para cada empresa: compara com nome + razao_social + fantasia + variacoes | | | | Jaccard(A, B) = |A intersecao B| / |A uniao B| | | | | score >= 0.75 --> confidence = 'alto' VERDE (dropdown fechado) | | score 0.45 a 0.74 --> confidence = 'medio' AMARELO (dropdown aberto) | | score < 0.45 --> confidence = 'nenhum' VERMELHO (campo livre) | +-------------------------------------------------------------------------------+ | credor contem 'scpc'|'boa vista'|'boavista'|'scpc/boa vista' v +--- ESTRATEGIA 3: Caso Especial Boa Vista -------------------------------------+ | Multiplas entradas Boa Vista no bureau --> AGRUPADAS em 1 item | | empresa = "Boa Vista" | valor = soma de todos | processavel = false | | Motivo: "Boa Vista nao integra o fluxo de processos" | +-------------------------------------------------------------------------------+ OUTPUT: { empresa_id, confidence, score, empresa_nome }
Estrategia Criterio Score Confidence UI Behavior
Match Exato Credor in nome_variacoes[] 1.0 Alto Dropdown fechado, link "corrigir" discreto
Fuzzy Jaccard Intersecao tokens ≥ 0.75 0.75–0.99 Alto Dropdown fechado, link "corrigir" discreto
Fuzzy Jaccard Intersecao tokens 0.45–0.74 0.45–0.74 Medio Dropdown aberto para revisao do usuario
Fuzzy Jaccard Intersecao tokens < 0.45 < 0.45 Nenhum Campo livre obrigatorio, sem pre-selecao
Boa Vista Especial scpc / boa vista / boavista — Alto Agrupado, nao processavel, badge especial

Tabelas Supabase

Tabela auditorias
-- Supabase: qdivfairxhdihaqqypgb
CREATE TABLE auditorias (
  -- Identidade
  id                              UUID         DEFAULT gen_random_uuid() PRIMARY KEY,
  organizacao_id                  UUID         NOT NULL REFERENCES organizacoes(id) ON DELETE CASCADE,

  -- Objeto auditado
  cpf                             TEXT         NOT NULL,
  bureau                          TEXT         CHECK (bureau IN ('apifull_115','boavista','serasa','spc')),
  lead_id                         UUID         REFERENCES leads(id),
  cliente_id                      UUID         REFERENCES clientes(id),
  processo_id                     UUID         REFERENCES processos(id),

  -- Status
  status                          TEXT         CHECK (status IN ('pendente','processando','concluida','falha')),
  erro_mensagem                   TEXT,

  -- Resultados
  possui_restricoes               BOOLEAN,
  possui_restricoes_processaveis  BOOLEAN,
  restricoes_encontradas          INT,
  valor_total_restricoes          BIGINT,       -- centavos | mascarar se !acesso_financeiro
  nome_titular                    TEXT,

  -- Dados brutos do bureau + URLs do PDF
  resultado_json                  JSONB,
  -- estrutura: { apifull_115: {...}, pdf_urls: { apifull_115: "https://..." } }

  -- Auditoria
  solicitante_user_id             UUID         REFERENCES auth.users(id),
  created_at                      TIMESTAMPTZ  DEFAULT now() NOT NULL,
  updated_at                      TIMESTAMPTZ  DEFAULT now() NOT NULL
  -- SEM deleted_at: auditorias sao imutaveis por compliance
);
Tabela restricoes
-- Supabase: qdivfairxhdihaqqypgb
CREATE TABLE restricoes (
  -- Identidade
  id                     UUID         DEFAULT gen_random_uuid() PRIMARY KEY,
  organizacao_id         UUID         NOT NULL REFERENCES organizacoes(id) ON DELETE CASCADE,

  -- Vinculo (lead ou cliente, mutuamente exclusivos)
  lead_id                UUID         REFERENCES leads(id),
  cliente_id             UUID         REFERENCES clientes(id),

  -- Empresa credora
  empresa_id             UUID         REFERENCES empresas(id),   -- FK quando empresa encontrada
  empresa_nome           TEXT,                                   -- fallback quando empresa nao existe

  -- Dados da restricao
  valor                  BIGINT,                                 -- centavos
  data_restricao         DATE,
  orgao_nome             TEXT,
  tipo                   TEXT,
  codigo_contrato        TEXT,

  -- Processabilidade
  processavel            BOOLEAN      NOT NULL DEFAULT false,
  motivo_nao_processavel TEXT,                                   -- ex: "Boa Vista nao integra o fluxo"

  -- Vinculo com processo (quando vira negocio)
  processo_id            UUID         REFERENCES processos(id),
  drive_pasta_id         TEXT,
  drive_pasta_url        TEXT,

  -- Auditoria
  created_by             UUID         REFERENCES auth.users(id),
  created_at             TIMESTAMPTZ  DEFAULT now() NOT NULL,
  updated_at             TIMESTAMPTZ  DEFAULT now() NOT NULL,
  deleted_at             TIMESTAMPTZ                             -- soft delete OBRIGATORIO
);

Regras de Negocio Importantes

⚠
Boa Vista nao e processavel: Restricoes SCPC/Boa Vista sao agrupadas em 1 item mas marcadas processavel=false. Motivo: "Boa Vista nao integra o fluxo de processos". Isso e intencional — nao corrigir.
🚫
Soft delete obrigatorio em restricoes: Toda query deve incluir .is('deleted_at', null). Hard delete proibido com FK ativa. Auditorias nao tem deleted_at (imutaveis por compliance).
🔓
Multi-tenant absoluto: Todas as queries incluem .eq('organizacao_id', orgId). RLS no banco como segunda linha de defesa.
👁
Cegueira financeira: Se acesso_financeiro === false: mascarar todos os valores monetarios com R$••••• — inclui valor_total_restricoes no CrmAuditoriaPanel.
🔨
Monetario em centavos: Banco armazena BIGINT em centavos. toCentavos = v => Math.round(v * 100) ao salvar. formatBRL = c => Intl.NumberFormat('pt-BR',{style:'currency',currency:'BRL'}).format(c/100) ao exibir.
📈
React Query staleTime: Toda query deve ter staleTime: 30_000. Realtime invalida manualmente via queryClient.invalidateQueries().
✌
Bureaus suportados vs disponiveis: Reconhecidos: ['apifull_115', 'boavista', 'serasa', 'spc']. Atualmente com disponivel: true apenas apifull_115. Boa Vista e SPC marcados como disponivel: false no frontend.
🔁
nome_variacoes e aprendizado persistente: Quando usuario confirma match de empresa, o credor normalizado e adicionado ao array nome_variacoes via RPC com array_append. Proxima auditoria ja usa match exato (estrategia 1).
🚫
Design System — proibicoes absolutas: Proibido hardcode de cor/hex/rgb. Sempre tokens CSS via hsl(var(--token)). Nunca className="bg-[#20c997]" — usar className="bg-primary". Glassmorphism: somente cards/modais/popovers.
Convencoes de Codigo — Resumo Rapido
snake_case: banco + nomes de arquivo  |  camelCase: JS/TS  |  PascalCase: componentes + tipos
Arquivo maximo: 500 linhas por .tsx  |  Hooks compartilhados: hooks/shared/  |  1 modal por vez (dirty-state obrigatorio)
Busca: unaccent(lower(campo)) ILIKE '%'||unaccent(lower($1))||'%' + debounce 300ms (min 2 chars)
Datas: TIMESTAMPTZ no banco | format(parseISO(str), 'dd/MM/yyyy', {locale: ptBR})
