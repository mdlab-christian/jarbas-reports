Contexto Geral

01
Cenário de origem

O cenário de geração de documentos existe no Make.com com 210 módulos, chamado [MDFlow] Gerar contratos. Foi completamente mapeado para migração para n8n no workflow MDFLOW-DOC-GERAR-CONTRATOS-v1.

Supabase
Produção MdFlowqdivfairxhdihaqqypgb
URLhttps://qdivfairxhdihaqqypgb.supabase.co
Onde e como é usado no produto

O webhook faz parte do fluxo MdZap sdr_leads — especificamente na Fase 7 do atendimento com o lead:

  1. Lead avança no funil CRM e tem uma restrição (negativação) cadastrada
  2. Advogado (Christian ou Heloísa) decide gerar procuração + contrato no MdFlow
  3. Frontend chama POST /webhook/mdflow/doc/gerar-documentos com os parâmetros
  4. Sistema gera os documentos no Google Drive e cria links ZapSign
  5. Julia (IA) envia o link ZapSign pelo MdZap para o lead
  6. Lead é movido para aguardando_assinatura
  7. Follow-ups automáticos em 1h e 24h

Fluxo Técnico

02
Webhook Trigger
POST /webhook/mdflow/doc/gerar-documentos — valida header mdflow: MdL1501@ — responde 200 imediato
Fetch & Validate (Supabase)
Busca restrição → cliente ou lead (fallback) → advogado (via OAB) → templates Google Docs para o tipo+oab
↳ ERRORMISSING_FIELDS | RESTRICAO_NOT_FOUND | NO_PESSOA → Respond Error 1
Get Google Token
OAuth2 refresh_token flow → obtém access_token para Drive + Docs API
↳ ERRORGOOGLE_TOKEN_ERROR → Respond Error 2
Generate & Upload Docs (loop por documento do bundle)
Para cada doc no bundle do tipo selecionado:
1. Copy template Drive (preserva original)
2. Replace placeholders via Google Docs API (batchUpdate)
3. Export como PDF via Drive API
4. Upload na pasta do cliente/lead no Drive (google_drive_folder_id)
↳ ERRORTEMPLATE_NOT_FOUND | Drive API error → Respond Error 3
Respond Success
action: "toast_refresh" + lista de documentos gerados com URLs

Envelope canônico de resposta (sempre HTTP 200, mesmo em erro):
Sucesso: {"success": true, "message": "...", "action": "toast_refresh", "data": {"docs": [...]}}
Erro: {"success": false, "message": "...", "action": "toast_error", "error_code": "RESTRICAO_NOT_FOUND"}

Pasta raiz do escritório no Google Drive compartilhado

Pasta do cliente ou lead
clientes.google_drive_folder_id
leads.google_drive_folder_id

Os PDFs gerados são salvos NESSA pasta, não na raiz

Fluxo de upload Drive 
1. Drive.Files.copy(templateDocId, { name: "NOME_GERADO", parents: [folderIdDoCliente] })
   → retorna copyDocId

2. Docs.Documents.batchUpdate(copyDocId, { requests: replacements[] })
   → substitui todos os {{PLACEHOLDER}} pelos valores reais

3. Drive.Files.export(copyDocId, { mimeType: "application/pdf" })
   → retorna buffer PDF

4. Drive.Files.create(pdfBuffer, { name: "NOME.pdf", parents: [folderIdDoCliente] })
   → salva PDF na pasta do cliente

5. (Opcional) Drive.Files.delete(copyDocId)
   → remove cópia .docx temporária se não for necessária

Error Codes

09
error_code Fase Causa Resposta ao frontend
MISSING_FIELDS Validação inicial lead_id, restricao_id ou tipo ausente ou vazio success: false, action: "toast_error"
RESTRICAO_NOT_FOUND Fetch Supabase Restrição com restricao_id não existe ou foi deletada success: false, action: "toast_error"
NO_PESSOA Fetch Supabase Restrição não tem cliente_id nem lead_id vinculado success: false, action: "toast_error"
GOOGLE_TOKEN_ERROR OAuth2 Falha ao obter access_token via refresh_token (token expirado, revogado, etc) success: false, action: "toast_error"
TEMPLATE_NOT_FOUND Geração docs Combinação tipo + oab não tem template mapeado success: false, action: "toast_error"

Observações do Gustavo / Erros Observados

10
Completar esta seção antes de enviar para o Claude Web

O Gustavo vai completar esta seção com os erros observados no cenário Make, comportamentos inesperados, campos que não estão sendo preenchidos corretamente, erros de API do Google, ou qualquer desvio do fluxo esperado.

O JSON completo do Make blueprint (make_blueprint_original.json — 210 módulos) deve ser colado ou anexado junto com este briefing ao iniciar a conversa no Claude Web.

Script de criação n8n

O arquivo .projeto/make-to-n8n-contratos/create_workflow.mjs (878 linhas) já tem a versão n8n deste workflow mapeada. Para executar:

Execução do script de criação 
N8N_BASE_URL=https://primary-production-e209.up.railway.app \
N8N_API_KEY=sua_api_key_aqui \
node .projeto/make-to-n8n-contratos/create_workflow.mjs

Atenção: verificar se o script tem os doc_ids corretos dos templates antes de executar. Comparar com as tabelas acima e com o template_placeholders.json.

Áreas que precisam de revisão antes da migração
  • Confirmar doc_ids dos termos (ciencia, veracidade, parceria) no blueprint original
  • Verificar se restricoes.codigo_contrato é o campo correto para o placeholder CONTRATO
  • Confirmar formato exato dos placeholders no Google Doc (ex: {{NOME COMPLETO}} vs [[NOME COMPLETO]])
  • Testar OAuth2 refresh_token — verificar se as credenciais Google estão no n8n
  • Confirmar se leads.google_drive_folder_id é o campo exato (ou há outro nome?)
