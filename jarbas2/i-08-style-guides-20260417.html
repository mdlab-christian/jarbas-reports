08 · Drifts / Gaps / Bugs

DRIFT-01
Prompt do subprojeto dizia "cascata advogado > org > tipo" — ordem INVERTIDA vs D-08.2 aprovada (tipo > advogado > org > virtual). Implementação segue a decisão aprovada, não o prompt. Atualizar prompt para evitar confusão em revisões futuras.
DRIFT-02
Subprojeto 08 previa soft delete deleted_at; schema real usa flag ativo BOOLEAN. Diverge do padrão MdFlow CLAUDE.md seção 4 ("soft delete obrigatório"). Impacto: T08-03 já normalizou cascata.mjs, mas migration T08-01 no worktree mostrava deleted_at — arquivo desatualizado vs banco real.
GAP-01 · CRÍTICO
TipTapEditor standalone não é importado em lugar algum — decisão de substituir EditorPeticaoV3 foi "deferida para W5" nos notes do T08-04. Hoje: editor real em /olivia NÃO tem Mention, style guide cascade nem autosave tripla.

GAP-03

09 · Avaliação

  • Backend é o mais forte: DDL rico (57 colunas cobrem headings, endereço, data, assinatura, cabeçalho/rodapé HTML além do corpo), RPC elegante, seed realista.
  • Fluxo de leitura sólido: 2 consumidores (frontend TipTapEditor + backend cascata.mjs) chamam a mesma função — R7 arquiteturalmente mitigado.
  • Fluxo de escrita funciona: /configuracoes/style-guide tem CRUD completo e performante (staleTime 30s, mutations com invalidate).
  • Ruim: "editor novo com tudo" (TipTapEditor.tsx) foi construído mas não plugado — permite dupla manutenção e risco de divergência.
  • Ruim: autosave tripla e Mention, que seriam os ganhos percebidos pelo advogado no Step 4, não chegam ao fluxo real.
  • Neutro: cascata.mjs tem cache 5min em memória — aceitável para pipeline ephemeral, mas se Mac Mini mantiver processo vivo e advogado editar guide, há janela de até 5min de inconsistência (resolvido por invalidarCache() manual — hoje ninguém chama).
