01 🎬

Resumo fiel do video — conceitos do Bruno

Apresentacao dos conceitos de Bruno Okamoto sem analise. Universal para qualquer "segundo cerebro" — Claude, OpenClaw, Hermes, Paper Clip.

1.1 — Os 3 pilares fundamentais
01

Contexto

Quem e a empresa, equipe, papeis, metricas, faturamento. Bruno extrai contexto de WhatsApp, Telegram, email e calendario.

"Sem contexto, seu agente e so um estagiario perdido."

02

Skills

Habilidades documentadas universais. Diferente de prompt: skill executa sempre o mesmo processo. Bruno recomenda nao baixar skills de terceiros — criar do proprio processo.

Skills funcionam em qualquer agente (Claude, OC, Hermes, Paper Clip).

03

Rotinas / Crons

Automacoes recorrentes que acordam o agente, executam skills e entregam resultado. Cron + skill = agente autonomo.

"Sem rotina, agente nao sai do reativo."

1.2 Mapas por pasta

Toda pasta precisa ter um mapa. AGENTS.md (OpenClaw) ou CLAUDE.md (Claude Code) eh o mapa que a IA le ao acordar. Mapa dentro de empresa/, skills/, areas/, marketing/, vendas/.

"Quando o Cloud acorda, ele puxa todas as informacoes principalmente porque tem o mapa."

1.3 Memoria de 48h

Quando o agente acorda, ele le o mapa de todas as pastas, todas as skills E le a memoria das ultimas 48 horas. Tanto faz se commit veio de Cloud Code ou OpenClaw — IA lembra todo o trabalho.

1.4 GitHub vs Obsidian

Obsidian = pessoa fisica. Visualmente lindo (grafo), mas nao escala para equipe.

GitHub = empresa. Multi-user, multi-agente, commit historico.

"Se voce e empresario, GitHub eh um no-brainer."

1.5 Multi-canal de absorcao de contexto

Bruno absorve contexto de WhatsApp (alunos, parceiros, investidores), Telegram (resumos diarios), Calendario e Email.

"Eu puxo dado de todos os lugares para ter o maximo de contexto possivel pro meu agente."

1.6 Skills universais

"As skills sao universais. Voce tira skill de um e gera a mesma skill no Cloud, no OpenClaw, no Paper Clip, no Hermes." Tudo que voce documenta como skill viaja.

1.7 Reindexacao forcada ao trocar LLM

Cada LLM funciona diferente. Quando voce troca uma por outra, precisa reindexar e fazer memory index na forca. Bruno aplica 2 prompts ao trocar Cloud → ChatGPT que forcam IA a "compreender o corpo dela".

⭐ 1.8 Conselho central de Bruno

"Se foca menos em ferramentas e mais em construcao do seu segundo cerebro. Ferramentas tem pros/contras, mas TODAS te levam mais ou menos pro mesmo lugar."

🟡 Parcial Sprint dedicada

  • Bruno: IA acorda lendo "memoria das ultimas 48 horas" automaticamente

Como fazer

1. Criar tabela mdos_core_memory_index no Supabase:

id, organizacao_id, agente (mia/olivia/cecilia/cc-pro/cc-air/cc-mini), evento_tipo (sessao/commit/relatorio), resumo (text), refs (jsonb), created_at

2. Triggers automaticos:

  • Pos /report → INSERT em memory_index
  • Pos commit beta-mdflow → INSERT (via git hook)
  • Pos sessao Telegram MIA → INSERT

3. Adicionar ao bootstrap CLAUDE.md global:

node ~/Documents/MdOs-vault/_tools/mdos-db.mjs select \
  mdos_core_memory_index "*" \
  "created_at=gte.$(date -v-48H -u +%Y-%m-%dT%H:%M:%S)"
Risco: Crescimento — rotina cleanup +90d necessaria
Estimativa: 4h (tabela + 3 triggers + skill /mdos:recall + CLAUDE.md)
