Por que isso existe

O Cecilia v3 funciona quando recebe uma lista de processos manualmente, mas o ciclo autonomo (cron) ainda falha. Este mecanismo de importacao e o backup definitivo: sempre que o cron falhar, o usuario faz upload de uma planilha com os CNJs que tiveram movimentacao, o sistema acessa o EPROC via Cecilia, analisa cada processo e alimenta o /controller com apenas as movimentacoes relevantes.

Problema

Cron autonomo instavel — circuit breaker v1 sem reset automatico

Solucao

Upload manual → Cecilia processa → mesmo pipeline de IA → mesmo /controller

Logica de Deduplicacao — 3 Casos

Caso 1 — Duplicata
signature_hash

Hash ja existe → Skip + marca ja_existente

Aparece no painel como "X ja existentes (pulados)"

Nao insere no controller. Nao gasta tokens de IA.

Caso 2 — Sem Movimentacao
sem_movimentacao

Processo acessado no EPROC, mas sem eventos desde data_inicio

Atualiza cecilia_ultima_analise_em

Nao entra no controller.

Caso 3 — Erro EPROC
ia_status='erro'

Timeout, sessao expirada, CNJ nao encontrado

INSERT intimacao com titulo '[Erro EPROC] '+CNJ

Aparece no /controller com badge de erro — prazo nao passa.

Vai funcionar na pratica?

Sim. O skill_acessar_processo.mjs ja funciona — o teste manual do Christian confirmou. A unica mudanca e chamar esse skill a partir de uma fila no Supabase em vez de um ciclo OAB completo. O pipeline de IA (Haiku → Sonnet) ja esta operacional. A deduplicacao por signature_hash ja existe. O risco principal e a estabilidade das sessoes EPROC (~25-35 min para 100 processos com delays anti-bot). Qualquer falha e recuperavel — processos com erro aparecem no controller para revisao manual.
