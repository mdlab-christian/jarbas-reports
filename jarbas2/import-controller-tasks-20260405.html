Contexto do Projeto

Objetivo

Criar mecanismo de importação manual de processos no /controller. O usuário faz upload de planilhas (CSV/XLSX) com CNJs, associa a um advogado (OAB), seleciona data de corte, e a Cecilia acessa o EPROC, analisa com IA e alimenta o controller com movimentações relevantes (classes B–G).

Motivação

O cron autônomo da Cecilia está instável. Este mecanismo é o backup definitivo — quando o cron falhar, o usuário importa a lista exportada do software jurídico e o sistema processa completamente sem intervenção adicional.

Mecanismo de Filas

  • Sem n8n — usa Supabase queue (tabela controller_importacoes como fila)
  • Cecilia faz poll a cada 30s via start.mjs
  • Realtime Supabase → frontend atualiza em tempo real sem polling
  • Dedup via signature_hash (campo já existe em intimacoes)

Fora do Escopo

  • Corrigir cron autônomo (projeto separado: cecilia-autonomo)
  • Modificar pipeline de IA (já funcional: Haiku triagem → Sonnet D/E/F)
  • Histórico de importações anteriores (v2)
  • Notificações Telegram (v2)
