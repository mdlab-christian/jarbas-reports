O que falta para o super agente

Organizadas por impacto. As P1 são o que separa o 7.8 do 9.5+.

⏰
Criar os 6 crons
🔴 P1 — Autonomia zero sem isso
O HEARTBEAT.md define 6 crons mas nenhum está criado. Sem crons, o Jarbas não age proativamente.
  • 08:00 daily-briefing → Sonnet
  • 08:30 agent-health-check → Haiku
  • 12:00 vault-checkpoint → Haiku
  • 18:00 custo-alert → Haiku
  • 22:45 memory-consolidation → Sonnet
  • 23:00 relatorio-diario → Sonnet
📁
Corrigir acesso ao mdlab-vault
🔴 P1 — Memória técnica inacessível
O mdlab-vault fica em /Users/jarbas/ (usuário legado). O usuário christian não tem acesso. Precisa symlink ou ajuste de permissão.
  • Criar symlink /Users/christian/mdlab-vault → /Users/jarbas/mdlab-vault
  • Ou ajustar permissões de grupo via sudo
  • Reativar extraPath no memorySearch
  • Testar leitura de 04_stack/agentes.md direto
📓
Instalar skill steipete/obsidian
🔴 P1 — Integração nativa Obsidian
A skill obsidian do ClawhHub dá acesso programático ao vault — busca, criação de notas, links. Sem isso, acesso ao vault é só via read/write.
  • clawhub install steipete/obsidian
  • Configurar path do jarbas-vault e mdlab-vault
  • Testar busca semântica dentro do vault
🧠
Popular self-improving com domínios
🟠 P2 — Qualidade cumulativa
~/self-improving/ só tem memory.md. Os domains/ e projects/ ainda estão vazios — o aprendizado por domínio não está ativo.
  • Criar domains/mdflow.md — padrões do projeto
  • Criar domains/supabase.md — decisões de banco
  • Criar projects/midas.md — operação jurídica
  • Migrar lições da JANA para os domínios
🦞
Auditar crons do Jarbinhas
🟠 P2 — Custo oculto no Mac Mini
O Jarbinhas pode ter crons legados com systemEvent (não funcionam) ou configurações antigas não otimizadas.
  • SSH Mac Mini → listar crons ativos
  • Remover/corrigir crons com systemEvent
  • Aplicar cacheRetention: long no Jarbinhas
  • Verificar custo do Mac Mini nos últimos dias
📊
Cost tracker diário automático
🟠 P2 — Visibilidade de gasto
Sem monitoramento ativo de custo, os picos só são descobertos depois. O cron de 18h precisa comparar com média histórica.
  • Cron custo-alert com threshold configurável
  • Relatório semanal de custo por agente
  • Alerta se gasto > 2x baseline normal
🔑
Configurar .env do usuário christian
🟡 P3 — Organização
As credenciais estão em ~/jarbas.env (raiz). Não está sendo lido automaticamente pelo OpenClaw — precisa checar se está no path correto.
  • Verificar se OpenClaw lê ~/jarbas.env
  • Ou mover para ~/.env (padrão)
  • Confirmar SUPABASE_KEY, OPENAI_API_KEY acessíveis
🛡️
LaunchAgent para reinício automático
🟡 P3 — Resiliência
Se o MacBook Air reiniciar, o Jarbas 2.0 precisa subir automaticamente. Checar se o LaunchAgent está configurado corretamente.
  • Verificar ~/Library/LaunchAgents/ai.openclaw.gateway.plist
  • Testar reinício automático
  • Confirmar que porta 18790 sobe sem intervenção
