Judit API × Cecília Controller

Análise de viabilidade e plano de integração entre a API de monitoramento judicial Judit.io e o sistema Cecília/Controller da MdFlow.

⚖️

VEREDICTO: INTEGRAR — não substituir

A Judit API não pode substituir a Cecília para o core do EPROC — a barreira técnica de CSRF em todas as URLs internas torna o Playwright insubstituível para leitura de intimações autenticadas. Porém, a Judit oferece 3 capacidades genuinamente novas que hoje a Cecília não tem: descoberta de processos por OAB, enriquecimento de metadados via datalake e jurimetria histórica. O plano correto é integrar seletivamente com escopo bem definido, iniciando por um piloto de R$1.000/mês.

O que é a Judit API
Plataforma brasileira de automação de consultas judiciais — fundada 2021
Requests Service
requests.prod.judit.io
ASYNC
Consultas assíncronas por CPF, CNPJ, OAB, CNJ ou nome. Fluxo: criar request → polling de status → buscar resposta. Suporta filtros por tribunal, data, valor, fase e tipo de parte.
Lawsuits Service
lawsuits.production.judit.io
SYNC / DATALAKE
Consultas síncronas contra o datalake Judit. Inclui Hot Storage para dados recentes. Mais barato que crawler em tempo real — ideal para metadados e histórico.
Tracking Service
tracking.prod.judit.io
MONITORAMENTO
Monitoramento contínuo com recorrência configurável em dias. Detecta novas movimentações, novas ações e mudanças em processos. Retorna tracking_id + webhooks.
Credential Vault
crawler.prod.judit.io
⚠️ AVALIAR
Cofre seguro para credenciais de tribunais. A Judit usa essas credenciais no seu próprio crawler para buscar dados em sistemas que requerem autenticação. Ponto chave: pode cobrir EPROC com nossas credenciais de OAB?
JUDIT IA
Módulo incluso
R$0,10/sumário
Sumarização automática de processos e entidades via IA. R$0,10/processo para sumário completo, R$0,15/entidade. Complementa a análise da OlivIA para o portal do cliente.

A Questão Crítica: EPROC
Por que a Judit não pode substituir o Playwright para EPROC — e o que isso significa
⛔
CSRF hash em TODAS as URLs internas do EPROC. Pesquisa anterior (Cecília v5.0, 15 Abr 2026) testou 5 alternativas ao Playwright — Skyvern, Browser Use, Firecrawl, Computer Use, Managed Agents. Todas descartadas. O EPROC gera tokens CSRF únicos por sessão nas URLs internas, impossibilitando qualquer acesso HTTP puro mesmo com cookies válidos. Playwright é insubstituível.
O que a Judit poderia ou não poderia fazer com EPROC
Tarefa EPROC Cecília (Playwright) Judit (API) Observação
Login autenticado OAB + TOTP ✅ Funciona ❓ Credential Vault Judit supostamente usa suas credenciais — se o crawler deles roda Playwright, pode funcionar. Não confirmado para EPROC.
Navegar URLs internas com CSRF ✅ Funciona ❌ Improvável CSRF hash único por sessão. HTTP puro falha sempre. Só browser session real resolve.
Ler texto de intimações ✅ Funciona ❌ Não exposto A Judit retorna metadados e andamentos, não o texto bruto de intimações EPROC.
Download de petições/despachos ✅ Funciona ❓ R$3,50/doc A Judit oferece download de anexos, mas a R$3,50/doc e sem confirmação de suporte EPROC.
Verificar prazos abertos ✅ Funciona ❌ Não tem A análise de prazo aberto para a parte representada é específica do contexto da OAB autenticada.
Descobrir processos novos por OAB ❌ Não faz 🌟 R$0,25 DIFERENCIAL. Busca por OAB retorna todos os processos — incluindo novos ainda não no banco.
Metadados: vara, comarca, valor ⚠️ Parcial 🌟 Datalake DIFERENCIAL. Enriquece campos NULL (advogado_reu_nome, vara, comarca) via datalake barato.
Jurimetria histórica ❌ Não faz 🌟 Histórico DIFERENCIAL. Consulta histórica por juiz, vara e empresa — alimenta /jurimetria do MdFlow.

Análise de Riscos
4 riscos identificados com mitigação
CRÍTICO

Suporte a EPROC não confirmado

A Judit lista "90+ tribunais" mas EPROC RS/SC/SP é um sistema proprietário do CJF com CSRF obrigatório. Não há nenhuma menção explícita de EPROC na documentação pública. O Credential Vault pode ou não ter integração real com EPROC. Mitigação: contatar comercial Judit antes de contratar e solicitar confirmação técnica específica de EPROC. Pilotar com 1 processo real.

ALTO

Dependência de serviço terceiro para descoberta

Se a Judit ficar offline ou mudar preços, o Job 1 (descoberta de novos processos) para de funcionar. Hoje a Cecília não faz isso — seria uma capacidade nova com dependência externa. Mitigação: implementar como job não-crítico. Cecília continua sendo fonte primária. Judit complementa, nunca bloqueia.

MÉDIO

Freshness do datalake vs. EPROC em tempo real

O datalake Judit tem delay (não especificado na documentação — pode ser horas ou dias). Para enriquecimento de metadados isso é aceitável, mas para status processual não. Mitigação: usar Judit apenas para enriquecimento (slow path). Cecília mantém o status em tempo real (fast path).

BAIXO

Custo escala se expandirmos monitoramento

Se no futuro quisermos usar Tracking Service para processos prioritários, o custo pode escalar rápido. Com 500 processos monitorados diariamente = R$22.500/mês. Mitigação: nunca usar Tracking para todos os 5.800. Apenas para casos com acordos em aberto ou prazos críticos — máx. 200-300 processos.
