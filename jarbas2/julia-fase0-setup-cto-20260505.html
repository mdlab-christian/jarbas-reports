3.1 — Criar vault remote no Obsidian Sync (web)

  1. Acessar https://obsidian.md/sync
  2. Login com sua conta Obsidian (mesma do MacBook Pro)
  3. Clicar em "Create new remote vault" (botão verde)
  4. Nome: julia-vault (exato, com hífen)
  5. Region: escolher US East ou EU West (qualquer uma — escolha a mais perto)
  6. Encryption: ⚠️ ATIVAR "End-to-end encryption" — gera passphrase NOVA
  7. Quando aparecer a passphrase, SALVE EM LOCAL SEGURO (1Password, Bitwarden, KeePass) — sem ela NADA recupera os dados se perdidos
🚨 PASSPHRASE — passo crítico

Se perder a passphrase, perde tudo do julia-vault. Sem recuperação, sem reset, sem suporte. Use gerenciador de senha confiável (1Password recomendado) e marque como pinned/favorito.

3.2 — Conectar Obsidian app no MacBook Pro (master)

  1. Abrir Obsidian.app no MacBook Pro
  2. Clicar no ícone do vault no canto inferior esquerdo → "Manage vaults"
  3. Clicar em "Open" → "Create new vault"
  4. Nome do vault: julia-vault
  5. Localização: /Users/christianpaz/julia-vault (mesmo padrão dos outros: jarbas-vault, mdos-vault)
  6. Clicar em "Create"
  7. Após o vault abrir: Settings → Community plugins → procurar "Sync" (oficial Obsidian) → ativar (se ainda não estiver)
  8. Settings → Sync → Login → conectar sua conta Obsidian (mesma do passo 3.1)
  9. Choose remote vault: selecionar julia-vault (criado no passo 3.1)
  10. Inserir a passphrase (a mesma que você salvou no 1Password)
  11. Início da sync: deve ser instantâneo — vault remoto está vazio, vault local também

Passo 4 — Confirmar Tailscale + sudoers no Mac Mini.

4

Confirmar Mac Mini Tailscale + sudoers admin

~3min

📋 O QUE

Confirmar que o user jarbas no Mac Mini tem sudoers sem senha (necessário para criar user julia, configurar LaunchAgents, dar Full Disk Access).

💡 POR QUE

O Claude vai executar comandos como sudo dscl . -create /Users/julia (criar usuário) e sudo launchctl bootstrap (registrar daemon Julia). Sem NOPASSWD no sudoers, automode trava em prompt de senha que ninguém digita.

⚙️ COMO

4.1 Verificar Tailscale online no Mac Mini:

ssh mac-mini-tail "sudo tailscale status"

Esperado: lista de devices com IP 100.x.x.x e status conectado.

4.2 Testar sudo sem senha:

ssh mac-mini-tail "sudo -n true && echo OK || echo FAIL"

Esperado: retorna OK sem prompt de senha.

4.3 Se retornar FAIL (precisa configurar sudoers):

  1. SSH no Mac Mini: ssh mac-mini-tail
  2. Editar sudoers: sudo visudo -f /etc/sudoers.d/jarbas-nopasswd
  3. Adicionar a linha: jarbas ALL=(ALL) NOPASSWD: ALL
  4. Salvar (Ctrl+X → Y → Enter no nano, ou Esc → :wq no vi)
  5. Testar de novo: sudo -n true && echo OK

✅ VERIFICAÇÃO

  • Tailscale status retorna lista de devices (Mac Mini incluso)
  • sudo -n true retorna OK sem pedir senha
