Por Que Este Projeto

S1

Premissa central: O escritório MP Advogados tem dados reais de receita (Organizze 2020–2026), histórico de vitórias (ADVBox + EPROC), e carteira ativa com ~5.136 processos. Este projeto cruza essas três fontes para calcular — não chutar — o valor financeiro real do escritório.

💰

Receitas Passadas Reais

O Organizze registra R $ 7.622.192 recebidos de 1.329 CNJs únicos entre 2020 e abril/2026. São dados financeiros reais — não projeções — que servem como base histórica concreta para qualquer modelo preditivo.

📋

Títulos Executivos Judiciais

~2.274 CNJs ganhos identificados via cruzamento de 4 fontes (Organizze, ADVBox API, Procedentes XLSX, Execução XLSX). Processos com decisão favorável mas ainda não recebidos representam receita futura mapeável.

⏱️

Ciclos Reais por Fase

Cruzando data de ajuizamento (EPROC) com data de pagamento (Organizze), é possível calcular o ciclo médio real — quantos dias do ajuizamento até o recebimento — por comarca, réu, fase e advogado.

📈

Projeção com Dados Reais

Para cada processo ativo: fase atual → ciclo médio da fase → data estimada de pagamento. Resultado: fluxo de caixa projetado 24 meses com intervalos de confiança P25/P50/P75. Não chutamos — calculamos.

Decisões Pendentes do CTO

S5

As 7 decisões abaixo definem o comportamento do ETL e dos modelos de ciclo. Cada uma tem uma recomendação fundamentada — o CTO responde em decisoes/aprovadas-2026-04-18.md.

D-01 Definição de “Processo Ganho”
A — Apenas Organizze (pagamento confirmado)
B — Apenas ADVBox fees_money > 0
C — União de todas as fontes (Organizze + fees + Procedentes + Execução) recomendado

C maximiza cobertura. O cruzamento real revelará sobreposições — a dedup por CNJ resolve.

D-02 Marco Zero do Ciclo
A — Data de distribuição ADVBox (created_at)
B — Ano do CNJ (7 dígitos = AAAA)
C — EPROC data_distribuicao se disponível, senão year do CNJ recomendado

C usa a fonte mais precisa disponível por processo, com fallback confiável.

D-03 Separação Acordo vs Decisão
A — Usar apenas Execução XLSX (mais confiável)
B — Inferir do stage ADVBox
C — Combinar todas as fontes + campo tipo_resultado no CSV recomendado

Acordo e decisão têm ciclos muito distintos. C garante máxima cobertura para o modelo.

D-04 Processos sem Data de Distribuição
A — Excluir do cálculo de ciclo
B — Imputar com média da coorte
C — Calcular P10/P50/P90 separado, sinalizar no CSV recomendado

C mantém transparência estatística — dados incompletos ficam visivelmente sinalizados.

D-05 Modelo de Ciclo para Ativos
A — Ciclo médio histórico geral (todos os processos)
B — Ciclo médio REAL calculado por fase ADVBox + réu + advogado recomendado
C — Modelo de regressão (complexidade alta)

B é o equilíbrio ideal: usa dados reais sem overfitting estatístico.

D-06 Armazenamento Obsidian
A — Apenas ~/Desktop (temporário)
B — mdlab-vault/jurimetria
C — jarbas-vault/02_projetos/jurimetria/ + symlinks para CSVs recomendado

C usa o vault primário com Obsidian Sync — persiste e sincroniza automaticamente.

D-07 Alimentação MDFlow Supabase
A — Importar já na Fase 1 do projeto
B — Nunca alimentar Supabase, manter só local
C — Fase 2: após validação manual do HTML gerado em S04 recomendado

C evita poluir o banco de produção com dados não validados. Primeiro validamos visualmente no HTML, depois alimentamos o MDFlow.

Riscos Identificados

S6
R01
Ausência de fees (64%)
64% dos processos ADVBox têm fees_money = 0 ou null. O valor financeiro real desses processos é desconhecido.
Alta Financeiro
R02
APIs bloqueadas
ADVBox movements e transactions bloqueados por Cloudflare. Dados de movimentação detalhada indisponíveis.
Média Cobertura
R03
Match CNJ falha
Formatações distintas do CNJ entre fontes podem causar falsos negativos no cruzamento.
Alta Join
R04
Organizze incompleto
Nem todas as receitas foram registradas com CNJ. Receitas de clientes pessoas físicas podem estar sem identificador.
Média Histórico
R05
EPROC sub-representado
SC/SP podem ter volume menor que RS. Ciclos calculados para essas praças podem ter baixa significância estatística.
Baixa Estatística
R06
Datas imprecisas
Processos com data_distribuicao ausente no EPROC forçam uso do year do CNJ — precisão reduzida nesses casos.
Baixa Ciclos
R07
Token ADVBox expira
Cache local pode ficar desatualizado se o token expirar antes de concluir o ETL. Renovação manual necessária.
Média Operacional
R08
Lacunas 2022–2023
Período com possíveis gaps de registro no Organizze. Ciclos para coortes 2022–2023 podem ter viés de sobrevivência.
Baixa Histórico
