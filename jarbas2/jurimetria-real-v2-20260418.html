Como os dados foram calculados

    Financeiro (2020–2026): 4 planilhas XLS do aplicativo Organizze → 4.379 lançamentos analisados → 1.706 receitas identificadas → 1.329 CNJs únicos cruzados → receita média real calculada por processo único
  • F2
    ADVBox Procedentes: 4 arquivos XLSX → 263 registros de compromissos conclusivos (JULGADO PROCEDENTE + DECISÃO 2º GRAU)
  • F3
    ADVBox Execução/Recursal: 2 arquivos XLSX → 244 processos com valores de honorários detalhados → R$ 1.484.912 em pipeline ativo
  • F4
    EPROC (RS Dr Christian): 8.597 processos com data de distribuição → mapeamento de coorte por ano → cálculo de taxa de sucesso por coorte
  • CX
    Chave de cruzamento: número CNJ (NNNNNNN-DD.AAAA.J.TT.OOOO) presente nos lançamentos financeiros. Deduplicação: múltiplos pagamentos do mesmo CNJ (execução parcelada) somados em valor único por processo

Plano ADVBox API — Conexão e Extração

Por que conectar
Os dados das planilhas são snapshots. A API permite:
• Consultar histórico completo de cada processo
• Identificar Acordo vs. Decisão Favorável
• Extrair data sentença → acórdão → trânsito
• CNJ → data sentença → data pagamento = ciclo real medido
Autenticação
Base: https://app.advbox.com.br/api/v1
Authorization: Bearer EXgoDJuTDkiRBTamOQncUtRdNW1E5CEX4aXhZmqZ3S1D1CbvfCH0DPbIOVri
Rate limits: 30 GET/min · 500 POST/dia
Problema: Cloudflare bloqueia curl direto → usar Playwright headless
Endpoint Método Uso na Jurimetria
/settings GET Obter IDs de configuração (necessário para filtros)
/customers GET Listar clientes com paginação
/lawsuits GET Listar processos (filtro por status, OAB, fase)
/lawsuits/{id} GET Detalhe: partes, valores, datas, resultado
/publications GET Movimentações/publicações por processo
/tasks GET Atividades: compromissos, prazos, resultados
/transactions GET Transações financeiras no processo

Acesso API
1 dia
  • Testar autenticação via Playwright/Node.js (contornar Cloudflare)
  • Alternativa: usar CSV export direto do ADVBox (mais simples)
  • Verificar /settings para obter IDs necessários
  • Validar rate limits e tokens de sessão
02
Extração — 1.329 CNJs com receita
3–5 dias
  • GET /lawsuits?cnj={numero} para cada CNJ do financeiro
  • Extrair: data_sentenca, data_transito, tipo_resultado, fase_atual, etiquetas
  • Identificar: ACORDO vs. DECISÃO FAVORÁVEL
  • Resultado: tabela processos_ganhos com ciclo completo
03
Extração — Pipeline ativo (244+ proc)
1–2 dias
  • GET /lawsuits?fase=execucao&fase=recursal (com paginação)
  • Extrair todos os processos ativos com valores
  • Classificar por urgência de pagamento
04
Normalização e Cruzamento
2–3 dias
  • JOIN financeiro × advbox por CNJ
  • Calcular: delta_dias(data_distribuição, data_sentença, data_pagamento)
  • Identificar outliers (R$ 120K max)
  • Deduplicar processos duplicados entre arquivos
05
Carga no MDFlow Supabase
2–3 dias
  • INSERT na tabela decisoes (processo ganho + valor + data)
  • INSERT na tabela acordos (acordos + valores)
  • INSERT/UPDATE processos (enriquecer com dados ADVBox)
  • Alimentar tabela kpis_mensais para página /jurimetria
