01

Chromium: Problema e Fix

BUG P0-07
Causa raiz

O Playwright abre instancias de chrome-headless-shell para cada ciclo. Quando o ciclo falha (timeout, erro de login, crash), o browser NAO e fechado — fica como processo zumbi consumindo RAM.

Evidencia

55 processos Chromium detectados num unico momento no Mac Mini.

Antes (bug)
const browser = await chromium.launch(); const page = await browser.newPage(); await fazerLogin(page); // se lanca excecao aqui... await browser.close(); // ...NUNCA CHEGA AQUI
Depois (fix)
const browser = await chromium.launch(); try { const page = await browser.newPage(); await fazerLogin(page); } finally { await browser.close(); // executa SEMPRE }

Fix completo em 3 partes

1
try/finally em S1, S4, S5
Todo chromium.launch() deve ter bloco try/finally com browser.close() garantido. Stages afetados: S1 (login), S4 (documentos), S5 (extracoes).
2
Limpeza de orfaos no startup
Primeiras linhas de start-cron.mjs verificam se ha mais de 2 instancias. Se sim, elimina com pkill e loga quantidade.
import { execSync } from 'child_process'; try { const count = execSync("pgrep -c chrome-headless-shell").toString().trim(); if (parseInt(count) > 2) { execSync('pkill -f chrome-headless-shell'); console.log(`[startup] ${count} orfaos eliminados`); } } catch {}
3
Limite de instancias simultaneas
Nunca abrir mais de 1 browser por OAB ao mesmo tempo. Mutex por OAB em start-cron.mjs — execucao serializada garantida.
Resultado esperado

De 55 processos zumbis para maximo 1 processo ativo por OAB + 0 orfaos. Alertas desaparecem completamente.

Pags tipicas/dia
Coluna-chave: Data/Hora Ult. Evento — ordenacao DECRESCENTE. Quando pagina inteira tem eventos mais antigos que a janela, parar paginacao. Economia: 55 paginas → tipicamente 1-3 para janela 24h.

Algoritmo de coleta (novo S2)

FUNCAO coletar_processos(oab, janela_horas): pagina = 1 processos = [] ENQUANTO pagina <= total_paginas: linhas = navegar_pagina(pagina) todos_antigos = TRUE PARA CADA linha: data_hora = parsear(linha.data_hora_ult_evento) SE data_hora >= agora - janela_horas: processos.append({ cnj, ultimo_evento, data_evento }) todos_antigos = FALSE SE todos_antigos: PARAR // paginas seguintes sao mais antigas pagina += 1 RETORNAR processos

Logica de janela temporal inteligente

Ciclo normal: janela = 24h a partir do horario do cron Retry (+6h): janela = 30h (24h + 6h de gap = zero lacuna) // Exemplo RS116571: Cron 10h00 coleta movimentos de 10h ontem ate 10h hoje Falhou retry 16h00 coleta 10h ontem ate 16h hoje (30h)
