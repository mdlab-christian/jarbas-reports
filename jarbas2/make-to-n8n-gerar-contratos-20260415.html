01. Schema Drift — Make vs DB Atual (CRITICO)

ALERTA CRITICO: 11 campos mudaram de nome ou foram removidos

O blueprint Make foi construido contra um schema antigo da tabela leads. Os nomes de coluna mudaram significativamente. O n8n DEVE usar os nomes atuais.

Campo Make (antigo) Campo DB Atual Tabela Status Nullable?
leads.name leads.nome leads RENOMEADO NOT NULL
leads.estado leads.endereco_estado / leads.endereco_uf leads RENOMEADO NULLABLE
leads.drive_cliente leads.google_drive_folder_id leads RENOMEADO NULLABLE
leads.logradouro leads.endereco_logradouro leads RENOMEADO NULLABLE
leads.numero_endereco leads.endereco_numero leads RENOMEADO NULLABLE
leads.cep leads.endereco_cep leads RENOMEADO NULLABLE
leads.cidade leads.endereco_cidade leads RENOMEADO NULLABLE
leads.codigo_advogado leads.advogado_id → advogados.oab leads + advogados REMOVIDO NULLABLE
leads.advogado (text) leads.advogado_id (UUID FK) leads TIPO MUDOU NULLABLE
leads.contratos_gerados — leads REMOVIDO —
restricoes_orgao (tabela) restricoes (tabela unica) restricoes MERGED —
Campos de Endereco: TODOS NULLABLE

Os campos endereco_logradouro, endereco_numero, endereco_cep, endereco_cidade, endereco_estado sao todos nullable. O n8n deve tratar null como string vazia no template — o documento tera campos em branco mas sera gerado mesmo assim.

04. Drive Folder Resolution

INCONSISTENCIA: drive_cliente pode ser ID ou URL completa

Alguns registros em clientes.drive_cliente contem apenas o folder ID (1PtAiAJde...), outros contem a URL completa (https://drive.google.com/drive/folders/1N2uL...). O n8n DEVE extrair o ID da URL quando necessario.

Cadeia de prioridade (primeiro encontrado vence)
1
restricoes.drive_pasta_id
Subpasta especifica da restricao (frequentemente NULL)
2
clientes.drive_cliente
Pasta do cliente — pode ser ID ou URL completa (precisa extrair)
3
leads.google_drive_folder_id
Pasta do lead (fallback final)
function resolveDriveFolder(lead, cliente, restricao) { if (restricao?.drive_pasta_id) return restricao.drive_pasta_id; if (cliente?.drive_cliente) { const val = cliente.drive_cliente; // Extrair ID de URL completa const match = val.match(/\/folders\/([a-zA-Z0-9_-]+)/); return match ? match[1] : val; // se nao e URL, assume que e ID } if (lead.google_drive_folder_id) return lead.google_drive_folder_id; return null; // ERRO: nenhuma pasta encontrada }

07. Code Node 7 — Document Queue Builder

Node mais complexo do workflow

Este Code Node recebe lead, advogado, restricoes e templates como input, e gera um array de documentos a serem criados. Cada item do array contem: template_id, doc_name, variables (mapa de placeholders), folder_id, tipo_doc e restricao_id.

findTemplate() — 3 niveis de fallback
function findTemplate(templates, tipo, grupo, subtipo) { // Nivel 1: match exato (tipo + grupo + subtipo) let t = templates.find(t => t.tipo === tipo && t.grupo_advogado === grupo && t.subtipo === subtipo ); if (t) return t; // Nivel 2: sem subtipo (templates compartilhados como termo_ciencia) t = templates.find(t => t.tipo === tipo && t.grupo_advogado === grupo && (!t.subtipo || t.subtipo === null) ); if (t) return t; // Nivel 3: universal (grupo_advogado = 'todos') t = templates.find(t => t.tipo === tipo && t.grupo_advogado === 'todos' ); if (t) return t; return null; // TEMPLATE NAO ENCONTRADO - adicionar ao errors[] }

10. Comparativo Make vs n8n

Make (atual — desligar)

Nodes: 210 nodes em 1 cenario
Error handling: Nenhum — falha silenciosa
DB writes: Nenhum — so leitura
Tracking: Nenhum — processos_documentos vazio
Webhook response: Fire-and-forget
Drive folder creation: Nao cria — assume existencia
Schema: Desatualizado — 11 campos errados
Router logic: Hardcoded por OAB em cada rota
Template lookup: IDs hardcoded em cada rota do router
Custo: Plano Make $29/mes (1.000 ops)
Restricoes: Tabela separada restricoes_orgao (nao existe mais)

n8n (novo — construir)

Nodes: 17 nodes + error handler
Error handling: Completo — catch + log + continue
DB writes: INSERT processos_documentos por doc
Tracking: Completo — status disponivel/falhou
Webhook response: Action-driven (toast_refresh)
Drive folder: Resolve com fallback 3 niveis
Schema: Atualizado — nomes corretos
Router logic: Dinamico via documento_templates
Template lookup: findTemplate() com 3 niveis fallback
Custo: Railway $5/mes (shared infra)
Restricoes: Tabela unica restricoes com empresa_nome + orgao_nome
Melhorias do n8n que Make NUNCA teve
FeatureMaken8nImpacto

Status de falha por documentoNaoSimIdentificar exatamente qual doc falhou e por que
Webhook response com contagemNaoSimUI mostra "12 docs gerados, 1 falhou"
Validacao de dados antes de gerarNaoSimErro claro em vez de doc com campos vazios
