Document Queue Builder Logic (Node 6)

function buildQueue(modo, lead, restricoes, templates, grupo, subtipo) { const queue = []; const findTemplate = (tipo, needSubtipo) => templates.find(t => t.tipo === tipo && (t.grupo_advogado === grupo || t.grupo_advogado === 'todos') && (needSubtipo ? t.subtipo === subtipo : t.subtipo === null) ); // ---- MODO COMPLETO ---- if (modo === 'completo' || modo === 'contrato_midas') { // 1.1 Contrato Midas (universal, grupo="todos") queue.push({ template: findTemplate('contrato_midas', false), vars: leadVars(lead) }); } if (modo === 'completo' || modo === 'termo_parceria') { // 1.2 Termo de Parceria (OAB-specific) queue.push({ template: findTemplate('termo_parceria', true), vars: leadVars(lead) }); } // ---- POR EMPRESA ---- if (modo === 'completo' || modo === 'empresa') { const empresas = restricoes.filter(r => r.empresa_id !== null); for (const emp of empresas) { const v = { ...leadVars(lead), ...empresaVars(emp) }; queue.push({ template: findTemplate('contrato_empresa', true), vars: v }); // OAB-specific queue.push({ template: findTemplate('termo_ciencia_empresa', false), vars: v }); // adv-only queue.push({ template: findTemplate('termo_veracidade_empresa',false), vars: v }); // adv-only queue.push({ template: findTemplate('procuracao_empresa', true), vars: v }); // OAB-specific } } // ---- POR ORGAO ---- if (modo === 'completo' || modo === 'orgao') { const orgaos = restricoes.filter(r => r.orgao_id !== null); for (const org of orgaos) { const v = { ...leadVars(lead) }; queue.push({ template: findTemplate('contrato_orgao', true), vars: v }); queue.push({ template: findTemplate('termo_ciencia_orgao', false), vars: v }); queue.push({ template: findTemplate('termo_veracidade_orgao',false), vars: v }); queue.push({ template: findTemplate('procuracao_orgao', true), vars: v }); } } // Filtrar templates nao encontrados (logar warning) return queue.filter(q => { if (!q.template) { console.warn('Template nao encontrado'); return false; } return true; }); }

Edge Cases e Validacoes

CRITICO: Heloisa + Lead SP
Heloisa nao possui OAB em SP. Se lead.estado = "SP" e advogado = Heloisa, a geracao vai falhar.
Solucao: Node 3 valida a combinacao. Retorna erro claro: "Advogada Heloisa nao possui OAB no estado SP. Reatribua o lead para Christian."
Template nao encontrado
Se documento_templates nao retorna match para tipo+grupo+subtipo especifico.
Solucao: Log warning, skip documento, continua batch. Response final lista docs faltantes. Nao crashear o batch inteiro.
Timeout (120s frontend)
Lead com 3 restricoes gera ~14 docs. Budget ~8s/doc = ~112s. Margem apertada.
Solucao: Webhook responde imediato com action "toast_refresh". Processamento assincrono. Polling via contratos_gerados flag ou Supabase Realtime.
Drive folder ausente
lead.drive_cliente pode ser NULL se pasta ainda nao foi criada.
Solucao: Node 6 verifica. Se NULL, cria pasta no Drive primeiro, salva ID no lead. Depois continua geracao.
Restricao sem empresa_nome ou orgao_nome
Dados de restricao incompletos impedem preenchimento correto do template.
Solucao: Validar campos obrigatorios antes de adicionar ao queue. Skip + log se incompleto.
Modo parcial (empresa/orgao) sem restricao_id
Se modo = "empresa" mas restricao_id nao informado, gera para todas empresas ou erro?
Solucao: Se restricao_id informado, gera so para aquela. Se nao, gera para todas restricoes do tipo. Documentar no contrato do webhook.

Restricoes - Tabela Unica (correcao do Make)

O Make antigo usava 2 tabelas separadas (restricoes + restricoes_orgao). O MdFlow atual usa 1 tabela unica com diferenciacao por campos:

EMPRESA (empresa_id IS NOT NULL)
restricoes.empresa_id -- UUID FK restricoes.empresa_nome -- "Serasa" restricoes.codigo_contrato restricoes.valor -- INT8 centavos
ORGAO (orgao_id IS NOT NULL)
restricoes.orgao_id -- UUID FK restricoes.orgao_nome -- "TJRS" -- sem codigo_contrato -- sem valor
Filtragem no Node 6: restricoes.filter(r => r.empresa_id !== null) para empresas, restricoes.filter(r => r.orgao_id !== null) para orgaos.
