1. Contexto de Negócio — Fluxo Pré-Cliente

Ponto Critico: Contratos são gerados ANTES do lead virar cliente

Os contratos e procurações são documentos que o lead precisa assinar para fechar o negócio. Nesta etapa:

  • Nenhum registro de cliente existe ainda (leads.cliente_id pode ser NULL)
  • Nenhum processo foi distribuído
  • O frontend cria um registro processo imediatamente antes de chamar o webhook (fix E08)
  • A pasta no Google Drive geralmente não existe — apenas 1 de 45 leads tem pasta
  • As restrições também não têm drive_pasta_id populado (0 de 77)

Formato dos placeholders nos Google Docs

Os templates usam placeholders no formato {{VARIAVEL}}. O Google Docs API replaceAllText substitui todas as ocorrências.

Placeholder Fonte (campo.tabela) Transform Nullable? Fallback

{{CPF}} leads.cpf Formatar: 000.000.000-00 WARN "___.___.___-__"
{{RG}} leads.rg Nenhum WARN "__________"
{{PROFISSAO}} leads.profissao Nenhum OK ""
{{ENDERECO}} leads.endereco_logradouro + endereco_numero Concatenar: "${logradouro}, ${numero}" OK ""
{{CIDADE}} leads.endereco_cidade Nenhum OK ""

{{UF}} leads.endereco_uf UPPER() OK ""

10 P0 Timeout (120s frontend) Budget ~7s/doc. Max ~17 docs por chamada. Se queue > 17, considerar split
11 P0 Google OAuth token expired n8n gerencia refresh automaticamente via credential. Verificar se credential está configurada
12 P0 Drive quota exceeded (rate limit) Implementar retry com backoff exponencial (1s, 2s, 4s). Max 3 retries
13 P1 Template Google Doc foi deletado manualmente Error no doc específico, log em webhook_requests, continuar com os demais
14 P1 Valor armazenado em centavos (bigint) Converter: (valor / 100).toLocaleString('pt-BR', {style:'currency', currency:'BRL'})
15 P2 drive_cliente em clientes pode ser URL completa (não ID) NAO RELEVANTE — usamos leads.google_drive_folder_id, não clientes.drive_cliente

17 P1 Termo de Parceria usa nome_completo especial (do webhook, não do lead) Verificar se webhook envia nome diferente. Se sim, usar. Se não, usar leads.nome
18 P2 Contrato Midas usa UPPER(estado), outros não Normalizar: sempre UPPER() no Code Node. Manter consistência

13. Resumo Executivo

O que muda (Make → n8n)
  • Make → n8n: Engine de automação
  • leads.name → leads.nome: Todos os campos renomeados
  • restricoes_orgao → restricoes (unificada): Tabela removida
  • leads.advogado (text) → leads.advogado_id (UUID FK)
  • Sem error handling → Webhook response canônico + logging
  • Sem tracking → processos_documentos + webhook_requests
O que NAO muda
  • Frontend (useGerarContrato.ts) — intacto
  • 3 endpoints separados — mantidos
  • Google Docs templates — mesmos 34 documentos
  • Fluxo de negócio — lead → contratos → assinatura → cliente
  • Upload para pasta do lead no Drive — mesmo destino
  • Timeout de 120s — mantido
Decisão final

Opção A (3 workflows separados) — sem alterar frontend.
6 fases sequenciais, começando pelo mais simples (Midas) e terminando com error handling.
34 templates mantidos no Google Docs, referenciados via documento_templates.google_doc_id_original.
18 edge cases mapeados com severidade e ação definida para cada um.
