Camada 1 — Agente

Definição do Agente

Você descreve o agente em linguagem natural via Claude Console. Ele gera automaticamente a configuração: nome, modelo (Haiku / Sonnet / Opus), description e system prompt completo.

Suporta Claude Haiku, Sonnet 3.5 e Opus 3
System prompt editável e versionado
Configuração via console.anthropic.com → aba "Agents"
Pode conectar múltiplos MCP servers
🎯
Camada 2 — Sessão

Instâncias de Execução

Cada sessão é uma instância de execução do agente. Reutilizável, versionada. Você cria via POST REST API e recebe código TypeScript pronto para embed em qualquer frontend.

POST /v1/agents/{id}/sessions para criar
Contexto persistido entre mensagens na sessão
Sessões independentes por usuário / caso
Código TypeScript gerado automaticamente para embed
🌐
Camada 3 — Ambiente

Controle de Execução

Container de execução gerenciado pela Anthropic. Sem necessidade de infraestrutura própria. Controle de acesso configurável: irrestrito ou limitado a domínios específicos.

Container hospedado na infraestrutura Anthropic
Acesso irrestrito ou whitelist de domínios
Zero DevOps — sem Docker, sem servidores
Scaling automático por demanda
🔐
Camada 4 — Credenciais

Vault Seguro

Vault de credenciais nativo: OAuth, API keys, tokens. O agente acessa recursos externos (Supabase, Notion, Gmail) sem expor credenciais no frontend ou no código.

OAuth 2.0 e API keys armazenadas com segurança
Conecta a qualquer MCP server disponível
Supabase MCP, Notion, Gmail, n8n e mais
Credenciais nunca expostas ao frontend

O que Managed Agents pode substituir no MdFlow?

SEÇÃO 03
🔀

n8n — Automações e Workflows

n8n e mais barato, mais flexivel, ja temos 30+ workflows maduros. Managed Agents seria 10x a 50x mais caro para o mesmo volume de execucoes. O n8n ja integra com Claude API diretamente quando necessario IA. Nao ha motivo para migrar.

Veredicto ❌ Nao substituir
📝

OlivIA em producao — Geracao de Peticoes (5.800 processos)

Inviavel economicamente. Chamar Claude API diretamente via n8n custa ~$0.003-$0.015 por 1K tokens (Haiku/Sonnet). Managed Agents adiciona overhead de sessao e hospedagem Anthropic. Para o volume do Midas, o custo mensal seria proibitivo.

Calculo de risco: O autor gastou R$3,77 em poucas mensagens de teste. Extrapolar isso para OlivIA processando centenas de peticoes/mes e inviavel operacionalmente.
Veredicto ❌ Nao substituir
🧪

Prototipagem de novos sub-agentes OlivIA

Excelente caso de uso. Antes de implementar um novo sub-agente na OlivIA v3 (que exige codigo TypeScript, EdgeFn, integracao n8n), criar um Managed Agent leva minutos. Valide o comportamento, o system prompt, as ferramentas necessarias. Depois migra para API direta.

Workflow ideal: Managed Agent (validar) → Spec confirmada → Implementar como sub-agente OlivIA com API direta.
Veredicto ✅ Otimo caso
👤

Portal do Cliente — Baixo volume

Depende do volume. Se cada cliente Midas fizer 5-10 perguntas/mes sobre seu processo, o custo pode ser aceitavel como diferencial de produto (cliente premium ve status do processo via chat IA). Mas precisa medir: quantos clientes ativos? Quantas mensagens/cliente/mes?

Veredicto ⚠️ Avaliar volume
💡

E util para MdFlow?

SIM — mas nao onde parece obvio. A utilidade real esta na velocidade de prototipagem e em nichos de baixo volume, nao em substituir a arquitetura existente.

✅ Usar aqui

✅ Prototipos rapidos de novos sub-agentes
✅ Portal do cliente — baixo volume
✅ Widget de suporte externo embed
✅ Validacao de system prompt antes de implementar
✅ MdChat interno (poucos usuarios, alta qualidade)
✅ Demo para prospect / pitch de produto

❌ Nao usar aqui

❌ Substituir n8n em producao
❌ OlivIA com volume alto de peticoes
❌ Automacoes EPROC / monitoramento frequente
❌ Qualquer fluxo com >500 interacoes/mes
❌ Processamento batch de processos Midas
❌ Substituir Cecilia (EPROC bot)
