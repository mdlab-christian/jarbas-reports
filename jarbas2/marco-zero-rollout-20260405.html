Comparação de Estratégia

Opção A — Automático via Crons
RECOMENDADO — LaunchAgents já existentes no Mac Mini (6h, 13h, 19h)

Cecilia lê fila, processa batch, pausa, continua sozinha. Notificações Telegram após cada sessão. Auto-retry para erros. Sem intervenção humana.

  • + Totalmente autônomo — sem necessidade de atenção constante
  • + Self-healing — erros vão para status 'erro', reprocessados automaticamente
  • + Respeita delays anti-bot automaticamente (800-2500ms, 5min/OAB)
  • + Notificações Telegram funcionam como dashboard de progresso
  • ~ Menos visibilidade granular (mitigado pelas notificações)
✅ Escolha Recomendada
Opção B — /projeto com Tasks por Lote
~145 tasks (100 processos cada), aprovação manual por task

Criar /projeto "Marco Zero Rollout". Cada lote de 100 processos vira uma task. Visibilidade total mas requer atenção constante.

  • + Controle granular — pausar/ajustar qualquer lote
  • + Visibilidade total do progresso
  • - Requer aprovação manual ~145 vezes
  • - Não escalável para 14.456 processos
  • - Depende da disponibilidade de Christian continuamente
Não recomendado para este volume
Solucao Hibrida Recomendada

Rodar em automático com notificações Telegram a cada sessão + criar uma view de progresso no MdFlow para monitorar em tempo real. Controle via Telegram sem necessidade de aprovação manual a cada lote.
