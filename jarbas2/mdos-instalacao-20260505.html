O que e o MdOs e o que vai acontecer

Leia antes de comecar — leva 2 minutos
MdOs e o sistema operacional CLI do MDLab. Pense nele como um assistente juridico que roda no terminal de cada computador da equipe. Uma vez instalado, o membro digita claude dentro da pasta do vault e tem acesso a todas as skills da equipe: gerar peticoes com a OlivIA, ver prazos do EPROC com a Cecilia, buscar processos e muito mais.
O que a instalacao faz, tecnicamente:
O script install.sh cria os atalhos de comandos (/mdos:*), configura as variaveis de ambiente com o email e nivel do membro, e tenta conectar ao servidor central (Mac Mini). Tudo automaticamente — o Gustavo so precisa rodar um comando em cada maquina.
Importante antes de comecar: cada membro precisa ter o Obsidian aberto com o vault MdOs-vault sincronizado. O sync pode levar 1-2 minutos na primeira vez. Se o vault nao estiver sincronizado, o install falha. Confirme o Obsidian em cada maquina antes de prosseguir.

Passo a passo da instalacao

Repetir em cada uma das 3 maquinas
1
Verificar o vault
Confirmar que o MdOs-vault esta sincronizado e completo

No terminal, rodar o comando abaixo. Deve aparecer uma lista de arquivos e pastas — se der "No such file or directory", o vault ainda nao sincronizou.

bash 
ls ~/Documents/MdOs-vault/
# Resultado esperado (deve ter esses itens):
# ACESSOS.md  CLAUDE.md  _tools  install.sh  skills  ...
Se a pasta aparecer vazia ou der erro: abrir o Obsidian, aguardar ate a barra de progresso de sync sumir por completo (canto superior direito), fechar e abrir o terminal, tentar novamente.
2
Rodar o install.sh
O script cuida de tudo automaticamente
bash 
bash ~/Documents/MdOs-vault/install.sh

Quando o script pedir o email, digitar exatamente como na tabela abaixo (sem espacos, sem maiusculas extras):

Membro Email para digitar Nivel
Joao Victor joao.1avila@gmail.com Gestor
Leonardo leonardops2020@gmail.com Gestor
Vinicius pazvinipaz@gmail.com Colaborador

Apos confirmar o email, o script vai executar automaticamente:

  • ✅ Validar o email no banco MdFlow
  • ✅ Detectar o nivel do membro (Gestor ou Colaborador)
  • ✅ Configurar variaveis de ambiente (TEAM_MEMBER + MDOS_EMAIL)
  • ✅ Criar symlinks de todos os comandos /mdos:*
  • ✅ Configurar respostas em portugues brasileiro
  • ✅ Gerar chave SSH e tentar conectar ao Mac Mini (servidor central)
3
Abrir novo terminal
Obrigatorio para carregar as configuracoes
Atencao: fechar completamente o terminal atual e abrir um novo antes de continuar. As variaveis de ambiente instaladas pelo script so ficam disponiveis em sessoes novas do terminal. Pular esse passo faz os comandos /mdos:* nao funcionarem.
4
Abrir o MdOs
Entrar no ambiente de trabalho
bash 
cd ~/Documents/MdOs-vault && claude
O terminal vai entrar no modo interativo do Claude Code. O prompt muda para indicar que esta dentro do MdOs. Agora os comandos /mdos:* estao disponiveis.
5
Login Claude.ai
Obrigatorio — cada membro precisa da propria conta

Dentro do Claude Code (apos o passo 4), digitar o comando de login:

mdos 
/login
  • → O browser abre automaticamente na pagina de login do Claude.ai
  • → O membro faz login com a conta Claude.ai pessoal dele
  • → Fecha o browser e volta ao terminal
  • → O Claude Code confirma o login com o nome do usuario
Cada membro precisa de uma conta Claude.ai propria. Nao compartilhar conta. Se algum membro ainda nao tem, criar em claude.ai antes de prosseguir.
6
Testar a instalacao
Confirmar que tudo funcionou

Rodar os 3 comandos de teste em sequencia:

teste 1 — status geral 
/mdos:status
# Resultado esperado: OlivIA online, Cecilia online, Supabase conectado
teste 2 — manual de comandos 
/mdos:ajuda
# Resultado esperado: lista completa de comandos disponiveis
teste 3 — skills do membro 
/mdos:minha-skills
# Resultado esperado: lista de skills do nivel (Gestor ou Colaborador)
Instalacao concluida com sucesso quando os 3 comandos respondem sem erros. O membro ja pode usar o MdOs normalmente a partir daqui.

Se der problema

Solucoes para os erros mais comuns
❌ "vault nao encontrado" ou "No such file or directory"
O vault do Obsidian ainda nao sincronizou. Abrir o Obsidian, aguardar a barra de progresso de sync no canto superior direito desaparecer por completo (pode levar 1-3 minutos na primeira vez). Depois fechar e abrir o terminal e tentar novamente.
❌ "email nao encontrado no banco" ou "user not found"
Verificar se o email foi digitado corretamente — sem espacos antes ou depois, sem letras maiusculas extras. Conferir com a tabela da Secao 3, Passo 2. Se o email estiver correto e ainda falhar, avisar o Christian — pode ser necessario verificar o cadastro no banco.
❌ "Mac Mini nao acessivel" ou "SSH connection refused"
Normal na primeira vez. O MdOs basico funciona sem o Mac Mini — busca de processos, OlivIA via web e dados do Supabase funcionam normalmente. Para usar a Cecilia e OlivIA CLI completa, o Gustavo precisa autorizar a chave SSH: o script exibe a chave publica gerada (~/.ssh/id_rsa.pub) — copiar e enviar para o Christian pelo WhatsApp com o nome do membro.
❌ "/mdos:status nao reconhecido" ou "command not found"
O terminal foi aberto antes do install terminar, ou o Passo 3 foi pulado. Fechar completamente o terminal atual e abrir um novo. As variaveis de ambiente so carregam em sessoes novas. Se persistir, verificar se o install.sh concluiu sem erros (deve mostrar "Instalacao concluida" no final).
❌ Claude Code em "API Key mode" — pedindo chave de API
O Claude Code esta configurado para usar chave de API em vez de login Claude.ai. Dentro do Claude Code, digitar /login e seguir o fluxo do browser. Se o modo nao mudar, pode ser necessario desinstalar e reinstalar o Claude Code: npm uninstall -g @anthropic-ai/claude-code && npm install -g @anthropic-ai/claude-code
❌ No Windows: "bash nao e reconhecido"
O Leonardo precisa usar o Git Bash (instalado junto com o Git para Windows) ou o WSL (Windows Subsystem for Linux). Nao usar o PowerShell nem o CMD para o install.sh. Clicar com botao direito no Desktop > "Git Bash Here" ou buscar "Git Bash" no menu iniciar.
