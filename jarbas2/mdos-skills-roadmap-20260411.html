Fase 1 — Critico

Semana 1-2 · 8 skills · alta frequencia + simples
1
/mdos:atividades-criar Rota: /atividades
CRITICO SIMPLES ⭐⭐⭐⭐⭐ ALLOW_WRITE ✅

Joao usa atividades multiplas vezes por dia — criar tarefas de prazo, follow-up de processo, lembrete de audiencia. Hoje exige abrir o MdFlow, navegar ate /atividades, clicar em "Nova Atividade", preencher formulario. Via terminal: uma frase resolve.

"cria atividade: verificar prazo processo 5000123, urgente, hoje" → Claude extrai: titulo, processo, data, prioridade → INSERT em atividades via mdos-db.mjs
Tabela atividades
Obrigatorios titulo, status='pendente', organizacao_id
Opcionais processo_id, data_vencimento, prioridade, responsavel_id
2
/mdos:atividades-concluir Rota: /atividades
CRITICO SIMPLES ⭐⭐⭐⭐⭐ ALLOW_WRITE ✅

Operacao mais repetida: marcar atividade como concluida. Hoje exige abrir /atividades, encontrar a tarefa, clicar no checkbox. Via terminal: busca por titulo e atualiza instantaneamente.

"conclui atividade 'verificar prazo processo 5000123'" → busca por titulo → UPDATE status='concluida', concluida_em=now()
Tabela atividades
UPDATE status='concluida', updated_at=now()
3
/mdos-controller:marcar-lido Rota: /controller
CRITICO SIMPLES ⭐⭐⭐⭐⭐ ALLOW_WRITE novo

Joao passa horas no /controller processando intimacoes. Marcar como "lido" e a operacao mais repetida. Via terminal: pode processar em lote sem abrir o MdFlow.

💡 Joao economiza 2-3 clicks por intimacao × ~30 intimacoes/dia = 60-90 clicks eliminados/dia
"marca como lida a intimacao do processo 5000123" → busca intimacao → UPDATE status='lida' via n8n webhook ou direto
Tabela intimacoes
UPDATE status='lida'
4
/mdos-controller:criar-atividade Rota: /controller → drawer → aba Acoes
CRITICO SIMPLES ⭐⭐⭐⭐⭐

Ao analisar intimacao no controller, Joao frequentemente precisa criar uma atividade vinculada. Hoje: abre drawer da intimacao → aba Acoes → clica "Criar Atividade" → preenche modal. Via terminal: uma frase.

🔗 Diferencial: skill ja conhece o contexto da intimacao — nao precisa preencher processo manualmente.
"cria atividade para a intimacao do processo 5000123: contestar prazo 15 dias" → busca intimacao_id → cria atividade vinculada ao processo
Tabelas intimacoes + atividades
Contexto intimacao_id auto-vinculado
5
/mdos-controller:registrar-decisao Rota: /controller → drawer → aba Acoes
CRITICO MEDIA ⭐⭐⭐⭐

Quando intimacao traz decisao judicial, Joao precisa registrar no sistema. Hoje: drawer → Acoes → "Registrar Decisao" → modal complexo com multiplos campos.

"registra decisao: processo 5000123, procedente, valor R$ 8.000, data 10/04/2026" → INSERT em decisoes → automaticamente dispara pipeline execucao
Tabelas decisoes + trigger → execucoes
Downstream pipeline execucao automatico
6
/mdos:decisoes-registrar Rota: /decisoes
CRITICO MEDIA ⭐⭐⭐⭐

Versao standalone do registro de decisao (sem passar pelo controller). Para quando advogado recebe decisao fora do EPROC — diretamente, por e-mail, etc.

"registra decisao: processo 5000123, parcialmente procedente, honorarios 15%, data sentenca 08/04/2026" → Claude extrai parametros → INSERT decisoes → propoe criar execucao se procedente
Tabela decisoes
Diferencial propoe execucao se procedente
7
/mdos:processos-criar Rota: /processos
CRITICO MEDIA ⭐⭐⭐⭐ ALLOW_WRITE novo

Criar processo novo hoje exige: /processos → botao "Novo Processo" → formulario com 12+ campos. Via terminal: uma entrevista curta resolve, especialmente util para importacoes manuais.

Modo entrevista: Claude pergunta CNJ, cliente, advogado, status, fase processual, tribunal → INSERT processos
Tabela processos
Modo entrevista interativa
Acao adicionar ALLOW_WRITE
8
/mdos:clientes-criar Rota: /clientes
CRITICO MEDIA ⭐⭐⭐⭐ ALLOW_WRITE novo

Criar cliente novo via terminal — especialmente util quando e referenciado durante uma peticao e ainda nao existe no sistema.

🔗 Diferencial: Claude pode criar cliente E ja vincular ao processo sendo trabalhado na mesma sessao.
"cria cliente: Maria Aparecida, CPF 123.456.789-00, Caxias do Sul RS, WhatsApp 54 99999-0000" → validacao CPF → INSERT clientes
Tabela clientes
Validacao CPF formato + unicidade

Fase 2 — Importante

Semana 3-4 · 8 skills · impacto operacional medio-alto
9
/mdos:execucoes-criar Rota: /execucoes
IMPORTANTE MEDIA ⭐⭐⭐⭐

Quando decisao e procedente, precisa abrir cumprimento de sentenca. Hoje: /execucoes → "Nova Execucao" → vincular decisao → preencher dados.

🔗 Sinergia: combinar com /mdos:calculadora → calcula o valor → ja cria a execucao com o valor correto.
"cria execucao para decisao do processo 5000123 — valor calculado R$ 17.666,71" → usa resultado do /mdos:calculadora → INSERT execucoes com valor_calculado
Tabela execucoes
Sinergia /mdos:calculadora
10
/mdos:audiencias-criar Rota: /audiencias
IMPORTANTE MEDIA ⭐⭐⭐

Agendar audiencias manualmente quando nao capturadas pelo EPROC automaticamente.

"agenda audiencia: processo 5000123, 15/05/2026 14h, Vara do Consumidor de POA, tipo instrucao" → INSERT audiencias → futuramente: sync Google Calendar
Tabela audiencias
Futura integr. Google Calendar (pendente MdFlow)
11
/mdos:acordos-registrar Rota: /acordos
IMPORTANTE SIMPLES ⭐⭐⭐

Registrar acordo extrajudicial ou judicial. Operacao mensal mas importante para rastreamento financeiro.

"registra acordo: processo 5000123, valor R$ 5.000, pagamento parcelado 3x, data 10/04/2026" → INSERT acordos → propoe criar atividade de acompanhamento de pagamento
Tabela acordos
Auto-acao propoe atividade de acompanhamento
12
/mdos:playbooks-consultar Rota: /playbooks (modelos)
IMPORTANTE SIMPLES ⭐⭐⭐

Consultar playbooks juridicos antes de gerar peticao — templates de contestacao, tipos de recurso, modelos de acao por tipo de restricao.

"qual o playbook para apelacao em TJSC negativacao indevida Serasa?" → SELECT olivia_playbooks WHERE tipo + tags → retorna secoes relevantes
🔗 Sinergia com OlivIA: o /mdos-olivia:peticao ja usa RAG de blocos, mas o controller pode consultar playbooks diretamente antes de iniciar geracao.
Tabela olivia_playbooks
Metodo SELECT por tipo + tags
13
/mdos:jurimetria-analisar-juiz Rota: /jurimetria
IMPORTANTE MEDIA ⭐⭐

Consultar jurimetria de um juiz antes de audiencia — taxa de procedencia, valor medio, tendencia. Versao aprofundada do /mdos:juiz existente, com analise por tipo de acao.

"como e o juiz Fulano na Vara 3 de POA para negativacao indevida?" → SELECT julgadores + decisoes WHERE julgador + tipo → calcula stats → apresenta analise
Tabelas julgadores + decisoes
vs. /mdos:juiz mais profundo, por tipo de acao
14
/mdos:docs-solicitar Rota: /documentos-pendentes
IMPORTANTE SIMPLES ⭐⭐⭐

Solicitar documento pendente ao cliente — procuracao, comprovante de residencia, CTPS, etc. Integra com MdZap para disparo automatico de WhatsApp.

"solicita procuracao ao cliente do processo 5000123" → busca cliente → dispara webhook MDZap → mensagem automatica WhatsApp → INSERT documentos_pendentes
📱 Sinergia MdZap: cliente recebe mensagem automatica no WhatsApp pedindo o documento — sem nenhuma acao manual adicional.
Tabela documentos_pendentes
Webhook MDZap → WhatsApp automatico
15
/mdos-controller:transferir Rota: /controller
IMPORTANTE SIMPLES ⭐⭐

Transferir intimacao de um advogado para outro — quando ha substituicao ou sobrecarga temporaria.

"transfere intimacoes do processo 5000123 para Heloisa" → UPDATE intimacoes SET advogado_id WHERE processo_id
Tabela intimacoes
UPDATE advogado_id por processo ou lote
16
/mdos:relatorios-gerar Rota: /relatorios
IMPORTANTE MEDIA ⭐⭐⭐

Solicitar geracao de relatorio periodico sem abrir o MdFlow. Enfileira na pipeline n8n e notifica quando pronto.

"gera relatorio de processos do mes de marco, advogado Christian, TJRS" → INSERT relatorios → dispara pipeline n8n → notifica quando pronto
Tabela relatorios
Pipeline n8n async + notificacao

O Que Nao Fazer — Analise de Exclusoes

6 grupos excluidos com justificativa
❌
/mdos:crm-* — Skills CRM
CRM do MdFlow e especifico da operacao Midas (restricoes de credito, leads WhatsApp, round-robin). Nao e operacao juridica — e operacao comercial. Se quiser, criar grupo separado /mdos-midas:* para nao misturar com skills juridicos.
❌
/mdos:mdzap-* — Skills MdZap
WhatsApp e canal de atendimento, nao operacao juridica. O MDZap ja tem automacao propria (Julia agent). Criar skill seria duplicacao desnecessaria. MdZap ja e usado como DOWNSTREAM por /mdos:docs-solicitar.
❌
/mdos:parceiros-* — Skills de Parceiros
Gestao de parceiros e administrativa, feita 1x/mes. Nao vale o custo de manutencao de skill para operacoes de baixissima frequencia.
❌
/mdos:financeiro-* — Skills Financeiros Completos
Financeiro tem cegueira por perfil (acesso_financeiro=false). A maioria da equipe nao tem acesso. Skill simples de lancamento (Fase 3, #20) e suficiente — evita exposicao de dados sensiveis via terminal.
❌
/mdos:mdacademy, homepage, mdlab — Interfaces
Sao interfaces de visualizacao, nao operacoes juridicas executaveis. Nao ha o que automatizar via terminal — sao telas de leitura.
❌
/mdos-admin:* — Admin Completo
Permissoes elevadas + alto risco de erro em producao. /mdos-admin:usuario-novo ja cobre o essencial. Operacoes de configuracao exigem contexto visual do MdFlow, nao terminal.

Arquitetura Tecnica

Como implementar cada skill
Padrao de implementacao: Skill (markdown) → mdos-db.mjs (SELECT/INSERT/UPDATE/RPC) → n8n webhook apenas para automacoes downstream. Sem nova ferramenta — reutilizar infraestrutura existente.
estrutura — skill atividades-criar.md
# Localizacao: skills/mdos/atividades-criar.md ← instrucao para o Claude _tools/mdos-db.mjs ← adicionar tabela ALLOW_WRITE n8n webhook ← apenas para downstream automations # Sem criar nova ferramenta — reutilizar: mdos-db.mjs select | insert | update | rpc
