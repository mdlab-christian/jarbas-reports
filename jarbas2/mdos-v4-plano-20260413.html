Nomenclatura Final — Padrao Adotado

Convencao
Regra definitiva: todas as skills seguem o formato /mdos-{grupo}:{acao} — compativel com Claude Code slash commands e semanticamente claro para toda equipe.
Formato Exemplo Status Motivo
/mdos-{grupo}:{acao} /mdos-atividades:criar ✓ Adotado Padrao oficial v4.0 — claro e compativel CC
/mdos-{grupo}:{acao} /mdos-decisoes:criar ✓ Adotado Todos os 11 novos grupos seguem este padrao
/mdos/{sub}:{acao} /mdos/decisao:criar ✗ Invalido Barra interna nao suportada pelo Claude Code
/mdos:{sub}-{acao} /mdos:decisao-criar ⚠ Funciona Polui o grupo generico /mdos — evitado

Arquitetura de Backend Integration

4 Caminhos
CONSULTA
→
mdos-db.mjs SELECT
→
Supabase (direto, RLS)
Sem side effects · Seguro para agentes
OPERACAO
→
mdos-db.mjs INSERT/UPDATE
→
Supabase ALLOW_WRITE
Sem automacoes · Apenas dados
AUTOMACAO
→
mdos-n8n.mjs POST /webhook
→
n8n → Supabase + notif.
WhatsApp, email, multi-step
TRANSACIONAL
→
mdos-db.mjs RPC {fn}
→
Supabase SECURITY DEFINER
Atomico · Multi-tabela seguro
Tabelas ALLOW_WRITE (10 tabelas autorizadas para escrita via MdOs)
Tabela Operacoes Via Regra principal
atividades INSERT UPDATE mdos-db criar, concluir, cancelar, editar
decisoes INSERT RPC RPC + n8n criar, aprovar via fn_decisao_aprovar
processos INSERT UPDATE n8n + mdos-db criar novo, editar campos nao-core
clientes INSERT UPDATE n8n + mdos-db CRM: criar leads, editar contatos
execucoes_etapas RPC RPC fn_execucao_avancar — atomico
audiencias INSERT UPDATE n8n + RPC criar, registrar resultado
acordos INSERT RPC RPC + n8n criar proposta, fn_acordo_marcar_pago
documentos_pendentes INSERT UPDATE mdos-db solicitar, concluir
intimacoes UPDATE mdos-db + n8n marcar lido, reprocessar via n8n
eventos_operacionais INSERT mdos-db audit trail de todas operacoes MdOs
