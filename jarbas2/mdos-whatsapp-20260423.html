Alertas criticos

🚫
BLOQUEANTE — Template proposta_acordo com status PENDING
O template principal de disparo automatico esta aguardando aprovacao da Meta. Nenhum disparo de proposta pode ser feito ate aprovacao. Acesse Facebook Business Manager > WhatsApp Manager > Message Templates. Se status for "Rejected", corrigir o conteudo e reenviar. Se "Pending", aguardar (geralmente 24-48h).
⏱
Access Token Meta tem validade — verificar expiracao
Tokens do tipo System User tem duracao configurada (60 ou 90 dias por padrao). Verifique em Facebook Business Manager > Business Settings > System Users. Configure um token permanente (Never expire) para producao. Se o token expirar, todos os disparos falharao com erro HTTP 190.
🕐
Janela de 24h — texto livre so funciona apos contato enviar mensagem primeiro
O comando send-text so funciona dentro da janela de 24h apos o destinatario ter enviado alguma mensagem. Para primeiro contato / outbound, use SEMPRE send-template com template de status "approved".

--- skill: status-mensagem grupo: mdos-whatsapp descricao: Verifica status de entrega de uma mensagem enviada pelo wamid tool: node _tools/mdos-whatsapp.mjs --- # Skill: Verificar Status de Mensagem ## Quando usar Quando o usuario quiser saber se uma mensagem enviada foi entregue ou lida, ou verificar o wamid de uma mensagem recente. ## Uso ``` node _tools/mdos-whatsapp.mjs get-status <wamid> ``` Onde <wamid> e o ID retornado pela Meta API no momento do envio (formato: wamid.XXXXXXXX...) ## Saida esperada ``` ── Status da mensagem ────────────────────────────── wamid: wamid_xxxxx status: sent | delivered | read | failed tipo: template | text direcao: saida destinatario: 5551999999999 created_at: 2026-04-23T10:00:00Z ``` ## Status possiveis - `sent` — Meta recebeu, aguardando entrega ao dispositivo - `delivered` — entregue ao dispositivo do destinatario - `read` — lido pelo destinatario - `failed` — falha na entrega (verificar erro no MdFlow) ## Observacoes - Status e atualizado via webhook da Meta (Edge Function mdzap-webhook) - Se wamid nao encontrado localmente, verificar em MdFlow /mdzap/sdr_acordos - O status no Supabase pode ter pequeno delay em relacao ao status real da Meta
