2. Fluxo Completo de Atendimento (6 Fases)

FASE 1

📱 Entrada do Lead

  • Lead clica anuncio Facebook Ads → WhatsApp abre com msg pre-preenchida (nome + CPF)
  • Julia recebe → cadastra contato automaticamente via inbound-upsert-lead
  • Julia envia boas-vindas:
    • Apresenta "Midas Restricoes" — empresa especialista em restricoes indevidas de credito
    • Explica: auditoria de credito gratuita, descobre brechas nas restricoes
    • Reforça: 100% gratuito
    • Pergunta: "tem interesse em descobrir sua situacao no CPF?"
    • Pede: documento de identificacao (CNH ou RG) + comprovante de residencia no nome do lead
↓
FASE 2

📄 Coleta de Documentos

  • Lead envia os 2 docs (geralmente em sequencia)
  • Doc Reader verifica legibilidade
  • Doc Reader extrai dados:
    CNH/RG: nome, CPF, RG, data_nascimento, sexo Comprovante: nome_titular, logradouro, numero, bairro, cidade, UF, CEP
  • Se comprovante NAO esta no nome do lead:
    • Julia pede comprovante no proprio nome
    • Julia sugere emitir boleto via banco digital
    • Julia NUNCA sugere extrato bancario
    • Se nao consegue: Julia pede declaracao de terceiro (modelo especifico + doc do terceiro obrigatorio)
↓
FASE 3

🔍 Pre-Cadastro e Auditoria

  • Doc Reader popula pending_lead_data (metadata JSONB da conversa)
  • Julia manda: "ja estamos fazendo seu cadastro"
  • Atendente humano assume → faz auditoria Boa Vista
  • Nome limpo → Julia parabeniza, encerra
  • Nome sujo → Julia informa SOMENTE empresas (NUNCA valores). Ex: "39 restricoes da RGE"
  • Prazo comunicavel: "90 dias para primeiro resultado" — NUNCA dar garantia
↓
FASE 4

📋 CTPS (Carteira de Trabalho)

  • Julia pede CTPS para comprovar AJG (Assistencia Juridica Gratuita)
  • Tetos por estado:
    RS: 5 salarios minimos SC: 3 salarios minimos SP: 3 salarios minimos
  • Doc Reader extrai: cargo + data_admissao (se contrato aberto)
  • Se contrato fechado → Julia pergunta se trabalha → autonomo/desempregado
  • Julia cadastra profissao nos casos sem contrato aberto
↓
FASE 5

💰 Renda Receita (Humano)

  • Humano faz renda receita (declaracoes IR) — NAO e a Julia
  • Julia tem contexto para poder cobrar se faltar
  • Julia nao executa esta etapa — apenas cobra o humano se necessario
↓
FASE 6

✍️ Procuracao (ZapSign)

  • Humano gera procuracao → envia link ZapSign via WhatsApp
  • Lead vai para aguardando_assinatura no CRM
  • Julia faz follow-up: 1h sem resposta → cobrar
  • Lead assina → Julia confirma encaminhamento para "escritorio de advocacia parceiro"
  • Midas agradece → fluxo encerrado
