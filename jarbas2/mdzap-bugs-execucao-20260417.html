BUG-01 — Duração de Áudio (OGG)

BUG-01 Áudio recebido mostra 0 segundos ✅ Corrigido c365fb02
Causa Raiz
OGG/Opus do WhatsApp não tem header DURATION. Browser retorna duration = Infinity ou 0. Código anterior usava setTimeout(100ms) após seek — muito curto para o browser processar o arquivo.
Fix em MdzapMsgAudio.tsx
aguarda evento canplay seek para 1e101 se duracao invalida evento seeked em vez de setTimeout reset a 0 apos seeked

Lógica: browser força download completo ao fazer seek para 1e101 → após o evento seeked a duração real está disponível no metadata → exibe corretamente e reseta playhead para 0.

BUG-02 — Áudio Gravado (Formato)

BUG-02 Áudio gravado não chega no WhatsApp ✅ Corrigido c365fb02
Causa Raiz
Chrome grava em audio/ogg;codecs=opus. Containers OGG do MediaRecorder não têm header DURATION. WhatsApp Android/iOS rejeita ou não reproduz OGG sem esse header.
Fix em MdzapAudioRecorder.tsx
// Ordem de prioridade de MIME types // audio/mp4 (AAC) é o formato nativo do WhatsApp — 100% compatível Android/iOS ['audio/mp4', 'audio/mpeg', 'audio/ogg;codecs=opus'] // Prioridade: tenta audio/mp4 primeiro — todos os browsers modernos suportam

Audio/MP4 (AAC-LC) é o codec nativo do WhatsApp em todas as plataformas. O MediaRecorder no Chrome 94+ suporta audio/mp4 — sem polyfill necessário.

BUG-03 — PDF "anonymous"

BUG-03 PDF com nome "anonymous" / sem download ✅ Corrigido c365fb02 c9a37649
Investigação
✅ access_token verificado no Supabase — AMBOS os números têm token configurado
⛔ Edge Function mdzap-webhook fazia upload SEM contentDisposition
⛔ Workflow /MDZAP — Document Received (ID: 7BvjpfcStEG2z27V) estava INATIVO
Fixes Aplicados
downloadAndStoreMeta(): contentDisposition adicionado downloadAndStorePublic(): contentDisposition adicionado MdzapMsgMedia.tsx: atributo download no link Workflow Document Received: ATIVADO
// Edge Function mdzap-webhook/index.ts // Antes: sem contentDisposition → browser usa fallback "anonymous" // Depois: contentDisposition: `attachment; filename="${safeName}"`

Sem Content-Disposition: attachment; filename="..." no upload para o Supabase Storage, o browser não tem como inferir o nome do arquivo e usa o fallback "anonymous". O fix garante que o filename seja preservado desde o download da mídia do WhatsApp até o Storage.
