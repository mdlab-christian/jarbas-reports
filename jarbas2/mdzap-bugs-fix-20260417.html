BUG-01 — Áudio recebido mostra 0 segundos

BUG-01 Áudio recebido mostra 0 segundos na duração ✅ Corrigido c365fb02
Causa raiz
OGG/Opus vindo do WhatsApp não tem o header DURATION no container. O browser retorna audio.duration = Infinity ou 0 até baixar o arquivo inteiro. O código anterior usava setTimeout(100ms) após seek — tempo insuficiente para o browser calcular a duração real.
Fix aplicado — src/components/mdzap/MdzapMsgAudio.tsx
  • Aguarda evento canplay antes de verificar duração
  • Se duração ainda inválida (Infinity / 0) → seek para 1e101 para forçar download completo do arquivo
  • Usa evento seeked (em vez de setTimeout) para resetar posição para 0
  • Registra durationchange para capturar duração no momento exato em que aparece

BUG-02 — Áudio gravado não chega no WhatsApp

BUG-02 Áudio gravado pelo operador não reproduz no WhatsApp do cliente ✅ Corrigido c365fb02
Causa raiz
Chrome grava em audio/ogg;codecs=opus. Containers OGG criados pelo MediaRecorder não têm o header DURATION. WhatsApp Android/iOS tem dificuldade para reproduzir OGG sem esse header. O array de MIME types priorizava OGG — causando incompatibilidade sistêmica.
Fix aplicado — src/components/mdzap/MdzapAudioRecorder.tsx
  • Array reordenado para: ['audio/mp4', 'audio/mpeg', 'audio/ogg;codecs=opus']
  • audio/mp4 (AAC) é o formato nativo do WhatsApp — compatível 100% com Android e iOS
  • OGG/Opus permanece como fallback apenas quando MP4 não é suportado pelo browser

BUG-04 — Preview de mídia era modal fullscreen

BUG-04 Seleção de arquivo abria modal que bloqueava toda a tela ✅ Corrigido c365fb02
Causa raiz
MdzapMediaPreview renderizava com classes fixed inset-0 bg-black/70 z-50 — modal que bloqueava toda a tela e não seguia o padrão UX esperado. Comportamento correto: preview inline acima do campo de texto, igual ao WhatsApp Web e ChatGPT.
Fix aplicado
  • MdzapMsgInput.tsx — novo strip inline com thumbnail/ícone, nome do arquivo, botão X e barra de progresso
  • MdzapMsgThread.tsx — removido modal MdzapMediaPreview; passa pendingFile / onFileSend / onFileClear para o input
  • Textarea vira campo de legenda quando há arquivo pendente — placeholder "Legenda (opcional)..."
  • Botão enviar habilitado quando há arquivo pendente, mesmo sem texto digitado

BUG-05 — Latência alta (3-8s) no envio/recebimento

BUG-05 Mensagens chegam com atraso de 3 a 8 segundos 🔵 Aguarda Decisão CTO
Causa raiz investigada
  • Recebimento: n8n /MDZAP — Message Grouper tem Wait node de 8 segundos intencional para agrupar mensagens fragmentadas antes de enviar para Julia AI — isso é DESIGN, não bug
  • Envio: pipeline sequencial no n8n — INSERT → webhook → Meta API → PATCH status. Cada hop adiciona ~500ms–1s
  • Race condition: se operador e cliente enviam simultaneamente, ambos chegam com delay de 8s podendo colidir na resposta da Julia
Opcoes de resolucao
Opcao A — Reduzir Wait
Reduzir Wait de 8s para 3-4s. Melhora latência percebida mas pode quebrar agrupamento de mensagens fragmentadas enviadas em sequência pelo WhatsApp.
Opcao B — Wait Adaptativo (recomendado)
Resetar timer se nova mensagem chegar dentro de 2s. Mantém agrupamento correto e reduz latência para casos onde não há fragmentação. Requer alteração no workflow n8n.
Impacto: qualquer alteração no Wait node afeta diretamente o comportamento da Julia AI. Requer aprovação do CTO antes de implementar.
