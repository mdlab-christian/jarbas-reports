Bugs P0 — Criticos (bloqueiam lancamento)

BUG-01 P0 — Critico
Orchestrator inexistente — Julia nunca responde
Problema: O workflow Message Grouper chama POST /webhook/mdflow/mdzap/orchestrator e recebe 404. O endpoint simplesmente nao existe no n8n.
Impacto: A Julia NUNCA responde automaticamente. Todo o fluxo de IA conversacional esta morto. O produto funciona apenas como visualizador de WhatsApp — sem automacao alguma.
Fix necessario
Criar workflow MDZAP-ORCHESTRATOR com rota mdflow/mdzap/orchestrator. O workflow deve receber o payload agrupado do Message Grouper, classificar a intencao e despachar para Julia, SDR Docs ou SDR Acordos conforme o contexto. Estimativa: ~4h de desenvolvimento.
BUG-02 P0 — Critico
whatsapp_mensagens sem coluna conteudo
Problema: A tabela whatsapp_mensagens nao tem coluna para armazenar o texto da mensagem. Impossivel recuperar conteudo de mensagens de texto via SQL.
Impacto: Busca, auditoria e historico de mensagens de texto sao inviaveis. Sem essa coluna, qualquer feature que dependa do conteudo armazenado (busca fulltext, exportacao, analise) falha silenciosamente.
Migration necessaria (ver secao SQL)
ALTER TABLE public.whatsapp_mensagens ADD COLUMN IF NOT EXISTS conteudo TEXT, ADD COLUMN IF NOT EXISTS conteudo_limpo TEXT;
BUG-03 P0 — Critico
AI Composer quebrado em producao (process.env bloqueado)
Problema: O workflow do AI Composer usa process.env.ANTHROPIC_API_KEY no Code node. O Railway bloqueia acesso a variaveis de ambiente via N8N_BLOCK_ENV_ACCESS_IN_NODE=true. Resultado: erro 500 a cada clique no botao ✨.
Impacto secundario: Modelo configurado como claude-haiku-4-5-20251001 (desatualizado). Deve ser claude-haiku-4-5.
Fix necessario
Substituir process.env.ANTHROPIC_API_KEY pela key hardcoded no Code node — padrao ja adotado por todos os outros workflows do projeto. Atualizar o ID do modelo para claude-haiku-4-5. Estimativa: ~30 min.
BUG-04 P0 — Critico
Conflito de rota em mdzap/enviar-template
Problema: Dois workflows ativos na mesma rota: 2DxMd7m0d2GMASFg (versao antiga, API v19.0) e D0nbZUxCJXkHdvfb (versao atual). O n8n usa o primeiro match — comportamento nao deterministico.
Impacto: Envio de templates HSM pode usar API v19.0 (obsoleta) em vez da versao atual, causando falhas silenciosas ou rejections da Meta.
Fix necessario
Desativar workflow 2DxMd7m0d2GMASFg imediatamente. Manter apenas D0nbZUxCJXkHdvfb. Estimativa: 5 minutos.

Bugs P1 — Importantes (impactam qualidade)

P1-01 P1 — Importante
Audio Transcribe completamente inativo
Situacao: Existem 3 versoes do workflow de transcricao — todas inativas. A melhor versao identificada e dTthEKJc78qa4Zcy (ElevenLabs STT, com auth).
Impacto: Audios recebidos nao sao transcritos para texto. A Julia nao consegue processar mensagens de voz. Usuarios que enviam audio ficam sem resposta contextual.
Fix necessario
Ativar workflow dTthEKJc78qa4Zcy. Desativar ou arquivar as outras 2 versoes para evitar confusao futura. Estimativa: 30 min (inclui teste de ponta a ponta).
P1-02 P1 — Importante
Auth ausente em endpoints criticos
Workflows sem validacao do header mdflow:
Upsert Lead Doc Reader Julia Lab Refine Julia Prompt Save Conversation Status
Impacto: Endpoints expostos sem protecao. Qualquer requisicao externa pode criar leads, sobrescrever prompts da Julia ou alterar status de conversas sem autenticacao.
Fix necessario
Adicionar validacao do header mdflow nos 5 workflows listados, seguindo o padrao ja implementado nos outros endpoints. Estimativa: 2h.
P1-03 P1 — Importante
Policy RLS do bucket mdzap-media sem isolamento de org
Problema: A policy SELECT do bucket mdzap-media permite qualquer usuario autenticado ler arquivos de outras organizacoes. Nao ha verificacao de organizacao_id.
Impacto: Vazamento de midia entre clientes. Arquivos de uma organizacao (fotos de documentos, comprovantes, imagens pessoais) sao acessiveis por usuarios de outras orgs.
Fix SQL necessario
DROP POLICY IF EXISTS mdzap_media_read ON storage.objects; CREATE POLICY mdzap_media_read ON storage.objects FOR SELECT TO public USING ( bucket_id = 'mdzap-media' AND (storage.foldername(name))[1] = get_org_id()::text );
P1-04 P1 — Importante
Policies INSERT sem WITH CHECK (leads, acordos, acordos_propostas)
Problema: As policies de INSERT nas tabelas leads, acordos e acordos_propostas nao tem clausula WITH CHECK.
Impacto: Um usuario pode inserir registros com organizacao_id de outra organizacao, corrompendo dados multi-tenant.
Fix necessario
Adicionar WITH CHECK (organizacao_id = org_id()) nas policies INSERT das 3 tabelas afetadas. Padrao ja implementado em outras tabelas do sistema.
P1-05 P1 — Importante
Bulk send sem delay entre mensagens
Problema: Os workflows bulk-negociacoes e bulk-documentos enviam mensagens em sequencia sem nenhum delay.
Impacto: Em listas com mais de ~50 contatos, a Meta API retorna 429 (rate limit). Mensagens perdem-se silenciosamente. O escritorio nao tem visibilidade sobre quais envios falharam.
Fix necessario
Adicionar no de delay de 200-500ms entre cada iteracao do loop de envio. Configurar retry com backoff exponencial para status 429. Estimativa: 30 min.
P1-06 P1 — Importante
Workflow Encerrar Conversa inativo sem motivo
Problema: O workflow 6ht8s7FVDKEnLLPz (Encerrar Conversa) esta com logica implementada e funcional, mas desativado.
Impacto: Usuarios nao conseguem encerrar conversas via interface. Conversas ficam presas em status ativo indefinidamente.
Fix necessario
Ativar workflow 6ht8s7FVDKEnLLPz no painel n8n. Estimativa: 5 minutos.
