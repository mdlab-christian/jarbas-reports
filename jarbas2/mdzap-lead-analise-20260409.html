Seção 2 — 3 Correções a Executar

Fix 1 · P1 Crítico
criarLeadEAuditarInline retorna sucesso + leadId
Problema: Após clicar "Auditoria + Cadastro", conversa.lead_id ainda é null no React state até o refetch da lista completar (Realtime demora 500ms–2s). A aba Restrições recebe leadId={conversa.lead_id} = null e pisca o empty state "Cadastre o lead".
Arquivos afetados
useLeadMutations.ts MdzapLeadPanel.tsx
Mudanças necessárias
  • criarLeadEAuditarInline deve retornar { success: boolean; leadId?: string } em vez de void
  • MdzapLeadPanel adiciona state const [localLeadId, setLocalLeadId] = useState<string | null>(null)
  • Passa localLeadId ?? conversa.lead_id para LeadRestricoesMdzapTab e MdzapLeadCadastroDrawer
Callback corrigido
const result = await criarLeadEAuditarInline(); if (result?.leadId) setLocalLeadId(result.leadId); if (result?.success) setActiveTab('restricoes');
Fix 2 · P1 Crítico
setActiveTab condicional — incorporado no Fix 1
Problema: await criarLeadEAuditarInline(); setActiveTab('restricoes') é chamado mesmo se o pipeline falhou — aba muda mas mostra empty state errado.
Resolvido automaticamente pelo Fix 1 — ao verificar result?.success antes de chamar setActiveTab, o bug desaparece sem nenhuma alteração adicional.
Fix 3 · P1 Novo (Lovable)
Capturar auditoria_id do RPC
Problema: rpc_disparar_auditoria retorna { success, auditoria_id, message } mas useLeadMutations.ts só lê { success, message } — auditoria_id é perdido. O CrmAuditoriaPanel precisa fazer refetch extra para encontrar a auditoria, causando delay desnecessário.
Arquivo afetado
useLeadMutations.ts
Mudança — antes × depois
// ANTES: const data = result as { success?: boolean; message?: string }; // DEPOIS: const data = result as { success?: boolean; message?: string; auditoria_id?: string }; if (data.auditoria_id) { // retornar junto com leadId para o caller }
Retorno de criarLeadEAuditarInline
return { success: true, leadId, auditoriaId };
