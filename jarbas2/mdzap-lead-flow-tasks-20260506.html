Matching automatico server-side com pg_trgm. Filtra cartorios/protestos, cria item Boa Vista consolidado para bureaus ApiFull, define requer_revisao por score de similaridade.
Regra Absoluta Credores com CARTORIO, TABELIONATO, PROTESTO, TABELI ou OFICIAL DE REGISTRO devem ser ignorados — nao cadastrar, nao somar, nao incluir no Boa Vista.
C-Prompt para Claude Code
Execute a T17 — RPC fn_mdzap_auto_cadastrar_restricoes. REGRA ABSOLUTA: qualquer restrição cujo credor contenha CARTORIO, TABELIONATO, PROTESTO, TABELI, OFICIAL DE REGISTRO deve ser IGNORADA — não cadastrar, não somar. Parâmetros: p_auditoria_id UUID, p_lead_id UUID, p_organizacao_id UUID Lógica: 1. Lê resultado_json->>'restricoes' da tabela auditorias WHERE id = p_auditoria_id 2. Para cada item do array: a. Se credor contém (case-insensitive): CARTORIO, TABELIONATO, PROTESTO, TABELI, OFICIAL → PULAR b. Normaliza credor: unaccent(lower(credor)), remove não-alfanumérico c. Busca melhor empresa via: SELECT id, nome, similarity(unaccent(lower(nome)), credor_norm) AS sc FROM empresas WHERE organizacao_id = p_organizacao_id AND deleted_at IS NULL ORDER BY sc DESC LIMIT 1 d. Também verificar nome_variacoes[] com match exato normalizado (score 1.0 se match) e. sc >= 0.75 → empresa_id definido, requer_revisao = false f. sc 0.45-0.74 → empresa_id definido, requer_revisao = true g. sc < 0.45 → empresa_id = null, requer_revisao = true h. NUNCA criar empresa nova 3. Se bureau IN ('apifull_115','apifull'): gerar 1 item extra "Boa Vista" com soma dos valores (excluindo cartórios), processavel=true, buscar empresa com nome ILIKE '%boa vista%' OR '%scpc%' 4. Chamar fn_restricao_salvar() ou INSERT direto em restricoes por item 5. UPDATE leads SET restricoes_revisadas = false, etapa = 'atendimento' WHERE id = p_lead_id 6. UPDATE auditorias SET lead_id = p_lead_id WHERE id = p_auditoria_id Retorna: total_criadas INT, pendentes_revisao INT, ignoradas_cartorio INT Usar apply_migration.
