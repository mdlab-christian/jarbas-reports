📋 Secao 1 — Visao Geral
O que e o MdZap
  • MdZap = modulo WhatsApp do MdFlow para SDR automatizado via agentes IA
  • 4 agentes SDR planejados, foco desta spec: Julia Leads 100%
  • Midas Restricoes = empresa especialista em restricoes indevidas de credito (Boa Vista, Serasa, SPC)
  • Pattern: Hub + Routes — /mdzap = Hub geral, /mdzap/leads = Julia (ja implementado)
  • Julia atende leads vindos de Facebook Ads via WhatsApp, coleta documentos, faz auditoria, cadastra no CRM
  • Fluxo completo: Ads → WhatsApp → Julia → Doc Reader → Auditoria → Cadastro → CTPS → Procuracao
Regras de confidencialidade
  • NUNCA citar nome do escritorio de advocacia parceiro em mensagens ao lead
  • NUNCA citar nome de advogado em mensagens ao lead
  • Referir-se apenas como "Midas Restricoes" e "escritorio parceiro especializado"
🔄 Secao 2 — Fluxo de Atendimento Julia (6 Fases)
1 Entrada do Lead (Facebook Ads)
  1. Lead clica anuncio Facebook → abre WhatsApp com msg pre-preenchida (nome + CPF)
  2. Julia recebe mensagem → cadastra contato automaticamente no sistema
  3. Julia envia boas-vindas: apresenta "Midas Restricoes", explica auditoria gratuita, pergunta interesse
  4. Julia pede: documento de identificacao (CNH ou RG) + comprovante de residencia
2 Coleta de Documentos
  1. Doc Identificacao: Doc Reader verifica legibilidade → extrai nome, CPF, RG, data_nascimento, sexo
  2. Comprovante Residencia: Doc Reader valida nome e extrai endereco
Branch: Nome no comprovante NAO bate
  1. Julia pergunta se lead tem outro comprovante em nome proprio
  2. Sugere: boleto de banco digital (Nubank, Inter, etc.)
  3. Se nao consegue → pede declaracao de terceiro:
"Eu, [nome terceiro], CPF [CPF terceiro], declaro que [nome lead], CPF [CPF lead], reside comigo no endereco [endereco completo]."
OBRIGATORIO: documento de identificacao do terceiro junto com a declaracao
ATENCAO Doc Reader: validar ENDERECO DO LEAD (nao o da empresa/Vivo do comprovante)
Julia envia docs apenas DEPOIS que lead manda os 2 documentos (geralmente manda na sequencia)
3 Auditoria + Cadastro
  1. Julia: "ja estamos fazendo seu cadastro e analisando sua situacao"
  2. Atendente humano assume → faz auditoria Boa Vista (bureau apifull_115)
  3. Pipeline: rpc_disparar_auditoria → n8n webhook → ApiFull API → resultado
  4. PDF auditoria abre em nova guia para analise visual
Branch: Resultado da Auditoria
Nome LIMPO
Julia parabeniza o lead → encerra fluxo
Nome SUJO
Julia informa SOMENTE empresas (NUNCA valores)
Ex: "Encontramos 39 restricoes da RGE"
  1. Cadastro automatico: dados pessoais (ja extraidos) + endereco + restricoes
  2. Lead move para status em_atendimento no CRM
4 CTPS (Carteira de Trabalho)
  1. Julia pede CTPS para comprovar elegibilidade AJG (Assistencia Judiciaria Gratuita)
  2. Tetos salariais por estado:
RS = 5 salarios minimos SC = 3 salarios minimos SP = 3 salarios minimos
Branch: Status CTPS
Contrato ABERTO
profissao = cargo do contrato
Contrato FECHADO
Julia pergunta se trabalha atualmente
Se nao → profissao = "Desempregado"
REGRA: Lead NAO pode sair de em_atendimento sem campo profissao preenchido
5 Renda Receita (Humano)
  1. Humano faz renda receita (declaracoes IR) — NAO e Julia
  2. Julia tem contexto para cobrar se faltar documentacao complementar
  3. Pendencia: como notificar Julia quando humano completa renda receita?
6 Procuracao (ZapSign)
  1. Humano gera procuracao → envia link ZapSign para assinatura digital
  2. Lead move para status aguardando_assinatura
  3. Julia faz follow-up: 1h sem resposta → lembrete automatico
  4. Lead assina → Julia confirma encaminhamento para escritorio parceiro
  5. Midas agradece e encerra atendimento

🔬 Secao 5 — CRM Audit: Como Funciona (para replicar no MdZap)
Pipeline Completo da Auditoria
1 Frontend: Botao "Consultar CPF Completo"
useCrmDispararAuditoria({ cpf, lead_id, organizacao_id })
2 RPC: rpc_disparar_auditoria (SECURITY DEFINER)
  1. Valida CPF (formato + nao vazio)
  2. Cria registro na tabela auditorias com status='pendente'
  3. Enfileira chamada webhook n8n
3 n8n Webhook: ApiFull API
  1. Recebe request com CPF + auditoria_id
  2. Chama ApiFull bureau apifull_115 (Boa Vista)
  3. Atualiza tabela auditorias com resultado + PDF
  4. Status: 'concluida' ou 'erro'
4 Frontend: Realtime + Polling
  1. Subscription Realtime para mudanca de status
  2. Fallback: polling a cada 5 segundos
  3. 4 estados visuais: idle | processing | sem restricoes | com restricoes
5 CadastrarRestricoesDrawer
  1. Fuzzy matching de empresas (nome_variacoes auto-learning)
  2. Regra de agrupamento Boa Vista
  3. Save por item via fn_restricao_salvar
RPCs Envolvidas
  • rpc_disparar_auditoria — SECURITY DEFINER
  • fn_restricao_salvar — save individual
  • rpc_crm_aprovar_triagem — mover status
Tabelas
  • auditorias — registro + resultado
  • restricoes — itens individuais
  • empresas — nome_variacoes (auto-learn)
