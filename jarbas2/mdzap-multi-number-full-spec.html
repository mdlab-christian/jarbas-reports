1. Contexto e Motivacao

Situacao Atual

2 numeros ativos (Midas Leads + Christian Acordos)

Rotas, cards de hub e agentes sao hardcoded em 5+ arquivos separados. Cada numero novo exige 6 mudancas em codigo + deploy completo.

⚠️
Problema criticoSe primaryNumero ficar offline, TODOS os envios do sistema param — mesmo em chats de outros numeros conectados.

Cada numero novo vira um INSERT em whatsapp_numeros + mdzap_ai_modules. Hub e rotas se atualizam automaticamente.

2. Arquitetura Multi-Numero

ENTRADA (Inbound) WhatsApp ──► Meta Cloud API ──► Edge Function (mdzap-webhook) │ ├─ match wa_instance_id ──► whatsapp_numeros row ├─ busca/cria conversa com numero_id ├─ INSERT whatsapp_mensagens └─ POST n8n Orchestrator (numero_id + agente_slug) │ ├─ busca mdzap_ai_modules por slug ├─ chama Claude com system_prompt do agente └─ envia resposta via Meta API usando wa_instance_id SAIDA (Outbound) — Frontend /mdzap ──► Hub (cards dinamicos de whatsapp_numeros + mdzap_ai_modules) /mdzap/:agentSlug ──► MdzapAgentPage (pagina generica) │ ├─ MdzapChatPanel(defaultSdrMode=agentSlug) ├─ useMdzapConversas(agente_slug) ──► fn_mdzap_conversas RPC ├─ Lista filtrada por agente_slug └─ Envio: conversa.numero_id ──► webhook ──► n8n FE ──► Meta API
💡
Principio arquiteturalO numero_id fica persistido em cada conversa no banco. Assim o frontend nunca precisa saber "qual numero esta conectado agora" — ele simplesmente usa o numero da conversa.

P2 4.2 Edge Function — .maybeSingle() fix Pendente

Na linha 386 de mdzap-webhook/index.ts, a query usa .maybeSingle() em uma clausula OR. Se 2 numeros derem match (cenario de edge case), o Supabase lanca erro 406.

Fix
// ANTES .maybeSingle() // DEPOIS .limit(1).maybeSingle()

1 linha de mudanca. Evita erro silencioso em producao.

6. Fix dos Baloes de Mensagem

P0 — BUG
🐛
Problema: Baloes sendo cortados Texto truncado, timestamps escondidos, emoji grandes clipados, descenders de letras cortados (g, p, q, y). Causa: 2 overflow-hidden desnecessarios em componentes aninhados.
Fix 1 — MdzapMsgBubble.tsx (linha 166)
Antes
'max-w-[min(75%,480px)] overflow-hidden break-words rounded-2xl ...'
Depois
'max-w-[min(75%,480px)] break-words rounded-2xl ...'

Por que: overflow-hidden foi adicionado para o border-radius, mas e desnecessario — rounded-2xl ja clipeia a pintura sozinho. O overflow recortava timestamps e descenders.

Fix 2 — MdzapMsgText.tsx (linha 11)
Antes
<p className="break-words overflow-hidden text-sm leading-relaxed">
Depois
<p className="break-words text-sm leading-relaxed">

Por que: Redundante com o container pai. Corta descenders de letras (g, p, q, y) e emoji grandes. break-words ja garante que texto nao vaze horizontalmente.
