Dúvidas Abertas — Perguntas para Gustavo

D1 — Integração Petição Inicial + OlivIA
Quando a OlivIA gerar a petição inicial, como ela chegará ao E7?
O carregarDocsMia() busca o último lote concluído em mia_geracao_docs_lotes. A petição precisa estar lá como novo tipo (ex: peticao_inicial)? Ou vem de outra tabela/fonte (ex: documentos_pendentes, Storage com path conhecido)?
D2 — E8 Pós-Confirmação em Produção
Com confirmar_ajuizamento: true, o EPROC abre modal antes de finalizar ou navega direto?
Precisamos saber se há interação extra após o click em confirmacaoParaFinalizarProcesso() para capturar o número CNJ atribuído. O pipeline precisa tratar esse passo antes de ir para produção real.
D3 — Resultado Esperado do E8 em Produção
Após a confirmação real, o EPROC exibe o número CNJ na tela? Em qual elemento DOM?
O worker precisa capturar esse número para gravar em distribuicao_lotes.numero_cnj_distribuido (ou campo equivalente). Sem esse dado, não conseguimos confirmar que o processo foi distribuído com sucesso.
D4 — TJSC: Diferenças Conhecidas vs TJRS
Existe alguma diferença conhecida no fluxo TJSC que o stub atual não cobre?
Ex: seleção de perfil diferente, competências diferentes, timing mais lento? O stub funciona para TJRS, mas pode falhar silenciosamente no TJSC se houver diferenças de DOM ainda não mapeadas.
D5 — TJSP: Competências e Assuntos
Para negativação indevida no TJSP, quais os valores corretos de competência e assunto?
São os mesmos do TJRS ("Cível - Geral" / "Indenização por Dano Moral") ou diferem? O adapter TJSP tem discoverCompetenciaAssuntoTJSP que tenta descobrir na primeira execução — mas seria melhor ter os valores corretos de antemão.
D6 — E3 Competência via Fallback
A seleção "Cível - Geral" via fallback no TJRS está correta para negativação indevida?
O E3 está selecionando via fallback: true — pode ser que a competência correta não esteja sendo selecionada de forma determinística. Se houver competência mais específica aplicável, o pipeline deveria selecioná-la diretamente.
