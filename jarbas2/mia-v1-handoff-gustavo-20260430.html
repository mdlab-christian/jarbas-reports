Como funciona o ciclo de execucao

A Mia fica em loop de 10 segundos fazendo poll no Supabase com SKIP LOCKED. Quando encontra um lote com status = 'fila', despacha para o adapter correto (TJRS/TJSC/TJSP), executa o pipeline E0 ate E8 e atualiza o status. Completamente autonomo — nenhuma intervencao humana necessaria no fluxo feliz.

Bugs corrigidos em 2026-04-30 durante smoke run
#BugCausaCorrecao aplicada

2 E6 marcava checkbox de doenca grave Text-matching generico batia em #chkDoencaGrave IDs exatos para cada checkbox obrigatorio
3 E6 para E7 causava navegacao prematura Codigo clicava #btnSalvar antes de E7 finalizar uploads Navegacao removida — E6 e E7 compartilham a mesma pagina
4 E8 nao detectava o botao correto Seletor #btnSalvar generico insuficiente Seletor com atributo onclick especifico adicionado

Quick Start — Ciclo de Deploy

1
Editar o arquivo no SSOT compartilhado
Sempre editar em mdlab-vault, nunca diretamente em /Users/mia/code/. Isso garante rastreabilidade e historico de alteracoes.
vim ~/Documents/mdlab-vault/02_projetos/mia/worker/adapters/distribuir-tjrs.mjs
2
Copiar para o ambiente da Mia (requer sudo)
O usuario mia tem permissao restrita. O deploy e feito com sudo cp + chown para garantir a propriedade correta.
sudo cp /tmp/distribuir-tjrs.mjs \ /Users/mia/code/worker/adapters/distribuir-tjrs.mjs sudo chown mia:staff \ /Users/mia/code/worker/adapters/distribuir-tjrs.mjs
3
Reiniciar o worker
O flag -k forca o kill do processo atual antes de reiniciar. Aguardar alguns segundos para o Node.js inicializar.
sudo launchctl asuser 508 launchctl kickstart \ -k gui/508/com.mia.worker
4
Smoke test via SQL — SEMPRE com confirmar_ajuizamento: false
Insere um lote de teste na fila. O worker vai pegar, executar E0 a E8 e parar antes de confirmar. O campo confirmar_ajuizamento=false e obrigatorio em testes.
INSERT INTO distribuicao_lotes ( processo_id, oab, tribunal, status, meta ) VALUES ( 'uuid-do-processo-de-teste', 'RS116571', 'TJRS', 'fila', '{"confirmar_ajuizamento": false}' -- NUNCA usar true em testes );
5
Monitorar logs em tempo real
Acompanhar execucao. Duracao esperada para E0 a E8: aproximadamente 90 a 100 segundos. Verificar ausencia de erros e confirmar que parou no E8.
sudo tail -f /Users/mia/logs/mia-worker.out.log # Via SSH no Mac Mini (de outra maquina) ssh mac-mini-tail \ "sudo tail -f /Users/mia/logs/mia-worker.out.log"
6
Documentar alteracoes em seletores DOM
Se qualquer seletor CSS ou ID foi alterado durante o desenvolvimento, atualizar obrigatoriamente antes do merge.
vim ~/Documents/mdlab-vault/02_projetos/mia/MAPA-CAMPOS-EPROC.md

Alertas Criticos

🔴
NUNCA usar confirmar_ajuizamento: true em testes

Confirmar o ajuizamento no EPROC e uma acao IRREVERSIVEL. Uma vez enviado, o processo entra no sistema judicial e nao pode ser cancelado. Em TODOS os testes e smoke runs, usar sempre "confirmar_ajuizamento": false no campo meta do lote. So liberar como true em producao, com autorizacao explicita do Christian Paz.

🔴
Regex de Justica Gratuita deve ter ancoras

O regex para o seletor de Justica Gratuita DEVE ser /^requerida$/i com as ancoras ^ e $. Sem elas, o regex casa com "Nao Requerida" e marca o checkbox errado no EPROC, causando distribuicao com dados incorretos. Bug validado e corrigido em 2026-04-30.

⚠️
SSOT primario fica no vault do Christian — nao alterar sem documentar

O vault primario e ~/jarbas-vault/02_projetos/mia/v1/ (vault Christian, Obsidian Sync). O vault compartilhado ~/Documents/mdlab-vault/02_projetos/mia/ e o ponto de colaboracao do Gustavo. Toda decisao arquitetural deve ser registrada antes de implementar. Nao tomar decisoes tecnicas sem documentar.

⚠️
Alteracoes em seletores DOM exigem atualizacao do mapa

Qualquer mudanca em seletores CSS ou IDs do EPROC deve ser documentada em MAPA-CAMPOS-EPROC.md ANTES do merge. O EPROC pode modificar IDs entre deployments sem aviso. Alteracao sem documentacao = bug silencioso em producao.

ℹ️
TJSP usa EPROC — nao eSAJ

O tribunal de Sao Paulo na Mia usa o sistema EPROC (igual RS e SC), NAO o sistema eSAJ. O eSAJ e usado apenas pela campanha de acordos SP (OAB SP535101 campanha). Os dois sistemas tem seletores DOM completamente diferentes. Nao confundir ao desenvolver o adapter TJSP.
