Step 1 Verificar se Node ja instalou
O brew install node foi iniciado antes. Pode ter terminado ou dado erro por causa de um restart. Vamos verificar.
node --version
Se mostrar uma versao (tipo v25.9.0) = Node ja instalou. Pode pular pro Step 3.
Se der "command not found" = precisa instalar. Vai pro Step 2.
✓
Step 2 Instalar Node.js (se necessario)
Atencao: No macOS 12 (Monterey), o Homebrew compila do source. Pode demorar 30 a 60 minutos. NAO fechar o terminal.
brew install node
Se der erro de lock (tipo "Another active Homebrew process"):
rm -rf /usr/local/var/homebrew/locks/* brew install node
✓
Step 3 Instalar Claude Code
Este e o comando principal. Instala o Claude Code globalmente.
npm install -g @anthropic-ai/claude-code
Se der "permission denied" ou "EACCES", rodar com sudo:
sudo npm install -g @anthropic-ai/claude-code
✓
Step 4 Verificar instalacao
claude --version
Deve mostrar um numero de versao (tipo 1.x.x). Se mostrou, Claude Code esta instalado com sucesso.
✓
Step 5 Criar alias (facilitar pro Joao)
Cria um atalho cc para o Joao nao precisar digitar "claude" inteiro.
echo '' >> ~/.zshrc echo '# Claude Code alias' >> ~/.zshrc echo 'alias cc="claude"' >> ~/.zshrc source ~/.zshrc
✓
Step 6 Verificar SSH (ja habilitado)
O SSH ja foi habilitado antes. So verificar que continua ativo.
sudo systemsetup -getremotelogin
Deve retornar "Remote Login: On". Se mostrar "Off", rodar:
sudo systemsetup -setremotelogin on

3. Executar o .msi: Next → Next → Next → Install
4. Fechar e reabrir o PowerShell depois de instalar
✓

Fechar e reabrir o PowerShell antes de rodar isso (senao nao vai encontrar).
node --version npm --version
Ambos devem mostrar um numero de versao. Se der erro, o PowerShell precisa ser reaberto.
✓
Step 3 Instalar Claude Code

✓
Step 4 Verificar Claude Code
claude --version
Deve mostrar um numero de versao. Se der "nao reconhecido", fechar e reabrir o PowerShell.
✓
Step 5 Habilitar SSH no Windows
Primeiro, verificar se o OpenSSH Server ja esta instalado:
Get-WindowsCapability -Online | Where-Object Name -like 'OpenSSH.Server*'
Se mostrar "NotPresent", instalar:
Add-WindowsCapability -Online -Name OpenSSH.Server~~~~0.0.1.0
Iniciar o servico SSH e configurar para iniciar automaticamente:
Start-Service sshd Set-Service -Name sshd -StartupType Automatic
Verificar que esta rodando:
Get-Service sshd
Deve mostrar Status = Running.
Liberar no firewall (se necessario):
New-NetFirewallRule -Name "OpenSSH-Server" -DisplayName "OpenSSH Server (sshd)" -Enabled True -Direction Inbound -Protocol TCP -Action Allow -LocalPort 22

Troubleshooting

▶ 🍎 Problemas no Mac do Joao
  • "command not found: brew"
    O Homebrew precisa dos Next Steps. Rodar:
    echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> ~/.zprofile eval "$(/opt/homebrew/bin/brew shellenv)"
  • "command not found: node"
    Node ainda nao instalou. Rodar brew install node e esperar compilar (30-60 min no macOS 12).
  • "brew install trava / demora muito"
    Normal no macOS 12 (compila do source). Se travar de verdade, Ctrl+C e rodar brew install node de novo.
  • "permission denied" no npm install
    Rodar com sudo:
    sudo npm install -g @anthropic-ai/claude-code
  • "Another active Homebrew process" / lock error
    rm -rf /usr/local/var/homebrew/locks/* brew install node
  • "claude" nao reconhecido apos npm install
    Fechar o terminal inteiro e abrir de novo. Depois tentar claude --version.
▶ 💻 Problemas no Windows do Leonardo
  • "winget nao reconhecido"
    Usar a Opcao B: baixar o instalador direto do site nodejs.org.
  • "npm nao reconhecido" / "node nao reconhecido"
    Fechar e reabrir o PowerShell. O Windows precisa recarregar o PATH depois de instalar.
  • "Acesso negado" / "Access denied"
    O PowerShell precisa estar como Administrador. Fechar e abrir de novo clicando com botao direito → "Executar como administrador".
  • "sshd nao encontrado" / "service not found"
    O OpenSSH Server nao esta instalado. Rodar:
    Add-WindowsCapability -Online -Name OpenSSH.Server~~~~0.0.1.0
  • "firewall bloqueando conexao SSH"
    New-NetFirewallRule -Name "OpenSSH-Server" -DisplayName "OpenSSH Server (sshd)" -Enabled True -Direction Inbound -Protocol TCP -Action Allow -LocalPort 22
  • "claude" nao reconhecido apos npm install
    Fechar e reabrir o PowerShell. Se ainda nao funcionar, tentar abrir um novo CMD (nao PowerShell) e rodar claude --version.
▶ 🚨 Se NADA funcionar
  • 1. Tirar um screenshot da tela com o erro
  • 2. Mandar pro Christian no WhatsApp com a descricao: "no computador do [Joao/Leonardo], no Step [X], deu esse erro"
  • 3. Nao tentar resolver sozinho se for algo diferente do troubleshooting acima
