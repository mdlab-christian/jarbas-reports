O que falta — Gap Analysis

Gap 1 — OC não escreve no vault. Quando você planeja algo comigo aqui no Telegram, eu leio mas não salvo automaticamente no vault. O CC não fica sabendo do que planejamos.
Gap 2 — Sem formato padrão de "sessão de planejamento". Falta um protocolo claro de como transformar uma conversa de planejamento aqui em tasks concretas no vault que o CC vai ler.
Gap 3 — Status de projeto no vault não tem granularidade de task. PROJETOS.md tem resumo por wave, mas não expõe o estado task-a-task que o OC precisa para discutir planejamento cirúrgico.
Gap 4 (nice-to-have) — OC não sabe o que CC fez na última sessão. Quando você executa no Pro e o CC atualiza o vault, eu não recebo notificação. Isso pode ser resolvido com um cron de leitura pós-sessão CC.

Implementação — O que precisamos fazer

🔴 P0 — OC escreve no vault
Habilitar permissão de escrita para que eu (OpenClaw) escreva arquivos em ~/Documents/jarbas-vault/. Já tenho o symlink ~/jarbas-vault. Preciso de autorização explícita sua para escrever.

Resultado: ao final de cada sessão de planejamento, eu salvo automaticamente o output no vault — sem você precisar fazer nada.
🟡 P1 — Protocolo de sessão de planejamento
Criar um formato padrão de arquivo de sessão no vault:

05_conversas/sessao-YYYYMMDD.md

Conteúdo: decisões tomadas, tasks novas ou atualizadas, próximas etapas, contexto da conversa. Formato que o CC entenda sem precisar ler o histórico do Telegram.
🟡 P1 — Formato de /projeto no vault
Padronizar como os projetos são representados no vault para que OC e CC usem o mesmo formato. Atualmente o CC usa .projeto/<nome>/tasks.json no repositório. Precisamos definir: o vault armazena o que? — spec, decisões, status por wave/task, ou tudo?
🟢 P2 — Cron de sync pós-CC
Cron que roda quando o vault é atualizado pelo CC (poll simples no arquivo PROJETOS.md). Se detectar mudança → me notifica com resumo do que foi executado. Fecha o loop: CC executou → OC sabe → você pode pedir review.
🟢 P2 — Skill /planejamento no CC
Criar uma skill /planejamento no Claude Code que gera o arquivo de sessão de planejamento no formato OC. Quando você rodar /planejamento olivia no CC, ele lê o spec, gera uma agenda de discussão e cria o arquivo em 05_conversas/ para você trazer pra cá.
⚡ Quick win — Já funciona hoje
Você já pode usar este grupo para planejamento. Eu já leio o vault. O que falta é só a escrita automática. Por enquanto: me peça "salva isso no vault" ao final de cada sessão e eu escrevo o arquivo de sessão manualmente.
