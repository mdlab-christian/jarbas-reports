OlivIA Chat — Redesign Blueprint

Análise completa da aba Chat OlivIA + proposta de reconstrução como interface conversacional full-power: upload de docs, geração de petições, download — tudo via chat.

27 Mar 2026 MdFlow / página Olivia Christian Paz → Jarbas 2.0 Stack: React + OlivIA Server v4
🔍 1. Diagnóstico — Estado Atual
O que existe hoje e por que não basta

⚠️ Chat Atual (OliviaChatPage)

  • Chat livre ou vinculado a processo (SearchBoxCNJ)
  • Manda mensagem → webhook olivia/chat → Supabase Realtime para resposta
  • Sem upload de documentos
  • Sem download de arquivos gerados
  • Sem geração de petições via chat
  • Sem análise de provas
  • Sem action cards / feedback estruturado
  • Sugestões rápidas hardcoded ("Resumir processo", etc.)
  • Sem integração com skills da OlivIA

✅ Chat Redesenhado (OlivIAChatV2)

  • Sempre inicia com Step 1: seleção de processo CNJ obrigatória
  • Upload de docs inline (provas, contratos, laudos)
  • OlivIA detecta intenção → rota para skill correta automaticamente
  • Gera petição via chat com progresso em tempo real
  • Download inline: petição .docx, relatório .pdf, laudo
  • Action cards estruturados para escolhas (tipo petição, modelo, etc.)
  • Funciona como fallback completo quando gerador está quebrado
  • Histórico de threads com contexto de processo persistido
  • Gravação de áudio direto no chat (já tem hook useAudioRecorder)
💡 Insight chave: O gerador de petições atual exige 5-6 steps sequenciais. O chat redesenhado deve colapsar esses steps em conversa natural, com a OlivIA conduzindo o fluxo via perguntas e action cards quando precisar de informação.

🚀 4. Proposta de Redesign — OlivIA Chat v2
Arquitetura conversacional completa
🗺️

Fluxo Conversacional Ideal

1

Start: Seleção de Processo (obrigatória)

Ao criar novo chat, usuário informa o número CNJ. OlivIA puxa dados do processo do Supabase e confirma: "Processo 0012345-67.2024.8.21.0001 — Banco Bradesco vs. [Cliente]. Carregado ✓ O que você quer fazer?"

= Step 1 do gerador SearchBoxCNJ reutilizável
2

Intenção: OlivIA detecta e roteia

Usuário digita ou fala. OlivIA classifica a intenção e chama a skill certa. Ex: "analisa as provas" → skill_olivia_analise; "gera a réplica" → fluxo completo de geração com triagem inline.

Intent detection no agente v4 Action cards para ambiguidades
3

Upload de Documentos

Botão de clip inline no composer. Suporta PDF, DOCX, imagens. Upload → Supabase Storage ou diretamente para o Mac Mini via endpoint dedicado. OlivIA recebe path + roda OCR/classifica automaticamente.

Novo: POST /olivia/chat/upload Multipart form-data
4

Triagem Conversacional (quando necessário)

Em vez de form com 10 campos, OlivIA faz as perguntas de triagem uma a uma via chat. Usuário responde em linguagem natural ou via action cards (sim/não, múltipla escolha).

= Steps 2-3 do gerador PerguntaDinamica reutilizável
5

Geração com Progresso em Tempo Real

OlivIA gera a petição seção por seção, transmitindo progresso via Realtime: "📝 Gerando seção Dos Fatos... (3/8)". Score automático + revisor se <7. Sem bloquear a UI.

= Steps 4-5 do gerador Streaming via messages parciais
6

Download Inline

Ao concluir, OlivIA posta action card com botões de download: [⬇ Baixar DOCX] [⬇ Baixar PDF] [📊 Ver Relatório]. Links diretos do Supabase Storage ou Google Drive.

= Step 6 do gerador action_cards[] na mensagem
⚙️ 5. Arquitetura Técnica — O Que Mudar
Backend, frontend e integração
🔧

Backend — Mudanças Necessárias

  • Novo endpoint: POST /olivia/chat/upload — Aceita multipart/form-data com o arquivo + thread_id + processo_id. Salva no Supabase Storage ou Mac Mini. Retorna URL/path para a skill usar.
  • Intent Router no agente v4 — Antes de gerar texto livre, classificar intenção: ANALISE, GERA_PETICAO, UPLOAD_DOC, DOWNLOAD, TRIAGEM, LIVRE. Cada intenção dispara skill diferente.
  • Session state por thread — Manter estado da conversa: processo carregado, documentos enviados, triagem respondida. Salvar em olivia_threads.metadata.
  • Action cards estruturados — Quando OlivIA precisar de escolha (tipo petição, modelo), retornar action_cards[] na mensagem. Frontend renderiza como botões inline.
  • Progresso em tempo real — Durante geração, inserir mensagens parciais de progresso no Supabase (role: 'system', content: '📝 Gerando...'). Frontend já tem Realtime.
  • Endpoint de download — GET /olivia/chat/download/:file_id com auth. Ou servir diretamente do Supabase Storage (URLs públicas temporárias).
🎨

Frontend — Mudanças Necessárias

  • Onboarding obrigatório de processo — Ao criar novo chat, substituir o dialog "livre vs processo" por: sempre pede o CNJ primeiro. Chat livre pode ser opção secundária.
  • Composer com upload inline — Adicionar botão clip (📎) ao lado do textarea. Suporta drag&drop. Preview de arquivo antes de enviar. Múltiplos arquivos.
  • Renderer de action cards — Detectar action_cards[] na mensagem e renderizar como botões inline abaixo do bubble. Click → envia escolha como mensagem.
  • Progress indicator — Mensagens de sistema com role='system' renderizadas diferente (tag pequena, não bubble), mostrando progresso de geração.
  • Download card — Tipo especial de action card com ícone de arquivo + botão de download. Detectar por action_cards[].type === 'download'.
  • Markdown rendering — Já tem ReactMarkdown no ChatDrawerOlivia. Trazer para OliviaChatPage também. Formata análises e relatórios inline.
  • Context header — Mostrar processo ativo no header do chat: número CNJ, cliente, tipo de réu. Clicável para ver detalhes.

📋 6. Plano de Implementação
4 fases — do MVP ao full-power
FASE 1 — MVP (2-3h) · Claude Code

Reconstruir OliviaChatPage: step 1 obrigatório com SearchBoxCNJ, header mostrando processo ativo, Markdown rendering nas mensagens, composer melhorado (textarea expandível, contador de chars). Sem mudança no backend ainda.

FASE 2 — Upload (3-4h) · Backend + Frontend

📎 Upload de Documentos Inline

Backend: endpoint POST /olivia/chat/upload no webhook-server.mjs, salva no Supabase Storage, cria registro em olivia_chat_docs, roda classifica-docs e OCR async. Frontend: botão clip no composer, preview de arquivo, pill de doc enviado na conversa.

FASE 3 — Intent Router (4-5h) · Backend OlivIA

🧠 Roteamento de Intenção + Action Cards

Backend: intent classifier no agente v4 antes do texto livre. Rota para skills corretas. Retorna action_cards na mensagem quando precisar de input (tipo petição, modelo). Frontend: renderer de action cards como botões inline, progresso de geração em tempo real.

FASE 4 — Download + Polimento (2-3h) · Full-stack

⬇️ Download Inline + UX Final

Action card de download com link temporário do Supabase Storage. Áudio no composer (hook useAudioRecorder já existe). Animações de progresso. Histórico de docs enviados visível na sidebar. E2E test de todo o fluxo.

⚠️ 7. Riscos & Decisões Pendentes
Antes de implementar, Christian precisa definir
🚨

Riscos Técnicos

  • Upload size limit — PDFs de petição podem ter 50MB+. Supabase tem limite de 5MB no Storage free tier. Precisa configurar bucket com limite maior ou usar upload direto para Mac Mini.
  • Timeout do webhook olivia/chat — Geração de petição leva 3-8min. Webhook atual tem timeout de 60s no frontend. Precisa de resposta imediata ("iniciando geração...") e progresso via Realtime.
  • Context flooding na OlivIA — Thread com muitas mensagens pode estourar contexto. Precisar de sumarização automática de histórico.
  • Múltiplos docs por thread — Classificar e usar o doc certo na hora certa. OlivIA precisa saber qual doc é prova vs qual é contrato quando gerar.
🤔

Decisões do Christian

  • Chat livre ainda existe? — Manter opção de chat sem processo ou tornar processo obrigatório sempre? Recomendo: obrigatório como padrão, com "chat livre" como exceção avançada.
  • Storage: Supabase vs Mac Mini? — Docs enviados ficam no Supabase Storage (mais simples, URLs públicas) ou Mac Mini local (mais privado, precisa endpoint de download)? Recomendo Supabase Storage.
  • Triagem conversacional ou form? — OlivIA faz perguntas uma a uma (mais natural) ou mostra mini-form com todas as perguntas de uma vez (mais rápido)? Recomendo conversacional para o MVP.
  • Compatibilidade com gerador atual? — O chat redesenhado substitui o gerador ou é paralelo? Recomendo: paralelo por ora, substitui quando chat estiver estável.
🎯

Recomendação Final

✅ Estratégia: Reconstruir OliviaChatPage do zero em 4 fases. Fase 1 (MVP) é segura e entrega 80% do valor — processo obrigatório + markdown + UX melhorada. Fases 2-4 adicionam o poder real: upload, intent router, download. Nenhuma fase quebra o gerador atual.
