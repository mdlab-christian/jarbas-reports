8 Decisoes do CTO

D01Polling 5s — manter ou substituir?
▼

SELECT leve com WHERE indexado no Mac Mini. Custo irrisorio (~0.001 CPU). Frontend ja usa Realtime para UI. Polling do Mac Mini e trigger de processamento, nao e para usuario.

✔ Recomendacao: MANTER polling 5s
D02Relatorios — template rigido, hibrido ou IA livre?
▼
A) Template rigido por tipo (10 templates)

Previsivel mas inflexivel. Nao adapta a particularidades do caso.

B) Template base + IA adapta (RECOMENDADO)

1 template base com secoes flexiveis. Playbooks por tipo orientam O QUE focar. Melhor equilibrio.

C) IA totalmente livre

Imprevisivel. Dificil manter qualidade consistente.

Implementacao: relatorio-template-base.md + relatorio-{tipo}.md por tipo no vault.

✔ Recomendacao: Opcao B (hibrido)
D03Perguntas Step 2 — formato JSON estruturado
▼

Tipos suportados: sim_nao, texto_livre, valor_numerico, selecao_unica, selecao_multipla

{ "perguntas": [ { "id": "q1", "texto": "Confirme o valor da restricao", "tipo": "valor_numerico", "valor_sugerido": 72804, "obrigatoria": true, "contexto": "Comprovante mostra R$728.04 mas procuracao menciona R$1.016,40" }, { "id": "q2", "texto": "O cartao de credito foi recebido pelo cliente?", "tipo": "sim_nao", "recomendacao": "nao", "obrigatoria": false } ] }
✔ Recomendacao: JSON estruturado com tipos definidos
D04Analise pos-geracao (Step 4) — modo APRENDIZADO
▼

Roda em background apos geracao completa. Input: historico completo do caso.

Output persiste em:

  • • padroes_empresa → MEMORY.md
  • • tipo_prova_re → MEMORY.md
  • • pontos_fortes / pontos_fracos → MEMORY.md
  • • juris_eficazes → jurimetria
✔ Recomendacao: Implementar como Step 4 background
D05Restricoes — fluxo completo de dados
▼
analise.mjs → SELECT restricoes WHERE processo_id |-- Dados completos → Injeta no payload IA |-- Dados parciais → IA gera pergunta com valor_sugerido |-- Sem dados → IA gera perguntas obrigatorias |-- User responde no Step 2 |-- Respostas salvam em tag_answers |-- geracao.mjs usa na peticao

Dados reais: 9.849 restricoes, so 47% com valor, 43% com data. Confirma necessidade do fluxo.

✔ Recomendacao: Fluxo completo com fallback progressivo
D06Multi-peticao — simples vs complexo
▼
Simples (sem tags)

emenda, manifestacao, alvara → gera direto do skeleton sem analise profunda

Complexo (com tags)

replica, apelacao, contrarrazoes → fluxo completo section-driven

Deteccao: COUNT olivia_modelo_tags. Se 0 → simples. Se >0 → complexo.

✔ Recomendacao: Deteccao automatica por COUNT tags
D07Formatacao — regras precisas
▼

Regras de formatacao para peticoes geradas:

  • • Recuo 2.5cm primeira linha de paragrafo
  • • Espacamento 6pt entre paragrafos
  • • Citacoes longas: recuo 4cm
  • • Jurisprudencias em blockquote
  • • Dados numericos em negrito

Implementar: formatarHTML() em geracao.mjs + FORMATACAO-v2.md no vault.

✔ Recomendacao: formatarHTML() + doc vault
