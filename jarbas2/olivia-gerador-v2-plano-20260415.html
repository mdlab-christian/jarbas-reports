Bug #1 — Estrutura Invertida
P0 CRITICO

DOCX/preview comeca com Christian Dener Paz — OAB/RS 116571 seguido de PROCESSO No:... antes do enderecamento ao juizo.

Estrutura atual (errada)
Christian Dener Paz — OAB/RS 116571 PROCESSO No: 5090163-05.2025.8.21.0001 CONTRARRAZOES DE APELACAO EXCELENTISSIMO(A) SENHOR(A)... <- deveria ser primeiro
Estrutura correta
EXCELENTISSIMO(A) SENHOR(A) DOUTOR(A) JUIZ(A) DE DIREITO DA [VARA] Proc. n. [CNJ] [qualificacao das partes] [corpo — topicos H2] Nestes termos, pede deferimento. [cidade], [data] ________________________ [advogado_nome] — OAB/[UF] [numero]
Root Cause

parsearParaBlocos() em criar-doc.mjs assume uma ordem que nao e garantida quando adicionarEstruturaCompleta() compos o HTML. O campo conteudo_html comeca com informacoes do advogado extraidas de um topico que Claude gerou violando a regra de blocos estruturais.

Bug #2 — OAB Duplicada
P0 CRITICO

Assinatura do advogado (nome + OAB) aparece 2 vezes no documento final.

Root Cause

Claude gera rodape com nome/OAB no conteudo_html (violando a regra de blocos estruturais do prompt) E adicionarEstruturaCompleta() adiciona outro rodape. A deteccao de idempotencia atual checa por "nestes termos" mas nao detecta se OAB ja esta no corpo.

Bug #3 — qualidade_ia ignorado na Geracao G7
P0 CRITICO

Tier "Avancado" usa Sonnet na geracao, igual ao padrao. O parametro qualidade_ia nao altera o modelo Claude no loop G7.

Codigo atual (errado) — geracao.mjs linha ~346
// hardcode MODEL_SONNET para TODAS as geracoes const response = await callClaude({ model: MODEL_SONNET, // <- nunca muda messages: [...], max_tokens: 4000, stream: true });
analise.mjs ja esta correto (linha 569)
const model = (qualidade === 'avancado') ? MODEL_OPUS : MODEL_SONNET;
Regra correta
Padrao Sonnet 4.6 para analise E geracao
Avancado Opus 4.6 para analise E geracao
Bug #4 — Jurisprudencias sem formatacao no DOCX
P1

No DOCX, jurisprudencias aparecem como texto corrido, sem o recuo e borda esperados de um BLOCKQUOTE.

Root Cause

parsearParaBlocos() em criar-doc.mjs usa regex que nem sempre captura todos os <blockquote> aninhados com estilos inline.

Secao 4 — Proposta de Arquitetura: Skeleton Generation

Pipeline Atual (sem skeleton) — Problemas
G7 Loop: Topico 1 -> Claude -> parseSecao() -> conteudo_html Topico 2 -> Claude -> parseSecao() -> conteudo_html ... concatenar todos -> peticao_html adicionarEstruturaCompleta() -> cabecalho + rodape substituirCamposDados() -> resolve placeholders
Problemas: Claude pode gerar estrutura extra, ordem depende da concatenacao, cabecalho/rodape nao integrados ao template.
Pipeline Proposto (com skeleton) — Solucao
ANTES do G7 Loop: gerarSkeletonTemplate(modelo) -> skeleton_html (completo) -> HTML com {{fatos}}, {{fund.}} G7 Loop: Topico 1 -> Claude -> parseSecao() -> skeleton_html.replace('{{fatos}}') Topico 2 -> Claude -> parseSecao() -> skeleton_html.replace('{{fund.}}') ... peticao_html = skeleton_html substituirCamposDados() -> campos de dados finais
Beneficio: Estrutura 100% controlada, sem concatenacao arbitraria, Claude nunca pode inserir estrutura errada.
Fallback e Onde Implementar
Fallback

Se o modelo nao tem template, continua com o pipeline atual (concatenacao + adicionarEstruturaCompleta). Nenhuma regressao para modelos sem template.

Onde Implementar

Step 2→3 da EF olivia-gerar. Ja monta peticao_referencia com modelo_id, secoes e template_placeholders. O skeleton seria gerado apos montar peticao_referencia, antes de disparar o G7 no Mac Mini.
