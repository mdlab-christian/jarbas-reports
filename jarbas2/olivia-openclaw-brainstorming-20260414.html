7. Roadmap de Implementacao

Fase 1 — Fundacao
Refatorar core + criar modulos
Base tecnica para o sistema batch
  • Refatorar treinamento.mjs — exportar inserirNoVault + gerarLogAbsorcao
  • Criar media-processor.mjs — download Telegram, Whisper, pdf-parse, Vision
  • Criar treinamento-batch.mjs — orquestrador do pipeline batch
  • Adicionar endpoint POST /olivia/treinar-session em server.mjs
Fase 2 — Testes curl
Validacao HTTP sem Telegram
Testar cada action via curl direto no server
  • curl start → verificar sessao JSON criada
  • curl add (text/audio/document/photo) → verificar itens acumulados
  • curl finish → verificar classificacao Opus + insercao vault
  • curl cancel → verificar cleanup sessao
Fase 3 — Prompt
System prompt Opus + testes classificacao
Calibrar a qualidade da classificacao batch
  • Criar treinamento-batch.md (system prompt Opus)
  • Testar com 3 cenarios reais: texto puro, audio+texto, mix completo
  • Validar formato JSON de saida, tipos corretos, fontes rastreadas
Fase 4 — Telegram
SOUL.md v3.0 + deploy + E2E
Integracao completa via Telegram
  • Atualizar SOUL.md v2.5 → v3.0 no Mac Mini (user olivia)
  • Reiniciar OpenClaw Gateway
  • E2E: /treinar com texto → enviar audio → enviar PDF → /treinar → verificar vault
Fase 5 — Polish
HTML report + cleanup + docs
Refinamento final e documentacao
  • /treinar html — gerar relatorio Jarbas 2.0 em feedbacks/reports/
  • Cron cleanup sessoes expiradas (>30min)
  • Documentacao em olivia-vault/agente/README.md

8. Riscos e Mitigacoes

RiscoProb.ImpactoMitigacao
OpenClaw nao detecta sessao ativa Media Alto POST status ao server antes de cada decisao no SOUL.md
Audio grande demora no Whisper Baixa Baixo Timeout 60s, resposta "recebido" imediata ao usuario
PDF sem OCR (scan) Media Medio Fallback Sonnet Vision page-by-page
Sessao abandonada Alta Baixo Auto-expire 30min + notificacao via Telegram
Telegram >4096 chars Media Baixo Truncar + split em multiplas mensagens + oferecer HTML
Conflito de sessao (2 treinos simultaneos) Baixa Baixo Uma sessao por user_id, rejeitar start se ja ativa
