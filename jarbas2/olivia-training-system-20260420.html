DRIFT-02 P2
Env vars não configuradas no Mini

Faltam: OLIVIA_TOOLS_ENABLED, ABSORCAO_V3, OLIVIA_VAULT_PATH, TELEGRAM_TEAM_OLIVIA_ID.

Impacto: tools não ligam, absorção V3 não ativa, alertas caem no fallback TELEGRAM_CHAT_ID.

Correção (após DRIFT-01):

ssh mac-mini-tail "echo '010597' | sudo -S bash -c 'cat >> /Users/olivia/olivia-server/.env << EOF
OLIVIA_TOOLS_ENABLED=false
ABSORCAO_V3=true
OLIVIA_VAULT_PATH=/Users/olivia/Library/Mobile Documents/com~apple~CloudDocs/Documents/olivia-vault
EOF'"
DRIFT-03 P1 operacional
Full Disk Access pendente para /bin/bash e /usr/bin/rsync

Sintoma: vault-sync.log continua com rsync: Operation not permitted. Christian reporta que habilitou FDA mas provavelmente só para Terminal app, não para os binários.

Correção manual (2min):

  1. System Settings → Privacy & Security → Full Disk Access
  2. Clicar + (desbloquear com Touch ID se precisar)
  3. Cmd+Shift+G → colar /bin/bash → Enter → Selecionar
  4. Repetir para /usr/bin/rsync
  5. Terminal: launchctl kickstart -k gui/$(id -u)/com.olivia.vault-sync
  6. Validar: tail -5 ~/Library/Logs/olivia-brain/vault-sync.log → sem "Operation not permitted"

Workaround atual: Obsidian Sync (LaunchAgent com.olivia.obsidian-vault) já cobre sync principal. DRIFT-03 é robustez, não bloqueio.

DRIFT-04 P3 baixo
30 treinamentos recentes com camada='global' (sem categoria_empresa_id)

Impacto: RAG ainda encontra por similaridade semântica (categoria é boost, não filtro). Relevância levemente menor.

Correção opcional: RPC fn_match_empresa_categoria (referenciada em T3 da wave pré-W7).

DRIFT-05 P3 natural

Causa: vault-consolidator requer gerações com processo_id + markdown. Primeiras gerações pós-deploy vão popular automaticamente.
