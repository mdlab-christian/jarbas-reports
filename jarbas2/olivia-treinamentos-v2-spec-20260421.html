Fluxo Completo de Geracao de Peticao — 9 Etapas

1
System Prompt Base
SOUL.md + geracao.md + estilo.md → system prompt fixo injetado em todo caso
2
Memoria do Vault (cap 40K chars)
GLOBAL.md + empresa-{slug}.md + processo-{CNJ}.md → contexto estruturado deterministico
3
Regras de Selecao de Playbooks
playbook_registry.json → quais playbooks existem e quando cada um se aplica
4
Override On-Demand (frontend)
input.playbook_ids[] do Step 2→3 → carrega playbooks especificos selecionados pelo advogado
5
Busca RAG Semantica
fn_rag_search_treinamentos(embedding do caso) → top 12 chunks → <treinamentos_relevantes>
6
Jurisprudencia Favoravel
decisoes-favoraveis/ → ate 3 decisoes relevantes injetadas no user message
7
Geracao Principal
Sonnet 4.6 (peticoes completas) ou Haiku (smart blocks) → HTML estruturado por secoes
8
Boris Juris-Validator
Valida citacoes → score 0-10 com penalidades (alucinacao 3x, distorcao 1.5x) → persiste em olivia_boris_outputs
9
Boris Revisor (se ENABLE_BORIS ativo)
Revisao adversarial — foco citacoes OU qualidade — preserva HTML — degradacao silenciosa

Recomendacoes Adicionais — Alem das Decisoes Formais

R1
Trigger PostgreSQL para RAG ingest
Trigger em olivia_treinamentos ON INSERT/UPDATE enfileira automaticamente. Hoje e manual e esquecido — causa os 74% sem indexacao.
R2
Deduplicacao por Boris
Boris identifica treinamentos com similaridade maior que 80% e propoe mesclagem para o advogado aprovar antes de executar.
R3
Feedback loop de peticoes
Peticao com score maior ou igual a 8 → boris-playbook-manager extrai o que funcionou e adiciona ao playbook correspondente automaticamente.
R4
Versioning de playbooks via git
Cada git commit dos playbooks e rastreado → OlivIA sabe qual versao usou para gerar cada peticao. Rastreabilidade completa para auditorias.
R5
Relatorio de uso de playbooks
Rastrear quais playbooks sao usados com mais frequencia e correlacionar com score das peticoes para identificar os mais eficazes.
R6
Vault Health Check automatico
Script periodico verifica integridade vault vs Supabase: detecta chunks sem embedding, arquivos orfaos, inconsistencias de slug.
