Template switcher no editor
Botao "Aplicar estilo Christian/Heloisa" que reformata a peticao inteira (topicos, fontes, blockquotes) sem regenerar conteudo. Util para processos que mudam de advogado.
FRONTEND #3
Diff visual entre versoes
Quando o operador regenera ou edita, diff colorido (verde=adicionado, vermelho=removido) entre versao anterior e atual.
FRONTEND #4
Minimap no editor
Como VS Code, um minimap na lateral direita do TipTap mostrando a estrutura do documento. Para peticoes longas (30+ paginas), navegacao rapida.
FRONTEND #5
Keyboard shortcuts no VibeLaw
Cmd+K para abrir command palette (como VS Code). Mais rapido que digitar "/" no meio do texto.
FRONTEND #6
Status persistente por tab
Quando o operador fecha o browser e volta, a tab retoma exatamente de onde parou (step, topicos selecionados, rascunho).
FRONTEND #7
Comparacao A/B de geracao
Gerar 2 versoes simultaneas (uma Sonnet, uma Opus) e deixar o operador escolher qual prefere. Alimenta loop de qualidade.
MECANISMO #8
Cache de vault-reader
Hoje le o vault inteiro a cada geracao. Implementar cache com TTL de 1h (vault muda pouco). Reduz latencia em ~2s.
MECANISMO #9
Streaming parcial
Em vez de esperar S6 terminar toda a peticao, streaming por secao: "Enderecamento pronto" aparece no editor, depois "Qualificacao pronta", etc.
MECANISMO #10
Fallback automatico de modelo
Se Sonnet 4.6 estiver com latencia >10s, fallback para Opus automaticamente. Redundancia sem intervencao humana.
MECANISMO #11
Deteccao de plagio/repeticao
Antes de finalizar, comparar secoes com as ultimas 50 peticoes. Se >80% similar, alertar: "Peticao muito similar ao processo X."
MECANISMO #12
Analise de sentenca IA aprimorada
Usar Opus para analisar sentencas desfavoraveis. Mapear argumentos aceitos/rejeitados pelo juiz. Alimenta playbook por vara/juiz.
