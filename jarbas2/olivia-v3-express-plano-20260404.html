Decisão Estratégica — Express First

01
Funcionalidade P0
O gerador de petições é o coração do MdFlow. Sem ele nada funciona. Express first = produto funcional antes de expandir Chat e Análise.
02
Escopo controlável
4 steps bem definidos, sem complexidade paralela. Cada step tem responsabilidade única. Mais fácil testar, debugar e entregar.
03
Validar mecanismo
Express conecta diretamente com pipeline-v3.mjs. Se funcionar ponta a ponta, temos confiança para expandir. Se falhar, descobrimos cedo.
04
Código 30% mais limpo
Apagar Chat, Análise, Editor e Smart Blocks elimina 4.600+ linhas não usadas. Frontend menor, mais rápido, menos bugs.

Fluxo Técnico Completo

1. Usuário abre /olivia
OliviaGeradorLanding → clica "Nova Petição Express" → OliviaGeradorExpress com Step 1
2. Step 1: Identificação
Seleciona processo + tipo + modelo + qualidade. Upload PDFs → olivia-extrair-texto (OCR). Clica "Analisar".
3. handleStep1Next() — FIX CRÍTICO DA RACE CONDITION
RPC fn_olivia_iniciar_geracao() → retorna historicoId. Webhook /olivia/gerar fase=1. AWAIT historicoId confirmado → SOMENTE ENTÃO setStepIndex(1). Mac Mini: S1 contexto + S2 RAG + S3 tópicos.
4. Step 2: Tópicos + Análise
Realtime aguarda status='AGUARDA_RESPOSTAS'. Carrega tópicos e análise. Usuário: DnD reordenar + responde perguntas + toggle. Clica "Gerar Petição".
5. handleStep2Next()
Webhook /olivia/gerar fase=2 (topicos_aprovados + instrucoes). setStepIndex(2) + GenerationProgressPanel. Mac Mini: S4 gerarPeticao + S5-S8 loopRevisao + S9 persistir.
6. Step 3: Editor
Realtime: status='CONCLUIDO' → carrega peticao_html no TipTap. Usuário edita manualmente. Autosave 30s em olivia_historico.
7. Step 4: Conclusão
Downloads PDF/DOCX/GDoc (URLs do banco). FeedbackSection → fn_olivia_registrar_feedback → S9 retroalimenta playbook. Botão "Nova Petição".
