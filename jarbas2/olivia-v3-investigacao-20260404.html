P0-01 · SECURITY

  • oliviaService.ts
  • useOliviaChat.ts

P0-02 · SCHEMA

olivia_flow NAO EXISTE

A tabela olivia_flow e referenciada por 2 RPCs e pela Tab "Flow" do frontend, mas nunca foi criada no banco de dados.

  • rpc_get_olivia_flow()
  • rpc_update_olivia_flow()
  • OliviaFlowTab.tsx (inteiro)
P0-03 · DATA INTEGRITY

Status inconsistente RPCs vs dados reais

RPCs retornam status hardcoded que nao refletem o estado real dos registros. 5 RPCs afetadas com logica de status desatualizada.

  • rpc_get_olivia_documents — status sempre "rascunho"
  • rpc_olivia_generate — nao atualiza status pos-geracao
  • rpc_olivia_review — campo review_status ignorado
  • rpc_get_olivia_history — timestamps incorretos
  • rpc_olivia_submit — status nao transiciona
P0-04 · ARCHITECTURE

3 pipelines coexistentes (3700L duplicadas)

Tres pipelines fazem a mesma coisa com logicas diferentes, causando resultados inconsistentes e manutencao impossivel.

  • Pipeline A: olivia-legacy/ (1200L) — webhook n8n
  • Pipeline B: olivia-v2/ (1500L) — Edge Function direta
  • Pipeline C: olivia-v3/ (1000L) — hibrido incompleto
P0-05 · QUALITY

converterReplicaParaHtml — gargalo de qualidade

Funcao central da geracao de peticoes. Conversao de replica para HTML produz formatacao inconsistente, perda de paragrafos e indentacao quebrada em 40% dos casos testados.

P0-06 · SECURITY

olivia_structural_templates sem RLS

Tabela de templates estruturais acessivel por qualquer organizacao. Dados de um escritorio podem ser lidos por outro tenant sem restricao.

-- FALTANDO: ALTER TABLE olivia_structural_templates ENABLE ROW LEVEL SECURITY; CREATE POLICY "tenant_isolation" ON olivia_structural_templates USING (organizacao_id = public.get_org_id());
