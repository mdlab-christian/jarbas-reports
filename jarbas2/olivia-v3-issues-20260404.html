3. Como Usar na Pratica

Passo a Passo
  1. Abrir 6 terminais no Claude Code Desktop (ou VS Code com extensao Claude)
  2. Em cada terminal, colar o prompt de recovery da secao 5 abaixo
  3. T-CTRL (este terminal) coordena — NAO implementa codigo
  4. Comecar SEMPRE pelo T-BACK com Sprint 0 (security fixes)
  5. Depois de Sprint 0, abrir T-FRONT + T-BACK em paralelo (Sprint 1 + Sprint 6)
  6. A cada sprint concluido, T-CTRL valida + roda code review GPT-5.4
  7. Se encontrar P0/P1, T-CTRL cria issue de fix e delega ao terminal correto
  8. Sprint 9 (QA) so comeca quando TODOS os outros terminaram
NUNCA 2 terminais mexendo no mesmo arquivo simultaneamente
Verificar pre-condicoes antes de iniciar cada issue
Se bloqueado por dependencia, pular para proxima issue do sprint
1 commit por issue — nao acumular
Recovery: se terminal perder contexto, colar prompt + “continua de onde parou”
ZERO TODO/placeholder — implementar completo ou PENDENCIA formal
