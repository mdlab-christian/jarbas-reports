Licoes Aprendidas — Fase 2

#1
Investigacao antes de correcao
Wave 10 mapeou todos os bugs e inconsistencias antes de tocar codigo. Resultado: zero retrabalho nas waves seguintes.
#2
Vault-driven prompts
Migrar prompts hardcoded para banco/vault permite iterar sem redeploy. Mudancas de prompt agora sao operacionais, nao tecnicas.
#3
Score como metrica
Medir consistencia (6.5→9/10) deu visibilidade objetiva ao progresso. Sem metrica, "melhorou" e subjetivo.
#4
Component splits pagam rapido
StepEditorVibeLaw 762→472L tornou T61 (toggle chat/edit) trivial de implementar. Arquivos menores = features mais rapidas.
#5
Payload alignment via E2E
T67 so encontrou 3 problemas de tipo porque testou com dados reais, nao mocks. Validacao E2E e insubstituivel.
