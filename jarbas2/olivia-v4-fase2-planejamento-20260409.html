Bugs Detalhados

8 bugs
▶ P0 B1 — Relatorio HTML retorna 502
Erro
olivia-relatorio 502 "IA retornou formato invalido"
Causa raiz
Claude retorna JSON dentro de markdown fences (```json ... ```). O regex que parseia a resposta nao remove as fences antes de fazer JSON.parse, causando falha silenciosa que resulta em 502.
Arquivo
supabase/functions/olivia-relatorio/index.ts:86
Fix
Strip markdown fences antes do regex. Correcao de 1-2 linhas.
▶ P0 B2 — Chat sem contexto do processo
Causa raiz
A Edge Function olivia-vibelaw no modo chat recebe APENAS tipo, empresa e score. Nao recebe analise_estruturada, topicos nem alertas. Claude responde sem conhecer o processo.
Arquivo
supabase/functions/olivia-vibelaw/index.ts:170-173
Fix
Injetar analise_estruturada + topicos + alertas no system prompt do chat. Aproximadamente 15 linhas.
▶ P0 B3 — Topicos sem perguntas
Causa raiz
O Mac Mini gera topicos com formato {ordem, titulo, descricao} mas sem campo perguntas. O SOUL.md regra R-08 define: confianca < 70% deve gerar pergunta ao advogado. Essa logica nunca foi implementada no step S2.
Arquivo
olivia-server/skills/olivia-pipeline-v3.mjs (S2 — geracao topicos)
Fix
Cruzar tags do modelo com provas analisadas. Para cada tag com confianca baixa, gerar pergunta especifica. Estimativa: 30-50 linhas.
▶ P0 CRIT B4 — peticao_referencia VAZIA
Evidencia
{secoes: [], tag_answers: {}, blocos_resolvidos: {}}
Cadeia de falha
Perguntas nao aparecem (B3) → tag_answers fica vazio → blocos nao sao mapeados → template do modelo e ignorado pela Edge Function → Claude gera peticao "do zero" sem usar os 13 placeholders do modelo.

O modelo Servico Credito Online V3 possui 13 placeholders (ex: {{preliminares}}, {{dano_moral}}) que deveriam ser preenchidos com blocos contextualizados. Nenhum chega.
Arquivo
supabase/functions/olivia-gerar/index.ts:80-147
Fix
Rewrite parcial: ler template, mapear placeholders para tags, resolver blocos por tag_id, montar peticao_referencia completa. Fix mais complexo da lista.
▶ P1 B5 — DOCX/PDF stuck em pct=1
Causa raiz
A etapa criar_doc do pipeline inicia (pct=1) mas nunca progride. A Edge Function olivia-criar-doc provavelmente falha silenciosamente sem atualizar o progresso.
Fix
Investigar pipeline de conversao. Verificar se a EF recebe o HTML corretamente e se a conversao DOCX/PDF esta funcional.
▶ P2 B6 — Chat scroll UX
Problema
O chat nao possui container com scroll proprio. Mensagens empurram a pagina inteira. Input deveria ser fixo no rodape do painel.
Fix
CSS: container com overflow-y: auto + input com position: sticky; bottom: 0.
▶ P2 B7 — Falta advogado no Step 1
Problema
O painel de identificacao (Step 1) nao exibe nome do advogado nem OAB. Informacao importante para contexto da peticao.
Fix
Buscar advogado_responsavel do processo e exibir no painel com nome + OAB.
▶ P2 B8 — Falta seletor de versao
Problema
Nao ha dropdown para selecionar entre V1, V2, V3 do modelo. O sistema usa a versao ativa por default sem opcao de escolha.
Fix
Adicionar dropdown no Step 1 que lista versoes disponiveis do grupo selecionado. Default: versao com ativo = true.

Decisoes CTO

3 decisoes

D1 Chat — Arquitetura

A) Enriquecer contexto Claude — injetar analise + topicos no system prompt RECOMENDADO
0 dependencias novas, 1 Edge Function editada. Rapido e eficaz.
B) OpenAI Assistants com thread_id — memoria persistente entre sessoes
Memoria automatica, porem custo alto e vendor lock-in.
C) Hibrido — Claude com contexto + thread_id para historico
Melhor dos mundos, mas adiciona complexidade significativa.

D2 VibeLaw — Modo

A) Toggle Chat / VibeLaw — botao alterna entre os dois modos RECOMENDADO
UX clara, sem ambiguidade. Advogado sabe em que modo esta.
B) Unificado auto-detect — sistema detecta se e pergunta ou comando
UX "magica" mas propenso a erros de deteccao.

D3 Relatorios — Escopo

Proposta: Mesma Edge Function com parametro modo
Step 2: Analise Provas
Qualidade da analise, cobertura de provas, alertas detectados. Rapido, parcial.
Step 4: Completo
Analise + peticao + jurisprudencia + score + recomendacoes. Final, exportavel.

Avaliacao Sincera

A V4 tem potencial real?
SIM. A V4 e objetivamente superior a V3: zero dependencia de n8n, arquitetura pull-based, super agente Mac Mini, 4 modos de operacao, RAG com pgvector. A base arquitetural esta correta.
O que falta entao?
MECANISMO. As pecas existem individualmente (modelos, tags, blocos, jurisprudencias, Edge Functions, pipeline) mas nao estao conectadas. O modelo V3 tem 13 placeholders, 1.258 blocos e 183 tags — nenhum chega no agente IA. E um problema de integracao, nao de arquitetura.
Qual o esforco real?
Cirurgico. Cada fix individual varia de 1 a 50 linhas de codigo. O plano organiza em 4 waves com 16+ tasks. Estimativa: 2-3 sessoes de Claude Code para completar Waves 0-3. Wave 4 (validacao) pode ser feita em paralelo com testes do CTO.
Riscos?
O maior risco e o fix de B4 (peticao_referencia). E o unico que requer rewrite parcial em vez de patch simples. Se a logica de mapeamento placeholder → tag → bloco tiver edge cases nao mapeados, pode precisar de iteracao extra. Todos os outros fixes sao diretos.
