O que esta quebrando (Root Causes)

O Fluxo de Dados esta Quebrado. As provas enviadas no Step 1 nunca chegam ao agente que gera a peticao no Step 3. As instrucoes adicionais somem. As respostas das perguntas do Step 2 nao sao coletadas. O modelo de peticao (skeleton/tags) nao e reenviado. Resultado: o Mac Mini gera sem contexto real do caso — o pipeline roda no escuro, com dados insuficientes, e a peticao sai generica independente do que o usuario enviou.

Mecanismo Ideal — Como Deve Funcionar

STEP 1 — Identificacao
  • Entrada:
  • processo + tipo peticao
  • modelo + skeleton (tags)
  • provas (PDFs/imagens)
  • instrucoes_adicionais
  • OCR: Claude Sonnet extrai texto de todas as provas
  • Salva: analise_provas no historico
  • Envia ao Mac Mini:
  • processo + cliente + empresa
  • modelo_skeleton (tags ordenadas)
  • texto_ocr_consolidado
  • instrucoes_adicionais
  • tipo_peticao
  • Resultado: topicos com perguntas
→
STEP 2 — Topicos e Perguntas
  • Usuario:
  • responde perguntas inteligentes
  • edita titulos, reordena (DnD)
  • desativa topicos desnecessarios
  • Envia ao Mac Mini:
  • topicos aprovados + respostas
  • analise_provas.texto_ocr
  • meta.instrucoes_adicionais
  • modelo_skeleton (re-buscado)
  • tipo_peticao
  • Pipeline S1-S9:
  • S1: Contexto rico (RAG + processo)
  • S3: Gera JSON por tag
  • S5-S6: Revisor (meta ≥ 8.5)
  • S8: Formatacao juridica
  • S9: Salva peticao_html + score
→
STEP 3 — Editor VibeLaw
  • Editor TipTap:
  • peticao HTML renderizada
  • formatacao juridica preservada
  • Sumario lateral:
  • topicos clicaveis
  • selecionar topico → foca no editor
  • vincula ao chat VibeLaw
  • Chat VibeLaw:
  • edita DENTRO do topico selecionado
  • nao reescreve a peticao inteira
  • Autosave 30s
  • olivia_historico.peticao_html
→
STEP 4 — Conclusao
  • Forca-save da peticao editada
  • Export:
  • DOCX/PDF = versao editada
  • NAO regenera — usa o que foi salvo
  • Relatorio HTML:
  • analise do caso enriquecida
  • gaps identificados pelo pipeline
  • Criar Atividade:
  • pre-preenchida com dados do processo
  • advogado + data + tipo automaticos
