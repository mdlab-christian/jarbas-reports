Decisoes Respondidas (D1-D8)

D1 Resposta: A
Manter Mac Mini Node.js como engine de geracao

Mac Mini M4 continua como motor principal. Edge Functions orquestram, Mac Mini processa. Pipeline-v3.mjs (108KB, 9 estagios) e o coracao.

D2 Resposta: A
Consolidar server.mjs — remover rotas legadas

Remover /olivia/gerar e /olivia/pipeline-v3. Manter apenas /olivia/pipeline-v4 (unificado topicos+gerar), /olivia/criar-doc, /olivia/eproc-baixar.

D3 Resposta: A — OCR Fallback
OCR apenas como fallback. Claude recebe docs diretamente.

Modelo de IA (Claude Sonnet 4.6) recebe PDFs/imagens como document blocks. Analisa layout real. OCR (olivia-extrair-texto) so quando analise direta falhar. Economia: 1 API call/doc, analise superior.

D4 Resposta: A
Smart Blocks: JSON referencia + IA

peticao_referencia construida com blocos curados + scoring IA (pgvector RAG 768-dim). Threshold 0.65 para auto-resolve, abaixo disso gera pergunta pro usuario.

D5 Resposta: A
EPROC integration: implementar agora

Auto-download de documentos do EPROC quando usuario seleciona processo no Step 1. Skill olivia-eproc-baixar ja existe no Mac Mini. Falta: trigger no frontend + storage Supabase.

D6 Resposta: B — Zero n8n
Manter tudo EF + Mac Mini. n8n fora do fluxo de geracao.

Nenhum workflow n8n no fluxo critico de geracao. EFs orquestram. Mac Mini processa. Supabase Realtime notifica. n8n pode ser usado para integracao lateral (notificacao Telegram), nunca no caminho critico.

D7 Resposta: A
HTML Report: EF separada (olivia-relatorio)

Criar nova EF olivia-relatorio. Separa responsabilidade de olivia-criar-doc (que fica so DOCX/PDF). Relatorio HTML e rico (10 secoes, score, pilares, timeline).

D8 Resposta: A
Retry: re-chamar EF inteira

Em caso de erro, frontend re-chama a EF (olivia-topicos ou olivia-gerar) com o historico_id existente. EF detecta retry, le estado do banco, retoma de onde parou. Nunca recria historico.

Principio Central: olivia_historico e o state machine. Todo estado persiste no banco ANTES de chamar Mac Mini. Se Mac Mini cair, retry le do banco e retoma. Payload minimo EF → Mac Mini: {historico_id, organizacao_id, modo}.

  • Recebe request do frontend (HTTP)
  • Valida JWT + autorizacao

  • Retorna imediatamente {historico_id}
  • Timeout max: 25s (Supabase limit)
🤖 Mac Mini (processamento)
  • Le TUDO do banco via historico_id
  • Executa pipeline S0-S9 (30-480s)
  • Chama Claude API (Sonnet/Opus)
  • Atualiza progresso no banco (Realtime)
  • Gera DOCX/PDF via Puppeteer
  • Rate limit: 5 concurrent, 50/hour/org
  • Self-improvement S9 (score >= 9.0)

Detalhamento: olivia-topicos (493L) — Orquestrador Step 2
Input
{processo_id, tipo_peticao_id, modelo_id, versao_id, advogado_id, organizacao_id, qualidade, instrucoes_adicionais, doc_urls[]}
Output
{success: true, historico_id: "uuid", message: "Analise iniciada"}
Fluxo Completo
  1. Auth: extrai userId do JWT sub (rapido, sem getUser())
  2. Cria olivia_historico via RPC fn_olivia_iniciar_geracao
  3. Carrega modelo_skeleton (fallback RPC fn_olivia_get_model_skeleton)
  4. Carrega jurisprudencias curadas (advogado-specific > general)
  5. Persiste meta (instrucoes + juris)
  6. ATUAL: OCR via olivia-extrair-texto p/ cada doc → NOVO: Passa doc_urls direto
  7. Tag RAG: p/ cada tag, busca blocos (pgvector >= 0.65 auto-resolve, < 0.65 gera pergunta)
  8. Persiste tag_answers + modelo_versao_id + modelo_skeleton
  9. Pre-flight: Health check Mac Mini (/health, 3s timeout)
  10. Fire-and-forget: POST /olivia/pipeline-v4 {historico_id, modo: "topicos"}
  11. Retorna {historico_id} imediatamente
Tabelas Afetadas
Escrita: olivia_historico (meta, texto_ocr, tag_answers, modelo_versao_id, modelo_skeleton), olivia_topicos (insert perguntas)
Leitura: olivia_modelo_juris, olivia_jurisprudencias, olivia_modelos_peticao, olivia_tags, fn_olivia_get_tags_com_blocos, fn_buscar_blocos_rag
Detalhamento: olivia-gerar (421L) — Orquestrador Geracao
Input
{historico_id, topicos_aprovados: [{id, titulo, ordem, instrucoes}], qualidade, tag_answers}
Output
{success: true, message: "Geracao iniciada"}
Fluxo Completo
  1. Auth: JWT passthrough
  2. Fetch historico (valida ownership via org)
  3. Update topicos: titulo, ordem, consideracoes, ativo=true
  4. Soft-delete topicos NAO selecionados (deleted_at = now)
  5. Update qualidade_ia
  6. RPC fn_olivia_atualizar_status → PROCESSANDO
  7. Resolve versao_id (se so modelo_id, busca versao ativa)
  8. Fetch modelo_skeleton
  9. T28: tag_answers → peticao_referencia (blocos + jurisprudencias por secao)
  10. Persiste peticao_referencia + tag_answers no DB
  11. Pre-flight: Health check Mac Mini
  12. Fire-and-forget: POST /olivia/gerar {historico_id} (EdgeRuntime.waitUntil, 30s)
peticao_referencia e a estrutura rica que guia a geracao multi-agente:
{modelo_id, secoes: [{secao_id, blocos: [{bloco_id, conteudo_html_rico, jurisprudencias: [...]}]}]}

Agentes e Sub-Agentes IA

O pipeline OlivIA usa 6 "agentes" conceituais — cada um e uma chamada Claude com system prompt especializado. Nao sao servicos separados.

🔍
Analista (S1-S2 + topicos)
Quando: Modo topicos. Chamado pelo pipeline no Mac Mini.
O que faz: Analisa provas (document understanding), gera topicos, avalia chance de exito, mapeia pilares probatorios, identifica alertas.
Modelo: Claude Sonnet 4.6 (doc blocks + texto)
Output: topicos[], analise_estruturada {score, chance_ganho, pilares[], alertas[], mapeamento_provas[]}
Prompt: Hardcoded em pipeline-v3.mjs (executarModoTopicos). 5 regras universais (STJ 385, ônus probatorio, dano moral faixas).
✍
Redator (S3-S4)
Quando: Modo gerar. Apos topicos aprovados.
O que faz: Redige a peticao completa em HTML semantico. 30+ diretivas no system prompt. Per-lawyer formatting (Calibri/Georgia, OAB, cidade). Variacao layer aleatorio.
Modelo: Sonnet 4.6 (default, 32K tokens) / Opus 4.6 (avancado, 40K tokens + Extended Thinking 8K budget)
Multi-agent: Se peticao_referencia tem secoes, gera cada secao em paralelo, depois sintetiza.
Prompt: buildSystemPromptV3() + buildUserPromptV3() em pipeline-v3.mjs.
📜
Revisor (S6)
Quando: Apos cada geracao. Loop ate 3 tentativas.
O que faz: Avalia qualidade juridica. Score 0-10. Feedback especifico (o que melhorar). Decide: aprovado ou reescrever.
Modelo: Claude Sonnet 4.6
Threshold: Atual: 7.0 | Spec sugere: 8.5
Skill: olivia-revisor.mjs (12KB)
⚡
VibeLaw (Step 3)
Quando: Step 3, usuario digita "/" ou usa chat.
O que faz: 5 comandos inline (reescrever, expandir, jurisprudencia, simplificar, formalizar). Recebe trecho selecionado, retorna versao melhorada.
Modelo: Claude Sonnet 4.6 (max 2048 tokens por comando)
Prompts: olivia_prompts table (slug-based, org-specific). Hardcoded fallback.
EF: olivia-vibelaw (742L)
💬
Chat Analise (Step 2/3)
Quando: Modo chat_analise no painel lateral.
O que faz: Conversa contextual sobre o caso. RAG v2 (pgvector 768-dim). Inteligencia juridica (julgador + empresa). Protocolo de diagnostico.
Modelo: Claude Sonnet 4.6 + OpenAI Embedding 3-S
Contexto: Caso + score + provas + 4 RAG chunks + jurimetria
EF: olivia-vibelaw modo "chat_analise"
👥
Multi-Agent Generator (S4 variant)
Quando: peticao_referencia.secoes.length > 0.
O que faz: Gera cada secao da peticao em paralelo (Promise.all). Depois sintetizarPeticao() combina tudo num HTML coeso. Pula S6 revisao (aprovado direto).
Modelo: Sonnet 4.6 por secao (paralelo)
Vantagem: Mais rapido para peticoes longas. Cada secao tem blocos + juris especificos.

OCR vs Document Understanding Direto

❌ Fluxo Atual (OCR separado)
// 2 API calls por documento 1. EF olivia-topicos recebe doc_urls 2. Chama olivia-extrair-texto p/ cada doc 3. olivia-extrair-texto envia doc ao Claude 4. Claude retorna texto extraido 5. Texto salvo em texto_ocr 6. Mac Mini le texto_ocr do banco 7. Mac Mini chama Claude novamente c/ texto
Problema: 2 chamadas Claude por doc. Perde layout (tabelas, formatacao). Mais lento. Mais caro.
✅ Fluxo Novo (Document Understanding)
// 1 API call por documento 1. EF olivia-topicos recebe doc_urls 2. Passa doc_urls pro Mac Mini 3. Mac Mini fetch docs (base64) 4. Claude recebe docs como document blocks 5. Claude analisa: layout + conteudo + provas 6. Resultado salvo em analise_estruturada 7. Texto extraido como subproduto (opcional)
Vantagem: 1 chamada Claude. Ve layout real (tabelas, assinaturas). Mais rapido. Mais barato.
Recomendacao: Document Understanding como primario. olivia-extrair-texto vira fallback (docs muito grandes, PDFs corrompidos). olivia-ocr.mjs no Mac Mini tambem fallback. Economiza ~50% do custo de analise de provas.
