11 decisões aprovadas (D-11.1 → D-11.11)

§5

D-11.1 Engine de orquestração Opção A

tmux + bash · 25 panes (layout tiled 5×5). Zero dependência extra, padrão Cecília v2 escalado. Cada worker recebe branch isolada + prompt copy-paste pré-formado + heartbeat file-based. Stagger start 20s obrigatório.

D-11.2 Waves e distribuição 6 waves

W1(14) → W2(12) → W3(18) → W4(14) → W5(6) → W6(4-8). Todas as tasks da wave rodam simultâneas quando deps permitem. Dep sequencial intra-wave fica no mesmo pane.

D-11.3 Branch strategy branch-per-task

olivia-1604/T<NN>-<slug>. Merge sequencial via master (pane 0). Até ~70 branches transientes. NUNCA trunk-based, NUNCA branch-por-subprojeto.

D-11.4 Skills obrigatórias pipeline /projeto v4

Sequência inegociável: /projeto:plan --adversarial → /projeto:execute --rescue-on-fail → /report. Adversarial em TODAS as tasks — over-engineering é menor risco que vibe-coding.

D-11.5 Heartbeat + monitoramento file-based

Workers escrevem heartbeat/T<NN>.txt a cada 60s. Master varre 24 arquivos a cada 30s. Auto-restart 1× via tmux respawn-pane. Segunda morte = escalate Telegram.

D-11.6 Dashboard de progresso markdown + TUI

logs/dashboard.md auto-atualizado 30s + tmux attach live. Telegram notify em eventos críticos (wave completa, P0 security, rescue, merge conflict, rate limit).

D-11.7 Rollback playbook git revert apenas

R9 mitigação dura. SEMPRE git revert — NUNCA reset --hard, NUNCA push --force, NUNCA --no-verify. Backup pré-wave em _archived/<ts>/.

D-11.8 Anti vibe-coding hooks enforced

Hooks em .claude/settings.local.json: PreToolUse bloqueia TODO/placeholder/hex-hardcoded/any/console.log/push --force. UserPromptSubmit bloqueia /projeto:execute sem ## Plan + ## Adversarial Findings.

D-11.9 Custos estimados ~$269

50 tasks Opus 4.7 ($175) + 20 tasks Sonnet 4.6 ($16) + rescue ($20) + codex externos ($15) + cold-start tax ($43) = $269 típico, $350 worst. Cache 90% off via disciplina de system prompt.

D-11.10 Master (pane 0) 11 responsabilidades

NÃO executa tasks. Apenas: merge coord · heartbeat · dashboard · wave gate · cost tracker · escalate bridge · backup archiver · PAUSED.lock listener · report generator · rate-limit guard · keep-warm scheduler.

D-11.11 Modelo por camada mix Opus/Sonnet

Opus 4.7 para backend/frontend/n8n/rescue (crítico). Sonnet 4.6 para polish/docs/tests (~30% das tasks). Codex externo (o4-mini) para reviews. Rescue sempre Opus 4.7.

Scripts do orquestrador (11 arquivos)

scripts/olivia1604/

Todos em ~/beta-mdflow/scripts/olivia1604/. Spec em orchestrator-skeleton.md — implementação na Fase 3.

🎬 olivia-orchestrator.sh

Entry point: setup + backup + dispatch. Uso: ./olivia-orchestrator.sh W1. Pre-check de tasks.json, deps, working tree clean, PAUSED.lock ausente.

🖥️ setup-session.sh

Cria sessão tmux olivia1604 com 25 panes (layout tiled 5×5). Pane 0 master + panes 1-24 workers. Exporta CLAUDE_PROJECT_DIR e OLIVIA_PANE_ID.

🚀 spawn-task.sh

spawn-task.sh <pane> <task>. Cria branch, monta prompt, injeta via tmux send-keys. Registra em logs/pane-N-task-TNN.log.

❤️ heartbeat-check.sh

Loop 30s no master. Varre 24 arquivos heartbeat/T*.txt. >5min sem update = respawn-pane 1×. Segunda morte = escalate Telegram.

📊 dashboard.sh

Loop 30s. Fonte: heartbeats + tasks.json + cost logs + git log. Escreve em logs/dashboard.md + symlink readonly para Obsidian vault.

🔀 merge-wave.sh

Merges sequenciais. SEMPRE um por vez. Auto-merge se 8 gates = clean. Senão escala para CTO manual. git merge --no-ff sempre.

🧪 smoke-test.sh

Playwright E2E após cada wave. Login → /olivia → thread → geração → download DOCX → valida estrutura. Falha bloqueia W+1.

⏪ rollback.sh

git revert -m 1 <merge-commit> — NUNCA reset --hard. Se branch não mergeada: git branch -D.

💰 cost-tracker.sh

Parse ~/.claude/logs/<session>/. Aplica pricing: Opus $15/$75 · Sonnet $3/$15 · cached 90% off. Output em logs/cost-live.log.

📦 backup-archive.sh

Pré-wave: _archived/<ts>-pre-W<N>/. Copia tasks/, logs/, heartbeat/, spec.md. Cria tag git olivia1604-pre-W<N>-<ts> pushada.

🔥 keep-warm.sh

Loop 3min. Envia # keep-warm ping em panes idle para manter session Anthropic alive e prompt cache quente. Anti cold-start tax.

Heartbeat e auto-restart

§D-11.5
Formato heartbeat/T<NN>.txt
timestamp: 2026-04-17T14:23:45Z phase: plan|execute|review|adversarial|fix|merge|idle last_log: "applied patch fix-2" pid: 12345 pane: 17 branch: olivia-1604/T15-rpc-anonimizar tokens_used: 48320
Loop do master (30s)
  • Varre 24 arquivos heartbeat/T*.txt
  • mtime > 5min → pane morto
  • 1ª morte: tmux respawn-pane -k + spawn-task.sh de novo
  • 2ª morte: marca FAILED + Telegram via webhook
  • ≥3 panes com 429 em 5min = pause-world automático

Inspiração direta: Cecília v2 start.mjs com health endpoint :19200/health + jarbas_heartbeat table + PID lock /tmp/cecilia.pid. Arquivos em disco local (NÃO no Obsidian vault — evita sync delay).

PAUSE WORLD — botão de emergência

§10 MANUAL

⚠️ Disparar pause-world

touch /Users/christianpaz/beta-mdflow/.projeto/olivia/olivia1604/v1/PAUSED.lock

O que acontece (≤30s)
  1. heartbeat-check.sh detecta, para de spawnar tasks novas
  2. merge-wave.sh para de mergear
  3. Workers terminam a task atual (podem levar até 90min)
  4. Master fica em phase=paused no dashboard
  5. Master envia Telegram: "olivia1604 PAUSED — manual review"
Quando usar (VERMELHO)
  • ≥3 workers com heartbeat >5min
  • ≥3 erros 429 em 5min (rate limit)
  • Gitleaks detectou secret
  • Smoke E2E fail após merge
  • Cost ultrapassou $300 (90% budget)
  • CTO quer analisar ou debugar

Para retomar: rm PAUSED.lock → master retoma em ≤30s no próximo loop. NÃO usar se 1 task falhou (use rollback pontual) ou 1 worker morreu (auto-restart cuida).

Riscos residuais (R19 → R25)

§decisoes.md

R19 MÉDIO

Custo disparar (cold cache em cold restarts). Mitigação: abrir terminais em batch + keep-warm.sh a cada 3min manter session alive.

R20 BAIXO

Heartbeat file system race (tmpfs vs disk). Mitigação: heartbeat/ em disco local Pro — não Obsidian vault — evita sync delay.

R21 MÉDIO

/codex:review timeout em wave grande. Mitigação: retry 1×; skip com warn se timeout 2× (flag review_status: skipped).

R22 🔴 CRÍTICO

24 Claude processes simultâneos → limite API rate. Mitigação dura: stagger start 20s; guard pane 0 = pause-world automático se >3 panes em 429 em 5min; preferir tier Anthropic 4+ na API key.

R23 MÉDIO

Obsidian Sync corrompe dashboard.md durante escrita. Mitigação: dashboard.md em logs/ git-tracked + symlink readonly para vault.

R24 MÉDIO

Fadiga CTO em monitorar 24 panes. Mitigação: dashboard textual + Telegram notify em eventos críticos; tmux attach só quando quiser debugar pane específico.

R25 BAIXO

24 branches simultâneas saturam git index. Git aguenta milhares; .git/packed-refs pode crescer — git gc automático após cada wave.

Riscos originais R1-R18 permanecem conforme RISCOS.md do projeto — este subprojeto não os altera. R9 (N terminais = conflito git): mitigado via branch-per-task + merge sequencial + pause-world em conflito, INDEPENDENTEMENTE do número de workers.
