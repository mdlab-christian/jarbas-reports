Estratégia de Correção — 2 Problemas, 2 Soluções
✅ Problema 1: Empresa Ré
VIA API AUTOMÁTICO
A ADVBOX não tem campo opposing_party real. O workaround confirmado: usar o campo notes via PUT para registrar as partes corretas. Isso é legível, auditável e reversível.

Formato padrão: Parte Autora: [NOME] | Parte Contrária: [EMPRESA]

Alternativa mais robusta: criar um customer novo na ADVBOX com o nome da empresa e vinculá-lo ao processo com origin: PARTE CONTRÁRIA via POST /lawsuits (recriação)... mas isso exige DELETE primeiro (não disponível). Então o caminho é notes.
PUT /lawsuits/{id} { "notes": "Parte Autora: KARINA PAMELA SANTOS | Parte Contrária: MERCADO PAGO INSTITUICAO DE PAGAMENTO LTDA" }
⚠️ Problema 2: Cliente Errado
MANUAL — API NÃO SUPORTA
A API ADVBOX não tem endpoint para trocar o customer vinculado a um processo existente. As opções são:

Opção A — Manual via interface web ADVBOX: João Victor acessa cada processo e troca o cliente manualmente. Com a lista dos 61 processos, dá ~60 cliques.

Opção B — Playwright: Automação via browser simulando login e edição. Mais complexo, mas 100% automático.

Opção C — Deletar e recriar: DELETE não disponível na API pública (405). Inviável.
# Inviável via API: PUT /customers/:id → 405 Method Not Allowed PUT /lawsuits/:id + customers_id → aceita mas não troca # Viável: Playwright login → /processos → editar → trocar cliente
