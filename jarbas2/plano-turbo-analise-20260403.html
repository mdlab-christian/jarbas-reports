MERGE

local · zero custo
Decisao de roteamento: Se Triador classifica como simples → pula Analista + Financeiro (so registra). Se complexo ou detecta valores → chama Financeiro apos Analista. Economia de ~40% no custo total.

Otimizacoes para Sistema Perfeito

1. Prompt Caching Automatico

System prompts dos 3 agentes sao estaticos (~15k tokens). Com caching da Anthropic API, input de sistema cai de $3/M para $0.30/M — economia de 90% no custo fixo.

Economia: ~48%
2. Triador como Filtro Inteligente

Triador classifica ~30-50% como "simples" (juntadas, certidoes, ruido). Esses NAO passam pelo Analista — economia direta de 30-50% em chamadas.

Economia: ~35%
3. Batch Processing (5 paralelos)

Processar 5 intimacoes em paralelo com Promise.allSettled(). Reduz tempo total de ~3h para ~40 min. Rate limit da API: 50 req/min.

Velocidade: 4.5x
4. Retry com Escalacao

Se Analista retorna confianca < 0.60, retry automatico com texto completo (sem truncar) e contexto de processo enriquecido. Max 1 retry.

Qualidade: +15%
5. Contexto de Processo

Para cada intimacao, buscar dados do processo (tipo, valor_causa, status, advogado_reu) e injetar no prompt do Analista. Melhora roteamento de responsavel e fase.

Precisao: +20%
6. Deduplicacao por Signature

Hash MD5 do conteudo evita re-analise de duplicatas. Constraint UNIQUE ja existe: (signature_hash, organizacao_id). Zero custo em re-runs.

Economia: duplicatas

Sistema a Prova de Falhas

Retry com Backoff

Falha na API? Retry com backoff exponencial (1s, 2s, 4s). Max 3 tentativas. Se falhar, marcar ia_status='erro' e seguir.

Validacao de Output

JSON.parse() obrigatorio. Se resposta nao e JSON valido, retry. Se retry falha, usar output do Triador como fallback.

Checkpoint a Cada 50

Salvar progresso em /tmp/turbo_checkpoint.json a cada 50 registros. Se script crashar, retomar do checkpoint.

Schema Validation

Validar que output do Analista tem campos obrigatorios (flag_ciencia_renuncia, atividade_tipo, resumo_controller, confianca). Se faltar campo, usar default seguro.
