Seção 8 — Perguntas para Christian

Questão 1 — TipTap: JSON ou HTML no banco
O TipTap pode salvar o conteúdo como JSON nativo (melhor para manipulação programática e diff entre versões) ou como HTML serializado (retrocompatível com o campo conteudo_html atual). A proposta é salvar JSON no campo conteudo_tiptap e manter conteudo_html em paralelo por compatibilidade, deprecando depois. Confirmar esta estratégia de dupla gravação durante a migração?
Questão 2 — Style Guides: trigger automático ou manual
A regeneração de Style Guides pode ser: (A) manual apenas — advogado clica [Regenerar] quando quiser; (B) automático após N petições aprovadas (ex: a cada 10 novas aprovações, roda em background via n8n). A opção B é mais elegante mas adiciona complexidade. Qual prefere para o V3?
Questão 3 — Reportagens: importação via URL
No modal de Nova Reportagem, seria útil um botão [Importar da URL] que usa um scraper simples (Edge Function + fetch + cheerio) para preencher título, fonte e texto automaticamente a partir de um link. Adiciona ~2h à Fase 6. Vale o esforço para o V3 ou deixa para V3.1?
Questão 4 — Busca de Jurisprudências: autenticação no tribunal
O portal público de jurisprudência do TJRS (e outros) às vezes exige CAPTCHA em buscas intensivas. O script Playwright pode usar a OAB do Christian (RS116571) para fazer login no EPROC antes de buscar, evitando CAPTCHA. Autoriza uso da OAB RS116571 como sessão de busca de jurisprudências?
Questão 5 — Permissões: quem pode criar/editar na /playbooks
A página atual não tem controle de permissão granular. Na V3, os perfis juridico e admin poderiam ter níveis diferentes: (A) ambos criam/editam tudo; (B) juridico cria blocos e reportagens, mas só admin publica modelos e playbooks. Qual modelo de permissão para /playbooks?
Questão 6 — Sequência de entrega: tudo junto ou por grupo
Posso entregar a refatoração em uma única sessão grande (~36h de trabalho técnico) ou em 3 PRs menores: PR1 — Limpeza + Backend + Redesign Visual (Fases 1-3), PR2 — TipTap + Style Guides + Reportagens (Fases 4-6), PR3 — Busca Juris + Edge Functions + QA (Fases 7-9). Prefere entregas incrementais ou big-bang?
