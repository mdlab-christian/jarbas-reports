Como Funciona o Monitoramento

Pipeline S0-S9 por OAB

2.1 Visao Geral

3 ciclos/dia (6h, 13h, 19h) no Mac Mini. Cada ciclo processa as 5 OABs em sequencia. Para cada OAB, 10 estagios (S0-S9).

2.2 Pipeline por OAB

S0
PLAN
Carrega vault (4 camadas: Core + Prompt + Playbooks + Memoria)
Verifica cecilia_capture_log para cada OAB
Calcula janela temporal: ultimo ciclo OK = 24h | ultimo ciclo FALHOU = 48h (FALLBACK)
Define lista de OABs + ordem de prioridade
S1
LOGIN
Abre navegador headless (Playwright)
Navega para EPROC da OAB (TJRS/TJSC/TJSP)
Preenche usuario + senha de jarbas_accounts no Supabase
Gera TOTP 2FA automaticamente via otplib
S2
SCRAPE
Acessa "Ultimas Movimentacoes" da OAB
Baixa XLS com todos os processos com movimentacao
Parsea XLS — lista de CNJs + timestamps
S3
FILTER
Aplica janela temporal (24h ou 48h fallback)
Deduplicacao SHA256(cnj + evento + data)
eventos-ruido.md — tipo_acao = 'ruido' sem IA (0 tokens)
Keywords Classe A — pre_classificado_completo = true
Restante entra para S4-S6
S4
NAVIGATE
Abre processo via hash de sessao (NUNCA URL direta)
Modo CronJob = Lista de Eventos | Modo on-demand = Arvore
Verifica PRAZO ABERTO / PRAZO FECHADO
S5
READ_DOCS
Baixa documentos vinculados ao evento
Classes C-F: extrai texto completo do PDF
Timeout por documento: 30s
Se vazio: erro_s5_vazio, pula S6, salva como pendente para retry
S6
ANALYZE Claude Sonnet 4.6 + RAG
Gera embedding do conteudo (gemini-embedding-2, 768 dims)
Busca top-5 casos similares no RAG (vector_cosine_ops + HNSW)
Monta prompt: vault Core + playbooks + top-5 exemplos RAG
Chama Claude Sonnet 4.6 (temp 0.1, retry com backoff)
Retorna: tipo_acao, urgente, prazo, atividade_sugerida, confianca
S7
PERSIST
Upsert em intimacoes (organizacao_id + cnj + evento_hash)
Campos: tipo_acao, urgente, analise_ia, conteudo_completo, modelo_ia, custo_ia
pipeline_status = 'concluido' | 'erro_s5' | 'pendente'
Atualiza cecilia_capture_log com timestamp do ciclo
S8
ACT
urgente_financeiro — Telegram imediato (Joao + Christian)
prazo_recursal com urgente=true — Telegram com prazo
Cria atividade_sugerida no MdFlow (1-click para Joao aprovar)
Processo em SP — verifica processo relacionado 2 grau
S9
REPORT
Resume ciclo: X processos verificados, Y intimacoes novas, Z urgentes
Ciclo das 19h: BRIEFING DIARIO com custo total + breakdown por OAB
Registra log no Supabase (cecilia_ciclos)

2.3 Sistema de Fallback 2 Dias

Problema que resolve: SP535101 tinha watchdog de 2h que matava o ciclo antes de terminar. No dia seguinte, movimentacoes perdidas nunca eram recuperadas.
