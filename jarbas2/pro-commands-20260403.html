Comandos MacBook Pro — v2

RODAR NO TERMINAL NATIVO (nao no Claude Code). Um de cada vez. Espere terminar antes do proximo.

O Claude Code do Pro tentou rodar tudo junto e deu problema. Rode esses comandos direto no Terminal.app, um por vez.
Parte A — Corrigir SSH Config
1
Reescrever SSH config completo
Apaga o config atual (corrompido) e cria um novo limpo com IdentitiesOnly yes (impede que o SSH tente outras chaves).
printf "Host air\n  HostName 192.168.0.149\n  User christian\n  IdentityFile ~/.ssh/id_ed25519_air\n  IdentitiesOnly yes\n  StrictHostKeyChecking accept-new\n  AddKeysToAgent yes\n" > ~/.ssh/config && chmod 600 ~/.ssh/config
2
Verificar config
Deve mostrar Host air com IdentitiesOnly yes.
cat ~/.ssh/config
Deve mostrar: Host air / HostName 192.168.0.149 / User christian / IdentityFile ... / IdentitiesOnly yes
3
Testar SSH com o Air
Agora sim deve conectar. Se pedir fingerprint, digite "yes".
ssh air "hostname && whoami && echo SSH_OK"
Esperado: Macbook-air---Jarbas-OS.local / christian / SSH_OK
Parte B — Tailscale
O Tailscale ja foi instalado e o servico iniciado. Falta autenticar. O comando abaixo vai abrir uma URL no browser.
4
Autenticar no Tailscale
Vai mostrar uma URL. Abra no browser, faca login com sua conta e autorize.
tailscale up
5
Verificar Tailscale
Deve mostrar jarbas-macbookair (100.101.208.82) na lista.
tailscale status
Esperado: 100.101.208.82 jarbas-macbookair na lista
6
Adicionar entrada Tailscale ao SSH config
Adiciona host "air-tail" que funciona de qualquer rede.
printf "\nHost air-tail\n  HostName 100.101.208.82\n  User christian\n  IdentityFile ~/.ssh/id_ed25519_air\n  IdentitiesOnly yes\n  StrictHostKeyChecking accept-new\n  AddKeysToAgent yes\n" >> ~/.ssh/config
7
Testar SSH via Tailscale
Conexao via VPN — funciona mesmo fora da rede local.
ssh air-tail "hostname && whoami && echo TAILSCALE_OK"
Esperado: Macbook-air---Jarbas-OS.local / christian / TAILSCALE_OK
Parte C — Testar Claude Code Remoto
8
Verificar Claude Code no Air via SSH
Confirma que o Claude Code do Air responde via SSH.
ssh air "claude --version"
Esperado: versao do Claude Code (ex: 2.1.89 ou similar)
