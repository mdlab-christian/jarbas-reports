Objetivo & Motivação

📋 O que foi construído

Mecanismo de importação manual de processos no /controller. O usuário faz upload de planilhas CSV/XLSX com números CNJ, associa cada planilha a um advogado (OAB) via dropdown, seleciona uma data de corte e o sistema — via Cecilia — acessa o EPROC, analisa cada processo com IA e alimenta a fila com as movimentações relevantes (classes B–G).

💡 Por que foi construído

O cron autônomo da Cecilia estava instável. Este mecanismo é o backup definitivo — quando o cron falha, o usuário importa a lista exportada do software jurídico e o sistema processa completamente. Zero dependência do cron para alimentar o controller.

✅ Escopo incluído

  • Botão "Importar" no header do /controller (ao lado de "Exportar")
  • Modal 3 steps: upload múltiplo + dropdown OAB + date picker → preview + validação → progress
  • Tabelas backend: controller_importacoes + controller_importacao_itens
  • Hook useImportacaoProgress com Supabase Realtime
  • ControllerImportProgress — progress panel inline persistente no /controller
  • skill_importar_lote.mjs (382 linhas) — poll + login + dedup + IA + INSERT
  • start.mjs — setInterval de 30s para poll automático de jobs pendentes
  • Normalização CNJ: formatado e bruto → formato canônico
  • Deduplicação via signature_hash
  • Fallback de erro EPROC → INSERT com ia_status='erro'

Fluxo Cecilia — skill_importar_lote.mjs

Step Ação Resultado
1 Poll Supabase a cada 30s SELECT controller_importacoes WHERE status='pendente' LIMIT 1
2 Tomar job UPDATE status='processando', SELECT itens pendentes
3 Agrupar por OAB Map: { RS116571: [cnj1, cnj2], SC074025: [cnj3] }
4 Para cada OAB: loginEproc(oab) Reutiliza skill_login_eproc.mjs (TOTP + otplib)
5 Para cada CNJ: skillAcessarProcesso Retorna eventos desde data_inicio. Delay 800–2500ms anti-bot.
6 Deduplicação signature_hash Skip se já existe em intimacoes → ja_existentes++
7 skillAnalisarIA(eventosNovos, { oab }) Classifica A-G. Skip classe A (irrelevante).
8 INSERT intimacoes fonte='importacao_manual', provider='cecilia_v3'. Classes B-G.
9 UPDATE item + contadores status='concluido', intimacao_id=uuid, processados++
10 Fallback erro EPROC INSERT intimacao ia_status='erro', UPDATE item status='erro', erros++
11 Finalizar job UPDATE importacao status='concluido'. Delay 5s entre OABs.

🔗 Skills Cecilia Reutilizadas

  • skill_login_eproc.mjs — Login com TOTP (otplib) para cada OAB
  • skill_acessar_processo.mjs — CNJ + desdeTimestamp → eventos EPROC
  • skill_analisar_ia.mjs — Haiku triagem A-G → Sonnet D/E/F análise completa
  • buildSignatureHash — SHA256(cnj|evento|data) para deduplicação

Observações Técnicas

⚠️ importacao_id não existe em intimacoes

A spec original mencionava um campo importacao_id na tabela intimacoes, mas verificação via Supabase MCP confirmou que o campo não existe. Traceabilidade implementada via controller_importacao_itens.intimacao_id (reverse lookup: dado uma intimação, achar o item de importação que a criou).

⚠️ ControllerImportModal.tsx — 801 linhas

O arquivo ultrapassou o limite de 500 linhas (P-10 CLAUDE.md). Refatoração em sub-componentes está pendente para tarefa futura. O accordion responsivo em <640px também foi diferido por depender desta refatoração.

✅ LaunchAgent Cecilia — ai.cecilia.skills

O label correto do LaunchAgent é ai.cecilia.skills (não com.cecilia.agent). Restart: launchctl kickstart -k gui/506/ai.cecilia.skills no Mac Mini. Cecilia confirmada rodando: status=ok, poll interval ativo.
