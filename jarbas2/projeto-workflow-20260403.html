Origem: Video da Deborah Folloni

🎬 "Vibe Coding nao funciona (Novo Workflow no Claude Code)" — 24min

Deborah mostra como criar aplicacoes profissionais com Claude Code usando um workflow de 4 etapas que resolve os problemas classicos do vibe coding.

❌ Problema 1: IA engasga

Tarefas grandes lotam a context window e o modelo falha

✅ Solucao: Quebrar em tarefas

Spec em tarefinhas pequenas, 1 por vez, com /clear entre elas

❌ Problema 2: Codigo bagunca

IA reinventa a roda, duplica codigo, complica o simples

✅ Solucao: Planejar antes

Buscar codigo reutilizavel + padroes comprovados antes de implementar

❌ Problema 3: IA desobediente

Sem saber quais arquivos mexer, IA modifica os errados

✅ Solucao: Plano explicito

Listar cada arquivo + acao exata. IA segue o plano estritamente

❌ Problema 4: Cobertor de pobre

Arruma uma coisa, quebra outra por falta de isolamento

✅ Solucao: Arquitetura + isolamento

Docs de arquitetura + tarefas isoladas por comportamento

Exemplo de Uso

# Sessao 1: Criar spec
/projeto spec "Sistema de notificacoes push com preferencias"
→ Spec criada em .projeto/spec.md (React + Supabase detectado)
→ Revise e ajuste. Quando pronta: /clear + /projeto break

# Sessao 2: Quebrar em tarefas
/projeto break
→ 6 tarefas criadas em .projeto/tasks.md
→ T01: Migration tabela notificacoes (S)
→ T02: RPC buscar_notificacoes (S)
→ T03: Componente NotificationBell (M)
→ T04: Painel de preferencias (M)
→ T05: Webhook push via n8n (M)
→ T06: Loading/empty states (S)

# Sessao 3: Planejar T01
/projeto plan T01
→ Encontrou: tabela processos como referencia de pattern
→ Plano: criar migration, RLS modelo A, indices
→ Salvo em .projeto/plans/task-T01.md

# Sessao 4: Executar T01
/projeto execute T01
→ Migration criada. Build OK. T01 concluida (1/6 = 16%)
→ Proximo: /clear + /projeto plan T02

# Checar progresso a qualquer momento
/projeto status
→ Progresso: 1/6 (16%) | Proxima: T02 — RPC buscar_notificacoes

Por que e Perfeito pro Caso Christian

Multi-Projeto: MdFlow, Skills, Automacoes, Clientes

Christian trabalha em 4+ contextos diferentes simultaneamente. O /projeto funciona em qualquer um deles sem configuracao extra — detecta stack, le CLAUDE.md, e adapta.

Substitui SDD + Gerenciador sem perder nada

O /sdd era bom mas nao trackava progresso. O gerenciador-projetos trackava mas precisava de Supabase. /projeto combina o melhor dos dois: tracking local + workflow estruturado + /clear entre fases.

Portavel entre maquinas

A skill fica em ~/.claude/skills/ (synca com o setup). Os .projeto/ ficam no repo de cada projeto. MacBook Air, Mac Mini, qualquer maquina com Claude Code tem acesso.

Anti-vibe-coding enforced

O workflow FORCA a disciplina: nao da pra pular direto pro execute sem ter spec+tasks+plan. Cada fase gera um arquivo que a proxima fase consome. Sem atalhos.
