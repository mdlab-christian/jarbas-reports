Objetivo

Criar uma tabela jarbas_cost_log no Supabase para registrar cada evento de custo da rede: crons, heartbeats, sessões interativas, Claude Code, subagentes e qualquer chamada de API que consuma tokens.

Com isso, teremos um dashboard real de custo por agente, por dia, por tipo de evento — identificando automaticamente o que está queimando dinheiro.

Schema — jarbas_cost_log
CREATE TABLE jarbas_cost_log (
  id                  BIGSERIAL PRIMARY KEY,
  created_at          TIMESTAMPTZ NOT NULL DEFAULT NOW(),

  -- Quem gerou o custo
  agent_id            TEXT NOT NULL,      -- 'jarbas2', 'jana', 'olivia', 'cecilia', 'beto', 'cc'
  agent_host          TEXT,               -- 'macbook-air', 'mac-mini'

  -- O que foi
  event_type          TEXT NOT NULL,      -- 'session', 'cron', 'heartbeat', 'subagent', 'claude_code', 'api_call'
  event_name          TEXT,               -- nome do cron, nome do subagente, etc
  session_id          TEXT,               -- UUID da sessão OpenClaw (quando aplicável)

  -- Modelo e tokens
  model               TEXT,              -- 'claude-sonnet-4-6', 'claude-haiku-4-5', 'gpt-4o', etc
  input_tokens        INTEGER DEFAULT 0,
  output_tokens       INTEGER DEFAULT 0,
  cache_read_tokens   INTEGER DEFAULT 0,
  cache_write_tokens  INTEGER DEFAULT 0,
  total_tokens        INTEGER GENERATED ALWAYS AS (input_tokens + output_tokens) STORED,

  -- Custo calculado
  cost_usd            NUMERIC(10,6),     -- custo em USD
  cost_brl            NUMERIC(10,4),     -- custo em BRL (USD × taxa do dia)

  -- Contexto da operação
  context_tokens      INTEGER,           -- tamanho do contexto no momento
  duration_ms         INTEGER,           -- duração da operação
  status              TEXT DEFAULT 'ok', -- 'ok', 'error', 'timeout'
  error_msg           TEXT,

  -- Metadados livres
  meta                JSONB DEFAULT '{}'
);

-- Índices para queries rápidas
CREATE INDEX idx_cost_log_agent   ON jarbas_cost_log (agent_id, created_at DESC);
CREATE INDEX idx_cost_log_event   ON jarbas_cost_log (event_type, created_at DESC);
CREATE INDEX idx_cost_log_date    ON jarbas_cost_log (DATE(created_at));
CREATE INDEX idx_cost_log_model   ON jarbas_cost_log (model, created_at DESC);

Queries de dashboard

Custo por agente — hoje

SELECT
  agent_id,
  COUNT(*) AS eventos,
  SUM(total_tokens) AS total_tokens,
  SUM(cost_usd) AS custo_usd,
  SUM(cost_brl) AS custo_brl
FROM jarbas_cost_log
WHERE DATE(created_at) = CURRENT_DATE
GROUP BY agent_id
ORDER BY custo_usd DESC;

Custo por tipo de evento — últimos 7 dias

SELECT
  event_type,
  event_name,
  COUNT(*) AS execucoes,
  AVG(total_tokens)::INT AS avg_tokens,
  SUM(cost_usd) AS total_usd
FROM jarbas_cost_log
WHERE created_at >= NOW() - INTERVAL '7 days'
GROUP BY event_type, event_name
ORDER BY total_usd DESC
LIMIT 20;

Top 5 sessões mais caras da semana

SELECT
  agent_id, session_id, model,
  total_tokens, cost_usd, cost_brl,
  duration_ms / 1000 AS duracao_s
FROM jarbas_cost_log
WHERE event_type = 'session'
  AND created_at >= NOW() - INTERVAL '7 days'
ORDER BY cost_usd DESC
LIMIT 5;

Rodar o SQL de criação via Management API ou Supabase Studio. Criar também jarbas_fx_rates para taxas de câmbio diárias.

2

Cron de coleta diária de sessions P0 · 30min

3

Adicionar ao payload de cada cron: ao final da execução, inserir registro em jarbas_cost_log com tokens usados e custo estimado. Via n8n webhook para não depender do modelo.

4

Dashboard MdFlow ou HTML diário P2 · futuro

Integrar ao relatório diário das 23h ou criar página dedicada no MdFlow. Alerta automático se projeção do dia ultrapassa threshold definido (ex: $20/dia).
