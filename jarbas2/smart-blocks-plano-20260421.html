Seção 1 — O que é o Smart Blocks

Conceito

Modo manual e determinístico de geração de petições na OlivIA. O advogado monta a petição selecionando blocos de texto pré-aprovados — sem IA.

Vantagem central: previsibilidade total, velocidade e blocos já aprovados juridicamente pela equipe. Ideal para petições recorrentes e simples onde a variação da IA é indesejada.

Fluxo de 4 Steps (CTO 2026-04-21: step 5 unificado com step 4)
1 Identificação — processo, tipo, modelo, versão
2 Blocos + Campos — seleção manual + auto-preenchimento
3 Editor TipTap — revisão e edição livre
4 Download + Registro — unificados (step 5 absorvido)

Step 1 → INSERT olivia_smart_blocks (advogado_id, versao_id...) Step 2 → UPDATE olivia_smart_blocks (blocos_selecionados, campos_preenchidos) Step 3 → POST /smart-blocks/montar monta HTML inline → salva editor_html Step 3 → TipTap edita HTML (autosave 30s) Step 4 → POST /smart-blocks/gerar-doc HTML → DOCX/PDF via Puppeteer Step 4 → Download + Registro (observacao, clonar, arquivar)
Engine de montagem — /smart-blocks/montar (a criar no olivia-server)
async function montarSmartBlock(smart_block_id, organizacao_id) { // 1. Buscar sessão + template base const session = await supabase .from('olivia_smart_blocks') .select(`*, modelo_base:olivia_modelos_peticao!modelo_base_id(conteudo_template_html_rico)`) .eq('id', smart_block_id).single(); // 2. Se já tem editor_html → rascunho retomado, retorna sem remontar if (session.editor_html) return res.json({ html: session.editor_html }); // 3. Template base (HTML com {{TAG:codigo}} como marcadores de seção) let html = session.modelo_base.conteudo_template_html_rico ?? ''; // 4. Para cada bloco selecionado: for (const bloco of session.blocos_selecionados) { const { data: blocoData } = await supabase .from('olivia_blocos_texto') .select('conteudo_html_rico, juris_ids, titulo') .eq('id', bloco.bloco_id).single(); const { data: tagData } = await supabase .from('olivia_tags').select('codigo').eq('id', bloco.placeholder_id).single(); const marcador = `{{TAG:${tagData?.codigo || bloco.placeholder_id}}}`; html = html.replace(marcador, blocoData.conteudo_html_rico || ''); if (blocoData.juris_ids?.length) { const { data: juris } = await supabase .from('olivia_jurisprudencias') .select('ementa, tribunal, numero') .in('id', blocoData.juris_ids); html += renderJuris(juris); } } // 5. Substituir placeholders de dados const campos = { ...session.campos_preenchidos, ...session.overrides_manuais }; for (const [key, value] of Object.entries(campos)) { html = html.replaceAll(`{{${key}}}`, value || ''); } // 6. Salvar no banco await supabase.from('olivia_smart_blocks') .update({ editor_html: html }).eq('id', smart_block_id); return { html }; }

Migration 1 — Corrigir FK advogado_id
-- Corrigir FK errada -- (causa do erro na tela) ALTER TABLE olivia_smart_blocks DROP CONSTRAINT olivia_smart_blocks_advogado_id_fkey; ALTER TABLE olivia_smart_blocks ADD CONSTRAINT olivia_smart_blocks_advogado_id_fkey FOREIGN KEY (advogado_id) REFERENCES advogados(id);
RPC 2 — fn_olivia_smart_blocks_preencher_campos
CREATE OR REPLACE FUNCTION public.fn_olivia_smart_blocks_preencher_campos( p_processo_id UUID, p_advogado_id UUID, -- advogados.id p_org_id UUID ) RETURNS JSONB LANGUAGE plpgsql SECURITY DEFINER AS $$ DECLARE v_result JSONB; BEGIN SELECT jsonb_build_object( 'advogado_nome', adv.nome, 'advogado_oab', adv.oab, 'advogado_oab_uf', adv.oab_uf, 'advogado_email', adv.email, 'cliente_nome', cli.nome, 'cliente_cpf', cli.cpf, 'cliente_rg', cli.rg, 'cliente_email', cli.email, 'cliente_uf', cli.estado, 'processo_cnj', pr.numero_cnj, 'processo_vara', pr.vara, 'processo_comarca',pr.comarca, 'valor_causa', pr.valor_causa::text, 'empresa_nome', emp.nome, 'empresa_cnpj', emp.cnpj, 'data_hoje', to_char(NOW(),'DD "de" TMMonth "de" YYYY'), 'estado_extenso', CASE cli.estado WHEN 'RS' THEN 'Rio Grande do Sul' WHEN 'SC' THEN 'Santa Catarina' WHEN 'SP' THEN 'Sao Paulo' ELSE cli.estado END ) INTO v_result FROM processos pr LEFT JOIN clientes cli ON cli.id = pr.cliente_id LEFT JOIN advogados adv ON adv.id = p_advogado_id LEFT JOIN empresas emp ON emp.id = pr.empresa_id WHERE pr.id = p_processo_id AND pr.organizacao_id = p_org_id; RETURN COALESCE(v_result, '{}'::jsonb); END; $$;

Seção 9 — Ideias para Smart Blocks 2.0

1
Preview inline antes do Editor
Botao "Pre-visualizar" no Step 2 monta HTML com blocos selecionados ate agora (sem salvar) e exibe em modal. Advogado ve como ficara antes de ir para Step 3.
2
Sugestao automatica de blocos
Ao abrir uma tag, destacar o bloco "mais usado" para aquele tipo + categoria. Badge "Mais usado" via contador de uso em blocos_selecionados.
3
Clonar para novo processo
No Step 4, alem de clonar como nova versao, botao "Reutilizar para outro processo" — abre modal, seleciona novo processo, recria sessao com mesmos blocos e novos dados.
4
Indicador de completude
Badge de progresso no topo do Step 2: "5/7 topicos preenchidos (2 obrigatorios pendentes)". Botao Avancar bloqueado ate obrigatorios estarem preenchidos.
5
Jurisprudencias por bloco expandivel
Ao selecionar um bloco, expandir painel com jurisprudencias vinculadas (juris_ids) com opcao de remover individualmente. Controle granular pelo advogado.
6
Rascunhos visiveis na listagem
Card na pagina /olivia mostrando "3 Smart Blocks em rascunho" com link para retomar. Query: olivia_smart_blocks WHERE status='rascunho' AND advogado_id=atual.
7
Historico integrado ao /olivia
Smart Blocks aparecem na mesma lista de historico do VibeLaw, com badge "Manual" para diferenciar. modo = 'smart_blocks' vs 'vibelaw'.
