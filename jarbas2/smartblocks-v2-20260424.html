Bug 1 — Step 3 em branco

P0 CRITICO
Causa Raiz
assembleHtml.ts linha 28 faz .select('conteudo_html') — coluna NAO EXISTE
Coluna Real
olivia_modelos_peticao.conteudo_template_html_rico (TEXT, nullable)
Impacto
Quando Mac Mini nao esta acessivel (Lovable preview), fallback local assembleHtmlLocally() busca conteudo_html → retorna null → templateHtml = '' → peticao em branco
Mac Mini acessivel
smart-blocks-montar.mjs tambem usa a coluna errada — verificacao necessaria
Consequencia Adicional
Substituicao {{TAG:codigo}} nunca ocorre — templates usam {{campo_nome}}, nao {{TAG:codigo}}. Blocos devem ser appendados apos o template em ordem.
🚨

Bug 2 — Step 4 "Failed to fetch"

P0 CRITICO
Causa
handleStep4Generate faz fetch('http://localhost:19100/smart-blocks/gerar-doc', ...)
Problema
localhost:19100 = Mac Mini. Do Lovable preview (nuvem), esse endereco nao existe → CORS block + network failure → "Failed to fetch"
Como VibeLaw Resolve
Usa supabase.functions.invoke('olivia-criar-doc', ...) → Edge Function chama Mac Mini internamente via Tailscale → resultado persiste em olivia_documentos_gerados → frontend recebe via Realtime
Fix Necessario
Criar Edge Function smart-blocks-gerar-doc (ou estender olivia-criar-doc) para fazer bridge Mac Mini <→ nuvem
🔍

Descobertas — Template Format

Template conteudo_template_html_rico — Formato Confirmado
<!-- Templates usam {{campo_nome}} — NAO {{TAG:codigo}} --> <p>{{advogado_enderecamento}}</p> <p>Autos de origem n°: {{numero_processo}}</p> <p>A parte exequente, {{parte_autora}}, {{qualificacao_sexo}}...</p> [corpo da peticao]
💡
Tags de bloco como olivia_tags.codigo (ex: preliminares_2a05db7d, dano_moral_7493ef36) identificam blocos de conteudo que devem ser appendados em ordem (sort por `ordem`) apos o template, antes do </body> ou no final do HTML. Substituicao de campos {{campo_nome}} deve ocorrer antes de appendar os blocos.
🛠
