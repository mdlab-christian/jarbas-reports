Como Usar

Cenario 1: Trabalhar no MdFlow pelo Pro

  1. Abrir CC no Pro:
cd ~/beta-mdflow && claude
  1. Trabalhar normalmente — agents/skills/memory ja estao disponiveis via symlinks
  2. Fazer commit e push:
git push origin main
  1. No Air, fazer pull se necessario:
git pull --rebase

Cenario 2: Comandar o Air remotamente do Pro

ssh air-tail "cd ~/beta-mdflow && git status" ssh air-tail "cd ~/beta-mdflow && claude -p 'roda /status'"

Cenario 3: Command-bus (async)

  1. No Pro, criar arquivo em ~/jarbas-vault/10_sync-macs/command-bus/inbox/
  2. Obsidian Sync propaga para Air (5-60s)
  3. Air processa e responde em outbox/

Cenario 4: Dois CCs simultaneos

  • Pode rodar CC em ambos ao mesmo tempo
  • Agents/skills/memory compartilhados em tempo real via Obsidian Sync
  • Codigo: coordenar via git (push/pull antes de editar)
  • NAO editar mesmo arquivo simultaneamente
