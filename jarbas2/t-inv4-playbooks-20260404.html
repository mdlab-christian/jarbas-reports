Q-02 · Reportagens
Reportagens — scraping automatico (n8n cron) ou entrada manual via form?
Impacta escopo da Fase 4. Scraping requer n8n workflow adicional + URL do scraper. Manual e mais simples porem menos util.
Q-03 · Aprovacoes
TabDetalhesAprovacoes — manter como read-only historico ou deletar junto?
Modal de 194L. Se ja existem aprovacoes no banco de dados anteriores ao descope, pode ser util manter para consulta historica.
Q-04 · Tipos Peticao
Campo modelo_ia em olivia_tipos_peticao — expor no form de edicao?
Campo existe no tipo TypeScript e no banco mas o form atual nao permite edita-lo. Define prompt base para geracao de peticoes daquele tipo.
Q-05 · D11 Self-Improving
Trigger do D11 — automatico pos-aprovacao, manual pelo advogado, ou ambos?
Automatico e mais simples mas pode gerar chamadas excessivas de Edge Function. Manual requer botao "Atualizar guide" na UI.
Q-06 · Gerador IA
Migrar Gerador IA para Edge Function agora (Fase 5) ou deixar para pos-MVP V3?
Webhooks n8n funcionando hoje. Migracao para Edge Function com Sonnet 4.6 e tecnicamente superior mas adiciona escopo. Pos-MVP e mais conservador.
