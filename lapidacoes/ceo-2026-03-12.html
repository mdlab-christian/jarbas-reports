Problemas Encontrados e Corrigidos

P0 — Design System (violações absolutas)

  • tab_knowledge_vault.tsx:23 — typeLabel usava emojis (📚🎤📋📄📁) como ícones. DS proíbe qualquer ícone fora do Lucide React.
    Fix: Substituído por BookMarked, Mic, ClipboardList, FileText, FolderOpen com cores chart tokens.
  • tab_board.tsx:130,144 — strings de texto ⚠ e → em <span> como marcadores visuais. DS requer Lucide exclusivamente.
    Fix: Substituído por <TriangleAlert /> e <ArrowRight />.

P1 — UX / Lógica

  • ceo/index.tsx:130 — MacroTabs com TabsList className="w-full justify-start overflow-x-auto h-9" — faltava border, bg-muted/50, rounded-lg conforme padrão DS canonical.
    Fix: bg-muted/50 border border-border rounded-lg p-1 h-auto gap-1 em todos os TabsList.
  • ceo/index.tsx — TabsTrigger — faltava data-[state=active]:bg-primary data-[state=active]:text-primary-foreground para active state visual correto.
    Fix: Constante TRIGGER_CLS aplicada em todos os triggers.
  • tab_juridico.tsx:31 — Query intimacoes.limit(1) com head: false retornava apenas 1 registro, então o filter de ia_status === 'pendente' só poderia resultar em 0 ou 1 (incorreto para muitas intimações urgentes).
    Fix: Duas queries separadas — uma para count total (head:true) e outra para ia_status=pendente count.
  • tab_juridico.tsx:35 — Query decisoes sem .is('deleted_at', null).
    Fix: Adicionado filtro de soft delete.
  • use_lolito_equipe.ts:28,30,33 — as any[], as any explícitos (2 ocorrências). CLAUDE.md proíbe any explícito no TypeScript.
    Fix: Criadas interfaces AtividadeRow e UserRow. Cast via as unknown as UserRow[]. Também corrigido bug: .sort() mutava o array original — agora usa [...atividades].sort().

P2 — Code Quality

  • tab_equipe_tempo_real.tsx:47 — Template literal ${condition ? 'class1' : 'class2'} para Tailwind classes (risco de purging em builds otimizados).
    Fix: Substituído por cn('base', condition1 && 'class1', condition2 && 'class2').
  • tab_relatorios.tsx:59,69 — Template literals em Icon className e Badge className.
    Fix: Substituído por cn() em ambos os casos.
  • ceo/index.tsx:259 — onAddDocument={() => {/* modal upload */}} era um noop silencioso — usuário clicava e nada acontecia.
    Fix: Emite toast.info('Upload de documentos em breve', ...) com feedback adequado.
  • ceo/index.tsx:71 — const [_selectedKrObjectiveId, setSelectedKrObjectiveId] — variável prefixada com _ indicando unused. Corrigido para const [, setSelectedKrObjectiveId] com comentário de PENDÊNCIA (modal KR não implementado).
