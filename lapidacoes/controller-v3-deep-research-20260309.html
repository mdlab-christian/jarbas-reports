Seção 3 — Revisão Específica
🔲 Análise: Modal v1 vs Drawer v2 vs Drawer v3
O ponto mais importante do merge — como o usuário interage com uma intimação

❌ Modal Antigo (v1) — Problemas

  • Bloqueava a lista inteira (modal overlay)
  • Não permitia comparar intimações lado a lado
  • Abas aninhadas dentro de modal = UX confusa
  • Largura fixa — sem flexibilidade
  • Sem footer de ações rápidas
  • Sem keyboard navigation (J/K) com modal aberto
  • Sem CopyableCNJ no header
  • ia_acao_recomendada não destacada

✅ Drawer v3 — O Que Melhora

  • Drawer lateral — lista ainda visível à esquerda
  • Draggable: 38% a 82% de largura
  • CopyableCNJ no header (click para copiar)
  • 6 abas organizadas por propósito
  • Footer fixo com 5 ações principais
  • J/K navega entre intimações sem fechar o drawer
  • ia_acao_recomendada em card verde destacado
  • Tempo em curso calculado (anos, meses, dias)
  • Auto-mark como lida ao abrir

🧠 Header do Drawer — CopyableCNJ

Um dos recursos mais pedidos: clicar no número do processo copia direto para o clipboard. Implementado com feedback visual (✓ copiado por 1.5s).

// CopyableCNJ component function CopyableCNJ({ cnj }: { cnj: string | null }) { const [copied, setCopied] = useState(false); const copy = () => { navigator.clipboard.writeText(cnj ?? ''); setCopied(true); setTimeout(() => setCopied(false), 1500); }; return ( <button onClick={copy} className="font-mono text-xs text-muted-foreground hover:text-primary"> {copied ? '✓ copiado' : cnj ?? '—'} </button> ); }

Seção 6
🔍 FilterBar v3 — Novidades

Toggle: Excluir Pauta Julgamento

Christian pediu especificamente este filtro. As intimações com tag "pauta_julgamento" geralmente não precisam de ação imediata — o toggle esconde elas da lista principal, reduzindo o ruído.

// Em ControllerV3FilterBar.tsx excluir_pauta_julgamento: boolean ← novo campo no filtro // Aplicado em ControllerV3Intimacoes.tsx if (filters.excluir_pauta_julgamento) { query = query.not('tags', 'cs', '{"pauta_julgamento"}'); }

Input: Filtro por Empresa

Filtra por empresa_nome — útil para campanhas de acordo (ex: "mostrar só Claro" quando for fazer proposta em massa para um réu específico).

// Filtro aplicado na query empresa_nome: string ← novo campo no filtro if (filters.empresa_nome) { query = query.ilike('empresa_nome', `%${filters.empresa_nome}%`); }
