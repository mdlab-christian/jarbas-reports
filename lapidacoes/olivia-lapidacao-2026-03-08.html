Bugs e Problemas Encontrados
Ordenados por severidade
P0
Wizard não avança de Step 1 → Step 2
O botão "Avançar" chama createHistoricoRecord() que faz INSERT em olivia_historico. O INSERT falha silenciosamente com FK constraint violation. O usuário vê toast "Erro ao salvar rascunho" e fica preso no Step 1 para sempre.
// useWizardTabs.ts linha 309 — payload do INSERT modelo_id: ws.modelo_id, // ← suspeita: pode ser null ou ID inválido versao_id: ws.versao_id, // ← idem advogado_id: ws.advogado_id, // ← idem // Diagnóstico: campo processo_id pode ser NULL quando processo_id // não foi selecionado, violando NOT NULL constraint em olivia_historico
src/hooks/olivia/useWizardTabs.ts:279
💥 Impacto: O gerador de petições IA Assistida está completamente inutilizável. Feature principal do produto.
P1
KPIs "IA Assistida" e "Manual" mostram 0 na landing
Os cards de KPI mostram Total=28, Sucesso=22, mas "IA Assistida=0" e "Manual=0". Claramente o campo modo em olivia_historico não está sendo populado corretamente ou a query está filtrando por um valor diferente do que está salvo no banco.
src/pages/olivia/components/gerador/OliviaGeradorLanding.tsx
📊 Impacto: Métricas do produto totalmente incorretas. Mostra ao usuário que nada foi gerado por IA.
P1
4 rascunhos duplicados do mesmo processo no Recovery Card
A UI mostra 4 cards de recovery idênticos (mesmo CNJ, mesmo step, horas parecidas). São resquícios de testes da sessão de debugging de ontem. Além de serem lixo, revelam um problema de UX: o recovery card exibe N rascunhos individualmente em vez de agrupá-los.
src/pages/olivia/components/gerador/OliviaGeradorLanding.tsx
🎨 Impacto: UX degradada. Usuário real verá múltiplos rascunhos do mesmo processo sem forma de gerenciar.
P1
Fila: coluna "Processo" mostra "Petição avulsa" para maioria
Na tab Fila de Geração, a coluna "Processo" exibe "Petição avulsa" para quase todos os registros em vez do número CNJ ou nome do cliente. Apenas 2 registros PENDENTE mostram o CNJ correto. O join com a tabela de processos não está sendo feito ou está falhando.
src/pages/olivia/components/fila/OliviaFilaTab.tsx:489L
📋 Impacto: Impossível identificar qual petição é de qual processo na fila.
P1
VibeLaw não funcional — workflows n8n ausentes
Os 4 workflows n8n necessários para o VibeLaw (WH-OLI-06 a 09) ainda não foram criados no n8n. O frontend está pronto e tem fallback gracioso, mas ao clicar em "Abrir VibeLaw" o usuário vai receber erros de API assim que interagir.
Pendência: criar WH-OLI-06, 07, 08, 09 no n8n
🚀 Impacto: Feature "NOVO" completamente não operacional. É um diferencial competitivo que não funciona.
P1
ModalHistoricoAnalises criado mas não integrado
O modal existe no código mas o botão "Ver outras análises" no Step 1 mostra um toast placeholder em vez de abrir o modal. O usuário não consegue ver o histórico de análises anteriores ao gerar uma petição.
src/pages/olivia/components/gerador/steps/StepIdentificacao.tsx:439L
📚 Impacto: Contexto de análises anteriores indisponível para o advogado no momento de gerar petição.
P2
Fila: coluna "Tempo" mostra "0m 00s" para petições concluídas
Todos os registros SUCESSO na fila mostram "0m 00s" no campo de tempo de geração. O campo finished_at provavelmente não está sendo populado corretamente ou o cálculo de diff está errado.
src/pages/olivia/components/fila/OliviaFilaTab.tsx
📊 Impacto: Métrica de performance de geração inutilizável.
P2
resolverPlaceholders não implementado
O helper resolverPlaceholders que substitui placeholders como [NOME_CLIENTE] na petição gerada não está implementado. O ModalConfirmacaoPlaceholders existe mas não está integrado. Usuário recebe petição com placeholders literais.
src/pages/olivia/components/gerador/steps/StepConclusao.tsx:354L
📝 Impacto: Petições geradas podem ter placeholders não substituídos. Embaraçoso em produção.
P2
StepRevisao.tsx acima do limite de 500L
StepRevisao.tsx tem 503 linhas (3 acima do limite). Além disso, mais 4 arquivos estão entre 441-489L, beirando o limite: OliviaFilaTab.tsx (489L), OliviaAnaliseProcesso.tsx (474L), OliviaAnaliseDecisao.tsx (456L), StepIdentificacao.tsx (439L).
Anti-Frankenstein: 1 arquivo viola, 4 beirando
