Modal: Novo Modelo de Petição P0 size: lg

Campos — spec ModalNovoModelo8 campos
1 tipo_peticao_id Select obrigatório
Source: olivia_tipos_peticao WHERE ativo=true, display=nome, value=id
→ olivia_modelos_grupo.tipo_peticao_id UUID FK
2 nome TextInput obrigatório
Placeholder: "ex: PI Banco do Brasil - Cartão Consignado"
minLength: 5 maxLength: 200 required
→ olivia_modelos_grupo.nome TEXT NOT NULL
3 descricao Textarea opcional
rows: 3
maxLength: 1000
→ olivia_modelos_grupo.descricao TEXT
4 modo_alcance RadioGroup obrigatório
Controla visibilidade dos campos 5 e 6
1 — Por Categoria de Empresa (padrão) 2 — Por Empresa Específica
→ olivia_modelos_grupo.modo_alcance INT2 DEFAULT 1
5 empresas_categorias MultiSelect condicional
Visível e obrigatório apenas quando modo_alcance === 1
Source: empresas_categorias WHERE ativo=true
→ olivia_modelos_peticao.empresas_categorias UUID[]
6 empresas_especificas MultiSelect (searchable) condicional
Visível e obrigatório apenas quando modo_alcance === 2
Source: empresas WHERE ativo=true, com busca
→ olivia_modelos_peticao.empresas_especificas UUID[]
7 versao_nome TextInput obrigatório
Label: "Versão" · Default: v1.0
pattern: ^v?\d+(\.\d+)*$
→ olivia_modelos_peticao.versao TEXT NOT NULL
8 arquivo FileUpload obrigatório
Accept: .pdf, .docx · Max: 10MB · Upload bucket="modelos", path="{org_id}/templates/{uuid}.{ext}"
Pós-upload: dispara Webhook WH-MOD-002 (detectar placeholders)
→ olivia_modelos_peticao.arquivo_bucket_path TEXT
Fluxo de Submit
  1. Validar formulário (todos os campos obrigatórios)
  2. Upload arquivo para Storage → bucket "modelos"
  3. Webhook WH-MOD-002 detectar placeholders no arquivo
  4. INSERT olivia_modelos_grupo (tipo, nome, descricao, modo_alcance)
  5. INSERT olivia_modelos_peticao (versão v1.0, empresas, arquivo_path, status='draft')
  6. INSERT olivia_flow (flow vazio vinculado ao grupo)
  7. UPDATE olivia_modelos_grupo.active_version_id = nova versão
  8. Toast sucesso · Fechar modal · Atualizar lista

Modal: Detalhes do Modelo P0 size: 90vw × 80vh

5 Tabs Internas — spec ModalModeloDetalhes
1 — Informações 2 — Versões 3 — Flow 4 — Tags 5 — Placeholders

Tab 1: Informações (two-column 60:40)

Esquerda: tipo, categoria, alcance, categorias/empresas, criado, descrição. Direita: stats (usos total, usos mês, nº versões, taxa sucesso %)

Tab 2: Versões (timeline)

Timeline de versões com botão "+ Nova Versão". Cada item: nº versão, badge status, indicador de versão ativa, data relativa, autor, ações (definir ativa, ver arquivo, duplicar).

Tab 3: Flow

Componente embarcado FlowRulesEditor (visualização somente, edição abre ModalFlowEditor completo)

Tab 4: Tags

Chips de tags extraídas do flow ativo via getTagsFromFlow(flow)

Tab 5: Placeholders

✓Botão "Redetectar" aciona WH-MOD-002
✓Tabela: placeholder · label · tipo · valid · source
!Aviso bloqueante se hasInvalidPlaceholders = true → modelo não pode ser publicado
