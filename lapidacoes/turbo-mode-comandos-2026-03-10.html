🧠 Turbo Mode — Gestão de RAM

O orquestrador monitora RAM a cada ciclo e ajusta automaticamente quantas páginas rodam em paralelo. Páginas conflitantes nunca rodam ao mesmo tempo.

RAM UsadaParalelasComportamentoStatus
< 50%🟢 3 páginasMáximo — noite sem Chrome abertook
50–65%🟡 2 páginasModo padrão durante o diamoderado
65–80%🟠 1 páginaConservador — outras apps abertasalto
80–88%🔴 0 novasAguarda terminar as ativascrítico
> 88%🆘 EmergênciaPausa + alerta Telegramemergência

🔀 Detecção de Conflito — Páginas Não Rodam Juntas

Conflitos definidos

/controller ≠ /distribuicao, /jurimetria
/acordos ≠ /execucoes
/olivia ≠ /crm

Como funciona

Quando o orquestrador vai lançar a próxima tarefa, checa se a nova página conflita com qualquer uma em execução. Se sim, pula para a próxima na fila que não conflite. Nunca para — só reordena.

📁 Sistema de Conversas — Auto-Save por Grupo

Cada grupo Telegram tem uma pasta em ~/jarbas/conversas/. Documentos gerados nas conversas são salvos automaticamente.

~/jarbas/conversas/
├── importacao-processos/ # grupo -5272935309
│ ├── projetos/ # esboços ativos
│ ├── relatorios/ # reports gerados
│ ├── arquivos/ # outros docs
│ └── index.md # índice auto-gerado
├── notificacoes/ # grupo -5251265489
└── christian-direto/ # chat privado

Formato de Armazenamento

Markdown (.md) — principal. Melhor para LLMs (sem ruído de HTML).

HTML (.html) — opcional, só quando o visual importa para referência futura.

PRDs sérios → ~/jarbas/docs/PRDs/
Conversas/ = esboços e projetos em andamento. Quando vira PRD definitivo, move para docs/PRDs/.

Configurar Grupos

Edite ~/jarbas/config/conversas-grupos.json para adicionar grupos. Grupos sem mapeamento não salvam documentos.
