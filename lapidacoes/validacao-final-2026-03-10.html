🐛 Bugs Corrigidos pelo Claude Code

✅ FIXED

BUG-001 — proximaTarefa() não era async

Usava await checkPausedRemoto() em função sync. Silenciosamente ignorado. Fix: declarado como async function.

✅ FIXED

BUG-002 — Auto-loop entre páginas

Após concluir uma página, o orchestrator não enfileirava a próxima. Novo comando continuar combina reportar + proxima + enqueue atomicamente.

✅ FIXED

BUG-003 — Sem garantia de resposta

Handler sem try/catch/finally. Erros deixavam tudo preso. Fix: wrapper garantido que notifica Telegram + atualiza MC mesmo em falha.

✅ FIXED

BUG-004 — Notificações no grupo errado

Todas as notificações iam pro grupo genérico. Resolvido via FEATURE-002: roteamento por config/page-groups.json.

✅ FIXED

BUG-005 — Playwright falha silenciosa

Screenshots tentavam login sem verificar dev server. Fix: curl --max-time 5 health check antes. Se offline: pula com aviso claro.

✅ FIXED

BUG-006 — Git push timeout curto

30s insuficiente para repos com screenshots. Fix: 120s + retry automático em falha.

⚡ Novas Features Implementadas

✅ FEAT

Turbo Mode — 2 páginas em paralelo

Triggers: turbo on / turbo off. Orchestrator suporta em_progresso[] (array). Pré-requisito: RAM < 70% e CPU < 60%. Use quando não estiver usando JARBAS.

✅ FEAT

Grupos Telegram por Página

Mapeamento em config/page-groups.json. Função getPageGroup(rota) roteia notificações para o grupo da página. Fallback: grupo de lapidações genérico.

✅ FEAT

Mission Control Kanban

Nova aba no dashboard com 6 colunas: Na Fila | Investigação | Aguarda Aprovação | Em Execução | Problema | Concluída. Dados em tempo real do Supabase + state JSON.

✅ FEAT

Graceful Failure — Continua após erro

Novo status problema (diferente de erro). Missão não para por uma página com falha — notifica e avança para a próxima automaticamente.

✅ FEAT

Anti-Stall Melhorado (15min)

Threshold expandido de 5 para 15min. Notificação vai para o grupo específico da página travada. Não cancela — só notifica para triagem.

🆕 NEW

Watchdog em Loop (5min)

scripts/lapidacao-watchdog.mjs — inicia automaticamente com "lapidar todas". Verifica stalls, impulsiona tarefas travadas, encerra quando missão conclui ou fica 30min idle.

⚠️ Checklist Antes de Dormir

  • Dev server rodando: cd ~/beta-mdflow && npm run dev (para screenshots Playwright)
  • Fechar Chrome e Claude Web (libera 3-4GB RAM para os agentes)
  • Confirmar RAM livre > 6GB antes de ativar turbo (turbo off por padrão)
  • Configurar grupos Telegram das páginas P0 em config/page-groups.json para receber notificações separadas
  • Grupo "JARBAS - NOTIFICAÇÕES" só recebe alertas gerais — lapidações vão pro grupo da página

🚀 Para Iniciar Agora

1. Crie grupos Telegram para as páginas P0 (/acordos, /decisoes, /execucoes, /distribuicao) e preencha em config/page-groups.json
2. Inicie o dev server no Mac Mini
3. Feche Chrome e Claude Web
4. Mande no Telegram: "lapida /acordos — [qualquer contexto que quiser passar]"
5. Watchdog sobe automaticamente. Acompanhe pelo grupo da página + Mission Control.
