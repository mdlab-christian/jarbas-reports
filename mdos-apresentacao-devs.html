01 · O que é o MdOs

Uma segunda
interface para
o MdFlow

Vocês já conhecem o MdFlow — o SaaS que gerencia nosso escritório jurídico com milhares de processos em 7 OABs (RS, SC, SP — RJ e MG sendo integrados). O MdOs é o mesmo sistema, mas acessível via terminal.

  • Mesmo Supabase — qualquer operação feita no MdOs aparece em tempo real no MdFlow
  • Mesmos webhooks n8n — as automações rodam igual
  • Mesma RLS multi-tenant — organizacao_id injetado automaticamente
  • Mesmos agentes OlivIA e Cecilia — compartilhados entre as três interfaces
  • Linguagem natural — "gera réplica para o processo X" ou "lista prazos de hoje"
# A equipe abre o MdOs assim: cd ~/Documents/MdOs-vault && claude # E opera o escritório inteiro: /mdos-olivia:peticao # → pipeline OlivIA /mdos-cecilia:prazos # → D-0/D-1 hoje /mdos-atividades:criar # → INSERT Supabase /mdos-decisoes:listar # → SELECT view_decisoes

03 · Distribuição

Plug-and-play
via Obsidian Sync

Qualquer MacBook da equipe recebe todas as atualizações automaticamente. Sem deploy, sem git, sem configuração manual.

📦
Instalação zero-touch
Obsidian Sync + Claude Code + um script de install. Em 5 minutos um novo membro tem acesso completo ao sistema operacional do escritório.
bash install.sh [nome]
🔄
Updates automáticos
Quando o Christian atualiza uma skill no vault, o Obsidian Sync distribui para todos os MacBooks da equipe em segundos. Sem nenhuma ação necessária.
vault.md → Obsidian → todos
🛡️
Ferramentas via SSH
Os arquivos .mjs e .sh não sincronizam pelo Obsidian — são enviados via SSH para o Mac Mini e para o Air com um único script.
bash admin/sync-tools.sh

05 · Obsidian como cérebro

Um vault para
governar tudo

Obsidian não é só onde a equipe anota coisas — é o sistema nervoso dos três agentes. Cada um tem seu próprio vault sincronizado com toda a knowledge base necessária para operar.

Estrutura dos vaults
📁 MdOs-vault/ ← Obsidian Sync
├── .claude/commands/ ← 94+ skills
├── _tools/ ← mdos-db.mjs · mdos-n8n.mjs
├── docs/ ← arquitetura · schema · guias
├── conhecimento/ ← playbooks · taxonomia A-G
└── vaults/ ← mirror olivia + cecilia
📁 olivia-vault/ ← Mac Mini user:jarbas
├── SOUL.md ← identidade + modos
├── playbooks/ ← estratégia por réu

└── STYLE.md ← estilo por advogado
📁 cecilia-vault/ ← Mac Mini user:cecilia
├── SOUL.md ← identidade + regras
├── taxonomia-A-G.md ← classificação intimações
├── treinamento/ ← 17 playbooks do João

Como a sincronização funciona
✏️

↓
📡

Em segundos, os MacBooks de Heloísa, João, Leonardo e Stefani recebem a atualização. Zero ação necessária.

🔧
Ferramentas (.mjs) sincronizam via SSH
