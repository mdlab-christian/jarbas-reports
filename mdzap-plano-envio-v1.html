🔧 Correção necessária no n8n: MDZAP-ENVIAR-MENSAGEM

O nó "Meta API Send" tem o body hardcoded como type:text. Precisa virar um switch por tipo:

// Nó Code (JavaScript) — substituir o body estático:
function buildBody(contato, tipo, mensagem, media_url, filename) {
  const base = { messaging_product: 'whatsapp', to: contato };
  switch (tipo) {
    case 'text':
      return { ...base, type: 'text', text: { body: mensagem } };
    case 'audio':
      return { ...base, type: 'audio', audio: { link: media_url } };
    case 'image':
      return { ...base, type: 'image', image: { link: media_url, caption: mensagem || '' } };
    case 'video':
      return { ...base, type: 'video', video: { link: media_url, caption: mensagem || '' } };
    case 'document':
      return { ...base, type: 'document', document: { link: media_url, filename: filename || 'arquivo' } };
    default:
      return { ...base, type: 'text', text: { body: mensagem } };
  }
}
⚠️ Atenção formato de áudio: A Meta API aceita .aac, .mp4, .mpeg, .amr, .ogg (somente OPUS codec). O MdzapAudioRecorder grava em audio/ogg;codecs=opus — compatível.

⚠️ Verificar: bucket mdzap-media é público?

A Meta API precisa baixar a mídia diretamente via URL pública (sem autenticação). O bucket Supabase mdzap-media precisa ser público (ou ter policy de acesso anônimo). Se for privado, o link gerado pelo getPublicUrl() não funciona com a Meta API.

Verificar no Supabase Dashboard: Storage → mdzap-media → Policies → deve ter SELECT for public.
