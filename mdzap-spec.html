Core — O que é o MDZap
Visão geral, posicionamento no ecossistema MdFlow e regras fundamentais
Missão do MDZap

O MDZap é o módulo de WhatsApp Business do MdFlow. É o primeiro ponto de contato com leads — onde o lead chega, é qualificado pela Julia (SDR IA), fornece documentos e eventualmente se torna um processo ou cliente registrado no sistema.

O MDZap abrange desde o primeiro "oi" do cliente até o momento em que ele vira um processo ativo. A partir daí, outros módulos assumem (acordos, documentos pendentes).

Separação de Agentes e Páginas
🤖 Julia — SDR Principal (ESTE MÓDULO)

Agente de leads novos. Qualifica, coleta documentos iniciais, faz handoff para humano quando necessário. Página: /mdzap. Número principal conectado.

✓ Leads novos ✓ Qualificação ✓ Docs iniciais ✓ Handoff CRM
📋 Documentos Pendentes (OUTRO AGENTE)

Pós-atendimento. Cobrar documentos de clientes já cadastrados com processos em andamento. Número e agente diferentes — não confundir com Julia.

⚠ Pós-atendimento ⚠ Outro número Futuro
⚖️ Acordos (OUTRO AGENTE)

Etapa avançada do processo — negociação de acordos. Processo já em estágio bem avançado. Número e agente próprios, desvinculados da Julia.

⚠ Etapa avançada ⚠ Outro número Futuro
📊 CRM — Conexão Principal

Quando Julia qualifica um lead, ele vai para o CRM. A conversa fica vinculada ao lead via lead_id e ao cliente via cliente_id. O Lead Panel (painel direito) exibe dados do CRM em tempo real.

Regras Fundamentais
⚠️
Julia NÃO é responsável por: Documentos pendentes de clientes já ativos (pós-cadastro), negociação de acordos, contato com processos em andamento. Esses têm números e agentes separados.
  • Um número WhatsApp, um agente: Cada número no sistema tem um propósito definido (comercial/operacional/suporte) e um agente IA correspondente.
  • Conversa vinculada ao lead: Toda conversa tem lead_id ou cliente_id assim que o lead é qualificado/cadastrado no CRM.
  • Handoff explícito: Julia só para quando o botão "Assumir" é clicado por um humano. A IA não desliga sozinha sem motivo.
  • Feedback interno: 👍👎 nas mensagens da Julia são visíveis apenas internamente para treinar e melhorar o prompt.
  • Memória da Julia: Cada conversa tem um campo de memória JSONB que a Julia atualiza progressivamente com dados coletados do lead (CPF, situação, empresa credora, etc.).
  • Templates Meta aprovados: Para iniciar conversa (outbound) é obrigatório usar template Meta aprovado. Respostas inbound podem ser texto livre dentro da janela de 24h.

RPC Principal — fn_mdzap_conversas
-- Retorna conversas com contexto completo (JOINs de lead, cliente, número)
CREATE OR REPLACE FUNCTION fn_mdzap_conversas(
  p_org_id   UUID,
  p_numero_id UUID DEFAULT NULL
)
RETURNS TABLE (
  id                  UUID,
  contato_numero      TEXT,
  contato_nome        TEXT,
  status              TEXT,
  nao_lidas           INT,
  ultimo_msg_at       TIMESTAMPTZ,
  metadata            JSONB,
  created_at          TIMESTAMPTZ,
  numero_display      TEXT,
  numero_nome         TEXT,
  numero_proposito    TEXT,
  lead_id             UUID,
  lead_nome           TEXT,
  lead_coluna         TEXT,
  lead_awaiting_human BOOLEAN,
  cliente_id          UUID,
  cliente_nome        TEXT
)
LANGUAGE sql SECURITY DEFINER SET search_path = public AS $$
  SELECT
    wc.id, wc.contato_numero, wc.contato_nome, wc.status,
    wc.nao_lidas, wc.ultimo_msg_at, wc.metadata, wc.created_at,
    wn.numero_e164, wn.nome, wn.proposito,
    l.id, l.nome, l.coluna, l.awaiting_human,
    cl.id, cl.nome
  FROM whatsapp_conversas wc
  JOIN whatsapp_numeros wn ON wn.id = wc.numero_id
  LEFT JOIN crm_leads l    ON l.id = wc.lead_id
  LEFT JOIN clientes cl    ON cl.id = wc.cliente_id
  WHERE wc.organizacao_id = p_org_id
    AND (p_numero_id IS NULL OR wc.numero_id = p_numero_id)
  ORDER BY wc.ultimo_msg_at DESC NULLS LAST
  LIMIT 200;
$$;

Especificação do Input Area (Redesign)
📝 Input de Mensagem — Comportamentos
  • Auto-resize: Textarea cresce conforme o texto digita — máximo 6 linhas antes de scroll interno
  • Enter = enviar, Shift+Enter = nova linha
  • / (barra) = templates: Digitar "/" abre dropdown de autocomplete com todos templates ativos. Filtrar por nome/conteúdo conforme digita
  • Ctrl+V imagem: Colar screenshot/imagem abre MdzapMediaPreview com preview antes de enviar
  • Ctrl+V texto/link: Cola normalmente no textarea (ou detecta URL e oferece link preview)
  • Formatação de texto: Suporte a WhatsApp markdown: *negrito*, _itálico_, ~tachado~, ```código``` (renderizado nos bubbles)
  • Indicador de digitação: Mostra "lead está digitando..." quando recebe typing indicator via webhook
  • Botão ✨ AI Composer: Ao lado do Send — abre dropdown com opções: "Tom formal", "Tom amigável", "Corrigir texto", "Resumir", "Traduzir"
🤖 AI Composer — Especificação

O botão ✨ reformula o texto digitado pelo humano usando IA antes de enviar.

  • Trigger: Botão ✨ visível no input area ao lado do Send
  • Dropdown com opções:
    • ✍️ Tom formal — reescreve mais profissional
    • 😊 Tom amigável — mais próximo e empático
    • ✔️ Corrigir gramática — apenas ortografia
    • 📝 Resumir — encurta o texto mantendo sentido
    • 🌐 Traduzir para inglês
  • Fluxo: Texto atual → webhook mdzap/ai-composer → Claude/OpenAI → substitui texto no input
  • Estado loading: Botão mostra spinner enquanto processa
  • Código existente: MdzapAIComposer.tsx já existe, melhorar UI do dropdown

📐 Regras de Renderização das Mensagens
  • Texto longo: Quebra de linha automática — white-space: pre-wrap para preservar quebras intencionais do lead
  • Largura máxima: 75% do container da thread para não encher toda a tela
  • Separadores de data: "Hoje", "Ontem", ou "12 de março de 2026" — centralizados entre grupos de msgs
  • Status da mensagem (saída): ✓ enviado, ✓✓ entregue, ✓✓ (azul) lido — via wa_message_id e status field
  • Remetente label: "🤖 Julia" (verde) / "👤 Humano + nome" (amarelo) — mostrado APENAS nas mensagens de saída, acima do texto
  • Imagens: Thumbnail até 200px de largura + botão de ampliar (lightbox) + hover para expandir
  • Áudio recebido: Player nativo com barra de progresso, tempo total e botão play
  • Vídeo: <video controls preload="metadata"> máximo 200px largura
  • Documento: Ícone + nome do arquivo + tamanho + link para download em nova aba
  • Feedback (👍👎): Visível APENAS nas mensagens da Julia (remetente_tipo === 'nina'). Não aparece para mensagens humanas nem do lead.
  • Scroll: Posição fixa no fundo ao receber novas mensagens. Se usuário rolar para cima, parar o auto-scroll e mostrar botão "↓ Nova mensagem"
