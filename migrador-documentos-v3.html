Qual abordagem adotar para o n8n?
  • Opção A — Manter Google Docs: n8n replica o fluxo Make (copia Google Doc → exporta PDF → sobe no Drive). Vantagem: PDFs idênticos aos atuais. Desvantagem: dependência do Google Drive, editar template = editar Doc no Drive.
  • Opção B — HTML no Supabase (recomendado por JARBAS): Templates em HTML no Supabase, PDF gerado pelo PDF.co. Vantagem: tudo versionado, sem Google, mais rápido. Desvantagem: precisamos converter todos os Google Docs para HTML — trabalho inicial.
  • Opção C — Híbrida: n8n chama a Google Docs API diretamente (sem Make), usando os mesmos Doc IDs. Menos trabalho de conversão, mantém os Docs originais.
