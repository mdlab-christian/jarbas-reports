🛠️ Plano de ação — o que fazer quando o cenário Make chegar
01
Refatorar skill_olivia_gerar_peticao para template substitution puro

Eliminar a geração IA do zero. Nova lógica:
1. Buscar template de olivia_modelos_peticao
2. Buscar dados: processo, cliente, advogado, restricao
3. Substituir {{placeholder}} → valor real
4. Converter template (texto/HTML) → PDF via Playwright
5. Upload Storage → signed URL → UPDATE processos_documentos

Esforço: 2h | Impacto: petição de 204s → ~5s | Score: determinístico (não mais 4.5)
02
Extrair modelos reais do cenário Make e cadastrar no banco

Quando você mandar o cenário Make:
1. Extrair templates de petição (com {{placeholders}})
2. Mapear tipos: SPC/Serasa, Boa Vista, Dívida Prescrita etc.
3. Inserir em olivia_modelos_peticao com tipo_peticao_id correto
4. Vincular automaticamente ao tipo de processo (pela empresa réu)

Esforço: 1h após receber o cenário | Resultado: templates reais funcionando
03
Auto-select inteligente de modelo no dispatcher

Hoje: pega o 1º modelo ativo (qualquer um).
Proposta: match por empresa réu → tipo_peticao_id → modelo mais recente ativo.
Lógica: empresa_nome ILIKE '%serasa%' → tipo SPC/Serasa → modelo correspondente

Esforço: 1h | Resultado: template sempre correto sem intervenção do advogado
04
[OPCIONAL] n8n como backup/fallback de petição

Se quiser redundância: manter WH-OLIVIA-GERAR-DOC-V3 ativo como fallback.
Se JARBAS falhar (Mac Mini offline), frontend tenta n8n automaticamente.
Não é prioridade agora, mas é viável.

Esforço: 3h | Resultado: zero downtime mesmo com Mac Mini off
