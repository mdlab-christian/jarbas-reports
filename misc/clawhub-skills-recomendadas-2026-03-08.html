🤖 Code Review — Comparativo
Codex (o4-mini) vs Claude Opus — qual usar?
Análise honesta para o contexto MdFlow. TL;DR: use os dois juntos.
⚡
Codex CLI (o4-mini)
OpenAI — raciocínio puro
✅ Excepcional em bugs de lógica e TypeScript
✅ Detecta race conditions, memory leaks, edge cases
✅ Muito barato vs Opus (~5x mais barato)
✅ Rápido — sem overhead de contexto MdFlow
❌ Não conhece DS v7, tokens, PATTERNS.md
❌ Pode não pegar violações de design system
❌ Mais nova dependência (OpenAI key + CLI)
⚠️ o3 seria melhor mas é 10x mais caro que o4-mini
Melhor para: bugs lógicos, segurança, TypeScript strict, SQL injection, queries sem RLS
🧠
Claude Sonnet (atual)
Anthropic — contexto profundo
✅ Conhece todo o contexto MdFlow (DS v7, patterns)
✅ Segue checklists complexos com precisão
✅ Detecta violações de design system e tokens hex
✅ Já integrado — zero setup extra
⚠️ Opus é caro (R$5–15 por review)
⚠️ Sonnet já é suficiente para 90% dos casos
❌ Não raciocina tão profundo quanto o3 em lógica pura
Melhor para: DS v7 compliance, padrões Supabase/RLS, UX/acessibilidade, org_id checks
🎯 Minha recomendação: pipeline híbrido
Fase 1
Codex o4-mini — revisão de lógica, TypeScript, segurança, SQL. Rápido e barato. Trava bugs antes de lapidar.
Fase 2
Claude Sonnet (não Opus) — DS v7 compliance, padrões MdFlow, org_id, deleted_at, staleTime. Contexto completo.
Opus?
Reservar só para páginas muito complexas (+500L, controller, financeiro). Para /clientes, /processos → Sonnet basta.
