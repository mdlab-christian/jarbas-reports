🎯 O que muda na prática?

Antes, tudo passava pelo Mac Mini — operação jurídica, build, Claude Code, análise de vídeo. Quando o Mini estava pesado (load 5.8), tudo atrasava junto.

Agora: Mini = operação pura (EPROC, intimações, Supabase, crons). Air = desenvolvimento puro (build, Claude Code, Gustavo trabalhando). Os dois rodam em paralelo, sem disputar recursos.

O ganho real não é "dobrar" no sentido absoluto — o gargalo ainda é custo de API. Mas o throughput paralelo dobra: enquanto JARBAS processa 50 processos no Mini, Gustavo + Claude Code constroem feature nova no Air. Isso antes era impossível sem travar tudo.

⚠️ O gargalo real: tokens de API, não hardware

Dois M4s com 32 GB RAM são mais que suficientes para qualquer carga local. O limite vai ser o custo de API Anthropic — cada subagent adicional consome tokens. Com 16 agentes rodando paralelo, uma lapidação pesada pode custar R$80–200 de uma vez. Usar com critério: paralelismo total para tarefas grandes (lapidação de página inteira), serial para consultas rápidas.
