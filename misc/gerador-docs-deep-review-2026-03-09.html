🔴 Blocker Crítico — Arquitetura HTTP JARBAS
🚧 JARBAS não tem servidor HTTP para receber chamadas do frontend
O frontend chama callWebhook('mdflow/doc/gerar-comprovante', ...) que aponta para n8n no Railway. Para JARBAS assumir, ele precisa expor esses mesmos endpoints localmente — mas JARBAS é um processo Node.js (poller + fila), sem Express ou servidor HTTP.

O Mac Mini está acessível da Vercel via domínio? Provavelmente não direto — precisa de ngrok/cloudflare tunnel OU proxy via Supabase Edge Function.
Solução recomendada (Opção A — Express local + Cloudflare Tunnel):
1. Adicionar Express em start.mjs (porta 3001)
2. Registrar rotas: POST /mdflow/doc/gerar-* → trigger skill
3. Cloudflare Tunnel: cloudflared tunnel --url http://localhost:3001 → URL pública estável
4. Atualizar VITE_JARBAS_WEBHOOK_URL no .env do beta-mdflow

Solução alternativa (Opção B — Supabase Edge Function proxy):
Edge Function recebe o POST → insere em jarbas_tarefas → JARBAS processa via polling → atualiza processos_documentos
✓ Sem exposição do Mac Mini | ✗ Adiciona latência + complexidade
Gaps Identificados por Componente
⚠️ ProcessoDrawerTabDocumentos — sem botão "Gerar"
O componente em /components/shared/processo_drawer/tabs/ProcessoDrawerTabDocumentos.tsx mostra docs existentes mas não renderiza as 4 categorias fixas quando ausentes nem tem botão para acionar geração. O DocumentosClientePanel.tsx já tem isso — precisa replicar o padrão.
Fix: importar useGerarDocumento + mostrar linhas vazias para categorias ausentes + botão "Gerar" por categoria. 30min de trabalho.
⚠️ useGerarDocumento — aponta apenas para n8n
WEBHOOK_MAP usa VITE_N8N_WEBHOOK_URL hardcoded para todos os endpoints. Quando JARBAS assumir, precisa de uma variável de ambiente separada VITE_JARBAS_WEBHOOK_URL com fallback para n8n.
Fix simples: adicionar VITE_JARBAS_WEBHOOK_URL no .env + lógica: se JARBAS_URL existe, usa JARBAS; senão, usa n8n. Permite migração gradual.
⚠️ ProcessoDrawer sem organizacaoId correto para geração
O ProcessoDrawerTabDocumentos recebe processoId e drivePastaId, mas não recebe organizacaoId — necessário para gerar documentos. Precisa adicionar a prop.
Fix: passar organizacaoId como prop do ProcessoDrawer para o tab de documentos (já está no ProcessoDrawer.tsx via organizacaoId prop).
🔵 ProcessoDrawerTabRestricao — sem botão "Gerar Comprovante"
A tab de Restrição mostra os dados da restrição, mas não tem o botão de gerar o comprovante diretamente dela. Seria UX ideal: ver a restrição + gerar comprovante na mesma tela.
Fix: adicionar botão "Gerar Comprovante" na tab Restrição, disparando useGerarDocumento('comprovante_restricao'). Melhoria de UX, não blocker.
🔵 CRM — useGerarDocumentosLead usa callWebhook sem JARBAS
O hook em useGerarDocumentosLead.ts usa WEBHOOK_PATH com paths diretos para n8n (sem VITE_N8N_WEBHOOK_URL base). Precisa unificar com useGerarDocumento.
Fix: deprecar useGerarDocumentosLead.ts e usar useGerarDocumento no painel CRM também. Unifica a lógica.
