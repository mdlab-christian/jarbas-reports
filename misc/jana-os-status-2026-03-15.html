Minhas ideias — o que vale implementar agora

🎯 preferred_host automático por tipo de skill P1 · 20min
Skills que usam browser EPROC (cerebro_documental, captura_intimacoes, importar_processo, download_pecas, login_eproc) devem ter preferred_host=macmini na hora de enfileirar. Skills puras de IA (lapidar_pagina, analise_juridica, olivia_*, gerar_prd) ficam com any — Air pega livremente. Isso zera os erros do Air e divide o trabalho de forma inteligente.
😴 Prevent Sleep no Air quando há tarefas na fila P1 · 10min
O Air com tela fechada pode dormir depois de 1h sem atividade mesmo com LaunchAgent ativo. Solução: pmset -g | grep sleep e configurar sudo pmset -a disksleep 0 sleep 0 quando plugado na tomada. Ou instalar caffeinate no LaunchAgent. Garante 24/7 sem precisar de ação manual.

🔄 Auto-sync de skills via git hook P2 · 15min
Hoje você roda sync-skills manualmente. Com um webhook simples no GitHub Actions, a cada push no repo jarbas o Air faz pull automaticamente e reinicia os workers. Zero intervenção manual para deploy de nova skill.
🧠 Air só pega tarefas de IA, Mini só pega tarefas de browser P2 · depende do P1
Após o P1 acima, o Air vira uma GPU de IA: lapidações, análises jurídicas, OlivIA, PRDs, relatórios. Mini vira o robô EPROC: capturas, downloads, logins, monitoramento. Divisão natural de responsabilidades sem conflito de recursos.
