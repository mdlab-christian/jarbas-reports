Seção 2 — A Pergunta Central
⚖️ Veredicto: Lapidação + Importação ao Mesmo Tempo?
Resposta direta baseada nos dados reais

✅ SIM — É viável, com uma condição

O Mac Mini M4 aguenta rodar lapidação + importação simultaneamente desde que nenhuma outra coisa esteja aberta (browsers, Claude Web, conversas extras). Veja por quê:

🟢 Lapidação (Claude Code)

  • 1 instância Claude Code: ~2-3GB RAM
  • Loop de 3 ciclos por página: ~2-4h por página
  • CPU intensivo APENAS durante geração
  • Entre ciclos: quase zero consumo

🔵 Importação (Playwright + IA API)

  • Playwright: ~800MB RAM
  • Haiku API: chamadas online, zero RAM local
  • pdftotext: leve (~50MB)
  • Sequencial por processo: não usa múltiplos subprocessos
Total estimado à noite: ~5-7GB RAM (35-45%) com ambos rodando. Safe zone. Mac Mini não vai travar.

❌ O Que NÃO Fazer

  • Deixar Claude Web ou Chrome aberto — add +3-4GB
  • Múltiplas lapidações simultâneas — cada uma = +2GB
  • Importação com concorrência > 2 processos simultâneos
  • Deixar conversas Telegram abertas com contexto longo
  • Rodar jurimetria-auto E importação ao mesmo tempo (duplica DB calls)

✅ O Setup Correto para Dormir

  1. Fechar Chrome, Claude Web, todas as abas
  2. Fechar esta conversa (libera ~500MB)
  3. Mandar lapidação noite → JARBAS entra em modo noturno
  4. Mandar importação noite → inicia pipeline de processos
  5. Dormir. Receber resumo às 6h no celular.

🤝 Coordenação Entre as Filas

As duas filas precisam ceder RAM uma à outra. Protocolo:

  • Lapidação pausada durante cada ciclo Claude Code → importação aproveita
  • Importação usa Playwright compartilhado entre processos (não abre 2 browsers)
  • Ambas leem o mesmo /tmp/jarbas-ram-state.json antes de executar
  • Se RAM > 75% → importação reduz para 1 simultâneo
  • Se RAM > 80% → lapidação pausa + importação também para
  • Se RAM > 85% → alerta Christian + ambas pausam
