🚀 Setup no MacBook Air — Passo a Passo

~30 min

Você vai clonar o ambiente do Mac Mini. Não vai refazer do zero — vai sincronizar a config.

  1. Instalar dependências base
    No MacBook Air, abrir terminal
    brew install node git
npm install -g openclaw
npm install -g @anthropic-ai/claude-code
  2. Clonar configuração do JARBAS
    Do Mac Mini via SSH ou copiar manualmente
    # Opção A: via SSH (se estiverem na mesma rede)
rsync -avz jarbas@mac-mini.local:/Users/jarbas/jarbas/ ~/jarbas/
rsync -avz jarbas@mac-mini.local:/Users/jarbas/.openclaw/workspace/ ~/.openclaw/workspace/

# Opção B: via GitHub (criar repo privado do jarbas config)
git clone git@github.com:mdlab-christian/jarbas-config.git ~/jarbas
  3. Copiar .env (credenciais)
    Mesmo arquivo .env do Mac Mini — mesmas API keys
    scp jarbas@mac-mini.local:/Users/jarbas/jarbas/.env ~/jarbas/.env
    Só ajustar: JARBAS_HOST=macbook e TELEGRAM_CHAT_ID se for bot separado
  4. Ajustar identidade do agente
    Para o orquestrador saber qual máquina é cada um
    # ~/jarbas/.env no MacBook Air — adicionar:
JARBAS_HOST=macbook
JARBAS_ROLE=interactive  # vs "worker" no Mac Mini
  5. Configurar OpenClaw com mesmo openclaw.json
    As skills, modelo e config são idênticos. Só o host muda.
    openclaw gateway start
openclaw status  # verificar que está rodando
  6. Testar conexão com Supabase e n8n
    Validar que ambas as máquinas conseguem acessar a fila
    node ~/jarbas/scripts/health-check.mjs

🎛️ Orquestrador de Tarefas — Como Vai Funcionar

O Supabase já é seu orquestrador. A tabela jarbas_tarefas funciona como fila centralizada. Vamos adicionar dois campos para roteamento inteligente.

Campos adicionais na jarbas_tarefas

ALTER TABLE jarbas_tarefas 
ADD COLUMN preferred_host TEXT DEFAULT 'any',  -- 'macbook' | 'macmini' | 'any'
ADD COLUMN assigned_host TEXT,                  -- quem pegou
ADD COLUMN host_priority INTEGER DEFAULT 5;     -- 1-10, maior = mais urgente

Lógica de roteamento automático

Tipo de Tarefa Vai para Motivo
cc: commands, pair programmer 💻 MacBook Você está presente e interagindo
EPROC, captura intimações 🖥️ Mac Mini Precisa de acesso ao navegador do escritório
Lapidações longas 🖥️ Mac Mini Pesado, pode rodar overnight sem travar seu trabalho
Relatórios HTML qualquer Fila pega quem estiver livre primeiro
Claude Code dev tasks 💻 MacBook Você monitora de perto no contexto de desenvolvimento
Crons noturnos (briefings, OlivIA) 🖥️ Mac Mini MacBook dorme, Mini fica ligado 24/7
Análise jurídica rápida qualquer Balanceamento automático por carga
🎯 Você não precisa pensar nisso. O JARBAS de cada máquina vai verificar a coluna preferred_host antes de pegar uma tarefa. Se for any, pega o primeiro que estiver livre. Se for específico, só aquela máquina pega.

🌐 Conexão Remota — Tailscale (recomendado)

Para acessar o Mac Mini remotamente (mesmo fora de casa/escritório), instale o Tailscale em ambas as máquinas. É gratuito, seguro e transparente — como se estivessem na mesma rede sempre.

  1. Instalar em ambas as máquinas
    brew install tailscale
sudo tailscale up
  2. Logar com mesma conta em ambas
    Depois de instalar, cada máquina aparece em tailscale.com/admin com IP fixo (100.x.x.x)
  3. SSH do MacBook para o Mac Mini
    ssh jarbas@mac-mini.tail12345.ts.net
    Funciona de qualquer lugar do mundo
  4. JARBAS controla ambas as máquinas
    O JARBAS do MacBook pode rodar comandos via SSH no Mac Mini quando necessário — total orquestração cross-machine

    Disparar uma lapidação no Mac Mini enquanto você faz um cc: command no MacBook Air — validar que não há conflito
