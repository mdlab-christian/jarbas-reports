✅ Veredicto: Sistema operacional — pode usar agora

Jana é um clone completo do JARBAS em capacidade. Mesmas 104 skills, mesmas credenciais, mesmo modelo, mesmos repos. A única diferença é identidade (SOUL.md separado) e Apple Notes (bloqueado intencionalmente).

Para começar a usar Jana: adicione @Jana_mdbot nos grupos que você usa com o JARBAS. Ela assume imediatamente com contexto completo.

Anti-travamento: quando Gustavo estiver fazendo lapidação pesada no JARBAS e você precisar de algo urgente, chama a Jana — ela não compartilha contexto, memória ou sessão com o JARBAS. São processos completamente independentes.

Sincronização automática: toda atualização de skill que o JARBAS faz e commita chega na Jana em até 1h via git pull automático.

💡 Dicas de Melhoria
⚡
Reduzir sync para 15min (opcional) O cron de git pull está em 1h. Se quiser skills novas na Jana mais rápido, pode reduzir para 15 min. Por ora 1h é suficiente — skills mudam raramente.
🔄
Comando migrar para handoff entre grupos Quando quiser passar um projeto do JARBAS para a Jana (ex: uma lapidação em andamento), use migrar jana no grupo. O contexto e arquivos relevantes são copiados automaticamente.
📁
GitHub Pages separados por agente HTMLs da Jana vão para /jana/, do JARBAS para /jarbas/. Dashboard: mdlab-christian.github.io/jarbas-reports/network-index.html
🧠
MEMORY.md: sincronizar periodicamente A memória de longo prazo diverge com o tempo (cada um escreve no seu). Use migrar jana após sessões importantes para manter Jana alinhada com decisões recentes do JARBAS.
🎯
Especializar Jana em dev, JARBAS em operação Com o tempo, os dois podem ter MEMORY.md diferentes: Jana focada em MdFlow dev, JARBAS em jurídico. Hoje os dois sabem tudo — o que é bom para cobertura, mas vai criar divergência natural.
