🔒 Isolamento de Contexto Entre Conversas
Como funciona a partir de agora — implementado no SOUL.md v3.1

✅ Chat Direto (1:1 com Christian)

  • Carrega: SOUL.md + USER.md + MEMORY.md + daily notes
  • NUNCA referencia contexto de grupos automaticamente
  • Para puxar contexto de grupo: dizer explicitamente
  • Trigger: "puxa o contexto do grupo [nome]"
  • Cada conversa tratada como universo isolado

✅ Grupos Telegram

  • Cada grupo carrega APENAS seu próprio arquivo de memória
  • Arquivo: memory/group-[chat_id].md
  • NUNCA carrega MEMORY.md (dados pessoais)
  • NUNCA cruza contexto entre grupos
  • Se não existe arquivo do grupo → cria na 1ª conversa

📁 Estrutura de Memória por Grupo

~/.openclaw/workspace/memory/
├── 2026-03-09.md ← chat direto, daily notes
├── MEMORY.md ← só chat direto
├── group-[id-grupo-acordos].md ← isolado, só /acordos
├── group-[id-grupo-olivia].md ← isolado, só /olivia
├── group-[id-grupo-decisoes].md ← isolado, só /decisoes
└── ... um por grupo

📋 Como criar grupos por página MdFlow

Quando Christian criar um grupo para tratar de uma página específica (ex: grupo para /acordos), JARBAS vai:

  1. Detectar o chat_id do grupo
  2. Criar memory/group-[chat_id].md automaticamente
  3. Não contaminar outros grupos nem o chat direto
  4. Contexto específico da página fica no grupo — não vaza
