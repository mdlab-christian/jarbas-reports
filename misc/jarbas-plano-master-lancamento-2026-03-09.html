Cada página precisa de contexto antes do loop. Como organizar.

📁 Estrutura de Documentação por Página

(1 arquivo por página)

📋 O que vai em cada PRD

  • Objetivo: para que serve a página
  • Usuário principal: advogado / Christian / operação
  • Funcionalidades obrigatórias (P0)
  • Melhorias desejadas (P1/P2)
  • Considerações da revisão manual do Christian
  • Queries Supabase relevantes
  • Tabelas envolvidas
  • Dependências (ex: /acordos depende de /processos)
  • Critérios de aprovação (o que precisa funcionar)

🤖 Geração Automática de PRDs — Comando

Christian passa as considerações de cada página, JARBAS gera o PRD formatado:

gerar prd /acordos — [suas considerações aqui] // JARBAS cria: beta-mdflow/docs/PRDs/PRD_ACORDOS.md // Com: objetivo, funcionalidades, queries, critérios // Inclui: considerações manuais do Christian + análise do código atual

📌 Fluxo: Considerações → PRD → Lapidação

  1. Christian passa considerações de uma página (áudio ou texto)
  2. JARBAS gera o PRD e salva em docs/PRDs/
  3. Christian aprova o PRD (ou ajusta)
  4. PRD aprovado entra na fila de lapidação como contexto obrigatório
  5. Lapidador lê o PRD no primeiro loop antes de tocar no código
