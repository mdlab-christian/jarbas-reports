II

Fase 2 — 6 etapas autônomas

~2-4h por página · notificação a cada etapa · CC fecha o ciclo

🤖 autônomo
1

🔍 Code Review inicial — Opus

Antes de qualquer linha de código, mdflow-code-reviewer (Opus) audita o estado atual. P0s críticos (Frankenstein, service_role exposto, any massivo, arquivos >700L) são resolvidos primeiro.

📢 Telegram: lista de P0s encontrados + nota estimada inicial
2

🔧 Loop de execução — mínimo 3 rodadas

Para cada rodada: Frontend (DS v7, abas, modais) → Backend (RLS, queries, hooks) → nota parcial por componente. Se nota < 9.0 ou rodadas < 3 → nova rodada (máx 5).

📢 Telegram: "Rodada X/5 — nota atual: Y/10" a cada rodada
3

📤 Commit + Push intermediário

git add src/pages/[pagina]/ && git commit && git push

📢 Telegram: "Push intermediário concluído"
4

🧪 Agente E2E — teste como usuário real

mdflow-e2e-tester com Playwright: login → navega página inteira → executa todos os fluxos do PRD → verifica banco → confirma UI atualiza. Nota E2E separada.

📢 Telegram: nota E2E + o que passou / o que falhou
5

🔎 CC Final Review — olhos frescos

Claude Code faz: git pull → lê o PRD completo → revisa toda a implementação contra o PRD → identifica gaps, drifts, itens faltando → corrige ele mesmo → git push final.

Esse passo é o diferencial: CC vê com olhos frescos o que o loop interno pode ter normalizado.

📢 Telegram: "CC Final Review — corrigidos: X itens | gaps pendentes: Y"
6

📋 Relatório HTML final + nota definitiva

HTML com: nota por dimensão (Frontend 70% + E2E 30%), screenshots before/after, resumo da execução. Publicado no GitHub Pages.

Nota Final = (Frontend × 0.7) + (E2E × 0.3) → precisa ≥ 9.0 para commit ser o último.

📢 Telegram: "🏆 [/pagina] LAPIDAÇÃO CONCLUÍDA — Nota: X/10 | 🌐 link"

Qualidade
📊 Sistema de Notas

Fórmula

Nota/componente = 10 - (P0×2) - (P1×0.5) - (P2×0.2) Nota Frontend = média das abas + modais Nota Backend = queries + RLS + tipos Nota E2E = fluxos completos + banco Nota Final = (Frontend × 0.7) + (E2E × 0.3) Commit final: Nota ≥ 9.0 AND zero P0s

Critérios de parada do loop

  • Mínimo 3 rodadas — sempre, mesmo se nota 10 na 1ª
  • Máximo 5 rodadas — para evitar loop infinito
  • Nota ≥ 9.0 e ≥ 3 rodadas → sai do loop e vai pro E2E
  • Após 5 rodadas → avança com a nota que tiver, CC Final Review tenta completar
  • CC Final Review — sempre executa, independente da nota
Resiliência
🚨 Protocolo de Erros Críticos
Um erro em uma página não para a fila. JARBAS pausa a página, avança, te avisa, retoma quando você responder.

Erro crítico — 3 tentativas sem solução

  1. Ativa mdflow-debug-specialist (Opus) para análise profunda
  2. Se debug não resolve em 2 tentativas → pausa a SUBTAREFA (não a fila)
  3. Avança para próxima página na fila
  4. Notifica no grupo com problema + tentativas + o que precisa
  5. Retoma quando você responder no grupo

Mensagem de erro no grupo

🚨 [/acordos] Subtarefa pausada Problema: Modal não fecha após submit Erro: Cannot read 'id' of undefined em AcordosModal.tsx:234 Tentativas: 1. null check → mesmo erro 2. undefined guard → mesmo erro 3. query refetch → mesmo erro Debug specialist ativado → sem solução Fila: avançando para /execucoes ✅ Preciso: você ver o modal em 192.168.0.4:5173/acordos Responda aqui para retomar ↩️

💡 Ideias para Evoluir
⚡

Fase 1 paralela entre páginas

Enquanto você revisa o HTML de /acordos, JARBAS já inicia o esboço de /decisoes em background. Economiza ~30min por página na Fase 1. Quando você aprovar o primeiro, o segundo já está adiantado.

fácil implementar alto impacto
📊

Notas por aba no Mission Control em tempo real

A cada subtarefa concluída, gravar nota na tabela jarbas_mission_control. O MC exibe uma tabela de notas por página sendo lapidada. Christian vê o progresso em tempo real enquanto dorme.

médio alto impacto
🔁

Loop de melhoria contínua — sem lapidação completa

Depois que todas as páginas forem lapidadas, um cron semanal roda o CC Final Review em todas as páginas. Se encontrar drifts ou melhorias → cria task de melhoria incremental. Zero retrabalho manual.

futuro alto valor
🧠

PRD gerado a partir de conversa de grupo

Você bate papo no grupo de lapidações sobre uma página, JARBAS extrai as decisões da conversa e já pré-preenche o PRD com o que você disse. Reduz o tempo de Fase 1 à metade.

criativo médio impacto
🏆

Leaderboard de páginas por nota

Relatório mensal: ranking de todas as páginas por nota, tendência (melhorou/piorou vs mês anterior), custo por página lapidada. Dashboard para você medir o ROI da lapidação autônoma.
