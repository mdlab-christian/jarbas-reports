🦞 MacBook Air — Setup JARBAS

Siga os passos em ordem · Clique em cada etapa para expandir · Clique no ✓ quando terminar
0 / 8 etapas concluídas
1
Abrir terminal e confirmar SSH do Mac Mini
~1 min

Confirmar que o Air consegue se conectar ao Mini:

ssh jarbas@100.78.173.81 "echo 'Mini OK'"

Deve aparecer: Mini OK

Se der "Permission denied", precisa configurar a chave SSH primeiro — me avisa no Telegram.
2
Espelhar arquivos do Mac Mini → Air
~5 min

Rodar no Mac Mini (não no Air). Abre o terminal do Mini e roda:

rsync -avz --progress ~/jarbas/ jarbas@100.101.208.82:~/jarbas/
rsync -avz --progress ~/.openclaw/ jarbas@100.101.208.82:~/.openclaw/
rsync -avz --progress ~/.claude/ jarbas@100.101.208.82:~/.claude/
rsync -avz --progress ~/.ssh/ jarbas@100.101.208.82:~/.ssh/
Aguarda cada comando terminar antes de rodar o próximo. O jarbas tem ~500MB, vai levar alguns minutos.
3
Corrigir permissões SSH e instalar dependências
~3 min

Rodar no Air:

chmod 600 ~/.ssh/id_* && chmod 700 ~/.ssh
cd ~/jarbas && npm install

Aguardar o npm install terminar (pode demorar 1-2 min).

4
Configurar identidade do Air no .env
~1 min

Rodar no Air:

echo "JARBAS_HOST=macbook" >> ~/jarbas/.env
echo "JARBAS_ROLE=worker" >> ~/jarbas/.env

Confirmar que foi adicionado:

grep "JARBAS_HOST\|JARBAS_ROLE" ~/jarbas/.env

Deve aparecer: JARBAS_HOST=macbook e JARBAS_ROLE=worker

5
Clonar repositórios GitHub no Air
~3 min

Rodar no Air:

git clone git@github.com:mdlab-christian/jarbas-reports.git ~/jarbas-reports
git clone git@github.com:mdlab-christian/beta-mdflow.git ~/beta-mdflow && cd ~/beta-mdflow && npm install
Se der erro de chave SSH: ssh -T git@github.com para testar. Se não funcionar, me avisa.
6
Instalar LaunchAgent (JARBAS inicia no boot)
~2 min

Copiar o LaunchAgent do Mini e adaptar para o Air. Rodar no Mini:

scp ~/Library/LaunchAgents/com.mdx.jarbas.plist jarbas@100.101.208.82:~/Library/LaunchAgents/com.mdx.jarbas.plist

Depois rodar no Air:

launchctl load ~/Library/LaunchAgents/com.mdx.jarbas.plist
launchctl start com.mdx.jarbas
O JARBAS vai iniciar agora e sempre que o Air ligar/reiniciar.
7
Copiar watchdog (reinício automático se JARBAS cair)
~1 min

Rodar no Mini:

scp ~/Library/LaunchAgents/com.mdx.jarbas-watchdog.plist jarbas@100.101.208.82:~/Library/LaunchAgents/com.mdx.jarbas-watchdog.plist

Depois rodar no Air:

launchctl load ~/Library/LaunchAgents/com.mdx.jarbas-watchdog.plist
8
Verificar se os 4 workers subiram no Air
~1 min

Rodar no Air (aguardar ~15s após o start):

tail -20 ~/jarbas/logs/launchd-out.log 2>/dev/null || grep -r "Pool paralelo\|4 workers\|macbook" ~/jarbas/logs/ 2>/dev/null | tail -5

Deve aparecer:

🚀 Pool paralelo iniciado — 4 workers · host=macbook
Se não aparecer, me avisa no Telegram que verifico o que aconteceu.

Adicionar aliases úteis no .zshrc do Air:

echo 'alias sync-from-mini="rsync -avz jarbas@100.78.173.81:~/jarbas/ ~/jarbas/ && echo ✅ sync ok"' >> ~/.zshrc
echo 'alias mini="ssh jarbas@100.78.173.81"' >> ~/.zshrc
source ~/.zshrc

🎉 Air configurado!

Me avisa no Telegram que confirmo os 4 workers rodando no Air e fazemos o teste final.
