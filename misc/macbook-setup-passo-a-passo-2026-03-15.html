1
🔌 Primeiro Boot — Configuração Inicial do macOS
⏱ ~10 min | Faça agora, antes de instalar qualquer coisa
  • 1
    Ligar e seguir o assistente do macOS
    Idioma: Português | Teclado: Brasileiro ABNT2 | Região: Brasil
  • 2
    Criar usuário Admin com SEU Apple ID
    Nome: Christian ou admin — este é o seu usuário pessoal com iCloud, AppStore, etc.
    ⚠️
    NÃO use Apple ID do Jarbas aqui. Este usuário é o seu — pessoal, com iCloud ativo.
  • 3
    Pular migração / Time Machine
    Selecionar "Não transferir informações agora" — vamos sincronizar manualmente depois.
  • 4
    Instalar Xcode Command Line Tools
    Abre o Terminal (⌘+Espaço → Terminal) e roda:
    xcode-select --install
    Confirmar na janela que aparecer. Aguardar (~5 min).
2
👤 Criar Usuário Dedicado do JARBAS
⏱ ~5 min | Usuário isolado para o agente
  • 1
    Acessar Configurações → Usuários e Grupos
    Menu Apple → Configurações do Sistema → Usuários e Grupos → clicar em +
  • 2
    Criar usuário jarbas
    Tipo: Administrador (precisa de admin para instalar apps e rodar OpenClaw)
    Nome completo: JARBAS
    Nome de conta: jarbas
    Senha: uma senha forte — salvar no seu cofre (1Password, etc.)
    💡
    Usuário Administrador porque o JARBAS precisa instalar pacotes npm globais, acessar o sistema e rodar serviços. Não precisa de Apple ID.
  • 3
    Habilitar login remoto (SSH) para o usuário jarbas
    Configurações do Sistema → Geral → Compartilhamento → ativar "Login Remoto" → adicionar usuário jarbas
3
🌐 Tailscale — Conexão Segura entre as Máquinas
⏱ ~5 min | Sem isso não dá pra sincronizar com o Mac Mini
  • 1
    Instalar Homebrew (no usuário Admin)
    /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
  • 2
    Instalar Tailscale
    brew install tailscale
sudo tailscale up
    Abre o browser para login — entrar com a mesma conta do Mac Mini
  • 3
    Verificar que o Mac Mini aparece na rede
    tailscale status
    Deve aparecer mac-mini (ou o hostname que você definiu) com IP 100.x.x.x
  • 4
    Testar SSH no Mac Mini
    ssh jarbas@mac-mini.local
# ou via Tailscale:
ssh jarbas@100.x.x.x  # IP do Mac Mini no tailscale status
4
⚙️ Instalar Stack no Usuário JARBAS
⏱ ~15 min | Logar no usuário jarbas e rodar esses comandos
⚠️
Trocar para o usuário jarbas antes de instalar — Menu Apple → Trocar usuário → jarbas. Tudo abaixo roda no terminal do usuário jarbas.
  • 1
    Instalar Homebrew no usuário jarbas
    /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
# Adicionar ao PATH (Apple Silicon):
echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> ~/.zprofile
eval "$(/opt/homebrew/bin/brew shellenv)"
  • 2
    Instalar Node.js, Git, ffmpeg
    brew install node git ffmpeg
node --version  # deve ser v20+
  • 3
    Instalar OpenClaw e Claude Code
    npm install -g openclaw
npm install -g @anthropic-ai/claude-code
  • 4
    Configurar chaves SSH para o GitHub
    ssh-keygen -t ed25519 -C "jarbas@macbook"
cat ~/.ssh/id_ed25519.pub  # copiar e adicionar no GitHub → Settings → SSH Keys
5
🔄 Sincronizar Configuração do Mac Mini
⏱ ~5 min | Clonar tudo que está no Mac Mini
💡
Rodando no usuário jarbas do MacBook, com Tailscale ativo. O Mac Mini precisa estar ligado.
  • 1
    Sincronizar pasta jarbas (skills, scripts, docs)
    rsync -avz --progress jarbas@MAC_MINI_IP:~/jarbas/ ~/jarbas/
    Substituir MAC_MINI_IP pelo IP Tailscale do Mac Mini (ex: 100.64.0.1)
  • 2
    Sincronizar workspace OpenClaw
    rsync -avz --progress jarbas@MAC_MINI_IP:~/.openclaw/ ~/.openclaw/
  • 3
    Sincronizar agents do Claude Code
    rsync -avz --progress jarbas@MAC_MINI_IP:~/.claude/ ~/.claude/
  • 4
    Ajustar .env para identidade do MacBook
    nano ~/jarbas/.env
    Alterar / adicionar estas linhas:
    # Identidade da máquina
JARBAS_HOST=macbook
JARBAS_ROLE=interactive

# Manter todas as outras chaves IGUAIS ao Mac Mini
  • 5
    Clonar repositório jarbas-reports
    git clone git@github.com:mdlab-christian/jarbas-reports.git ~/jarbas-reports
  • 6
    Instalar dependências do JARBAS
    cd ~/jarbas && npm install
6
🚀 Ligar o JARBAS no MacBook
⏱ ~2 min | Reta final
  • 1
    Iniciar o gateway OpenClaw
    openclaw gateway start
openclaw status
  • 2
    Testar no Telegram
    Mandar /status pelo Telegram — o JARBAS vai responder indicando qual máquina respondeu
    ✅
    Se aparecer host: macbook na resposta, o setup foi um sucesso!
  • 3
    Configurar gateway para iniciar automaticamente no login
    openclaw gateway install-autostart
    Ou via launchctl — o JARBAS vai iniciar sozinho sempre que o usuário jarbas logar

📁 Sistema de Contexto — "O arquivo está em qual máquina?"
Como o JARBAS vai saber onde estão seus arquivos
🧠 Como o JARBAS resolve isso
📡
Auto-detect
JARBAS verifica automaticamente em qual máquina o arquivo existe antes de abrir/editar
🗄️
Índice Supabase
Cada arquivo importante é registrado com localização em jarbas_files
☁️
Pastas Compartilhadas
GitHub, iCloud Drive ou Supabase Storage para arquivos que precisam estar nos dois lugares

Como você vai falar de arquivos no Telegram

Você diz JARBAS entende O que faz
o arquivo na mesa do mac mini Desktop do Mac Mini Acessa via SSH e pega o arquivo
o arquivo que vc salvou hoje Último arquivo criado Verifica log de criações recentes no Supabase
o html do relatório de ontem GitHub Pages Busca em jarbas-reports por data
no macbook: [tarefa] Explícito: MacBook Roteia para MacBook via preferred_host
no mac mini: [tarefa] Explícito: Mac Mini Roteia para Mac Mini via preferred_host
💡
Regra geral de arquivos: Tudo que é produto (relatórios HTML, análises, PDFs) vai para o GitHub ou Supabase Storage — acessível de qualquer lugar. Só ficam locais os arquivos temporários e de processo.
