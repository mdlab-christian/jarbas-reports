🗺️ Plano de Execução (o que JARBAS vai fazer)

FASE 1 Preparação da Conta Facebook
1
Login na conta nova
Acessar facebook.com com as credenciais. Verificar se há restrições, 2FA pendente ou confirmação de identidade. Se tiver, resolver com os códigos do site que Gustavo vai passar.
2
Criar Meta Business Manager
Acessar business.facebook.com → "Criar conta" → preencher nome da empresa (MP Advocacia ou MdLab), email de contato, CNPJ.
3
Iniciar verificação de negócio
Configurações do Business → "Verificação da empresa" → enviar CNPJ + documentos se solicitado. Pode levar de minutos a dias. Em contas novas, a Meta às vezes pula essa etapa inicialmente.
FASE 2 Criar App no Meta for Developers
4
Acessar developers.facebook.com
Criar app → tipo "Business" → vincular ao Business Manager criado na Fase 1.
5
Adicionar produto WhatsApp
No painel do app → "Add Product" → WhatsApp → "Set up". A Meta criará automaticamente uma WABA de teste vinculada ao app.
6
Copiar Token temporário
A Meta gera um token de 24h para teste. Já dá para testar o envio de mensagens enquanto configuramos o token permanente.
FASE 3 Registrar Número de Telefone
7
Adicionar número ao WABA
WhatsApp → Phone Numbers → Add phone number → inserir o número do chip novo. Escolher verificação por SMS.
8
Verificar com código SMS
Meta envia SMS com 6 dígitos para o chip. Gustavo precisa me passar esse código em tempo real (ou eu acesso o site que ele vai passar com os códigos).
9
Configurar display name
Nome que aparece para os clientes (ex: "MP Advocacia" ou "Midas Acordos"). A Meta aprova em minutos a horas — precisa ser nome relacionado ao negócio.
FASE 4 Gerar Token Permanente
10
Criar System User no Business Manager
Business Settings → System Users → Add → "Admin". Esse usuário de sistema não expira como uma conta pessoal.
11
Atribuir permissões ao System User
Atribuir o app + a WABA ao System User com permissão whatsapp_business_messaging e whatsapp_business_management.
12
Gerar Token Permanente
System Users → Generate Token → selecionar permissões → copiar token. Esse token não expira (diferente do token de 24h). Salvar no .env do LDZAP.
FASE 5 Integração com LDZAP / n8n
13
Configurar variáveis de ambiente
Salvar no n8n / .env: WHATSAPP_TOKEN, WHATSAPP_PHONE_ID, WHATSAPP_WABA_ID. Esses 3 dados ficam no painel do app.
14
Configurar Webhook
App → WhatsApp → Configuration → Webhook URL → apontar para o n8n (mdzap/inbound). Verify token = segredo customizado.
15
Teste end-to-end
Enviar mensagem de teste via API → confirmar chegada no webhook → confirmar resposta saindo pelo número oficial.

⚠️ Riscos & Como Mitigar

Conta nova bloqueada por "atividade suspeita"
A Meta pode bloquear contas novas que acessam rapidamente configurações de negócio/API. Mitigação: aguardar alguns minutos entre cada passo, não usar VPN durante a configuração.
Verificação de negócio demorada
Em alguns casos a Meta pede documentos (contrato social, CNPJ no site da Receita). Pode levar 2–5 dias úteis. Mas dá para continuar em modo "teste" (100 conversas/dia) enquanto aguarda.
Número rejeitado pela Meta
Se o chip tiver histórico no WhatsApp comum, o cadastro falha. Gustavo comprou chips novos — isso elimina o risco. Se rejeitar, tentar o segundo chip.
✅ Plano tem 3 chips de backup
Gustavo comprou 3 chips. Se o primeiro falhar, já tem backup imediato sem precisar comprar novo.
