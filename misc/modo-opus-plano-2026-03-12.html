O Problema — Por Que Precisamos Disto?

O JARBAS usa claude-sonnet-4-6 por padrão — rápido, econômico, excelente para 95% das tarefas cotidianas. Mas existe uma classe de tarefas onde velocidade não é o critério principal: projetos arquiteturais complexos, PRDs multi-módulo, decisões técnicas irreversíveis, refatorações estruturais do MdFlow.

Para essas tarefas, o Claude Opus 4.6 tem raciocínio profundo significativamente superior, planeja com mais passos, comete menos erros lógicos e produz arquiteturas mais sólidas — mesmo que custe mais e demore mais.

Especificações Técnicas da Implementação

  • Trigger archon: [tarefa] — detectado no smart router antes do roteamento normal
  • Modelo claude-opus-4-6 via API Anthropic direta
  • Max tokens output 16.000–32.000 tokens (configurável por tarefa)
  • Thinking Extended thinking ativado quando tarefa contém palavras-chave de arquitetura
  • System prompt Prompt ARCHON especializado: "Você é o Arquiteto-Chefe do MdFlow. Não use atalhos. Documente cada decisão."
  • Contexto injetado SOUL.md + MEMORY.md + arquivos relevantes do beta-mdflow/ (auto-detectados)
  • Timeout 300 segundos (5min) — tarefas complexas levam mais tempo
  • Persistência Auto-salva em memory/archon-[slug].md + jarbas_tarefas com status
  • Notificações Update no início ("⚡ ARCHON ativado...") + updates de progresso + entrega final
  • HTML output Quando tarefa produz relatório → publica automaticamente no GitHub Pages

Plano de Implementação — 4 Etapas

Criar skill_archon.mjs

Nova skill em ~/jarbas/src/skills/ — detecta trigger archon:, monta contexto, chama Opus 4.6 com system prompt especializado, retorna resposta estruturada.

Registrar no smart router (start.mjs)

Adicionar archon: como trigger de alta prioridade no roteamento — antes do check de verbos de ação, com priority: MAX.

System prompt + contexto dinâmico

Criar ~/jarbas/prompts/archon-system.txt. Implementar injeção automática de arquivos relevantes: detecta palavras-chave na tarefa → carrega arquivos do beta-mdflow/src/pages/ correspondentes.

Persistência + HTML automático

Toda saída do ARCHON é salva em memory/archon-[slug]-[data].md. Se contiver seções estruturadas (PRD, arquitetura, análise), gerar HTML e publicar no GitHub Pages automaticamente.

⚠️
Custo estimado de implementação ~2–3 horas de trabalho JARBAS. Skill nova (~200 linhas), registro no router (5 linhas), system prompt (1 arquivo). Nenhuma mudança no banco — só skill + router. Custo de API: ~R$10–R$20 por chamada ARCHON em projetos grandes.
