Quando criar Agente · Quando criar Skill
🧠 Crie um Agente Claude quando...
A tarefa exige julgamento especializado — análise jurídica, revisão de código, estratégia
Precisa de persona específica com conhecimento de domínio diferente (ex: mdflow-backend-specialist sabe só de DB)
A qualidade depende de instruções densas que não cabem no system prompt principal
O trabalho é iterativo com loop — revisa, melhora, volta, revisa (lapidador, debug specialist)
Precisa de modelo diferente — Opus para qualidade, Haiku para velocidade
Exemplos: mdflow-lapidador, mdflow-debug-specialist, especialista_juridico, mdflow-code-reviewer
⚙️ Crie uma Skill JARBAS quando...
A tarefa é determinística e repetível — mesma entrada, mesma saída esperada
Precisa de acesso a APIs externas — Supabase, n8n, EPROC, Telegram, GitHub
É um trigger do Telegram — Christian manda "lapida /controller" → skill executa
O trabalho tem sequência definida — passo 1, passo 2, passo 3 (monitorar, capturar, analisar)
Envolve filesystem ou processos do sistema — ler arquivo, executar bash, git push
Exemplos: skill_monitorar, skill_captura_intimacoes, skill_lapidar_pagina, skill_ler_documento
Regra de ouro: Skill = executor confiável · Agente = especialista que pensa. Na prática: skills chamam agentes quando precisam de julgamento. O lapida.mjs (skill) chama mdflow-lapidador (agente) para construir, depois chama mdflow-code-reviewer (agente) para validar.

Scanner de Segurança — Prompt Injection
Usar Codex o4-mini (~R$0.20/scan)
🚨 O que é prompt injection em skills
Uma skill maliciosa pode: (1) Exfiltrar .env e credenciais via fetch para URL externa · (2) Adicionar instruções ocultas em texto branco ou zero-width chars · (3) Sobrescrever SOUL.md ou system prompt · (4) Criar backdoor via cron ou LaunchAgent.
⚠️ Repos com flag de crypto
claude-mem ($CMEM) e get-shit-done ($GSD) têm tokens crypto Solana vinculados. O código técnico pode ser legítimo, mas incentivos econômicos tornam exfiltração mais provável. Sempre auditar antes de instalar.
✅ Como o scanner vai funcionar
skill_security_scan.mjs: lê todos os arquivos .mjs/.js/.md de uma skill → envia para Codex o4-mini com prompt de análise → retorna SEGURO / SUSPEITO / BLOQUEADO com razão específica.
📋 Checklist de auditoria manual
Buscar: fetch( para URLs externas · process.env enviado para fora · eval( ou Function( com string dinâmica · comentários com espaços em branco excessivos · arquivos .png/.jpg com dados embutidos.
