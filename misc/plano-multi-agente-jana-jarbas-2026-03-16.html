🔴 Problema Identificado

Context Blocking — Um agente só não aguenta

Com Gustavo + Christian operando em paralelo, o JARBAS trava. Um único agente tem contexto limitado e quando uma sessão pesada de lapidação está rodando, o outro usuário fica bloqueado. A solução: dois agentes independentes com estrutura compartilhada e handoff dinâmico.

🔄 Comando migra — Handoff Dinâmico

Como vai funcionar

Christian digita "migra Jana" ou "migra Jarbas" em qualquer grupo.
O agente receptor:
1. Recebe o contexto atual (últimas N mensagens + arquivos abertos + estado do projeto)
2. Puxa os relatórios relevantes do GitHub Pages
3. Lê os arquivos de memória do projeto (memory/project-*.md)
4. Confirma: "✅ Contexto recebido — estou pronto para continuar"

📋 Comandos planejados

migra Jana → JARBAS empacota contexto e passa para JANA
migra Jarbas → JANA empacota contexto e passa para JARBAS
migra Jana [resumo] → Inclui nota customizada no handoff
status migra → Ver último handoff realizado

O que o handoff inclui automaticamente

  • 📄 Arquivo de projeto ativo (memory/project-[chat_id]-*.md)
  • 🔗 Links dos últimos HTMLs publicados no GitHub Pages
  • 📋 Lista de tarefas pendentes + concluídas
  • 🗂️ Arquivos abertos / em edição no momento
  • 🧠 Últimas decisões técnicas tomadas na sessão
📅 Fases de Implementação
1

Estrutura GitHub — Renomear / Criar Repo Jana

✅ Viável agora

Criar jana-reports repo no GitHub com Pages ativo. Decidir: pastas compartilhadas em jarbas-reports com prefixo jana/, ou repo separado. Recomendação: repo separado — evita colisão de nomes e facilita permissões por agente.

  • Criar mdlab-christian/jana-reports com GitHub Pages
  • Jana publica em https://mdlab-christian.github.io/jana-reports/...
  • Cada agente tem identidade visual própria no report
2

Skill skill_migra.mjs — Empacotamento de Contexto

✅ Viável agora

Skill que detecta o comando migra [agente], monta o pacote de handoff e o salva em local acessível ao agente destino.

  • Salvar handoff em Supabase (jarbas_handoffs) com timestamp + chat_id
  • Agente destino lê o handoff na primeira mensagem do grupo
  • Notificar no grupo: "📦 Handoff pronto — adicione @Jana_mdbot no grupo e dê 'retoma contexto'"
3

Jana OS — Configuração como Agente Separado OpenClaw

🟡 Médio — 1-2h

Criar segunda instância OpenClaw para JANA no mesmo Mac Mini, com Telegram bot separado (@Jana_mdbot) e configuração própria.

  • Verificar se OpenClaw suporta múltiplas instâncias no mesmo host
  • Criar ~/.openclaw-jana/ com openclaw.json próprio
  • Porta separada (ex: 18790 vs 18789 do JARBAS)
  • Ou: usar o mesmo OpenClaw com roteamento por bot token
4

Comunicação Direta Entre Agentes via Supabase

🔵 Fase futura

JARBAS e JANA podem se "avisar" de tarefas via tabela jarbas_mensagens_agentes. Polling a cada 30s. Possibilita delegação automática sem precisar de comando do Christian.

  • JARBAS enfileira tarefa para JANA quando está sobrecarregado
  • JANA avisa JARBAS quando concluiu algo relevante
  • Base para futura orquestração multi-agente autônoma
