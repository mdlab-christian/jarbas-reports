📋 Plano de Correção — Não executar ainda
PC-01
Push JARBAS — 55 commits pendentes
Problema: SSH git push trava por timeout. Repo local está 55 commits à frente.
Causa provável: Chave SSH expirada, instabilidade de rede, ou repo remoto tem divergência.
Correção:
1. cd ~/jarbas && ssh -T git@github.com — verificar se SSH funciona
2. git fetch origin && git log origin/main..HEAD --oneline | wc -l — confirmar divergência
3. Se SSH OK: GIT_TRACE=1 git push --verbose 2>&1 | tee /tmp/push.log
4. Se repo divergiu: git push origin main --force-with-lease
Esforço: 15min | Prioridade: P0
PC-02
Verificar bucket 'comprovantes' — URL pública
Problema: skill_gerar_comprovante usa getPublicUrl() — se bucket for privado, URL não abre.
Correção:
1. Supabase Dashboard → Storage → bucket 'comprovantes' → verificar se é público
2. Se privado: criar signed URL com supabase.storage.from('comprovantes').createSignedUrl(path, 86400) e salvar no arquivo_url
3. Ou: tornar bucket público via UPDATE storage.buckets SET public=true WHERE name='comprovantes'
Esforço: 20min | Prioridade: P0 (URL do comprovante pode estar inacessível)
PC-03
Testar E2E petição via JARBAS + ModalGerarPeticao
Problema: skill_olivia_gerar_peticao foi corrigida mas não E2E testada.
Correção:
1. Executar direto: cd ~/jarbas && node --input-type=module → importar skill + passar tarefa com processo_id real
2. Verificar olivia_modelos_peticao tem templates compatíveis com o processo de teste
3. Verificar se JARBAS atualiza processos.peticao_status=sucesso + processos_documentos=disponivel
4. Testar via browser: /distribuicao → processo → Gerar Petição (ModalGerarPeticao)
Esforço: 1h | Prioridade: P1
PC-04
Race condition: processos_documentos status no refetch
Problema: Quando frontend faz INSERT jarbas_tarefas + pre-cria processando, o refetchInterval é 3s. Mas se JARBAS demorar >3s para pegar a tarefa, há um ciclo extra de "gerando" antes de realmente começar.
Correção:
1. Aumentar timeout de polling para 5s (JARBAS poll interval é 5s)
2. Ou: adicionar campo jarbas_tarefa_id em processos_documentos para rastrear qual tarefa está processando
3. Garantir que refetchInterval detecta status='processando' (já corrigido)
Esforço: 30min | Prioridade: P2
PC-05
useGerarDocumentosBulk — migrar para JARBAS
Problema: Hook ainda usa n8n para geração em lote (WH-DOC-LOTE). Não coeso com a nova arquitetura.
Correção:
1. Ler o hook atual e entender uso em distribuição
2. Substituir por N INSERTs em jarbas_tarefas (1 por processo, JARBAS processa sequencialmente)
3. Adicionar feedback visual de progresso no lote
Esforço: 2h | Prioridade: P2
PC-06
Fallback comprovante se JARBAS falhar
Problema: n8n WH-DOC-COMPROVANTE está INATIVO. Sem fallback para comprovante.
Opção A (recomendada): Adicionar timeout de 120s no frontend — se processos_documentos não chegar a disponivel, mostrar alerta + link para contato.
Opção B: Reativar n8n comprovante com uma API HTML-to-PDF alternativa (browserless.io free tier ou gotenberg self-hosted).
Opção C: Keep as-is — JARBAS estável, risco aceitável por enquanto.
Esforço: 1-4h dependendo da opção | Prioridade: P2
