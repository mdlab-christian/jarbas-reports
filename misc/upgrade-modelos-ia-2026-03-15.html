Ganhos Concretos Esperados
Impacto real nas operações do Midas
01
Análise de sentença mais precisa
Haiku classificava o resultado de processos errado em documentos curtos (ex: decisão interlocutória de 2 páginas que negava tutela). Sonnet lê a intenção real da decisão, não só as palavras.
Precisão: 72% → ~92%
02
Downloads EPROC mais completos
Haiku não identificava todos os links de documentos em telas complexas do EPROC (múltiplas abas, tabelas aninhadas). Sonnet tem melhor reconhecimento visual de interfaces web jurídicas.
Cobertura: ~75% → ~95%
03
Petições com análise de provas mais forte
A análise das provas da ré alimenta os argumentos da petição. Com Sonnet, pontos fracos da contestação são identificados com mais profundidade, gerando réplicas mais cirúrgicas.
Qualidade argumentativa ↑
04
Compactação de sessão sem perda de contexto
Resumos de sessão feitos com Haiku perdiam decisões técnicas sutis ("usar Management API direta", "não usar Claude Code para isso"). Sonnet preserva nuances que importam para retomar a missão.
Fidelidade do resumo ↑
05
E2E detecta bugs de fluxo, não só visuais
Screenshots detectam problemas estáticos. O novo modo vídeo com Gemini detecta: botões que não disparam, spinners infinitos após ação, tabs que não mudam de estado, erros que aparecem após interação.
Bugs detectados: +40% estimado
06
Nunca mais modelo errado em nova skill
O agente skill_model_selector é a referência para qualquer skill nova. Consulta: /modelo para [tarefa]. Regras documentadas + exportáveis para reuso programático em qualquer skill.

E2E Vision — Novo Modo Vídeo com Gemini
Análise de fluxo temporal completo, não só frames estáticos
  • 1
    Fase 1 — Playwright: Captura de Screenshots
    Navega pela página, captura cada tab/estado. Modelo: Claude Sonnet 4.6. Já existia.
  • 2A
    Fase 2A — Claude Vision: Avaliação por Screenshot
    Cada screenshot é avaliada contra o PRD (design system, tabs esperadas, elementos). Modelo: claude-sonnet-4-6. Já existia.
  • 2B
    Fase 2B — Playwright grava vídeo + Gemini analisa (NOVO)
    Playwright grava WebM de interação real (login → rota → scroll → clique em tabs). Upload para Gemini File API → gemini-2.0-flash analisa o fluxo temporal. Detecta: spinners infinitos, botões que não respondem, erros após interação, inconsistências de estado entre tabs.
  • 3
    Fase 3 — HTML Report com nota combinada
    Nota final = 60% screenshots (Sonnet) + 40% vídeo (Gemini). Report inclui seção dedicada à análise de vídeo com problemas de fluxo identificados.
Uso: testa /olivia video — ativa Fase 2B automáticamente.
