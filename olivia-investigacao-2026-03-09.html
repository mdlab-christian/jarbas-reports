P0 — Crítico Loop de regeneração não usa feedback do revisor
📄 ~/jarbas/src/skills/skill_olivia_gerar_peticao.mjs — linha 254
Quando o score do revisor é < 7, o código tenta adicionar o feedback ao próximo prompt — mas não reassigna a variável prompt. A expressão prompt + "..." cria uma string temporária que é descartada. Resultado: as 2 tentativas de regeneração usam o mesmo prompt, desperdiçando tokens e não melhorando a qualidade.
// Loop de geração — tentativa ${tentativas} if (!revisaoResult.regenerar || scoreFinal >= 7) break; log(`Score < 7, regenerando com feedback...`); - prompt + `\n\nFEEDBACK DO REVISOR (melhorar):\n${revisaoResult.feedback}\nPontos fracos: ${revisaoResult.pontos_fracos?.join(', ')}`; + prompt = prompt + `\n\nFEEDBACK DO REVISOR (melhorar):\n${revisaoResult.feedback}\nPontos fracos: ${revisaoResult.pontos_fracos?.join(', ')}`;
🔧 Fix: 1 caractere (=) · Impacto: qualidade das regenerações sobe imediatamente · ETA: 5min
P1 — UI Typo quebra classe CSS no KpiCard
📄 ~/beta-mdflow/src/pages/olivia-v2/index.tsx — KpiBanner
Aspas extras na className do label inferior do KpiCard causam parsing incorreto do Tailwind. Pode não renderizar a cor e espaçamento corretos dos labels dos KPIs.
- className="mt-0.5 text-[9px] uppercase tracking-widest text-muted-foreground' + className="mt-0.5 text-[9px] uppercase tracking-widest text-muted-foreground"
🔧 Fix: remover ' extra no final · ETA: 2min
P2 — UX VibeLaw Studio com dados hardcoded
📄 ~/beta-mdflow/src/pages/olivia-v2/index.tsx — VibelawStudio component
O VibeLaw Studio (Step 4) tem mensagens de chat mockadas no useState inicial. Não recebe o historicoId da petição gerada, não carrega o HTML real. O hook useVibeLaw existe mas não está conectado à petição do wizard.
📋 Fix: passar historicoId como prop ao VibelawStudio + conectar useVibeLaw · ETA: 2-3h lapidação
P2 — Dados olivia_modelos.perguntas_triagem pode estar vazio
📄 supabase: tabela olivia_modelos
A skill de triagem busca perguntas_triagem do modelo selecionado. Se o campo estiver null/empty, o JARBAS não tem perguntas para responder e a triagem retorna vazia — o Step 2 ficará sem campos. Precisa verificar se os 14 tipos de petição têm perguntas_triagem seeded.
📋 Fix: query rápida no Supabase para verificar + seed das perguntas padrão por tipo
P3 — Info jarbas_style_guide não tem mecanismo de preenchimento
📄 skill_olivia_gerar_peticao.mjs + tabela jarbas_style_guide
A skill consulta o style guide do advogado para personalizar o estilo da petição (tom, forma de citação, abertura de seções). A tabela existe mas nunca é populada — nenhuma skill faz o "deep analysis" das petições aprovadas para gerar o guide. O código tem um fallback gracioso (usa valores padrão), mas a personalização real nunca acontece.
📋 Futuro: criar skill_olivia_gerar_style_guide que roda semanalmente nos aprovados

Estimativa para lapidação completa (L1→L5): 6-8h de cc: — split em 2 sessões, começando pelos hotfixes e terminando com teste real de ponta a ponta.
