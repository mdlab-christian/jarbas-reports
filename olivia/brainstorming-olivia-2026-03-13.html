🧱 Smart Blocks — Modo Legado / Backup

Conceito: Modo mecânico. Tags = perguntas ordenadas. Blocos de texto = respostas. IA não gera — apenas monta. Inspirado no Typeform antigo. Versão advogado determina qual bloco usar.

Pergunta: Pode rodar sem N8N? (Só frontend + Supabase)

✅ SIM — Smart Blocks pode rodar 100% sem N8N

O mecanismo é: buscar tags do modelo → apresentar em ordem → usuário responde → buscar bloco de texto correspondente à tag + versão → montar texto concatenado. Tudo isso é queries Supabase + frontend puro. O único momento que precisava de N8N era para gerar o Google Doc via Apps Script.

Nova abordagem: geração do documento final via edge function ou JARBAS (converte texto concatenado → DOCX via biblioteca + salva no bucket peticoes/). Zero N8N necessário.

Decisão: 1 Super-Workflow ou Separados?

1 Workflow (ideia do Christian)
  • ✓ Manutenção centralizada
  • ✓ Lógica de roteamento clara
  • ✗ N8N não processa em paralelo por padrão → pode engarrafar com múltiplos usuários
  • ✗ Um bug paralisa todos os modos
2 Workflows Separados (Recomendado)
  • ⭐ OLIVIA-ANALISE: análise de provas (independente)
  • ⭐ OLIVIA-GERADOR: triagem + Express + Smart Blocks
  • ⭐ Execução paralela real (cada webhook tem sua queue)
  • ⭐ Falha em um não afeta o outro
  • ⭐ Ainda é só 2 para manter

Resposta sobre paralelismo N8N: N8N Community/Cloud processa webhooks em paralelo (cada request abre uma execução independente). Então 1 ou 2 workflows — ambos suportam múltiplos usuários simultâneos. A separação é por clareza e resiliência, não por limitação técnica.
