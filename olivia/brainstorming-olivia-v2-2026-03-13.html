Regra de Jurisprudências (todos os modos)

  1. Busca: assunto + advogado_id + estado_processo → usa esta versão
  2. Fallback 1: assunto + advogado_id (qualquer estado) → mais recente
  3. Fallback 2: assunto (qualquer advogado e estado) → mais recente
  4. Sem nenhuma → campo informativo no relatório ("sem jurisprudência cadastrada para este assunto")

As jurisprudências são substituídas automaticamente como placeholders no template — o usuário nunca seleciona manualmente.

🧱 Smart Blocks — Frontend + Supabase + Apps Script

Modo legado/backup. Mecânico e determinístico. Tags = perguntas em sequência Typeform. Blocos = respostas. IA não gera — apenas monta e substitui. 100% sem N8N.
STEP 1
Identificação
Processo + Tipo + Modelo + Versão/Advogado
STEP 2
Perguntas
Tags em ordem (Typeform). Uma por vez.
STEP 3
Revisão
Preview texto montado + observações livres
STEP 4
Conclusão
Apps Script gera Google Doc → link + salvar historico

Step 1 — Identificação: campos detalhados

Busca de processo
Search box com debounce. Após selecionar → painel resumo: nº CNJ, cliente, empresa ré, valor restrição, data restrição, código contrato, estado.
Tipo de petição
Select com os 10 tipos ativos (filtrado por ativo:true). Obrigatório.
Modelo de petição
Select filtrado por: tipo_peticao_id + categoria_empresa do processo. Mostra nome do modelo. Obrigatório.
Versão / Advogado
NOVO — Select dos advogados da org. Determina qual bloco de texto apresentar. Obrigatório.

Step 2 — Perguntas (Tags): comportamento

  • Tags buscadas do modelo selecionado → ordenadas por olivia_tags.ordem ASC
  • Apresentadas uma por vez (estilo Typeform) com animação de transição
  • multipla_selecao:true → checkbox (pode marcar N opções)
  • multipla_selecao:false → radio/select (apenas 1 opção)
  • selecao_obrigatoria:true → não avança sem responder; botão "Pular" oculto
  • Para cada tag respondida → sistema já sabe qual bloco de texto buscar (tag_id + advogado_id + categoria_empresa_id)
  • Tipos sem tags → pula direto para Step 3 com campo de observação livre
  • Progresso visual: "3 de 12 perguntas"

Step 3 — Revisão: campos

Preview texto
Texto montado com os blocos selecionados. Só leitura. Scroll. Formatação básica.
Instruções adicionais
Textarea grande — campo livre para o advogado adicionar observações para o Apps Script. Ex: "Incluir parágrafo sobre dano moral majorado".
Jurisprudências
Lista auto-selecionada (assunto → advogado → estado → fallback). Somente leitura. O advogado vê quais foram incluídas mas não precisa selecionar.

Step 4 — Conclusão: fluxo de geração

  1. Frontend chama JARBAS com: blocos_ids[], juris_ids[], modelo_id, advogado_id, instrucoes, processo_id
  2. JARBAS chama Google Apps Script via HTTP POST com payload JSON
  3. Apps Script: monta Doc base a partir do template Google Doc do modelo → substitui cada tag pelo Google Doc do bloco correspondente → insere jurisprudências nos placeholders → retorna URL do Google Doc gerado
  4. JARBAS salva em olivia_historico: google_doc_url, status:SUCESSO, modo:MANUAL, motor:smart-blocks
  5. Frontend apresenta: link do Google Doc + botão copiar + link para o processo

🚀 Modo Express — N8N como Motor

Evolução do Smart Blocks. A IA (Gemini análise + Claude geração) responde as tags automaticamente. Só pergunta ao usuário o que ficou em dúvida. N8N roda no Railway — funciona mesmo se Mac Mini cair.
STEP 1
Identificação
Processo + Tipo + Modelo. Painel resumo do processo.
STEP 2
Documentos
Docs JARBAS (by evento) + upload manual
STEP 3
Análise IA
Auto ao avançar. Gemini analisa. Relatório gerado.
STEP 4
Triagem
Perguntas da IA + relatório visível + observação livre
STEP 5
Geração
Claude gera petição. Progress realtime. Google Doc link.

Step 1 — Identificação

Busca de processo
= Smart Blocks Step 1. Search + painel resumo: valor, data, empresa, estado, código contrato.
Tipo de petição
Select. Obrigatório.
Modelo de petição
Select filtrado por tipo + categoria empresa.
Versão/Advogado
Select. Determina o style guide para a IA e versão dos blocos de referência.

Step 2 — Documentos

Docs JARBAS
Lista agrupada por evento EPROC (ex: "Contestação — 15/01/2026 [3 arquivos]"). Checkbox para selecionar evento inteiro ou arquivo individual.
Upload manual
Drag & drop ou botão. Aceita PDF/imagem. Campo "Tipo de documento" (select: contestação, prova, extrato, tela, decisão, etc.).
Bloqueio
Não avança sem ao menos 1 documento. Mensagem clara: "Selecione ou envie os documentos para análise".

Step 3 — Análise IA

Disparo
Automático ao avançar do Step 2. Sem botão manual. N8N recebe docs → OLIVIA-ANALISE processa.
Loading
Tela de carregamento com barra de progresso atualizada via realtime Supabase (meta.progresso_key). Mensagens: "Extraindo documentos... Analisando provas... Preparando relatório..."
Relatório
Apresentado em seções com negrito. Ex: "Análise da Contestação", "Provas da Empresa Ré", "Pontos de Impugnação", "Valor Cobrado vs. Valor Real". Texto humanizado, não JSON.
Falha
Se análise falhar → NÃO avança. Mensagem de erro + botão retry. A triagem só funciona se a análise foi concluída.

Step 4 — Triagem

Relatório visível
Seção superior com o relatório do Step 3 (colapsável). O advogado pode ler sem sair da tela.
Perguntas da IA
Geradas pelo N8N baseadas na análise. Tipos: opções de clique (radio/checkbox) + SEMPRE textarea de observação em cada pergunta.
Inputs de texto
Textarea grande (min 4 linhas). Botão de áudio em todos os campos. Nunca input de linha única.
Chat OlivIA
Drawer à direita disponível para dúvidas contextualizadas (thread do processo).
Campo livre
Área de observações gerais ao final: "Alguma instrução adicional para a petição?"

Step 5 — Geração

Fluxo N8N
Respostas da triagem → OLIVIA-GERADOR → Claude recebe: template do modelo + blocos Smart Blocks como base + respostas da triagem + style guide do advogado → gera petição melhorada → Apps Script formata no Google Doc
Progresso
Barra realtime via olivia_historico.progresso. "Gerando petição... 40%... Revisando... 80%... Concluído!"
Saída
Link do Google Doc gerado. Botão "Abrir no Google Docs". Botão "Ver no processo". Resumo salvo em olivia_historico.

OLIVIA-ANALISE

Geração da petição. Recebe respostas da triagem + contexto → Claude gera com base no modelo + blocos → Apps Script formata.

Webhooks: olivia/gerar (Express) + olivia/gerar-manual (Smart Blocks)
Input: respostas triagem, modelo_id, blocos_ids[], advogado_id
Output: google_doc_url + historico_id
