⚙️ O que foi construído

📄
POST /olivia/docx-converter NOVO
Recebe HTML ou Markdown da petição gerada → converte com pandoc 3.9 usando template ABNT → faz upload para bucket peticoes no Supabase Storage → gera signed URL (1h) → faz UPDATE no olivia_historico com pdf_url, pdf_path, status=SUCESSO.
Path no Storage: org_id/historico_id/filename.docx
📤
POST /olivia/upload-documento NOVO
Recebe arquivo via multipart/form-data → parser nativo Node.js (zero dependências externas) → valida tipo (pdf/doc/docx/txt) e tamanho (max 10MB) → salva em bucket olivia-provas → retorna signed URL 1h.
Path: org_id/processo_id/timestamp-filename.ext
🔧
parseMultipart() NOVO
Parser multipart/form-data implementado do zero em Node.js puro. Sem dependências externas. Suporta múltiplos fields + files, boundary com quotes, detecção de Content-Type.
🤖
N8N dGw50wQ4xjdsjR3n — Google Drive removido CORRIGIDO
6 nós Google Drive removidos: Criar Google Doc, Inserir Texto, Drive Prep, Ensure Folder, Move Doc, Salvar Drive ID.
Substituídos por HTTP: DOCX Converter → localhost:3012/olivia/docx-converter.
Sequência final conforme SPEC 13.1:
Prompt OK?→ Claude Gerar Petição→ Processar Resposta→ Salvar Histórico
EM_PROGRESSO + texto_gerado + peticao_html→ DOCX Converter
