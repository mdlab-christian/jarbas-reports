Gaps e Riscos
🔴 CRÍTICO — versao_id pode bloquear todos os usuários
Se olivia_versoes estiver vazia em produção, versao_id = null e canAdvance = false. O usuário não consegue nem clicar em "Avançar" no Step 1. Não tem fallback gracioso.
Fix: em OliviaGeradorIA.tsx, checar versoes.length === 0 — se vazio, não exigir versao_id no canAdvance. OU: no useWizardTabs.nextStep(), o auto-resolve já tenta buscar versao por modelo_id — se funcionar, bastaria mover canAdvance para não exigir versao_id quando modelo_grupo_id está preenchido.
🟡 MENOR — Autosave 30s no StepTriagem ausente
Spec §07.2 item 5: "Autosave a cada 30s em olivia_historico.meta.wizard_state". StepTriagem não tem debounce de save. Se usuário fechar o browser no meio das perguntas, perde as respostas.
Fix: adicionar useAutosave hook no StepTriagem. Já existe o hook, só falta wiring.
🟡 MENOR — Upload documentos não está no StepTriagem
Spec §07.2 item 4: "Aceita upload de documentos (max 10MB, PDF/DOC/DOCX/TXT)" — no StepTriagem, não no Step de Identificação. O upload atual está em StepIdentificacao (UploadProvasInline). Está no lugar errado.
Para C1b: mover ou duplicar o upload para o rodapé do StepTriagem, junto com instrucoes_adicionais.
ℹ️ INFO — RecoveryCard component dedicado não usado
O componente shared/RecoveryCard.tsx existe mas não é renderizado. A Landing usa lógica inline (tab bar + painel). Funcionalmente equivalente, mas o componente ficou órfão.
Não é blocking. Pode usar inline ou migrar para RecoveryCard — decisão arquitetural.
