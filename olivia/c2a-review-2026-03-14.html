📦 O que foi implementado nesta sessão

Step 1
Identificação
Seletor de processo (Popover + search debounce), tipo de petição, advogado. Reset dos blocos ao trocar processo ou tipo.
Step 2
Seleção de Blocos (DnD)

🐛 Bugs encontrados e corrigidos

🔴 BUG-SB-01 — Status filter errado

Em SmartBlocksStep2Blocos, a query filtrava .eq('status', 'ativo').

Os valores reais em olivia_blocos_texto são draft | published | archived — nenhum bloco seria retornado com 'ativo'.

// ANTES (bugado)
.eq('status', 'ativo')

// DEPOIS (correto)
.eq('status', 'published')

Corrigido em a3b8791. Impacto: teria exibido "Nenhum bloco encontrado" para todos os usuários.

🔴 BUG-SB-02 — Props inexistentes no PageHeader

Passei description, icon e breadcrumbs (plural) para <PageHeader>.

O componente aceita apenas: title, breadcrumb (singular ReactNode), actions, showOlivIA, oliviaContext.

TypeScript não acusou erro porque props extras são ignoradas silenciosamente em JSX com tipos strict — mas o build teria gerado warnings. Corrigido em a3b8791.
