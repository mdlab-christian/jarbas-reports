🔧 Bugs corrigidos nesta sessão
✅
Bug crítico: SSE invisível quando thread tinha histórico
messages useMemo preferia messagesData (DB) ao invés de localMessages. Resultado: durante streaming, a bolha do assistant não aparecia se a thread já tinha mensagens anteriores.
Fix: localMessages é agora fonte única, hidratada via useEffect do DB. Streaming sempre visível.
✅
Gap spec: JURIS_TERMS sem botão "Copiar citação"
Spec 07.8 exige botão. Adicionado: copia todos os termos formatados como citação jurídica para clipboard via navigator.clipboard.
✅
Gap spec: NEWS_RESEARCH sem badge relevância
Spec 07.8 exige badge. Adicionado: RelevanceBadge com 3 níveis (alta/média/baixa) com cores verde/âmbar/cinza.
✅
Breakpoint errado: studio só mostrava split-view em 1280px+
Spec diz mobile em <640px. Corrigido para 768px — tablets veem o split-view correto. Mobile (<768px) continua com tabs Chat | Petição.
✅
Query StepProcesso com join PostgREST inválido
.select('clientes!processos_cliente_id_fkey(nome)') + .or('clientes.nome.ilike...') falha silenciosamente no PostgREST.
Fix: usa cliente_nome, empresa_nome direto da tabela processos (colunas verificadas existem).
✅
Documento não persistia no banco (refresh = perda total)
documentoAtual vivia apenas em estado local. Refresh = sumia.
Fix: autosave com debounce 5s → olivia_threads.documento_atual. Na abertura do studio, carrega da thread se existir.
Também criado endpoint POST /olivia/vibelaw-apply-patch no JARBAS que persiste o documento após cada patch aplicado.
✅
Endpoint /olivia/vibelaw-apply-patch inexistente no JARBAS
O applyPatch chamava callWebhook() que roteia para N8N (workflow "Verificar" — potencialmente inexistente).
Fix: usa fetch diretamente para o JARBAS endpoint recém-criado. Fire-and-forget, não bloqueia a UI.
