Apps Script — Entendimento completo

  • 📥 Input: secoes[] (conteúdo já montado), tags_substituicao{}, google_folder_id, titulo_documento, config{}
  • 📤 Output: google_doc_url, google_doc_id, estatisticas{}
  • 🎨 Formatação: Markdown-lite (**bold**, *italic*, ## heading) → Google Docs nativo. Padrão ABNT forense.
  • 🏷️ Tags: {{cliente_nome}} ou {{cliente.nome}} → substituídos. Tags não encontradas → [REVISAR: nome] em vermelho.
  • 🔄 Idempotência: Janela de 60s via PropertiesService. Retry automático com backoff.
  • ❗ NÃO faz: Não busca conteúdo do Google Drive. Quem monta as secoes[] é o chamador (N8N/JARBAS/Edge Function).

Recomendação: Loop próprio via PRD, sem skill de lapidação dedicada.
A skill de lapidação genérica foi feita para páginas isoladas. Este projeto é um sistema completo (frontend + N8N + JARBAS + Supabase + Apps Script). O loop deve ser guiado pelo PRD de execução, onde cada tarefa especifica: arquivo a editar, teste a executar, critério de conclusão. JARBAS executa task por task usando write/edit/exec diretamente. Mais previsível, mais rastreável.

Loop por tarefa do PRD

  5. Atualiza PRD: marca tarefa como ✅ concluída + nota o que foi feito

  7. Notifica Christian no Telegram: "✅ [tarefa] — próximo: [o que vem]"

💬 O Que Vai Melhorar nos Prompts e Playbooks

olivia_triagem — Melhoria crítica

Adicionar ao output JSON o campo blocos_sugeridos:

"perguntas": [
  {
    "id": "q1",
    "pergunta": "Qual o produto contratado?",
    "tipo": "opcoes",
    "opcoes": ["Empréstimo", "Cartão de crédito", "Conta corrente"],
    "blocos_sugeridos": {
      "Empréstimo": ["uuid-bloco-emprestimo-arg1", "uuid-bloco-emprestimo-arg2"],
      "Cartão de crédito": ["uuid-bloco-cartao-arg1"]
    }
  }
]

Isso permite ao Express selecionar o bloco correto automaticamente (Smart Blocks automatizado).
