Fluxo detalhado — Workflow OlivIA Principal (101 nodes)

Entrada: Webhook recebe payload completo (processo_context, triagem_respostas, blocos, modelo, versão)
Etapa 1 — Criação Thread: OpenAI Create Thread → Create Message com contexto completo
Etapa 2 — Execução: Create Run → loop Wait → Get Run Status → If (completo/pendente) → Get Messages
Etapa 3 — Análise docs: Se tem documentos → PDF.co OCR → baixa .txt → processa em batches
Etapa 4 — Triagem: Loop Over Items → Create Message triagem → Create Run3 → polling status
Etapa 5 — Análise processo: Sub-workflow Execute (N8N-PROC-OLIVIA-RESUMO) → GPT extrai dados
Etapa 6 — Redação: Switch tipo (decisão desfavorável/favorável/despacho) → GPT Message específico
Etapa 7 — Revisão: Agente principal Thread → Create Message revisão → polling
Saída: Supabase UPDATE olivia_historico status=SUCESSO + texto_gerado + meta

⚠️ Ponto crítico: A qualidade da petição depende do contexto passado na Etapa 1. Se dados_processo chega vazio, as etapas 5 e 6 não têm base para gerar conteúdo jurídico específico.

🎯 7. Plano de Lapidação Final

📌
Estratégia: Resolver primeiro os P0 que causam falha de geração (B1 conteúdo genérico + B2 RLS + B3 blocos repetidos), depois os P1 de UX/feedback, e por último os refinamentos de interface. Cada fase com E2E Vision ao final.

⚡ FASE 1 — Correções P0 (estimativa: 3-4h)

  • 🔴
    B1 — Enriquecer dados_processo antes do envio ao N8N P0
    Em useGerarPeticao, garantir que dados_processo tem joins completos (clientes, empresas, advogados). Se processo_id existe mas dados_processo está sem joins → fazer query adicional antes de buildProcessoContext(). Arquivo: useGerarPeticao.ts + useStepIdentificacaoData.ts
  • 🔴
    B2 — Corrigir RLS jarbas_tarefas para Modo Agente P0
    Migration Supabase: CREATE POLICY "olivia_insert" ON jarbas_tarefas FOR INSERT TO authenticated WITH CHECK (organizacao_id = auth.jwt()->'organizacao_id'); ou usar função service_role para o INSERT da triagem.
  • 🔴
    B3 — Corrigir filtro blocos Smart Blocks (sem repetição) P0
    SmartBlocksStep2Blocos.tsx: query deve fazer JOIN olivia_modelo_tags → olivia_blocos_texto WHERE tag_id IN (tags_do_modelo) + DISTINCT para evitar duplicatas. Remover seleção manual de jurisprudências (ficam nos placeholders dos blocos).
  • 🔴
    B8 — Triagem personalizada com contexto de provas P0
    StepTriagem.tsx: passar processo_context + provas_urls[] no payload do webhook triagem. N8N deve usar esses dados para gerar perguntas contextuais específicas ao caso.

  • /clear — salva estado atual + instrui nova conversa. Usar quando contexto ~80%
  • retoma contexto olivia — JARBAS lê arquivo salvo e continua missão
  • Arquivo de projeto: ~/jarbas/memory/project--5219977942-olivia-rebuild.md
  • Para lapidações longas (4+ fases), JARBAS abre nova conversa a cada fase automaticamente
