Gaps Extras — Fora do PRD
7 findings
EX-01 Smart Blocks — bug P0-01 presente (modelos duplicados) 🔴 Crítico
ManualStep1Processo busca de olivia_modelos_peticao sem DISTINCT — mesma tabela que causava os 328 modelos duplicados no Express. O PRD corrigiu o Express (migrou para olivia_modelos_grupo), mas o Smart Blocks ficou para trás.
📁 src/pages/olivia/components/gerador/manual/ManualStep1Processo.tsx:63
Fix
Migrar query para olivia_modelos_grupo com filtro tipo_peticao_id + status='published', igual ao Express.
EX-02 Smart Blocks — modelo_id (versão) vs modelo_grupo_id (grupo) — mismatch 🔴 Crítico
ManualStep1 salva em state.modelo_id um ID de olivia_modelos_peticao (versão). Mas ManualStep2Conteudo busca olivia_modelos_grupo.tags_ids usando esse mesmo state.modelo_id como se fosse o grupo. Resultado: a busca de tags/blocos pode falhar ou retornar dados errados.
📁 ManualStep2Conteudo.tsx:26 — busca olivia_modelos_grupo WHERE id = state.modelo_id
Fix
Após migrar para olivia_modelos_grupo (EX-01), o state.modelo_id passará a ser o grupo ID — mismatch se resolve automaticamente.
EX-03 Jurisprudências — status 'ativo' vs 'published' 🟠 Alto
ManualStep2 busca jurisprudências com status = 'ativo', mas todos os outros recursos (blocos, modelos, playbooks) usam status = 'published'. Se o banco usa published, a query nunca retorna resultados — zero jurisprudências na UI.
📁 ManualStep2Conteudo.tsx:96 — .eq('status', 'ativo')
Fix
Verificar valor real no banco. Provavelmente mudar para .eq('status', 'published') ou aceitar ambos: .in('status', ['ativo', 'published'])
EX-04 Smart Blocks sem recovery/autosave 🟠 Alto
Express tem sistema completo de recovery: cria olivia_historico ao avançar do Step 1, salva estado no meta.wizard_state, permite retomar. Smart Blocks usa estado React local puro — se o usuário fechar a aba ou navegar para outra página, perde tudo o que selecionou.
📁 OliviaGeradorManual.tsx — useState local sem persistência
Fix (sugestão)
Criar olivia_historico com modo='MANUAL' ao avançar do Step 1, salvar blocos_ids e jurisprudencias_ids no meta. Reutilizar pattern do Express.
EX-05 Express — busca de processo filtra só CNJ (não nome do cliente) 🟠 Alto
useStepIdentificacaoData filtra por numero_cnj.ilike apenas. Smart Blocks filtra por numero_cnj OR cliente_nome. Usuário do Express que digitar o nome do cliente não encontra o processo — comportamento inconsistente.
📁 useStepIdentificacaoData.ts:18 — query processos
Fix
Adicionar .or(`numero_cnj.ilike.%${s}%,cliente_nome.ilike.%${s}%`) na query, igual ao Smart Blocks.

O PRD (P1-04) pedia botão de download no Modo Agente. Existe "Abrir no VibeLaw" e "Relatório HTML", mas não um botão direto para o Google Doc gerado. Express e Smart Blocks também não expõem download direto.
📁 OliviaGeradorJarbas.tsx — StepConclusaoJarbas
