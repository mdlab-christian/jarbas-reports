🔴 Problemas Identificados

Problema #1 — categoria_empresa_id NULO em 100% dos modelos

TabelaCampoCom valorSem valorImpacto

Root cause: A importação copiou os dados do banco antigo mas não mapeou o campo categoria_empresa_id (que no banco antigo era derivado do nome da empresa no processo, não uma FK explícita). O campo existe no schema novo mas ficou null.

Problema #2 — olivia_modelo_juris vazia (0 registros)

Os vínculos entre modelos e jurisprudências não foram importados. As 1.000+ jurisprudências estão no banco mas desvinculadas de qualquer modelo.

Impacto: Modo Express e Modo Agente não encontram jurisprudências contextuais para enriquecer as petições geradas.

Problema #3 — Tags duplicadas sem vínculo de tipo_peticao

Tags com nomes iguais existem intencionalmente (ex: "Preliminares" para BV vs "Preliminares" para Telefonia). Correto. Mas sem o secao_id e sem o vínculo por categoria_empresa_id, o SmartBlocks não consegue filtrar quais tags mostrar para cada tipo de empresa.

⚖️ Veredicto: Reimportar ou Corrigir In-Place?

❌ Opção A — Reimportar do zero

DELETE em cascata de todos os dados OlivIA + reimportação do banco antigo com mapeamento correto.

  • Risco alto: histórico de geração (olivia_historico) zerado
  • Threads de chat perdidas
  • Análises existentes perdidas
  • Tempo: 2-3h de importação completa
  • Só necessário se os dados estiverem corrompidos estruturalmente
  • Parte boa: banco limpo, sem lixo acumulado
Conclusão: NÃO necessário. Os dados estão corretos — só faltam vínculos.

✅ Opção B — Correção cirúrgica in-place

UPDATEs direcionados para preencher os campos vazios sem apagar nada.

  • Histórico e threads preservados
  • Dados corretos — só vínculos faltando
  • Tempo: ~30 min de SQL
  • Reversível (pode desfazer se errar)
  • Sem risco de perda de dados
Conclusão: ✅ Esta é a solução definitiva. Corrigir os 4 problemas via SQL.

🛠️ Plano de Execução — Correção Cirúrgica

1

Popular categoria_empresa_id em olivia_modelos_grupo via nome

Detectar padrões no nome do modelo ([BV], [Lojas], [Telefonia]...) e fazer UPDATE com os IDs corretos de empresas_categorias. Modelos sem padrão detectável ficam com NULL (genéricos, aparecem para todos).

SQL:
UPDATE olivia_modelos_grupo SET categoria_empresa_id = '1bf519a7...' WHERE nome ILIKE '%[BV]%' OR nome ILIKE '%BV Financ%';
× 6 categorias detectáveis
2

Popular categoria_empresa_id em olivia_blocos_texto via tag_ids

Cada bloco tem tag_ids[]. Cada tag tem o nome com o padrão de categoria (ex: "(Serviço de crédito) Preliminares"). Fazer UPDATE nos blocos herdando a categoria da sua tag principal.

Estratégia: JOIN entre olivia_blocos_texto.tag_ids[1] e olivia_tags.nome → detectar categoria pelo nome da tag → UPDATE categoria_empresa_id.
3

Popular secao_id nas 10 tags órfãs

As 10 tags sem secao_id precisam ser vinculadas à seção correta de olivia_tags_secao. UPDATE direto por código da tag.

4

Importar vínculos olivia_modelo_juris

Reconstruir os vínculos modelo↔jurisprudência a partir do banco antigo. Se o banco antigo tiver a tabela equivalente, é um INSERT direto com mapeamento de IDs. Se não tiver, usar as juris_ids[] nos blocos como proxy para inferir quais jurisprudências cada modelo usa.

Nota: Verificar se olivia_blocos_texto.juris_ids já contém os vínculos (campo existe no schema).
5

Validar filtragem no SmartBlocks

Após os UPDATEs, testar: Step 1 → selecionar empresa "BV" → modelos devem filtrar para Instituições Financeiras → Step 2 → tags e blocos devem aparecer filtrados corretamente.
