FASE 1 — Página Playbooks (base da OlivIA)

Executar primeiro. Playbooks alimenta OlivIA — a base tem que estar certa antes do gerador.

⚠️

Por que Playbooks primeiro?

O SmartBlocks, Express e Modo Agente dependem dos dados de Tipos, Modelos, Tags, Blocos e Jurisprudências corretamente vinculados. Se a base está errada, todos os geradores falham. Consertar a fundação primeiro.

T1.1 — Corrigir ordem das tabs P0 Frontend
~15 min
Reordenar as tabs da página Playbooks seguindo a hierarquia SmartBlocks: Tipos → Modelos → Tags → Blocos → Jurisprudências → Playbooks → Agentes IA → Gerador IA → Aprovações.

A aba "Tipos" é o ponto de partida — tudo parte do tipo de petição. Após tipos vêm os modelos vinculados, depois as tags do modelo, depois os blocos de cada tag, depois as jurisprudências. Essa ordem guia o usuário na configuração do sistema.
src/pages/playbooks/index.tsx — array TABS[]
T1.2 — Corrigir vinculação Tags → Modelo P0 Frontend DB
~2h
No wizard do Smart Blocks (ManualStep2Conteudo), as tags são buscadas sem filtro de modelo — traz todas as tags da org. Corrigir para filtrar tags pelo modelo_id selecionado no Step 1.

Hierarquia correta: Tipo de petição → seleciona Modelo → Modelo tem Tags vinculadas → cada Tag é uma "pergunta" no wizard → cada Tag tem Blocos vinculados → Blocos são as opções de resposta → usuário seleciona 1 bloco por tag (ou múltiplos se multipla_selecao=true) → petição montada substituindo tags pelos blocos.

Verificar se a tabela olivia_tags tem FK para olivia_modelos_peticao ou se o vínculo é via olivia_flow. Ajustar query para usar o vínculo correto.
manual/ManualStep2Conteudo.tsx olivia_tags / olivia_flow / olivia_modelos_peticao

~1h
Blocos de texto devem ser exibidos como opções de resposta para cada tag. A query atual busca blocos com tag_ids mas não garante que só mostre blocos da tag atual sendo respondida.

Lógica esperada: Para cada tag (pergunta), exibir somente os blocos que têm aquela tag vinculada. O usuário seleciona 1 bloco (ou múltiplos). Ao finalizar, os blocos selecionados formam o corpo da petição na ordem das tags.
