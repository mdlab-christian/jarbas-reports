🔍 Root Causes Identificadas
ROOT CAUSE #1 · P0 CRÍTICO
Modelos duplicados — sem agrupamento por versão
O banco tem 328 registros em olivia_modelos_peticao, mas são ~109 modelos únicos cada um com versões 1, 2 e 3. O frontend está exibindo TODOS os 328 como itens separados no dropdown. Resultado: 72 "tags" no Smart Blocks (que na verdade são 24 tags × 3 versões).

Fix: Agrupar por modelo_grupo_id no dropdown. Após seleção, exibir seletor de versão. Filtrar tags/blocos apenas pela versão selecionada.
SELECT DISTINCT ON (modelo_grupo_id) * FROM olivia_modelos_peticao WHERE tipo_peticao_id = $1 ORDER BY modelo_grupo_id, versao DESC
ROOT CAUSE #2 · P0 CRÍTICO
processo_id não persiste entre steps do wizard
O estado do wizard perde o processo_id ao avançar para o Step 3 (Estrutura). O botão "Iniciar" retorna "processo_id obrigatório". O processo foi selecionado no Step 1, mas o estado não está sendo passado corretamente para a chamada de geração.

Fix: Verificar o hook useWizardTabs — garantir que processo_id está no estado persistido durante toda a sessão do wizard.
// useWizardTabs.ts — wizardState.processo_id deve estar disponível em todos os steps
ROOT CAUSE #3 · P0 CRÍTICO
Triagem — webhook retorna sem perguntas
O StepTriagem chama o webhook mdflow/olivia/triagem-get-questions via n8n. O workflow N8N-MODELOS-FLOW-VALIDATE está ativo, mas o fluxo de triagem não está retornando perguntas. Pode ser: (1) query SQL que busca as perguntas está quebrada, (2) o modelo selecionado não tem perguntas cadastradas, (3) o webhook está retornando [] silenciosamente.

Fix: Verificar o workflow n8n + popular perguntas de triagem para os modelos principais (Réplica, Apelação, Petição Simples).
ROOT CAUSE #4 · P0 CRÍTICO
VibeLaw — resposta em JSON cru
O VibeLawChat está recebendo a resposta da IA como um objeto JSON serializado e renderizando como string. O componente precisa parsear a resposta e extrair o campo de texto.

Fix: Em VibeLawChat.tsx — verificar o handler de resposta e garantir que o texto é extraído do JSON antes de renderizar.
// VibeLawChat.tsx — const text = typeof resp === 'string' ? resp : resp?.output ?? resp?.text ?? JSON.stringify(resp)
ROOT CAUSE #5 · P1
Estrutura de petição ignora tipo selecionado
Ao selecionar "Réplica", a estrutura gerada pela IA é de um Recurso (com seções "Do Cabimento e Tempestividade", "Do Mérito Recursal"). O prompt de geração não está incluindo o tipo de petição como contexto prioritário.

Fix: Injetar o tipo de petição no system prompt da geração. Criar estruturas base por tipo (Réplica, Apelação, Petição Simples, Contestação).

💡 Brainstorming — Ideias para o Futuro da OlivIA

Insights capturados dos vídeos + análise do contexto jurídico do escritório. Para aprovação e priorização.

🔍
Análise Comparativa de Documentos
IA compara automaticamente restrição vs cédula de crédito: datas, valores, número de contrato. Destaca divergências com highlight. A equipe jurídica faz isso manualmente hoje — é uma análise crítica para as réplicas contra Mercado Pago.
💰 Alto impacto
🧠
RAG Histórico por Cliente
Se o cliente já tem uma réplica contra a mesma empresa feita nos últimos N meses, a IA detecta, reutiliza os prints e argumentos já validados. Evita trabalho duplicado e melhora a consistência das peças.
🔬 Médio prazo
📚
Playbooks Contextuais Automáticos
Ao identificar tipo de petição + empresa (ex: Réplica + Mercado Pago), o sistema sugere automaticamente o playbook correspondente. Cada empresa tem suas peculiaridades (consumer credits, cartão, empréstimo pessoal).
💰 Alto impacto
⚖️
Identificação Automática de Tipo de Débito
Mercado Pago tem 3 modalidades: consumer credits (empréstimo para compras), cartão de crédito, empréstimo pessoal. A IA deve identificar qual é, pois determina quais argumentos usar e quais provas exigir da parte ré.
⚡ Fase 4
🖼️
Prints Integrados ao Texto
Quando a IA identifica prints relevantes nas provas, ela os insere no texto da petição com referências corretas ("verifica-se na imagem abaixo...", "conforme tela acostada..."). Nunca prints jogados no meio — sempre integrados narrativamente.
🎯 Diferencial
🏛️
Monitor de Gratuidade de Justiça
A IA verifica automaticamente se a AJG foi concedida no processo. Se não foi, sinaliza para incluir pedido de reiteiração na réplica. O escritório só atua com clientes que têm AJG — essa verificação é obrigatória.
💰 Alta prioridade
🔄
Loop de Qualidade Autônomo
Após geração: agente revisor avalia a petição com critérios jurídicos (completude, argumentação, estrutura). Score mínimo 8/10. Se não atingir, gera novamente com instrução de melhoria. Loop até aprovação ou limite de 3 tentativas.
🔬 Médio prazo
💡
Sugestões Contextuais no Campo "Instruções Adicionais"
O campo de instruções adicionais hoje é uma caixa em branco. A IA poderia sugerir pré-prontos baseados na triagem já feita: "Detectamos que há divergência de datas. Deseja adicionar instrução para abordar isso com mais ênfase?"
🎯 UX
⚖️ Contexto Jurídico — O que a IA Precisa Saber

Insights capturados do vídeo da equipe jurídica. Regras de negócio implícitas que a OlivIA deve conhecer.

📋 Checklist de Análise de Contestação (Mercado Pago)
  • →Identificar modalidade: consumer credits / cartão de crédito / empréstimo pessoal
  • →Verificar preliminares, especialmente litigância predatória (atuação abusiva)
  • →Comparar valor da restrição vs valor da cédula de crédito
  • →Comparar datas: data da restrição vs data de vencimento na cédula
  • →Verificar número de contrato: se ausente na cédula → argumento de impugnação
  • →Verificar se há comprovação da destinação do crédito (para consumer credits)
  • →Verificar gratuidade de justiça — se não concedida → reiterar na réplica
  • →Identificar prints juntados na contestação e verificar o que provam (ou deixam de provar)
📝 Regras de Formatação das Petições
  • ✓Prints integrados ao texto com referência: "na imagem acostada abaixo", "conforme tela seguinte"
  • ✓Palavras-chave em negrito — sinaliza melhor para o juiz
  • ✓Abordagem contundente em divergências de datas/valores — não amenizar
  • ✓Estrutura de réplica: qualificação → preliminares → origem do débito → mérito
  • ✗Nunca gerar estrutura de recurso quando o tipo for réplica
  • ✗Nunca usar prints "soltos" sem referência no texto
  • ✗Nunca gerar petição sem verificar tipo de petição no system prompt
