Spec Técnica
Nó DOCX-Upload — Implementação no n8n V3
Nó a adicionar após B8-Salvar (Express) e após D1 (Smart Blocks)
  • Receber texto_gerado (HTML) e historico_id + organizacao_id do nó anterior
  • Salvar HTML em arquivo temp: /tmp/{historico_id}.html
  • Executar: pandoc /tmp/{historico_id}.html -o /tmp/{historico_id}.docx --from=html --to=docx
  • Ler o DOCX gerado como buffer (base64)
  • Upload ao Supabase Storage: PUT /storage/v1/object/peticoes/{org_id}/{processo_id}/{historico_id}.docx com Content-Type: application/vnd.openxmlformats-officedocument.wordprocessingml.document
  • Obter URL pública: GET /storage/v1/object/sign/peticoes/{path}?expiresIn=86400
  • UPDATE: PATCH /rest/v1/olivia_historico?id=eq.{historico_id} → {pdf_url: url_assinada, pdf_path: path_bucket}
  • Cleanup: deletar /tmp/{historico_id}.html e /tmp/{historico_id}.docx
Path do Storage
  • Bucket: peticoes (já existe, public=false)
  • Path: {organizacao_id}/{processo_id}/{historico_id}.docx
  • URL de download: URL assinada (signed URL, expiresIn 24h) — para acesso privado por usuário autenticado
