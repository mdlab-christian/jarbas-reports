🤖 MODO 3 — Modo Agente (JARBAS Autônomo)

Rota: /olivia/gerador/modo-agente → OliviaGeradorJarbas.tsx

🟡 Frontend existe, backend parcial
OliviaGeradorJarbas.tsx usa os mesmos steps do Express, mas motor='jarbas'. A triagem usa jarbas_tarefas (olivia_gerar_triagem). A geração usa jarbas_tarefas (olivia_gerar_peticao). A skill olivia_gerar_peticao.mjs precisa estar funcional no JARBAS.

Arquitetura do Modo Agente

Frontend → jarbas_tarefas (olivia_gerar_peticao) → JARBAS skill
  → Busca modelo + conteudo_template no banco
  → Busca dados do processo (partes, ré, comarca)
  → Resolve placeholders ({{numero_processo}}, {{parte_autora}}, etc.)
  → Chama Claude API com system prompt jurídico especializado
  → Salva resultado em olivia_historico (texto_gerado + google_doc_url)
  → Notifica frontend via Supabase Realtime

Diferencial vs Express

AspectoExpress (n8n)Modo Agente (JARBAS)
Motorn8n workflow chainJARBAS skill autônoma
TriagemPerguntas fixas por modeloJARBAS analisa processo e gera perguntas contextuais
GeraçãoTemplate + substituiçãoIA jurídica com contexto completo do processo
RevisãoRevisor automático n8nJARBAS auto-revisa e itera até aprovação
Google DocVia script Apps ScriptVia Google Drive API (rclone token)
FallbackConcatena blocosJARBAS mantém contexto e retry automático

System Prompt base (Modo Agente)

Você é um advogado especialista em direito do consumidor brasileiro,
focado em ações de restrição de crédito indevida (negativação indevida).

Escritório: Perinotti Paz Advogados | Midas Advocacia
Estados: RS, SC, SP
Tipos de ação mais comuns: negativação indevida, dano moral, 
  cobrança indevida, protesto, cartão de crédito, empréstimos,
  bureaus (SPC, SERASA, Boa Vista)

Ao gerar petições:
1. Use o template fornecido como estrutura base
2. Substitua placeholders com dados reais do processo
3. Adapte argumentação ao tipo de réu e fundamento
4. Aplique jurisprudência do tribunal competente (TJRS/TJSC/TJSP)
5. Mantenha linguagem formal e técnica jurídica
