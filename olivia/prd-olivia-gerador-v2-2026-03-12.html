4 Modo Agente ⏳ Próximo

🤖

Modo Agente — JARBAS Direct

Geração avançada via skill no Mac Mini M4. Anti-fingerprint, seção por seção, RAG com jurisprudências, score de qualidade. O modo principal para casos complexos.

⏳ Implementar

Arquitetura

Endpoint
POST jarbas.jarbas-mdlab.com/olivia/gerar-modo-agente
Skill
skill_olivia_gerar_peticao.mjs (existente)
Modelo padrão
claude-sonnet-4-6 (streaming)
Motor salvo
jarbas_agente
Timeout
5 min (geração seção-a-seção)
Frontend ref
useOliviaJarbas hook

Diferencial vs Express

  • Anti-fingerprint: seed baseado no CNJ → 4 camadas de variação (conectivos, tratamentos, aberturas)
  • RAG: Busca jurisprudências relevantes no ChromaDB filtradas por UF do processo
  • Outline: Aceita estrutura de seções customizada (drag-and-drop no frontend)
  • Score: Revisor automático (score < 7 → regenera, máx 2x)
  • Progresso: Atualiza olivia_historico.progresso[] em tempo real
  • Google Doc: Cria doc via Drive API, salva google_doc_url

Fluxo de execução

1
Webhook n8n → JARBAS
POST mdflow/olivia/gerar-doc com modo=agente → n8n roteia para jarbas.jarbas-mdlab.com/olivia/gerar-modo-agente
2
JARBAS recebe + valida
Express server porta 3012 → enfileira skill_olivia_gerar_peticao
3
Buscar contexto completo
Supabase: processo + cliente + advogado + empresa + modelo_grupo + playbook + jurisprudências RAG
4
Montar outline
Se outline fornecido → usar • senão → gerar seções padrão por tipo_peticao
5
Gerar seção por seção
Claude streaming por seção → concatena HTML → atualiza progresso no Supabase em tempo real
6
Score + revisão
Revisor automático → score < 7 → regenerar seções fracas (máx 2x)
7
Google Doc (opcional)
Drive API → cria doc → salva google_doc_url no historico
8
Salvar SUCESSO
PATCH olivia_historico: status=SUCESSO, peticao_html, score_revisor, motor=jarbas_agente

Roteamento no n8n (a criar)

O webhook mdflow/olivia/gerar-doc precisa de um nó de roteamento que identifica o modo:

// Nó "Rotear Modo" — adicionar após Validar+Init
const modo = $json.data?.modo || 'express';
// modo='express' → continua fluxo atual (n8n_express)
// modo='agente' → POST jarbas.jarbas-mdlab.com/olivia/gerar-modo-agente
return [{ json: { ...$json, modo } }];
⚠️
Atenção na implementação: O endpoint JARBAS (jarbas.jarbas-mdlab.com) é o Mac Mini via Cloudflare Tunnel. O n8n no Railway faz POST para esse endpoint — isso é uma chamada externa. Timeout deve ser 120s mínimo no nó HTTP Request.

VibeLaw — Chat Studio Editor

Petição já gerada (pelo Express ou Agente) é refinada via chat interativo. Editor split-view: petição à esquerda, chat à direita. Patches aplicados seção a seção.

Conceito chave

💡
VibeLaw não gera do zero — refina. O usuário abre o studio com uma petição JÁ gerada (Express ou Agente). O chat permite: "deixa mais agressivo", "adiciona jurisprudência do STJ", "reformula o terceiro parágrafo". Patches são aplicados no HTML sem regenerar tudo.
