⚡ Riscos e Mitigações
CRÍTICO
Google Drive API sem acesso
Se os docs forem privados e sem service account configurada, não conseguimos extrair os textos. Os 2.926 documentos ficam como conteúdo vazio.
Mitigação: Testar ANTES de executar as fases. Se falhar, opção alternativa é tornar os docs públicos temporariamente, ou exportar manualmente como .docx e processar localmente.
ALTO
Mapeamento tipo_peticao antigo → novo
O antigo usava texto livre em tipo_peticao. O novo usa UUID FK. Um mapeamento errado vincula modelos ao tipo errado.
Mitigação: Criar tabela de mapeamento explícita (codigo antigo → id novo) antes de qualquer INSERT. Validar com Christian os 14 tipos.
ALTO
Docs deletados ou movidos no Drive
Alguns google_doc_ids podem não existir mais — docs deletados desde que foram cadastrados no antigo.
Mitigação: Logar todos os 404/403 durante a extração. Marcar como 'draft' + conteúdo placeholder. Gerar relatório de erros para revisão.
MÉDIO
227 blocos "extras" no antigo
O antigo tem 1.244 blocos, o novo tem 1.017. Houve 227 blocos deletados/não importados antes. Podem ser duplicatas ou versões obsoletas.
Mitigação: Importar TUDO do antigo (1.244). Deixar revisão de qualidade para depois.
MÉDIO
Rate limit Google Docs API
300 req/min por projeto. Com 2.926 docs, levaria ~10 min. Mas se houver erros e retries, pode estourar a quota.
Mitigação: Implementar exponential backoff, processar em lotes de 5 com delay de 300ms, checkpoint a cada 100 registros.
MÉDIO
historico_gerador referencia IDs antigos
Existe olivia_historico_gerador que pode ter FKs para IDs dos modelos/blocos antigos. Após reimportação, esses IDs mudam.
Mitigação: Verificar se vale preservar os IDs originais do antigo via INSERT com id explícito (manter os UUIDs antigos nos registros novos).
