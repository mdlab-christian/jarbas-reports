Gestão de Contexto — Estratégia para Execução
Uma conversa para /olivia + /playbooks
Mesmo mecanismo de dados (blocos, modelos, RAG, skills). Trabalhar junto é mais eficiente e evita decisões inconsistentes entre as páginas.
Âncoras de contexto para novas sessões
Quando a sessão sobrecarregar, usar /clear → JARBAS salva estado → nova sessão com:
1. SPEC /olivia (gerado Opus 4.6) — contrato definitivo do sistema
2. SPEC /playbooks (gerado Opus 4.6) — contrato definitivo
3. Prompt Universal (1 página):
   - IDs fixos: Supabase, N8N workflow IDs, org_id, test_user_id
   - Regras críticas: sem fallback triagem, nota_minima 9, zero placeholders
   - Estado atual: fases concluídas + próximo passo
   - Credenciais relevantes: GOOGLE_REFRESH_TOKEN, N8N_API_KEY
Alertas de contexto automáticos
JARBAS avisa quando sessão atingir ~80% do limite. Recomenda: compactar, continuar ou nova conversa. Em sessões de lapidação com HTML, alertar mais cedo (contexto consome mais).
