Backend — Diagnóstico e Ações
Causa Provável — Bug "Processo não encontrado"
1
Lookup de processo vinculado ao lead
A lógica de geração de documentos provavelmente busca um processo_id vinculado ao lead. No estágio "Em Atendimento", o processo ainda pode não ter sido criado/distribuído — por isso o erro "não encontrado".
2
Geração de documento usa dados do lead (não do processo)
Para os documentos iniciais (Honorários, Procuração), os dados são do cliente/lead — CPF, nome, endereço, parte reclamada, valor. Não deveria depender de processo criado.
3
Retorno da URL do Drive
A função de geração precisa retornar a file_url ou folder_url do Drive para o frontend usar no botão "Abrir".
Endpoints Afetados (estimativa)
POST
/api/crm/leads/{id}/documentos/gerar
Gera documento para um lead — precisa funcionar com dados do lead, sem depender de processo
GET
/api/crm/leads/{id}/documentos
Lista documentos gerados com URLs do Drive — deve retornar file_url
🤖
Automação (n8n) — Fluxo de Documentos
⚙️ A geração de documentos provavelmente passa por um workflow n8n que preenche os templates no Google Docs/Drive com os dados do cliente.
1
Trigger: Frontend chama endpoint de geração
MdFlow → POST para backend → dispara n8n webhook com dados do lead
2
n8n preenche template no Google Drive
Usa dados do lead (nome, CPF, endereço, parte, valor) → cria cópia do template → salva na pasta do cliente
3
n8n retorna URL do arquivo gerado
Deve retornar { file_url, folder_url } → backend salva no banco → frontend exibe botão "Abrir" ativo
4
⚠️ Ponto de falha suspeito
O n8n pode estar buscando um processo_id que não existe no estágio "Em Atendimento" → retorna erro → "processo não encontrado". Precisa ser corrigido para usar apenas dados do lead.
