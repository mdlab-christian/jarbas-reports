⚠️ Objetivo
Garantir que cada usuário veja na sidebar apenas as páginas liberadas para ele, e que o acesso via URL direto seja bloqueado por PageGuard se não tiver permissão. Sistema de permissões individual por página, do zero.

1 Diagnóstico — O que está errado hoje

🔴 Bug crítico — Mismatch de rota: /calculo vs /calculadora

Código usa /calculo mas banco tem /calculadora. Resultado: calculadora não aparece na sidebar de ninguém, mesmo quem tem permissão. E permissões do banco não casam com a rota do frontend.

🔴 PageGuard ausente em rotas protegidas

Qualquer usuário autenticado consegue acessar via URL direto: /atividades, /mdchat, /mdacademy, /usuario, /mdzap, /homepage. Sidebar filtra, mas a rota não bloqueia.

🟡 Permissões desatualizadas no banco

Vários usuários estão com permissões divergentes da spec (ex: Yuri com 12 páginas mas spec pede 12 diferentes, Carlos Daniel faltando /marketing e /processos). Solução: reset completo e aplicação do zero.

🟡 Heloísa com perfil errado

Heloísa está com perfil = 'juridico' no banco. Spec define ela como Admin (29 páginas). Precisa ser corrigido para 'admin'.

🔵 Tabelas de equipe não populadas

equipe_juridico, equipe_comercial e vendedores estão vazias. Setores dos usuários também não preenchidos. Comerciais precisam ser registrados como vendedores.

Tanto app_sidebar.tsx quanto mobile_sidebar_drawer.tsx já filtram por user.permissoes.includes(item.url). Após corrigir /calculo → /calculadora, a sidebar vai funcionar corretamente.
