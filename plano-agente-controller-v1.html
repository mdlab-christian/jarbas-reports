1. Visão Geral & Objetivos

O que é o Agente Controller?

Uma instância OpenClaw completamente separada rodando no Mac Mini M4, com um usuário macOS próprio, dedicada exclusivamente ao monitoramento, análise e gestão dos processos jurídicos do MP Advocacia.

Hoje essas responsabilidades estão fragmentadas entre JARBAS e JANA. O Controller centraliza tudo isso com foco total, sem interferência de outras tarefas.

✅ Responsabilidades

  • Monitorar ~5.800 processos
  • Capturar movimentações EPROC
  • Analisar intimações em lote
  • Alimentar página Controller MdFlow
  • Popular knowledge base Obsidian

⚡ Fora de escopo

  • Distribuição de processos (JARBAS)
  • Frontend/lapidação (JANA)
  • OlivIA / petições (OlivIA)
  • n8n workflow management
  • Chat pessoal / comandos gerais

🎯 KPIs de Sucesso

  • 3x por dia: levantamento
  • 100% movimentações capturadas
  • Análise em batch (não 1 a 1)
  • Obsidian atualizado após cada ciclo
  • Zero intervenção manual rotineira

⚠️ P0 — Confirmar antes de executar. Criação de usuário e configuração do OpenClaw são ações de sistema irreversíveis.
1
Criar usuário macOS controller
Usuário padrão (não admin), sem acesso Sudo, diretório home em /Users/controller. Isolamento total das credenciais do Christian.
2
Instalar OpenClaw no novo usuário
Instalar via npm como controller: npm install -g openclaw. Configurar ~/.openclaw/ com token Anthropic dedicado (sub-key ou key separada para billing isolado).
3
Criar bot Telegram dedicado
Via @BotFather: @Controller_mdbot. Canal dedicado ou grupo privado Controller. NÃO usar o mesmo bot do JARBAS.
4
Configurar .env do Controller
Copiar apenas as credenciais necessárias: ANTHROPIC_API_KEY, SUPABASE_ACCESS_TOKEN, SUPABASE_SERVICE_KEY, OABs (2FA), TELEGRAM_BOT_TOKEN.
5
Conectar no grupo Controller (-5156512750)
Adicionar o novo bot ao grupo Controller existente no Telegram. JARBAS permanece lá como backup/supervisor.
6
Configurar como LaunchAgent (autostart)
Criar ~/Library/LaunchAgents/ai.openclaw.controller.plist para subir automaticamente no login. Separado do JARBAS daemon.

5. Fase 4 — Arquitetura de Monitoramento (Pipeline em Subagentes)

💡 Cada ciclo diário é dividido em subagentes independentes. Isso evita timeout em lotes grandes e permite paralelismo por OAB/tribunal.

Pipeline de um Ciclo (3x por dia: 8h, 13h, 18h)

①
LEVANTAMENTO — Screenshot de todas as movimentações
Subagente paralelo por OAB × Tribunal
A
Login EPROC por OAB (5 OABs × 3 tribunais)
Usar Playwright com perfis Chrome isolados por OAB. 2FA via 2fa-manager. Cada OAB roda em subagente separado.
B
Capturar lista de movimentações desde último ciclo
Screenshot completo da lista + extrair texto via pdf-extract. Salvar em jarbas_tarefas com status pendente_analise.
C
Consolidar: quantos processos, por tribunal, por OAB
Gravar stats em Supabase. Notificar Controller group com resumo: "📊 Levantamento concluído: 47 movimentações capturadas (RS: 31, SC: 10, SP: 6)"
②
TRIAGEM — Leitura em conjunto de todos os prints
Único subagente, analisa o lote todo de uma vez
A
Carregar todos os prints do ciclo atual
Sonnet-4-6 lê todos os prints em uma única chamada longa. Classifica cada processo: 🔴 urgente 🟡 atenção 🟢 rotina
B
Planejar análises individuais necessárias
Listar quais processos precisam de acesso profundo ao EPROC (decisões, prazos, intimações). Salvar lista priorizada.
③
ANÁLISE INDIVIDUAL — Processos prioritários
Subagente por processo urgente/atenção
A
Acessar processo individualmente no EPROC
Ler decisão completa, extrair prazo, identificar necessidade de contestação/petição. Verificar teses em Obsidian.
B
Atualizar MdFlow + criar tarefa para OlivIA se necessário
Gravar análise em jarbas_tarefas. Se precisar de petição → criar item para OlivIA processar.
④
RELATÓRIO — Alimentar Controller MdFlow + Obsidian
Fechamento do ciclo
A
Atualizar página Controller no MdFlow
Stats do ciclo, lista de processos analisados, alertas pendentes.
B
Registrar insights no Obsidian
Padrões novos, empresas com táticas novas, argumentos bem-sucedidos. Atualizar insights-controller.md e padroes-empresas.md.

7. Fase 6 — Integração Obsidian ↔ Cérebro do Agente

Como o agente consome o Obsidian

  • Antes de qualquer análise: ler empresa-ré em 02-empresas/
  • Antes de classificar urgência: checar 03-teses/ e 05-tribunais/
  • Ao detectar padrão novo: escrever em 04-padroes/insights-controller.md
  • Ao finalizar ciclo: gravar log em 07-logs/YYYY-MM/
  • Ao receber áudio de instrução: transcrever e salvar em 08-audio-transcricoes/

Como o Obsidian alimenta as análises

  • SOUL.md aponta para o vault como fonte de verdade jurídica
  • MEMORY.md mantém referência para insights recentes
  • PLAYBOOKS.md no workspace = índice de links para vault
  • A cada nova empresa encontrada: criar nota automaticamente
  • A cada decisão favorável: atualizar decisoes-favoraveis.md

🔁 Loop de aprendizado contínuo

O Controller não é só executor — ele aprende. Após cada ciclo:

1
Detectar padrão → gravar em Obsidian
2
Próximo ciclo lê Obsidian → aplica conhecimento acumulado
3
Qualidade de análise melhora progressivamente
