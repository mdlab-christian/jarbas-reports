MISSÃO 1 — Login com Google: Root Cause Encontrado
Google OAuth não está configurado no Supabase
BUG P1
⚠️ Causa raiz confirmada: external_google_enabled: false e external_google_client_id: null no projeto Supabase qdivfairxhdihaqqypgb. O botão chama signInWithOAuth mas o Supabase rejeita porque o provider nunca foi habilitado.
O que precisa ser feito
1
Criar OAuth 2.0 Client no Google Cloud Console
Acesse console.cloud.google.com → APIs & Services → Credentials → Create OAuth 2.0 Client ID.
Tipo: Web application
Authorized redirect URIs: https://qdivfairxhdihaqqypgb.supabase.co/auth/v1/callback
Copiar: Client ID e Client Secret
2
Habilitar Google Provider no Supabase via Management API
JARBAS executa diretamente via PATCH /v1/projects/{id}/config/auth com as credenciais do Client ID e Secret. 
# JARBAS executa via Management API
PATCH https://api.supabase.com/v1/projects/qdivfairxhdihaqqypgb/config/auth
{
  "external_google_enabled": true,
  "external_google_client_id": "CLIENT_ID_AQUI",
  "external_google_secret": "CLIENT_SECRET_AQUI"
}
3
Adicionar redirect URL autorizada no Google Cloud
Após habilitar, o Supabase gera a callback URL. Confirmar que está no Google Console:
https://qdivfairxhdihaqqypgb.supabase.co/auth/v1/callback
4
Verificar o redirectTo no botão Google
O botão atual redireciona para /homepage após login. Confirmar que essa rota existe e tem guard de autenticação correto. O código está correto — nenhuma alteração necessária.
5
Teste E2E: clicar no botão e verificar fluxo completo
Browser headless abre /login → clica em "Continuar com Google" → verifica redirect para Google → confirma que volta para /homepage após auth.
