Playbook: Criar Novo Agente OpenClaw

Manual definitivo para criar agentes isolados baseados na arquitetura JARBAS — reutilizável quantas vezes for necessário.

Mac Mini M4 Mac Book Air M4 OpenClaw Gateway Telegram Bot Node.js + PM2 Supabase + n8n
💡

O Que É Este Playbook

🎯
Objetivo

Este documento é um manual reproduzível. Dado um novo bot Telegram + nome do agente, você segue os passos e cria um agente completo, isolado, com suas próprias skills, workspace OpenClaw, usuário macOS e PM2. Funciona no Mac Mini e no MacBook Air.

⚠️
Problema com JARBAS atual (o motivo desta mudança)

O JARBAS original virou um monolito: 102+ skills registradas no mesmo processo, skills jurídicas (OlivIA), skills de Controller e skills de Dev todas juntas. Resultado: contexto inflado (>20KB), heartbeats interferindo na fila, Claude semaphore bloqueando tudo. A nova abordagem divide por responsabilidade: cada agente é enxuto, focado e isolado.

📝
Como criar o bot no Telegram

1. Abra @BotFather no Telegram → /newbot → informe nome e username → copie o token.
2. Envie /setdescription, /setuserpic conforme necessário.
3. O token vai para o .env do novo usuário — nunca commitado no Git.

Script para copiar skills selecionadas do JARBAS para o novo agente
# Rodar como jarbas — copia skills selecionadas para /Users/beto/beto/src/skills/
DEST=/Users/beto/beto/src/skills

SKILLS=(
  skill_ping skill_status skill_memoria skill_modo_plano
  skill_git_digest skill_lapidar_pagina skill_supabase
  skill_monitor_n8n skill_gerar_prd skill_relatorio_html
  skill_claude_code skill_cc_orchestrator skill_pair_programmer
  skill_e2e_vision skill_documentador skill_custo_tracker
  skill_acesso_remoto skill_network_status skill_verificar_pagamentos
  skill_immune_system skill_download_documentos skill_voz_resposta
  skill_video_async skill_self_update skill_session_health
  skill_session_cleaner skill_contexto_monitor skill_sintetizar_aprendizado
  skill_checkpoint_projeto skill_projeto skill_projeto_delegado
  skill_handoff_prompt skill_mission_orchestrator skill_openclaw_agent
)

for s in "${SKILLS[@]}"; do
  f=$(ls /Users/jarbas/jarbas/src/skills/${s}.* 2>/dev/null | head -1)
  if [ -n "$f" ]; then
    sudo cp "$f" $DEST/
    echo "✅ $f"
  else
    echo "⚠️  Não encontrado: $s"
  fi
done

sudo chown -R beto:staff $DEST

💡
Regra dos 8KB

O workspace total dos arquivos injetados NÃO deve exceder 8KB. Workspace > 20KB causa flood de sendMessage no gateway (problema documentado no JARBAS). Manter MEMORY.md < 4KB.

Workspace OpenClaw — Arquivos a Criar

  • SOUL.md — Identidade do agente. Adaptar de ~jarbas/.openclaw/workspace/SOUL.md. Alterar nome, missão, Telegram handle. Remover referências a OlivIA/Controller. Manter < 2KB.
  • USER.md — Perfil do dono. Pode copiar igual de ~jarbas/.openclaw/workspace/USER.md.
  • MEMORY.md — Começa vazio/minimal. Manter < 4KB rigoroso. Sem histórico do JARBAS.
  • HEARTBEAT.md — Apenas 3 checks: custo acima de $200, tarefas presas >15min, gateway estável.
  • AGENTS.md — Regras de sessão. Adaptar do JARBAS. Remover grupos jurídicos.
  • TOOLS.md — Notas de ferramentas. Adaptar configurações SSH/browser para o novo agente.
