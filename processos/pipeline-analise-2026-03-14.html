🔴 Bug do download — o que provavelmente está acontecendo

Hipóteses prováveis

  • 1
    Sessão não está logada quando o download é disparado
    O download é enfileirado pelo importar_processo (que está logado), mas quando o task-consumer pega a task de download, ele abre um browser sem sessão. Precisa de relogin.
  • 2
    Seletor CSS #tblEventos desatualizado
    O EPROC pode ter mudado a estrutura HTML da tabela de eventos — o skill não encontra mais os links.
  • 3
    Processo sem documentos (só movimentações textuais)
    Processos novos podem ainda não ter peças anexadas. Mas a taxa de 100% "sem doc" é suspeita.
  • 4
    Payload sem processo_id
    A task de download tem só {cnj, oab, tribunal} — sem processo_id. O skill precisa fazer query extra no banco para relacionar o arquivo ao processo.

Correção recomendada antes de escalar

  • 1
    Testar manualmente 1 processo com logs detalhados
    --skill download_documentos --cnj 5000001-00.2024.8.21.0001
    Ver exatamente o que o browser vê na página
  • 2
    Garantir relogin no início da skill
    Usar o mesmo padrão da skill_importar_processo.mjs — verificar sessão e fazer login se necessário
  • 3
    Adicionar processo_id no payload do enfileiramento
    Salvar diretamente a FK para evitar query dupla
  • 4
    Screenshot na falha para debug visual
    Salvar print da página quando "nenhum doc encontrado" para validar o que o browser está vendo
