🚨 O que PRECISA ser resolvido antes de escalar
🔴 P0 — BLOQUEIA ESCALA
2.173 processos sem tribunal_id
A RPC fn_jarbas_buscar_para_analisar filtra por t.sigla = p_tribunal — processos com tribunal_id = NULL são invisíveis para análise IA.

Fix: Inferir tribunal pelo numero_cnj (código TJ é o 7º bloco: 8.21 = TJRS, 8.24 = TJSC, 8.26 = TJSP) e fazer UPDATE em batch.
🔴 P0 — BLOQUEIA ESCALA
ORPHAN_TIMEOUT_MIN = 10 muito agressivo para downloads
Um processo TJSC com 15 docs leva 8.5 minutos. Downloads TJRS estão morrendo por timeout (orphan=10min). Com 56 docs por processo, pode levar 30-40 min.

Fix: Aumentar ORPHAN_TIMEOUT_MIN para download_documentos para 45 min, ou criar timeout diferenciado por tipo de skill.
🔴 P0 — BLOQUEIA UTILIDADE
Análise IA não lê documentos do Storage
skill_analisar_ia_documentos navega ao EPROC e tenta baixar o PDF na hora. Não consome os PDFs que já estão no bucket.

Fix: Priorizar PDFs já salvos em processo_documentos antes de navegar ao EPROC. Economiza tempo e evita re-login.
🟡 P1 — DEGRADA ANÁLISE
Filtro de relevância não implementado
O download atual baixa todos os documentos do processo (56 por processo = muita coisa irrelevante como guias de custas, cartas de citação).

Fix: Filtrar por prioridade: SENT > DEC > ACORD > TUT > PET1. Baixar apenas os top 3-5 relevantes por processo.
🟡 P1 — DEGRADA ESCALA
processo_documentos não registrando
35 PDFs no bucket mas tabela processo_documentos com 0 registros. O insert está falhando silenciosamente.

Fix: Verificar schema da tabela, adicionar log detalhado no catch, confirmar colunas obrigatórias.
🟡 P1 — BLOQUEIA TJSP
TJSP com 8% de análise — bug ainda não identificado
eprocBaseUrl foi corrigido mas TJSP ainda pode estar falhando por seletor CSS ou estrutura de página diferente.

Fix: Testar 1 processo TJSP com logs visuais + screenshot para identificar onde trava.
🔵 P2 — QUALIDADE
Vision fallback não salva resultado no banco
Quando Claude Vision identifica documentos via screenshot, os links não são persistidos. Se o processo rodar novamente, repete o Vision.

Fix: Salvar links identificados por Vision em processo_documentos com fonte = 'vision'.
🔵 P2 — JURIMETRIA
Pipeline download → análise → jurimetria desconectado
Os 3 estágios rodam independentes. Quando download conclui, não dispara análise IA automaticamente.

Fix: Ao concluir download com documentos relevantes, enfileirar analisar_ia_documentos automaticamente para aquele processo.

🧱 Dificuldades Encontradas
🔴
EPROC muda estrutura de HTML por tribunal e versão
Mesmo seletor (#tblEventos) pode não existir em certas versões. TJSP usa estrutura diferente. Solução: seletores múltiplos + Vision fallback para quando nada bate.
🔴
Anti-bot do EPROC detecta automação agressiva
Delays de 3-8s por ação são obrigatórios. Sem isso, a sessão é bloqueada. Isso é o principal limitante de velocidade — não tem como acelerar muito.
🟡
Sessão EPROC expira durante processamento longo
Batches de 30+ CNJs podem expirar a sessão no meio. reloginFn foi implementado mas com custo de tempo adicional.
🟡
Downloads TJSP com links em iframes ou JavaScript
Alguns documentos abrem via onclick JavaScript ou em iframes — o seletor a[href] não captura esses. Vision fallback ajuda mas não é 100%.
