🗺️ As 3 rotas de execução

1
Análise das movimentações já no banco (zero custo de download)
Rota principal — funciona agora, sem Playwright
Como funciona: Para cada processo → pega as últimas N movimentações com texto (decisão, despacho, sentença) → manda para Claude Sonnet → extrai resultado, motivo, valor, fase → salva em ia_resumo, processos_decisoes.

Custo estimado: ~0.3–0.5s por processo · ~R$0,002/processo · 7.959 processos = ~R$16 total
Tempo: ~6–8h para todos os processos (rate limit API)
Implementação: Nova skill skill_analisar_movimentacoes.mjs — ~200 linhas
2
Extração ao vivo do EPROC (quando movimentação não tem texto suficiente)
Rota de enriquecimento — abre EPROC e copia texto da decisão
Como funciona: Para processos onde o texto da movimentação é curto (<200 chars) → abre EPROC → navega até a movimentação → extrai o HTML/texto completo → analisa com IA.

Vantagem sobre download: Não baixa PDF — só lê o HTML da página. Playwright mais estável nesse modo.
Já existe: o skill_analisar_ia_documentos.mjs faz isso. Só precisa passar texto extraído em vez de PDF.
3
Download de PDF (apenas quando necessário)
Rota premium — para contestações, provas, laudos
Quando usar: Provas específicas (CTPS, RG, comprovante renda), contestação da ré, laudos periciais. Não para decisões e despachos.
Status atual: Funcionando (fix aplicado hoje). Continua rodando em background.
