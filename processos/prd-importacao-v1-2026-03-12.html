🐛 02 — Bugs Críticos Encontrados
BUG-01 · Movimentações não gravadas no banco
P0 CRÍTICO
A tabela movimentacoes não tem constraint UNIQUE em external_id. A função saveMovimentacoesSupabase usa .upsert(rows, {onConflict: 'external_id'}), que falha silenciosamente retornando erro sem lançar exceção. Resultado: 0 movimentações salvas, embora o campo movimentacoes_total seja atualizado corretamente na tabela processos.
Fix: Criar migration ALTER TABLE movimentacoes ADD CONSTRAINT movimentacoes_external_id_unique UNIQUE (external_id). Alternativa sem migration: usar .insert(rows) com tratamento de erro duplicata ou buscar por processo_id + numero_evento no upsert.
BUG-02 · Download de PDFs retorna 0 documentos
P0 CRÍTICO
O skill skill_download_documentos_eproc.mjs busca links com o seletor #tblEventos a[href] filtrando por .pdf|.doc|.docx na URL. O EPROC atual usa URLs dinâmicas (ex: acessar_documento.php?id=XXX) sem extensão visível, ou os documentos estão em uma aba/iframe separado. Resultado: 100% dos downloads retornaram documentos: [].
Fix: Revisar a estrutura HTML do EPROC atual com um processo real. Atualizar o seletor para capturar links de documentos independente de extensão. Verificar se existe botão "Documentos" separado dos eventos. Adicionar log detalhado do HTML da página para debug.
BUG-03 · ia_status nunca vira 'concluido' (skill analise_juridica v2)
P0 CRÍTICO
O skill skill_analise_juridica_processo.mjs salva o JSON completo em ia_resumo mas não atualiza ia_status = 'concluido'. Usa a RPC fn_jarbas_analise_v2_upsert que pode estar com schema incorreto, ou a função só atualiza campos específicos. Resultado: todos os 50 processos ficam com ia_status='pendente' ou 'erro', nunca sendo removidos da fila de análise.
Fix: Verificar o corpo da RPC fn_jarbas_analise_v2_upsert e garantir que inclui ia_status = 'concluido'. Adicionar fallback direto: após salvar ia_resumo, fazer UPDATE processos SET ia_status='concluido' WHERE id = $1.
BUG-04 · fn_jarbas_buscar_para_analisar exclui processos com ia_status='erro'
P1 ALTO
A função de seleção de processos para análise filtra apenas ia_status='pendente'. Processos com ia_status='erro' (33 dos 50) nunca são re-analisados automaticamente, mesmo que o erro tenha sido transitório. Isso cria um "limbo permanente" para processos que falharam.
Fix: (1) Adicionar flag force=true no payload do task para re-processar erros. (2) Criar cron que reseta ia_status='pendente' para processos com ia_error='...' após 24h. (3) Atualizar a RPC para incluir ia_status IN ('pendente', 'erro') quando p_force=true.
BUG-05 · chance_sucesso_pct não é copiada do JSON para a coluna
P1 ALTO
O ia_resumo v2 contém chance_sucesso_pct: 75 mas a coluna processos.chance_sucesso_pct fica NULL. O frontend usa a coluna diretamente para filtros e dashboards. Mesmo problema para acordo_propensao_pct, polo_representado, fase_processual, acordo_candidato.
Fix: Após gravar ia_resumo, extrair e persistir campos individuais: UPDATE processos SET chance_sucesso_pct = (ia_resumo->>'chance_sucesso_pct')::int, acordo_candidato = ..., polo_representado = .... Criar função fn_sync_ia_fields(processo_id).
BUG-06 · Partes processuais não extraídas no enriquecimento
P1 ALTO
O skill de enriquecimento extrai vara/comarca/juiz mas retorna "0 partes extraídas" para quase todos os processos (log: WARN: 0 partes extraidas). O seletor de partes não está funcionando com a estrutura atual do EPROC. Campos reu_nome, advogado_reu_nome, advogado_reu_oab ficam vazios.
Fix: Debugar o seletor de partes com screenshot do EPROC. Atualizar para capturar partes da tabela de cabeçalho do processo. Mínimo necessário: polo ativo (cliente), polo passivo (empresa ré), advogado da ré.
BUG-07 · Skill supabase não executa SQL livre
P2 MÉDIO
A task do tipo 'supabase' só aceita queries pré-mapeadas (decisoes hoje, clientes hoje, etc.). O pipeline tentou usar para marcar ultima_analise_completa via SQL livre e falhou com "Query não mapeada". Workaround atual: Management API manual.
Fix: Adicionar handler para queries livres autorizadas (via lista de operações permitidas), ou criar task type dedicado marcar_analise_completa, ou usar diretamente a Management API no pipeline sem passar pela skill supabase.
BUG-08 · download_documentos na fila principal usa loginEproc não importado
P2 MÉDIO
O handler de download_documentos no start.mjs chama loginEproc(tarefa.payload?.oab) mas a função loginEproc não está no escopo correto (importada em outro bloco). Causa erro "loginEproc is not defined" quando a fila pega um task de download.
Fix: Mover o import de loginEproc para dentro do handler de download_documentos (inline import, como os outros handlers fazem).

Inconsistências de schema encontradas

Dois formatos de ia_resumo coexistindo:
• v1 (antigo): {resumo, resultado, analisado_em, data_decisao, tipo_decisao, modelo_analise} — usado pelos ~1.308 processos já analisados
• v2 (novo): {resumo, tipo_acao, fundamentos, recomendacao, pontos_fortes, pontos_atencao, chance_sucesso_pct, eventos_relevantes, ...} — muito mais rico
O frontend precisa suportar ambos ou fazer migração dos v1 para v2.
Processos órfãos (2.035 registros): Registros com origem_dados='mdflow_legacy' e tipo_acao='empresa' sem número CNJ. São processos de empresas (não de consumidores) importados do sistema legado. Não devem ser arquivados — precisam de tratamento especial ou separação em outra tabela.

Agentes propostos

🎯 Agente: Classificador de Polo
NOVO
Identifica automaticamente se o escritório representa autor ou réu, e qual parte é o cliente. Usa nome do cliente + partes extraídas do EPROC. Preenche polo_representado, cliente_nome verificado, reu_nome.
⏰ Agente: Detector de Prazos
NOVO
Varre as movimentações recentes e detecta prazos: citações, contestações, apelações, cumprimento de sentença. Cria registros na tabela prazos (ou envia notificação). Integrar com OlivIA para gerar peça antes do prazo.
📄 Agente: Analisador de Documentos
NOVO
Após download dos PDFs, faz análise específica por tipo: sentença (extrai resultado + valor), contestação (extrai argumentos da ré para preparar réplica), audiência (extrai data + pauta). Alimenta OlivIA com contexto rico.
📊 Agente: Monitor de Saúde do Pipeline
MELHORAR
Dashboard em tempo real do pipeline: quantos processos em cada estado, taxa de erros por skill, tempo médio de processamento, alertas quando alguma etapa trava. Substitui o monitoramento manual atual.

Tabela de erros proposta (nova)
