🎯 Estratégia de Match — CPF + Empresa

A chave de cruzamento entre os dois sistemas é a combinação CPF do cliente + empresa ré. Um processo no antigo é o mesmo que no novo se o CPF do cliente bate E a empresa bate (normalizada).

Campo no Antigo Campo no Novo Cobertura Confiança
clientes.cpf clientes.cpf_cnpj 182/191 (95%) Alta — CPF é único
processos.contrato_restricao processos.restricao_contrato ~68% Alta quando presente — código único do contrato
processos.valor_restricao (text) processos.restricao_valor (numeric) ~68% Média — converter texto→decimal p/ comparar
processos.data_restricao (date) processos.restricao_data (date) estimado ~60% Alta quando presente — data exata da negativação
processos.empresa_re processos.empresa_nome 146/191 (76%) Média — nomes diferem entre sistemas (normalizar)

💡 Estratégia em camadas (prioridade):
1️⃣ CPF + contrato → match quase certo
2️⃣ CPF + data_restricao + valor → confirma restrição específica
3️⃣ CPF + empresa (normalizada) → fallback
Quanto mais camadas batem, mais pontos de confiança. Só IDs são proibidos como chave — todos os outros campos da restrição são válidos.

### PASSO 1 — Extrair distribuídos do antigo com CPF

Projeto: `hzqoxkmxvwxejjanstdh`

```sql
SELECT 
  p.id AS proc_id_antigo,
  p.numero_processo,
  p.empresa_re,
  p.data_distribuicao,
  p.valor_restricao,
  p.contrato_restricao,
  p.data_restricao,
  p.estado,
  p.local_distribuicao,
  p.tipo_acao,
  REGEXP_REPLACE(c.cpf, '[^0-9]', '', 'g') AS cpf_limpo,
  c.name AS cliente_nome
FROM processos p
LEFT JOIN clientes c ON p.id_cliente = c.id
WHERE p.distribuido = true
  AND p.data_distribuicao >= '2026-03-16'
ORDER BY p.data_distribuicao;
```

Guardar resultado como `processos_antigo` (array de objetos).

### PASSO 2 — Para cada processo do antigo, buscar no novo

Projeto: `qdivfairxhdihaqqypgb`

Para cada item em `processos_antigo` com CPF:

```sql
SELECT 
  p.id AS proc_id_novo,
  p.numero_processo,
  p.empresa_nome,
  p.etapa_distribuicao,
  p.distribuido,
  p.restricao_valor,
  p.restricao_contrato,
  p.restricao_data,
  REGEXP_REPLACE(c.cpf_cnpj, '[^0-9]', '', 'g') AS cpf_limpo
FROM processos p
JOIN clientes c ON p.cliente_id = c.id
WHERE REGEXP_REPLACE(c.cpf_cnpj, '[^0-9]', '', 'g') = '{cpf_limpo}'
  AND p.ativo = true
  AND p.deleted_at IS NULL
ORDER BY p.created_at;
```

#### Sistema de pontuação por candidato (score de confiança):
Para cada candidato encontrado no novo, calcular um score somando pontos:

| Critério | Pontos |
|---|---|
| Contrato bate exato | +40 |
| Data restrição bate exato | +30 |
| Valor bate (±R$0,01 após conversão texto→decimal) | +20 |
| Empresa normalizada bate (primeiras 3 palavras) | +10 |

**Classificação final:**
- **0 candidatos** → GRUPO C (não encontrado)
- **1 candidato com score ≥ 40** → GRUPO A (match confiável)
- **1 candidato com score < 40** → GRUPO B (ambíguo — confirmar manualmente)
- **2+ candidatos** → pegar o de maior score; se score_1 - score_2 ≥ 30 → GRUPO A; caso contrário → GRUPO B

#### Normalização do nome de empresa:
Remover pontuação (`.,/-`), remover stopwords (`S.A.`, `SA`, `LTDA`, `ME`), deixar maiúsculas, comparar primeiras 3 palavras.
Exemplos de equivalência conhecida:
- "BOA VISTA SERVIÇOS S.A." ≈ "BOA VISTA SERVICOS S.A."
- "NU PAGAMENTOS SA" ≈ "NU PAGAMENTOS S.A. - INSTITUICAO DE PAGAMENTO"
- "BANCO BRADESCO SA" ≈ "BANCO BRADESCO S.A."

#### Conversão valor (antigo text → decimal):
Substituir `.` (milhar) por nada, `,` por `.`. Ex: `"1.250,00"` → `1250.00`. Comparar com `restricao_valor::numeric`.
