### ESTRATÉGIA DE MATCH (em ordem de prioridade)

**Opção A — Match por numero_processo (mais confiável):**
```sql
-- No NOVO banco (qdivfairxhdihaqqypgb)
SELECT p.id, p.numero_processo, p.distribuido, p.etapa_distribuicao,
       REGEXP_REPLACE(c.cpf_cnpj, '[^0-9]', '', 'g') AS cpf_limpo
FROM processos p
JOIN clientes c ON p.cliente_id = c.id
WHERE p.numero_processo = '{numero_processo_do_antigo}'
  AND p.ativo = true
  AND p.deleted_at IS NULL;
```
→ Se encontrar 1 resultado: verificar CPF bate → **match confirmado → GRUPO A**
→ Se 0 resultados: tentar Opção B

**Opção B — Match por CPF + janela de data_distribuicao (±3 dias):**
```sql
SELECT p.id, p.numero_processo, p.distribuido, p.etapa_distribuicao,
       p.data_distribuicao,
       REGEXP_REPLACE(c.cpf_cnpj, '[^0-9]', '', 'g') AS cpf_limpo
FROM processos p
JOIN clientes c ON p.cliente_id = c.id
WHERE REGEXP_REPLACE(c.cpf_cnpj, '[^0-9]', '', 'g') = '{cpf_limpo}'
  AND p.ativo = true
  AND p.deleted_at IS NULL
  AND p.distribuido = false
ORDER BY p.created_at;
```
→ Se 1 resultado com CPF batendo: **match provável → GRUPO A**
→ Se 2+ resultados: **GRUPO B (ambíguo)**
→ Se 0: **GRUPO C (não encontrado)**
