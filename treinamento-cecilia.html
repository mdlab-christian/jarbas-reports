Visão Geral — O que é o Controller
Entenda o propósito antes dos detalhes técnicos
O que é o Controller?

O Controller é o painel central do MdFlow onde o João (e o escritório) gerencia as intimações judiciais capturadas automaticamente do EPROC. É a interface principal de trabalho do dia a dia — tudo que chega pelo tribunal aparece aqui para ser analisado, classificado e distribuído para os advogados.

O que o JARBAS fazia aqui?
  • 🔐 Fazer login no EPROC (RS, SC, SP)
  • 🔍 Navegar processo a processo capturando movimentações
  • 📥 Gravar intimações novas no banco Supabase
  • 🤖 Acionar a IA (OlivIA) para analisar cada intimação
  • ✅ Atualizar status → Controller frontend reflete em tempo real
  • 🎤 Transcrever áudios recebidos (Whisper)
  • 📄 Gerar HTMLs de relatório → publicar no GitHub Pages
O que você vai assumir?
  • 🎯 Receber comandos do João via Telegram/Controller
  • 🏃 Acionar capturas de intimações quando solicitado
  • 🔄 Importar lotes de intimações para teste
  • 📊 Gerar relatórios HTML e enviar o link
  • 🧹 Limpar intimações inválidas/fantasmas
  • 👀 Monitorar status geral do pipeline
  • 🎤 Transcrever áudios quando pedido
💡
Regra de ouro: O Controller frontend (React/Supabase) é passivo — ele apenas exibe o que está no banco. Quem alimenta o banco somos nós (JARBAS/você) via EPROC + IA. Então tudo que você fizer aqui reflete diretamente na tela do João.

EPROC — Acesso processo a processo
Como navegar o portal judicial e capturar movimentações
O que é o EPROC e por que acessamos?

O EPROC é o sistema eletrônico de processo judicial usado pelos tribunais estaduais. Ele exibe todas as movimentações (despachos, decisões, intimações) de cada processo. Nós acessamos ele automaticamente para capturar essas movimentações e alimentar o Controller.

🔐 Login e autenticação

  • Acessar a URL do tribunal correspondente à OAB desejada (RS, SC ou SP)

  • Fazer login com CPF e senha da OAB — credenciais ficam em ~/jarbas/.env

  • O sistema retorna HTTP 302 redirect SSO quando bem-sucedido — isso é normal!

  • Manter a sessão ativa: o EPROC expira a sessão após inatividade, então verificamos sempre antes de navegar

⚠️
Nunca tente logar em vários tribunais ao mesmo tempo com a mesma OAB — pode invalidar a sessão e bloquear o acesso.
🔍 Navegação processo a processo

O JARBAS usa o Playwright (browser automatizado) para navegar. O fluxo de captura é assim:

  • Buscar lista de processos ativos no banco Supabase (processos com status='ativo')

  • Para cada processo: acessar pelo número CNJ no EPROC

  • Extrair o último evento da página — verificar se é uma intimação/despacho relevante

  • Usar a lista de keywords para decidir se captura ou ignora:
    ✅ Captura: "INTIMAÇÃO", "CITAÇÃO", "DESPACHO", "DECISÃO", "SENTENÇA"
    ❌ Ignora: "CONCLUSOS PARA DESPACHO", "JUNTADA DE PETIÇÃO", "CERTIDÃO"

  • Extrair o texto completo do documento de intimação (PDF ou texto)

  • Calcular signature_hash para evitar duplicatas

  • Gravar na tabela intimacoes com status='nova' e ia_status='pendente'

🤖 O que a OlivIA analisa em cada intimação?

CAMPOS GERADOS PELA IA

  • ia_classificacao — tipo da intimação
  • ia_risco_nivel — crítico / alto / médio / baixo
  • ia_acao_recomendada — o que fazer
  • ia_prazo_sugerido — data limite sugerida
  • resumo_ia — texto resumido em pt-BR
  • urgente — boolean (true se prazo < 48h)

🧹 Limpeza de intimações fantasmas

Se o Controller mostrar badge de "não lidas" mas não aparecer nenhuma intimação na tela, há intimações fantasmas no banco. Para limpar:

  1. Peça ao JARBAS: "limpa as intimações não lidas"
  2. O JARBAS vai verificar o conteúdo antes de apagar — intimações com título "Conclusos para despacho" são candidatas à limpeza
  3. Serão arquivadas (não deletadas) com observação explicando o motivo

🔄 Por que o frontend atualiza automaticamente?

O MdFlow usa Supabase Realtime — WebSocket que envia updates instantâneos para o browser. Quando você (ou o pipeline IA) atualiza um registro no banco, o João vê a mudança em menos de 1 segundo, sem precisar recarregar a página.

Whisper — Transcrição de Áudio
Converta áudios do Telegram em texto com IA local
O que é o Whisper?

O OpenAI Whisper é um modelo de transcrição de voz que roda localmente no Mac Mini — sem precisar de API key, sem custo extra, e com suporte nativo a português. É usado para transcrever áudios que o João (ou o Christian) envia pelo Telegram.

Como usar o Whisper

  • Receba um áudio via Telegram (o arquivo fica em /Users/jarbas/.openclaw/media/inbound/)

  • Execute o Whisper no arquivo:

$ whisper /caminho/arquivo.ogg --model turbo --output_format txt --output_dir /tmp/

  • O Whisper detecta o idioma automaticamente (usa --language pt para forçar português)

  • A transcrição aparece no terminal e também salva em .txt no diretório de saída

  • Você envia o texto transcrito para o João via Telegram

💡
Modelos disponíveis: tiny (rápido) → base → small → medium → large → turbo (padrão — melhor custo-benefício). Use large para áudios com sotaque ou qualidade ruim.
Formatos suportados

.ogg (Telegram) · .mp3 · .m4a · .wav · .mp4 · .webm

Geração de HTMLs — GitHub Pages
Como criar e publicar relatórios visuais
Por que usar GitHub Pages?

Em vez de enviar código HTML bruto no Telegram (que ficaria ilegível), publicamos em GitHub Pages — um link público estável que o João pode abrir no browser. O repositório jarbas-reports é dedicado para isso.

🚀 Como publicar um HTML

  • Escreva o HTML em ~/jarbas-reports/[nome-do-relatorio].html

  • Commit e push:

# No terminal do Mac Mini:
cd ~/jarbas-reports
git add [nome].html
git commit -m "Add: [nome do relatório]"
git push

  • O link fica disponível em segundos:

🔗
Formato do link:
https://mdlab-christian.github.io/jarbas-reports/[nome].html

  • Envie apenas o link para o João — nunca o HTML completo no chat

  • Acento secundário: #00d4aa (teal)
  • Fonte: system-ui / Segoe UI

📊 Queries úteis
-- Ver intimações por status
SELECT status, count(*) FROM intimacoes GROUP BY status;

-- Ver últimas intimações novas
SELECT titulo, status, ia_status, created_at
FROM intimacoes
WHERE status = 'nova'
ORDER BY created_at DESC LIMIT 10;

-- Ver intimações com erro na IA
SELECT id, titulo, ia_error
FROM intimacoes
WHERE ia_status = 'erro'
ORDER BY created_at DESC;

-- Ver view completa do Controller
SELECT titulo, processo_cnj, cliente_nome, status_prazo
FROM v_controller_intimacoes_completa
WHERE status = 'pendente'
ORDER BY dias_ate_prazo ASC NULLS LAST LIMIT 20;
